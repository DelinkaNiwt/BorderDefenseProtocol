# 基因系统结构与继承体系

**总览**：基因系统是Biotech DLC的核心系统，为Pawn提供可遗传、可组装、可植入的属性修改框架。系统由三层架构组成：`GeneDef`（定义层，描述基因的所有属性和效果）、`Gene`（实例层，附加在Pawn上的运行时对象）、`Pawn_GeneTracker`（管理层，维护Pawn的全部基因并提供聚合查询）。

## 三层架构

| 层级 | 核心类 | 命名空间 | 职责 | 行数 |
|------|--------|---------|------|------|
| **定义层** | `GeneDef : Def` | Verse | 描述基因的80+字段（效果、外观、代谢、能力等） | 807 |
| **实例层** | `Gene : IExposable` | Verse | 附加在Pawn上的运行时基因对象，管理生命周期 | 168 |
| **管理层** | `Pawn_GeneTracker : IExposable` | RimWorld | 维护xenogenes/endogenes双列表，提供聚合属性查询 | 1402 |

## Xenogene vs Endogene 双轨机制

Pawn_GeneTracker内部维护两个独立的`List<Gene>`：

| 维度 | Xenogene（异种基因） | Endogene（内源基因） |
|------|---------------------|---------------------|
| **存储** | `Pawn_GeneTracker.xenogenes` | `Pawn_GeneTracker.endogenes` |
| **来源** | Xenogerm植入、基因组装器 | 出生时继承、种族默认 |
| **可遗传** | 否（除非XenotypeDef.inheritable=true） | 是（通过PregnancyUtility.GetInheritedGenes） |
| **覆盖规则** | Xenogene优先覆盖同exclusionTag的Endogene | 被Xenogene覆盖时标记为Overridden |
| **移除方式** | Xenogerm植入时清除全部旧Xenogene | 通常不可移除（代码可操作） |

> **关键机制**：`Gene.Overridden`属性——当一个基因被另一个基因覆盖时（通过`exclusionTags`匹配），`overriddenByGene`字段指向覆盖者，`Active`返回false，效果不生效但基因仍存在于列表中。

## Gene基类生命周期方法

| 方法 | 触发时机 | 典型用途 |
|------|---------|---------|
| `PostMake()` | Gene对象创建后 | 初始化自定义字段 |
| `PostAdd()` | 基因添加到Pawn后 | 刷新图形、应用初始效果 |
| `PostRemove()` | 基因从Pawn移除后 | 清理效果、刷新图形 |
| `Tick()` | 每tick调用（如果重写） | 高频更新逻辑 |
| `TickInterval(int delta)` | 间隔调用 | 低频更新（如mentalBreakMtbDays检查） |
| `Notify_IngestedThing(Thing, int)` | Pawn进食后 | 资源型基因的补充（如Hemogen） |
| `GetGizmos()` | UI渲染时 | 提供Gizmo按钮（如资源条） |
| `OverrideBy(Gene)` | 基因覆盖时 | 设置overriddenByGene |

## Gene子类完整继承树（11个子类，源码验证）

```
Gene (根基类, Verse) ─ IExposable, ILoadReferenceable
├── Gene_Resource (抽象, RimWorld) ─ 资源型基因基类
│   └── Gene_Hemogen (RimWorld) ─ 血源素，IGeneResourceDrain
├── Gene_Deathrest (RimWorld) ─ 死眠系统管理
├── Gene_HemogenDrain (RimWorld) ─ 血源素消耗（非资源型）
├── Gene_Bloodfeeder (Verse) ─ 吸血能力
├── Gene_ChemicalDependency (Verse) ─ 化学品依赖
├── Gene_Clotting (Verse) ─ 凝血/止血
├── Gene_Deathless (Verse) ─ 不死基因
├── Gene_Healing (Verse) ─ 自愈基因
├── Gene_PollutionRush (Verse) ─ 污染兴奋
└── Gene_PsychicBonding (Verse) ─ 心灵连接
```

> **开发者关键认知**：绝大多数基因不需要自定义Gene子类——它们直接使用基类`Gene`，通过GeneDef的80+字段配置效果。只有需要特殊运行时逻辑（如资源追踪、周期性行为）的基因才需要继承Gene写新子类。

## XenotypeDef（异种类型定义）

XenotypeDef定义一组预设基因的集合（如Sanguophage吸血鬼、Hussar骠骑兵），是"种族模板"。

| 字段 | 类型 | 说明 |
|------|------|------|
| `genes` | `List<GeneDef>` | 该异种包含的所有基因 |
| `inheritable` | `bool` | 基因是否可遗传给后代 |
| `combatPowerFactor` | `float` | 战斗力倍率（影响袭击点数计算） |
| `canGenerateAsCombatant` | `bool` | 是否可作为战斗员生成 |
| `factionlessGenerationWeight` | `float` | 无派系生成权重 |
| `nameMaker` / `nameMakerFemale` | `RulePackDef` | 异种专属命名规则 |
| `forbiddenWeaponClasses` | `List<WeaponClassDef>` | 禁止使用的武器类别 |
| `doubleXenotypeChances` | `List<XenotypeChance>` | 双异种概率（混血） |

> `Archite`属性：如果genes列表中任何基因属于`GeneCategoryDefOf.Archite`分类，则该异种被标记为Archite异种。

## Genepack与Xenogerm容器体系

```
GeneSetHolderBase (抽象基类, ThingWithComps)
├── Genepack ─ 基因包，存储1~4个基因，放入基因库保存
└── Xenogerm ─ 异种胚芽，组装完成的完整基因方案，可植入Pawn
```

| 容器 | 类 | 核心字段 | 用途 |
|------|-----|---------|------|
| **GeneSet** | `RimWorld.GeneSet` | `List<GeneDef> genes` | 基因集合的数据容器（非Thing） |
| **Genepack** | `RimWorld.Genepack` | `geneSet`, `targetContainer`, `autoLoad` | 物理基因包物品，可存入基因库 |
| **Xenogerm** | `RimWorld.Xenogerm` | `geneSet`, `xenotypeName`, `iconDef` | 组装完成的异种胚芽，植入后替换Pawn的全部Xenogene |

**Genepack生命周期**：
1. 基因提取器（Building_GeneExtractor）从Pawn提取 → 生成Genepack
2. Genepack存入基因库（CompGenepackContainer）→ 通电时修复HP，断电时劣化
3. 基因组装器（Building_GeneAssembler）选择多个Genepack → 组装为Xenogerm
4. Xenogerm植入Pawn → 替换全部Xenogene

| 方法/属性 | 说明 |
|----------|------|
| `GenesListForReading` | 返回xenogenes + endogenes的合并列表（缓存） |
| `Xenogenes` / `Endogenes` | 分别访问两个列表 |
| `Xenotype` | 当前XenotypeDef（默认Baseliner） |

| `SetXenotype(XenotypeDef)` | 设置异种类型 |
| `HasGene(GeneDef)` | 检查是否拥有某基因 |
| `GetGene(GeneDef)` | 获取指定GeneDef的Gene实例 |
| `PainOffset` / `PainFactor` | 聚合所有活跃基因的疼痛修改 |

## 开发者要点

1. **三层分离**：GeneDef负责"是什么"，Gene负责"怎么运行"，Pawn_GeneTracker负责"怎么管理"。模组新增基因通常只需定义GeneDef XML
2. **Xenogene覆盖Endogene**：通过`exclusionTags`机制，同标签的Xenogene自动覆盖Endogene，被覆盖的基因`Active=false`但不被删除
3. **Gene_Resource是资源型基因的标准模式**：需要追踪数值资源的基因应继承Gene_Resource，获得内置的cur/max/targetValue + GeneGizmo_Resource UI
4. **Genepack→Xenogerm→植入是标准流程**：模组如需自定义基因获取途径，可直接调用`Pawn_GeneTracker.AddGene()`绕过物理流程
5. **Gene.Active的三重检查**：未被覆盖 + 年龄≥minAgeActive + 未被MutantDef禁用，三个条件全部满足才激活
