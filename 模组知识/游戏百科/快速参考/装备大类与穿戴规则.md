# 装备大类与穿戴规则

**总览**：RimWorld装备分为三大类——**武器**（`Pawn_EquipmentTracker`管理，仅1把主武器）、**服装**（`Pawn_ApparelTracker`管理，多件叠穿）、**腰带工具**（Belt层，归属服装系统但不与其他层冲突）。服装穿戴受**层级**（ApparelLayerDef）和**身体部位组**（BodyPartGroupDef）双重约束，冲突判定由`ApparelUtility.CanWearTogether()`执行。

## 装备三大类对比

| 大类 | 管理器 | 存储结构 | 数量限制 | 核心特征 |
|------|--------|---------|---------|---------|
| **武器** | `Pawn_EquipmentTracker`（Verse） | `ThingOwner<ThingWithComps>` | 仅1把Primary | 近战/远程二分，`EquipmentType.Primary` |
| **服装** | `Pawn_ApparelTracker`（RimWorld） | `ThingOwner<Apparel>` | 多件叠穿 | 层级+部位组冲突规则 |
| **腰带工具** | 同服装（Belt层） | 同服装 | 多件 | 特殊层级，不与其他层冲突 |

### 武器系统要点

- **存储**：`ThingOwner<ThingWithComps>`，通过`Primary`属性访问主武器
- **限制**：同一时间仅1把Primary武器（`EquipmentType.Primary`）
- **绑定武器**：`bondedWeapon`字段支持心灵武器绑定（Royalty DLC）
- **Verb系统**：武器通过`CompEquippable`提供`Verb`（攻击动作）

### 服装系统要点

- **存储**：`ThingOwner<Apparel>`，通过`WornApparel`列表访问
- **锁定系统**：`lockedApparel`列表，锁定的服装不可手动脱下（如奴隶项圈）
- **裸体判定**：`AnyClothing`属性检查`countsAsClothingForNudity`字段
- **需求系统**：支持Royalty头衔和Ideology角色的服装需求（`ApparelRequirement`）

## 6层服装层级系统（ApparelLayerDef）

| # | 层级defName | drawOrder | label | 说明 | 典型实例 |
|---|------------|-----------|-------|------|---------|
| 1 | **OnSkin** | 0 | skin | 贴身层，最内层 | T恤、裤子、内衣 |
| 2 | **Middle** | 100 | middle | 中间层 | 衬衫、防弹衣、连衣裙 |
| 3 | **Shell** | 200 | outer | 外层，最外层衣物 | 夹克、板甲、大衣、防尘外套 |
| 4 | **Belt** | 300 | utility | 腰带/工具层 | 护盾腰带、弹药带、工具腰带 |
| 5 | **Overhead** | 400 | headgear | 头部覆盖层 | 头盔、帽子、兜帽 |
| 6 | **EyeCover** | 500 | eyes | 眼部覆盖层 | 眼罩、蒙眼布 |

**关键机制**：
- `drawOrder`决定渲染顺序（数值越大越后渲染，即越外层）
- 一件服装可以属于**多个层级**（如某些全身甲同时占OnSkin+Shell）
- Belt层设计为不与其他层冲突——因为Belt层服装的`bodyPartGroups`通常只有`Waist`
- `LastLayer`属性返回服装最外层的层级（用于渲染和AI评估）

## 人类身体部位组系统（BodyPartGroupDef）

服装通过`bodyPartGroups`字段声明覆盖哪些身体部位组。以下是人类相关的部位组：

| # | defName | label | listOrder | 典型服装覆盖 |
|---|---------|-------|-----------|-------------|
| 1 | **UpperHead** | upper head | 200 | 帽子、头盔 |
| 2 | **FullHead** | full head | 199 | 全脸头盔、面罩 |
| 3 | **Eyes** | eyes | 198 | 眼罩、蒙眼布 |
| 4 | **Teeth** | teeth | 197 | — |
| 5 | **Mouth** | mouth | 196 | — |
| 6 | **Neck** | neck | 180 | 围巾、项圈 |
| 7 | **Torso** | torso | 100 | 大多数上身衣物 |
| 8 | **Arms** | arms | 90 | 长袖衣物 |
| 9 | **Shoulders** | shoulders | 85 | 肩甲、披风 |
| 10 | **Hands** | hands | 80 | 手套 |
| 11 | **Waist** | utility slot | 50 | 腰带工具 |
| 12 | **Legs** | legs | 10 | 裤子、腿甲 |
| 13 | **Feet** | feet | 9 | 靴子 |

**关键机制**：
- `listOrder`决定在UI中的显示顺序（数值越大越靠前）
- 服装的`bodyPartGroups`是一个列表，一件衣服可覆盖多个部位组（如板甲覆盖Torso+Shoulders+Arms+Legs）
- 身体部位（`BodyPartRecord`）可属于多个部位组（如头部同时属于UpperHead和FullHead）

## 穿戴冲突规则

### 核心判定：`ApparelUtility.CanWearTogether()`

```
输入：ThingDef A, ThingDef B, BodyDef body
输出：bool（能否同时穿戴）

Step 1: 检查两件服装是否有共同层级
  → 遍历 A.apparel.layers × B.apparel.layers
  → 无共同层级 → return true ✅（可以同时穿戴）
  → 有共同层级 → 进入Step 2

Step 2: 计算干涉部位组
  → A的干涉组 = GetInterferingBodyPartGroups(A, body)
  → B的干涉组 = GetInterferingBodyPartGroups(B, body)

Step 3: 交叉检查（双向）
  → A.bodyPartGroups ∩ B的干涉组 ≠ ∅ → return false ❌（冲突）
  → B.bodyPartGroups ∩ A的干涉组 ≠ ∅ → return false ❌（冲突）
  → 否则 → return true ✅（可以同时穿戴）
```

### 干涉部位组计算：`GetInterferingBodyPartGroups()`

```csharp
// 逻辑：找到服装覆盖的所有身体部位 → 收集这些部位的所有BodyPartGroupDef
body.AllParts
  .Where(part => part.groups中有任何一个在服装的bodyPartGroups中)
  .SelectMany(part => part.groups)  // 展开所有部位组
  .Distinct()                        // 去重
  .Where(group => apparelRelevantGroups中包含该group)  // 只保留服装相关的组
```

**原因**：一个身体部位可能属于多个部位组。例如头部同时属于UpperHead和FullHead，所以戴了覆盖UpperHead的帽子后，其干涉组也包含FullHead，从而阻止同时戴全脸头盔。

### 穿戴冲突实例

| 场景 | 服装A | 服装B | 共同层级？ | 干涉检查 | 结果 |
|------|-------|-------|-----------|---------|------|
| T恤+裤子 | OnSkin/Torso | OnSkin/Legs | OnSkin ✓ | Torso干涉组∩Legs=∅ | ✅可穿 |
| 两件夹克 | Shell/Torso | Shell/Torso | Shell ✓ | Torso∩Torso干涉组≠∅ | ❌冲突 |
| 帽子+面罩 | Overhead/UpperHead | Overhead/FullHead | Overhead ✓ | UpperHead∈FullHead干涉组 | ❌冲突 |
| 夹克+腰带 | Shell/Torso | Belt/Waist | 无共同层级 | — | ✅可穿 |
| 帽子+眼罩 | Overhead/UpperHead | EyeCover/Eyes | 无共同层级 | — | ✅可穿 |

## 护甲减免机制

### 护甲处理流程：`ArmorUtility.GetPostArmorDamage()`

```
输入：Pawn, 伤害量, 穿甲值, 受击部位, 伤害类型
输出：护甲后伤害量

Step 1: 检查伤害类型是否有armorCategory
  → 无armorCategory → 直接返回原始伤害（如心理伤害）
  → 有armorCategory → 获取对应armorRatingStat

Step 2: 逐层检查服装护甲（从外到内遍历WornApparel）
  → 服装是否覆盖受击部位？（CoversBodyPart）
  → 是 → 调用ApplyArmor()
  → 伤害降为0 → 完全偏转，返回0

Step 3: 检查Pawn自身护甲（皮肤/机械体）
  → 调用ApplyArmor()
  → 返回最终伤害
```

### 单层护甲判定：`ApplyArmor()`

```
输入：伤害量, 穿甲值(armorPenetration), 护甲值(armorRating)

Step 1: 护甲磨损
  → 护甲物品受到 伤害量×25% 的损伤

Step 2: 计算有效护甲
  → effectiveArmor = max(armorRating - armorPenetration, 0)

Step 3: 随机判定（Rand.Value ∈ [0,1)）
  → 随机值 < effectiveArmor × 0.5 → 完全偏转（伤害=0）
  → 随机值 < effectiveArmor       → 半减（伤害÷2，锐器→钝器）
  → 否则                          → 全额伤害穿透
```

### 三类护甲Stat

| 伤害类型 | DamageArmorCategory | 对应护甲Stat |
|---------|--------------------|----|
| 锐器（Sharp） | `Sharp` | `ArmorRating_Sharp` |
| 钝器（Blunt） | `Blunt` | `ArmorRating_Blunt` |
| 热能（Heat） | `Heat` | `ArmorRating_Heat` |

**关键细节**：
- 护甲值上限常量：`MaxArmorRating = 2f`
- 半减时锐器伤害转为钝器（`DamageDefOf.Blunt`），模拟"没穿透但有冲击"
- 金属护甲偏转有特殊视觉/音效（`metalArmor`标记）
- 每层护甲独立判定，多层叠加时伤害逐层递减

## 特殊机制

### 服装锁定系统

```csharp
// Pawn_ApparelTracker
private List<Apparel> lockedApparel;
// 锁定的服装不可手动脱下
// 生命值低于50%时自动解锁（AutoUnlockHealthPctThreshold = 0.5f）
```

### 裸体判定

```csharp
public bool AnyClothing {
    get {
        foreach (Apparel item in wornApparel) {
            if (item.def.apparel.countsAsClothingForNudity)
                return true;
        }
        return false;
    }
}
// 腰带工具的countsAsClothingForNudity通常为false
// 所以只穿腰带仍算裸体
```

### 服装磨损

- `wearPerDay`字段控制每日磨损量（默认0.4f）
- 磨损在`ApparelTrackerTick()`中处理

### 性别与年龄限制

- `ApparelProperties.gender`：限制特定性别穿戴
- `developmentalStageFilter`：限制发育阶段（Biotech DLC，区分婴儿/儿童/成人）
