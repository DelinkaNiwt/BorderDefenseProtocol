# 机械师系统与带宽控制体系

**总览**：机械师（Mechanitor）是Biotech DLC的核心系统，允许Pawn通过Mechlink植入体控制机械体。系统围绕"带宽容量/占用"模型运作——机械师拥有带宽上限，每个机械体占用一定带宽，超出则失控。

## 1. 核心架构

三层协作：**植入体**（Hediff_Mechlink，激活条件）→ **Tracker**（Pawn_MechanitorTracker，核心逻辑）→ **Gizmo**（UI呈现）。

| 层级 | 类 | 命名空间 | 职责 |
|------|---|---------|------|
| **激活条件** | `Hediff_Mechlink` | Verse | 脑部植入体，提供基础带宽6+控制组2，死后可回收 |
| **核心Tracker** | `Pawn_MechanitorTracker` | RimWorld | 管理监管列表、带宽计算、控制组、Gizmo生成 |
| **监管对象** | `CompOverseerSubject` | RimWorld | 机械体侧Comp，管理失控倒计时和野化逻辑 |
| **机械体Comp** | `CompMechanoid` | RimWorld | 继承CompCanBeDormant，管理激活/停用/休眠 |
| **控制组** | `MechanitorControlGroup` | RimWorld | 分组管理，每组独立工作模式和目标 |

**激活前提**：Pawn脑部安装Mechlink植入体 → `PawnComponentsUtility.AddAndRemoveDynamicComponents()` 检测到Hediff → 创建`Pawn_MechanitorTracker`实例挂载到`pawn.mechanitor`。

**监管关系建立**：通过`PawnRelationDefOf.Overseer`关系连接机械师和机械体。`OverseenPawns`属性实时遍历`pawn.relations.DirectRelations`过滤Overseer关系。

## 2. 带宽系统（容量/占用模型）

核心公式：

```
可用带宽 = TotalBandwidth - UsedBandwidth
UsedBandwidth = UsedBandwidthFromSubjects + UsedBandwidthFromGestation
```

### 2.1 带宽容量来源（3种叠加）

| # | 来源 | 实现方式 | 典型值 | 叠加方式 |
|---|------|---------|--------|---------|
| 1 | **Mechlink植入体** | `HediffStage.statOffsets` | 6 | Stat加法 |
| 2 | **装备（头盔/背包）** | `equippedStatOffsets` | 3/6/9/12/15 | Stat加法 |
| 3 | **Band Node建筑** | `Hediff_BandNode` 动态Stage | 每个+1 | Stat加法 |

`TotalBandwidth`通过`pawn.GetStatValue(StatDefOf.MechBandwidth)`统一计算，所有来源通过Stat系统自动叠加。

**装备带宽梯度**（5级）：

| 装备 | 带宽 | 科研前置 | 层级 |
|------|------|---------|------|
| Airwire Headset | +3 | BasicMechtech | Overhead |
| Array Headset | +6 | StandardMechtech | Overhead |
| Integrator Headset | +9 | HighMechtech | Overhead |
| Control Pack | +9 | HighMechtech | Belt |
| Mech Commander Helmet | +12 | UltraMechtech | Overhead |

### 2.2 带宽占用来源（2种）

| # | 来源 | 计算方式 | 说明 |
|---|------|---------|------|
| 1 | **已监管机械体** | `Σ mech.GetStatValue(BandwidthCost)`（排除孵化中） | 实时统计 |
| 2 | **孵化中机械体** | `Σ bill.BandwidthCost` | 预占用 |

**机械体带宽成本**（按重量级）：

| 重量级 | 典型单位 | BandwidthCost |
|--------|---------|---------------|
| Light | Lifter, Cleansweeper | 1 |
| Medium | Militor, Constructoid, Agrihand | 1-2 |
| Heavy | Lancer, Scyther, Centipede | 2-4 |
| SuperHeavy | Diabolus, War Queen | 5 |

### 2.3 带宽变化通知链

`Notify_BandwidthChanged()` 是带宽系统的核心刷新方法，触发时机：

| 触发事件 | 调用路径 |
|---------|---------|
| 装备变化 | `Notify_ApparelChanged()` → `Notify_BandwidthChanged()` |
| Hediff变化 | `Notify_HediffStateChange()` → `Notify_BandwidthChanged()` |
| Band Node上下线 | `Hediff_BandNode.RecacheBandNodes()` → `Notify_BandwidthChanged()` |
| Pawn生成 | `Notify_PawnSpawned()` → `Notify_BandwidthChanged()` |

**超带宽处理**：`Notify_BandwidthChanged()`按分配顺序从后往前遍历，超出带宽的机械体调用`Notify_DisconnectedFromOverseer()`断开连接。

## 3. 控制组系统

### 3.1 控制组容量

控制组数量由`StatDefOf.MechControlGroups` Stat决定，来源：

| 来源 | 提供量 | 说明 |
|------|--------|------|
| Mechlink植入体 | 2 | 基础 |
| Control Sublink（标准） | +1~3 | Hediff_Level，severity=安装次数 |
| Control Sublink（高级） | +4~6 | 需先安装3级标准 |
| Control Pack | +1 | 装备 |

最大控制组数 = 2（基础）+ 6（Sublink满级）+ 1（背包）= **9组**。

### 3.2 控制组工作模式（4种MechWorkModeDef）

| defName | 标签 | 行为描述 | 特殊标志 |
|---------|------|---------|---------|
| **Work** | 工作 | 按机械体类型自主工作，需要时自动充电 | 默认模式 |
| **Escort** | 护送 | 跟随机械师，攻击附近敌人，可跟随离开地图 | `WorkModeDrawer_Escort` |
| **Recharge** | 充电 | 前往充电器充电，即使电量充足 | `ignoreGroupChargeLimits=true` |
| **SelfShutdown** | 休眠自充 | 进入休眠状态，极缓慢自充电 | 囚禁机械师时自动切换 |

每个控制组独立设置工作模式和目标（`GlobalTargetInfo`）。

### 3.3 控制组Gizmo交互

`MechanitorControlGroupGizmo`（继承Gizmo）：
- 显示组内机械体图标列表
- 右键菜单切换工作模式（`GetWorkModeOptions()`遍历所有MechWorkModeDef）
- 支持多机械师选中时合并显示（`GroupsWith()` + `MergeWith()`）
- 点击选中组内所有机械体

## 4. Gizmo UI体系

机械师选中时显示3类Gizmo：

| # | Gizmo类 | 继承 | 功能 |
|---|---------|------|------|
| 1 | `MechanitorBandwidthGizmo` | Gizmo | 带宽条——方块网格可视化，已用/总量/超额三色 |
| 2 | `Command_CallBossgroup` | Command | 召唤Boss组——从地图外召唤指定Boss机械体 |
| 3 | `MechanitorControlGroupGizmo` | Gizmo | 控制组面板——每组一个，显示成员+工作模式 |
| 4 | `Command_Action`（全选） | Command | 选中所有机械体 |

**带宽Gizmo视觉设计**：
- 方块网格（非连续条），每格代表1带宽
- 三色状态：`FilledBlockColor`（已占用）、`EmptyBlockColor`（空闲）、`ExcessBlockColor`（超额）
- 悬停显示详细分组统计（按机械体类型汇总带宽占用）

## 5. 监管与失控机制

### 5.1 监管状态（OverseerSubjectState）

`CompOverseerSubject`管理机械体的监管状态：

| 状态 | 含义 |
|------|------|
| **受控** | 有Overseer关系 + 在带宽内 → 正常工作 |
| **断开** | 超带宽/机械师倒下/离开地图 → 开始失控倒计时 |
| **野化** | 倒计时结束 + MTB概率触发 → 变为敌对 |

### 5.2 失控流程

```
断开连接（Notify_DisconnectedFromOverseer）
  → 取消征召 + 掉落携带物 + 中断Job
  → 启动延迟计时器（delayUntilFeralCheck）
  → 每Tick检查：延迟结束后按MTB概率（feralMtbDays）尝试野化
  → TryMakeFeral() → 变为敌对派系
```

### 5.3 机械师失能影响

| 事件 | 影响 |
|------|------|
| **倒下** | `Notify_Downed()` → 所有机械体取消征召 |
| **离开地图** | `Notify_DeSpawned()` → 所有机械体取消征召 |
| **被囚禁** | `Notify_ChangedGuestStatus()` → 所有控制组切换为SelfShutdown |
| **Mechlink移除** | `Notify_MechlinkRemoved()` → 断开所有Overseer关系 → 动态组件移除 |
| **阵营变化** | `Notify_ChangedFaction()` → 所有机械体跟随变更阵营 |

## 6. 机械体孵化系统

### 6.1 孵化流程

```
机械师在MechGestator工作台下单（Bill_ProductionMech/Bill_ResurrectMech）
  → 预占用带宽（BandwidthCost计入UsedBandwidthFromGestation）
  → 带宽不足时Bill自动暂停（CheckAvailableBandwidthForBills）
  → 孵化完成 → Notify_FormingCompleted() → 建立Overseer关系
  → 自动分配控制组（AssignPawnControlGroup）
```

### 6.2 关键类

| 类 | 职责 |
|---|------|
| `Building_MechGestator` | 孵化器建筑，继承Building_WorkTableAutonomous |
| `Bill_Mech` | 抽象基类，定义BandwidthCost属性 |
| `Bill_ProductionMech` | 生产新机械体，成本从recipe.ProducedThingDef读取 |
| `Bill_ResurrectMech` | 复活机械体尸体，成本从尸体内Pawn读取 |

## 7. 命令范围限制

机械师对征召机械体的命令有距离限制：

```csharp
private const float MechCommandRange = 24.9f;
// CanCommandTo: pawn.Position.DistanceToSquared(target.Cell) < 620.01f
```

- 征召状态的机械体必须在机械师24.9格范围内才能接受命令
- 选中征召机械体时自动绘制命令范围圈（`DrawCommandRadius()`）
- 超出范围的命令被拒绝
