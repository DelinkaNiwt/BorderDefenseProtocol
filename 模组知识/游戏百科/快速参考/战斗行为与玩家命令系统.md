# 战斗行为与玩家命令系统

**总览**：RimWorld的战斗行为由三层系统协作驱动——**征召系统**（Draft，切换Pawn进入战斗模式）→ **Job系统**（具体战斗行为：射击、近战、移动、待命、逃跑、施法）→ **Verb+Stance系统**（单次攻击的预热→连发→冷却节奏控制）。玩家通过**Gizmo命令按钮**下达指令，AI通过**JobGiver_AIFightEnemy**自动决策。

## 1. 战斗行为总览

```
┌─────────────────────────────────────────────────────────────────────┐
│                    战斗行为三层架构                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  第1层：征召系统（Pawn_DraftController）                              │
│    ├── 征召 → 清除工作队列、唤醒休眠、离开Lord                       │
│    ├── 解除征召 → 释放预订、中断Job、重置动物                        │
│    └── AutoUndrafter → 10000 ticks无威胁后自动解除                   │
│                                                                     │
│  第2层：Job系统（战斗行为）                                           │
│    ├── Wait_Combat ← 征召待命（自动射击引擎）                        │
│    ├── AttackStatic ← 远程射击（固定位置）                           │
│    ├── AttackMelee ← 近战攻击（追踪+攻击）                          │
│    ├── Goto ← 移动命令（玩家右键）                                   │
│    ├── CastAbility ← 能力施放                                       │
│    └── Flee/FleeAndCower ← 逃跑                                     │
│                                                                     │
│  第3层：Verb + Stance系统（单次攻击节奏）                             │
│    ├── TryStartCastOn → Stance_Warmup（预热/瞄准）                   │
│    ├── WarmupComplete → TryCastShot（连发/Bursting）                 │
│    └── BurstComplete → Stance_Cooldown（冷却）                       │
│                                                                     │
│  玩家命令入口：Gizmo按钮                                             │
│    ├── 征召/解除征召按钮（DraftController.GetGizmos）                │
│    ├── 自由射击开关（FireAtWill，仅远程武器）                        │
│    └── 能力按钮（AbilityTracker.GetGizmos）                          │
│                                                                     │
│  AI决策入口：ThinkTree                                               │
│    └── JobGiver_AIFightEnemy → 目标获取 → 能力/近战/远程Job          │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 2. 征召系统（Pawn_DraftController）

### 2.1 核心属性

`Pawn_DraftController`（RimWorld命名空间）管理Pawn的征召状态。

| 属性 | 类型 | 说明 |
|------|------|------|
| `Drafted` | bool | 征召状态（setter含大量副作用逻辑） |
| `FireAtWill` | bool | 自由射击开关（默认true，征召时重置） |
| `ShowDraftGizmo` | bool | 是否显示征召按钮 |
| `autoUndrafter` | AutoUndrafter | 自动解除征召管理器 |

### 2.2 Drafted setter — 征召/解除征召时的副作用

**征召时（Drafted = true）**：
```csharp
// 1. 清除工作队列
pawn.mindState.priorityWork.ClearPrioritizedWorkAndJobQueue();
pawn.jobs.ClearQueuedJobs();

// 2. 离开自愿加入的Lord
lord.Notify_PawnLost(pawn, PawnLostCondition.Drafted);

// 3. 通知自动解除征召器
autoUndrafter.Notify_Drafted();

// 4. 丢弃组队打包物品
pawn.inventory.DropAllPackingCaravanThings();

// 5. 唤醒休眠单位
pawn.TryGetComp<CompCanBeDormant>()?.WakeUp();

// 6. 重置FireAtWill为true
fireAtWillInt = true;
```

**解除征召时（Drafted = false）**：
```csharp
// 1. 释放所有目的地预订
pawn.Map.pawnDestinationReservationManager.ReleaseAllClaimedBy(pawn);

// 2. 中断当前Job（如果可中断）
if (pawn.jobs.curJob != null && pawn.jobs.IsCurrentJobPlayerInterruptible())
    pawn.jobs.EndCurrentJob(JobCondition.InterruptForced);

// 3. 重置动物释放状态
pawn.playerSettings.animalsReleased = false;

// 4. 特殊处理：携带婴儿→创建"带婴儿到安全地点"Job
// 5. 特殊处理：携带物品→尝试放下
```

**通知驯养动物**：征召/解除征召时对所有驯养动物调用 `jobs.Notify_MasterDraftedOrUndrafted()`。

### 2.3 AutoUndrafter — 自动解除征召

```csharp
// 触发条件（全部满足时自动解除征召）：
// 1. 当前Job是Wait_Combat
// 2. 没有活跃的敌对威胁（AnyHostilePreventingAutoUndraft()）
// 3. 超过延迟时间（10000 ticks ≈ 2.8分钟）
// 4. 非机械体（机械体不自动解除）
```

### 2.4 FireAtWill — 自由射击开关

```csharp
public bool FireAtWill
{
    get => fireAtWillInt;
    set {
        fireAtWillInt = value;
        // 关键：关闭时立即取消正在进行的Warmup瞄准
        if (!fireAtWillInt && pawn.stances.curStance is Stance_Warmup)
            pawn.stances.CancelBusyStanceSoft();
    }
}
```

**影响范围**：
1. `JobDriver_Wait.CheckForAutoAttack()` — FireAtWill为false时跳过远程目标搜索
2. 设置为false时立即中断正在进行的瞄准（Stance_Warmup）
3. 不影响近战自动攻击——仅控制远程
4. 每次征召时重置为true

### 3.4 能力按钮（Pawn_AbilityTracker.GetGizmos）

```csharp
// Pawn_AbilityTracker.GetGizmos()
foreach (Ability a in AllAbilitiesForReading)
{
    // 可见性条件：
    // 1. 玩家控制的殖民者/机械体/亚人类/动物
    // 2. 征召状态 OR ability.def.displayGizmoWhileUndrafted
    // 3. ability.GizmosVisible() 返回true

    {
        foreach (Command c in a.GetGizmos())
            yield return c;
        foreach (Gizmo g in a.GetGizmosExtra())
            yield return g;
```

> - 自定义ThingComp/HediffComp/Gene子类的 `GetGizmos()` 也会被收集

## 4. 战斗Job系统

### 4.1 6个核心战斗Job对比

| # | JobDef | JobDriver | 触发方式 | 核心行为 | 结束条件 |
|---|--------|-----------|---------|---------|---------|
| 1 | **Wait_Combat** | `JobDriver_Wait` | 征召后默认 | 原地待命 + 自动射击 | 玩家下达新命令 |
| 2 | **AttackStatic** | `JobDriver_AttackStatic` | 玩家右键敌人(远程) | 固定位置远程射击 | 目标死亡/倒下/超出射程 |
| 3 | **AttackMelee** | `JobDriver_AttackMelee` | 玩家右键敌人(近战) | 追踪+近战攻击 | 目标死亡/倒下/不可达 |
| 4 | **Goto** | `JobDriver_Goto` | 玩家右键地面 | 移动到目标位置 | 到达目的地 |
| 5 | **CastAbility** | `JobDriver_CastAbility` | 玩家点击能力按钮 | 施放能力(含预热) | 施放完成/目标消失 |
| 6 | **Flee** | `JobDriver_Flee` | AI逃跑决策 | 逃离威胁源 | 到达安全位置 |

### 4.2 Wait_Combat — 征召待命与自动射击

Wait_Combat是征召状态下的默认Job，也是**自动射击的引擎**。

**CheckForAutoAttack() 核心逻辑**（每4 tick执行一次）：

```csharp
private void CheckForAutoAttack()
{
    // 前置条件：非倒地、非全身忙碌、非携带Pawn、非蹒跚者
    if (pawn.Downed || pawn.stances.FullBodyBusy || pawn.IsCarryingPawn())
        return;

    // 优先级1：近战自动攻击（周围9格）
    for (int i = 0; i < 9; i++)
    {
        IntVec3 c = pawn.Position + GenAdj.AdjacentCellsAndInside[i];
        // 检查敌对Pawn → pawn.meleeVerbs.TryMeleeAttack()
    }

    // 优先级2：灭火（仅玩家派系+非暴力禁止）
    if (fire != null)
    {
        pawn.natives.TryBeatFire(fire);
        return;
    }

    // 优先级3：远程自动射击
    // 关键检查：FireAtWill必须为true
    if (!job.canUseRangedWeapon || job.def != JobDefOf.Wait_Combat
        || (pawn.drafter != null && !pawn.drafter.FireAtWill))
        return;

    Verb verb = pawn.CurrentEffectiveVerb;
    if (verb != null && !verb.verbProps.IsMeleeAttack)
    {
        TargetScanFlags flags = NeedLOSToAll | NeedThreat | NeedAutoTargetable;
        Thing target = AttackTargetFinder.BestShootTargetFromCurrentPosition(pawn, flags);
        if (target != null)
            pawn.TryStartAttack(target);  // 发起攻击
    }
}
```

**调用时机**：
1. Job初始化时调用一次
2. 每4 tick检查一次（`GenTicks.IsTickIntervalDelta(pawn.thingIDNumber, 4, delta)`）
3. Stance变化时（切换回`Stance_Mobile`时）

> **自动射击优先级**：近战（9格内敌人）> 灭火 > 远程射击。近战不受FireAtWill控制。

## 5. AI自动战斗决策（JobGiver_AIFightEnemy）

### 5.1 核心架构

`JobGiver_AIFightEnemy`（RimWorld命名空间，抽象类）是AI战斗决策的核心，继承自`ThinkNode_JobGiver`。

**关键配置参数**：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `targetAcquireRadius` | 56格 | 获取新目标的搜索半径 |
| `targetKeepRadius` | 65格 | 保持已有目标的最大距离 |
| `chaseTarget` | false | 是否追逐目标 |
| `allowTurrets` | false | 是否允许攻击炮塔 |
| `ignoreNonCombatants` | false | 是否忽略非战斗人员 |
| `humanlikesOnly` | false | 是否仅针对类人生物 |

### 5.2 TryGiveJob 决策流程

```
TryGiveJob(pawn)
  │
  ├── 1. 检查敌对响应模式
  │     └── 殖民者：hostilityResponse != Attack → return null
  │
  ├── 2. UpdateEnemyTarget(pawn)
  │     ├── 验证当前目标（被摧毁？超出距离？超时？）
  │     └── 寻找新目标：AttackTargetFinder.BestAttackTarget()
  │
  ├── 3. 无敌人目标 → return null
  │
  ├── 4. 尝试能力Job（非殖民者/亚人类优先）
  │     └── GetAbilityJob(pawn, enemyTarget) → 有可用能力 → return abilityJob
  │
  ├── 5. 尝试近战攻击
  │     └── MeleeAttackJob(pawn, enemyTarget)
  │
  └── 6. 尝试远程攻击
        ├── TryFindShootingPosition(pawn, out dest, verb)
        └── 创建Goto(射击位置) 或 AttackStatic Job
```

### 5.3 敌对响应模式（HostilityResponseMode）

| 模式 | 行为 | 适用对象 |
|------|------|---------|
| `Ignore` | 忽略威胁，不参与战斗 | 非战斗殖民者 |
| `Attack` | 主动攻击敌人 | 战斗殖民者（默认） |
| `Flee` | 逃离威胁 | 非战斗殖民者 |

> 征召状态下的殖民者不受HostilityResponseMode限制——征召后由玩家命令和Wait_Combat的自动射击驱动。

### 5.4 目标获取与丢失逻辑

**目标丢失条件**（`ShouldLoseTarget`）：
1. 目标被摧毁
2. 目标超出`targetKeepRadius`（65格）
3. 接战后超过`TicksSinceEngageToLoseTarget`（400 ticks）未再次接战

**目标获取**：
```csharp
AttackTargetFinder.BestAttackTarget(
    pawn,
    flags: TargetScanFlags.NeedReachable | NeedThreat,
    maxDist: targetAcquireRadius,  // 56格
    validator: ExtraTargetValidator  // 额外验证
);
```

**Job过期间隔**（控制AI重新评估频率）：

| 场景 | 过期间隔 | 说明 |
|------|---------|------|
| 射手成功射击 | 450-550 ticks | 射击后较长间隔 |
| 近战攻击 | 360-480 ticks | 近战中等间隔 |
| 能力使用 | 30 ticks | 能力后快速重新评估 |

## 6. Stance状态机（攻击节奏控制）

### 6.1 Stance继承体系

```
Stance (抽象基类)
  ├── Stance_Mobile        ← 默认状态，可自由移动
  └── Stance_Busy (抽象)   ← 忙碌状态，无法移动
        ├── Stance_Warmup   ← 预热/瞄准（ticksLeft倒计时）
        └── Stance_Cooldown ← 冷却（ticksLeft倒计时）
```

### 6.2 Stance_Busy 基类

| 属性 | 类型 | 说明 |
|------|------|------|
| `ticksLeft` | int | 剩余tick数（每tick递减） |
| `verb` | Verb | 关联的Verb |
| `focusTarg` | LocalTargetInfo | 焦点目标 |
| `neverAimWeapon` | bool | 是否不瞄准武器 |

**核心方法**：
- `StanceTick()` — 每tick递减ticksLeft，到0时调用`Expire()`
- `Expire()` — 过期时切换回`Stance_Mobile`

### 6.3 Stance_Warmup — 预热/瞄准

```csharp
public Stance_Warmup(int ticks, LocalTargetInfo focusTarg, Verb verb)
    : base(ticks, focusTarg, verb)
{
    // 通知目标护盾保持显示
    // 初始化视觉效果：瞄准线(aimLineMote)、充能特效、目标标记
    // 播放瞄准音效
}

protected override void Expire()
{
    verb?.WarmupComplete();  // 预热完成→触发Verb射击
    effecter?.Cleanup();
    base.Expire();
}
```

**中断条件**：
- 目标倒下（如果开始时未倒下）
- 目标消失或超出射程
- 被眩晕时暂停计时（不中断，只暂停）
- FireAtWill设为false时被`CancelBusyStanceSoft()`取消

### 6.4 Stance_Cooldown — 冷却

非常简单的类，主要作用是在冷却期间阻止Pawn行动。选中时绘制冷却圆圈UI。

### 6.5 完整攻击节奏流程

```
Pawn.TryStartAttack(target)
  → Verb.TryStartCastOn(target)
    → 有预热时间？
      ├── Yes → 创建Stance_Warmup(warmupTicks)
      │           → ticksLeft倒计时...
      │           → Expire() → Verb.WarmupComplete()
      └── No  → 直接Verb.WarmupComplete()
    → Verb.WarmupComplete()
      → burstShotsLeft = ShotsPerBurst
      → state = VerbState.Bursting
      → TryCastNextBurstShot()
        → TryCastShot()  // 抽象方法，由Verb_Shoot/Verb_MeleeAttackDamage实现
        → burstShotsLeft--
        → burstShotsLeft > 0?
          ├── Yes → 创建Stance_Cooldown(TicksBetweenBurstShots)  // 连发间隔
          │           → Expire() → TryCastNextBurstShot()  // 继续连发
          └── No  → 创建Stance_Cooldown(AdjustedCooldownTicks)  // 武器冷却
                    → Expire() → 回到Stance_Mobile
```

**关键时间参数**：

| 参数 | 来源 | 说明 |
|------|------|------|
| WarmupTime | `VerbProperties.warmupTime` | 预热时间（秒），受`AimingDelayFactor` stat影响 |
| TicksBetweenBurstShots | `VerbProperties.ticksBetweenBurstShots` | 连发间隔（ticks） |
| ShotsPerBurst | `VerbProperties.burstShotCount` | 每次连发射击数 |
| CooldownTicks | `VerbProperties.AdjustedCooldownTicks()` | 武器冷却时间，受`RangedCooldownFactor` stat影响 |

## 7. Verb系统在战斗中的角色

### 7.1 Verb与Job的关系

```
Job系统（宏观行为）          Verb系统（微观攻击）
┌──────────────┐           ┌──────────────┐
│ Wait_Combat  │──自动──→  │              │
│ AttackStatic │──手动──→  │ TryStartAttack│──→ Verb.TryStartCastOn
│ AttackMelee  │──手动──→  │              │──→ Stance_Warmup
│ CastAbility  │──手动──→  │              │──→ Verb.TryCastShot
└──────────────┘           └──────────────┘
```

Job决定"做什么"（攻击谁、在哪攻击），Verb决定"怎么做"（预热多久、射几发、冷却多久）。

### 7.2 Pawn.TryStartAttack — Job到Verb的桥梁

```csharp
public bool TryStartAttack(LocalTargetInfo targ)
{
    if (stances.FullBodyBusy) return false;           // Stance忙碌→不能攻击
    if (WorkTagIsDisabled(WorkTags.Violent)) return false;  // 非暴力→不能攻击

    bool allowManualCastWeapons = !IsColonist;        // 殖民者不自动使用手动武器
    Verb verb = TryGetAttackVerb(targ.Thing, allowManualCastWeapons);

    return verb?.TryStartCastOn(targ) ?? false;       // 委托给Verb
}
```

### 7.3 关键Verb子类

| Verb子类 | 攻击类型 | TryCastShot行为 |
|---------|---------|----------------|
| `Verb_Shoot` | 远程射击 | 调用基类`TryCastShot()`生成投射物 |
| `Verb_LaunchProjectile` | 投射物发射（基类） | 创建Projectile并Launch |
| `Verb_MeleeAttackDamage` | 近战伤害 | 生成DamageInfo，伤害随机化±20% |
| `Verb_MeleeAttack` | 近战攻击（基类） | 命中判定→闪避判定→ApplyMeleeDamage |
| `Verb_CastAbility` | 能力施放 | 调用Ability.Activate() |
