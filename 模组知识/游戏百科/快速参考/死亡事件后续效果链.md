# 死亡事件后续效果链

**总览**：Pawn死亡由`Pawn.Kill()`驱动，分**6个阶段**按顺序执行——DoKillSideEffects（叙事者/统计/杀手记录）→ PreDeathPawnModifications（Hediff/服装/关系通知）→ DropBeforeDying（心情Debuff分发）→ SetDead+尸体生成+物品掉落+DeathActionWorker → 组件清理 → 死后通知链（派系/任务/基因/意识形态/20+系统）。整个流程涉及**7大核心系统**（健康、心情、关系、派系、任务、叙事、物品），触发**30+个Notify回调**，是RimWorld中副作用链最长的单一事件。

## 一、心情Debuff系统

### 核心入口

`PawnDiedOrDownedThoughtsUtility.TryGiveThoughts(victim, dinfo, PawnDiedOrDownedThoughtsKind.Died)` 在阶段4（DropBeforeDying）中调用，分发两类心情：**个体心情**（IndividualThoughtToAdd，给特定Pawn）和**全殖民者心情**（ThoughtToAddToAll，给所有殖民者）。

### 1.1 目击死亡类（需在视线范围内）

| ThoughtDef | 心情 | 持续 | 触发条件 |
|-----------|------|------|---------|
| `WitnessedDeathAlly` | -5 | 6天 | 目击同派系非奴隶成员死亡 |
| `WitnessedDeathNonAlly` | -2 | 6天 | 目击非敌对者/奴隶死亡 |
| `WitnessedDeathFamily` | -6 | 6天 | 目击血亲死亡（叠加在上述之上） |
| `WitnessedDeathBloodlust` | +5 | 6天 | 嗜血特性者目击任何死亡（覆盖负面） |

**判定逻辑**：`ThoughtUtility.Witnessed(item, victim)` — 在同一地图且视线范围内。未目击但同派系的殖民者获得`KnowColonistDied`。

### 1.2 知晓死亡类（全殖民者）

| ThoughtDef | 心情 | 持续 | 触发条件 |
|-----------|------|------|---------|
| `KnowColonistDied` | -3 | 6天 | 同派系殖民者死亡（非奴隶/非寄宿者/非亚人类），未目击 |
| `KnowPrisonerDiedInnocent` | -5 | 6天 | 无辜囚犯死亡（非处决） |
| `KnowGuestExecuted` | -6 | 6天 | 客人被处决 |
| `KnowColonistExecuted` | -6 | 6天 | 殖民者被处决 |

**豁免条件**：`victim.guilt.IsGuilty` — 有罪Pawn死亡不触发大部分全殖民者心情。

### 1.3 家庭关系类（通过AppendThoughts_Relations分发）

| ThoughtDef | 心情 | 持续 | 关系 |
|-----------|------|------|------|
| `MyHusbandDied` / `MyWifeDied` | -20 | 30天 | 配偶 |
| `MySonDied` / `MyDaughterDied` | -20 | 30天 | 子女 |
| `MyFianceDied` / `MyFianceeDied` | -18 | 30天 | 未婚夫/妻 |
| `MyLoverDied` | -16 | 30天 | 恋人 |
| `MyBrotherDied` / `MySisterDied` | -14 | 30天 | 兄弟姐妹 |
| `MyGrandchildDied` | -12 | 30天 | 孙辈 |
| `MyFatherDied` / `MyMotherDied` | -8 | 30天 | 父母 |
| `MyNieceDied` / `MyNephewDied` | -5 | 30天 | 侄子/侄女 |
| `MyHalfSiblingDied` | -5 | 30天 | 同父异母兄弟姐妹 |
| `MyAuntDied` / `MyUncleDied` | -4 | 30天 | 叔伯姑姨 |
| `MyGrandparentDied` | -4 | 30天 | 祖父母 |
| `MyCousinDied` | -4 | 30天 | 堂/表兄弟姐妹 |
| `MyKinDied` | -4 | 30天 | 其他亲属 |

### 1.4 社交关系类

| ThoughtDef | 心情 | 持续 | 触发条件 |
|-----------|------|------|---------|

| `PawnWithBadOpinionDied` | +10 | 10天 | 好感度≤-20的Pawn死亡 |
| `BondedAnimalDied` | -8 | 20天 | 绑定动物死亡 |

## 二、关系系统影响

### 核心入口

`Pawn_RelationsTracker.Notify_PawnKilled(dinfo, map)` 在阶段3（PreDeathPawnModifications）中调用。

### 2.1 执行内容

```csharp
// Pawn_RelationsTracker.Notify_PawnKilled() L891
1. 所有有关系的Pawn → Notify_SituationalThoughtsDirty()  // 刷新情境心情
2. RemoveMySpouseMarriageRelatedThoughts()                // 移除配偶婚姻正面心情
3. RemoveRelationsOnDeath()                               // 移除特定关系类型
4. AffectBondedAnimalsOnMyDeath()                         // 绑定动物行为变化
5. Notify_FailedRescueQuest()                             // 失败救援任务
```

### 2.2 关系保留与移除

| 关系类型 | 死亡后 | 说明 |
|---------|--------|------|
| 配偶 | 保留（变为"已故配偶"） | 移除婚姻正面心情 |
| 父母/子女 | 保留 | 永久关系 |
| 恋人/未婚夫妻 | 移除 | `RemoveRelationsOnDeath` |
| 绑定动物 | 保留 | 动物获得debuff |

### 2.3 绑定动物影响

`AffectBondedAnimalsOnMyDeath()` — 主人死亡时，绑定动物可能：
- 获得心情debuff（`BondedAnimalMasterDied`）
- 概率发狂（`MentalState_Manhunter`）
- 概率出走（`MentalState_Wander`）

## 三、物品与装备掉落

### 核心入口

`DropAndForbidEverything()` 在阶段5中调用（仅spawned状态）。

### 3.1 掉落规则

| 条件 | 行为 |
|------|------|
| `kindDef.destroyGearOnDrop = true` | 所有装备和服装**销毁**（不掉落） |
| 在封闭容器内 | 物品转移到容器 |
| 正常掉落 | 装备/背包/携带物掉落到附近，设为**禁止拾取** |
| 被猎人杀死 | 尸体自动**预留**给猎人 |
| 被屠宰/仪式处死 | 尸体**不设禁止** |

### 3.2 尸体生成（MakeCorpse）

`MakeCorpse(assignedGrave, currentBed)` 创建`Corpse`对象：
- 尸体内部保存完整的Pawn对象（`InnerPawn`）
- 继承所有Hediff和装备状态
- 记住分配的墓地
- 在床上死亡保持床的旋转角度

### 3.3 尸体腐烂判定

```csharp
// Pawn.Kill() 阶段5
if (ToxicBuildup != null && Rand.Value < ToxicBuildup.Severity)
    corpse.GetComp<CompRottable>().RotImmediately();  // 毒素积累→概率立即腐烂
if (Scaria != null && Rand.Chance(difficulty.scariaRotChance))
    corpse.GetComp<CompRottable>().RotImmediately();  // 狂暴病→概率立即腐烂
```

### 3.4 额外掉落物

`GenLeaving.DoLeavingsFor(this, map, DestroyMode.KillFinalize)` — 根据种族定义生成肉、皮革等资源。火焰附着物会转移到尸体位置。

## 四、DeathActionWorker（种族特定死亡动作）

### 触发时机

阶段6中，`RaceProps.DeathActionWorker.PawnDied(corpse, prevLord)` — 仅在spawned状态且非Shambler时触发。

### 6个子类

| Worker | 命名空间 | 行为 | 典型使用者 |
|--------|---------|------|-----------|
| `DeathActionWorker_Simple` | Verse | 无特殊效果（默认） | 大多数Pawn |
| `DeathActionWorker_SmallExplosion` | RimWorld | 小型爆炸 | 爆炸虫 |
| `DeathActionWorker_BigExplosion` | RimWorld | 大型爆炸 | 自爆机械体 |
| `DeathActionWorker_ToxCloud` | RimWorld | 释放毒云 | 毒气虫 |
| `DeathActionWorker_Vanish` | Verse | 消失（不留尸体） | 特殊实体 |
| `DeathActionWorker_Divide` | Verse | 分裂成多个单位 | 分裂型生物 |

> **模组扩展**：自定义DeathActionWorker只需继承基类并重写`PawnDied(Corpse, Lord)`，在RaceProperties XML中配置`deathActionWorkerClass`即可。

## 五、派系/任务/叙事者通知

### 5.1 叙事者适应系统（阶段2）

`Find.Storyteller.Notify_PawnEvent(this, AdaptationEvent.Died)` → `StoryWatcher_Adaptation` 降低adaptDays：
- 殖民者死亡：adaptDays -= 20~30天（按死后人口查曲线）
- 效果：后续威胁点数降低（adaptDays -30 → 威胁倍率0.40）

### 5.2 派系通知（阶段6）

`HomeFaction.Notify_MemberDied(this, dinfo, wasWorldPawn, wasGuilty, map)` — 可能影响：
- 派系好感度变化（杀死友好派系成员→好感度下降）
- 成员统计更新
- 领袖死亡特殊处理

`Find.FactionManager.Notify_PawnKilled(this)` — 全局派系关系更新。

### 5.3 任务系统（阶段6）

`Find.QuestManager.Notify_PawnKilled(this, dinfo)` — 可能导致：
- 任务失败（死者是任务目标/保护对象）
- 任务状态更新
- 任务奖励变化

### 5.4 其他系统通知

| 系统 | 方法 | 说明 |
|------|------|------|
| 殖民者统计 | `StoryWatcher.statsRecord.Notify_ColonistKilled()` | 记录死亡统计 |
| Tale系统 | `TaleUtility.Notify_PawnDied()` | 生成故事记录（可成为艺术品主题） |
| 战斗日志 | `Find.BattleLog.Add()` | 添加死亡日志条目 |
| 账单系统 | `BillUtility.Notify_ColonistUnavailable()` | 清除死者的工作账单 |
| Boss组 | `Find.BossgroupManager.Notify_PawnKilled()` | Boss组状态更新 |
