# 战斗极端情况与安全网机制

**总览**：RimWorld的战斗系统内置了**7层安全网机制**，从伤害最终化到倒地后行为，层层递进地给予Pawn（尤其是殖民者）存活机会。核心状态机是`PawnHealthState`三状态（Mobile→Down→Dead），由`CheckForStateChange()`统一调度。即死保护仅限殖民者，倒地即死概率按生物类型和难度动态调节，不死基因提供额外的死亡替代路径。

```
┌─────────────────────────────────────────────────────────────────────┐
│                    7层战斗安全网（按管线位置排列）                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  第1层：即死保护（阶段6 FinalizeAndAddInjury）                       │
│    └── 仅殖民者 + 外部暴力 → 最多7次迭代削减伤害至不致死            │
│                                                                     │
│  第2层：preventsDeath Hediff（ShouldBeDead检查）                     │
│    └── 有preventsDeath=true的Hediff → 跳过所有死亡判定              │
│                                                                     │
│  第3层：不死基因安全网（CheckForStateChange）                        │
│    └── Deathless基因 + 大脑完好 → 死眠/昏迷替代死亡                 │
│                                                                     │
│  第4层：倒地替代死亡（CheckForStateChange）                          │
│    └── ShouldBeDowned() → MakeDowned() 而非 Kill()                  │
│                                                                     │
│  第5层：倒地即死概率（CheckForStateChange）                          │
│    └── 非玩家Pawn → 按类型/难度/人口意图概率判定                    │
│                                                                     │
│  第6层：爬行机制（倒地后行为）                                       │
│    └── 人形 + 清醒 + 操作≥15% → 可爬行逃离/去床位                  │
│                                                                     │
│  第7层：恢复机制（CheckForStateChange）                              │
│    └── 伤势好转 → MakeUndowned() 恢复Mobile状态                     │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 1. ★ 即死保护（Instant Kill Protection）

### 1.1 难度设置：allowInstantKillChance

| 字段 | 类型 | 默认值 | 范围 | 说明 |
|------|------|--------|------|------|
| `allowInstantKillChance` | float | 1.0 | 0~1 | 殖民者被一击致死的概率 |

- **1.0（默认）**：100%允许即死 = 无保护
- **0.0**：0%允许即死 = 完全保护
- 所有难度预设默认1.0，玩家可在自定义难度中手动调低
- UI标签："Colonist instant kills"

> **官方说明**（Menus_Main.xml L247）：*"Adjust the chance that a colonist can die instantly due to an attack. The incoming damage is adjusted so it will down the colonist instead. This only prevents some kinds of instant kills - colonists can still be downed and bleed to death, die from fire, explosions, disease, and so on."*

### 1.2 FinalizeAndAddInjury 保护逻辑

**位置**：`Verse.DamageWorker_AddInjury.FinalizeAndAddInjury()` L226

```csharp
protected float FinalizeAndAddInjury(Pawn pawn, Hediff_Injury injury, DamageInfo dinfo, DamageResult result)
{
    injury.TryGetComp<HediffComp_GetsPermanent>()?.PreFinalizeInjury();
    float partHealth = pawn.health.hediffSet.GetPartHealth(injury.Part);

    // ★ 即死保护核心逻辑
    if (pawn.IsColonist                              // 条件1：仅殖民者
        && !dinfo.IgnoreInstantKillProtection         // 条件2：伤害未标记忽略保护
        && dinfo.Def.ExternalViolenceFor(pawn)        // 条件3：外部暴力伤害
        && !Rand.Chance(Find.Storyteller.difficulty.allowInstantKillChance))  // 条件4：概率判定
    {
        float num = (injury.IsLethal ? (injury.def.lethalSeverity * 1.1f) : 1f);
        float min = 1f;
        float max = Mathf.Min(injury.Severity, partHealth);

        for (int i = 0; i < 7; i++)  // 最多7次迭代
        {
            if (!pawn.health.WouldDieAfterAddingHediff(injury))
                break;  // 不再致死→停止削减

            float num2 = Mathf.Clamp(partHealth - num, min, max);
            injury.Severity = num2;  // 削减伤害
            num *= 2f;               // 每次翻倍削减量
            min = 0f;                // 第二次起允许削减到0
        }
    }

    pawn.health.AddHediff(injury, null, dinfo, result);
    // ... 后续处理
}
```

**保护算法流程**：

```
原始伤害 → WouldDieAfterAddingHediff?
  ├── No → 正常应用伤害
  └── Yes → 迭代削减（最多7次）
        ├── 第1次：Severity = partHealth - 1（或lethalSeverity×1.1）
        ├── 第2次：Severity = partHealth - 2
        ├── 第3次：Severity = partHealth - 4
        ├── ...（每次翻倍）
        └── 第7次：Severity = partHealth - 64
        直到 WouldDieAfterAddingHediff = false
```

**四个触发条件**：

| # | 条件 | 说明 |
|---|------|------|
| 1 | `pawn.IsColonist` | 仅玩家殖民者（不含囚犯、奴隶、访客） |
| 2 | `!dinfo.IgnoreInstantKillProtection` | 伤害未标记忽略（处决等特殊伤害会标记） |
| 3 | `ExternalViolenceFor(pawn)` | 外部暴力（不含疾病、自伤、手术） |
| 4 | `!Rand.Chance(allowInstantKillChance)` | 概率判定（默认1.0=不触发） |

> **关键发现**：即死保护不阻止倒地——它只确保伤害不直接致死，Pawn仍可能倒地后流血致死。这是"延缓死亡"而非"阻止死亡"。

## 2. 倒地状态机（Downed State Machine）

### 2.1 PawnHealthState 三状态

```csharp
public enum PawnHealthState : byte
{
    Mobile,  // 正常活动
    Down,    // 倒地（失能但存活）
    Dead     // 死亡
}
```

**状态转换由`CheckForStateChange()`统一调度**：

```
Mobile ──(ShouldBeDead)──→ Dead
  │                          ↑
  │                          │ forceDeathOnDowned
  │                          │ 或 倒地即死概率命中
  ├──(ShouldBeDowned)──→ Down ──(ShouldBeDead)──→ Dead
  │                        │
  │                        │ !ShouldBeDowned
  └────────────────────────┘ MakeUndowned
```

### 2.2 ShouldBeDowned() — 倒地判定

**位置**：`Verse.Pawn_HealthTracker.ShouldBeDowned()` L614

```csharp
public bool ShouldBeDowned()
{
    if (!InPainShock
        && capacities.CanBeAwake
        && (capacities.CapableOf(PawnCapacityDefOf.Moving) || pawn.RaceProps.doesntMove))
    {
        return pawn.ageTracker.CurLifeStage.alwaysDowned;
    }
    return true;
}
```

**四个触发条件**（满足任一即倒地）：

| # | 条件 | 判定方式 | 典型场景 |
|---|------|---------|---------|
| 1 | **疼痛休克** | `PainTotal ≥ PainShockThreshold` stat | 大量伤害累积 |
| 2 | **无法保持清醒** | `!CanBeAwake`（意识能力为0） | 麻醉、脑损伤 |
| 3 | **无法移动** | `!CapableOf(Moving)` 且非固定生物 | 双腿被毁、脊椎损伤 |
| 4 | **生命阶段强制** | `alwaysDowned`（婴儿等） | 新生儿 |

### 2.3 疼痛休克机制

**位置**：`Verse.Pawn_HealthTracker.InPainShock` L95

```csharp
public bool InPainShock
{
    get
    {
        if (!pawn.kindDef.ignoresPainShock)
            return hediffSet.PainTotal >= pawn.GetStatValue(StatDefOf.PainShockThreshold);
        return false;
    }
}
```

| 要素 | 说明 |
|------|------|
| `PainTotal` | 所有Hediff的疼痛贡献之和 |
| `PainShockThreshold` | Stat，默认0.8，可被特性/装备/基因修改 |
| `ignoresPainShock` | PawnKindDef标记，某些生物免疫疼痛休克 |

> **模组扩展点**：通过`statOffsets`/`statFactors`修改`PainShockThreshold`可改变倒地阈值，无需C#。

## 3. 倒地即死概率系统（Death on Downed）

### 3.1 触发条件

倒地即死概率仅在以下条件全部满足时计算：

| # | 条件 | 说明 |
|---|------|------|
| 1 | `!forceDowned` | 非强制倒地 |
| 2 | 外部暴力或可应用DoD的Hediff | 非自然原因 |
| 3 | `!pawn.IsWildMan()` | 非野人 |
| 4 | `!pawn.IsDeactivated()` | 非停用状态 |
| 5 | 非玩家派系 | `Faction != OfPlayer` |
| 6 | 非玩家宿主派系 | `HostFaction != OfPlayer` |

### 3.2 概率计算优先级链（8级）

```csharp
float deathChance =
    (overrideDeathOnDownedChance >= 0f)           ? overrideDeathOnDownedChance :        // ①手动覆盖
    (pawn.IsMutant && mutant.Def.dodChance >= 0f) ? mutant.Def.deathOnDownedChance :     // ②变异体定义
    (hasDeathless && Faction == OfPlayer)          ? 0f :                                 // ③不死基因+玩家
    (kindDef.overrideDoD.HasValue)                ? kindDef.overrideDoD.Value :           // ④PawnKind覆盖
    (Faction == OfEntities)                        ? EntityThreatCurve.Evaluate(...) :    // ⑤实体派系
    (RaceProps.Animal)                             ? 0.5f :                               // ⑥动物
    (RaceProps.IsMechanoid)                        ? 1f :                                 // ⑦机械体
    PopulationIntentCurve.Evaluate(...) * difficultyFactor;                               // ⑧人形生物
```

### 3.3 人形生物倒地即死曲线

**HealthTuning.DeathOnDownedChance_NonColonyHumanlikeFromPopulationIntentCurve**：

| 人口意图 | 倒地即死概率 | 说明 |
|---------|------------|------|
| -1（人口远超需求） | 86.7% | 不需要更多殖民者→高死亡率 |
| 0（人口刚好） | 80% | 基准 |
| 1（需要更多人口） | 56.6% | 需要殖民者→降低死亡率 |
| 2（急需人口） | 50% | 急需→更多存活 |
| 8（极度缺人） | 26.3% | 极度缺人→大幅降低 |

**动摇囚犯变体**（`unwaveringPrisoners=false`时使用）：

| 人口意图 | 倒地即死概率 |
|---------|------------|
| -1 | 92% |
| 0 | 85% |
| 1 | 62% |
| 2 | 55% |
| 8 | 30% |

**难度系数**（`enemyDeathOnDownedChanceFactor`）：

| 难度 | 系数 |
|------|------|
| 和平建造者 | 0.5 |
| 冒险故事 | 0.9 |
| 血与尘埃/失落部落/富有挑战 | 1.0 |

### 3.4 各生物类型倒地即死概率汇总

| 生物类型 | 倒地即死概率 | 说明 |
|---------|------------|------|
| **玩家殖民者** | 不适用 | 不触发倒地即死系统 |
| **机械体** | 100% | 倒地=死亡 |
| **动物** | 50% | 固定概率 |
| **敌对人形** | 26%~87% | 基于人口意图×难度系数 |
| **实体（Anomaly）** | 基于威胁曲线 | 100点→0%，1000点→50%，5000点→90% |
| **forceDeathOnDowned** | 100% | 无人机、某些实体 |
| **Deathless基因+玩家** | 0% | 完全免疫 |

### 3.5 Deathless基因的特殊处理

当倒地即死概率命中且Pawn有Deathless基因时：

```csharp
if (flag && !pawn.Dead)
{
    // 不直接Kill，而是摧毁大脑
    pawn.health.AddHediff(HediffDefOf.MissingBodyPart, pawn.health.hediffSet.GetBrain(), dinfo);
}
else
{
    pawn.Kill(dinfo);
}
```

> **设计意图**：Deathless基因的Pawn不能被简单Kill（因为ShouldBeDead会被Deathless拦截），所以通过摧毁大脑绕过Deathless保护。

## 4. 不死基因安全网（Deathless Gene）

### 4.1 ShouldBeDeathrestingOrInComa 判定

**位置**：`RimWorld.SanguophageUtility.ShouldBeDeathrestingOrInComaInsteadOfDead()` L13

```csharp
public static bool ShouldBeDeathrestingOrInComaInsteadOfDead(Pawn pawn)
{
    if (!ModsConfig.BiotechActive) return false;
    if (!pawn.health.ShouldBeDead()) return false;

    if (pawn.genes != null && pawn.genes.HasActiveGene(GeneDefOf.Deathless))
    {
        BodyPartRecord brain = pawn.health.hediffSet.GetBrain();
        if (brain != null
            && !pawn.health.hediffSet.PartIsMissing(brain)
            && pawn.health.hediffSet.GetPartHealth(brain) > 0f)
        {
            return true;  // Deathless + 大脑完好 → 可以替代死亡
        }
    }
    return false;
}
```

**三个条件**：
1. Biotech DLC激活
2. 拥有激活的Deathless基因
3. 大脑存在且未缺失且HP > 0

### 4.2 ForceDeathrestOrComa 执行流程

**位置**：`Verse.Pawn_HealthTracker.ForceDeathrestOrComa()` L789

```csharp
private void ForceDeathrestOrComa(DamageInfo? dinfo, Hediff hediff)
{
    if (pawn.CanDeathrest())
    {
        // 路径A：可以死眠 → 进入死眠 + 清空Hemogen
        SanguophageUtility.TryStartDeathrest(pawn, DeathrestStartReason.LethalDamage);
        GeneUtility.OffsetHemogen(pawn, -9999f);
    }
    else
    {
        // 路径B：不能死眠 → 进入再生昏迷
        SanguophageUtility.TryStartRegenComa(pawn, DeathrestStartReason.LethalDamage);
    }

    if (!Downed)
    {
        forceDowned = true;
        MakeDowned(dinfo, hediff);  // 强制倒地
    }
}
```

**在CheckForStateChange中的位置**：

```
ShouldBeDead() == true
  ├── ShouldBeDeathrestingOrInComa() == true
  │     └── ForceDeathrestOrComa() → 死眠/昏迷 + 强制倒地（不死亡）
  └── ShouldBeDeathrestingOrInComa() == false
        └── pawn.Kill() → 死亡
```

> **关键发现**：Deathless基因的保护有一个致命弱点——大脑被摧毁时保护失效。这就是为什么处决和倒地即死系统都通过摧毁大脑来绕过Deathless。

## 5. preventsDeath Hediff

### 5.1 在ShouldBeDead中的检查位置

**位置**：`Verse.Pawn_HealthTracker.ShouldBeDead()` L623

```csharp
public bool ShouldBeDead()
{
    if (Dead) return true;
```

### 5.2 原版使用

原版中`preventsDeath`的使用非常有限，主要出现在：
- 某些Anomaly DLC的特殊Hediff
- 模组扩展点——模组可通过XML设置`<preventsDeath>true</preventsDeath>`在HediffDef或HediffStage上

> **模组扩展点**：`preventsDeath`是最简单的"不死"实现——纯XML，无需C#。但要注意：它阻止所有死亡判定，包括ShouldBeDead的全部5个条件。

## 6. 爬行机制（Crawling）

### 6.1 CanCrawl 五个条件

**位置**：`Verse.Pawn_HealthTracker.CanCrawl` L119

```csharp
public bool CanCrawl
{
    get
    {
        if (!pawn.RaceProps.Humanlike) return false;                          // 条件1：人形生物
        if (!pawn.Awake()) return false;                                      // 条件2：清醒
        if (capacities.GetLevel(PawnCapacityDefOf.Manipulation) < 0.15f)     // 条件3：操作能力≥15%
            return false;
        if (pawn.ageTracker.AgeBiologicalYears < 8) return false;            // 条件4：年龄≥8岁
        if (hediffSet.AnyHediffPreventsCrawling) return false;               // 条件5：无阻止爬行Hediff
        return true;
    }
}
```

### 6.2 倒地ThinkTree行为

倒地Pawn的行为由专用ThinkTree `Downed`驱动：

```
ThinkTree_Downed
  ├── JobGiver_FleeDanger          ← 爬行逃离危险
  ├── JobGiver_PatientGoToBed      ← 玩家殖民者爬向床位
  ├── ThinkNode_CrawlInterrupt     ← 每隔几秒中断爬行（模拟挣扎）
  ├── 在床上时执行队列任务
  └── JobGiver_IdleForever         ← 无法爬行时永久待机
```

## 7. 恢复机制（Recovery）

### 7.1 MakeUndowned 触发与流程

**位置**：`Verse.Pawn_HealthTracker.MakeUndowned()` L883

**触发时机**：`CheckForStateChange()`中，当`Downed == true`且`ShouldBeDowned() == false`时自动调用。

```csharp
private void MakeUndowned(Hediff hediff)
{
    if (!Downed) return;

    pawn.pather?.StopDead();                    // 停止寻路
    pawn.jobs?.CheckForJobOverride();           // 检查任务覆盖
    healthState = PawnHealthState.Mobile;        // 状态恢复

    // 发送恢复消息（除非是死亡安息恢复）
    if (PawnUtility.ShouldSendNotificationAbout(pawn)
        && (hediff == null || hediff.def != HediffDefOf.Deathrest))
    {
        Messages.Message("MessageNoLongerDowned".Translate(...), pawn, MessageTypeDefOf.PositiveEvent);
    }

    PortraitsCache.SetDirty(pawn);
    GlobalTextureAtlasManager.TryMarkPawnFrameSetDirty(pawn);

    pawn.guest?.Notify_PawnUndowned();          // 通知访客系统
    pawn.GetLord()?.Notify_PawnUndowned(pawn);  // 通知Lord
}
```

**恢复条件**（ShouldBeDowned返回false）：
- 疼痛降至阈值以下（治疗、止痛药）
- 移动能力恢复（安装义肢）
- 意识恢复（麻醉消退）

### 7.2 救援与俘虏

倒地Pawn可被其他Pawn交互：
- **玩家殖民者倒地**：其他殖民者自动生成救援Job（`JobGiver_RescueNearby`）
- **敌对Pawn倒地**：可被俘虏（`JobDriver_TakeToBed` → 设为囚犯）
- **动物倒地**：可被驯养或屠宰

## 8. 处决机制（Execution）

### 8.1 ExecutionUtility.ExecutionInt 流程

**位置**：`RimWorld.ExecutionUtility.ExecutionInt()` L20

```csharp
private static void ExecutionInt(Pawn executioner, Pawn victim, ...)
{
    // 1. 生成血液污物
    int bloodCount = Max(RoundRandom(victim.BodySize * bloodPerWeight), 1);

    // 2. 选择切割部位
    BodyPartRecord cutPart = ExecuteCutPart(victim);  // 通常是颈部
    int partHealth = (int)victim.health.hediffSet.GetPartHealth(cutPart);

    // 3. 造成处决伤害
    DamageInfo damageInfo = new DamageInfo(
        DamageDefOf.ExecutionCut,
        Min(partHealth - 1, 1),  // 几乎致命但不立即杀死
        999f,                     // 护甲穿透=999（忽略所有护甲）
        ...
    );
    damageInfo.SetIgnoreArmor(true);

    // 4. Deathless基因特殊处理
    if (victim.genes?.HasActiveGene(GeneDefOf.Deathless) == true)
    {
        damageInfo.SetAmount(9999f);                  // 超量伤害
        damageInfo.SetAllowDamagePropagation(false);  // 不传播
    }

    victim.TakeDamage(damageInfo);

    // 5. 确保死亡
    if (!victim.Dead)
        victim.Kill(damageInfo);
}
```

**关键设计**：
- 护甲穿透999 + `IgnoreArmor=true`：双重保险忽略护甲
- Deathless基因：9999伤害摧毁大脑，绕过Deathless保护
- 最终保险：如果TakeDamage没杀死，强制`Kill()`
