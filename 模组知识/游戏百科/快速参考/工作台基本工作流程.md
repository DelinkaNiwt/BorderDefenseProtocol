# 工作台基本工作流程

**总览**：工作台（`Building_WorkTable`）是RimWorld生产系统的核心，完整工作流程经过6个阶段：**玩家添加配方**（ITab_Bills UI）→ **系统分配工作**（WorkGiver_DoBill扫描）→ **搜索材料**（TryFindBestBillIngredients）→ **搬运材料**（CollectIngredientsToils）→ **执行制造**（DoRecipeWork）→ **产品入库**（FinishRecipeAndStartStoringProduct）。整个流程涉及7个核心类协作。

## 1. 核心类架构总览

```
玩家UI层          数据层              调度层                执行层              产出层
──────────       ──────────         ──────────           ──────────         ──────────
ITab_Bills  →  BillStack/Bill  →  WorkGiver_DoBill  →  JobDriver_DoBill  →  GenRecipe
(添加配方)     (配方队列)         (扫描+找材料)        (搬运+制造)          (生成产品)
```

**7个核心类职责对比**：

| 核心类 | 职责 | 关键方法/字段 |
|--------|------|-------------|
| `Building_WorkTable` | 工作台实体，实现`IBillGiver` | `billStack`, `CurrentlyUsableForBills()`, `UsedThisTick()` |
| `BillStack` | 有序配方队列 | `bills`, `AddBill()`, `FirstShouldDoNow`, `AnyShouldDoNow` |
| `Bill` → 子类 | 配方实例（含玩家设置） | `recipe`, `ShouldDoNow()`, `ingredientFilter`, `repeatMode` |
| `ITab_Bills` | 配方管理UI面板 | 添加/删除/排序/配置配方 |
| `WorkGiver_DoBill` | 工作分配器（继承WorkGiver_Scanner） | `StartOrResumeBillJob()`, `TryFindBestBillIngredients()` |
| `JobDriver_DoBill` | Job执行器 | `MakeNewToils()`, `workLeft`, `ticksSpentDoingRecipeWork` |
| `GenRecipe` | 产品生成器 | `MakeRecipeProducts()`, `PostProcessProduct()` |

## 2. IBillGiver接口与工作台可用性

`IBillGiver`是工作台的核心接口，`Building_WorkTable`实现它：

```csharp
public interface IBillGiver {
    BillStack BillStack { get; }              // 配方队列
    IEnumerable<IntVec3> IngredientStackCells; // 材料放置格（=建筑占据格）
    bool CurrentlyUsableForBills();            // 当前是否可用
    bool UsableForBillsAfterFueling();         // 加燃料后是否可用
}
```

**工作台可用性判定**（`CurrentlyUsableForBills()`）：

| # | 条件 | 说明 |
|---|------|------|
| 1 | `UsableForBillsAfterFueling()` | 基础可用（已生成、未销毁、未被焚烧） |
| 2 | 电力检查 | 无CompPower或已通电或`unpoweredWorkTableWorkSpeedFactor > 0` |
| 3 | 燃料检查 | 无CompRefuelable或有燃料 |

> **关键认知**：`unpoweredWorkTableWorkSpeedFactor > 0`允许工作台在断电时以降低速度工作（如手动工作台），而非完全停止。

## 3. Bill体系（配方实例）

Bill是RecipeDef的运行时实例，携带玩家的所有配置。三个子类覆盖所有场景：

### Bill继承体系

```
Bill（抽象基类）
├── Bill_Production          ← 标准生产配方（电炉、裁缝台等）
│   └── Bill_ProductionWithUft ← 需要UnfinishedThing的长工期配方
│       └── Bill_Mech        ← 机械体孵化配方（Biotech）
├── Bill_Medical             ← 手术/医疗配方
└── Bill_Autonomous          ← 自主运行配方（无需殖民者操作）
```

### Bill核心字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `recipe` | RecipeDef | 关联的配方定义 |
| `suspended` | bool | 是否暂停 |
| `ingredientFilter` | ThingFilter | 玩家可调整的材料过滤器 |
| `ingredientSearchRadius` | float | 材料搜索半径（默认999） |
| `allowedSkillRange` | IntRange | 允许的技能等级范围 |
| `pawnRestriction` | Pawn | 指定执行者（可选） |
| `slavesOnly` / `mechsOnly` | bool | 限制执行者类型 |

### BillRepeatMode（三种重复模式）

| 模式 | defName | 行为 |
|------|---------|------|
| **指定次数** | `RepeatCount` | 制造N次后停止（默认模式，初始1次） |
| **无限制** | `Forever` | 永不停止，持续制造 |
| **目标数量** | `TargetCount` | 维持库存数量（低于目标时自动制造） |

**TargetCount模式**额外字段：`targetCount`（目标数量）、`pauseWhenSatisfied`（满足后暂停）、`unpauseWhenYouHave`（低于此数量时恢复）、`includeEquipped`（是否计入已装备的）、`hpRange`/`qualityRange`（计数过滤条件）。

### BillStoreMode（三种存储模式）

| 模式 | defName | 行为 |
|------|---------|------|
| **丢在地上** | `DropOnFloor` | 产品直接放在工作台旁 |
| **最佳储存区** | `BestStockpile` | 搬运到最佳可用储存区 |
| **指定储存区** | `SpecificStockpile` | 搬运到玩家指定的储存区/货架 |

### ShouldDoNow()判定逻辑

`Bill_Production.ShouldDoNow()`按序检查：
1. `suspended` → false
2. `repeatMode == Forever` → true
3. `repeatMode == RepeatCount` → `repeatCount > 0`
4. `repeatMode == TargetCount` → 当前库存 < `targetCount`（通过`RecipeWorkerCounter`计数）

┌──────────────────────────────────────────────────────────────────────┐

## 5. 材料搜索机制（WorkGiver_DoBill）

### 搜索流程

`TryFindBestBillIngredients()`的核心逻辑：

1. **收集候选材料**：在`ingredientSearchRadius`范围内，按距离排序所有可达+未禁止+可预留的Thing
2. **逐项匹配**：遍历`RecipeDef.ingredients`列表，对每个`IngredientCount`从候选中选取满足条件的材料
3. **两种匹配模式**：
   - `allowMixingIngredients=false`（默认）：每个IngredientCount只能用一种ThingDef满足（`TryFindBestBillIngredientsInSet_NoMix`）
   - `allowMixingIngredients=true`：可混合多种ThingDef满足同一IngredientCount（`TryFindBestBillIngredientsInSet_AllowMix`，烹饪配方常用）

### 搜索失败处理

找不到材料时，设置`bill.nextTickToSearchForIngredients = TicksGame + RandomInRange(500~600)`，避免每tick重复搜索。浮动菜单（右键菜单）中会显示缺失材料列表。

## 6. 制造执行（JobDriver_DoBill）

### Toil序列（7步）

| # | Toil | 说明 |
|---|------|------|
| 1 | `Notify_DoBillStarted` | 通知Bill开始 |
| 2 | `JumpIf(无材料→跳到步骤4)` | 无材料队列时跳过搬运 |
| 3 | `CollectIngredientsToils` | 逐个搬运材料到工作台（多次往返） |
| 4 | `GotoThing(工作台)` | 前往工作台交互格 |
| 5 | `MakeUnfinishedThingIfNeeded` | 长工期配方创建UnfinishedThing |
| 6 | `DoRecipeWork` | 核心制造循环（每tick扣减workLeft） |
| 7 | `FinishRecipeAndStartStoringProduct` | 生成产品并入库 |

### 工作速度计算公式

`DoRecipeWork`的`tickIntervalAction`中：

```
workSpeed = pawn.GetStatValue(recipe.workSpeedStat)     // 殖民者工作速度
          × workTable.GetStatValue(recipe.workTableSpeedStat)  // 工作台速度因子
workLeft -= workSpeed × delta                            // 每interval扣减
```

- `workSpeedStat`：殖民者的工作速度Stat（如`CookSpeed`、`GeneralLaborSpeed`）
- `workTableSpeedStat`：工作台的速度因子Stat（默认`WorkTableWorkSpeedFactor`，受设施加成影响）
- 当`workLeft ≤ 0`时，调用`ReadyForNextToil()`进入产品生成阶段

### 技能经验获取

- **使用UnfinishedThing的配方**：制造过程中持续获取经验（`0.1 × workSkillLearnFactor × delta`/interval）
- **不使用UnfinishedThing的配方**：完成时一次性获取经验（`ticksSpentDoingRecipeWork × 0.1 × workSkillLearnFactor`）

## 7. 产品生成（GenRecipe）

### MakeRecipeProducts流程

1. 计算效率：`efficiency = pawn.efficiencyStat × workTable.workTableEfficiencyStat`
2. 遍历`recipe.products`列表，对每个产品：
   - 创建Thing（如果MadeFromStuff则使用主要材料的Stuff）
   - `stackCount = Ceil(count × efficiency)`（效率影响产出数量）
   - 如果`useIngredientsForColor`，继承材料颜色
   - 记录成分（CompIngredients）
3. 处理`specialProducts`（Butchery屠宰/Smelted熔炼）
4. `PostProcessProduct()`：品质判定 → 风格应用 → 最小化（Minifiable）

### 品质判定

`PostProcessProduct()`中，如果产品有`CompQuality`：
- 调用`QualityUtility.GenerateQualityCreatedByPawn(worker, recipe.workSkill)`
- 品质由殖民者的`workSkill`等级+灵感+随机因素决定
- 技能越高，获得高品质的概率越大

### 产品存储

按`BillStoreMode`决定存储方式：
1. `DropOnFloor`：直接`GenPlace.TryPlaceThing`放在殖民者位置附近
2. `BestStockpile`：`StoreUtility.TryFindBestBetterStoreCellFor`找最佳储存格 → 殖民者搬运
3. `SpecificStockpile`：`StoreUtility.TryFindBestBetterStoreCellForIn`在指定储存区找格子 → 殖民者搬运

多产品时，仅第一个产品由殖民者搬运，其余直接丢在地上。

## 8. UnfinishedThing（长工期机制）

需要`unfinishedThingDef`的配方（如制造组件、武器、服装）使用`Bill_ProductionWithUft`子类：

| 维度 | 说明 |
|------|------|
| **创建时机** | `MakeUnfinishedThingIfNeeded` Toil中，材料搬运完成后 |
| **存储内容** | 材料列表 + 剩余工作量(workLeft) + 绑定的Bill和Worker |
| **中断恢复** | 殖民者被打断后，UnfinishedThing留在工作台上，下次可继续 |
| **绑定规则** | `BoundBill`和`BoundWorker`——同一个UnfinishedThing只能由原殖民者在原Bill上继续 |
| **长工期检查** | 工作超过3000 ticks后，每1000 ticks检查一次JobOverride（允许被更高优先级工作打断） |

## 9. 关键类速查表

| 需求 | 推荐方案 | 说明 |
|------|---------|------|
| 新增工作台 | XML继承`BenchBase` + `thingClass=Building_WorkTable` | 自动获得ITab_Bills配方系统 |
| 新增生产配方 | XML定义RecipeDef + `recipeUsers`或ThingDef.`recipes`关联 | 无需C# |
| 自定义产品生成 | 自定义RecipeWorker子类 + 重写`Notify_IterationCompleted()` | 需C# |
| 自定义材料计量 | 自定义IngredientValueGetter子类 | 需C#，原版仅Volume和Nutrition |
| 自定义产品计数 | 自定义RecipeWorkerCounter子类 | 用于TargetCount模式的库存计数 |
| 断电可工作的工作台 | 设置`unpoweredWorkTableWorkSpeedFactor` > 0 | 断电时以降低速度工作 |
| 工作台加热 | 设置`heatPerTickWhileWorking` | 工作时产热（如电炉） |
