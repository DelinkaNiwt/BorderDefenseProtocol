# 模组自定义派系与扩展模式

**总览**：社区模组通过**4种扩展模式**自定义派系——纯XML定义（FactionDef字段配置）、XML+DefModExtension（VEF框架扩展字段）、XML+自定义WorldObjectComp（定居点交互逻辑）、完整派系子系统（多派系+头衔+任务+文化协作）。本文分析3个代表性模组：机械族全面战争（双派系友敌分离+驯服系统）、闪耀毁灭者5（任务驱动单派系）、米莉拉（三派系体系化设计）。

```
┌─────────────────────────────────────────────────────────────────┐
│         模组自定义派系的4种扩展模式（按复杂度排列）              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  模式1：纯XML定义 ← 最简单，零C#                               │
│    └── FactionDef字段配置：敌对性/隐藏/袭击曲线/PawnGroupMaker  │
│                                                                 │
│  模式2：XML + DefModExtension ← VEF框架扩展                   │
│    └── FactionDefExtension附加生物群系/好感度/功能控制等字段    │
│                                                                 │
│  模式3：XML + 自定义WorldObjectComp ← 定居点交互逻辑           │
│    └── WorldObjectComp实现商队Gizmo/派系关系动态转换            │
│                                                                 │
│  模式4：完整派系子系统 ← 多类协作的复杂系统                    │
│    └── 多FactionDef+RoyalTitle+Quest+Culture+WorldComponent     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 1. 机械族全面战争(NCL/TW)——双派系友敌分离

### 1.1 派系架构

**双派系设计**：同一模组定义两个FactionDef，通过`permanentEnemy`区分阵营。

| 字段 | NCL_faction（友方） | NCL_factionEnemy（敌方） |
|------|-------------------|------------------------|
| **ParentName** | FactionBase | FactionBase |
| **permanentEnemy** | false | true |
| **hidden** | true | true |
| **humanlikeFaction** | false | false |
| **techLevel** | Archotech | Archotech |
| **requiredCountAtGameStart** | 1 | 1 |
| **raidsForbidden** | true | true |
| **autoFlee** | false | false |
| **earliestRaidDays** | 90 | 90 |
| **displayInFactionSelection** | false | false |

**设计要点**：
- 两个派系共用同一`factionNameMaker`（NamerFactionNyar → 固定名"NCL"），玩家感知为同一势力
- `hidden=true` + `displayInFactionSelection=false`：不在世界地图生成定居点，不在世界创建界面显示
- `raidsForbidden=true`：禁止常规袭击流程，所有战斗通过模组自定义事件触发
- `requiredCountAtGameStart=1`：强制在游戏开始时生成，确保派系始终存在
- `maxPawnCostPerTotalPointsCurve`极高上限（10000点对应10000cost），允许生成任意强度单位

### 1.2 ★ 驯服系统（CompTameableMech → SetFaction）

**核心机制**：通过`CompTameableMech`实现机械体从野生/敌方到玩家派系的转换。

**流程**：
```
Gizmo按钮（选中无派系机械体时显示）
  → 检查条件：Faction==null && IsMechanoid && Spawned
  → 检查资源：地图上ComponentIndustrial数量 ≥ componentsPerTame
  → 查找执行者：FindBestTamer()
  → 创建Job：TW_TameMech
    → Toil1: 检查背包零件
    → Toil2: 搜索最近零件
    → Toil3: 前往零件位置
    → Toil4: 拾取零件到背包
    → Toil5: 前往机械体
    → Toil6: 等待250 ticks（约4秒）
    → Toil7: 消耗零件 + 50%概率判定
      → 成功: Mech.SetFaction(Faction.OfPlayer) ← 关键调用
      → 失败: 重新入队TW_TameMech Job，自动重试
```

**`SetFaction()`的效果**：原版API，一次调用完成派系切换——AI行为、敌对关系、Gizmo显示、渲染颜色全部自动更新。

### 1.3 派系切换的模组意义

NCL/TW的双派系+驯服系统展示了一种**"同一势力，不同阵营"**的设计模式：
- 友方NCL派系的机械体可以被驯服后加入玩家
- 敌方NCL派系的机械体在特殊事件中作为敌人出现
- 两个派系的`pawnGroupMakers`可以配置不同的单位组合

## 2. 闪耀毁灭者5(GD5)——任务驱动单派系

### 2.1 派系定义

**单派系设计**：`BlackMechanoid`（黑色机械蜂巢），独立于原版Mechanoid派系。

| 字段 | BlackMechanoid | 原版Mechanoid（对比） |
|------|---------------|---------------------|
| **ParentName** | FactionBase | MechanoidFactionBase |
| **permanentEnemyToEveryoneExceptPlayer** | true | — |
| **permanentEnemy** | —（注释掉） | true |
| **hidden** | true | true |
| **humanlikeFaction** | false | false |
| **techLevel** | Ultra | Ultra |
| **requiredCountAtGameStart** | 1 | 1 |
| **earliestRaidDays** | 999999 | 45 |
| **raidCommonality** | 300→0, 700→1, 4000→2.6 | 300→0, 700→1, 4000→2.6 |

**关键设计差异**：
- `permanentEnemyToEveryoneExceptPlayer`：对所有派系敌对，但**对玩家不自动敌对**——通过任务系统动态控制关系
- `earliestRaidDays=999999`：实质禁止常规袭击（需等约2740年），所有战斗通过QuestScriptDef触发
- raidCommonality曲线与原版Mechanoid完全相同，但因earliestRaidDays极高而不会被常规袭击系统选中

### 2.2 任务驱动交互

GD5通过`MissionComponent`（WorldComponent）管理所有与黑色机械蜂巢的交互：

**任务类型**：
| QuestScriptDef | 类型 | 说明 |
|---------------|------|------|
| GD_Quest_SendCorpse | 合作 | 向蜂巢送交尸体换取奖励 |
| GD_Quest_HelpMech | 合作 | 帮助蜂巢防御，获得机械体援军 |
| GD_Quest_BuildCluster | 挑战 | 蜂巢在地图上建造机械集群 |
| GD_Quest_Cluster_L | 挑战 | 大型机械集群攻击 |
| GD_Quest_Cluster_Fortress | 挑战 | 三级要塞攻击 |
| GD_Quest_Drysea | 剧情 | 干海任务线 |
| GD_Quest_Militor | 剧情 | 守护Militor任务 |

**设计模式**：派系本身不主动袭击，所有交互通过任务系统调度——玩家可以选择合作或对抗。

### 2.3 PawnGroupMaker配置

GD5的BlackMechanoid派系仅配置1个Combat组，单位种类少但个体极强：

```xml
<pawnGroupMakers>
  <li>
    <kindDef>Combat</kindDef>
    <commonality>100</commonality>
    <options>
      <Mech_BlackScyther>1</Mech_BlackScyther>
      <Mech_BlackLancer>1</Mech_BlackLancer>
    </options>
  </li>
</pawnGroupMakers>
```

Boss级单位（如`Mech_BlackApocriton`，护甲2.0/2.0）通过任务脚本直接生成，不走PawnGroupMaker。

## 3. 米莉拉(Milira)——三派系体系化设计

### 3.1 三派系架构

Milira是分析的3个模组中派系系统最复杂的，定义了3个相互关联的FactionDef：

| 字段 | Milira_Faction | Milira_AngelismChurch | Milira_PlayerFaction |
|------|---------------|----------------------|---------------------|
| **label** | Sky Elves | Skyfeathered Church | Milira Ground Survey Station |
| **ParentName** | FactionBase | FactionBase | PlayerFactionBase |
| **isPlayer** | — | — | true |
| **techLevel** | Ultra | Spacer | Ultra |
| **humanlikeFaction** | 默认true | 默认true | 默认true |
| **hidden** | — | — | — |
| **permanentEnemy** | — | — | — |
| **敌对模式** | permanentEnemyToEveryoneExcept [Milira_PlayerFaction] | 无（可外交） | — |
| **requiredCountAtGameStart** | 1 | 1 | — |
| **leaderTitle** | President | Pontiff | — |
| **royalTitleTags** | — | Milira_ChurchTitle | — |
| **allowedCultures** | MiliraCulture | Milira_ChurchCulture | MiliraCulture |
| **Combat PawnGroupMaker数** | 16个 | 4个 | — |

### 3.2 Milira_Faction（天空精灵）

**核心特征**：
- `permanentEnemyToEveryoneExcept`列表仅含`Milira_PlayerFaction`——对所有其他派系永久敌对，但对Milira玩家派系友好
- 16个Combat PawnGroupMaker，按战术分类（全米莉拉/带大剑/士兵+战车/全骑士/全家福/双要塞/无畏等），`commonality`从20到80不等
- 混合编队：人形单位（Milira_Valkyr/Artemis/YoungWarrior/Breacher）+ 机械单位（Milian_Mechanoid系列Pawn/Knight/Bishop/Rook各级）
- 自定义贸易：`baseTraderKinds`含`Base_Milira_Standard`
- 自定义空投：`dropPodActive=Milira_ActiveDropPod`，`dropPodIncoming=Milira_DropPodIncoming`

### 3.3 ★ WorldObjectCompMiliraSettlement（定居点交互）

**机制**：通过`WorldObjectComp`在Milira定居点上添加商队Gizmo，实现派系关系动态转换。

```
商队到达Milira定居点
  → WorldObjectCompMiliraSettlement.GetCaravanGizmos()
  → 条件检查：ValidSettlement（派系==Milira_Faction）
              && FallenMilira != null（存在堕落天使Pawn）
              && caravan.ContainsPawn(FallenMilira)（商队携带该Pawn）
  → 显示Gizmo："交还堕落天使"
  → 执行：
    1. caravan.RemovePawn(FallenMilira) — 从商队移除
    2. turnToFriend_Pre = true — 设置友好标记
    3. CheckMiliraPermanentEnemyStatus() — 重新检查永久敌对状态
    4. GoodwillSituationManager.RecalculateAll() — 重算好感度
    5. SetRelation(baseGoodwill=-60, Neutral) — 设置为中立（-60好感度）
```

**设计意义**：这是一个**条件触发的派系关系转换**——玩家通过特定行为（交还堕落天使）将永久敌对派系转为中立，开启外交可能。

### 3.4 Milira_AngelismChurch（天羽教会）

**核心特征**：
- 人类派系（非机械体），Spacer科技等级
- `royalTitleTags: Milira_ChurchTitle`——完整的皇家头衔系统
- 自定义`RoyalTitleAwardWorker_ChurchTitle`授衔逻辑
- 头衔层级：Follower → Acolyte → ...（多级，类似原版Empire）
- `royalFavorLabel: Church Reputation`——自定义荣誉名称
- `xenotypeSet`配置：4% Neanderthal + 12% Hussar + 1% Genie
- 混合编队：教会专属人形单位（Church_Warrior/Trooper/Janissary/Knight/Cataphract）+ Milian机械体 + 原版佣兵/帝国单位
- `canSiege=true`，`canStageAttacks=true`——可执行围攻和分阶段进攻

### 3.5 Milira_PlayerFaction（玩家派系）

**核心特征**：
- 继承`PlayerFactionBase`，提供Milira主题的玩家开局
- `allowedCultures: MiliraCulture`——自定义文化
- `startingResearchTags: MiliraRefugeeStart`——自定义初始科技
- `backstoryFilters`限定`Milira_General`背景故事
- `apparelStuffFilter`限制服装材料（禁止Synthread/Hyperweave/贵金属）

## 4. VEF FactionDefExtension框架

Vanilla Expanded Framework提供`FactionDefExtension`（继承`DefModExtension`），是社区事实标准的派系扩展框架。通过在FactionDef的`modExtensions`中添加即可使用，无需自定义C#。

### 4.1 字段清单

| 类别 | 字段 | 类型 | 说明 |
|------|------|------|------|
| **定居点生成** | `hasCities` | bool | 是否生成城市定居点（默认true） |
| | `settlementGenerationSymbol` | string | 自定义定居点生成符号 |
| | `minDistanceToOtherSettlements` | float | 与其他定居点最小距离 |
| | `neverConnectToRoads` | bool | 不连接道路网络 |
| **生物群系限制** | `allowedBiomes` | List\<BiomeDef\> | 允许生成的生物群系白名单 |
| | `disallowedBiomes` | List\<BiomeDef\> | 禁止生成的生物群系黑名单 |
| | `requiredHillLevels` | List\<Hilliness\> | 要求的地形起伏等级 |
| | `spawnOnCoastalTilesOnly` | bool | 仅在沿海地块生成 |
| **初始好感度** | `startingGoodwillByFactionDefs` | List\<StartingGoodwillByFaction\> | 对特定派系的初始好感度范围 |
| **功能控制** | `excludeFromCommConsole` | bool | 从通讯台排除 |
| | `excludeFromQuests` | bool | 从任务系统排除 |
| | `allowedStrategies` | List\<RaidStrategyDef\> | 限制可用袭击策略 |
| **围攻参数** | `siegeParameterSetDef` | SiegeParameterSetDef | 自定义围攻参数集 |
| **强制派系** | `forcedFactionData` | ForcedFactionData | 世界生成/游戏中强制派系存在 |
| **其他** | `strangerInBlackReplacement` | PawnKindDef | 替换"黑衣人"事件的PawnKind |
| | `packAnimalTexNameSuffix` | string | 驮兽贴图后缀 |

### 4.2 ForcedFactionData子系统

VEF提供完整的派系强制存在机制：

| 字段 | 说明 |
|------|------|
| `requiredFactionCountAtWorldGeneration` | 世界生成时要求的派系数量 |
| `preventRemovalAtWorldGeneration` | 阻止玩家在世界创建时移除 |
| `requiredFactionCountDuringGameplay` | 游戏中要求的派系数量 |
| `forceAddFactionIfMissing` | 缺失时自动添加 |
| `forcePlayerToAddFactionIfMissing` | 缺失时强制玩家添加 |
| `factionDiscoveryMinimumDistanceFromPlayer` | 派系发现最小距离 |

### 4.3 Harmony补丁实现

VEF通过Harmony补丁将扩展字段注入原版流程：
- `Faction.TryMakeInitialRelationsWith` Postfix → 应用`startingGoodwillByFactionDefs`
- `TileFinder.RandomSettlementTileFor` → 应用生物群系/地形/海岸限制
- `SiteMakerHelper.FactionCanOwn` → 应用`excludeFromQuests`

## 5. 4种扩展模式对比

| 维度 | 纯XML定义 | XML+DefModExtension | XML+WorldObjectComp | 完整派系子系统 |
|------|----------|---------------------|---------------------|---------------|
| **C#需求** | 零 | 零（用VEF） | 少量 | 大量 |
| **代表模组** | 简单派系模组 | VFE系列 | GD5 | Milira |
| **可控字段** | FactionDef原版~40字段 | +15字段（VEF） | +自定义交互逻辑 | 无限制 |
| **敌对控制** | permanentEnemy/naturalEnemy | +startingGoodwillByFactionDefs | +动态关系转换 | +条件触发转换 |
| **袭击控制** | raidCommonality曲线 | +allowedStrategies | +任务驱动袭击 | +自定义事件系统 |
| **定居点** | settlementGenerationWeight | +生物群系/地形/距离限制 | +自定义WorldObjectComp | +自定义定居点类型 |
| **适用场景** | 简单敌对/友好派系 | 需要地理/外交控制 | 需要定居点交互 | 完整文明体系 |
| **开发成本** | 低 | 低 | 中 | 高 |

## 6. 与原版派系的核心差异

模组自定义派系能做到而原版不能的6点：

| # | 能力 | 原版限制 | 模组实现方式 |
|---|------|---------|-------------|
| 1 | **动态派系关系转换** | permanentEnemy锁死-100 | WorldObjectComp + CheckPermanentEnemyStatus() |
| 2 | **任务驱动交互** | 袭击由Storyteller自动调度 | earliestRaidDays=999999 + QuestScriptDef |
| 3 | **混合编队** | 人形/机械体分属不同派系 | 同一FactionDef的pawnGroupMakers混合人形+机械PawnKindDef |
| 4 | **派系切换** | Pawn派系固定 | CompTameableMech → SetFaction() |
| 5 | **自定义头衔体系** | 仅Empire有royalTitle | royalTitleTags + 自定义RoyalTitleAwardWorker |
| 6 | **生物群系限定** | 所有派系全球分布 | VEF allowedBiomes/disallowedBiomes |
