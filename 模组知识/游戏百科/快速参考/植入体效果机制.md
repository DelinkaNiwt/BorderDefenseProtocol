# 植入体效果机制

**总览**：植入体效果通过两条路径实现——**替换型**（Hediff_AddedPart）直接替换身体部位并设定部位效率，**附加型**（Hediff_Implant）附加在部位上通过stages属性修改器和comps组件提供效果。

## 两大类植入体对比

| 维度 | 替换型（AddedPart） | 附加型（Implant） |
|------|---------------------|-------------------|
| C#类 | `Hediff_AddedPart` | `Hediff_Implant` |
| XML模板 | `AddedBodyPartBase` | `ImplantHediffBase` |
| 核心行为 | PostAdd时调用`pawn.health.RestorePart()`恢复部位，再将子部位标记为MissingBodyPart | 仅验证Part不为null，无特殊逻辑（标记性基类） |
| 效果来源 | `addedPartProps.partEfficiency` 设定部位效率 | `stages`属性修改器 + `comps`组件 |
| 典型实例 | 义肢臂、仿生眼、超凡臂 | 心灵连接器、止痛器、学习辅助器 |

## 替换型三级效率体系

| 等级 | 代表 | partEfficiency | betterThanNatural | 效果描述 |
|------|------|---------------|-------------------|---------|
| 义肢级 | SimpleProstheticArm | 0.50 | false | 功能恢复但低于自然 |
| 仿生级 | BionicEye | 1.25 | true | 超越自然部位 |
| 超凡级 | ArchotechArm | 1.50 | true | 远超自然部位 |

## AddedBodyPartProps字段速查

| 字段 | 类型 | 默认值 | 作用 |
|------|------|--------|------|
| `partEfficiency` | float | 1f | 部位效率乘数，直接影响该部位相关能力值 |
| `solid` | bool | true | 是否为固体（固体部位不会流血） |
| `isGoodWeapon` | bool | false | 是否算作好武器（影响近战评估） |
| `betterThanNatural` | bool | false | 是否优于自然部位（影响AI和UI判断） |

## 效果实现机制总表

植入体效果通过三层机制实现，从最常用到最特殊：

### 层1：HediffStage配置（XML声明式，最常用）

| 机制 | 字段 | 实例 |
|------|------|------|
| 属性修改 | `statOffsets` / `statFactors` | MechlinkImplant: MechBandwidth+6; LearningAssistant: GlobalLearningFactor+0.2 |
| 疼痛修改 | `painFactor` / `painOffset` | Painstopper: painFactor=0（完全消除疼痛） |
| 部位效率偏移 | `partEfficiencyOffset` | Joywire: -0.20（降低脑效率，代价换快乐） |
| 能力值修改 | `capMods` | PawnCapacityModifier列表，直接修改移动、操作等能力 |
| 饥饿速率 | `hungerRateFactor` | ReprocessorStomach: 0.75（降低25%饥饿速率） |
| 休息速率 | `restFallFactor` | CircadianAssistant: 0.8（降低20%休息需求） |
| 疾病免疫 | `makeImmuneTo` | ReprocessorStomach: 免疫GutWorms |

### 层2：HediffComp组件（C#+XML组合，功能扩展）

| Comp类 | 效果 | 实例 |
|--------|------|------|
| `HediffComp_VerbGiver` | 赋予近战攻击 | ArchotechArm: 14钝器; SimpleProstheticArm: 8.2钝器 |
| `HediffComp_GiveAbility` | 赋予能力 | Psylink: 超能力系统 |
| `HediffComp_ReactOnDamage` | 对特定伤害反应 | CircadianAssistant: EMP→BrainShock; ReprocessorStomach: EMP→呕吐 |
| `HediffComp_ChangeImplantLevel` | 改变植入体等级 | 机械师升级植入体 |
| `HediffComp_LetterOnDeath` | 死亡时发信 | MechlinkImplant: 通知可回收 |

### 层3：HediffDef直接配置（特殊字段）

| 字段 | 效果 | 实例 |
|------|------|------|
| `renderNodeProperties` | 视觉渲染（贴图覆盖） | BionicEye: 仿生眼贴图 |
| `spawnThingOnRemoved` | 移除时掉落物品 | 所有植入体: 移除后掉落对应ThingDef物品 |

## 典型植入体XML解析

### 示例1：止痛器（Painstopper）—— 纯Stage效果
```xml
<HediffDef ParentName="ImplantHediffBase">
  <defName>Painstopper</defName>
  <hediffClass>Hediff_Implant</hediffClass>
  <stages>
    <li>
      <painFactor>0</painFactor>          <!-- 完全消除疼痛 -->
    </li>
  </stages>
</HediffDef>
```

### 示例2：仿生眼（BionicEye）—— 替换型 + 视觉渲染
```xml
<HediffDef ParentName="AddedBodyPartBase">
  <defName>BionicEye</defName>
  <hediffClass>Hediff_AddedPart</hediffClass>
  <addedPartProps>
    <partEfficiency>1.25</partEfficiency>  <!-- 125%视力效率 -->
    <betterThanNatural>true</betterThanNatural>
  </addedPartProps>
  <renderNodeProperties>...</renderNodeProperties>  <!-- 仿生眼贴图 -->
</HediffDef>
```

### 示例3：超凡臂（ArchotechArm）—— 替换型 + Comp近战攻击
```xml
<HediffDef ParentName="AddedBodyPartBase">
  <defName>ArchotechArm</defName>
  <hediffClass>Hediff_AddedPart</hediffClass>
  <addedPartProps>
    <partEfficiency>1.50</partEfficiency>  <!-- 150%操作效率 -->
    <betterThanNatural>true</betterThanNatural>
  </addedPartProps>
  <comps>
    <li Class="HediffCompProperties_VerbGiver">
      <tools>
        <li>
          <label>fist</label>
          <power>14</power>               <!-- 14点钝器伤害 -->
          <capacities><li>Blunt</li></capacities>
        </li>
      </tools>
    </li>
  </comps>
</HediffDef>
```

### 示例4：机械连接器（MechlinkImplant）—— 多层效果叠加
```xml
<HediffDef ParentName="ImplantHediffBase">
  <defName>MechlinkImplant</defName>
  <hediffClass>Hediff_Implant</hediffClass>
  <stages>
    <li>
      <statOffsets>
        <MechBandwidth>6</MechBandwidth>   <!-- Stage: 属性偏移 -->
      </statOffsets>
    </li>
  </stages>
  <comps>
    <li Class="HediffCompProperties_LetterOnDeath">  <!-- Comp: 死亡通知 -->
      ...
    </li>
  </comps>
  <spawnThingOnRemoved>MechlinkImplant</spawnThingOnRemoved>  <!-- Def: 移除掉落 -->
</HediffDef>
```

> **开发者要点**：
> 1. **优先用Stage配置**：大多数植入体效果（属性修改、疼痛、效率）只需XML中的`stages`字段，无需写C#代码
> 2. **Comp用于行为扩展**：需要赋予攻击、能力、事件反应等"行为"时，使用HediffComp组件
> 3. **替换型 vs 附加型选择**：如果植入体概念上是"替换身体部位"（如假肢、仿生器官），用`Hediff_AddedPart`；如果是"附加功能"（如芯片、连接器），用`Hediff_Implant`
> 4. **效果可叠加**：一个植入体可以同时使用Stage + Comp + Def字段，三层效果互不冲突
> 5. **EMP脆弱性**：通过`HediffComp_ReactOnDamage`实现，这是植入体的常见弱点设计模式
