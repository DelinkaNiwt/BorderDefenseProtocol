# 科研系统架构与研究流程

**总览**：科研系统是RimWorld的核心进度系统，控制建筑、配方、装备等内容的解锁。系统由三层架构组成：`ResearchProjectDef`（定义层，描述科研项目的所有属性）、`ResearchManager`（管理层，追踪进度和完成状态）、`JobDriver_Research`（执行层，Pawn在研究台上执行研究工作）。每个科研项目属于一个`ResearchTabDef`标签页，通过`prerequisites`形成有向无环图（DAG）结构的科研树。

## 1. 三层架构

| 层级 | 核心类 | 命名空间 | 职责 |
|------|--------|---------|------|
| **定义层** | `ResearchProjectDef : Def` | Verse | 描述科研项目的30+字段（成本、前置、科技等级、解锁条件等） |
| **管理层** | `ResearchManager : IExposable` | RimWorld | 追踪当前项目、进度字典、techprint计数、anomaly知识，执行完成逻辑 |
| **执行层** | `JobDriver_Research : JobDriver` | RimWorld | Pawn前往研究台→每tick贡献研究点数→4000 ticks后重新评估 |

## 2. 研究速度公式

`JobDriver_Research`的核心Toil每tick执行：

```
研究点数/tick = ResearchSpeed(Pawn) × ResearchSpeedFactor(研究台) × 0.00825 × difficulty.researchSpeedFactor / CostFactor(科技等级差)
```

**各因子详解**：

| 因子 | 来源 | 说明 |
|------|------|------|
| `ResearchSpeed` | Pawn Stat | 殖民者研究速度（受智力技能、特性、植入体影响） |
| `ResearchSpeedFactor` | 研究台 Stat | 研究台速度因子（Simple=0.75, HiTech=1.0） |
| `0.00825` | `ResearchPointsPerWorkTick` | 硬编码常量，将工作tick转换为研究点数 |
| `researchSpeedFactor` | 难度设置 | 难度倍率 |
| `CostFactor` | 科技等级差 | 玩家科技等级低于项目等级时的惩罚倍率 |

**CostFactor科技等级惩罚**：

```csharp
CostFactor = 1 + (min(projectTechLevel, Industrial) - factionTechLevel) × 0.5
```

| 玩家科技等级 | 研究Industrial项目 | 研究Spacer项目 |
|------------|-------------------|---------------|
| Neolithic | 1.5× (慢50%) | 1.5× (Spacer被cap到Industrial) |
| Medieval | 1.0× (无惩罚) | 1.0× |
| Industrial | 1.0× | 1.0× |

> **关键认知**：科技等级惩罚被cap在Industrial（值=4），即Spacer/Ultra/Archotech项目的CostFactor与Industrial项目相同——高科技项目的难度体现在baseCost而非CostFactor。

**研究台房间加成**：研究台在Laboratory房间中工作时获得加成（`workTableRoomRole=Laboratory`），不在实验室中则乘以`workTableNotInRoomRoleFactor=0.8`（减速20%）。

**书架加成**：研究台附近的书架（Bookcase）提供额外研究速度加成（通过`PlaceWorker_DrawLinesToBookcasesInRoom`可视化连接线）。

## 3. 两种研究台对比

| 维度 | SimpleResearchBench | HiTechResearchBench |
|------|--------------------|--------------------|
| **ResearchSpeedFactor** | 0.75 | 1.0 |
| **需要电力** | 否 | 是（250W） |
| **支持设施** | 无 | MultiAnalyzer |
| **材料** | 75 Stuff + 25 Steel | 150 Metal Stuff + 100 Steel + 10 组件 |
| **尺寸** | 3×2 | 5×2 |
| **解锁条件** | 无（游戏开始可建） | 需研究Microelectronics |
| **可研究项目** | 无`requiredResearchBuilding`的项目 | 所有项目（含需HiTech的高级项目） |

**MultiAnalyzer**：HiTech研究台的唯一可连接设施（`CompAffectedByFacilities`），某些高级科研项目通过`requiredResearchFacilities`要求连接MultiAnalyzer。

## 4. ResearchProjectDef核心字段分类

### 按功能分7大类

| # | 功能类别 | 关键字段 | 说明 |
|---|---------|---------|------|
| 1 | **基础信息** | `defName`, `label`, `description`, `baseCost` | 标识和基础成本 |
| 2 | **科研树位置** | `tab`, `researchViewX`, `researchViewY` | 标签页归属和UI坐标 |
| 3 | **前置条件** | `prerequisites`, `hiddenPrerequisites`, `techLevel` | 解锁前置和科技等级 |
| 4 | **研究台要求** | `requiredResearchBuilding`, `requiredResearchFacilities` | 需要的研究台类型和设施 |
| 5 | **DLC解锁条件** | `techprintCount`, `requiredAnalyzed`, `requiresMechanitor`, `knowledgeCategory`, `knowledgeCost` | DLC特殊前置 |
| 6 | **完成效果** | `researchMods`, `recalculatePower`, `discoveredLetterTitle`, `teachConcept` | 完成时的副作用 |
| 7 | **标签/分类** | `tags`, `heldByFactionCategoryTags`, `hideWhen` | 分类标签和可见性控制 |

## 5. 科研树结构

### 5.1 ResearchTabDef（标签页）

| defName | label | DLC | 特殊属性 |
|---------|-------|-----|---------|
| **Main** | Main | Core | 默认标签页，所有未指定tab的项目归入此页 |
| **Anomaly** | anomaly | Anomaly | `visibleByDefault=false`, `minMonolithLevelVisible=1` |

> **关键认知**：模组可通过定义新的`ResearchTabDef`创建独立科研标签页——UFHeavyIndustries（联合重工）就定义了`KT_Research`标签页。

### 5.2 前置条件体系

| 字段 | 类型 | 说明 |
|------|------|------|
| `prerequisites` | `List<ResearchProjectDef>` | 显式前置——UI中显示连线，必须全部完成 |
| `hiddenPrerequisites` | `List<ResearchProjectDef>` | 隐式前置——不显示连线，但必须完成 |
| `requiredByThis` | `List<ResearchProjectDef>` | 反向引用——被此项目依赖的项目列表 |

**CanStartNow判定链**：`PrerequisitesCompleted`（所有prerequisites+hiddenPrerequisites完成）→ `TechprintRequirementMet`（techprint数量足够）→ `AnalyzedThingsRequirementsMet`（分析物品完成）→ `InspectionRequirementsMet`（检查要求满足）→ `PlayerMechanitorRequirementMet`（机械师要求满足）。

### 5.3 TechLevel（7级科技等级）

```
TechLevel枚举（byte）：
  Undefined(0) → Animal(1) → Neolithic(2) → Medieval(3) → Industrial(4) → Spacer(5) → Ultra(6) → Archotech(7)
```

科技等级影响两个方面：
1. **研究速度**：玩家派系科技等级低于项目等级时，CostFactor增加（见第2节）
2. **项目分布**：原版科研树按科技等级从左到右排列，Neolithic→Industrial→Spacer→Ultra

## 6. DLC特殊解锁条件

### 6.1 Techprint系统（Royalty DLC）

| 字段 | 类型 | 说明 |
|------|------|------|
| `techprintCount` | int | 需要的techprint数量（1~2） |
| `techprintCommonality` | float | techprint在贸易中的出现概率 |
| `techprintMarketValue` | float | techprint的市场价值 |
| `heldByFactionCategoryTags` | List\<string\> | 持有techprint的派系类别标签（如"Empire"） |

**流程**：获取techprint物品 → `ResearchManager.ApplyTechprint()` → 计数+1 → 达到`techprintCount`后可开始研究。

### 6.2 requiredAnalyzed系统（Biotech DLC）

| 字段 | 类型 | 说明 |
|------|------|------|
| `requiredAnalyzed` | List\<ThingDef\> | 需要分析的物品列表（如SignalChip、PowerfocusChip） |

**流程**：获取特定物品 → 在研究台分析（`CompAnalyzableUnlockResearch`）→ 分析完成后解锁研究。用于机械师科技树（BasicMechtech→StandardMechtech→HighMechtech→UltraMechtech），每级需要分析更高级的芯片。

### 6.3 requiresMechanitor（Biotech DLC）

`requiresMechanitor=true`的项目需要殖民地中有机械师（拥有Mechlink植入体的Pawn）才能开始研究。

### 6.4 Anomaly知识系统（Anomaly DLC）

Anomaly DLC引入了独立于baseCost的知识研究系统：

| 字段 | 类型 | 说明 |
|------|------|------|
| `knowledgeCategory` | KnowledgeCategoryDef | 知识类别（Basic/Advanced） |
| `knowledgeCost` | float | 知识成本（替代baseCost） |

**两个KnowledgeCategoryDef**：

| defName | 说明 | overflowCategory |
|---------|------|-----------------|
| **Basic** | 基础知识 | 无 |
| **Advanced** | 高级知识 | Basic（溢出到基础） |

**知识获取**：通过研究异常实体（`CompStudiable`）获得知识点 → `ResearchManager.ApplyKnowledge(category, amount)` → 分配到当前选择的该类别项目。

**溢出机制**：Advanced知识项目完成后，多余的知识点溢出到Basic类别的当前项目。如果没有选择Advanced项目，Advanced知识也会溢出到Basic。

**双轨并行**：玩家可同时选择一个Basic项目和一个Advanced项目，两者独立进度。

## 7. ResearchManager完成流程

`FinishProject()`执行以下步骤：

1. **递归完成前置**：如果前置未完成，先递归完成所有前置
2. **补齐techprint**：如果techprint不足，自动补齐
3. **补齐分析**：如果requiredAnalyzed未完成，强制完成分析
4. **设置进度**：`progress[proj] = baseCost`（标记完成）
5. **记录故事**：`TaleRecorder.RecordTale(FinishedResearchProject)`
6. **应用ResearchMod**：`ReapplyAllMods()`——调用所有已完成项目的`researchMods`
7. **重算电力**：如果`recalculatePower=true`，重算所有电力设备
8. **完成对话框**：显示研究完成弹窗
9. **发现信件**：如果有`discoveredLetterTitle`，发送信件
10. **教学概念**：如果有`teachConcept`，触发教学
11. **清除当前项目**：`currentProj = null`
12. **通知解锁**：遍历`UnlockedDefs`，调用`ThingDef.Notify_UnlockedByResearch()`
13. **发送信号**：`SendSignal("ResearchCompleted")`
