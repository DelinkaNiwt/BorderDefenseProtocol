# 事件系统大类与触发架构

**总览**：RimWorld的事件系统采用三层架构——`IncidentDef`（定义层，50+字段，描述事件的类别、触发条件、信件配置）、`IncidentWorker`（执行层，60+子类，实现事件的具体逻辑）、`Storyteller` + `StorytellerComp`（调度层，21个Comp子类，每1000 ticks决定触发哪些事件）。事件通过13个`IncidentCategoryDef`分类，Storyteller按类别权重随机选择，再从该类别中按`baseChance`加权选择具体事件。

## 一、13个IncidentCategoryDef总表

所有类别定义在`Core/Defs/Misc/IncidentCategoryDefs/IncidentCategoryDefs.xml`中：

| # | defName | needsParmsPoints | canUseAnomalyChance | tale | 说明 |
|---|---------|-----------------|--------------------|----- |------|
| 1 | **Special** | false | false | — | 特殊事件（不参与常规随机选择） |
| 2 | **Misc** | true | false | — | 杂项事件（好事/中性/小坏事） |
| 3 | **ThreatSmall** | true | true | — | 小型威胁（单体攻击、小群敌人） |
| 4 | **ThreatBig** | true | true | MajorThreat | 大型威胁（袭击、虫巢、机械集群） |
| 5 | **FactionArrival** | false | false | — | 派系到访（友好/中立Pawn到来） |
| 6 | **OrbitalVisitor** | false | false | — | 轨道访客（轨道贸易商） |
| 7 | **ShipChunkDrop** | false | false | — | 飞船碎片坠落 |
| 8 | **DiseaseHuman** | false | false | — | 人类疾病 |
| 9 | **DiseaseAnimal** | false | false | — | 动物疾病 |
| 10 | **AllyAssistance** | false | false | — | 盟友援助 |
| 11 | **EndGame_ShipEscape** | false | false | — | 终局飞船逃离 |
| 12 | **GiveQuest** | true | false | — | 任务生成 |
| 13 | **DeepDrillInfestation** | true | false | — | 深钻虫巢（深钻触发） |

**关键字段说明**：
- `needsParmsPoints`：为true时，Storyteller需要计算威胁点数（`IncidentParms.points`）才能触发该类别事件
- `canUseAnomalyChance`：为true时，Anomaly DLC可以将该类别事件替换为异常事件
- `tale`：触发时记录的故事片段（仅ThreatBig记录MajorThreat）

## 二、按功能分类的事件大类

将13个IncidentCategoryDef归纳为6大功能类别：

### 1. 威胁类（ThreatBig / ThreatSmall）

最核心的事件类别，驱动游戏的战斗和生存压力。

| 类别 | 典型事件 | 特征 |
|------|---------|------|
| **ThreatBig** | 敌人袭击(RaidEnemy)、虫巢(Infestation)、猎杀兽群(ManhunterPack)、机械集群(MechCluster)、坠毁飞船(CrashedShipPart) | 需要威胁点数、记录MajorThreat故事、Storyteller保证最大间隔 |
| **ThreatSmall** | 短路(ShortCircuit)、作物枯萎(CropBlight)、单体动物发狂(AnimalInsanitySingle) | 需要威胁点数但规模较小 |

**Anomaly DLC威胁**（`canUseAnomalyChance=true`）：
- ThreatBig：蹒跚者突袭(ShamblerAssault)、心灵仪式围攻(PsychicRitualSiege)、仇恨吟唱者(HateChanters)、食人魔突袭(GorehulkAssault)、奇美拉突袭(ChimeraAssault)、吞噬者突袭(DevourerAssault)
- ThreatSmall：食尸鬼攻击(GhoulAttack)、小型蹒跚者群(SmallShamblerSwarm)

### 2. 杂项类（Misc）

涵盖好事、中性事件和小型坏事，是事件多样性的主要来源。

| 子类 | 典型事件 |
|------|---------|
| **好事** | 资源舱坠落(ResourcePodCrash)、心灵安抚(PsychicSoothe)、自驯(SelfTame)、仙馐丛生(AmbrosiaSprout)、农场动物加入(FarmAnimalsWanderIn) |
| **中性** | 独角兽经过(ThrumboPasses)、兽群迁徙(HerdMigration)、流浪者加入(WandererJoin)、难民舱坠落(RefugeePodCrash) |
| **小坏事** | 赎金要求(RansomDemand)、心灵无人机(PsychicDrone) |

### 3. 派系/贸易类（FactionArrival / OrbitalVisitor / AllyAssistance）

与派系交互相关的事件。

| 类别 | 典型事件 | 说明 |
|------|---------|------|
| **FactionArrival** | 贸易商队到访(TraderCaravanArrival)、旅行者经过(TravelerGroup) | 地面派系到访 |
| **OrbitalVisitor** | 轨道贸易商(OrbitalTraderArrival) | 轨道贸易 |
| **AllyAssistance** | 盟友军事援助 | 盟友派系支援 |

### 4. 疾病类（DiseaseHuman / DiseaseAnimal）

疾病事件独立于威胁类别，有专门的`StorytellerComp_Disease`调度。

| 类别 | 典型事件 | 说明 |
|------|---------|------|
| **DiseaseHuman** | 流感(Flu)、瘟疫(Plague)、疟疾(Malaria)、肌肉寄生虫(MuscleParasites) | 人类殖民者疾病 |
| **DiseaseAnimal** | 动物流感、动物瘟疫 | 动物疾病 |

### 5. 任务类（GiveQuest / EndGame_ShipEscape）

任务生成事件，通过`IncidentWorker_GiveQuest`桥接到Quest系统。

| 类别 | 典型事件 | 说明 |
|------|---------|------|
| **GiveQuest** | 各种QuestScriptDef驱动的任务 | 需要威胁点数，桥接Quest系统 |
| **EndGame_ShipEscape** | 飞船逃离终局事件 | 特殊终局触发 |

### 6. 特殊/独立类（Special / ShipChunkDrop / DeepDrillInfestation）

不参与常规随机选择的特殊事件。

| 类别 | 典型事件 | 说明 |
|------|---------|------|
| **Special** | 游戏开始事件、叙事者特殊事件 | 由`StorytellerComp_ClassicIntro`等专用Comp触发 |
| **ShipChunkDrop** | 飞船碎片坠落 | 由`StorytellerComp_ShipChunkDrop`专用Comp触发 |
| **DeepDrillInfestation** | 深钻虫巢 | 由`StorytellerComp_DeepDrillInfestation`专用Comp触发 |

### IncidentWorker核心API

```csharp
public class IncidentWorker {
    public IncidentDef def;

    // 能否触发（前置条件检查）
    public bool CanFireNow(IncidentParms parms);
    protected virtual bool CanFireNowSub(IncidentParms parms);  // 子类重写

    // 触发概率因子
    public virtual float ChanceFactorNow(IIncidentTarget target);

    // 执行事件（入口）
    public bool TryExecute(IncidentParms parms);
    protected virtual bool TryExecuteWorker(IncidentParms parms);  // 子类重写

    // 发送信件
    protected void SendStandardLetter(...);
}
```

## 六、Storyteller调度架构

### 主循环流程

```
Storyteller.StorytellerTick()                    ← 每1000 ticks执行
  │
  ├── incidentQueue.IncidentQueueTick()          ← 处理排队事件
  │
  └── MakeIncidentsForInterval()                 ← 生成本轮事件
      │
      ├── 遍历所有 StorytellerComp:
      │   └── comp.MakeIntervalIncidents(target) ← 每个Comp独立生成
      │       │
      │       ├── minDaysPassed检查
      │       ├── allowedTargetTags / disallowedTargetTags过滤
      │       └── yield return FiringIncident(def, comp, parms)
      │
      ├── 遍历所有进行中Quest:
      │   └── IIncidentMakerQuestPart.MakeIntervalIncidents()
      │
      └── TryFire(FiringIncident)                ← 执行事件
          ├── allowBigThreats检查
          ├── Anomaly金属地狱冷却检查
          └── incident.def.Worker.TryExecute(parms)
```

### StorytellerComp_RandomMain核心逻辑

这是最重要的StorytellerComp——负责大多数随机事件的生成：

```csharp
// StorytellerComp_RandomMain.MakeIntervalIncidents
// 1. MTB检查：Rand.MTBEventOccurs(mtbDays, 60000f, 1000f)
// 2. 选择类别：ChooseRandomCategory()
//    - 如果距上次ThreatBig超过maxThreatBigIntervalDays → 强制ThreatBig
//    - 否则按categoryWeights加权随机
// 3. 生成参数：GenerateParms(category, target)
//    - points = StorytellerUtility.DefaultParmsNow() × randomPointsFactorRange
// 4. 选择事件：TrySelectRandomIncident(UsableIncidentsInCategory())
//    - 按baseChance × ChanceFactorNow × 人口因子 加权随机
// 5. 返回FiringIncident
```

**categoryWeights配置**（在StorytellerDef XML中）：
- 每个叙事者（Cassandra/Phoebe/Randy）有不同的类别权重配置
- 典型权重比：ThreatBig > Misc > ThreatSmall > FactionArrival > ...
