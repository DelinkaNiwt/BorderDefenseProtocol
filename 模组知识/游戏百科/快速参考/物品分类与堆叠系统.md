# 物品分类与堆叠系统

**总览**：RimWorld的物品分类采用**双层体系**——底层`ThingCategory`枚举（11值）定义Thing的大类归属（Pawn/Item/Building等），上层`ThingCategoryDef`树（60+节点）提供面向玩家的细粒度分类（食物/原材料/武器/服装等）。堆叠系统通过3层判定链（Thing基础→ThingWithComps Comp→TransferableUtility远行队）决定物品能否合并，只有`ThingCategory.Item`类的物品才能参与堆叠。

## 1. 双层分类体系

### ThingCategory枚举（11值）

`Verse.ThingCategory : byte`——定义Thing的底层大类，决定Thing在游戏引擎中的基本行为。

| # | 枚举值 | 说明 | 可堆叠 |
|---|--------|------|--------|
| 0 | `None` | 未分类 | ✗ |
| 1 | `Pawn` | 生物/人形/机械体 | ✗ |
| 2 | `Item` | **物品（含装备）** | **✓** |
| 3 | `Building` | 建筑/家具/工作台 | ✗ |
| 4 | `Plant` | 植物 | ✗ |
| 5 | `Projectile` | 投射物 | ✗ |
| 6 | `Filth` | 污物 | ✗ |
| 7 | `Gas` | 气体 | ✗ |
| 8 | `Attachment` | 附着物 | ✗ |
| 9 | `Mote` | 视觉粒子 | ✗ |
| 10 | `Ethereal` | 虚体 | ✗ |
| 11 | `PsychicEmitter` | 心灵发射器 | ✗ |

> **关键认知**：`CanStackWith()`的首个检查就是`def.category == ThingCategory.Item`。武器、服装、原材料、药品等在枚举层面都是`Item`，区分它们的是`ThingCategoryDef`树。

### ThingCategoryDef树（60+节点）

`Verse.ThingCategoryDef`——面向玩家的树形分类系统，用于存储区过滤器、贸易界面、资源面板等UI。

**核心字段**：
- `parent: ThingCategoryDef`——父类别
- `childCategories`——子类别列表（运行时填充）
- `childThingDefs`——直属物品列表（运行时填充）
- `resourceReadoutRoot: bool`——是否在资源面板显示（仅Root的直接子节点为true）

**两层关系**：ThingDef同时拥有`category`（枚举）和`thingCategories`（ThingCategoryDef列表）。枚举决定引擎行为（能否堆叠、能否建造等），ThingCategoryDef决定UI分类（存储区过滤、贸易界面等）。

## 3. 非装备物品6大类详解

### 3.1 Foods（食物）

| 子分类 | 典型物品 | stackLimit | 关键特性 |
|--------|---------|------------|---------|
| FoodMeals | 简单餐(10)、精致餐(10)、豪华餐(10)、营养膏(10) | 10 | 有CompIngredients（成分不同不可堆叠）、会腐烂 |
| MeatRaw | 人肉、虫肉、鸡肉、牛肉等 | 75 | 按动物种类区分ThingDef，会腐烂 |

| AnimalProductRaw | 牛奶(75)、蛋类(25) | 25~75 | 受精蛋可孵化 |

**食物核心机制**：
- **营养值**（`nutritionValue`）：进食时恢复的Food Need值
- **腐烂**（`CompRottable`）：有`daysToRotStart`天数后开始腐烂，冷藏可延缓
- **成分追踪**（`CompIngredients`）：熟食记录原材料，成分不同的同类食物不可堆叠

## 5. Stuff材料系统

### StuffCategoryDef（5个材料类别）

| # | 类别 | defName | 典型材料 | 影响属性 |
|---|------|---------|---------|---------|
| 1 | **金属** | Metallic | Steel, Plasteel, Gold, Silver, Uranium | 高耐久、高护甲、高锐利 |
| 2 | **木质** | Woody | WoodLog | 中等耐久、可燃、低护甲 |
| 3 | **石质** | Stony | Granite/Marble/Limestone/Slate/Sandstone, Jade | 高耐久、高钝器护甲、重 |
| 4 | **织物** | Fabric | Cloth, Devilstrand, Synthread, Hyperweave | 低耐久、保暖、轻 |
| 5 | **皮革** | Leathery | 各种动物皮革（30+种） | 中等耐久、保暖、轻 |

### 材料对物品的影响

- **Stuff影响堆叠**：`CanStackWith()`检查`Stuff == other.Stuff`，不同材料的同类物品不可堆叠
- **属性修改**：材料通过`StuffProperties.statOffsets`和`statFactors`修改物品属性
- **外观**：材料决定物品颜色（`stuffProps.color`）
- **ThingDef.stuffCategories**：定义物品可接受的材料类别列表
- **ThingDef.costStuffCount**：制造时消耗的材料数量

## 6. 堆叠系统完整分析

### 3层判定链

物品能否堆叠由3层判定依次检查，任一层返回false即不可堆叠：

```
第1层：Thing.CanStackWith(other)          — 基础检查
  ├── 未被摧毁（Destroyed == false）
  ├── def.category == ThingCategory.Item   — 只有Item类可堆叠
  ├── 不是圣物（IsRelic == false）
  ├── 同一个ThingDef（def == other.def）
  └── 同一种材料（Stuff == other.Stuff）

第2层：ThingWithComps.CanStackWith(other)  — Comp扩展检查
  └── 遍历所有Comp的AllowStackWith()
      ├── CompQuality：品质不同 → false
      ├── CompArt：有艺术品属性 → false
      └── CompIngredients：成分不同 → false

第3层：TransferableUtility.CanStackTogether() — 远行队/贸易额外检查
  ├── Genepack：基因集合+标签必须相同
  └── Xenogerm：基因集合+标签必须相同
```

### 堆叠阻断因素完整表

| # | 阻断因素 | 判定层 | 判定代码 | 说明 |
|---|---------|--------|---------|------|
| 1 | **category≠Item** | Thing基础 | `def.category != ThingCategory.Item` | 建筑、Pawn、植物等不可堆叠 |
| 2 | **已被摧毁** | Thing基础 | `Destroyed` | 已摧毁的物品不可堆叠 |
| 3 | **是圣物** | Thing基础 | `this.IsRelic()` | Ideology圣物不可堆叠 |
| 4 | **不同ThingDef** | Thing基础 | `def != other.def` | 不同种类物品不可堆叠 |
| 5 | **不同Stuff材料** | Thing基础 | `Stuff != other.Stuff` | 不同材料的同类物品不可堆叠 |
| 6 | **不同品质** | CompQuality | `qualityInt != other.qualityInt` | 品质不同不可堆叠 |
| 7 | **有艺术品属性** | CompArt | 有CompArt且已初始化 | 艺术品不可堆叠 |
| 8 | **不同成分** | CompIngredients | 原材料列表不同 | 不同肉类做的饭不可堆叠 |
| 9 | **不同基因集** | TransferableUtility | Genepack/Xenogerm基因不同 | 远行队/贸易中额外检查 |
| 10 | **Book** | Book子类 | `Book.CanStackWith`返回false | 书籍永远不可堆叠 |

### stackLimit典型值表

| stackLimit | 物品类型 | 典型实例 |
|------------|---------|---------|
| **1** | 装备、身体部件、特殊物品 | 武器、服装、义肢、训练器、书籍、碎块、尸体 |
| **5** | 少量药物 | 烟叶 |
| **10** | 熟食、药物 | 简单餐、精致餐、Yayo、Go-juice、Luciferium |
| **15** | 高级药品 | 闪光药品 |
| **25** | 普通药品、弹药、蛋类、药物 | 草药、工业药品、迫击炮弹、啤酒 |
| **50** | 组件 | 工业组件、高级组件 |
| **75** | 大宗原材料、食材 | 钢铁、木材、石砖、布料、皮革、生肉、蔬菜 |

> **规律**：stackLimit与物品的"独特性"成反比——越独特（有品质、有特殊属性）的物品stackLimit越低，越通用（原材料、食材）的物品stackLimit越高。

### TryAbsorbStack合并机制

当两个可堆叠物品合并时，`Thing.TryAbsorbStack(other, respectStackLimit)`执行以下逻辑：

```csharp
// 源码：Verse.Thing.TryAbsorbStack
public virtual bool TryAbsorbStack(Thing other, bool respectStackLimit) {
    if (!CanStackWith(other)) return false;

    int num = ThingUtility.TryAbsorbStackNumToTake(this, other, respectStackLimit);

    // HitPoints加权平均
    if (def.useHitPoints) {
        HitPoints = Mathf.CeilToInt(
            (float)(HitPoints * stackCount + other.HitPoints * num)
            / (float)(stackCount + num)
        );
    }

    stackCount += num;
    other.stackCount -= num;

    if (other.stackCount <= 0) {
        other.Destroy();
        return true;  // 完全吸收
    }
    return false;  // 部分吸收（受stackLimit限制）
}
```

**HitPoints加权平均公式**：
```
新HitPoints = ⌈(自身HP × 自身数量 + 对方HP × 吸收数量) / (自身数量 + 吸收数量)⌉
```

**关键行为**：
- `respectStackLimit=true`时，吸收数量受`def.stackLimit`限制
- `respectStackLimit=false`时，可超过stackLimit（如远行队打包）
- 合并后HitPoints取加权平均（向上取整），不是简单相加
- 对方stackCount归零时自动Destroy
