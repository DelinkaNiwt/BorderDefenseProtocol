# 袭击流程与结束条件

**总览**：袭击（Raid）是RimWorld最核心的威胁事件，由`IncidentWorker_Raid`（抽象基类）和`IncidentWorker_RaidEnemy`（敌对袭击实现）驱动。完整流程分为三大阶段——**信息生成**（10步解析派系/策略/到达/点数/Pawn）、**执行与通知**（信件/Lord创建/战利品）、**行为与结束**（Lord状态机驱动袭击者行为，7种结束条件）。

## 一、袭击执行完整流程（10步）

### 流程总图

```
Storyteller.StorytellerTick() → StorytellerComp选择RaidEnemy事件
  → IncidentWorker_RaidEnemy.TryExecuteWorker(parms)
    → TryGenerateRaidInfo(parms, out pawns)  ← 10步信息生成
      │
      ├── ①ResolveRaidPoints          ← 解析威胁点数
      ├── ②TryResolveRaidFaction      ← 选择敌对派系
      ├── ③ResolveRaidStrategy        ← 选择袭击策略
      ├── ④TryResolveRaidArriveMode   ← 选择到达模式
      ├── ⑤ResolveRaidAgeRestriction  ← 解析年龄限制(Biotech)
      ├── ⑥TryGenerateThreats        ← 策略特定威胁生成
      ├── ⑦TryResolveRaidSpawnCenter  ← 确定生成位置
      ├── ⑧AdjustedRaidPoints        ← 多层点数调整
      ├── ⑨GeneratePawns / SpawnThreats ← 生成袭击者
      └── ⑩PostProcessSpawnedPawns    ← 后处理
    │
    ├── GenerateRaidLoot              ← 生成战利品
    ├── SendStandardLetter            ← 发送信件
    └── MakeLords                     ← 创建Lord(行为控制)
```

### 10步详解

#### ①解析威胁点数（ResolveRaidPoints）
- **源码**：`IncidentWorker_Raid.TryGenerateRaidInfo` L73
- **逻辑**：使用`IncidentParms.points`（由Storyteller通过`StorytellerUtility.DefaultThreatPointsNow()`计算）
- **影响因素**：殖民者数量、财富值、难度系数、叙事者配置

#### ②选择敌对派系（TryResolveRaidFaction）
- **源码**：`IncidentWorker_RaidEnemy.TryResolveRaidFaction` → `FactionCanBeGroupSource` L9-41
- **选择方式**：`PawnGroupMakerUtility.TryGetRandomFactionForCombatPawnGroupWeighted()`加权随机
- **验证条件**（全部满足才可选）：
  - 派系必须敌对玩家（`HostileTo(Faction.OfPlayer)`）
  - 派系未被停用
  - 满足`earliestRaidDays`要求（L20）
  - 通过`raidArrivalLayerWhitelist/Blacklist`检查（L24-30）
  - 至少有一个可用的RaidStrategy（L34）

#### ③选择袭击策略（ResolveRaidStrategy）
- **源码**：`IncidentWorker_RaidEnemy.ResolveRaidStrategy` L92-119
- **默认groupKind**：`PawnGroupKindDefOf.Combat`
- **选择方式**：从所有`RaidStrategyDef`中按权重随机
- **权重计算**：`def.Worker.SelectionWeightForFaction(map, faction, points)`
  - 基础权重：`selectionWeightPerPointsCurve`按点数查曲线
  - 派系特殊权重：`selectionWeightCurvesPerFaction`覆盖
- **失败回退**：默认为`RaidStrategyDefOf.ImmediateAttack`（L100）

#### ④选择到达模式（TryResolveRaidArriveMode）
- **源码**：`IncidentWorker_Raid.TryResolveRaidArriveMode` L25-42
- **选择范围**：策略的`arriveModes`列表
- **权重计算**：`ModeWeight(parms, mode)`
  - 基础权重：`selectionWeightCurve`按点数查曲线
  - 空间地图特殊权重：`minSpaceSelectionWeight`（L44-51）
- **失败回退**：默认为`PawnsArrivalModeDefOf.EdgeWalkIn`（L58）

#### ⑤解析年龄限制（ResolveRaidAgeRestriction）
- **源码**：`IncidentWorker_Raid.TryGenerateRaidInfo` L84
- **条件**：Biotech DLC启用时，有概率生成儿童袭击者（L77-80）
- **影响**：`RaidAgeRestrictionDef.threatPointsFactor`调整点数

#### ⑥策略特定威胁生成（TryGenerateThreats）
- **源码**：`parms.raidStrategy.Worker.TryGenerateThreats(parms)` L87
- **用途**：围攻策略在此阶段生成炮台等特殊威胁
- **大多数策略**：返回true但不做额外操作

#### ⑦确定生成位置（TryResolveRaidSpawnCenter）
- **源码**：`parms.raidArrivalMode.Worker.TryResolveRaidSpawnCenter(parms)` L89
- **逻辑**：由到达模式Worker决定——边缘步行选地图边缘，空投选目标区域

#### ⑧多层点数调整（AdjustedRaidPoints）
- **源码**：`IncidentWorker_Raid.AdjustedRaidPoints` L191-211
- **调整链**（连乘）：
  1. 到达模式点数曲线：`raidArrivalMode.pointsFactorCurve`
  2. 策略点数曲线：`raidStrategy.pointsFactorCurve`
  3. 年龄限制因子：`ageRestriction.threatPointsFactor`
  4. 地图层因子：`tile.LayerDef.raidPointsFactor`（Odyssey）
- **下限保护**：`max(调整后点数, raidStrategy.Worker.MinimumPoints() × 1.05)`

#### ⑨生成袭击者Pawn
- **源码**：`IncidentWorker_Raid.TryGenerateRaidInfo` L97-115
- **两条路径**：
  - **策略自定义生成**：`parms.raidStrategy.Worker.SpawnThreats(parms)`返回非null时使用
  - **标准生成**：`PawnGroupMakerUtility.GeneratePawns(groupParms)`
    1. 从`faction.def.pawnGroupMakers`中选择匹配`groupKind`的PawnGroupMaker
    2. 从`options`列表中按权重+点数成本选择PawnKindDef
    3. 使用`PawnGenerator.GeneratePawn(request)`生成每个Pawn
    4. 调用`parms.raidArrivalMode.Worker.Arrive(pawns, parms)`放置Pawn

#### ⑩后处理（PostProcessSpawnedPawns）
- **源码**：`IncidentWorker_Raid.TryGenerateRaidInfo` L117
- **操作**：设置Pawn的额外属性、标记等

## 二、IncidentWorker_RaidEnemy特有逻辑

### 战利品生成（GenerateRaidLoot）
- **源码**：`IncidentWorker_RaidEnemy.GenerateRaidLoot` L155-173
- **仅敌对袭击**：友军支援不生成战利品
- **价值计算**：
  ```
  lootValue = raidLootValueFromPointsCurve(points)
             × raidStrategy.raidLootValueFactor
             × difficulty.EffectiveRaidLootPointsFactor
  ```
- **生成器**：`faction.def.raidLootMaker`（ThingSetMakerDef）
- **分配**：通过`RaidLootDistributor`分配给袭击者携带（L171）
- **围攻例外**：`Siege`策略的`raidLootValueFactor=0`，不生成战利品

### 信件生成
- **标签格式**：`"{raidStrategy.letterLabelEnemy}: {faction.Name}"`
- **正文组成**：到达模式文本 + 策略文本 + 领袖提示 + 年龄限制额外文本
- **信件类型**：`LetterDefOf.ThreatBig`

### Lord创建
- **源码**：`parms.raidStrategy.Worker.MakeLords(parms, pawns)` L169
- **逻辑**：为每个Pawn群组创建独立的Lord，调用策略的`MakeLordJob()`创建LordJob

## 三、Lord/LordJob系统架构

### 核心概念

| 概念 | 类 | 职责 |
|------|---|------|
| **Lord** | `Verse.AI.Group.Lord` | 管理一组Pawn的AI控制器，持有LordJob和当前LordToil |
| **LordJob** | `Verse.AI.Group.LordJob` | 定义Lord的目标和行为，创建StateGraph |
| **StateGraph** | `Verse.AI.Group.StateGraph` | 状态机，包含LordToil列表和Transition列表 |
| **LordToil** | `Verse.AI.Group.LordToil` | 具体的行为状态，决定Pawn的DutyDef |
| **Transition** | `Verse.AI.Group.Transition` | 状态转换规则，包含源状态、目标状态、触发器列表 |
| **Trigger** | `Verse.AI.Group.Trigger` | 触发状态转换的条件（时间、伤害、目标检测等） |

### LordJob_AssaultColony完整状态机

**构造参数**：
```csharp
LordJob_AssaultColony(
    Faction assaulterFaction,
    bool canKidnap = true,
    bool canTimeoutOrFlee = true,
    bool sappers = false,
    bool useAvoidGridSmart = false,
    bool canSteal = true,
    bool breachers = false,
    bool canPickUpOpportunisticWeapons = false
)
```

**4个状态节点（LordToil）**：

| # | 状态 | LordToil类 | 条件 | 行为 |
|---|------|-----------|------|------|
| 1 | **工兵攻击** | `LordToil_AssaultColonySappers` | `sappers=true` | 工兵挖掘路径绕过防御，使用避障网格 |
| 2 | **破坏攻击** | `LordToil_AssaultColonyBreaching` | `breachers=true` | 使用爆破武器破坏墙壁，使用避障网格 |
| 3 | **普通攻击** | `LordToil_AssaultColony` | 默认 | 直接攻击殖民地，可拾取武器 |
| 4 | **撤退** | `LordToil_ExitMap` | 转换触发 | 紧急程度Jog，始终使用避障网格 |

**7个状态转换（Transition）**：

| # | 转换 | 触发器 | 条件 | 消息 |
|---|------|--------|------|------|
| 1 | 工兵→攻击 | `Trigger_NoFightingSappers` | 无战斗中工兵 | — |
| 2 | 攻击→撤退（超时） | `Trigger_TicksPassed` | `canTimeoutOrFlee` + 人类派系 | "袭击者放弃离开" |
| 3 | 攻击→撤退（满意） | `Trigger_FractionColonyDamageTaken` | `canTimeoutOrFlee` | "袭击者满意离开" |
| 4 | 攻击→绑架 | `Trigger_KidnapVictimPresent` | `canKidnap` | "袭击者正在绑架" |
| 5 | 攻击→偷窃 | `Trigger_HighValueThingsAround` | `canSteal` | "袭击者正在偷窃" |
| 6 | 攻击→撤退（机械体） | `Trigger_GameEnding` | 机械族派系 | "机械体继续前进" |
| 7 | 攻击→撤退（关系） | `Trigger_BecameNonHostileToPlayer` | 派系变非敌对 | "袭击者离开" |

**避障机制**：
- 实现`ILordAvoidTraps`接口
- `AvoidTrapRatio = 0.3`（30%避障率）
- 仅在`useAvoidGridSmart=true`时启用

## 四、袭击结束条件（7种）

### 1. 全部击杀/倒下
- **触发**：所有袭击者死亡或倒下
- **机制**：Lord自动解散（`Lord.CheckTransitionOnSignal`检测到所有成员不可用）
- **结果**：袭击自然结束

### 2. 超时撤退
- **触发器**：`Trigger_TicksPassed`
- **条件**：`canTimeoutOrFlee=true` 且人类派系
- **时间常量**：
  - 普通攻击：26000-38000 ticks（约7.2-10.6游戏小时）
  - 工兵攻击：33000-38000 ticks（约9.2-10.6游戏小时）
  - 破坏攻击：33000-38000 ticks
- **过滤器**：`TriggerFilter_MapExitable()`——必须可以离开地图
- **消息**：`"MessageRaidersGivenUpLeaving"`
- **源码**：`LordJob_AssaultColony.CreateGraph` L118-124

### 3. 满意撤退
- **触发器**：`Trigger_FractionColonyDamageTaken(0.25-0.35, 900f)`
- **条件**：`canTimeoutOrFlee=true`
- **逻辑**：造成殖民地25-35%伤害后撤退（随机阈值）
- **伤害计算窗口**：900 ticks（15秒）
- **过滤器**：`TriggerFilter_MapExitable()`
- **消息**：`"MessageRaidersSatisfiedLeaving"`
- **源码**：`LordJob_AssaultColony.CreateGraph` L125-131

### 4. 转入绑架
- **触发器**：`Trigger_KidnapVictimPresent()`
- **条件**：`canKidnap=true`
- **目标**：倒下的殖民者
- **行为**：切换到`LordJob_Kidnap`子图，抓取目标并撤离
- **消息**：`"MessageRaidersKidnapping"`
- **源码**：`LordJob_AssaultColony.CreateGraph` L133-141

### 5. 转入偷窃
- **触发器**：`Trigger_HighValueThingsAround()`
- **条件**：`canSteal=true`
- **阈值**：`StealAIUtility.StartStealingMarketValueThreshold(lord)`
- **行为**：切换到`LordJob_Steal`子图，偷取物品并撤离
- **消息**：`"MessageRaidersStealing"`
- **源码**：`LordJob_AssaultColony.CreateGraph` L142-150

### 6. 机械体结束
- **触发器**：`Trigger_GameEnding()`
- **条件**：派系 = `Faction.OfMechanoids`
- **消息**：`"MechanoidsMovingOn"`
- **源码**：`LordJob_AssaultColony.CreateGraph` L152-159

### 7. 关系改善撤退
- **触发器**：`Trigger_BecameNonHostileToPlayer()`
- **条件**：派系变为非敌对（如玩家与派系和解）
- **消息**：`"MessageRaidersLeaving"`
- **源码**：`LordJob_AssaultColony.CreateGraph` L160-167

## 五、Pawn生成流程

### 标准生成链

```
PawnGroupMakerUtility.GeneratePawns(PawnGroupMakerParms)
  → faction.def.pawnGroupMakers 中选择匹配 groupKind 的 PawnGroupMaker
  → PawnGroupMaker.options 列表中按权重+点数成本选择 PawnKindDef
  → PawnGenerator.GeneratePawn(PawnGenerationRequest) 生成每个Pawn
  → parms.raidArrivalMode.Worker.Arrive(pawns, parms) 放置Pawn
```

### PawnGenerationRequest关键参数

| 参数 | 值 | 说明 |
|------|---|------|
| `faction` | 袭击派系 | 决定装备、服装、科技水平 |
| `context` | `PawnGenerationContext.NonPlayer` | 非玩家Pawn |
| `mustBeCapableOfViolence` | `true` | 必须能战斗 |
| `biocodeWeaponsChance` | `parms.biocodeWeaponsChance` | 生物编码武器概率 |
| `biocodeApparelChance` | `parms.biocodeApparelChance` | 生物编码服装概率 |
| `allowFood` | `def.pawnsCanBringFood` | 是否携带食物 |

### 策略自定义生成
- **方法**：`RaidStrategyWorker.SpawnThreats(IncidentParms)`
- **默认**：返回`null`，使用标准生成
- **围攻策略**：重写此方法生成炮台和操作员

## 六、战利品生成机制

### 价值计算公式

```
lootValue = raidLootValueFromPointsCurve(points)
           × raidStrategy.raidLootValueFactor
           × difficulty.EffectiveRaidLootPointsFactor
```

### 关键配置

| 配置 | 说明 |
|------|------|
| `faction.def.raidLootMaker` | 战利品生成器（ThingSetMakerDef） |
| `raidStrategy.raidLootValueFactor` | 策略战利品倍率（Siege=0，不生成） |
| `RaidLootDistributor` | 将战利品分配给袭击者携带 |
