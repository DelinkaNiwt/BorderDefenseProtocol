# 装备属性叠加计算

**总览**：装备通过三条路径影响Pawn属性——**equippedStatOffsets**（ThingDef上声明的穿戴属性偏移）、**装备自身Stat值**（如护甲值通过StatPart_GearStatOffset传递给Pawn）、**装备Stat因子**（通过StatPart_GearStatFactor对Pawn属性施加乘法）。这三条路径在Stat计算管线中由不同的StatPart负责收集和叠加。

## 核心概念区分

| 概念 | 定义位置 | 作用对象 | 说明 |
|------|---------|---------|------|
| **equippedStatOffsets** | `ThingDef.equippedStatOffsets` | Pawn的Stat | 穿戴时对Pawn属性的加法偏移 |
| **装备自身Stat** | 装备的`statBases` | 装备本身 | 装备自己的属性值（如护甲值） |
| **StatPart_GearStatOffset** | Pawn的StatDef.parts | Pawn的Stat | 收集所有装备的某个Stat值，加到Pawn属性上 |
| **StatPart_GearStatFactor** | Pawn的StatDef.parts | Pawn的Stat | 收集所有装备的某个Stat值，乘到Pawn属性上 |

## 路径1：equippedStatOffsets（穿戴属性偏移）

### 数据声明（XML）

```xml
<ThingDef>
  <defName>Gun_Revolver</defName>
  <equippedStatOffsets>
    <ShootingAccuracyPawn>0.5</ShootingAccuracyPawn>
    <MoveSpeed>-0.1</MoveSpeed>
  </equippedStatOffsets>
</ThingDef>
```

### 读取逻辑：`StatWorker.StatOffsetFromGear()`

```csharp
public static float StatOffsetFromGear(Thing gear, StatDef stat) {
    // 1. 从ThingDef.equippedStatOffsets读取基础偏移
    float val = gear.def.equippedStatOffsets.GetStatOffsetFromList(stat);

    // 2. 叠加心灵武器特质的偏移（Royalty DLC）
    CompBladelinkWeapon comp = gear.TryGetComp<CompBladelinkWeapon>();
    if (comp != null) {
        foreach (WeaponTraitDef trait in comp.TraitsListForReading)
            val += trait.equippedStatOffsets.GetStatOffsetFromList(stat);
    }

    // 3. 如果偏移非零，还要经过Stat的StatParts处理
    //    （例如品质会影响equippedStatOffsets的值）
    if (Math.Abs(val) > float.Epsilon && !stat.parts.NullOrEmpty()) {
        foreach (StatPart part in stat.parts)
            part.TransformValue(StatRequest.For(gear), ref val);
    }

    return val;
}
```

**关键细节**：
- `equippedStatOffsets`是**加法偏移**，直接加到Pawn的对应Stat上
- 心灵武器特质（Bladelink）的偏移也会叠加
- 偏移值本身还会经过StatParts处理（如品质倍率），所以传奇品质武器的equippedStatOffsets效果更强

## 路径2：StatPart_GearStatOffset（装备Stat加法传递）

将装备**自身的某个Stat值**作为加法偏移传递给Pawn的另一个Stat。

### 工作原理

```csharp
public class StatPart_GearStatOffset : StatPart {
    private StatDef apparelStat;    // 要从装备上读取的Stat
    private bool subtract;          // 是否减去（而非加上）
    private bool includeWeapon;     // 是否包含武器

    public override void TransformValue(StatRequest req, ref float val) {
        Pawn pawn = req.Thing as Pawn;

        // 遍历所有穿戴的服装
        foreach (Apparel apparel in pawn.apparel.WornApparel) {
            float statValue = apparel.GetStatValue(apparelStat);
            statValue += StatWorker.StatOffsetFromGear(apparel, apparelStat);
            val += (subtract ? -statValue : statValue);
        }

        // 可选：包含武器
        if (includeWeapon && pawn.equipment?.Primary != null) {
            float statValue = pawn.equipment.Primary.GetStatValue(apparelStat);
            statValue += StatWorker.StatOffsetFromGear(Primary, apparelStat);
            val += (subtract ? -statValue : statValue);
        }
    }
}
```

### 典型应用

| Pawn的Stat | apparelStat | 含义 |
|-----------|-------------|------|
| `ArmorRating_Sharp` | `ArmorRating_Sharp` | 服装的锐利护甲值加到Pawn的锐利护甲上 |
| `MoveSpeed` | `MoveSpeed` | 服装的移动速度修饰加到Pawn移速上 |
| `PsychicSensitivity` | `PsychicSensitivityOffset` | Eltex装备的心灵敏感度偏移 |

**注意**：这里读取的是装备**自身的Stat值**（经过品质、材质等修饰后的最终值），不是equippedStatOffsets。

## 路径3：StatPart_GearStatFactor（装备Stat乘法传递）

将装备**自身的某个Stat值**作为乘法因子应用到Pawn的另一个Stat。

### 工作原理

```csharp
public class StatPart_GearStatFactor : StatPart {
    private StatDef apparelStat;    // 要从装备上读取的Stat
    private bool includeWeapon;     // 是否包含武器

    public override void TransformValue(StatRequest req, ref float val) {
        Pawn pawn = req.Thing as Pawn;

        // 遍历所有穿戴的服装——乘法叠加
        foreach (Apparel apparel in pawn.apparel.WornApparel) {
            val *= apparel.GetStatValue(apparelStat);
        }

        // 可选：包含武器
        if (includeWeapon && pawn.equipment?.Primary != null) {
            val *= pawn.equipment.Primary.GetStatValue(apparelStat);
        }
    }
}
```

### 典型应用

| Pawn的Stat | apparelStat | 含义 |
|-----------|-------------|------|
| `ShootingAccuracyPawn` | `ShootingAccuracyFactor` | 装备的射击精度因子 |
| `MeleeDodgeChance` | `MeleeDodgeChanceFactor` | 装备的近战闪避因子 |

**关键区别**：
- **GearStatOffset**：加法叠加（`val += statValue`），多件装备效果累加
- **GearStatFactor**：乘法叠加（`val *= statValue`），多件装备效果连乘

## 完整计算管线

一个Pawn的Stat最终值经过以下计算链：

```
1. 基础值（StatDef.defaultBaseValue 或 Pawn的statBases）
2. + 各种加法偏移（StatPart们的TransformValue）
   ├── StatPart_GearStatOffset：装备Stat值加法传递
   ├── StatPart_Age、StatPart_Health等：其他加法修饰
   └── ...
3. × 各种乘法因子
   ├── StatPart_GearStatFactor：装备Stat值乘法传递
   └── ...
4. 最终值 = clamp(计算值, StatDef.minValue, StatDef.maxValue)
```

### equippedStatOffsets在管线中的位置

`equippedStatOffsets`不直接参与上述管线。它通过`StatWorker.StatOffsetFromGear()`被`StatPart_GearStatOffset`间接读取：

```
StatPart_GearStatOffset.TransformValue()
  → 对每件装备：
    → apparel.GetStatValue(apparelStat)     // 装备自身Stat值
    → + StatWorker.StatOffsetFromGear(...)   // equippedStatOffsets偏移
    → 两者之和加到Pawn的Stat上
```

## 实例分析：Pawn穿戴板甲后的护甲计算

```
假设：
- 板甲（Excellent品质，钢材质）
  - statBases中ArmorRating_Sharp = 1.0
  - 品质倍率（Excellent）= 1.3
  - 材质倍率（Steel）= 另外的StatPart处理

计算过程：
1. 板甲自身的ArmorRating_Sharp = 1.0 × 1.3（品质） = 1.3
2. StatPart_GearStatOffset读取板甲的ArmorRating_Sharp = 1.3
3. 加到Pawn的ArmorRating_Sharp上
4. 如果Pawn还穿了其他有护甲的服装，继续累加
```

## 开发者要点

1. **equippedStatOffsets vs statBases**：前者是"穿上后给Pawn的加成"，后者是"装备自身的属性"。两者通过不同路径影响Pawn
2. **品质影响双重叠加**：品质既影响装备自身Stat（如护甲值），也影响equippedStatOffsets的偏移量
3. **加法vs乘法**：GearStatOffset是加法叠加（多件累加），GearStatFactor是乘法叠加（多件连乘）。设计装备效果时需注意选择
4. **includeWeapon标记**：不是所有StatPart都包含武器，需要在StatDef的XML中配置
5. **subtract标记**：GearStatOffset支持减法模式（如重甲降低移速），GearStatFactor不支持
