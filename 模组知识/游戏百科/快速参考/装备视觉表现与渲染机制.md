# 装备视觉表现与渲染机制

**总览**：RimWorld的装备视觉表现分两条完全独立的渲染路径——**衣物**（Apparel）通过`PawnRenderNode_Apparel`在渲染树内渲染，随Pawn身体一起绘制；**武器和携带物**通过`PawnRenderNodeWorker_Carried`的`PostDraw()`在渲染树外独立绘制。两条路径的贴图解析、层级控制、位置计算完全不同。

```
┌──────────────────────────────────────────────────────────────────────────┐
│                    装备视觉渲染双路径架构                                  │
├──────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  路径A：衣物（渲染树内）                                                  │
│  ──────────────────────                                                  │
│  DynamicSetup_Apparel                                                    │
│  ├── 遍历 pawn.apparel.WornApparel                                       │
│  ├── 每件衣物 → ApparelGraphicRecordGetter.TryGetGraphicApparel()        │
│  │   ├── 普通身体衣物: wornGraphicPath + "_" + bodyType.defName          │
│  │   ├── Overhead/EyeCover: wornGraphicPath（无体型后缀）                 │
│  │   └── Pack(Belt层): wornGraphicPath（无体型后缀）+ WornGraphicData偏移 │
│  ├── 创建 PawnRenderNode_Apparel 节点                                    │
│  │   ├── 头部衣物 → Worker: PawnRenderNodeWorker_Apparel_Head            │
│  │   └── 身体衣物 → Worker: PawnRenderNodeWorker_Apparel_Body            │
│  └── 注入到渲染树（ApparelHead/ApparelBody父节点下）                      │
│                                                                          │
│  路径B：武器/携带物（渲染树外）                                           │
│  ──────────────────────────                                              │
│  PawnRenderNodeWorker_Carried.PostDraw()                                 │
│  ├── 有携带物? → PawnRenderUtility.DrawCarriedThing()                    │
│  └── 无携带物? → PawnRenderUtility.DrawEquipmentAndApparelExtras()       │
│      ├── 武器渲染（两种模式）                                             │
│      │   ├── 瞄准中: DrawEquipmentAiming(weapon, pos, aimAngle)          │
│      │   └── 公开携带: DrawCarriedWeapon(weapon, pos, facing, scale)     │
│      └── 衣物额外渲染: Apparel.DrawWornExtras()                          │
│          └── 遍历Comp → CompDrawWornExtras()（如护盾气泡）                │
│                                                                          │
└──────────────────────────────────────────────────────────────────────────┘
```

## 1. 衣物贴图解析（ApparelGraphicRecordGetter）

`TryGetGraphicApparel()`根据衣物层级选择不同的贴图路径解析策略：

| 衣物类型 | 层级条件 | 贴图路径 | 体型后缀 | 说明 |
|---------|---------|---------|---------|------|
| **普通身体衣物** | 非Overhead、非EyeCover、非Pack | `wornGraphicPath + "_" + bodyType.defName` | ✅ 有 | 如`_Male`、`_Female`、`_Thin` |
| **头部衣物** | Overhead 或 EyeCover | `wornGraphicPath` | ❌ 无 | 头盔/眼罩，所有体型共用 |
| **背包物品** | Belt层 + `RenderAsPack()=true` | `wornGraphicPath` | ❌ 无 | 护盾腰带等，通过WornGraphicData控制偏移 |

**RenderAsPack判定**：
```csharp
// PawnRenderUtility.RenderAsPack()
public static bool RenderAsPack(this Apparel apparel)
{
    if (apparel.def.apparel.LastLayer.IsUtilityLayer)  // Belt层
    {
        if (apparel.def.apparel.wornGraphicData != null)
            return apparel.def.apparel.wornGraphicData.renderUtilityAsPack;
        return true;  // 默认Belt层都是Pack
    }
    return false;
}
```

**着色器选择**：默认`ShaderDatabase.Cutout`，StyleDef可覆盖，`useWornGraphicMask=true`时使用`CutoutComplex`（支持双色遮罩）。

## 2. 衣物层级渲染

衣物在渲染树中的层级由`DynamicSetup_Apparel.ProcessApparel()`决定：

| 衣物类型 | 父节点 | Worker类 | 特殊层级处理 |
|---------|--------|---------|------------|
| **头部衣物** | `ApparelHead` | `PawnRenderNodeWorker_Apparel_Head` | 无 |
| **普通身体衣物** | `ApparelBody` | `PawnRenderNodeWorker_Apparel_Body` | 无 |
| **Shell层衣物** | `ApparelBody` | `PawnRenderNodeWorker_Apparel_Body` | 北向layer=88（头部后方） |
| **Pack背包** | `ApparelBody` | `PawnRenderNodeWorker_Apparel_Body` | 北向layer=93，南向layer=-3 |

**Shell层北向特殊处理**：Shell层衣物（如防弹衣外套）在面朝北时需要渲染在头部后方（layer=88），否则会遮挡头部。这通过`DrawData.RotationalData`实现：

```csharp
// DynamicSetup_Apparel.ProcessApparel() L117-120
if (lastLayer == ApparelLayerDefOf.Shell)
{
    props.drawData = DrawData.NewWithData(new DrawData.RotationalData(Rot4.North, 88f));
}
```

**Pack背包层级**：Pack物品在北向时layer=93（在Shell后方），南向时layer=-3（在身体前方），实现"背在背上"的视觉效果。

## 3. WornGraphicData系统

`WornGraphicData`为Pack物品提供**方向×体型**的二维偏移和缩放控制：

```
WornGraphicData
├── 方向数据（4个WornGraphicDirectionData）
│   ├── north: offset + scale
│   ├── south: offset + scale
│   ├── east:  offset + scale
│   └── west:  offset + scale
└── 体型数据（5个WornGraphicBodyTypeData）
    ├── male:   offset + scale
    ├── female: offset + scale
    ├── thin:   offset + scale
    ├── hulk:   offset + scale
    └── fat:    offset + scale
```

**偏移计算**：`BeltOffsetAt(facing, bodyType)` = 方向基础偏移 + 体型偏移（加法叠加）
**缩放计算**：`BeltScaleAt(facing, bodyType)` = 方向基础缩放 × 体型缩放（乘法叠加）

`PawnRenderNodeWorker_Apparel_Body`在`OffsetFor()`和`ScaleFor()`中应用这些值：
```csharp
// PawnRenderNodeWorker_Apparel_Body.OffsetFor() L24-29
if (apparel.def.apparel.wornGraphicData != null && apparel.RenderAsPack())
{
    Vector2 vector = apparel.def.apparel.wornGraphicData.BeltOffsetAt(parms.facing, parms.pawn.story.bodyType);
    result.x += vector.x;
    result.z += vector.y;
}
```

## 4. 武器渲染双模式

武器不在渲染树内，而是通过`PawnRenderNodeWorker_Carried.PostDraw()`独立绘制。两种渲染模式：

### 4.1 瞄准模式（DrawEquipmentAiming）

当Pawn处于`Stance_Busy`且有有效目标时，武器旋转指向目标：

```csharp
// PawnRenderUtility.DrawEquipmentAiming() 核心逻辑
float angle = aimAngle - 90f;
if (aimAngle > 20f && aimAngle < 160f)       // 面朝右
    mesh = MeshPool.plane10;                  // 正常朝向
    angle += equippedAngleOffset;
else if (aimAngle > 200f && aimAngle < 340f)  // 面朝左
    mesh = MeshPool.plane10Flip;              // 水平翻转
    angle -= 180f - equippedAngleOffset;
else                                          // 面朝前/后
    mesh = MeshPool.plane10;
    angle += equippedAngleOffset;
```

**后坐力效果**：`EquipmentUtility.Recoil()`根据当前Verb计算偏移和角度偏移，叠加到武器位置上。

**武器定位参数**：
- `equippedAngleOffset`：武器静态角度偏移（ThingDef字段）
- `equippedDistanceOffset`：武器距Pawn中心的距离偏移（ThingDef字段）
- `equipmentDrawDistanceFactor`：生命阶段缩放因子（儿童武器更近）

### 4.2 携带模式（DrawCarriedWeapon）

当`CarryWeaponOpenly()`返回true但无瞄准目标时，武器在固定位置显示：

| 朝向 | 偏移量 | 默认角度 | 说明 |
|------|--------|---------|------|
| **North** | `EqLocNorth` | 143° | 武器在身后 |
| **East** | `EqLocEast` | 143° | 武器在右侧 |
| **South** | `EqLocSouth` | 143° | 武器在身前 |
| **West** | `EqLocWest` | 217° | 武器在左侧（翻转） |

### 4.3 CarryWeaponOpenly判定链（6条件）

```csharp
// PawnRenderUtility.CarryWeaponOpenly() — 任一为true即显示武器
1. pawn.carryTracker.CarriedThing != null  → false（携带物品时不显示武器）
2. pawn.Drafted                            → true（征召状态）
3. pawn.CurJob.def.alwaysShowWeapon        → true（Job要求显示）
4. pawn.mindState.duty.def.alwaysShowWeapon → true（Duty要求显示）
5. lord.LordJob.AlwaysShowWeapon           → true（Lord要求显示）
6. pawn.CompsWantHoldWeapon()              → true（Comp请求显示）
```

> **关键发现**：`CompsWantHoldWeapon()`是模组让武器在非征召状态下显示的标准扩展点——自定义ThingComp重写`CompWantHoldWeapon()`返回true即可。

## 5. DrawWornExtras扩展通道

`Apparel.DrawWornExtras()`在武器渲染之后调用，遍历衣物的所有Comp调用`CompDrawWornExtras()`：

```csharp
// Apparel.DrawWornExtras()
public virtual void DrawWornExtras()
{
    foreach (ThingComp comp in AllComps)
        comp.CompDrawWornExtras();  // 每个Comp都有机会绘制额外视觉
}
```

**原版唯一使用者**：`CompShield.CompDrawWornExtras()` → 绘制护盾能量气泡。

> **关键发现**：`CompDrawWornExtras()`是在Pawn身上绘制额外视觉效果（如光环、粒子、护盾）的标准通道——不在渲染树内，每帧独立调用，适合动态特效。

## 6. 携带物渲染

当Pawn搬运物品时（`carryTracker.CarriedThing != null`），`DrawCarriedThing()`替代武器渲染：

```csharp
// PawnRenderUtility.DrawCarriedThing()
public static void DrawCarriedThing(Pawn pawn, Vector3 drawLoc, Thing carriedThing)
{
    // ... 计算偏移
    CalculateCarriedDrawPos(pawn, carriedThing, ref carryDrawPos, out flip);
    carriedThing.DrawAt(carryDrawPos, flip);
}
```

携带物显示在Pawn手部位置，位置由`CalculateCarriedDrawPos()`根据朝向计算。

| # | 扩展方式 | 路径 | 难度 | 适用场景 |
