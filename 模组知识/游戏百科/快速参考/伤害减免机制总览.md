# 伤害减免机制总览

**总览**：RimWorld有6种减少或避免伤害的机制，按伤害管线中的生效位置从早到晚排列——**未命中**（阶段1，攻击根本没打中）→ **闪避**（阶段1，近战专属）→ **掩体**（阶段1，远程专属）→ **护盾**（阶段1投射物拦截/阶段4伤害吸收）→ **伤害因子**（阶段3-5，基因/Hediff/Stat倍率）→ **护甲**（阶段5，逐层随机判定）。前3种是"避免伤害"（二元判定，要么命中要么不命中），后3种是"减少伤害"（数值削减，修改伤害量）。

## 3. ★ 护甲系统（ArmorUtility）

### 3.1 ArmorUtility.GetPostArmorDamage — 逐层判定流程

`ArmorUtility`（Verse命名空间）是护甲计算的核心类。

**关键常量**：
```csharp
public const float MaxArmorRating = 2f;           // 最大护甲值200%
public const float DeflectThresholdFactor = 0.5f;  // 完全偏转阈值系数
```

**主方法签名**：
```csharp
public static float GetPostArmorDamage(
    Pawn pawn,
    float amount,                    // 原始伤害
    float armorPenetration,          // 护甲穿透值
    BodyPartRecord part,             // 受击部位
    ref DamageDef damageDef,         // 伤害类型（可能被修改：Sharp→Blunt）
    out bool deflectedByMetalArmor,  // 是否被金属护甲偏转
    out bool diminishedByMetalArmor  // 是否被金属护甲削弱
)
```

**处理流程**：
```csharp
// 1. 检查伤害类型是否有护甲类别
StatDef armorRatingStat = damageDef.armorCategory?.armorRatingStat;
if (armorRatingStat == null) return amount;  // 无护甲类别→不计算护甲

// 2. 从外到内遍历所有覆盖受击部位的装备
List<Apparel> wornApparel = pawn.apparel.WornApparel;
for (int num = wornApparel.Count - 1; num >= 0; num--)  // 倒序=从外到内
{
    Apparel apparel = wornApparel[num];
    if (apparel.def.apparel.CoversBodyPart(part))  // 只检查覆盖该部位的装备
    {
        ApplyArmor(ref amount, armorPenetration,
                   apparel.GetStatValue(armorRatingStat),
                   apparel, ref damageDef, pawn, out var metalArmor);
        if (amount < 0.001f)  // 伤害已归零→提前返回
        {
            deflectedByMetalArmor = metalArmor;
            return 0f;
        }
    }
}

// 3. 最后应用Pawn自身护甲（种族天生护甲，如机械体）
ApplyArmor(ref amount, armorPenetration,
           pawn.GetStatValue(armorRatingStat),
           null, ref damageDef, pawn, out var metalArmor2);
```

> **关键特性**：
> - **独立判定**：每层护甲独立进行随机判定
> - **伤害递减**：外层削弱后，内层对剩余伤害再次判定
> - **提前终止**：任何一层完全偏转后，内层不再计算
> - **部位覆盖**：只有覆盖受击部位的装备才参与计算

### 3.2 ArmorUtility.ApplyArmor — 单层护甲判定

```csharp
private static void ApplyArmor(
    ref float damAmount,
    float armorPenetration,
    float armorRating,
    Thing armorThing,       // null=Pawn自身护甲
    ref DamageDef damageDef,
    Pawn pawn,
    out bool metalArmor)
{
    // 1. 护甲磨损：装备承受原始伤害的25%
    if (armorThing != null)
    {
        float f = damAmount * 0.25f;
        armorThing.TakeDamage(new DamageInfo(damageDef, GenMath.RoundRandom(f)));
    }

    // 2. 计算有效护甲值
    float effectiveArmor = Mathf.Max(armorRating - armorPenetration, 0f);

    // 3. 随机判定
    float roll = Rand.Value;
    float deflectThreshold = effectiveArmor * 0.5f;  // 完全偏转阈值
    float reduceThreshold = effectiveArmor;           // 削弱阈值

    if (roll < deflectThreshold)
    {
        // 完全偏转：伤害归零
        damAmount = 0f;
    }
    else if (roll < reduceThreshold)
    {
        // 部分削弱：伤害减半，Sharp→Blunt转换
        damAmount = GenMath.RoundRandom(damAmount / 2f);
        if (damageDef.armorCategory == DamageArmorCategoryDefOf.Sharp)
        {
            damageDef = DamageDefOf.Blunt;  // 锐器伤害转为钝器
        }
    }
    // else: 护甲无效，伤害全额穿透

    // 4. 金属护甲判定（影响视觉效果）
    metalArmor = armorThing != null &&
        (armorThing.def.apparel.useDeflectMetalEffect ||
         (armorThing.Stuff != null && armorThing.Stuff.IsMetal));
}
```

### 3.3 护甲判定决策树

```
输入：armorRating, armorPenetration, damAmount
  │
  ├── effectiveArmor = max(armorRating - armorPenetration, 0)
  │
  ├── roll = Random(0, 1)
  │
  ├── roll < eA × 0.5 ?
  │     └── YES → 完全偏转（damAmount = 0）
  │               概率 = eA × 50%
  │
  ├── roll < eA ?
  │     └── YES → 伤害减半 + Sharp→Blunt转换
  │               概率 = eA × 50%
  │               damAmount = damAmount / 2（四舍五入）
  │
  └── else → 护甲无效，全额穿透
              概率 = 1 - eA
```

**实战案例**：90%护甲 vs 10%穿透
```
有效护甲 = 0.9 - 0.1 = 0.8
完全偏转概率 = 0.8 × 0.5 = 40%
削弱概率     = 0.8 × 0.5 = 40%
穿透概率     = 1 - 0.8   = 20%
期望伤害     = 原伤害 × (0% × 40% + 50% × 40% + 100% × 20%) = 原伤害 × 40%
```

### 3.6 Sharp→Blunt转换机制

当护甲"削弱"判定成功时：
- **Sharp类伤害**：转换为Blunt伤害（`damageDef = DamageDefOf.Blunt`），伤害减半
- **Blunt类伤害**：保持Blunt，伤害减半
- **Heat类伤害**：保持Heat，伤害减半

> **设计意图**：锐器被护甲挡住后变成钝击冲击——刀砍在板甲上不会切开，但冲击力仍然传递。转换后的Blunt伤害会在下一层护甲中用`ArmorRating_Blunt`重新判定。

## 4. 掩体系统（CoverUtility）

### 4.1 掩体基础阻挡率

`CoverUtility`（Verse命名空间）计算掩体对远程射击的阻挡效果。

```csharp
public static float BaseBlockChance(this ThingDef def)
{
    if (def.Fillage == FillCategory.Full)
        return 0.75f;       // 完全填充物体：固定75%
    return def.fillPercent;  // 其他物体：使用fillPercent值
}

public static float BaseBlockChance(this Thing thing)
{
    if (thing is Building_Door { Open: not false })
        return 0f;           // 开启的门：0%
    return thing.def.BaseBlockChance();
}
```

**常见掩体fillPercent值**：

| 掩体 | fillPercent | 基础阻挡率 | 说明 |
|------|-----------|-----------|------|
| 墙壁 | 1.0 (Full) | 75% | Full填充固定75%，非100% |
| 沙袋 | 0.55 | 55% | 最常用掩体 |
| 岩石块 | 0.35 | 35% | 自然掩体 |
| 树木 | 0.25 | 25% | 弱掩体 |
| 开门 | — | 0% | 开启的门无掩体效果 |
| 角落 | — | 75% | `CoverPercent_Corner = 0.75f` |

### 4.2 CoverUtility.TryFindAdjustedCoverInCell — 角度/距离调整

掩体效果受射击角度和距离两个维度调整：

**角度调整**（射手→目标方向 vs 掩体→目标方向的角度差）：

```csharp
float angleDiff = GenGeo.AngleDifferenceBetween(
    (adjCell - cell).AngleFlat, angleFlat);

// 对角线掩体角度差放大1.75倍（惩罚）
if (!cell.AdjacentToCardinal(adjCell))
    angleDiff *= 1.75f;
```

| 角度差 | 掩体效果倍率 | 说明 |
|--------|------------|------|
| < 15° | × 1.0 (100%) | 正面掩体，完全有效 |
| 15°~27° | × 0.8 (80%) | 略偏，仍然有效 |
| 27°~40° | × 0.6 (60%) | 明显偏移 |
| 40°~52° | × 0.4 (40%) | 大幅偏移 |
| 52°~65° | × 0.2 (20%) | 几乎无效 |
| ≥ 65° | 无效 | 掩体不在射击线上 |

**距离调整**（射手到掩体的距离）：

| 距离 | 掩体效果倍率 | 说明 |
|------|------------|------|
| < 1.9格 | × 0.333 (33%) | 近距离掩体大幅削弱 |
| 1.9~2.9格 | × 0.667 (67%) | 中等距离 |
| ≥ 2.9格 | × 1.0 (100%) | 正常距离，完全有效 |

> **关键发现**：近距离射击时掩体效果大幅降低——<1.9格仅33%效果。这解释了为什么近距离战斗中掩体不太有用。

### 4.3 CalculateOverallBlockChance — 多掩体递减累加

```csharp
// 遍历目标周围8个格子，累加掩体效果
float totalBlockChance = 0f;
for (int i = 0; i < 8; i++)
{
    if (TryFindAdjustedCoverInCell(..., out var result))
    {
        totalBlockChance += (1f - totalBlockChance) * result.BlockChance;
    }
}
```

**递减公式**：`total += (1 - total) × newBlockChance`

这意味着多个掩体的效果递减累加，而非简单相加。例如：
- 第1个掩体55%：total = 55%
- 第2个掩体35%：total = 55% + (1-55%) × 35% = 55% + 15.75% = 70.75%
- 而非简单相加的90%

### 4.4 掩体在命中公式中的位置

```csharp
// ShotReport
public float PassCoverChance => 1f - coversOverallBlockChance;
public float TotalEstimatedHitChance => Mathf.Clamp01(AimOnTargetChance * PassCoverChance);
```

掩体作为独立乘数影响最终命中率：`总命中率 = 瞄准概率 × (1 - 掩体阻挡率)`。

## 5. 闪避系统（Verb_MeleeAttack.GetDodgeChance）

### 5.1 闪避判定流程

闪避仅在近战中存在，发生在命中判定之后：

```csharp
// Verb_MeleeAttack.TryCastShot()
if (Rand.Chance(GetNonMissChance(thing)))     // 先判定命中
{
    if (!Rand.Chance(GetDodgeChance(thing)))   // 再判定闪避
    {
        ApplyMeleeDamageToTarget(currentTarget);  // 命中且未闪避→造成伤害
    }
    else
    {
        MoteMaker.ThrowText(drawPos, map, "TextMote_Dodge".Translate(), 1.9f);
    }
}
```

**近战实际命中率** = `GetNonMissChance × (1 - GetDodgeChance)`

### 5.2 GetDodgeChance 完整逻辑

```csharp
private float GetDodgeChance(LocalTargetInfo target)
{
    // 4个禁用条件（任一满足→0%闪避）
    if (surpriseAttack) return 0f;                    // 突袭攻击
    if (IsTargetImmobile(target)) return 0f;          // 目标不可移动
    if (!(target.Thing is Pawn pawn)) return 0f;      // 非Pawn目标
    if (pawn.stances.curStance is Stance_Busy { verb: not null } sb
        && !sb.verb.verbProps.IsMeleeAttack)
        return 0f;                                     // 目标正在使用远程武器

    // 基础闪避率
    float dodgeChance = pawn.GetStatValue(StatDefOf.MeleeDodgeChance);

    // Ideology DLC：4种光照/室内外偏移
    if (ModsConfig.IdeologyActive)
    {
        if (DarknessCombatUtility.IsOutdoorsAndLit(target.Thing))
            dodgeChance += pawn.GetStatValue(StatDefOf.MeleeDodgeChanceOutdoorsLitOffset);
        else if (DarknessCombatUtility.IsOutdoorsAndDark(target.Thing))
            dodgeChance += pawn.GetStatValue(StatDefOf.MeleeDodgeChanceOutdoorsDarkOffset);
        else if (DarknessCombatUtility.IsIndoorsAndDark(target.Thing))
            dodgeChance += pawn.GetStatValue(StatDefOf.MeleeDodgeChanceIndoorsDarkOffset);
        else if (DarknessCombatUtility.IsIndoorsAndLit(target.Thing))
            dodgeChance += pawn.GetStatValue(StatDefOf.MeleeDodgeChanceIndoorsLitOffset);
    }

    return dodgeChance;
}
```

### 5.3 闪避禁用条件总表

| # | 条件 | 检查方式 | 说明 |
|---|------|---------|------|
| 1 | 突袭攻击 | `surpriseAttack` | 偷袭时目标无法闪避 |
| 2 | 目标不可移动 | `IsTargetImmobile()` | 倒地、非站立姿态、被眩晕 |
| 3 | 非Pawn目标 | `!(target.Thing is Pawn)` | 建筑等无法闪避 |
| 4 | 目标使用远程武器 | `Stance_Busy.verb非近战` | 正在射击的Pawn无法闪避近战 |

> **设计意图**：条件4是RimWorld的核心战术设计——远程单位在近战中无法闪避，这使得近战突击远程单位成为有效战术。

### 5.4 IsTargetImmobile 判定

```csharp
private bool IsTargetImmobile(LocalTargetInfo target)
{
    if (target.Thing is Pawn pawn)
    {
        if (pawn.Downed) return true;                    // 倒地
        if (pawn.GetPosture() != PawnPosture.Standing) return true;  // 非站立
        if (pawn.stances.stunner.Stunned) return true;   // 被眩晕
    }
    return true;  // 非Pawn默认不可移动
}
```

## 6. 未命中系统（ShotReport + GetNonMissChance）

### 6.1 远程命中：ShotReport 命中公式

`ShotReport`（Verse命名空间）计算远程射击的完整命中率。

**最终命中率**：
```csharp
public float TotalEstimatedHitChance => Mathf.Clamp01(AimOnTargetChance * PassCoverChance);
```

**公式分解**：
```
TotalHitChance = Clamp01(AimOnTargetChance × PassCoverChance)

AimOnTargetChance = AimOnTargetChance_StandardTarget × factorFromTargetSize × FactorFromPosture

AimOnTargetChance_StandardTarget = max(
    factorFromShooterAndDist
    × factorFromEquipment
    × factorFromWeather
    × factorFromCoveringGas
    × FactorFromExecution
    + offsetFromDarkness,
    0.0201)                          ← 最低命中率2.01%

PassCoverChance = 1 - coversOverallBlockChance
```

### 6.2 射手与距离因子（HitFactorFromShooter）

```csharp
public static float HitFactorFromShooter(Thing caster, float distance, float? acc = null)
{
    // 基础精度
    float accuracy = acc ?? caster.GetStatValue(StatDefOf.ShootingAccuracyPawn);

    // 距离衰减：accuracy^distance（指数衰减）
    float factor = Mathf.Pow(accuracy, distance);

    // Pawn额外的距离段调整（4段线性插值）
    if (caster is Pawn)
    {
        if (distance <= 3f)
            factor *= caster.GetStatValue(StatDefOf.ShootingAccuracyFactor_Touch);
        else if (distance <= 12f)
            factor *= Mathf.Lerp(Touch, Short, (distance - 3f) / 9f);
        else if (distance <= 25f)
            factor *= Mathf.Lerp(Short, Medium, (distance - 12f) / 13f);
        else if (distance <= 40f)
            factor *= Mathf.Lerp(Medium, Long, (distance - 25f) / 15f);
        else
            factor *= caster.GetStatValue(StatDefOf.ShootingAccuracyFactor_Long);
    }

    return Mathf.Max(factor, 0.0201f);  // 最低2.01%
}
```

**4个距离段**：

| 距离段 | 范围 | Stat | 说明 |
|--------|------|------|------|
| Touch（接触） | 0~3格 | `ShootingAccuracyFactor_Touch` | 近身射击 |
| Short（近距离） | 3~12格 | `ShootingAccuracyFactor_Short` | 常规交战距离 |
| Medium（中距离） | 12~25格 | `ShootingAccuracyFactor_Medium` | 中远程 |
| Long（远距离） | 25~40格 | `ShootingAccuracyFactor_Long` | 远程狙击 |

> **核心公式**：`accuracy^distance` 是指数衰减——精度0.96的射手在10格距离的基础因子为 0.96^10 ≈ 0.665，在30格为 0.96^30 ≈ 0.294。

### 6.4 姿态因子详解

```csharp
// 处决因子：近距离+倒地 = 750%命中
private float FactorFromExecution
{
    get
    {
        if (target.Thing is Pawn p && distance <= 3.9f
            && p.GetPosture() != PawnPosture.Standing)
            return 7.5f;
        return 1f;
    }
}

// 姿态因子：远距离+倒地 = 50%命中
private float FactorFromPosture
{
    get
    {
        if (target.Thing is Pawn p && distance >= 4.5f
            && p.GetPosture() != PawnPosture.Standing)
            return 0.5f;
        return 1f;
    }
}
```

> **设计意图**：倒地的Pawn在远距离更难被命中（50%），但在近距离反而更容易被命中（750%处决加成）。3.9~4.5格之间两个因子都不生效。

### 6.5 近战命中：GetNonMissChance

```csharp
private float GetNonMissChance(LocalTargetInfo target)
{
    // 突袭或目标不可移动：100%命中
    if (surpriseAttack || IsTargetImmobile(target))
        return 1f;

    // 基础命中率
    float hitChance = CasterPawn.GetStatValue(StatDefOf.MeleeHitChance);

    // Ideology DLC：4种光照/室内外偏移
    if (ModsConfig.IdeologyActive && target.HasThing)
    {
        if (DarknessCombatUtility.IsOutdoorsAndLit(target.Thing))
            hitChance += caster.GetStatValue(StatDefOf.MeleeHitChanceOutdoorsLitOffset);
        else if (DarknessCombatUtility.IsOutdoorsAndDark(target.Thing))
            hitChance += caster.GetStatValue(StatDefOf.MeleeHitChanceOutdoorsDarkOffset);
        else if (DarknessCombatUtility.IsIndoorsAndDark(target.Thing))
            hitChance += caster.GetStatValue(StatDefOf.MeleeHitChanceIndoorsDarkOffset);
        else if (DarknessCombatUtility.IsIndoorsAndLit(target.Thing))
            hitChance += caster.GetStatValue(StatDefOf.MeleeHitChanceIndoorsLitOffset);
    }

    return hitChance;
}
```

> **近战 vs 远程命中的关键区别**：
> - 近战命中由单一Stat（`MeleeHitChance`）决定，远程命中由多因子乘积决定
> - 近战没有距离衰减，远程有指数衰减
> - 近战有闪避（命中后第二次判定），远程没有闪避
> - 近战没有掩体，远程有掩体

## 7. 伤害因子系统（Gene/Hediff/Stat三层倍率）

### 7.1 三层伤害因子在管线中的位置

```
阶段3 Thing.TakeDamage
  └── ThingDef.damageMultipliers ←── 【第1层】Thing级倍率

阶段4 PreApplyDamage
  ├── Pawn_GeneTracker.FactorForDamage ←── 【第2层】基因倍率
  └── HediffSet.FactorForDamage ←── 【第3层】Hediff倍率

阶段5 DamageWorker_AddInjury.ApplyDamageToPart
  └── pawn.GetStatValue(IncomingDamageFactor) ←── 【第4层】Stat倍率
```

> **关键区别**：基因/Hediff倍率在**护甲之前**生效（阶段4），IncomingDamageFactor在**护甲之后**生效（阶段5）。

### 7.2 基因伤害因子（Pawn_GeneTracker.FactorForDamage）

```csharp
public float FactorForDamage(DamageInfo dinfo)
{
    if (!ModLister.BiotechInstalled || dinfo.Def == null
        || GenesListForReading.NullOrEmpty())
        return 1f;

    // 缓存机制
    if (cachedDamageFactors.TryGetValue(dinfo.Def, out var value))
        return value;

    float factor = 1f;
    List<Gene> genes = GenesListForReading;
    for (int i = 0; i < genes.Count; i++)
    {
        Gene gene = genes[i];
        if (!gene.Active || gene.def.damageFactors.NullOrEmpty())
            continue;
        for (int j = 0; j < gene.def.damageFactors.Count; j++)
        {
            if (gene.def.damageFactors[j].damageDef == dinfo.Def)
                factor *= gene.def.damageFactors[j].factor;  // 乘算累积
        }
    }

    cachedDamageFactors.Add(dinfo.Def, factor);
    return factor;
}
```

**XML配置示例**：
```xml
<GeneDef>
    <defName>FireResistant</defName>
    <damageFactors>
        <li>
            <damageDef>Flame</damageDef>
            <factor>0.5</factor>  <!-- 火焰伤害×50% -->
        </li>
    </damageFactors>
</GeneDef>
```

### 7.3 Hediff伤害因子（HediffSet.FactorForDamage）

```csharp
public float FactorForDamage(DamageInfo dinfo)
{
    if (dinfo.Def == null || hediffs.NullOrEmpty())
        return 1f;

    if (cachedDamageFactors.TryGetValue(dinfo.Def, out var value))
        return value;

    float factor = 1f;
    for (int i = 0; i < hediffs.Count; i++)
    {
        HediffStage stage = hediffs[i].CurStage;
        if (stage == null || stage.damageFactors.NullOrEmpty())
            continue;
        for (int j = 0; j < stage.damageFactors.Count; j++)
        {
            DamageFactor df = stage.damageFactors[j];
            if (df.damageDef == dinfo.Def)
                factor *= df.factor;  // 乘算累积
        }
    }

    cachedDamageFactors.Add(dinfo.Def, factor);
    return factor;
}
```

**XML配置示例**：
```xml
<HediffDef>
    <defName>TrionShield</defName>
    <stages>
        <li>
            <damageFactors>
                <li>
                    <damageDef>Bullet</damageDef>
                    <factor>0.5</factor>  <!-- 子弹伤害×50% -->
                </li>
            </damageFactors>
        </li>
    </stages>
</HediffDef>
```

> **Hediff vs Gene的区别**：Hediff的damageFactors是**按Stage**配置的——不同严重度阶段可以有不同的伤害因子。Gene的damageFactors是**全局**的——只要基因Active就生效。

### 7.4 IncomingDamageFactor Stat

```csharp
// DamageWorker_AddInjury.ApplyDamageToPart (Line 151)
// 在护甲计算之后应用
num *= pawn.GetStatValue(StatDefOf.IncomingDamageFactor);
```

- **默认值**：1.0（100%）
- **最小值**：0
- **生效条件**：仅`ExternalViolenceFor(pawn)`为true时应用
- **影响来源**：可被基因statOffsets/statFactors、Hediff stages、装备equippedStatOffsets等修改

### 7.5 ThingDef.damageMultipliers

```csharp
// Thing.TakeDamage (阶段3)
if (def.damageMultipliers != null)
{
    for (int i = 0; i < def.damageMultipliers.Count; i++)
    {
        if (def.damageMultipliers[i].damageDef == dinfo.Def)
            dinfo.SetAmount(dinfo.Amount * def.damageMultipliers[i].multiplier);
    }
}
```

这是Thing级别的伤害倍率，在TakeDamage入口处（阶段3）应用，早于所有其他伤害因子。

### 7.6 伤害因子生效顺序总结

| # | 层级 | 位置 | 护甲前/后 | 累积方式 | 配置方式 |
|---|------|------|----------|---------|---------|
| 1 | ThingDef.damageMultipliers | 阶段3 | 护甲前 | 乘算 | XML |
| 2 | Gene damageFactors | 阶段4 | 护甲前 | 乘算 | XML |
| 3 | Hediff damageFactors | 阶段4 | 护甲前 | 乘算 | XML |
| 4 | IncomingDamageFactor | 阶段5 | 护甲后 | Stat系统 | Stat修改 |

> **模组设计建议**：
> - 需要"减少某类伤害"→ 用Gene/Hediff的damageFactors（护甲前，针对特定DamageDef）
> - 需要"全局减伤"→ 用IncomingDamageFactor stat（护甲后，影响所有伤害）
> - 两者可叠加——先乘基因/Hediff倍率，再过护甲，最后乘IncomingDamageFactor
