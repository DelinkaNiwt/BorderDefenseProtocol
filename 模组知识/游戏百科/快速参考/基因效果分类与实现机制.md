# 基因效果分类与实现机制

**总览**：基因效果通过两条路径实现——**GeneDef字段配置**（大多数效果，纯XML）和**Gene子类行为**（少数需要运行时逻辑的效果，需C#）。GeneDef拥有80+字段，覆盖属性修改、能力赋予、外观改变、免疫、行为限制等10+种效果类型。

## GeneDef效果字段完整分类表

### 属性修改类

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `statOffsets` | `List<StatModifier>` | null | 属性加值（如MoveSpeed +0.5） |
| `statFactors` | `List<StatModifier>` | null | 属性倍率（如MeleeDamageFactor ×1.5） |
| `conditionalStatAffecters` | `List<ConditionalStatAffecter>` | null | 条件属性修改（如"在阳光下"时） |
| `painOffset` | `float` | 0 | 疼痛加值 |
| `painFactor` | `float` | 1 | 疼痛倍率 |
| `capMods` | `List<PawnCapacityModifier>` | null | 能力修改器（视力、操作等） |

### 能力/特质/Need类

| 字段 | 类型 | 说明 |
|------|------|------|
| `abilities` | `List<AbilityDef>` | 赋予的能力列表 |
| `forcedTraits` | `List<GeneticTraitData>` | 强制添加的特质 |
| `suppressedTraits` | `List<GeneticTraitData>` | 压制的特质 |
| `enablesNeeds` | `List<NeedDef>` | 启用的Need |

## Gene子类行为效果详表

| Gene子类 | 核心行为 | 关键方法 |
|---------|---------|---------|
| `Gene_Resource`（抽象） | 资源追踪：cur/max/targetValue + GeneGizmo_Resource UI | `PostAdd`→Reset, `GetGizmos`→资源条 |
| `Gene_Hemogen` | 血源素：每日消耗 + 进食人肉补充 + 自动消耗血源素包 | `TickInterval`→消耗, `Notify_IngestedThing`→补充 |
| `Gene_HemogenDrain` | 血源素消耗源：实现IGeneResourceDrain接口 | `ResourceLossPerDay`属性 |
| `Gene_Deathrest` | 死眠系统管理：死眠需求、死眠建筑效果、死眠Hediff | `PostAdd`→添加Need_Deathrest |
| `Gene_Bloodfeeder` | 吸血能力：添加吸血能力、吸血后补充Hemogen | `PostAdd`→添加Bloodfeed能力 |
| `Gene_ChemicalDependency` | 化学品依赖：定期检查是否满足化学品需求 | `PostAdd`→添加Hediff_ChemicalDependency |
| `Gene_Clotting` | 凝血/止血：加速伤口愈合 | `Tick`→检查并治愈出血伤口 |

## 开发者要点

1. **优先用GeneDef字段**：80+字段覆盖绝大多数效果需求，只有需要运行时逻辑（如周期性检查、资源追踪）才需要自定义Gene子类
2. **statOffsets vs statFactors**：加值用`statOffsets`，倍率用`statFactors`，两者可同时使用且独立计算
3. **conditionalStatAffecters实现条件效果**：如"在阳光下移动速度-50%"，无需C#代码
4. **forcedTraits/suppressedTraits是双向的**：基因可以强制添加特质，也可以压制已有特质
5. **abilities字段直接赋予能力**：无需自定义Gene子类，XML中列出AbilityDef即可
