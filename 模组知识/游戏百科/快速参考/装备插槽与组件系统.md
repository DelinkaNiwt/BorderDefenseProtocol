# 装备插槽与组件系统

**总览**：RimWorld中"装备上有插槽/组件"的概念由3个DLC系统实现——**Odyssey独特武器**（CompUniqueWeapon，1-3个随机特质=插槽）、**Royalty心灵武器**（CompBladelinkWeapon，1-2个特质+绑定机制）、**Anomaly装备能力**（CompEquippableAbilityReloadable，充能/弹药附加能力）。三者共享`WeaponTraitDef`定义层，通过`WeaponCategoryDef`区分适用范围。

## 1. 三大DLC系统对比总表

| 维度 | Odyssey独特武器 | Royalty心灵武器 | Anomaly装备能力 |
|------|----------------|----------------|-----------------|
| **核心Comp** | `CompUniqueWeapon` | `CompBladelinkWeapon` | `CompEquippableAbilityReloadable` |
| **继承关系** | `ThingComp` | `CompBiocodable` | `CompEquippableAbility` |
| **特质数量** | 1-3个 | 1-2个 | 0（能力而非特质） |
| **特质来源** | `WeaponTraitDef`（多类别） | `WeaponTraitDef`（仅BladeLink类别） | `CompProperties`配置 |
| **绑定机制** | 无 | Biocode绑定+bondedHediffs | 无 |
| **品质** | 强制Super品质 | 无特殊品质 | 继承基础武器品质 |
| **命名系统** | 随机生成独特名称+颜色 | 无自定义名称 | 无 |
| **弹药/充能** | 特质可注入abilityProps | 无 | ammoDef+maxCharges |
| **DLC检查** | `ModLister.CheckOdyssey` | `ModLister.CheckRoyalty` | 无DLC限制 |
| **"插槽"类比** | 武器类别=插槽类型，特质=组件 | 固定BladeLink类别 | 单一能力槽 |

## 2. ★ Odyssey独特武器系统（核心重点）

Odyssey的独特武器系统是RimWorld中最接近"装备插槽/组件"概念的实现：
- **WeaponCategoryDef** = "插槽类型"（决定哪些武器接受哪些组件）
- **exclusionTags** = "插槽冲突"（同类组件互斥）
- **WeaponTraitDef** = "可插入的组件"（属性/弹药/能力/伤害类型修改）
- **1-3个特质** = "1-3个插槽位"

### 2.1 CompUniqueWeapon 完整源码分析（310行）

```csharp
public class CompUniqueWeapon : ThingComp
{
    // === 核心数据 ===
    private List<WeaponTraitDef> traits = new List<WeaponTraitDef>();  // 1-3个随机特质
    private ColorDef color;          // 武器颜色（随机或特质强制）
    private string name;             // 独特名称（语法生成）
    private CompStyleable styleable; // 风格化组件引用
    private static readonly IntRange NumTraitsRange = new IntRange(1, 3);  // 特质数量范围
    private bool? ignoreAccuracyMaluses;  // 缓存：是否忽略精度惩罚
}
```

**生命周期方法**：

| 方法 | 触发时机 | 核心行为 |
|------|---------|---------|
| `PostPostMake()` | 物品创建后 | 初始化特质→设品质(Super)→选颜色→生成名称 |
| `PostExposeData()` | 存档读写 | 保存/加载traits、color、name；加载后调用Setup |
| `Notify_Equipped(pawn)` | 装备时 | 遍历所有特质调用Worker.Notify_Equipped |
| `Notify_EquipmentLost(pawn)` | 卸下时 | 遍历所有特质调用Worker.Notify_EquipmentLost |
| `GetStatOffset(stat)` | Stat计算 | 遍历所有特质的statOffsets**累加** |
| `GetStatFactor(stat)` | Stat计算 | 遍历所有特质的statFactors**连乘** |
| `ForceColor()` | 渲染 | 返回color.color |
| `TransformLabel(label)` | 显示名称 | 返回独特名称（优先于原始label） |
| `Setup(fromSave)` | 初始化/加载后 | 特质中有abilityProps时注入到CompEquippableAbilityReloadable |

**PostPostMake() 初始化流程**：

```
PostPostMake()
  ├── ModLister.CheckOdyssey() — DLC检查
  ├── InitializeTraits() — 随机选择1-3个特质
  │   ├── NumTraitsRange.RandomInRange → 确定特质数量
  │   ├── for each slot:
  │   │   ├── allDefs.Where(CanAddTrait) → 过滤可用特质
  │   │   └── RandomElementByWeight(commonality) → 按权重随机选择
  │   └── Setup(fromSave: false) — 注入abilityProps
  ├── CompQuality.SetQuality(Super) — 强制Super品质
  ├── 随机选择武器颜色（ColorType.Weapon且randomlyPickable）
  ├── 遍历特质：forcedColor覆盖颜色，收集traitAdjectives
  └── GrammarRequest + NamerUniqueWeapon → 生成独特名称
```

**Setup() 能力注入机制**：
```csharp
public void Setup(bool fromSave) {
    foreach (WeaponTraitDef trait in traits) {
        if (trait.abilityProps != null) {
            // 找到武器上的CompEquippableAbilityReloadable
            var comp = parent.TryGetComp<CompEquippableAbilityReloadable>();
            comp.props = trait.abilityProps;  // 替换props为特质的abilityProps
            if (!fromSave) comp.Notify_PropsChanged();  // 刷新充能数
        }
    }
}
```
> **关键设计**：特质的`abilityProps`字段可以为武器注入一个完整的充能能力（如榴弹发射器），这是Odyssey系统最强大的扩展点。

**Stat叠加机制**：
```csharp
// 加法叠加：所有特质的statOffsets累加
public override float GetStatOffset(StatDef stat) {
    float num = 0f;
    foreach (var item in TraitsListForReading)
        num += item.statOffsets.GetStatOffsetFromList(stat);
    return num;
}

// 乘法叠加：所有特质的statFactors连乘
public override float GetStatFactor(StatDef stat) {
    float num = 1f;
    foreach (var item in TraitsListForReading)
        num *= item.statFactors.GetStatFactorFromList(stat);
    return num;
}
```

**Overlaps() 互斥检查**：
```csharp
public bool Overlaps(WeaponTraitDef other) {
    if (other == this) return true;  // 同一个特质
    if (exclusionTags.NullOrEmpty() || other.exclusionTags.NullOrEmpty()) return false;
    return exclusionTags.Any(x => other.exclusionTags.Contains(x));  // 任一标签重叠即互斥
}
```

### 2.3 WeaponTraitWorker 生命周期（72行，5个回调+1个空方法）

```csharp
public class WeaponTraitWorker {
    public WeaponTraitDef def;

    // 5个生命周期回调
    public virtual void Notify_Bonded(Pawn pawn);        // 绑定时：施加bondedHediffs到大脑
    public virtual void Notify_Equipped(Pawn pawn);       // 装备时：施加equippedHediffs到大脑
    public virtual void Notify_EquipmentLost(Pawn pawn);  // 卸下时：移除equippedHediffs
    public virtual void Notify_KilledPawn(Pawn pawn);     // 击杀时：给予killThought
    public virtual void Notify_Unbonded(Pawn pawn);       // 解绑时：移除bondedHediffs

    // 空方法（供子类重写）
    public virtual void Notify_OtherWeaponWielded(CompBladelinkWeapon weapon);
}
```

### 2.5 特质兼容性检查流程（CanAddTrait）

```
CanAddTrait(trait) — CompUniqueWeapon版本（3步判定）
  ├── ① 武器类别匹配：Props.weaponCategories.Contains(trait.weaponCategory)?
  │   └── 不包含 → return false
  ├── ② 首个特质限制：traits为空 && !trait.canGenerateAlone?
  │   └── 是 → return false（某些特质不能单独存在）
  └── ③ 排斥标签检查：遍历已有traits，任一与新trait Overlaps?
      └── 是 → return false（exclusionTags冲突）
  → 全部通过 → return true

CanAddTrait(trait) — CompBladelinkWeapon版本（2步判定）
  ├── ① 固定类别：trait.weaponCategory != BladeLink?
  │   └── 不是BladeLink → return false
  └── ② 排斥标签检查：同上
  → 全部通过 → return true
```

## 3. Royalty心灵武器系统

### 3.1 CompBladelinkWeapon 源码要点（285行）

```csharp
public class CompBladelinkWeapon : CompBiocodable  // 继承Biocode绑定基类
{
    private int lastKillTick = -1;                          // 上次击杀时间
    private List<WeaponTraitDef> traits = new List<>();     // 1-2个特质
    private static readonly IntRange TraitsRange = new IntRange(1, 2);  // 比Odyssey少
}
```

**与Odyssey系统的关键差异**：

| 维度 | CompBladelinkWeapon | CompUniqueWeapon |
|------|--------------------|--------------------|
| 继承 | `CompBiocodable`（绑定机制） | `ThingComp`（无绑定） |
| 特质数 | 1-2个 | 1-3个 |
| 类别过滤 | 硬编码`BladeLink` | 配置`weaponCategories`列表 |
| 绑定机制 | Biocode绑定+bondedHediffs/Thought | 无 |
| 击杀追踪 | `lastKillTick`+`TicksSinceLastKill` | 无 |
| 品质控制 | 无特殊品质 | 强制Super品质 |
| 命名系统 | 无自定义名称 | 随机生成独特名称+颜色 |
| Stat修改 | 无GetStatOffset/Factor | 有（累加/连乘） |
| 能力注入 | 无 | Setup()注入abilityProps |

**绑定机制流程**：
```
装备心灵武器 → Notify_Equipped()
  ├── base.Notify_Equipped() → CompBiocodable检查
  └── 遍历traits → Worker.Notify_Equipped()

首次装备（未绑定）→ CodeFor(pawn)
  ├── 发送绑定信件
  ├── base.CodeFor() → 设置CodedPawn
  └── OnCodedFor() → lastKillTick重置 + bondedWeapon设置 + Worker.Notify_Bonded()

解绑/摧毁 → UnCode()
  ├── bondedWeapon = null
  ├── 遍历traits → Worker.Notify_Unbonded()
  └── base.UnCode() → 清除CodedPawn
```

```
使用能力 → UsedOnce()
  ├── RemainingCharges-- (由Ability系统处理)
  └── replenishAfterCooldown? → replenishInTicks = baseReloadTicks

CompTick() 每tick检查
  └── replenishAfterCooldown && RemainingCharges == 0?
      ├── replenishInTicks > 0 → replenishInTicks--
      └── replenishInTicks <= 0 → RemainingCharges = MaxCharges（自动恢复）

手动装填 → ReloadFrom(ammo)
  ├── ammoCountToRefill != 0 → 消耗固定数量，充满
  └── ammoCountToRefill == 0 → 按ammoCountPerCharge逐次充能
```

### 4.2 地狱猫步枪XML完整解析

**基础版**（Anomaly DLC，Gun_HellcatRifle）：
```xml
<li Class="RimWorld.CompProperties_EquippableAbilityReloadable">
    <abilityDef>HellcatBurner</abilityDef>    <!-- 火焰喷射能力 -->
    <maxCharges>2</maxCharges>                  <!-- 2次充能 -->
    <ammoDef>Bioferrite</ammoDef>               <!-- 弹药：生物铁 -->
    <ammoCountPerCharge>10</ammoCountPerCharge>  <!-- 每次10个生物铁 -->
    <baseReloadTicks>90</baseReloadTicks>        <!-- 装填时间90ticks -->
    <chargeNoun>charge</chargeNoun>
</li>
```

**独特版**（Odyssey DLC，Gun_HellcatRifle_Unique）：
- 继承基础版的所有配置
- 额外添加`CompProperties_UniqueWeapon`
- 属于7个武器类别（Ranged/BulletFiring/Rifle/Sighted/BurstFire/Gun/Attachable）
- 如果随机到Attachment类特质（如GrenadeLauncher），其`abilityProps`会**覆盖**原始的HellcatBurner配置

## 5. 共享架构：WeaponTraitDef

Royalty和Odyssey**共用同一个`WeaponTraitDef`类**，通过`WeaponCategoryDef`区分适用范围：

```
WeaponTraitDef（共享定义层）
  ├── weaponCategory = BladeLink → 心灵武器专用
  └── weaponCategory = Ranged/BulletFiring/... → 独特武器专用

CompBladelinkWeapon.CanAddTrait()
  └── 硬编码检查 trait.weaponCategory == BladeLink

CompUniqueWeapon.CanAddTrait()
  └── 配置检查 Props.weaponCategories.Contains(trait.weaponCategory)
```

**Worker属性的DLC检查**：
```csharp
public WeaponTraitWorker Worker {
    get {
        if (!ModLister.CheckRoyaltyOrOdyssey("Weapon traits"))  // 两个DLC都可用
            return null;
        // ...
    }
}
```

> **设计启示**：WeaponTraitDef是一个优秀的"共享定义+分类过滤"架构范例——同一个Def类型服务于多个系统，通过Category字段实现隔离。
