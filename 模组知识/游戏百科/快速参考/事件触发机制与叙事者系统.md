# 事件触发机制与叙事者系统

**总览**：RimWorld的事件触发由三层系统驱动——`Storyteller`（调度层，每1000 ticks执行主循环）遍历所有`StorytellerComp`（生成层，21个子类各自独立生成`FiringIncident`），最终通过`TryFire`执行事件（`IncidentWorker.CanFireNow` + `TryExecute`）。威胁点数由`StorytellerUtility.DefaultThreatPointsNow()`根据殖民地财富、Pawn战力、适应度、难度倍率综合计算。

## 一、Storyteller主循环

### 核心调用链

```
Storyteller.StorytellerTick()                    ← 每1000 ticks执行
  ├── incidentQueue.IncidentQueueTick()          ← 处理排队事件
  └── MakeIncidentsForInterval()                 ← 生成本轮事件
      ├── 遍历所有 StorytellerComp:
      │   ├── minDaysPassed检查
      │   ├── allowedTargetTags / disallowedTargetTags过滤
      │   ├── allowBigThreats + Anomaly冷却检查
      │   └── comp.MakeIntervalIncidents(target)
      ├── 遍历所有进行中Quest的IIncidentMakerQuestPart
      └── TryFire(FiringIncident)
          ├── CanFireNow(parms)  ← 15+项前置检查
          ├── TryExecute(parms)  ← 执行事件逻辑
          └── StoryState.Notify_IncidentFired(fi)
```

### 关键常量

| 常量 | 值 | 说明 |
|------|---|------|
| `CheckInterval` | 1000 ticks | 主循环检查间隔 |
| `IntervalsPerDay` | 60 | 每天60个检查间隔（60000/1000） |
| `GlobalPointsMax` | 10000 | 威胁点数上限 |
| `GlobalPointsMinRangeFloor` | 35 | 威胁点数下限底值 |

## 二、StorytellerComp_RandomMain（Randy使用）

### 核心逻辑

Randy的事件调度完全基于MTB（Mean Time Between）随机机制：

```
MakeIntervalIncidents(target):
  1. MTB检查: Rand.MTBEventOccurs(mtbDays=1.35, 60000, 1000)
     → 未通过则本次不触发
  2. 选择类别: ChooseRandomCategory()
     → 距上次ThreatBig超过maxThreatBigIntervalDays(13天)? 强制ThreatBig
     → 否则按categoryWeights加权随机
  3. 生成参数: DefaultThreatPointsNow() × randomPointsFactorRange(0.5~1.5)
  4. 选择事件: UsableIncidentsInCategory() → TryRandomElementByWeight(baseChance)
  5. 失败重试: 该类别无可用事件 → 尝试下一个类别，直到所有类别都试过
```

### Properties字段

| 字段 | Randy配置 | 说明 |
|------|----------|------|
| `mtbDays` | 1.35 | MTB天数（平均1.35天触发一次） |
| `categoryWeights` | 见下表 | 类别权重 |
| `maxThreatBigIntervalDays` | 13 | 大威胁最大间隔（超过则强制） |
| `randomPointsFactorRange` | 0.5~1.5 | 点数随机乘数 |
| `skipThreatBigIfRaidBeacon` | true | RaidBeacon跳过大威胁 |
| `spaceMtbDayFactor` | 2 | 太空MTB倍率 |
| `spaceMinSpacingDays` | 2 | 太空最小间隔 |

### Randy categoryWeights

| 类别 | 权重 | 占比 |
|------|------|------|
| Misc | 3.5 | 37.8% |
| FactionArrival | 2.4 | 25.9% |
| ThreatBig | 1.4 | 15.1% |
| OrbitalVisitor | 1.1 | 11.9% |
| ThreatSmall | 0.6 | 6.5% |
| ShipChunkDrop | 0.22 | 2.4% |

## 三、StorytellerComp_OnOffCycle（Cassandra/Phoebe使用）

### 核心逻辑

Cassandra/Phoebe的事件调度基于确定性的开关周期：

```
MakeIntervalIncidents(target):
  1. 计算acceptFraction（接受率）:
     × acceptFractionByDaysPassedCurve(天数)
     × acceptPercentFactorPerThreatPointsCurve(威胁点数)
     × acceptPercentFactorPerProgressScoreCurve(进度分数)
  2. 调用IncidentCycleUtility.IncidentCountThisInterval():
     → 根据onDays/offDays周期、numIncidentsRange、acceptFraction
     → 确定性计算本次间隔应触发的事件数
  3. 生成事件:
     → 早期强制: DaysPassedSinceSettle < forceRaidEnemyBeforeDaysPassed → 强制RaidEnemy
     → 指定事件: Props.incident不为null → 使用指定事件
     → 随机选择: UsableIncidentsInCategory() → TryRandomElementByWeight
```

### 三大叙事者ThreatBig配置对比

| 字段 | Cassandra | Phoebe | 说明 |
|------|-----------|--------|------|
| `minDaysPassed` | 11.0 | 13.0 | 最早触发天数 |
| `onDays` | 4.6 | 8.0 | "开"期天数 |
| `offDays` | 6.0 | 8.0 | "关"期天数 |
| `minSpacingDays` | 1.9 | — | 最小间隔天数 |
| `numIncidentsRange` | 1~2 | 1~1 | 每个开期事件数 |
| `forceRaidEnemyBeforeDaysPassed` | 20 | 20 | 强制敌袭天数 |
| **周期总长** | **10.6天** | **16天** | onDays + offDays |

## 四、威胁点数计算

### DefaultThreatPointsNow() 完整公式

```csharp
// 源码：StorytellerUtility.cs
最终点数 = Clamp(
  (财富点数 + Σ Pawn点数) × 随机因子 × 适应因子 × threatScale × 天数因子,
  GlobalPointsMin(), 10000
)
```

### 各因子详解

#### 1. 财富点数（PointsPerWealthCurve）

| 殖民地财富 | 0 | 14,000 | 400,000 | 700,000 | 1,000,000 |
|-----------|---|--------|---------|---------|-----------|
| 财富点数 | 0 | 0 | 2,400 | 3,600 | 4,200 |

#### 2. 殖民者点数（PointsPerColonistByWealthCurve）

每个自由殖民者贡献的点数随财富增长：

| 殖民地财富 | 0 | 10,000 | 400,000 | 1,000,000 |
|-----------|---|--------|---------|-----------|
| 每殖民者点数 | 15 | 15 | 140 | 200 |

#### 3. 其他Pawn点数

| Pawn类型 | 计算公式 | 说明 |
|---------|---------|------|
| 可释放动物 | `0.08 × combatPower` | 商队中×0.7 |
| 殖民地机械体 | `combatPower × PointsFactorForColonyMechsCurve` | 0→0.2, 400000→0.3, 1000000→0.4 |
| 亚人类 | `combatPower × PointsFactorForColonySubhumanCurve` | 同机械体曲线 |

#### 4. Pawn点数修正

| 修正 | 倍率 | 条件 |
|------|------|------|
| 冬眠舱中 | ×0.3 | `ParentHolder is Building_CryptosleepCasket` |
| 健康状态 | `Lerp(1.0, healthPercent, 0.65)` | 受伤Pawn贡献降低 |
| 奴隶 | ×0.75 | `IsSlaveOfColony` |
| 年龄（Biotech） | 3岁→0, 13岁→0.5, 18岁→1.0 | 儿童贡献降低 |

#### 5. 随机因子

| 目标类型 | 范围 | 说明 |
|---------|------|------|
| 地图（默认） | 0.8~1.2 | `IncidentPointsRandomFactorRange` |
| 商队 | 0.7~0.9 | `CaravanPointsRandomFactorRange` |
| 重力飞船 | 0.7~0.9 | `GravshipPointsRandomFactorRange` |

#### 6. 天数因子（pointsFactorFromDaysPassed）

| 定居天数 | 10 | 40+ |
|---------|----|----|
| 天数因子 | 0.70 | 1.00 |

## 五、适应系统（StoryWatcher_Adaptation）

### 核心机制

`adaptDays`是适应系统的核心值，通过`pointsFactorFromAdaptDays`曲线转换为威胁倍率。

### adaptDays变化机制

| 机制 | 触发条件 | 计算 |
|------|---------|------|
| **自然增长** | 每30000 ticks（半天） | `+= 0.5 × growthRateCurve(adaptDays) × adaptationGrowthRateFactorOverZero` |
| **殖民者死亡** | 殖民者死亡时 | `-= lossFromColonistLost(postPopulation)` |
| **殖民者倒下** | 暴力倒下时 | `-= lossFromViolentlyDowned(population)` |
| **开局保护** | 前30天 | adaptDays≥0时不增长 |

### adaptDaysGrowthRateCurve（增长速率曲线）

| adaptDays | <-30 | -30 | 0 | 60 | 120 |
|-----------|------|-----|---|----|----|
| 增长速率 | 6→3 | 3 | 1 | 0.5 | 0.25 |

实际增长 = 0.5 × 速率 × adaptationGrowthRateFactorOverZero（adaptDays>0时）

### adaptDaysLossFromColonistLostByPostPopulation（死亡损失）

| 死后人口 | 0~11 | 20 |
|---------|------|-----|
| adaptDays损失 | 30天 | 20天 |

### adaptDaysLossFromColonistViolentlyDownedByPopulation（倒下损失）

| 当前人口 | 1 | 2~3 | 11 | 20 |
|---------|---|-----|----|----|
| adaptDays损失 | 8天 | 6天 | 5天 | 3.5天 |

### pointsFactorFromAdaptDays（适应天数→威胁倍率）

| adaptDays | -30 | 0 | 30 | 60 | 120 | 180 |
|-----------|-----|---|----|----|-----|-----|
| 威胁倍率 | 0.40 | 0.80 | 1.00 | 1.20 | 1.60 | 2.00 |

### 范围限制

| 参数 | 值 |
|------|---|
| `adaptDaysMin` | -60 |
| `adaptDaysMax` | 100 |
| `adaptDaysGameStartGraceDays` | 30 |

## 七、冷却/重触发机制

### FiredTooRecently()

```csharp
// 源码：IncidentWorker.cs
public bool FiredTooRecently(IIncidentTarget target)
{
    Dictionary<IncidentDef, int> lastFireTicks = target.StoryState.lastFireTicks;
    int ticksGame = Find.TickManager.TicksGame;
    // 检查自身冷却
    if (lastFireTicks.TryGetValue(def, out var value)
        && (float)(ticksGame - value) / 60000f < def.minRefireDays)
        return true;
    // 检查关联事件冷却
    List<IncidentDef> refireCheckIncidents = def.RefireCheckIncidents;
    if (refireCheckIncidents != null)
    {
        for (int i = 0; i < refireCheckIncidents.Count; i++)
        {
            if (lastFireTicks.TryGetValue(refireCheckIncidents[i], out value)
                && (float)(ticksGame - value) / 60000f < def.minRefireDays)
                return true;
        }
    }
    return false;
}
```

### 关键机制

- `lastFireTicks`：每个IIncidentTarget维护的字典，记录每个IncidentDef的上次触发tick
- `minRefireDays`：IncidentDef字段，最小重触发天数
- `RefireCheckIncidents`：关联冷却列表，任一关联事件未冷却则阻止触发

```csharp
    {
        // 事件执行逻辑

```

### 4种融入方式

| # | 方式 | 实现 | 适用场景 |
|---|------|------|---------|
