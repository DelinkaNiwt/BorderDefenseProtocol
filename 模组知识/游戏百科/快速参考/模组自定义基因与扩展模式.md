# 模组自定义基因与扩展模式

**总览**：社区模组扩展基因系统主要有4种模式——**纯XML配置**（最简单，利用GeneDef 80+字段）、**DefModExtension扩展**（中等复杂度，VEF的GeneExtension是标杆）、**自定义Gene子类**（最灵活，需C#编程）、**Harmony补丁**（侵入式，修改原版行为）。大多数模组组合使用多种模式。

## 4种扩展模式对比

| # | 模式 | 实现方式 | 需要C# | 代表模组 | 适用场景 |
|---|------|---------|--------|---------|---------|
| 1 | **纯XML配置** | 利用GeneDef原生80+字段 | 否 | Altered Carbon | 属性修改、能力赋予、外观改变等标准效果 |
| 2 | **DefModExtension** | 自定义扩展字段挂载到GeneDef | 是 | VEF、AutoBlink | 原版字段不够用，需要额外数据但不需要运行时逻辑 |
| 3 | **自定义Gene子类** | 继承Gene，重写生命周期方法 | 是 | AutoBlink、AncotLibrary | 需要运行时逻辑（Tick、条件触发、动态Comp注入） |
| 4 | **Harmony补丁** | 拦截原版方法注入自定义逻辑 | 是 | VEF、HAR | 修改原版基因行为、种族限制、UI扩展 |

**模式选择决策树**：

```
原版GeneDef字段能满足？ → 是 → 模式1：纯XML
                        → 否 → 需要运行时逻辑？ → 否 → 模式2：DefModExtension
                                                → 是 → 需要修改原版行为？ → 否 → 模式3：Gene子类
                                                                          → 是 → 模式4：Harmony补丁
```

## 代表性模组技术方案

| # | 模组 | 核心技术 | 扩展模式 | 特色 |
|---|------|---------|---------|------|
| 1 | **VEF** (Vanilla Expanded Framework) | `GeneExtension : DefModExtension` + Harmony补丁 | 2+4 | 50+扩展字段，社区事实标准 |
| 2 | **Altered Carbon 2** | 纯XML + `GeneUtils`工具类 | 1+辅助C# | 意识堆栈基因，纯XML定义+C#工具类辅助 |
| 3 | **AutoBlink** (自动折跃) | `Gene_AutoBlink : Gene` + 运行时Comp注入 | 3 | 基因激活时动态注入ThingComp到Pawn |
| 4 | **HAR** (Humanoid Alien Races) | `RaceRestrictionSettings.CanHaveGene()` | 4 | 种族级基因过滤，白名单/黑名单机制 |
| 5 | **AncotLibrary** | `Gene_GainHediff_UnderHealthPct : Gene` | 3 | 条件触发型：血量低于阈值时自动施加Hediff |
| 6 | **Character Editor** | 反射+直接操作`Pawn_GeneTracker` | 工具型 | 编辑器模组，运行时增删基因 |

## VEF GeneExtension 关键字段分类

## 自定义Gene子类典型模式

### 模式A：运行时Comp注入（AutoBlink）

基因激活时动态创建ThingComp并注入Pawn，移除时清理。适合需要持续运行逻辑的场景。

```csharp
public class Gene_AutoBlink : Gene
{
    private CompAutoBlink runtimeComp;

    public override void PostAdd()
    {
        base.PostAdd();
        // 创建Comp并注入Pawn.AllComps
        runtimeComp = new CompAutoBlink { parent = pawn, props = Ext.ToThingCompProps() };
        pawn.AllComps.Add(runtimeComp);
    }

    public override void PostRemove()
    {
        base.PostRemove();
        pawn.AllComps.Remove(runtimeComp); // 清理
    }

    public override void Tick()
    {
        base.Tick();
        if (Active != previouslyActive) // 响应Active状态变化
        { /* 动态附加/分离Comp */ }
    }
}
```

**关键点**：必须处理`ExposeData()`保存/加载，加载后重新设置`parent`和`props`。

### 模式B：条件触发型（AncotLibrary）

基因在Tick中检测条件，满足时施加效果。适合"血量低于X时触发"等场景。

```csharp
public class Gene_GainHediff_UnderHealthPct : Gene
{
    private int cooldownTicksLeft = 0;
    public ModExtension_GeneHediff Props => def.GetModExtension<ModExtension_GeneHediff>();

    public override void TickInterval(int delta)
    {
        base.TickInterval(delta);
        cooldownTicksLeft -= delta;
        if (cooldownTicksLeft < 0 && pawn.health.summaryHealth.SummaryHealthPercent < Props.healthPctThreshold)
        {
            HealthUtility.AdjustSeverity(pawn, Props.hediff, Props.severity);
            cooldownTicksLeft = Props.cooldownTicks;
        }
    }
}
```

**关键点**：使用`TickInterval`而非`Tick`减少性能开销；配合DefModExtension存储配置参数。

## 常见陷阱与注意事项

### 1. Gene.Active ≠ 基因存在

基因被`exclusionTags`覆盖时`Active=false`但仍在列表中。自定义逻辑必须检查`Active`状态：

```csharp
// 错误：基因存在就执行
if (pawn.genes.HasGene(myGeneDef)) { DoEffect(); }

// 正确：检查Active状态
Gene gene = pawn.genes.GetGene(myGeneDef);
if (gene != null && gene.Active) { DoEffect(); }
```

### 2. PostAdd/PostRemove不等于激活/停用

`PostAdd()`在基因添加到列表时调用，此时基因可能因覆盖而`Active=false`。需要在`Tick()`中监控`Active`状态变化（参考AutoBlink的`previouslyActive`模式）。

### 3. ExposeData必须完整

自定义Gene子类的所有运行时字段都必须在`ExposeData()`中保存/加载，否则存档后状态丢失。特别注意动态注入的Comp需要`Scribe_Deep.Look()`。

### 4. DefModExtension的null检查

`def.GetModExtension<T>()`在XML未配置时返回null。所有访问必须null检查：

```csharp
// 安全访问模式
public GeneExtension Ext => def?.GetModExtension<GeneExtension>();
if (Ext?.hediffToWholeBody != null) { /* ... */ }
```

### 5. Harmony补丁的加载顺序

多个模组可能补丁同一方法。VEF通过`Notify_GenesChanged_Patch`、`Gene_PostAdd_Patch`等Postfix补丁扩展基因系统。自定义补丁需注意与VEF的兼容性，避免覆盖其效果。

### 6. 性能：避免每Tick重复查询

基因的`Tick()`每游戏Tick调用一次（60次/秒）。避免在Tick中执行昂贵操作：

```csharp
// 错误：每Tick查询DefModExtension
public override void Tick() {
    var ext = def.GetModExtension<MyExtension>(); // 每Tick反射查询
}

// 正确：缓存引用
private MyExtension cachedExt;
public override void PostAdd() {
    cachedExt = def.GetModExtension<MyExtension>();
}
```

### 7. 种族兼容性

HAR的`CanHaveGene()`会过滤不兼容种族的基因。如果模组基因需要限制种族，应通过HAR的白名单/黑名单XML配置，而非硬编码检查。

## 开发者要点

1. **优先用原版GeneDef字段**——80+字段覆盖绝大多数需求，不要过早引入C#
2. **VEF GeneExtension是社区标准**——如果依赖VEF，优先用其50+扩展字段而非自己造轮子
3. **Gene子类的生命周期方法**：`PostAdd()`→`Tick()`→`PostRemove()`，加上`ExposeData()`存档
4. **Active状态是核心**——所有自定义逻辑必须尊重`Gene.Active`，处理覆盖/恢复场景
5. **性能敏感**——Tick中缓存引用、用`TickInterval`替代`Tick`、避免每帧反射查询
