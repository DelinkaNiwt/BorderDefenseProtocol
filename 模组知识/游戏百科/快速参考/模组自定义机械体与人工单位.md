# 模组自定义机械体与人工单位

**总览**：社区模组通过**4种扩展模式**自定义机械体——纯XML继承（改数值/外观）、自定义Comp（新行为/新机制）、自定义Verb+PawnFlyer（新攻击方式/位移攻击）、完整子系统（晋升/贸易/驯服等跨多类协作的系统）。本文分析3个代表性模组：米莉拉（棋子主题体系化设计）、闪耀毁灭者5（原版强化变体+特殊能力）、机械族全面战争（大规模单位库+多种独创系统）。

## 1. 米莉拉(Milira)——棋子主题机械体系

### 1.1 单位体系

**棋子主题命名**：所有单位按国际象棋棋子分类，每类有多个等级变体。

**抽象模板继承链**：
```
BaseMechanoid（原版）
  └── Milira自定义抽象模板（Milian_BasePawn / Milian_BaseKnight / ...）
        └── 具体单位（Milian_Mechanoid_PawnI ~ PawnIV / KnightI ~ III / ...）
```

**全单位速查表**：

| 棋子类 | 等级 | 定位 | 特殊机制 |
|--------|------|------|----------|
| **Pawn** | I~IV | 基础步兵 | PawnIV(Captain)有指挥半径CompMechCommandRadius |
| **Knight** | I~III | 冲锋突击 | MiliraPawnFlyer_KnightCharge冲锋攻击 |
| **Bishop** | I~III | 远程支援 | 远程攻击+晋升目标 |
| **Rook** | I~IV | 重装防御 | RookIV(Eternal Fortress)有堡垒模式+浮空系统 |
| **King** | — | Boss级 | CompClassAmplificationLoop职阶增幅循环 |

独立派系：`Milira_Faction`，有自定义世界据点（`WorldObjectCompMiliraSettlement`）。

### 1.2 ★ 晋升系统（CompSendPromotionRequest + CompClassAmplificationLoop）

**核心思路**：高级单位死亡时，向附近低级Pawn发送"晋升请求"，通过Hediff标记职阶，实现战场动态升级。

**流程**：
```
高级单位(Bishop/Knight/Rook)死亡
  → CompSendPromotionRequest.PostDestroy()
    → 搜索40格内同派系Pawn类单位（PawnI~PawnIV）
    → 排除已有晋升Hediff的单位
    → 选最近的Pawn → AdjustSeverity(promotionHediffType, 0.01f)
```

```csharp
// CompSendPromotionRequest — 死亡时发送晋升请求
public override void PostDestroy(DestroyMode mode, Map previousMap)
{
    // 仅非玩家派系 + 设置启用 + 地图有效时触发
    if (parent.Faction != Faction.OfPlayer
        && MiliraRaceSettings.MiliraRace_ModSetting_MilianDifficulty_Promotion
        && previousMap != null)
    {
        SendPromotionToPawnClosest(previousMap);
    }
}

protected void SendPromotionToPawnClosest(Map map)
{
    // 13种晋升Hediff（Bishop/Knight/Rook各4级 + Queen）
    List<HediffDef> list = new List<HediffDef> { ... };

    foreach (Pawn item in map.mapPawns.AllPawnsSpawned)
    {
        if (IsPawnAffected(item))  // 同派系的PawnI~PawnIV
        {
            float num = item.Position.DistanceTo(parent.Position);
            // 选最近且未在晋升中的Pawn
            if (num < closestDistance && !list.Any(h => item.health.hediffSet.HasHediff(h)))
            {
                closestPawn = item; closestDistance = num;
            }
        }
    }
    if (closestPawn != null && (closestDistance < 40f || WidePromotion设置))
    {
        // 快速晋升：+0.4f，普通：+0.01f
        HealthUtility.AdjustSeverity(closestPawn, Props.promotionHediffType,
            FastPromotion ? 0.4f : 0.01f);
    }
}
```

**CompClassAmplificationLoop**（King单位专属）：周期性检查附近同派系机械体，重置其职阶Hediff的Severity为0.1f，并附加`Milian_KingTuningWeak`增益。本质是King对周围单位的"职阶维持"——King存在时，周围单位保持增强状态。

**设计模式**：**死亡触发 + Hediff标记 + Severity驱动**——用PostDestroy()触发晋升请求，用HediffDef区分晋升目标职阶（Bishop/Knight/Rook各4级），用Severity控制晋升进度。这种"死亡→附近单位升级"的模式可复用于任何"单位阵亡触发队友强化"场景。

### 1.3 ★ 自定义PawnFlyer（MiliraPawnFlyer继承体系）

**核心思路**：完全自定义的PawnFlyer系统（继承Thing而非原版PawnFlyer），实现4种冲锋/跳跃攻击。

**继承体系**：
```
Thing
  └── MiliraPawnFlyer（基类：IThingHolder，管理飞行状态/轨迹/渲染）
        ├── MiliraPawnFlyer_Hammer  — 锤击冲锋（落地AOE伤害）
        ├── MiliraPawnFlyer_Blade   — 刀刃突进（路径上连续斩击）
        ├── MiliraPawnFlyer_KnightCharge — Knight冲锋（直线冲撞）
        ├── MiliraPawnFlyer_Rook    — Rook冲撞（重型碾压）
        └── MiliraPawnFlyer_Lance   — 长枪突刺（精准单体）
```

**基类核心字段**：
- `innerContainer`：ThingOwner，持有飞行中的Pawn
- `startVec` / `destCell`：起点/终点
- `ticksFlightTime` / `ticksFlying`：飞行总时间/已飞行时间
- `RecomputePosition()`：每tick重算位置（groundPos + effectiveHeight）

**设计模式**：**自定义Thing + IThingHolder**——不使用原版PawnFlyer（继承自Thing, IThingHolder），而是完全自建。原因：原版PawnFlyer的飞行轨迹是简单抛物线，Milira需要更复杂的路径控制（直线冲锋、路径斩击等）。每种攻击类型有配套的`MiliraFlyUtility_XXX`工具类负责发起飞行。

### 1.4 特殊机制

**指挥半径**（`CompMechCommandRadius`）：PawnIV(Captain)专属，在指定半径内为同派系单位提供增益。

**堡垒模式**（`JobDriver_FortressMode` + `CompFloatingSystem`）：RookIV(Eternal Fortress)专属，进入固定防御姿态+浮空状态。

**载体系统**（`CompMechCarrier_Consul`）：继承AncotLibrary的`CompMechCarrier_Custom`，可搭载/释放小型单位。

### 2.2 ★ 隐身系统（CompPawnInvisibility）

**模组**：闪耀毁灭者5
**核心思路**：Firefly单位作为"隐身发射器"，周期性为附近同派系机械体附加隐身Hediff（通过原版`HediffComp_Invisibility`实现视觉隐身）。

```csharp
// CompPawnInvisibility — Firefly的区域隐身能力
public class CompPawnInvisibility : ThingComp
{
    public override void CompTick()
    {
        Pawn pawn = parent as Pawn;
        // 能量不足(≤5)或正在充电/关机时禁用
        if (energy != null && energy.CurLevel <= 5 || pawn.CurJobDef == JobDefOf.MechCharge)
        {
            this.abilityActivate = false; return;
        }

        if (flag && this.abilityActivate && !pawn.stances.stunner.StunFromEMP)
        {
            // 搜索范围内同派系机械体（排除自身Firefly）
            IEnumerable<Pawn> targets = from x in pawn.Map.mapPawns.AllPawns
                where x.Position.DistanceTo(pawn.Position) < this.distance
                   && x.RaceProps.IsMechanoid
                   && x.def.defName != "Mech_Firefly"  // 排除其他Firefly
                   && x.Faction == pawn.Faction
                select x;

            // 玩家派系时消耗能量（按目标BodySize计算）
            if (pawn.Faction == Faction.OfPlayer && targets.Count() > 0 && energy != null)
                this.ApplyEnergyCost(targets.ToList());

            foreach (Pawn target in targets)
            {
                // 已有隐身Hediff → 刷新持续时间；没有 → 新增
                Hediff existing = target.health?.hediffSet?.GetFirstHediffOfDef(Props.hediffToAdd);
                if (existing != null)
                    existing.TryGetComp<HediffComp_Disappears>().ticksToDisappear = 600;
                else
                {
                    Hediff hediff = HediffMaker.MakeHediff(Props.hediffToAdd, target);
                    hediff.TryGetComp<HediffComp_Disappears>().ticksToDisappear = 600;
                    target.health.AddHediff(hediff);
                }
            }
        }
    }

    // 能量消耗：每个目标消耗 energyCostNum × BodySize
    public void ApplyEnergyCost(List<Pawn> list) { ... }
}
```

**设计模式**：**区域Buff发射器**——Comp周期性扫描范围内友军，附加/刷新带`HediffComp_Disappears`的临时Hediff。隐身效果本身由原版`HediffComp_Invisibility`实现（Anomaly DLC引入），GD5只负责"谁获得隐身"和"持续多久"。玩家控制时有Gizmo开关+能量消耗。

### 2.3 ★ 观察者链接（CompObserverLink辅助瞄准）

**模组**：闪耀毁灭者5
**核心思路**：Observer单位通过Gizmo选择一个友军目标，周期性为其附加射程增强Hediff，距离近时效果更强。

```csharp
// CompObserverLink — 辅助瞄准链接
public class CompObserverLink : ThingComp
{
    public List<Pawn> tmpPawns = new List<Pawn>();  // 链接目标（最多1个）

    public override void CompTick()
    {
        if (this.tmpPawns.Count > 0 && this.readyToUseTicks > 63)
        {
            // 目标死亡/消失 → 断开链接
            if (!tmpPawns[0].Spawned || tmpPawns[0].Dead) { tmpPawns.Clear(); return; }

            Pawn p = this.tmpPawns[0];
            // 仅对同派系的机械体/人形生效
            if ((p.RaceProps.IsMechanoid || p.RaceProps.Humanlike) && p.Faction == parent.Faction)
            {
                // 附加/刷新射程增强Hediff（90tick持续）
                Hediff hediff = p.health?.hediffSet?.GetFirstHediffOfDef(Props.hediff);
                if (hediff == null) { /* 新增Hediff */ }
                // ★ 距离决定效果强度：近距离Severity=1.0，远距离=0.5
                hediff.Severity = parent.Position.DistanceTo(p.Position) <= Props.range ? 1.0f : 0.5f;
            }
        }
    }

    // Gizmo：点击后进入目标选择模式，选择友军Pawn建立链接
    public override IEnumerable<Gizmo> CompGetGizmosExtra() { ... }

    // 选中Observer时绘制到目标的连线
    public override void PostDraw()
    {
        if (Find.Selector.SelectedObjectsListForReading.Contains(this.parent) && tmpPawns.Count > 0)
            GenDraw.DrawLineBetween(parent.DrawPos, tmpPawns[0].DrawPos);
    }
}
```

**设计模式**：**单目标链接 + 距离衰减**——Gizmo选择目标→Comp周期性维持Hediff→距离决定Severity→Hediff的Stage按Severity分级提供不同属性加成。需要机械师监管（`MechanitorUtility.GetOverseer`检查）。

### 3.2 ★ 驯服系统（CompTameableMech：Gizmo→Job→派系切换）

**核心思路**：对无派系的野生机械体，通过Gizmo按钮发起驯服流程——消耗工业组件，派遣殖民者执行自定义Job，成功后将机械体派系切换为玩家。

```csharp
// CompTameableMech — 机械体驯服系统
public class CompTameableMech : ThingComp
{
    // 驯服条件：无派系 + 是机械体 + 已生成
    public bool CanBeTamed => parent is Pawn { Faction: null } mech
        && mech.RaceProps.IsMechanoid && mech.Spawned;

    public override IEnumerable<Gizmo> CompGetGizmosExtra()
    {
        if (CanBeTamed)
        {
            Command_Action tameCommand = new Command_Action
            {
                defaultLabel = "TameMech".Translate(),
                defaultDesc = "TameMechDesc".Translate(Props.componentsPerTame),
                action = StartTamingProcess,
            };
            // 组件不足 → 禁用按钮
            if (!HasEnoughComponentsInMap())
                tameCommand.Disable("NotEnoughComponents".Translate(Props.componentsPerTame));
            // 无可用驯服者 → 禁用按钮
            else if (FindBestTamer(parent as Pawn) == null)
                tameCommand.Disable("NoAvailableTamer".Translate());
            yield return tameCommand;
        }
    }

    private void StartTamingProcess()
    {
        Pawn tamer = FindBestTamer(mech);
        AddTameDesignation(mech);       // 添加驯服标记
        CreateTameJob(tamer, mech);      // 创建驯服Job
    }

    private void CreateTameJob(Pawn tamer, Pawn mech)
    {
        Job job = JobMaker.MakeJob(TameMechefOf.TW_TameMech, mech);
        job.count = Props.componentsPerTame;
        job.targetB = FindClosestComponents(tamer);  // 找最近的工业组件
        tamer.jobs.TryTakeOrderedJob(job, JobTag.Misc);
    }

    // ★ 驯服成功：切换派系为玩家
    public static void OnTameSuccess(Pawn mech)
    {
        mech.SetFaction(Faction.OfPlayer);
        mech.Map.designationManager.RemoveAllDesignationsOn(mech);
        Messages.Message("MessageMechTamed".Translate(mech.LabelShort), mech, MessageTypeDefOf.PositiveEvent);
    }
}
```

**设计模式**：**Gizmo + Designation + Job + 派系切换**——完整的4步流程：①Comp提供Gizmo按钮（含前置条件检查）→②添加Designation标记目标→③创建自定义Job（JobDriver_TameMech）让殖民者执行→④成功后`SetFaction(Faction.OfPlayer)`。这种"Gizmo触发→Job执行→状态改变"的模式是RimWorld模组中最常见的交互模式。

### 3.3 ★ 自动恢复护盾（CompMechanoidShield：反射重激活+解除倒地）

**核心思路**：敌对机械体定期检查自身状态，如果被停用或倒地，通过反射强制重新激活。使玩家无法通过EMP永久瘫痪这些机械体。

```csharp
// CompMechanoidShield — 自动恢复系统
public class CompMechanoidShield : ThingComp
{
    // ★ 通过反射访问CompMechanoid的私有字段
    private static readonly FieldInfo deactivatedField =
        typeof(CompMechanoid).GetField("deactivated", BindingFlags.Instance | BindingFlags.NonPublic);
    private static readonly FieldInfo activeField =
        typeof(CompMechanoid).GetField("active", BindingFlags.Instance | BindingFlags.NonPublic);

    public override void CompTick()
    {
        Pawn pawn = parent as Pawn;
        // 仅对已生成的敌对机械体生效，按间隔检查
        if (!pawn.Spawned || !pawn.HostileTo(Faction.OfPlayer)
            || Find.TickManager.TicksGame % Props.checkIntervalTicks != 0) return;

        CompMechanoid mechanoidComp = pawn.TryGetComp<CompMechanoid>();
        if (mechanoidComp != null)
        {
            if (IsDeactivated(mechanoidComp))
                ReactivateMech(pawn, mechanoidComp);  // 被停用 → 强制重激活
            if (pawn.Downed)
                UndownMech(pawn);                      // 倒地 → 强制站起
        }
    }

    private static void UndownMech(Pawn pawn)
    {
        pawn.health.Notify_Resurrected();
        pawn.health.forceDowned = false;
        // 移除所有致命Hediff
        pawn.health.hediffSet.hediffs
            .Where(h => h.def.lethalSeverity > 0f).ToList()
            .ForEach(h => pawn.health.RemoveHediff(h));
        pawn.jobs?.StartJob(new Job(JobDefOf.Goto, pawn.Position), JobCondition.InterruptForced);
    }
}
```

**设计模式**：**反射访问私有字段 + 周期性状态修复**——因为`CompMechanoid.deactivated`和`active`是私有字段，无法通过公共API修改，所以用反射强制写入。`UndownMech`通过`Notify_Resurrected()`+移除致命Hediff+重置forceDowned实现"强制站起"。这是一种"作弊级"的恢复机制，适合Boss级敌人。

### 3.4 死亡生成与召唤

**Comp_SpawnPawnsOnDestroy**（死亡生成子单位）：

```csharp
public class Comp_SpawnPawnsOnDestroy : ThingComp

}
```

**CompAbilityEffect_SummonDropPawns**（召唤空投）：继承`CompAbilityEffect`，使用时在施法者附近空投指定种类的机械体援军。通过`DropPodUtility`实现空投舱效果。

### 3.5 机械体贸易系统

**Comp_MechEmployable + Tradeable_MechanoidEmploy**：资本家小铳（Capitalistic Militor）商人系统。

- `Comp_MechEmployable`：附加在可交易的机械体上，标记其为"可雇佣"
- `Tradeable_MechanoidEmploy`：继承`Tradeable`，自定义贸易UI中机械体的显示和交易逻辑
- `Patch_TradeOptions`：Harmony补丁，在贸易对话中注入"雇佣机械体"选项
- `MechanoidTradeUtility`：处理交易完成后的机械体派系切换

**设计模式**：**自定义Tradeable + Harmony补丁注入贸易选项**——因为原版贸易系统不支持"交易Pawn"，所以需要自定义Tradeable子类+补丁注入。

## 4. 4种扩展模式对比与选型指南

| 维度 | 纯XML继承 | 自定义Comp | 自定义Verb+PawnFlyer | 完整子系统 |
|------|----------|-----------|---------------------|-----------|
| **需要C#** | 否 | 是 | 是 | 是 |
| **复杂度** | 低 | 中 | 中~高 | 高 |
| **兼容风险** | 无 | 低 | 低 | 中（反射/补丁） |
| **灵活性** | 低 | 中 | 高 | 最高 |
| **适用场景** | 改数值/外观/武器 | 单一新行为 | 新攻击/位移方式 | 跨系统协作 |
| **典型实例** | GD5 BlackMilitor | GD5隐身/NCL护盾 | Milira冲锋 | Milira晋升/NCL驯服 |

**选型决策树**：

```
需要新机械体单位？
  ├── 只改数值/外观/武器 → 纯XML继承BaseMechanoid系列
  └── 需要新行为？
        ├── 单一新能力（隐身/护盾/生成） → 自定义Comp
        ├── 新攻击方式（射线/冲锋/跳跃） → 自定义Verb或PawnFlyer
        └── 多系统协作（晋升/驯服/贸易） → 完整子系统（Comp+Job+Hediff+Gizmo）
```
