# 袭击策略与到达模式

**总览**：袭击的战术多样性由两个Def系统驱动——`RaidStrategyDef`（10个，决定"怎么打"）和`PawnsArrivalModeDef`（13个，决定"怎么来"）。每个策略定义了可用的到达模式列表（`arriveModes`），Storyteller在策略确定后从该列表中按权重随机选择到达模式。策略通过`RaidStrategyWorker`子类实现具体行为，到达模式通过`PawnsArrivalModeWorker`子类实现放置逻辑。

## 二、各策略Worker行为详解

### RaidStrategyWorker_ImmediateAttack
- **MakeLordJob**：`LordJob_AssaultColony(faction, canKidnap, canTimeoutOrFlee, sappers=false, useAvoidGridSmart=false, canSteal, breachers=false, canPickUpOpportunisticWeapons=false)`
- **最基础的策略**，直接攻击无特殊行为

### RaidStrategyWorker_ImmediateAttackSmart
- **MakeLordJob**：同上但`useAvoidGridSmart=true`
- **避障率30%**：袭击者会避开陷阱和炮塔射界

### RaidStrategyWorker_ImmediateAttackSappers
- **MakeLordJob**：`sappers=true, useAvoidGridSmart=true`
- **Pawn验证**：
  - 第一个Pawn必须是好工兵（`SappersUtility.IsGoodSapper()`）或备用工兵
  - 工兵必须有建筑破坏武器
- **行为**：工兵挖掘路径绕过墙壁和防御，其他袭击者跟随

### RaidStrategyWorker_ImmediateAttackBreaching
- **MakeLordJob**：`breachers=true, useAvoidGridSmart=true`
- **行为**：使用爆破武器（如火箭筒）破坏墙壁，自行决定突破路径

### RaidStrategyWorker_ImmediateAttackBreachingSmart
- **MakeLordJob**：`breachers=true, useAvoidGridSmart=true`
- **行为**：破坏+智能避障的组合

### RaidStrategyWorker_StageThenAttack
- **MakeLordJob**：`LordJob_StageThenAttack`
- **行为**：先在地图边缘集结（LordToil_Stage），等待一段时间后统一发起攻击
- **机械族也使用**：selectionWeightCurvesPerFaction中Mechanoid权重=1.0

### RaidStrategyWorker_Siege
- **MakeLordJob**：`LordJob_Siege(siegeSpot, artilleryPoints)`
- **炮台点数分配**：总点数的20-30%，最少60点
- **要求**：`faction.def.canSiege=true`，不允许动物Pawn
- **行为**：在殖民地外围建立炮台阵地，远程轰击
- **raidLootValueFactor=0**：围攻不生成战利品

### RaidStrategyWorker_PsychicRitualSiege（Anomaly）
- **行为**：Horax邪教徒到达后执行心灵仪式，玩家需要打断
- **特殊**：可多次执行仪式

### RaidStrategyWorker_ShamblerAssault（Anomaly）
- **行为**：蹒跚者（僵尸）从多个方向缓慢逼近
- **特殊**：蹒跚者会在数小时后自行崩溃

| 2 | **EdgeDrop** | `Worker_EdgeDrop` | ✗ | Industrial | (300,0)→(700,0.3) | — | 运输舱空投到地图边缘 |

### 到达模式关键字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `workerClass` | Type | Worker实现类 |
| `selectionWeightCurve` | SimpleCurve | 按威胁点数的选择权重曲线 |
| `pointsFactorCurve` | SimpleCurve | 点数调整因子曲线 |
| `minTechLevel` | TechLevel | 最低科技水平要求（空投需Industrial） |
| `walkIn` | bool | 是否步行进入（影响到达动画） |
| `layerWhitelist/Blacklist` | List | Odyssey星球层级白/黑名单 |
| `minSpaceSelectionWeight` | float | 空间地图最低选择权重 |
| `textEnemy/textFriendly` | string | 信件文本模板 |
| `forQuickMilitaryAid` | bool | 是否可用于快速军事援助 |

**规律总结**：
- **破坏策略**仅支持EdgeWalkIn——需要从边缘步行才能选择破坏路径
- **围攻策略**不支持分组和中心空投——需要集中在一个位置建立阵地
- **特殊策略**（水中、蹒跚者、心灵仪式）各有专用到达模式

## 七、XML配置示例

### 自定义袭击策略

```xml
<RaidStrategyDef>
    <defName>MyCustomStrategy</defName>
    <workerClass>MyMod.RaidStrategyWorker_Custom</workerClass>
    <minPawns>3</minPawns>
    <selectionWeightPerPointsCurve>
        <points>
            <li>(500, 0)</li>
            <li>(1000, 0.5)</li>
        </points>
    </selectionWeightPerPointsCurve>
    <pointsFactorCurve>
        <points>
            <li>0, 0.8</li>
        </points>
    </pointsFactorCurve>
    <arriveModes>
        <li>EdgeWalkIn</li>
        <li>EdgeDrop</li>
    </arriveModes>
    <letterLabelEnemy>Custom Raid</letterLabelEnemy>
    <arrivalTextEnemy>Custom raiders are approaching!</arrivalTextEnemy>
</RaidStrategyDef>
```

### 自定义到达模式

```xml
<PawnsArrivalModeDef>
    <defName>MyCustomArrival</defName>
    <workerClass>MyMod.PawnsArrivalModeWorker_Custom</workerClass>
    <selectionWeightCurve>
        <points>
            <li>(0, 1)</li>
        </points>
    </selectionWeightCurve>
    <textEnemy>A group of {0} from {1} have arrived via custom method.</textEnemy>
    <walkIn>true</walkIn>
</PawnsArrivalModeDef>
```
