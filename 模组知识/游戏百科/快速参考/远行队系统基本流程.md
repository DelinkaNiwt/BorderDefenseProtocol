# 远行队系统基本流程

**总览**：远行队（Caravan）是RimWorld世界地图层的核心移动单位，继承自`WorldObject`。完整生命周期经过4个阶段：**组建**（LordJob状态机驱动殖民者收集物资并离开地图）→ **移动**（Caravan_PathFollower在世界地图上A*寻路移动）→ **旅途**（8个子Tracker协作处理需求/觅食/医疗/负重）→ **到达**（CaravanArrivalAction决定到达后行为）。

## 1. 核心类架构

```
WorldObject（基类）
└── Caravan（远行队实例）
    ├── Caravan_PathFollower    ← 寻路与移动
    ├── Caravan_Tweener         ← 平滑动画
    ├── Caravan_TraderTracker   ← 贸易功能
    ├── Caravan_ForageTracker   ← 觅食系统
    ├── Caravan_NeedsTracker    ← 需求追踪
    ├── Caravan_CarryTracker    ← 搬运追踪
    ├── Caravan_BedsTracker     ← 床铺追踪
    └── Caravan_BabyTracker     ← 婴儿追踪（Biotech）
```

**Caravan核心字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `pawns` | ThingOwner&lt;Pawn&gt; | 成员列表（Pawn容器） |
| `pather` | Caravan_PathFollower | 寻路器 |
| `trader` | Caravan_TraderTracker | 贸易追踪器 |
| `forage` | Caravan_ForageTracker | 觅食追踪器 |
| `needs` | Caravan_NeedsTracker | 需求追踪器 |
| `autoJoinable` | bool | 是否允许自动加入 |
| `storyState` | StoryState | 叙事状态（事件目标） |

**Caravan关键属性**：

| 属性 | 说明 |
|------|------|
| `IsPlayerControlled` | 是否玩家控制 |
| `ImmobilizedByMass` | 是否因超重无法移动 |
| `MassCapacity` / `MassUsage` | 负重容量/已用负重 |
| `NightResting` | 是否夜间休息中 |
| `Visibility` | 可见度（影响伏击概率） |
| `TicksPerMove` | 每格移动tick数 |
| `DaysWorthOfFood` | 剩余食物天数 |

## 2. 组建流程（LordJob_FormAndSendCaravan）

远行队组建由`LordJob_FormAndSendCaravan`状态机驱动，6个阶段+暂停机制：

```
┌──────────────────────────────────────────────────────────────────┐
│              远行队组建状态机 6阶段流程                              │
├──────────────────────────────────────────────────────────────────┤
│  阶段1: GatherAnimals（集合动物）                                  │
│    LordToil_PrepareCaravan_GatherAnimals → meetingPoint          │
│    触发: "AllAnimalsGathered" memo                                │
│                                                                  │
│  阶段2: GatherItems（收集物品）                                    │
│    LordToil_PrepareCaravan_GatherItems → meetingPoint            │
│    触发: "AllItemsGathered" memo → EndAllJobs                    │
│                                                                  │
│  阶段3: GatherDownedPawns（搬运倒地者）                            │
│    LordToil_PrepareCaravan_GatherDownedPawns → exitSpot          │
│    触发: "AllDownedPawnsGathered" memo → EndAllJobs              │
│                                                                  │
│  阶段4: Wait（等待疲劳者醒来）                                     │
│    LordToil_PrepareCaravan_Wait → meetingPoint                   │
│    触发: NoPawnsVeryTiredAndSleeping → WakeAll                   │
│                                                                  │
│  阶段5: CollectAnimals（收拢动物到出口）                           │
│    LordToil_PrepareCaravan_CollectAnimals → exitSpot             │
│    触发: "AllAnimalsCollected" memo                               │
│                                                                  │
│  阶段6: Leave（离开地图）                                          │
│    LordToil_PrepareCaravan_Leave → exitSpot                      │
│    触发: "ReadyToExitMap" memo → SendCaravan()                   │
│                                                                  │
│  任意阶段: "ForceDepartNow" → 直接跳到Leave                       │
│  每个阶段都有对应的Pause状态（暂停/恢复）                           │
└──────────────────────────────────────────────────────────────────┘
```

**SendCaravan()核心**：调用`CaravanFormingUtility.FormAndCreateCaravan()` → `CaravanExitMapUtility.ExitMapAndCreateCaravan()`，将地图上的Pawn转移到世界地图Caravan对象中。

**组建入口**：玩家通过World面板选择殖民者和物资 → `Dialog_FormCaravan` UI → 创建LordJob → 分配Duty给每个参与者。

## 3. 世界地图移动（Caravan_PathFollower）

### 寻路机制

`Caravan_PathFollower`使用世界地图A*寻路（`WorldPathFinder`），逐格移动：

| 字段 | 说明 |
|------|------|
| `destTile` | 目标地块 |
| `nextTile` | 下一个地块 |
| `nextTileCostLeft` | 到达下一格剩余成本 |
| `nextTileCostTotal` | 到达下一格总成本 |
| `arrivalAction` | 到达后执行的行为 |
| `paused` | 是否暂停 |

### 移动成本公式

```
moveCost = caravanTicksPerMove × movementDifficulty × roadMultiplier
moveCost = clamp(moveCost, 1, 30000)
```

- `caravanTicksPerMove`：远行队基础移动速度（由成员移动速度、负重等决定）
- `movementDifficulty`：目标地块地形难度（`WorldPathGrid.CalculatedMovementDifficultyAt`）
- `roadMultiplier`：道路加速倍率（`WorldGrid.GetRoadMovementDifficultyMultiplier`）

**速度显示**：`60000 / moveCost` = 每天移动格数（TilesPerDay）

### 移动流程

`PatherTickInterval(delta)` 每tick执行：
1. 检查是否暂停/夜间休息/无法移动
2. `CostToPayThisTick()` 计算本tick应扣减的成本
3. `nextTileCostLeft -= costToPay`
4. 当`nextTileCostLeft ≤ 0`时，`TryEnterNextPathTile()` 进入下一格
5. 到达目标格时，`PatherArrived()` → 执行`arrivalAction`

### 夜间休息

`NightResting`属性：当世界地图当前位置处于夜间（22:00~06:00）时返回true，远行队停止移动。`LeftRestTicks`/`LeftNonRestTicks`显示剩余休息/行进时间。

## 4. 旅途系统（Caravan.TickInterval）

`TickInterval(delta)`每tick驱动8个子系统：

| # | 子系统 | 方法 | 功能 |
|---|--------|------|------|
| 1 | **寻路** | `pather.PatherTickInterval` | 移动到下一格 |
| 2 | **动画** | `tweener.TweenerTickInterval` | 平滑移动动画 |
| 3 | **觅食** | `forage.ForageTrackerTickInterval` | 按生物群系觅食（每15 ticks更新） |
| 4 | **搬运** | `carryTracker.CarryTrackerTickInterval` | 搬运倒地者 |
| 5 | **床铺** | `beds.BedsTrackerTickInterval` | 床铺分配 |
| 6 | **需求** | `needs.NeedsTrackerTickInterval` | 食物/休息/心情等Need更新 |
| 7 | **婴儿** | `babies.TickInterval` | 婴儿照料（Biotech） |
| 8 | **辅助** | 药物/医疗/污染 | `CheckTakeScheduledDrugs`/`CheckTend`/`CheckDamageFromPollution` |

### 觅食系统（Caravan_ForageTracker）

- 每15 ticks更新觅食进度
- 觅食速率由`ForagedFoodPerDayCalculator`计算（基于成员种植技能+生物群系`foragedFood`）
- 进度达到1.0时生成食物（受负重容量限制）
- 不同生物群系提供不同觅食食物（如温带森林=浆果）

### 需求追踪（Caravan_NeedsTracker）

远行队中Pawn的Need持续更新：
- **食物**：按正常速率消耗，从远行队库存中自动进食
- **休息**：夜间休息时恢复
- **心情**：旅途中有专用Thought（如"旅途中"、"睡在地上"）
- **娱乐**：停止移动时每1250 ticks获得0.1 Joy

## 5. 远行队事件

远行队作为`IncidentTarget`（标签`Caravan`），可触发专属事件。事件按`mtbDaysByBiome`（生物群系MTB天数）概率触发：

| # | 事件 | IncidentDef | 类别 | 效果 |
|---|------|------------|------|------|
| 1 | **敌对伏击** | `Ambush` | ThreatBig | 生成临时地图，敌对派系攻击 |
| 2 | **猎人伏击** | `ManhunterAmbush` | ThreatBig | 生成临时地图，疯狂动物攻击 |
| 3 | **勒索** | `CaravanDemand` | ThreatSmall | 敌对派系要求交出物品或战斗 |
| 4 | **友好遭遇** | `CaravanMeeting` | Misc | 遇到友好商队，可贸易 |

**MTB天数对比**（温带森林/冰原/海冰）：

| 事件 | 温带森林 | 冰原 | 海冰 |
|------|---------|------|------|
| 伏击 | 20天 | 40天 | 60天 |
| 猎人伏击 | 20天 | 40天 | 60天 |
| 勒索 | 20天 | 40天 | 60天 |
| 友好遭遇 | 10天 | 35天 | 80天 |

**伏击地图生成**：`CaravanIncidentUtility.GetOrGenerateMapForIncident()` 在远行队位置生成临时地图（`CaravansBattlefield` WorldObject），战斗结束后可重新组建远行队离开。

## 6. 到达行为（CaravanArrivalAction）

远行队到达目标后，由`CaravanArrivalAction`子类决定行为：

| # | ArrivalAction | 目标 | 行为 |
|---|--------------|------|------|
| 1 | `CaravanArrivalAction_Enter` | 玩家据点 | 进入已有地图 |
| 2 | `CaravanArrivalAction_AttackSettlement` | 敌对据点 | 生成地图+战斗 |
| 3 | `CaravanArrivalAction_VisitSettlement` | 友好据点 | 进入据点地图 |
| 4 | `CaravanArrivalAction_Trade` | 友好据点 | 打开贸易界面 |
| 5 | `CaravanArrivalAction_OfferGifts` | 友好据点 | 赠送礼物提升好感 |
| 6 | `CaravanArrivalAction_VisitSite` | 任务地点 | 进入Site地图 |
| 7 | `CaravanArrivalAction_VisitPeaceTalks` | 和谈地点 | 触发和谈事件 |
| 8 | `CaravanArrivalAction_VisitEscapeShip` | 逃生飞船 | 进入飞船地图 |

**到达流程**：`PatherArrived()` → `arrivalAction.Arrived(caravan)` → 根据子类执行具体行为（生成地图/打开UI/触发事件）。

## 7. 传送与穿梭机系统

RimWorld有5种跨地图/世界地图传送机制，按范围从小到大：

| # | 机制 | DLC | 范围 | 核心类 | 说明 |
|---|------|-----|------|--------|------|
| 1 | **PawnFlyer** | Core | 地图内 | `PawnFlyer` + `PawnFlyerWorker` | 地图内瞬移（跳跃包、Skip） |
| 2 | **Skip传送** | Royalty | 地图内 | `CompAbilityEffect_Teleport` | 灵能传送，目标位置瞬移+眩晕 |
| 3 | **运输舱** | Core | 世界地图 | `CompLaunchable` → `TravellingTransporters` | 单向发射，需燃料，可载人/物 |
| 4 | **帝国穿梭机** | Royalty | 世界地图 | `CompShuttle` + `Building_PassengerShuttle` | 皇室许可召唤，可往返 |
| 5 | **重力船** | Odyssey | 世界+轨道 | `Gravship` + `GravshipLaunch` | 可重复使用飞船，支持轨道层 |

### PawnFlyer（地图内飞行）

`PawnFlyer`是地图内Pawn瞬移的标准机制——Pawn被临时替换为PawnFlyer Thing，沿抛物线/直线飞行到目标位置，到达后恢复Pawn。

- `PawnFlyerProperties`定义飞行参数（速度、特效、着陆音效）
- `PawnFlyerWorker`控制飞行行为（默认抛物线）
- 模组可自定义PawnFlyer子类实现特殊飞行（如Milira的5种冲锋攻击）

### Skip传送（Royalty灵能）

`CompAbilityEffect_Teleport`实现灵能传送：
- 直接修改`target.Thing.Position = destination.Cell`
- 传送后对目标施加眩晕（`Props.stunTicks`）
- 发送`SkipUsedSignal`信号（任务系统可监听）
- 3种目标模式：Selected（玩家选择）、Caster（施法者位置）、RandomInRange（随机范围）

### 运输舱（Core）

`CompLaunchable`驱动运输舱发射：
1. 装载Pawn/物品到`CompTransporter`
2. 选择世界地图目标 → `TryLaunch()`
3. 生成`TravellingTransporters` WorldObject在世界地图上飞行
4. 到达后执行`TransportersArrivalAction`（着陆/赠送/攻击）

### 穿梭机（Royalty/Odyssey）

帝国穿梭机通过`CompShuttle`管理：
- 皇室许可`CallTransportShuttle`召唤
- `Building_PassengerShuttle`（Odyssey）支持玩家自有穿梭机
- 穿梭机可往返（不同于运输舱的单向发射）
- Odyssey的`PassengerShuttle` WorldObjectDef在世界地图上飞行

### 重力船（Odyssey）

`Gravship`是Odyssey DLC的核心移动方式：
- 继承WorldObject，在世界地图上移动（类似Caravan但可进入轨道层）
- `GravshipLaunch` WorldObject标记发射位置
- 支持`PlanetLayerDef`多层行星（地表层+轨道层）
- `requiresSignalJammerToReach`控制某些目标需要信号干扰器才能接近

## 8. 关键类速查表

| 需求 | 推荐方案 | 说明 |
|------|---------|------|
| 新增远行队事件 | IncidentDef(targetTags=Caravan) + IncidentWorker子类 | 配置mtbDaysByBiome控制触发频率 |
| 自定义到达行为 | CaravanArrivalAction子类 + 重写Arrived() | 需在FloatMenuOption中注册 |
| 地图内传送 | CompAbilityEffect_Teleport或PawnFlyer | Skip用于灵能，PawnFlyer用于物理跳跃 |
| 世界地图传送 | CompLaunchable + TravellingTransporters | 运输舱模式，需自定义TransportersArrivalAction |
| 修改远行队移动速度 | Stat修改TicksPerMove相关因子 | 通过装备/基因/Hediff的statOffsets |
| 远行队Gizmo | Caravan.GetGizmos()中添加 | 通过WorldObjectComp.GetGizmos() |
| 扎营系统 | Camp WorldObjectDef | 生成临时200×200地图 |
