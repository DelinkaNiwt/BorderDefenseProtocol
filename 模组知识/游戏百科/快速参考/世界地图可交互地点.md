# 世界地图可交互地点

**总览**：RimWorld世界地图上的所有可交互地点都是`WorldObject`的子类实例，由`WorldObjectsHolder`（`Find.WorldObjects`）统一管理。地点分两大类：**静态地点**（据点、任务地点、遗迹等，继承`MapParent`，可生成地图）和**动态地点**（远行队、运输舱、重力船等，直接继承`WorldObject`，在世界地图上移动）。

## 1. WorldObject类继承体系

```
WorldObject（基类，IExposable+ISelectable+IThingHolder）
├── 动态WorldObject（在世界地图上移动）
│   ├── Caravan                    ← 远行队
│   ├── TravellingTransporters     ← 飞行中的运输舱/穿梭机
│   ├── Gravship                   ← 飞行中的重力船（Odyssey）
│   ├── GravshipLaunch             ← 重力船发射标记（Odyssey）
│   ├── RoutePlannerWaypoint       ← 路线规划路标（临时）
│   └── AbandonedSettlement        ← 废弃据点（无地图）
│
└── MapParent（地图容器基类，canHaveMap=true）
    ├── Settlement                 ← 据点（玩家/NPC）
    ├── Site                       ← 任务地点（通用，SitePartDef组合）
    ├── Camp                       ← 临时营地
    ├── EscapeShip                 ← 逃生飞船
    ├── CaravansBattlefield        ← 远行队战场（伏击）
    ├── DestroyedSettlement        ← 被摧毁的据点
    ├── PocketMapParent            ← 口袋地图（隐藏）
    ├── SpaceMapParent             ← 轨道地图（Odyssey）
    │   ├── ResourceAsteroidMapParent  ← 资源小行星
    │   └── BasicAsteroidMapParent     ← 基础小行星
    └── AbandonedArchotechStructures   ← 废弃超凡建筑（Odyssey）
```

## 3. WorldObjectComp组件系统

WorldObject通过`WorldObjectComp`组件扩展功能，类似Thing的ThingComp：

| Comp | 功能 | 典型使用者 |
|------|------|-----------|
| `WorldObjectComp_FormCaravan` | 允许在此地点组建远行队 | Settlement, Site, Camp, Ambush |
| `WorldObjectComp_TimedDetectionRaids` | 定时检测袭击（停留过久触发） | Settlement, Site, Camp |
| `WorldObjectComp_Abandon` | 允许放弃据点 | Settlement, EscapeShip |
| `WorldObjectComp_EnterCooldown` | 进入冷却（防止反复进出） | Settlement, Site |
| `WorldObjectComp_Timeout` | 超时自动消失 | PeaceTalks, Site, AbandonedCamp |
| `WorldObjectComp_DefeatAllEnemiesQuest` | 击败所有敌人完成任务 | Site |
| `WorldObjectComp_ItemStashContents` | 物资藏匿内容 | Site |
| `WorldObjectComp_TimedMakeFactionHostile` | 定时使派系敌对 | Site |
| `WorldObjectComp_EscapeShip` | 逃生飞船逻辑 | EscapeShip |
| `WorldObjectComp_DownedRefugee` | 倒地难民 | Site |
| `WorldObjectComp_PrisonerWillingToJoin` | 愿意加入的囚犯 | Site |
| `WorldObjectComp_TradeRequest` | 贸易请求 | Settlement |

## 4. Site任务地点系统

`Site`是最通用的世界地图地点——通过`SitePartDef`组合实现任意任务场景：

```
Site（WorldObject）
├── def: WorldObjectDef（Site/SpaceSite/ClaimableSite等）
├── parts: List<SitePart>
│   └── SitePart
│       ├── def: SitePartDef（定义地图内容）
│       └── parms: SitePartParams（威胁点数等参数）
├── siteTexture（动态图标，由SitePartDef决定）
└── comps: 9个WorldObjectComp
```

**SitePartDef**定义地图中的内容（敌人营地、物资藏匿、倒地难民等），多个SitePartDef可组合在同一Site上。

**Site生成流程**：`SiteMaker.MakeSite(siteParts, tile, faction)` → 创建Site实例 → 添加SitePart → 设置派系 → 添加到WorldObjects。

## 5. 据点交互（Settlement）

`Settlement`是最重要的静态地点，支持5种交互：

| # | 交互 | CaravanArrivalAction | 条件 |
|---|------|---------------------|------|
| 1 | **进入** | `CaravanArrivalAction_Enter` | 玩家据点 |
| 2 | **攻击** | `CaravanArrivalAction_AttackSettlement` | 敌对据点 |
| 3 | **访问** | `CaravanArrivalAction_VisitSettlement` | 友好/中立据点 |
| 4 | **贸易** | `CaravanArrivalAction_Trade` | 友好/中立据点 |
| 5 | **赠礼** | `CaravanArrivalAction_OfferGifts` | 友好/中立据点 |

**据点地图生成**：首次进入时由`MapGenerator`根据`MapGeneratorDef`生成地图。玩家据点使用`Base_Player`，NPC据点使用`Base_Faction`。

## 6. Odyssey轨道层（PlanetLayerDef）

Odyssey DLC引入多层行星体系：

| 层级 | PlanetLayerDef | 说明 |
|------|---------------|------|
| **地表层** | `Surface`（默认） | 传统RimWorld地图 |
| **轨道层** | `Orbit` | 太空地点（小行星、轨道据点） |

- `SpaceMapParent`是轨道层地图的基类
- `fullyExpandedInSpace=true`的WorldObject在轨道层完全展开显示
- `requiresSignalJammerToReach=true`的目标需要重力船装备信号干扰器
- 重力船（Gravship）可在地表层和轨道层之间移动

## 7. 关键类速查表

| 需求 | 推荐方案 | 说明 |
|------|---------|------|
| 新增世界地图地点 | WorldObjectDef(XML) + WorldObjectComp | 大多数不需要自定义C#类 |
| 自定义地点交互 | CaravanArrivalAction子类 | 重写Arrived()和FloatMenuOption |
| 地点定时消失 | WorldObjectComp_Timeout | XML配置timeoutDays |
| 地点触发袭击 | WorldObjectComp_TimedDetectionRaids | 停留过久触发检测袭击 |
| 任务地点 | Site + SitePartDef组合 | 最灵活的地点类型 |
| 自定义地图生成 | MapGeneratorDef + GenStepDef | 配置在WorldObjectDef.mapGenerator |
| 轨道层地点 | 继承SpaceMapParent + SpaceSite模板 | 需要Odyssey DLC |
| 世界地图Comp | 自定义WorldObjectComp子类 | 类似ThingComp，重写CompTick/GetGizmos等 |
