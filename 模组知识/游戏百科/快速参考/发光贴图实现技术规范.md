# RimWorld发光贴图实现技术规范

## 1. 技术概述

### 1.1 核心机制
RimWorld的发光贴图效果基于**双层贴图渲染系统**，通过特殊着色器将基础贴图和发光层贴图进行混合渲染，实现自发光视觉效果。

### 1.2 技术栈
- **渲染引擎**: Unity引擎
- **着色器**: CutoutComplex Shader
- **混合模式**: Additive/Screen Blending
- **文件格式**: PNG (支持Alpha通道)
- **配置方式**: XML定义 + 贴图文件

### 1.3 适用范围
- 装备 (Apparel)
- 武器 (Weapon)
- 建筑物 (Building)
- 物品 (Thing)
- 载具/飞行器 (Aerocraft)

## 2. 文件结构规范

### 2.1 贴图文件命名规则

#### 规则A：插入式命名（推荐）
```
基础贴图: {BaseName}_{BodyType}_{Direction}.png
发光贴图: {BaseName}Light_{BodyType}_{Direction}.png
```

**示例**:
```
CMC_uniform_Male_east.png       → 基础贴图
CMC_uniLight_Male_east.png      → 发光贴图
```

#### 规则B：后缀式命名
```
基础贴图: {BaseName}_{Direction}.png
发光贴图: {BaseName}_{Direction}_Light.png
```

**示例**:
```
JX_Skyfall.png                  → 基础贴图
JX_Skyfall_Light.png            → 发光贴图
```

## 3. XML配置规范

### 3.1 装备类配置

#### 完整配置示例
```xml
<?xml version="1.0" encoding="utf-8"?>
<Defs>
    <ThingDef ParentName="ApparelMakeableBase">
        <defName>MyGlowingArmor</defName>
        <label>发光装甲</label>
        <description>一套带有发光线条的高科技装甲。</description>

        <!-- 显示贴图配置 (物品栏/地面) -->
        <graphicData>
            <texPath>Things/Apparel/Display/MyArmor_Display</texPath>
            <graphicClass>Graphic_Single</graphicClass>
            <shaderType>CutoutComplex</shaderType>  <!-- 关键1: 启用复杂着色器 -->
        </graphicData>

        <!-- 穿戴贴图配置 -->
        <apparel>
            <bodyPartGroups>
                <li>Torso</li>
                <li>Shoulders</li>
                <li>Arms</li>
            </bodyPartGroups>
            <layers>
                <li>Middle</li>
            </layers>

            <!-- 关键2: 指定穿戴贴图路径 -->
            <wornGraphicPath>Things/Apparel/MyArmor/MyArmor</wornGraphicPath>

            <!-- 关键3: 启用遮罩/发光层 -->
            <useWornGraphicMask>true</useWornGraphicMask>
        </apparel>

        <!-- 其他配置... -->
        <statBases>
            <MaxHitPoints>200</MaxHitPoints>
            <WorkToMake>15000</WorkToMake>
            <Mass>5</Mass>
        </statBases>
    </ThingDef>
</Defs>
```

#### 关键配置项解析

**1. shaderType配置**
```xml
<graphicData>
    <shaderType>CutoutComplex</shaderType>
</graphicData>
```
- **作用**: 指定使用支持多层渲染的着色器
- **可选值**:
  - `Cutout` - 标准着色器（不支持发光）
  - `CutoutComplex` - 复杂着色器（支持发光层）
  - `Transparent` - 透明着色器
  - `TransparentPostLight` - 后光照透明着色器
- **必需性**: 必须设置为 `CutoutComplex` 才能启用发光效果

**2. useWornGraphicMask配置**
```xml
<apparel>
    <useWornGraphicMask>true</useWornGraphicMask>
</apparel>
```
- **作用**: 告诉游戏引擎加载对应的发光层贴图
- **默认值**: `false`
- **必需性**: 必须设置为 `true`

**3. wornGraphicPath配置**
```xml
<apparel>
    <wornGraphicPath>Things/Apparel/MyArmor/MyArmor</wornGraphicPath>
</apparel>
```
- **作用**: 指定基础贴图的路径（不含文件扩展名）
- **自动查找规则**:
  - 基础贴图: `{wornGraphicPath}_{BodyType}_{Direction}.png`
  - 发光贴图: `{wornGraphicPath}Light_{BodyType}_{Direction}.png`

## 4. 贴图制作规范

### 4.1 基础贴图要求

**尺寸规范**:
- 装备贴图: 128x128 或 256x256 像素
- 武器贴图: 128x128 或 256x256 像素
- 建筑贴图: 根据建筑大小，通常 128x128 到 512x512

**格式要求**:
- 文件格式: PNG
- 色彩模式: RGBA (32位)
- Alpha通道: 必需（用于透明度）
- 压缩: 无损压缩

**内容要求**:
- 包含完整的物体外观
- 使用适当的阴影和高光
- 边缘需要抗锯齿处理
- 透明区域使用Alpha=0

### 4.2 发光层贴图要求

**核心原则**: 发光层只包含需要发光的部分，其他区域完全透明

**制作步骤**:
1. 复制基础贴图作为起点
2. 删除所有不需要发光的部分（设为完全透明）
3. 保留需要发光的线条、图案、标志等
4. 调整发光部分的颜色和亮度

**颜色选择**:
- **高亮白色** (255,255,255): 最强发光效果
- **青色** (0,255,255): 科技感发光
- **蓝色** (100,150,255): 冷色调发光
- **橙色** (255,150,50): 暖色调发光
- **绿色** (50,255,100): 能量感发光

**Alpha通道使用**:
- Alpha = 255 (100%): 完全不透明，最强发光
- Alpha = 200 (78%): 强发光
- Alpha = 150 (59%): 中等发光
- Alpha = 100 (39%): 微弱发光
- Alpha = 0 (0%): 完全透明，不发光

**示例对比**:
```
基础贴图 (MyArmor_Male_east.png):
- 包含完整的装甲外观
- 所有细节都可见
- 正常的颜色和阴影

发光层贴图 (MyArmorLight_Male_east.png):
- 只有发光线条可见（如胸口的能量核心、肩部的指示灯）
- 其他区域完全透明
- 使用明亮的颜色（如青色、白色）
- 可以比基础贴图更简化
```

### 4.3 Photoshop/GIMP制作流程

#### Photoshop流程
```
1. 打开基础贴图
2. 复制图层
3. 添加图层蒙版
4. 使用画笔工具，在蒙版上：
   - 黑色画笔：隐藏不需要发光的部分
   - 白色画笔：保留需要发光的部分
5. 调整发光部分的颜色：
   - 图像 > 调整 > 色相/饱和度
   - 图像 > 调整 > 亮度/对比度
6. 导出为PNG，确保保留Alpha通道
```

#### GIMP流程
```
1. 打开基础贴图
2. 图层 > 透明 > 添加Alpha通道
3. 使用橡皮擦工具删除不需要发光的部分
4. 颜色 > 色相-饱和度，调整发光颜色
5. 颜色 > 曲线，增加亮度
6. 文件 > 导出为 > PNG
```

## 5. 渲染原理

### 5.1 着色器工作流程

```
游戏渲染管线:

1. 加载ThingDef定义
   ↓
2. 检测 shaderType="CutoutComplex"
   ↓
3. 检测 useWornGraphicMask="true"
   ↓
4. 加载基础贴图 (Base Texture)
   ↓
5. 查找并加载发光层贴图 (Emissive Texture)
   ↓
6. 着色器处理:
   - Pass 1: 渲染基础贴图（正常光照）
   - Pass 2: 渲染发光层（加法混合，忽略光照）
   ↓
7. 混合输出到屏幕
```

### 5.2 混合模式详解

**加法混合 (Additive Blending)**:
```
最终颜色 = 基础颜色 + 发光层颜色
```
- 发光层的颜色会"叠加"到基础颜色上
- 白色发光层会让基础颜色变得更亮
- 即使在暗处也能看到发光效果（自发光）

**屏幕混合 (Screen Blending)**:
```
最终颜色 = 1 - (1 - 基础颜色) × (1 - 发光层颜色)
```
- 类似加法混合，但更柔和
- 不会过度曝光
- 适合大面积发光效果

### 5.3 光照独立性

**关键特性**: 发光层不受游戏光照系统影响

```
场景光照: 黑暗 (0% 亮度)
├── 基础贴图: 几乎不可见
└── 发光层: 完全可见 (100% 亮度)

场景光照: 明亮 (100% 亮度)
├── 基础贴图: 完全可见
└── 发光层: 完全可见 (叠加效果)
```

这就是为什么发光效果在黑暗中特别明显。

## 8. 性能考虑

### 8.1 性能影响

**渲染开销**:
- 基础贴图: 1x 渲染调用
- 发光贴图: +1x 渲染调用
- 总计: 2x 渲染调用

**优化建议**:
1. 发光层贴图尽可能简化
2. 避免大面积发光（增加透明区域）
3. 合理使用Alpha值（避免过度混合）
4. 对于大量相同装备，共享材质实例

### 8.2 内存占用

```
单个装备的内存占用:
- 基础贴图 (256x256 RGBA): ~256KB
- 发光层贴图 (256x256 RGBA): ~256KB
- 总计: ~512KB

如果有6个体型 × 3个方向 = 18个变体:
- 总内存: ~9MB per 装备
```

**优化策略**:
- 使用较小的贴图尺寸（128x128）
- 共享相同的发光层贴图（如果可能）
- 使用纹理压缩（Unity自动处理）

## 9. 常见问题

### Q1: 发光层不显示
**原因**:
- `shaderType` 未设置为 `CutoutComplex`
- `useWornGraphicMask` 未设置为 `true`
- 文件命名不正确

**解决方案**: 检查XML配置和文件命名

### Q2: 发光效果在某些方向不显示
**原因**: 缺少对应方向的发光层贴图

**解决方案**: 为所有方向准备发光层贴图

### Q3: 发光层位置偏移
**原因**: 基础贴图和发光层贴图尺寸不一致

**解决方案**: 确保两者尺寸完全相同

### Q4: 发光效果受光照影响
**原因**: 可能使用了错误的着色器

**解决方案**: 确认使用 `CutoutComplex` 着色器

### Q5: 发光层显示为纯色块
**原因**: 发光层贴图的Alpha通道不正确

**解决方案**: 重新导出PNG，确保保留Alpha通道
