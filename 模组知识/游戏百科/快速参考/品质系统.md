# 品质系统

**总览**：RimWorld品质系统通过`CompQuality`组件为物品赋予7级品质（Awful→Legendary），品质通过`StatPart_Quality`（倍率）和`StatPart_Quality_Offset`（偏移）两种StatPart影响物品的各项属性。每个Stat可以独立配置品质倍率曲线，因此不同属性受品质影响的幅度不同。

## 7级品质定义（QualityCategory）

| # | 枚举值 | 英文名 | 中文名 | 说明 |
|---|--------|--------|--------|------|
| 0 | `Awful` | Awful | 粗劣 | 最低品质 |
| 1 | `Poor` | Poor | 低劣 | 低于标准 |
| 2 | `Normal` | Normal | 普通 | 基准品质（默认值） |
| 3 | `Good` | Good | 良好 | 高于标准 |
| 4 | `Excellent` | Excellent | 优秀 | 高品质 |
| 5 | `Masterwork` | Masterwork | 杰作 | 极高品质 |
| 6 | `Legendary` | Legendary | 传奇 | 最高品质 |

## CompQuality组件

```csharp
public class CompQuality : ThingComp {
    private QualityCategory qualityInt = QualityCategory.Normal;  // 默认普通
    public QualityCategory Quality => qualityInt;

    // 设置品质（同时触发艺术品初始化）
    public void SetQuality(QualityCategory q, ArtGenerationContext? source);

    // 品质相同才能堆叠
    public override bool AllowStackWith(Thing other);
}
```

**关键特性**：
- 品质存储在`CompQuality`组件中，需要`CompProperties_Quality`才会附加
- 默认品质为`Normal`（普通）
- **堆叠限制**：品质不同的物品不能堆叠（`AllowStackWith`检查品质一致性）
- 品质设置时自动触发`CompArt`艺术品初始化和`CompFacilityQualityBased`设施更新

## 品质影响属性的两种机制

### 机制1：StatPart_Quality（倍率型）

最常用的品质影响方式。对Stat值施加**乘法倍率**，每个Stat可独立配置7级倍率。

```csharp
public class StatPart_Quality : StatPart {
    // 7级品质各自的倍率（默认均为1.0）
    private float factorAwful = 1f;
    private float factorPoor = 1f;
    private float factorNormal = 1f;
    private float factorGood = 1f;
    private float factorExcellent = 1f;
    private float factorMasterwork = 1f;
    private float factorLegendary = 1f;

    // 7级品质各自的最大增益上限（默认无限）
    private float maxGainAwful = 9999999f;
    // ... 其余6级同理

    // 是否对负值也应用倍率
    private bool applyToNegativeValues;

    public override void TransformValue(StatRequest req, ref float val) {
        if (val <= 0f && !applyToNegativeValues) return;
        float gain = val * QualityMultiplier(quality) - val;
        gain = Mathf.Min(gain, MaxGain(quality));  // 限制最大增益
        val += gain;
    }
}
```

**计算公式**：
```
增益 = 原始值 × 品质倍率 - 原始值
增益 = min(增益, maxGain)
最终值 = 原始值 + 增益
```

**典型倍率配置示例**（以护甲为例）：
| 品质 | 倍率 | 效果 |
|------|------|------|
| Awful | 0.5 | 护甲减半 |
| Poor | 0.75 | 护甲×0.75 |
| Normal | 1.0 | 基准值 |
| Good | 1.15 | 护甲×1.15 |
| Excellent | 1.3 | 护甲×1.3 |
| Masterwork | 1.55 | 护甲×1.55 |
| Legendary | 1.85 | 护甲×1.85 |

### 机制2：StatPart_Quality_Offset（偏移型）

对Stat值施加**加法偏移**，可限定只对特定ThingDef生效。

```csharp
public class StatPart_Quality_Offset : StatPart {
    // 7级品质各自的偏移量
    public float offsetAwful;
    public float offsetPoor;
    public float offsetNormal;
    public float offsetGood;
    public float offsetExcellent;
    public float offsetMasterwork;
    public float offsetLegendary;

    // 可选：限定只对特定ThingDef生效
    public List<ThingDef> thingDefs;

    public override void TransformValue(StatRequest req, ref float val) {
        if (ApplyTo(req))
            val += QualityOffset(quality);
    }
}
```

**计算公式**：`最终值 = 原始值 + 品质偏移量`

### 机制3：StatModifierQuality（品质修饰符）

用于`ThingDef`的`statBases`中，为每个品质级别指定不同的Stat值。

```csharp
public class StatModifierQuality {
    public StatDef stat;
    private float awful, poor, normal, good, excellent, masterwork, legendary;
    public float GetValue(QualityCategory qc);  // 按品质返回对应值
}
```

## 品质影响的主要属性

| 属性类别 | 典型Stat | 品质影响方式 | 说明 |
|---------|---------|-------------|------|
| **耐久度** | MaxHitPoints | StatPart_Quality（倍率） | 品质越高越耐用 |
| **护甲** | ArmorRating_Sharp/Blunt/Heat | StatPart_Quality（倍率） | 品质越高防护越强 |
| **保暖** | Insulation_Cold/Heat | StatPart_Quality（倍率） | 品质越高保暖/隔热越好 |
| **美观** | Beauty | StatPart_Quality（倍率/偏移） | 品质越高越美观 |
| **舒适度** | Comfort | StatPart_Quality（倍率） | 品质越高越舒适 |
| **武器伤害** | 各种DPS相关 | StatPart_Quality（倍率） | 品质越高伤害越高 |
| **市场价值** | MarketValue | StatPart_Quality（倍率） | 品质越高越值钱 |
| **工作速度** | 各种WorkSpeed | StatPart_Quality（倍率） | 工作台品质影响工作速度 |

## 品质与堆叠

- 有品质的物品**不能与不同品质的同类物品堆叠**
- `CompQuality.AllowStackWith()`检查品质一致性
- 分割物品时品质继承（`PostSplitOff`复制品质）
