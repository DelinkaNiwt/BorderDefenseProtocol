# 护盾机制与伤害吸收系统

**总览**：RimWorld的护盾系统有5种实现，全部基于ThingComp架构，**没有任何基于Hediff的护盾机制**。两大核心Comp——`CompShield`（个人能量护盾，拦截阶段4的PostPreApplyDamage）和`CompProjectileInterceptor`（区域投射物拦截器，拦截投射物飞行阶段的CheckIntercept）——覆盖了所有护盾行为。其余3种（BroadshieldPack、MechShield、Skipshield）都是在这两个核心Comp基础上的变体封装。

## 1. 5种护盾实现全面对比表

| 维度 | CompShield | CompProjectileInterceptor | BroadshieldPack | MechShield | Skipshield |
|------|-----------|--------------------------|-----------------|------------|------------|
| **核心类** | `CompShield` | `CompProjectileInterceptor` | `BroadshieldPack` | `MechShield` | AbilityDef `BulletShield` |

| **拦截范围** | 仅穿戴者/宿主 | radius范围内所有友方 | 部署后radius范围 | 跟随目标机械体 | 施放位置radius范围 |

## 2. ★ CompShield 个人能量护盾（核心重点）

### 2.1 类架构

`CompShield : ThingComp`（RimWorld命名空间，333行）

**核心字段**：

```csharp
public class CompShield : ThingComp
{
    protected float energy;                    // 当前能量（0~EnergyMax）
    protected int ticksToReset = -1;           // 击破后重置倒计时（-1=未击破）
    protected int lastKeepDisplayTick = -9999; // 上次保持显示的tick
    private Vector3 impactAngleVect;           // 受击角度向量（视觉抖动用）
    private int lastAbsorbDamageTick = -9999;  // 上次吸收伤害的tick
}
```

**关键属性**：

| 属性 | 类型 | 来源 | 说明 |
|------|------|------|------|
| `EnergyMax` | float | `StatDefOf.EnergyShieldEnergyMax` | 最大能量（护盾腰带默认1.1） |
| `EnergyGainPerTick` | float | `StatDefOf.EnergyShieldRechargeRate / 60f` | 每tick恢复量（护盾腰带0.13/秒） |
| `Energy` | float | `energy`字段 | 当前能量 |
| `ShieldState` | ShieldState | 计算属性 | Active/Resetting/Disabled |
| `PawnOwner` | Pawn | parent转换 | 穿戴者（Apparel→Wearer）或宿主（Pawn直接） |
| `IsApparel` | bool | `parent is Apparel` | 是否为服装组件（vs内置组件） |

### 2.2 ShieldState 三状态机

```csharp
public enum ShieldState : byte
{
    Active,     // 激活中：可吸收伤害，每tick恢复能量
    Resetting,  // 重置中：击破后倒计时，不可吸收伤害
    Disabled    // 禁用：Pawn充电中/自动关机/休眠中
}
```

**状态转换流程**：

```
Active ──(能量耗尽/EMP)──→ Break() ──→ Resetting
  ↑                                        │
  │                                        │ ticksToReset倒计时
  │                                        ↓
  └────────── Reset() ←──────────── ticksToReset ≤ 0

Disabled ←── Pawn.IsCharging() || Pawn.IsSelfShutdown() || CompCanBeDormant.!Awake
```

### 2.3 CompProperties_Shield 配置字段表

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `startingTicksToReset` | int | 3200 | 击破后重置倒计时（约53秒） |
| `minDrawSize` | float | 1.2 | 护盾气泡最小绘制尺寸 |
| `maxDrawSize` | float | 1.55 | 护盾气泡最大绘制尺寸（满能量时） |
| `energyLossPerDamage` | float | 0.033 | 每点伤害消耗的能量（1点伤害=0.033能量） |
| `energyOnReset` | float | 0.2 | 重置后的初始能量 |
| `blocksRangedWeapons` | bool | true | 是否阻止穿戴者使用远程武器 |

> **关键计算**：护盾腰带EnergyMax=1.1，energyLossPerDamage=0.033，因此满能量可吸收约 1.1/0.033 ≈ 33点伤害。

### 2.4 PostPreApplyDamage 伤害拦截逻辑（阶段4拦截点④）

```csharp
public override void PostPreApplyDamage(ref DamageInfo dinfo, out bool absorbed)
{
    absorbed = false;
    if (ShieldState != ShieldState.Active || PawnOwner == null) return;

    if (dinfo.Def == DamageDefOf.EMP)
    {
        energy = 0f;
        Break();                    // EMP直接击破，不吸收伤害
    }
    else if (!dinfo.Def.ignoreShields && (dinfo.Def.isRanged || dinfo.Def.isExplosive))
    {
        energy -= dinfo.Amount * Props.energyLossPerDamage;
        if (energy < 0f)
            Break();                // 能量耗尽→击破
        else
            AbsorbedDamage(dinfo);  // 成功吸收→视觉/音效反馈
        absorbed = true;            // 无论击破与否，伤害都被完全吸收
    }
    // 注意：近战伤害、ignoreShields伤害 → 不拦截，直接穿过
}
```

**拦截条件三重检查**：
1. `ShieldState == Active` — 护盾必须激活
2. `!dinfo.Def.ignoreShields` — 伤害类型不绕过护盾
3. `dinfo.Def.isRanged || dinfo.Def.isExplosive` — 必须是远程或爆炸伤害

**关键行为**：
- EMP伤害：直接击破护盾（energy=0 + Break()），但**不设absorbed=true**，EMP伤害仍然穿透
- 远程/爆炸伤害：无论是否击破护盾，`absorbed = true`，伤害被完全吸收
- 近战伤害：完全不拦截，直接穿过护盾

### 2.5 CompTick 能量恢复逻辑

```csharp
public override void CompTick()
{
    if (PawnOwner == null)
        energy = 0f;                              // 无主→能量归零
    else if (ShieldState == ShieldState.Resetting)
    {
        ticksToReset--;
        if (ticksToReset <= 0) Reset();           // 倒计时结束→重置
    }
    else if (ShieldState == ShieldState.Active)
    {
        energy += EnergyGainPerTick;              // 每tick恢复
        if (energy > EnergyMax) energy = EnergyMax;
    }
}
```

**恢复速率**：`EnergyGainPerTick = EnergyShieldRechargeRate / 60f`。护盾腰带RechargeRate=0.13/秒，即每tick恢复0.00217，从0恢复到满(1.1)需约507 ticks（约8.5秒）。

### 2.6 Break/Reset 击破与重置

**Break()**：
```csharp
private void Break()
{
    // 视觉效果：Shield_Break特效 + 闪光 + 6个灰尘粒子
    EffecterDefOf.Shield_Break.SpawnAttached(parent, parent.MapHeld, scale);
    energy = 0f;
    ticksToReset = Props.startingTicksToReset;  // 默认3200 ticks（约53秒）
}
```

**Reset()**：
```csharp
private void Reset()
{
    // 音效：EnergyShield_Reset + 闪电光效
    SoundDefOf.EnergyShield_Reset.PlayOneShot(...);
    ticksToReset = -1;
    energy = Props.energyOnReset;  // 默认0.2（不是满能量）
}
```

### 2.7 CompAllowVerbCast 远程武器限制

```csharp
public override bool CompAllowVerbCast(Verb verb)
{

        return !(verb is Verb_LaunchProjectile);  // 阻止发射投射物
```

### 3.3 CheckIntercept 拦截流程

```csharp
public bool CheckIntercept(Projectile projectile, Vector3 lastExactPos, Vector3 newExactPos)
{
    // 1. 距离预检：投射物是否在radius+速度+0.1范围内
    // 2. Active检查：护盾是否激活
    // 3. InterceptsProjectile：投射物类型检查（地面/空中）
    // 4. 敌对检查：投射物发射者是否敌对
    // 5. 出射检查：!interceptOutgoingProjectiles时，从内部射出的不拦截
    // 6. 几何检查：GenGeo.IntersectLineCircleOutline（线段与圆交叉）
    // 7. 记录拦截角度和时间
    // 8. EMP处理：disarmedByEmpForTicks > 0 时调用BreakShieldEmp
    // 9. HP扣减：currentHitPoints -= projectile.DamageAmount
    //    HP归零 → startedChargingTick + BreakShieldHitpoints
    return true;  // 拦截成功
}
```

**与CompShield的关键区别**：
- CompShield在**伤害管线阶段4**拦截（PostPreApplyDamage，伤害已命中目标）
- CompProjectileInterceptor在**投射物飞行阶段**拦截（CheckIntercept，投射物未到达目标）

### 3.4 两种能量模式

**HP模式**（hitPoints > 0）：
```
拦截投射物 → currentHitPoints -= DamageAmount
  ├── HP > 0 → 继续拦截
  └── HP ≤ 0 → BreakShieldHitpoints() → 开始充能
      → rechargeHitPointsIntervalTicks后恢复1点HP（或hitPointsRestoreInstantlyAfterCharge恢复满）
```

**充能模式**（chargeIntervalTicks > 0）：
```
Active → chargeIntervalTicks后 → Charging（chargeDurationTicks）→ Active
  └── Charging期间不拦截投射物
```

### 3.5 PostPreApplyDamage（直接伤害处理）

CompProjectileInterceptor也重写了PostPreApplyDamage，但**仅处理EMP**：

```csharp
public override void PostPreApplyDamage(ref DamageInfo dinfo, out bool absorbed)
{
    base.PostPreApplyDamage(ref dinfo, out absorbed);
    if (dinfo.Def == DamageDefOf.EMP && Props.disarmedByEmpForTicks > 0)
        BreakShieldEmp(dinfo);  // EMP直接命中护盾实体时禁用
}
```

> **注意**：CompProjectileInterceptor不在PostPreApplyDamage中吸收伤害（不设absorbed=true），它的拦截完全在CheckIntercept中完成。

## 6. Apparel.CheckPreAbsorbDamage 调用链

护盾腰带的伤害吸收通过以下调用链实现：

```
Pawn_HealthTracker.PreApplyDamage()          ← 伤害管线阶段4第3层
  └── for each Apparel in WornApparel:
      └── Apparel.CheckPreAbsorbDamage(dinfo)  ← 遍历所有服装
          └── for each ThingComp in AllComps:
              └── comp.PostPreApplyDamage(ref dinfo, out absorbed)
                  └── CompShield.PostPreApplyDamage()  ← 护盾拦截
                      └── absorbed = true → return true → 伤害被完全吸收
```

> **关键发现**：`Apparel.CheckPreAbsorbDamage`不是护盾腰带的自定义方法——它是`Apparel`基类的虚方法，遍历所有Comp的PostPreApplyDamage。任何带有能设置`absorbed=true`的Comp的服装都可以吸收伤害。

## 7. 护盾在伤害管线中的位置

```
阶段4 PreApplyDamage拦截链
  ├── ② 基因伤害倍率
  ├── ③ Hediff伤害倍率
  ├── ④ ThingComp.PostPreApplyDamage ←── CompShield（装备级，如机械体内置护盾）
  └── ⑤ Apparel.CheckPreAbsorbDamage ←── CompShield（服装级，如护盾腰带）

投射物飞行阶段（阶段1-2之间）
  └── Projectile.CheckForFreeIntercept ←── CompProjectileInterceptor（区域级）
```

> **拦截顺序**：CompProjectileInterceptor在投射物飞行时拦截（最早），CompShield在伤害应用前拦截（阶段4）。如果投射物被CompProjectileInterceptor拦截，根本不会到达CompShield。
