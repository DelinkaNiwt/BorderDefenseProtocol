**规模**：57个C#文件

**核心功能**：
- 装甲穿戴时自动赋予战术能力

```

├─ Ability_MagazineLaunch.cs          (能力释放实现)
├─ Ability_Xxx*.cs                    (具体能力变种)
├─ Apparel*.cs                        (装甲穿戴相关)
├─ Barrage*.cs                        (多重发射系统)
├─ Command_AbilityMagazine.cs         (UI命令)
├─ CompAbility_Magazine.cs            (弹匣组件) ← 核心
```

**关键设计**：
- `CompApparelGiveAbility` 负责装甲穿戴时赋予能力

**复杂度评级**：⭐⭐⭐⭐ (4/5) - 中等偏高

## 第七部分：架构演进的逻辑

### 7.1 为什么选择这样的项目划分？

```
设计原则：
1. 按功能独立性划分
   ├─ 护盾系统是独立的防御功能 → 单独项目
   ├─ 背包系统是独立的能力赋予 → 单独项目
   ├─ 炮塔系统是独立的火力 → 单独项目
   └─ 激光/电磁炮是炮塔的变种 → 可独立或合并

2. 按前置依赖整合
   ├─ 高复杂度系统（护盾）单独项目
   ├─ 中等系统（背包、海军炮）各自项目
   └─ 简单系统可以合并

3. 按编译单元优化
   ├─ 每个项目生成一个DLL
   ├─ 便于版本控制与迭代
   └─ 减少编译时间
```

### 7.2 如果要扩展模组，如何添加新功能？

```
场景1：添加新的护盾类型
  → 在 AbsoluteTerrorField 项目中添加新类
  → 不需要新建项目

场景2：添加新的炮塔类型
  → 在 HeavyNavalGunTurret 项目中添加
  → 或建立新项目 (如果差异很大)

场景3：添加新的装甲能力
  → 在 TurbojetBackpack 项目中添加
  → 或建立新项目 (如果是完全不同的系统)

场景4：添加完全新的防御/火力系统
  → 建立新项目，遵循命名规范
  → 在 About.xml 中注册
  → 如果使用Harmony补丁，放在项目内
```
