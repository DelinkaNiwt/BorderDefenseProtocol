## 1. 核心架构设计

### 1.1 RimWorld模组架构基础

联合重工模组基于RimWorld的核心架构模式：

| 架构层级 | 核心概念 | 实现方式 | 项目应用 |
|---------|--------|--------|--------|
| **Thing系统** | 游戏世界中的所有对象 | ThingComp组件 | 所有6个项目 |
| **MapComponent** | 地图级全局状态 | 单例MapComponent | ATFieldManager |
| **Harmony补丁** | 非侵入式代码扩展 | Prefix/Postfix | AbsoluteTerrorField(12+补丁) |
| **Def系统** | XML配置驱动 | ThingDef、AbilityDef等 | 88个XML配置文件 |

### 2.2 MapComponent单例模式

`ATFieldManager`演示了MapComponent的经典单例应用：

```csharp
public class ATFieldManager : MapComponent
{
    // 单例引用
    private static ATFieldManager _instance;
    public static ATFieldManager Instance => _instance;

    // 全局字段（地图级）
    private List<Comp_AbsoluteTerrorField> activeShields = new();
    private Dictionary<int, FieldState> stateCache = new();

    // 关键方法：中央调度点
    public bool TryInterceptProjectile(Projectile projectile, Vector3 pos)
    {
        // 遍历所有活跃护盾，尝试拦截
        foreach (var shield in activeShields)
        {
            if (shield.CheckIntercept(projectile, pos))
                return true;  // 第一个成功拦截即返回
        }
        return false;
    }

    // 注册/注销机制
    public void RegisterShield(Comp_AbsoluteTerrorField shield)
    {
        if (!activeShields.Contains(shield))
            activeShields.Add(shield);
    }

    public void UnregisterShield(Comp_AbsoluteTerrorField shield)
    {
        activeShields.Remove(shield);
    }
}
```

**优势分析**：
1. **中央协调**：所有护盾的拦截请求集中处理
2. **性能缓存**：避免重复计算
3. **生命周期管理**：护盾生成/销毁自动注册/注销
4. **补丁入口**：Harmony补丁仅需调用Manager，不需改动原始代码

## 3. 设计模式应用

### 3.1 状态机模式（State Machine）

**应用案例1：护盾能量状态**

```csharp
public class Comp_AbsoluteTerrorField : ThingComp
{
    // 状态枚举
    public enum ShieldState { Active, Resetting, Offline }

    private ShieldState currentState = ShieldState.Active;
    private int resetTicks = 0;
    private const int RESET_DURATION = 1000;

    public bool IsActive => currentState == ShieldState.Active;

    public override void CompTick()
    {
        switch (currentState)
        {
            case ShieldState.Active:
                // 正常运作：检测、拦截、消耗能量
                UpdateActive();
                if (energy <= 0)
                    TransitionTo(ShieldState.Resetting);
                break;

            case ShieldState.Resetting:
                // 能量回复阶段
                resetTicks++;
                RechargeEnergy();
                if (resetTicks >= RESET_DURATION)
                    TransitionTo(ShieldState.Active);
                break;

            case ShieldState.Offline:
                // 故障状态（可选）
                break;
        }
    }

    private void TransitionTo(ShieldState newState)
    {
        currentState = newState;
        resetTicks = 0;
        OnStateChanged(newState);
    }
}
```

**应用案例2：背包充能状态**

```csharp
public class CompAbility_Magazine : CompAbilityEffect
{
    private int charges;
    private int ticksToNextCharge;

    public float ReloadProgress =>
        1f - (float)ticksToNextCharge / Props.reloadTicks;

    public override void CompTick()
    {
        if (charges < Props.maxCharges)
        {
            ticksToNextCharge--;
            if (ticksToNextCharge <= 0)
            {
                ReloadOneCharge();  // 状态转移
            }
        }
    }

    public bool TryConsumeCharge()
    {
        if (charges > 0)
        {
            charges--;
            ticksToNextCharge = Props.reloadTicks;  // 重置充能计时器
            return true;
        }
        return false;
    }
}
```

**状态机优势**：
- 明确的状态转移逻辑
- 避免复杂的条件分支
- 易于调试和扩展

### 3.2 装饰器模式（Decorator）

Harmony补丁本质上是**运行时装饰器**，为原始方法添加功能：

```csharp
// 原始方法（RimWorld核心）
public static bool Projectile_TryImpact(Projectile __instance, ...)
{
    // 原始逻辑：弹幕撞击
}

// 装饰器（Harmony补丁）
[HarmonyPatch(typeof(Projectile), "TryImpact")]
public class Patch_Projectile_TryImpact
{
    public static bool Prefix(Projectile __instance, ...)
    {
        // 前置装饰：护盾拦截检查
        var manager = ATFieldManager.For(__instance.Map);
        if (manager != null && manager.TryInterceptProjectile(__instance, __instance.ExactPosition))
        {
            return false;  // 阻止原始方法执行
        }

        return true;  // 继续执行原始方法
    }

    public static void Postfix(Projectile __instance, ...)
    {
        // 后置装饰：记录日志、统计数据
    }
}
```

**装饰器优势**：
- 无需修改原始代码
- 支持动态添加/移除功能
- 多个补丁可以栈式组合

### 3.3 观察者模式（Observer）

护盾能量变化通知系统：

```csharp
public class Comp_AbsoluteTerrorField : ThingComp
{
    // 观察者列表
    private List<IShieldObserver> observers = new();

    public void RegisterObserver(IShieldObserver observer)
    {
        observers.Add(observer);
    }

    public void UnregisterObserver(IShieldObserver observer)
    {
        observers.Remove(observer);
    }

    // 能量变化时通知所有观察者
    private void ConsumeEnergy(float amount)
    {
        energy -= amount;

        // 通知观察者
        foreach (var observer in observers)
        {
            observer.OnEnergyChanged(energy, energy + amount);
        }

        if (energy <= 0)
        {
            foreach (var observer in observers)
            {
                observer.OnShieldDeactivated();
            }
        }
    }
}

// 观察者接口
public interface IShieldObserver
{
    void OnEnergyChanged(float newEnergy, float oldEnergy);
    void OnShieldActivated();
    void OnShieldDeactivated();
}

// 具体观察者：UI更新
public class ShieldUI : IShieldObserver
{
    private TextureRect energyBar;

    public void OnEnergyChanged(float newEnergy, float oldEnergy)
    {
        energyBar.UpdateValue(newEnergy / maxEnergy);
    }

    public void OnShieldDeactivated()
    {
        energyBar.Color = Color.red;
    }
}
```

### 3.5 策略模式（Strategy）

```csharp
public interface IInterceptionStrategy
{
    bool CanIntercept(Projectile projectile);
    float CalculateEnergyCost(float damage);
    void SpawnInterceptEffect(Vector3 position);
}

public class KineticInterceptionStrategy : IInterceptionStrategy
{
    public bool CanIntercept(Projectile projectile)
    {
        return projectile.def.projectile.damageType != DamageTypeDefOf.Flame;
    }

    public float CalculateEnergyCost(float damage)
    {
        return damage * 1.0f;  // 动能弹1:1消耗
    }

    public void SpawnInterceptEffect(Vector3 position)
    {
        MoteMaker.MakeExplosion(position, 5f, ThingDefOf.Mote_Smoke);
    }
}

public class EnergyInterceptionStrategy : IInterceptionStrategy
{
    public bool CanIntercept(Projectile projectile)
    {
        return projectile.def.IsRangedWeapon;
    }

    public float CalculateEnergyCost(float damage)
    {
        return damage * 0.7f;  // 能量弹70%消耗（反射）
    }

    public void SpawnInterceptEffect(Vector3 position)
    {
        MoteMaker.ThrowExplosion(position, 3f, ThingDefOf.Mote_Plasma);
    }
}

// 使用策略
public class Comp_AbsoluteTerrorField : ThingComp
{
    private Dictionary<DamageTypeDef, IInterceptionStrategy> strategies;

    public bool TryIntercept(Projectile projectile, float damage)
    {
        var strategy = GetStrategy(projectile.def.projectile.damageType);
        if (strategy == null || !strategy.CanIntercept(projectile))
            return false;

        float cost = strategy.CalculateEnergyCost(damage);
        if (!TryConsumeEnergy(cost))
            return false;

        strategy.SpawnInterceptEffect(projectile.ExactPosition);
        return true;
    }
}
```

## 5. 性能优化策略

### 5.1 Tick优化

RimWorld的Tick系统需要严格控制性能开销：

**优化原则**：
```csharp
// ❌ 糟糕：CompTick()中执行昂贵操作
public override void CompTick()
{
    // 每tick调用一次（频率：60Hz）
    foreach (var shield in FindAllShieldsOnMap())  // 昂贵！
    {
        var distance = Vector3.Distance(pos, shield.pos);  // 昂贵！
    }
    // 结果：严重FPS下降
}

// ✅ 优化：昂贵操作移到CompTickRare()
public override void CompTickRare()
{
    // 每250tick调用一次（频率：0.24Hz）
    RecalculateNearbyShields();
}

public override void CompTick()
{
    // 仅更新缓存数据
    UpdateShieldState();
}
```

**Tick优化表**：

| 操作 | 成本 | 优化位置 |
|-----|------|--------|
| 查询所有物体 | 昂贵 | CompTickRare |
| 距离计算 | 中等 | CompTickRare或缓存 |
| 状态更新 | 便宜 | CompTick |
| UI刷新 | 便宜 | CompTick |
| 列表遍历 | 取决于大小 | 优化算法 |

**实现示例**（来自ATFieldManager）：
```csharp
public class ATFieldManager : MapComponent
{
    private List<Comp_AbsoluteTerrorField> cachedShields = new();
    private int cacheUpdateTick = -1;

    public List<Comp_AbsoluteTerrorField> GetActiveShields()
    {
        // 每250tick更新一次缓存
        if (Find.TickManager.TicksGame % 250 == 0)
        {
            cachedShields.Clear();
            cachedShields.AddRange(FindAllShieldsOnMap());
            cacheUpdateTick = Find.TickManager.TicksGame;
        }
        return cachedShields;
    }
}
```

### 8.3 架构演进建议
