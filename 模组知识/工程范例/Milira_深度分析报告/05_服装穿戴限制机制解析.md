## 问题陈述

**是否存在穿戴限制？** ✅ **是的，确实存在！**

米莉拉和米莉安的服装都有专门的穿戴限制机制，**禁止其他种族的单位穿戴它们的衣服**。

## 第一部分：限制机制概览

### 1.1 核心机制：Harmony补丁

**目标类**：`AlienRace.RaceRestrictionSettings`
**目标方法**：`CanWear(ThingDef apparel, ThingDef race)`
**补丁类**：`MiliraAndMilian_RaceRestrictionSettings_CanWear_Patch`

**补丁代码**：
```csharp
[HarmonyPatch(typeof(RaceRestrictionSettings), "CanWear")]
public static class MiliraAndMilian_RaceRestrictionSettings_CanWear_Patch
{
	[HarmonyPrefix]
	public static bool Prefix(ThingDef apparel, ThingDef race, ref bool __result)
	{
		// 米莉拉服装限制处理
		if (race.defName == "Milira_Race" &&
		    !MiliraRaceSettings.MiliraRace_ModSetting_RaceRestrictedApparel)
		{
			__result = true;  // 米莉拉可以穿任何服装
			return false;     // 跳过原方法
		}

		// 米莉安服装限制处理
		if (!__result && MilianUtility.IsMilian(race))
		{
			__result = true;  // 米莉安可以穿该服装（通常是米莉安服装）
			return false;     // 跳过原方法
		}

		return true;  // 其他情况执行原方法
	}
}
```

**文件位置**：
```
Source/Milira/MiliraAndMilian_RaceRestrictionSettings_CanWear_Patch.cs
```

## 第二部分：限制机制详解

### 2.1 工作原理

#### 调用链：

```
游戏代码询问：
  "这个种族可以穿这件衣服吗？"
  → RaceRestrictionSettings.CanWear(apparel, race)

  ↓

Milira的Harmony补丁拦截：
  1. 检查是否米莉拉种族
  2. 检查是否米莉安种族
  3. 其他情况交给原方法

  ↓

返回结果：
  true  = 可以穿
  false = 不可以穿
```

### 2.2 米莉拉的限制逻辑

**代码片段**：
```csharp
if (race.defName == "Milira_Race" &&
    !MiliraRaceSettings.MiliraRace_ModSetting_RaceRestrictedApparel)
{
	__result = true;  // 允许穿戴
	return false;     // 跳过原方法的检查
}
```

**含义**：
- 检查条件1：是否米莉拉种族 (`race.defName == "Milira_Race"`)
- 检查条件2：用户设置是否**禁用**了服装限制
  - 如果设置**禁用限制** → `!MiliraRaceSettings...` 为 `true`
  - 如果设置**启用限制** → `!MiliraRaceSettings...` 为 `false` (不会进入此分支)

- **结果**：
  - 启用限制时：交给原方法处理（原方法会检查衣服是否是米莉拉衣服）
  - 禁用限制时：直接允许任何衣服 (`__result = true`)

**含义转化**：
```
如果（是米莉拉 && 设置要求有限制）{
	执行默认的HAR限制逻辑（原方法）
} 否则 {
	允许穿任何衣服
}
```

### 2.3 米莉安的限制逻辑

**代码片段**：
```csharp
if (!__result && MilianUtility.IsMilian(race))
{
	__result = true;  // 允许穿戴
	return false;     // 跳过原方法
}
```

**含义**：
- 前置条件：原方法返回 `false`（原本不允许穿）
- 再检查：是否米莉安种族
- **结果**：
  - 如果原本不允许但是米莉安 → 强制允许穿
  - 意思是：米莉安服装不受其他限制的约束

**逻辑转化**：
```
如果（原方法说不能穿 && 是米莉安）{
	改为允许穿（强制覆盖原结果）
}
```

## 第三部分：工作流程示例

### 示例1：米莉拉穿米莉拉服装

```
游戏询问：Milira_Race 能穿 Milira_Apparel_BaseUniform 吗？

Harmony补丁拦截：
  ✓ race.defName == "Milira_Race"  (是米莉拉)
  ? MiliraRaceSettings... 的值是什么？

  假设用户启用了限制：
    ✓ 条件都满足 → 执行原方法
    → 原方法检查：Milira_Apparel_BaseUniform 是米莉拉衣服吗？
    → 是！ → 返回 true

  假设用户禁用了限制：
    ✗ 第二个条件不满足 → 直接返回 true
    → 允许穿任何衣服

最终结果：允许穿戴 ✅
```

### 示例2：普通人类穿米莉拉服装

```
游戏询问：Human 能穿 Milira_Apparel_BaseUniform 吗？

Harmony补丁拦截：
  ✗ race.defName != "Milira_Race"  (不是米莉拉)
  → 第一个条件失败，进入第二个检查

  ✗ MilianUtility.IsMilian(race) = false  (不是米莉安)
  → 第二个条件也失败

  → return true (让原方法处理)

原方法（HAR的RaceRestrictionSettings.CanWear）：
  检查：Milira_Apparel_BaseUniform 有没有限制?
  → 有限制，只能米莉拉穿
  → 返回 false

最终结果：禁止穿戴 ❌
```

### 示例3：米莉安穿米莉安服装

```
游戏询问：Milian_Knight_I 能穿 Milian_Apparel_MilianBase 吗？

Harmony补丁拦截：
  ✗ race.defName != "Milira_Race"  (米莉安不等于米莉拉)
  → 第一个条件失败，进入第二个检查

  ✓ MilianUtility.IsMilian(race) = true  (是米莉安)
  ? !__result（原方法结果）是什么？

  假设原方法返回 false（原本不允许）：
    ✓ 条件满足 → 强制设为 true
    → return false (跳过原方法)

  假设原方法返回 true（原本允许）：
    ✗ 条件不满足（!true = false）
    → return true (让原方法的 true 结果生效)

最终结果：允许穿戴 ✅
```

## 第五部分：原理深度分析

### 5.1 为什么用 Harmony Prefix 补丁？

**Prefix 的特点**：
1. **拦截**：在原方法执行前运行
2. **控制**：可以修改参数和返回值
3. **短路**：可以通过返回 `false` 跳过原方法

**为什么选择 Prefix？**

```csharp
[HarmonyPrefix]
public static bool Prefix(ThingDef apparel, ThingDef race, ref bool __result)
{
	// 返回 false = 跳过原方法，使用修改后的 __result
	// 返回 true = 执行原方法，用原方法的结果覆盖 __result
}
```

**优势**：
- ✅ 可以有条件地跳过原方法（性能优化）
- ✅ 可以强制覆盖结果（override 逻辑）
- ✅ 可以基于多个条件做出决定

### 5.2 为什么针对 HAR 的 RaceRestrictionSettings？

**HAR（Humanoid Alien Races）的职责**：
- 提供种族框架
- 提供种族限制接口（`RaceRestrictionSettings.CanWear`）
- 允许模组通过补丁扩展限制逻辑

**Milira 的做法**：
- 不修改 HAR 的代码
- 通过 Harmony 补丁拦截 `CanWear` 方法
- 在补丁中实现自己的限制逻辑

**好处**：
- ✅ 不侵入 HAR 的代码
- ✅ 与 HAR 版本兼容
- ✅ 可以动态启用/禁用限制

## 第七部分：如何自己实现类似机制

### 7.1 标准范例：基于种族的服装限制

```csharp
using AlienRace;
using HarmonyLib;
using Verse;

namespace YourMod;

[HarmonyPatch(typeof(RaceRestrictionSettings), "CanWear")]
public static class YourRace_RaceRestrictionSettings_CanWear_Patch
{
	[HarmonyPrefix]
	public static bool Prefix(ThingDef apparel, ThingDef race, ref bool __result)
	{
		// 方案1：你的种族可以穿任何衣服
		if (race.defName == "YourRace_Race")
		{
			__result = true;
			return false;  // 跳过原方法
		}

		// 方案2：只有你的种族能穿你的衣服
		if (!__result && IsYourCustomRace(race))
		{
			__result = true;
			return false;  // 强制允许
		}

		return true;  // 让原方法处理其他情况
	}

	private static bool IsYourCustomRace(ThingDef race)
	{
		return race?.defName?.Contains("YourRace") == true;
	}
}
```

### 7.2 关键要点

✅ **必须引入正确的命名空间**
```csharp
using AlienRace;  // 获取 RaceRestrictionSettings
using HarmonyLib; // 获取 Harmony 相关属性
```

✅ **方法签名必须完全匹配**
```csharp
public static bool Prefix(ThingDef apparel, ThingDef race, ref bool __result)
//                                                                   ↑
//                      out 参数，用 ref 修饰，Harmony 会自动填充
```

✅ **正确理解返回值**
```csharp
return false;  // 跳过原方法，使用 __result 的值
return true;   // 执行原方法，用原方法的结果覆盖 __result
```

## 总结

### 核心发现

✅ **米莉拉和米莉安的服装确实有穿戴限制**

✅ **限制机制**：通过 Harmony 补丁拦截 `RaceRestrictionSettings.CanWear` 方法

✅ **实现方式**：
- 米莉拉：可配置的限制（用户可在 mod 选项中启用/禁用）
- 米莉安：固定的限制（不可改）

✅ **技术特点**：
- 非侵入式补丁（不修改 HAR 源代码）
- 条件判断（根据种族和设置动态决定）
- 结果覆盖（可以强制允许或禁止）

这个实现展示了：
1. **如何使用 Harmony Prefix 补丁拦截方法**
2. **如何实现基于种族的装备限制**
3. **如何在补丁中读取模组设置**
4. **如何优雅地扩展第三方库的功能**
