## 3. 核心系统分解

### 3.1 种族系统 (Race System)

**定义**：通过 Humanoid Alien Races 框架为玩家提供可选的异种族体验。

**核心组件**：
- **种族定义** → ThingDef_Races 配置
- **身体结构** → Bodies, BodyParts, BodyPartGroups 定义
- **视觉渲染** → PawnRenderTree, HeadTypes, Hairs 定义
- **文化体系** → Cultures 定义（1.6新增）
- **背景故事** → Backstories（成人与儿童）

**参考文件**：
```
Defs/ThingDefs_Races/Race_*.xml
Defs/BodyDefs/Bodies_*.xml
Defs/BodyPartDefs/BodyParts_*.xml
Defs/CultureDefs/Cultures.xml
Defs/BackstoryDefs/Backstory_*.xml
```

---

### 3.2 能力系统 (Ability System)

**定义**：基于 RimWorld Biotech DLC 的能力框架，为米莉拉提供独特的战斗和生活能力。

**特征**：
- 种族特有能力
- 能力分类管理
- 多级别能力升级
- 与装备/修饰的联动

**文件组织**：
```
Defs/AbilityDefs/
├── AbilityCategories.xml          # 能力分类定义
├── MiliraAbilities.xml             # 米莉拉主要能力
├── MiliraAbilities_Misc.xml        # 杂项能力
└── MilianAbilities.xml             # 米莉安（亚种族）能力
```

**代码实现**：
```
Source/Milira/
├── CompAbilityEffect_*.cs          # 12+ 个能力效果类
├── CompAbilityCheckCarrier.cs      # 能力前置条件检查
└── [其他能力支撑类]
```

---

### 3.3 机械体/浮空单位系统 (Mechanical Unit System)

**定义**：实现 16+ 种可召唤的机械化单位，包括浮空单位、战斗机械人等。

**单位类型分析**：
- **浮空单位** (Float Unit)：无地面限制的飞行单位
- **米莉安守卫** (Milian Guard)：工程化生命体
- **战斗无人机** (Combat Drones)：自动化战斗单位
- **运输机械** (Mech Carriers)：携带其他单位的母体

**核心组件**：
- **单位定义** → PawnKinds_*.xml
- **能力配置** → 各种 Ability 定义
- **智能系统** → Job 和行为定义
- **生产流程** → Recipe 和 Research 链

**参考文件**：
```
Defs/PawnKindDef/PawnKinds_*.xml
Defs/ThingDefs_Misc/[单位装备和武器]
Source/Milira/
├── CompFloatingSystem.cs           # 浮空单位核心
├── CompMechCarrier_*.cs            # 机械运输母体
├── CompDrone_*.cs                  # 无人机系统
└── [其他单位控制类]
```

## 4. 前置依赖架构

### 4.1 必要前置模组

#### A. Humanoid Alien Races 2.0 (erdelf.HumanoidAlienRaces)
**职责**：提供异种族框架

**关键API**：
- `AlienRace.RaceSettings` - 种族配置类
- 种族定制的身体结构、头部、发型系统
- 视觉渲染框架

**集成点**：
```xml
<!-- LoadFolders.xml -->
<li IfModActive="erdelf.HumanoidAlienRaces">...</li>

<!-- About.xml -->
<packageId>erdelf.HumanoidAlienRaces</packageId>
<loadAfter>erdelf.HumanoidAlienRaces</loadAfter>
```

#### B. Ancot Library (Ancot.AncotLibrary)
**职责**：提供通用工具库和组件

**关键API**：
- `AncotLibrary` 命名空间（30+ 个工具类）
- `AncotUtility.QualityFactor()` - 品质计算
- `CompDrone`, `CompMechCarrier_Custom` 等基类
- `CompWeaponCharge`, `CompThingContainer_*` 等组件基类

**集成点**：
```csharp
using AncotLibrary;

// 示例：品质因子计算
float damageAmount = AncotUtility.QualityFactor(quality) * baseDamage;

// 示例：继承基类
public class CompMechCarrier_Consul : CompMechCarrier_Custom
```

### 4.2 必要DLC集成

#### A. Royalty (Ludeon.RimWorld.Royalty)
**职责**：提供能力框架、仪式系统等

**关键集成**：

**条件加载**：
```xml
<li IfModActive="Ludeon.RimWorld.Royalty">1.6/Mods/Royalty</li>
```

#### B. Biotech (Ludeon.RimWorld.Biotech)
**职责**：提供机械化、基因、蜂群系统

**关键集成**：
- 机械体框架（CompMechCarrier）

**条件加载**：
```xml
<li IfModActive="Ludeon.RimWorld.Biotech">1.6/Mods/Biotech</li>
```

## 7. 设计模式与架构亮点

### 7.1 版本化架构 (Multi-Version Design)

**模式**：
```
Module
├── 1.5/                # RimWorld 1.5 专用版本
├── 1.6/                # RimWorld 1.6 专用版本
└── Content/            # 跨版本共享内容
```

**好处**：
- 同时支持多个游戏版本
- 版本特定的DLC适配（Odyssey仅在1.6）
- 清晰的版本管理

### 7.2 条件加载模式 (Conditional Loading)

**机制**：
```xml
<li IfModActive="Ancot.KiiroRace">1.5/Mods/Ancot.KiiroRace</li>
<li IfModActive="Ludeon.RimWorld.Royalty">1.6/Mods/Royalty</li>
```

**优点**：
- 只加载必需的内容
- 避免前置模组不存在时的错误
- 灵活的功能启用/禁用

### 7.3 模块化组件设计 (Modular Components)

**特点**：
- 充分利用 Harmony 补丁进行拓展
- 组件松耦合，独立功能
- 易于添加新能力、新单位

**示例**：
```
CompAbilityEffect_* → 每个能力一个独立类
CompMechCarrier_* → 每个机械母体一个类
```

### 7.4 前置框架充分利用 (Framework Leverage)

**特点**：
- 继承 HAR 的种族框架，最小化重复代码
- 继承 AncotLibrary 的基类（CompDrone, CompMechCarrier）
- 减少维护工作

## 10. 总体评价

### 10.1 架构优势

✅ **清晰的模块化** - 功能划分明确
✅ **充分利用前置框架** - 代码复用度高
✅ **多版本支持** - 不同游戏版本兼容
✅ **灵活的条件加载** - DLC和模组适配优雅
✅ **丰富的内容** - 292个源文件+187个Defs，内容深度高

### 10.2 技术特色

✅ **深度系统集成** - 从种族、能力、单位、建筑到武器
✅ **视觉渲染复杂度** - 多层PawnRenderTree实现精细渲染
✅ **游戏平衡设计** - 品质系统、蓄能系统等平衡机制
✅ **良好的代码组织** - 命名规范、模式一致

### 10.3 学习价值

这个模组展示了：
