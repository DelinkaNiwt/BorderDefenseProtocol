## 第三部分：核心设计模式总结

### 3.1 五大代码模式

**模式1：组件模式 (Component Pattern)**
- 继承 `CompDrone`, `CompMechCarrier_Custom` 等基类
- 简洁且易于扩展
- 典型文件：`CompDrone_Milira.cs`, `CompMechCarrier_Consul.cs`

**模式2：能力效果模式 (Ability Effect Pattern)**
- 继承 `CompAbilityEffect`
- 实现 `Apply()`, `GetPreCastActions()`, `DrawEffectPreview()`
- 典型文件：`CompAbilityEffect_Excalibur.cs` (12+个)

**模式3：建筑实现模式 (Building Pattern)**
- 继承专化的建筑基类
- 静态资源预加载、生命周期管理
- 典型文件：`Building_TurretGunFortress.cs`, `Building_SunBlastFurnace.cs`

**模式4：任务系统模式 (Job Pattern)**
- `JobDriver` 定义任务步骤，`JobGiver` 定义任务生成
- 使用 `Toil` 原子步骤实现复杂任务
- 典型文件：`JobDriver_DisassembleBroadShieldUnit.cs` (8+个)

**模式5：Harmony补丁模式 (Harmony Patch Pattern)**
- 使用 `[HarmonyPatch]` 非侵入式修改游戏逻辑
- `[StaticConstructorOnStartup]` 自动启动补丁
- 启动文件：`Ancot_MiliraRace_Patch.cs`

### 3.2 代码组织特点

✅ **强隔离**
- 所有代码在 `namespace Milira;` 下
- 避免与其他模组冲突

✅ **充分利用继承**
- 继承 AncotLibrary 基类减少代码重复
- 继承 RimWorld 基类获得核心功能

✅ **清晰的命名约定**
- `Comp*` - 组件
- `Building_*` - 建筑
- `Job*` - 任务
- `CompAbilityEffect_*` - 能力

✅ **防御性编程**
- 大量 null 检查
- 安全的类型转换
- 组件间通过公共接口通信

### 3.3 关键技术亮点

| 技术 | 用途 | 文件 |
|------|------|------|
| Harmony补丁 | 非侵入式扩展游戏逻辑 | Ancot_MiliraRace_Patch.cs |
| DefOf模式 | 编译时类型检查，避免字符串硬编码 | MiliraDefOf.cs |
| 属性计算链 | 复杂数值计算的模块化实现 | CompMechCarrier_Consul.cs |
| 组件获取 | `TryGetComp<T>()` 安全访问 | 292个源文件广泛使用 |
| 运行时条件 | `ModsConfig.IsActive()` 处理可选模组 | 多个源文件 |

## 第五部分：前置模组集成特色

### 5.1 About.xml 声明写法

**标准结构**：
```xml
<modDependencies>
	<li>
		<packageId>前置模组ID</packageId>
		<displayName>显示名称</displayName>
		<steamWorkshopUrl>https://steamcommunity.com/...</steamWorkshopUrl>
	</li>
</modDependencies>

<loadAfter>
	<li>前置模组ID</li>
</loadAfter>
```

**Milira 的声明**：
- ✅ 清晰列出4个前置（2个模组 + 2个DLC）
- ✅ 提供 Steam 链接（方便用户查找）
- ✅ 在 loadAfter 中保证加载顺序

### 5.2 LoadFolders.xml 版本化加载

**模式**：
```xml
<v1.5>
	<li>1.5</li>                                        <!-- 基础内容 -->
	<li>Content</li>                                    <!-- 共享内容 -->
	<li IfModActive="Ludeon.RimWorld.Royalty">...</li> <!-- 条件加载 -->
</v1.5>

<v1.6>
	<li>1.6</li>                                        <!-- 新版本内容 -->
	<li>Content</li>
	<li IfModActive="Ludeon.RimWorld.Odyssey">...</li> <!-- 1.6新增 -->
</v1.6>
```

**特点**：
- ✅ 支持多个RimWorld版本（1.5, 1.6）
- ✅ 动态加载DLC和可选模组内容
- ✅ 避免文件缺失的错误

### 5.3 源代码API集成方式

**命名空间引入**：
```csharp
using AncotLibrary;              // 前置库
using RimWorld;                  // 游戏核心
using Verse;                     // 基础框架
using UnityEngine;               // 引擎
```

**基类继承**：
```csharp
public class CompDrone_Milira : CompDrone { }
public class CompMechCarrier_Consul : CompMechCarrier_Custom { }
```

**工具类使用**：
```csharp
float damageAmount = AncotUtility.QualityFactor(quality) * baseDamage;
```

**可选模组检查**：
```csharp
if (ModsConfig.IsActive("Ancot.MilianModification")) { /* ... */ }
```

### 6.2 对模组开发者的建议

**阶段1：架构设计**
1. 参考 Milira 的系统划分方式
2. 明确前置依赖和版本支持
3. 规划文件组织结构

**阶段2：前置集成**
1. 在 About.xml 中清晰声明前置
2. 在 LoadFolders.xml 中配置版本和条件加载
3. 在代码中正确引入和使用前置库API

**阶段3：代码开发**
1. 学习五大核心设计模式
2. 使用 DefOf 模式管理常量
3. 采用防御性编程习惯

**阶段4：配置设计**
1. 遵循 Milira 的命名规范
2. 充分使用 Parent 继承
3. 分离配置为模块化的XML文件

### 6.3 常见错误与避坑指南

| 错误 | 后果 | 避免方法 |
|------|------|---------|
| About.xml中的前置ID错误 | 用户找不到前置 | 仔细核对packageId |
| LoadFolders.xml条件拼写错误 | 文件无法加载 | 使用与About.xml相同的ID |
| 源代码中硬编码字符串 | 拼写错误导致功能失效 | 使用DefOf模式 |
| 忘记检查null | 游戏崩溃 | 所有组件获取都要null检查 |
| 不适当的类型转换 | 类型异常 | 使用TryGetComp<T>() |
