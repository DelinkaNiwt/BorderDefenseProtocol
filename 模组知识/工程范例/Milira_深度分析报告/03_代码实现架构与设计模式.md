### 1.1 命名空间设计

**主命名空间**：`Milira`

所有 292 个源文件都在 `namespace Milira;` 下，实现强隔离：

```csharp
namespace Milira;

public class ComponentName : BaseClass
{
	// ...
}
```

**优势**：
- ✅ 避免与其他模组的类名冲突
- ✅ 清晰的代码来源标识
- ✅ 便于代码组织和查找

### 2.1 模式1：组件模式 (Component Pattern)

**定义**：继承 `ThingComp` 或专化的组件基类，为对象添加功能。

#### 示例1：简单组件继承

**文件**：`CompDrone_Milira.cs`

```csharp
using AncotLibrary;

namespace Milira;

/// 简单的无人机组件
public class CompDrone_Milira : CompDrone
{
	// 覆写虚属性
	public override bool Draftable => MiliraDefOf.Milira_DroneControl.IsFinished;
}
```

**特点**：
- 继承 AncotLibrary 的 `CompDrone` 基类
- 只覆写必要的虚方法/属性
- 简洁且高度可维护

### 2.2 模式2：能力效果模式 (Ability Effect Pattern)

**定义**：继承 `CompAbilityEffect`，实现自定义的能力效果。

#### 示例：Excalibur能力

**文件**：`CompAbilityEffect_Excalibur.cs`

```csharp
using System.Collections.Generic;
using System.Linq;
using AncotLibrary;
using RimWorld;
using UnityEngine;
using Verse;

public class CompAbilityEffect_Excalibur : CompAbilityEffect
{

	private float damageAmount
		=> AncotUtility.QualityFactor(quality) * damageAmountBase;

	// 属性2：利用品质计算护甲穿透
	private float armorPenetration
		=> AncotUtility.QualityFactor(quality) * armorPenetrationBase;

	// 属性3：快速访问蓄能组件
	public CompWeaponCharge compCharge
		=> ((Thing)weapon).TryGetComp<CompWeaponCharge>();

	// 属性4：获取配置
	private new CompProperties_AbilityExcalibur Props
		=> (CompProperties_AbilityExcalibur)props;

	// 核心方法：执行能力效果
	public override void Apply(LocalTargetInfo target, LocalTargetInfo dest)
	{
		// 1. 检查蓄能
		if (compCharge != null && !compCharge.CanBeUsed)
		{
			return;
		}

		// 2. 消耗蓄能
		CompWeaponCharge obj = compCharge;
		if (obj != null)
		{
			obj.UsedOnce();
		}

		// 3. 计算效果范围
		List<IntVec3> affectedCells = AffectedCells(target);

		// 4. 应用爆炸效果
		Map map = Pawn.Map;
		foreach (IntVec3 cell in affectedCells)
		{
			GenExplosion.DoExplosion(
				cell, map, 0.9f,
				MiliraDefOf.Milira_PlasmaBomb_TwoHandSword,
				Pawn,
				damageAmount: 3 * (int)damageAmount,
				armorPenetration: 2f * armorPenetration,
				// ... 其他参数
			);
		}

		// 5. 添加视觉效果
		parent.AddEffecterToMaintain(
			MiliraDefOf.Milira_TwoHandSwordFire.Spawn(
				parent.pawn.Position, target.Cell, parent.pawn.Map),
			Pawn.Position, target.Cell, 50, Pawn.MapHeld
		);

		// 6. 调用基类逻辑
		base.Apply(target, dest);
	}

	{

		{
			action = delegate

					MiliraDefOf.Milira_TwoHandSwordWarmup.Spawn(
						Pawn, Pawn.Map),
					Pawn.Position, 480, Pawn.Map
				);
			},
			ticksAwayFromCast = 600
		};
	}

	// UI预览绘制
	public override void DrawEffectPreview(LocalTargetInfo target)
	{
		GenDraw.DrawFieldEdges(AffectedCells(target));
	}

	// 计算影响范围的私有方法
	private List<IntVec3> AffectedCells(LocalTargetInfo target)
	{
		// ... 复杂的范围计算逻辑
	}
}
```

**关键设计要点**：
1. **分离关注点**：每个方法负责一个明确的职责
2. **工具使用**：充分利用 `AncotUtility.QualityFactor()`
3. **组件获取**：通过 `TryGetComp<T>()` 安全获取组件
4. **多步骤流程**：能力效果分解为可管理的步骤

### 2.5 模式5：Harmony补丁模式

**定义**：使用 Harmony 库对游戏逻辑进行非侵入式修改。

#### 启动模式

**文件**：`Ancot_MiliraRace_Patch.cs`

```csharp
using System.Reflection;
using HarmonyLib;
using Verse;

namespace Milira;

[StaticConstructorOnStartup]
public class Ancot_MiliraRace_Patch
{
	static Ancot_MiliraRace_Patch()
	{
		// 创建Harmony实例
		Harmony harmony = new Harmony("Ancot.MiliraRaceHarmonyPatch");

		// 自动扫描并应用所有补丁
		harmony.PatchAll(Assembly.GetExecutingAssembly());
	}
}
```

**特点**：
- `[StaticConstructorOnStartup]` - 游戏启动时自动执行
- `Assembly.GetExecutingAssembly()` - 扫描当前程序集中的所有补丁类
- 无需手动注册每个补丁

#### 补丁示例结构

```csharp

		// 昂贵的初始化只执行一次
		return new List<Pawn>();
	}
}
```

**优势**：
- 游戏启动更快
- 只初始化实际使用的组件

		ExpensiveOperation();

**优势**：

## 第八部分：最佳实践建议

### 对于模组开发者

1. **使用 DefOf 模式**
   - 避免字符串硬编码
   - 获得编译时检查

2. **利用继承减少重复**
   - 继承前置库的基类
   - 只覆写需要改变的部分

3. **组件间通信**
   - 通过 `TryGetComp<T>()` 查询
   - 避免直接访问内部状态

4. **防御性编程**
   - 大量的 null 检查
   - 安全的类型转换

5. **性能优化**
   - 属性缓存以避免重复计算
   - 间隔执行昂贵操作
   - 延迟初始化
