     ↓

```

    ↓
[检查触发条件]
    ├─ 威胁点数 > 20?
    ├─ 距离上次触发 > 10 小时?
    ├─ 随机通过 MilianClusterChance 概率?
    └─ 米莱拉未转向友好?
    ↓
[选择事件类型]
    ├─ 30% → 集群袭击 (IncidentWorker_MiliraCluster)

    │        ↓ GenStep_MilianCluster 创建地图
    │
    └─ 直接：使用 IncidentWorker_RaidEnemy 的标准逻辑
    ↓
[后处理]
    ├─ 增加威胁点数 (+威胁值)
    ├─ 启动冷却计时器 (lastThreatTick)
    └─ 自动衰减机制 (每 60000 tick -1 点，当 > 20 时)
```

## 第2部分：核心机制详解

### 2.1 威胁点数系统 (Threat Point System)

#### 2.1.1 威胁点数的生命周期

| 阶段 | 操作 | 触发条件 | 代码位置 |
|------|------|---------|---------|
| **初始化** | `miliraThreatPoint = 0` | 游戏开始 | MiliraGameComponent_OverallControl.StartedNewGame():118 |
| **增加** | 每次事件触发 +威胁值 | 事件执行后 | 游戏组件接收事件回调（隐式） |
| **衰减** | 每 60000 tick -1 点 | 点数 > 20 | MiliraGameComponent_OverallControl.GameComponentTick():63-66 |
| **查询** | MilianClusterChance.Evaluate(miliraThreatPoint) | 决定触发概率 | MiliraGameComponent_OverallControl.GameComponentTick():67 |

#### 2.1.2 MilianClusterChance 触发曲线

```csharp
// 源代码：MiliraGameComponent_OverallControl.cs:33-39
private static readonly SimpleCurve MilianClusterChance = new SimpleCurve
{
    new CurvePoint(0f, 0f),      // 0点   → 0%    概率（无触发）
    new CurvePoint(20f, 0.01f),  // 20点  → 1%    概率（很低）
    new CurvePoint(100f, 0.8f),  // 100点 → 80%   概率（较高）
    new CurvePoint(1000f, 1f)    // 1000点 → 100%  概率（必然触发）
};
```

**曲线分析**：
- **0-20点**：线性增长从 0% 到 1%（大约需要 1800+ 小时的游戏时间）
- **20-100点**：急剧增长从 1% 到 80%（游戏进度加速）
- **100-1000点**：逐渐逼近 100%（后期威胁增加）

#### 2.1.3 触发间隔保护

```csharp
// 源代码：MiliraGameComponent_OverallControl.cs:67
if (Find.TickManager.TicksGame > 600000 + lastThreatTick && ...)
```

- **600000 tick** = 约 10 小时游戏时间
- **用途**：防止事件频繁触发，保证玩家有充足的准备时间
- **lastThreatTick**：记录上次触发的游戏时间

### 2.2 事件触发决策树 (Decision Tree)

```
[第 27000 tick (3小时) 的检查点]
    ↓
[检查威胁点数] miliraThreatPoint > 20 ?
    ├─ NO  → [跳过本轮检查，继续等待]
    └─ YES ↓

[检查冷却时间] TicksGame > 600000 + lastThreatTick ?
    ├─ NO  → [还在冷却中，跳过本轮]
    └─ YES ↓

[随机概率检查] Rand.Chance(MilianClusterChance.Evaluate(miliraThreatPoint)) ?
    ├─ NO  → [本轮不触发，继续等待]
    └─ YES ↓

[检查派系状态] !turnToFriend ?
    ├─ NO  → [已转向友好，停止所有袭击]
    └─ YES ↓

[选择事件类型]
    ├─ 30% → SendMilianClusterRaid()  [集群袭击]
    │        ├─ 创建 IncidentParms
    │        ├─ points = DefaultThreatPointsNow() [动态难度]
    │        └─ Queue: Milira_MilianCluster
    │
    └─ 70% → SendMiliraRaid()  [直接袭击]
             ├─ 创建 IncidentParms
             ├─ faction = Milira_Faction
             ├─ raidArrivalMode = EdgeDrop
             ├─ raidStrategy = ImmediateAttack
             ├─ points = DefaultThreatPointsNow() [动态难度]
             └─ Queue: Milira_Raid
```

**关键决策点**：
1. **威胁点数过滤**：只有积累足够的威胁点数才会触发
2. **冷却保护**：连续触发的最小间隔是 10 小时
3. **随机性**：概率曲线使早期袭击稀少，后期频繁
4. **派系检查**：通过转向友好来停止所有袭击

3. **敌方数量**：

4. **防御设施**：
   - 激活器数量：ProblemCauserCountCurve.Evaluate(1560) ≈ 0.92 → 约 1 个

#### 2.4.3 集群位置选择算法 (MiliraClusterUtility.FindClusterPosition)

```csharp
// 源代码：MiliraClusterUtility.cs:28-79
public static IntVec3 FindClusterPosition(Map map, MechClusterSketch sketch,
                                          int maxTries = 100,
                                          float spawnCloseToColonyChance = 0f)
{
    // 第1优先级：靠近殖民地（如果触发概率）
    if (Rand.Chance(spawnCloseToColonyChance))  // 默认 0f，可在特定情况启用
    {
        for (int i = 0; i < 20; i++)
        {
            // 使用 DropCellFinder 查找离殖民地近的空投位置
            if (DropCellFinder.TryFindRaidDropCenterClose(out var spot, map, ...))
            {
                float score = GetClusterPositionScore(spot, map, sketch);
                if (score == 100f)  // 完美分数立即返回
                    return spot;
                if (score > bestScore)
                    bestScore = score;
            }
        }
    }

    // 第2优先级：远离殖民地（默认）
    for (int j = 0; j < maxTries; j++)  // 尝试 100 次
    {
        // 随机选择一个可站立位置
        IntVec3 entrySpot = CellFinderLoose.RandomCellWith(
            (IntVec3 x) => x.Standable(map), map
        );

        // 计算 RimWorld 围攻阵地（围攻相关的最优阵地）
        IntVec3 intVec = RCellFinder.FindSiegePositionFrom(entrySpot, map, ...);

        float score = GetClusterPositionScore(intVec, map, sketch);
        if (score == 100f)  // 完美分数立即返回
            return intVec;
    }

    // 第3降级：如果无法找到完美位置，返回随机有效位置
    return CellFinder.RandomCell(map);
}
```

**位置选择策略**：
1. **优先级 1**：靠近殖民地（特定情况，默认不启用）
   - 用于某些特殊事件（如教会入侵）
   - 增加威胁感

2. **优先级 2**：远离殖民地、围攻位置（默认）
   - 使用 RCellFinder.FindSiegePositionFrom()
   - 模拟真实围攻的位置选择

3. **优先级 3**：任意有效位置（降级）
   - 确保集群必定生成

## 第3部分：设计模式提炼

### 3.1 触发机制模式 (Event Triggering Pattern)

#### 模式特征
- **基于游戏组件的周期性检查**
- **多层条件过滤**
- **概率曲线驱动**
- **内部资源积累（威胁点数）**

#### 可复用模板

```csharp
/// <summary>
/// 通用的周期性事件触发框架
/// 用于实现需要积累和触发的系统（如：援军到达、敌方扩大、灾难事件）
/// </summary>
public class GameComponent_PeriodicEventTrigger : GameComponent
{
    // 资源积累
    private int resourcePoints = 0;
    private int lastTriggerTick = -99999;

    // 触发曲线定义
    private static readonly SimpleCurve TriggerChanceCurve = new SimpleCurve
    {
        new CurvePoint(0f, 0f),
        new CurvePoint(50f, 0.05f),
        new CurvePoint(200f, 0.8f),
        new CurvePoint(1000f, 1f)
    };

    // 周期常数
    private const int CheckIntervalTicks = 27000;  // 3小时
    private const int CooldownTicks = 600000;      // 10小时
    private const int DecayIntervalTicks = 60000;  // 1小时
    private const int DecayThreshold = 20;

    public override void GameComponentTick()
    {
        // 检查点 1：周期性检查
        if (Find.TickManager.TicksGame % CheckIntervalTicks != 0)
            return;

        // 检查点 2：资源阈值
        if (resourcePoints < DecayThreshold)
            return;

        // 检查点 3：冷却时间
        if (Find.TickManager.TicksGame <= CooldownTicks + lastTriggerTick)
            return;

        // 检查点 4：概率检查
        if (!Rand.Chance(TriggerChanceCurve.Evaluate(resourcePoints)))
            return;

        // 检查点 5：其他条件（如派系状态）
        if (!CanTriggerNow())
            return;

        // 触发事件！
        TriggerEvent();
        lastTriggerTick = Find.TickManager.TicksGame;
    }

    // 自动衰减机制
    public override void GameComponentTick()
    {
        if (resourcePoints > DecayThreshold &&
            Find.TickManager.TicksGame % DecayIntervalTicks == 0)
        {
            resourcePoints--;
        }
    }

    protected virtual void TriggerEvent() { /* 子类覆盖 */ }
    protected virtual bool CanTriggerNow() { return true; }
}
```

#### 应用场景
- 敌方强度系统（资源 = 威胁点数）
- 随机灾难频率（资源 = 灾难积累度）
- NPC 派系压力（资源 = 压力值）
- 探险队回归概率（资源 = 旅行天数）

#### 关键优势
✓ **自平衡**：资源自动衰减，防止系统失控
✓ **玩家可控**：通过游戏进度自动调整触发频率
✓ **易于扩展**：曲线参数可轻松调整触发行为
✓ **降低频率**：周期性检查而非每 tick 检查

### 3.2 集群生成模式 (Cluster Generation Pattern)

#### 模式特征
- **参数化生成**：使用多条曲线定义各个维度
- **随机性与确定性结合**：曲线决定范围，随机数决定具体值
- **分层生成**：大小 → 敌方数量 → 防御设施 → 激活机制

#### 可复用模板

```csharp
/// <summary>
/// 通用的参数化集群生成框架
/// 用于生成具有一定复杂度的敌方阵地（机械集群、建筑群、敌方营地等）
/// </summary>
public static class ParameterizedClusterGenerator
{
    /// 核心配置结构
    public class ClusterConfig
    {
        // 尺寸
        public SimpleCurve SizeCurve { get; set; }          // 点数 → 集群大小(格子)

        // 单位
        public SimpleCurve PawnCountCurve { get; set; }     // 点数 → 敌方数量
        public List<(PawnKindDef, float)> UnitWeights;      // 单位及权重

        // 防御
        public SimpleCurve DefenseBuildingCurve { get; set; } // 点数 → 防御建筑数量
        public List<ThingDef> DefenseBuildings;             // 防御建筑列表

        // 激活
        public SimpleCurve ActivatorCurve { get; set; }     // 点数 → 激活器数量
        public ThingDef ActivatorDef { get; set; }
    }

    /// 生成方法
    public static void GenerateCluster(
        float points,
        ClusterConfig config,
        out int clusterSize,
        out List<Pawn> enemies,
        out List<Thing> buildings,
        out List<Thing> activators)
    {
        // 第1步：计算集群大小
        clusterSize = (int)config.SizeCurve.Evaluate(points);

        // 第2步：生成敌方单位
        int pawnCount = (int)config.PawnCountCurve.Evaluate(points);
        enemies = GenerateWeightedPawns(pawnCount, config.UnitWeights);

        // 第3步：生成防御建筑
        int buildingCount = (int)config.DefenseBuildingCurve.Evaluate(points);
        buildings = GenerateDefenseBuildings(buildingCount, config.DefenseBuildings);

        // 第4步：生成激活机制
        int activatorCount = (int)config.ActivatorCurve.Evaluate(points);
        activators = GenerateActivators(activatorCount, config.ActivatorDef);
    }

    private static List<Pawn> GenerateWeightedPawns(
        int count,
        List<(PawnKindDef, float)> weights)
    {
        var pawns = new List<Pawn>();
        for (int i = 0; i < count; i++)
        {
            // 使用权重随机选择单位类型
            var kind = weights.RandomElementByWeight(w => w.Item2).Item1;
            pawns.Add(PawnGenerator.GeneratePawn(kind));
        }
        return pawns;
    }
}
```

#### 应用场景
- 机械集群布置
- 敌方营地生成
- BOSS 战场布置（BOSS + 小怪 + 陷阱）
- 地下城/遗迹生成

### 3.3 事件队列模式 (Incident Queue Pattern)

#### 模式特征
- **异步事件执行**：将事件加入队列而非立即执行
- **优先级隐含**：队列按添加顺序执行
- **参数预准备**：IncidentParms 提前计算（如点数、派系等）

#### 代码示例

```csharp
// 源代码：MiliraGameComponent_OverallControl.cs:179-191
public void SendMilianClusterRaid()
{
    Map currentMap = Find.CurrentMap;
    if (currentMap != null)
    {
        // 第1步：准备参数
        IncidentParms parms = new IncidentParms
        {
            target = currentMap,
            points = StorytellerUtility.DefaultThreatPointsNow(currentMap)
        };

        // 第2步：加入队列（而非立即执行）
        Find.Storyteller.incidentQueue.Add(
            MiliraDefOf.Milira_MilianCluster,      // 事件定义
            Find.TickManager.TicksGame,             // 触发时间
            parms                                   // 参数
        );
    }
}
```

#### 关键优势
✓ **减少帧率波动**：事件分散执行而非集中执行
✓ **故事讲述者控制**：故事讲述者决定实际触发时机
✓ **易于取消/修改**：事件在执行前仍可修改

### 3.4 Def 驱动模式 (Def-Driven Pattern)

#### 模式特征
- **配置化实现**：大量行为由 XML Def 定义而非硬编码
- **DefOf 强类型引用**：编译时安全

#### 关键 Def 配置

```xml
<!-- Incidents_Map_Threats.xml:61-79 -->
<IncidentDef>
  <defName>Milira_MilianCluster</defName>
  <label>Milian Cluster</label>
  <workerClass>Milira.IncidentWorker_MiliraCluster</workerClass>

  <!-- 触发条件 -->
  <earliestDay>90</earliestDay>            <!-- 最早第 90 天 -->
  <minRefireDays>27</minRefireDays>        <!-- 触发间隔 27 天 -->
  <minThreatPoints>400</minThreatPoints>   <!-- 最小威胁值 400 -->

  <!-- 难度参数 -->
  <baseChance>1.9</baseChance>             <!-- 基础触发概率 -->
  <pointsScaleable>true</pointsScaleable>  <!-- 根据点数缩放 -->

  <!-- 故事相关 -->
  <category>ThreatBig</category>
  <letterLabel>Milian Cluster</letterLabel>
  <letterText>...</letterText>
</IncidentDef>
```

#### Def 的可调参数

| 参数 | 作用 | 调优方向 |
|------|------|---------|
| `baseChance` | 触发概率权重 | ↑ 增加频率，↓ 降低频率 |
| `minRefireDays` | 触发间隔 | ↑ 降低频率，↓ 增加频率 |
| `earliestDay` | 首次触发时间 | ↑ 延迟触发，↓ 早期触发 |
| `minThreatPoints` | 触发最低威胁值 | ↑ 需要更强威胁，↓ 更早触发 |

### 5.1 转向友好 (Turn to Friend) 机制的深层设计

#### 当前实现的巧妙之处

米莱拉不是通过"敌对度"而是通过"派系列表"来控制敌对状态：

```csharp
// 核心逻辑：修改派系的 permanentEnemyToEveryoneExcept 列表
if (turnToFriend)
{
    MiliraDefOf.Milira_Faction.permanentEnemyToEveryoneExcept.Add(
        Faction.OfPlayer.def
    );
}
```

**优势**：
✓ 一个布尔值 (turnToFriend) 即可完全改变派系关系
✓ 所有敌对行为自动停止（因为派系不再敌对）
✓ 玩家可通过故事事件或代码触发转向

**应用扩展**：
- 敌方派系可通过完成任务转向友好
- 中立派系可根据玩家选择转向敌对/友好
- 联盟系统可基于这个机制实现

## 总结与思考

### 关键设计洞察

1. **资源积累模式**
   - 威胁点数作为"内部资源"驱动事件触发
   - 自动衰减防止系统失控
   - 给玩家准备时间

2. **多层级过滤**
   - 周期检查 → 资源阈值 → 冷却时间 → 概率 → 其他条件
   - 多层保护防止频繁触发
   - 每层可独立调整

3. **参数化生成**
   - 集群规模由多条曲线决定
   - 曲线参数可轻松调整难度
   - 避免硬编码

4. **派系关系驱动**
   - 通过修改派系关系停止袭击
   - 故事驱动的游戏设计
   - 玩家有解决方案（转向友好）
