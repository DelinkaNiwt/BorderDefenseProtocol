## 1. 编译环境配置

### 1.1 前置条件
- Visual Studio 2019 或更新版本（或 VS Code + dotnet CLI）
- .NET Framework 4.7.2 或更高
- RimWorld 游戏本体已安装
- RimWorld 源代码/Assembly 可访问

### 1.2 RimWorld 路径配置

**游戏安装路径**：
```
C:\NiwtGames\Steam\steamapps\common\RimWorld
```

**关键文件位置**：
```
RimWorld/Managed/
├── Assembly-CSharp.dll      (游戏核心代码)
├── UnityEngine.dll          (Unity引擎)
└── ... 其他依赖
```

### 1.3 项目配置文件 (.csproj)

```xml
<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <TargetFramework>net472</TargetFramework>
    <AssemblyName>MyMod</AssemblyName>
    <RootNamespace>MyMod</RootNamespace>
  </PropertyGroup>

  <ItemGroup>
    <Reference Include="Assembly-CSharp">
      <HintPath>C:\NiwtGames\Steam\steamapps\common\RimWorld\Managed\Assembly-CSharp.dll</HintPath>
    </Reference>
    <Reference Include="UnityEngine">
      <HintPath>C:\NiwtGames\Steam\steamapps\common\RimWorld\Managed\UnityEngine.dll</HintPath>
    </Reference>
  </ItemGroup>
</Project>
```

## 2. 编译步骤

### 2.1 从命令行编译

```bash
# 进入项目目录
cd C:\NiwtDatas\Projects\RimworldModStudio

# 清理旧构建
dotnet clean

# 还原依赖
dotnet restore

# 编译（Debug模式）
dotnet build -c Debug

# 编译（Release模式）
dotnet build -c Release
```

### 2.2 输出目录

```
bin/
├── Debug/
│   └── net472/
│       ├── MyMod.dll       (生成的DLL)
│       └── ... 其他文件
└── Release/
    └── net472/
        └── MyMod.dll
```

### 2.3 部署到游戏

```bash
# 复制DLL到模组文件夹
copy bin\Release\net472\MyMod.dll "C:\NiwtGames\Steam\steamapps\common\RimWorld\Mods\MyMod\Assemblies\"
```

**或使用批处理脚本**：
```batch
@echo off
set GAME_PATH=C:\NiwtGames\Steam\steamapps\common\RimWorld
set MOD_NAME=MyMod
set SOURCE=bin\Release\net472\%MOD_NAME%.dll
set DEST=%GAME_PATH%\Mods\%MOD_NAME%\Assemblies\

echo Copying %SOURCE% to %DEST%...
copy %SOURCE% %DEST%
echo Done!
pause
```

## 3. 常见编译错误及解决方案

### 3.1 找不到Assembly-CSharp.dll

**错误消息**：
```
error CS0006: Metadata file 'Assembly-CSharp.dll' could not be found
```

**原因**：引用路径不正确或文件不存在

**解决方案**：
1. 确认游戏安装路径正确：`C:\NiwtGames\Steam\steamapps\common\RimWorld`
2. 确认 `Managed` 文件夹存在：`RimWorld\Managed\`
3. 更新 `.csproj` 中的 HintPath 路径
4. 重新加载项目

### 3.2 类型不匹配错误

**错误消息**：
```
error CS0029: Cannot implicitly convert type 'X' to 'Y'
```

**原因**：类型转换或赋值错误

**解决方案**：
```csharp
// ✗ 错误
Pawn p = thing; // thing 可能不是Pawn

// ✓ 正确
if (thing is Pawn p)
{
    // 使用p
}
```

### 3.3 方法/属性不存在

**错误消息**：
```
error CS0117: 'X' does not contain a definition for 'Y'
```

**原因**：
1. 拼写错误
2. 方法/属性在当前RimWorld版本不存在
3. 命名空间导入错误

**解决方案**：
1. 检查拼写（区分大小写）
2. 查阅 API 参考文档
3. 添加必要的 `using` 语句
   ```csharp
   using RimWorld;
   using Verse;
   ```

### 3.4 命名空间冲突

**错误消息**：
```
error CS0104: 'X' is an ambiguous reference between 'A' and 'B'
```

**解决方案**：
```csharp
// 指定完整命名空间
using RimWorld;  // 选择一个
// using OtherNamespace; // 注释掉冲突的

// 或使用全路径
var x = RimWorld.ClassName.Method();
```

### 3.5 DLL 版本不匹配

**错误消息**：
```
error: Assembly version mismatch or runtime error
```

**解决方案**：
1. 检查 RimWorld 版本：`About.xml` 中的 `<supportedVersions>`
2. 确认项目目标框架：`.csproj` 中的 `<TargetFramework>net472</TargetFramework>`
3. 重新生成，确保使用最新的 Assembly-CSharp.dll

## 4. 运行时错误及处理

### 4.1 找不到 Comp 类

**症状**：游戏加载模组时报错，找不到自定义Comp

**原因**：XML 中的 `<compClass>` 与实际C#类路径不匹配

**解决方案**：
```xml
<!-- 检查 compClass 是否正确 -->
<li Class="MyMod.Comp_MyFeature">  <!-- 应该是 命名空间.类名 -->
```

```csharp
// 确保C#类在正确的命名空间
namespace MyMod
{
    public class Comp_MyFeature : ThingComp
    {
    }
}
```

### 4.2 数据序列化错误

**症状**：保存游戏后读档时报错

**原因**：忘记实现或错误实现 `ExposeData()`

**解决方案**：
```csharp
public override void ExposeData()
{
    base.ExposeData();  // 必须调用base
    // 保存所有关键字段
    Scribe_Values.Look(ref _myValue, "myValue", 0);
}
```

### 4.3 空引用异常 (NullReferenceException)

**症状**：运行时崩溃，显示NPE

**原因**：引用为null但被直接使用

**解决方案**：
```csharp
// ✗ 危险
Pawn parent = this.parent as Pawn;
parent.needs.food.CurLevel -= 1;  // 可能NPE

// ✓ 安全
Pawn parent = this.parent as Pawn;
if (parent != null && parent.needs != null)
{
    parent.needs.food.CurLevel -= 1;
}
```

## 6. 部署检查清单

编译完成后，在部署前确认：

- [ ] 编译无错误（0个错误，可有警告）
- [ ] DLL 文件生成在 `bin/Release/` 中
- [ ] 复制 DLL 到模组目录：`Mods/ModName/Assemblies/`
- [ ] `About.xml` 中的 `<assemblyVersion>` 与DLL版本匹配
- [ ] 所有 XML Def 文件都在正确位置
- [ ] 纹理文件都在 `Textures/` 目录
- [ ] 没有硬编码的绝对路径
- [ ] 模组加载顺序正确（dependencies）

## 7. 完整编译-部署脚本

创建 `deploy.bat` 文件：

```batch
@echo off
setlocal enabledelayedexpansion

REM 配置
set PROJECT_DIR=C:\NiwtDatas\Projects\RimworldModStudio
set MOD_NAME=MyMod
set GAME_PATH=C:\NiwtGames\Steam\steamapps\common\RimWorld
set MOD_DEST=%GAME_PATH%\Mods\%MOD_NAME%\Assemblies

REM 清理并编译
echo ========== Building ==========
cd /d %PROJECT_DIR%
dotnet clean
dotnet build -c Release

REM 检查编译结果
if %ERRORLEVEL% neq 0 (
    echo Build failed!
    pause
    exit /b 1
)

REM 部署
echo ========== Deploying ==========
if not exist "%MOD_DEST%" mkdir "%MOD_DEST%"
copy "bin\Release\net472\%MOD_NAME%.dll" "%MOD_DEST%"

if %ERRORLEVEL% equ 0 (
    echo Deploy successful!
    echo DLL copied to: %MOD_DEST%
) else (
    echo Deploy failed!
)

pause
```

**使用**：
```bash
# 在命令行执行
deploy.bat
```
