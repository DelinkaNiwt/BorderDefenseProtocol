## 2. 架构设计

### 2.1 整体架构图
```
┌─────────────────────────────────────┐
│        UI/Gizmo 交互层              │ (负责展示和用户交互)
└────────────┬────────────────────────┘
             │
┌────────────▼────────────────────────┐
│      核心系统 (业务逻辑层)          │ (负责功能实现)
│  ├─ 模块A（功能A的核心逻辑）         │
│  ├─ 模块B（功能B的核心逻辑）         │
│  └─ 模块C（通用工具函数库）          │
└────────────┬────────────────────────┘
             │
┌────────────▼────────────────────────┐
│     数据存储层（Comp + XML）        │ (负责数据定义和序列化)
│  ├─ ThingDef XML 定义                │
│  ├─ CompProperties 定义              │
│  └─ 数据序列化(ExposeData)           │
└─────────────────────────────────────┘
```

### 2.2 关键设计决策
- **Comp vs 直接继承**：采用Comp模式，通过XML配置组合功能
- **数据序列化**：使用 Scribe API 保存关键数据
- **UI交互**：通过 Gizmo 提供按钮和命令

## 5. 文件组织

### 5.1 目录结构
```
ModName/
├── About/
│   ├── About.xml
│   └── Preview.png
├── Defs/
│   ├── ThingDefs/
│   ├── HediffDefs/ (如需要)
│   └── ...其他Def
├── Source/
│   ├── ModuleA/
│   ├── ModuleB/
│   ├── Utils/
│   └── UI/
├── Textures/
└── README.md
```

### 5.2 命名约定
- **C# 类名**：PascalCase (如 Comp_MyFeature, Utility_Helper)
- **C# 方法名**：PascalCase
- **C# 字段/属性**：camelCase (私有) 或 PascalCase (公有)
- **XML 文件**：蛇形命名 (如 thing_defs.xml)
- **defName**：蛇形或PascalCase（保持一致）

## 6. 功能流程设计

### 6.1 （功能名）工作流
```
用户操作 → UI触发 → 核心逻辑处理 → 数据更新 → 保存到磁盘
```

**详细步骤**：
1. 用户点击Gizmo按钮
2. 触发 Action 回调函数
3. 调用 ModuleA 的核心方法
4. 更新 Comp 内的数据
5. ExposeData 自动保存

## 7. 已确定的技术选择

| 选择项 | 决策 | 理由 |
|--------|------|------|
| 框架模式 | Comp 组件模式 | 灵活、易于扩展 |
| 数据持久化 | Scribe API | RimWorld 标准做法 |
| UI 控件 | Gizmo | 游戏内原生UI系统 |
