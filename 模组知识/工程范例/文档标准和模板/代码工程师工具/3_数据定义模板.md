# 数据定义模板

**摘要**：提供XML配置文件的模板和字段定义，代码工程师可直接填充而无需理解数据设计。
**版本**：v1.0
**修改时间**：2026-01-04
**关键词**：XML定义, ThingDef, CompProperties, 配置文件, 数据结构
**标签**：[定稿]

---

## 设计理由
- **为什么需要**：XML配置涉及具体的字段类型、有效范围、默认值等细节，代码工程师需要参考模板而不是反复询问
- **如何使用**：按功能类型选择模板，填充各个字段值（字段说明已标注），保存到正确的XML文件

## 1. ThingDef 基础模板

**用途**：定义物品/建筑的基本属性
**保存位置**：`Defs/ThingDefs/`
**文件名示例**：`thing_defs_buildings.xml`

### 1.1 完整示例结构
```xml
<?xml version="1.0" encoding="utf-8" ?>
<Defs>

  <ThingDef ParentName="BuildingBase">
    <!-- 基本信息 -->
    <defName>MyMod_BuildingName</defName>
    <label>建筑名称</label>
    <description>建筑的详细描述</description>

    <!-- 外观 -->
    <thingClass>Verse.Building</thingClass>
    <graphicData>
      <texPath>Things/Buildings/MyMod_Building</texPath>
      <graphicClass>Graphic_Single</graphicClass>
      <drawSize>(2,2)</drawSize>
    </graphicData>

    <!-- 功能组件 -->
    <comps>
      <!-- 组件1：我的自定义功能 -->
      <li Class="MyMod.Comp_MyFeature">
        <字段1>值1</字段1>
        <字段2>值2</字段2>
      </li>
      <!-- 组件2：功率消耗（如需要） -->
      <li Class="CompProperties_Power">
        <compClass>CompPowerTrader</compClass>
        <basePowerConsumption>100</basePowerConsumption>
      </li>
    </comps>

    <!-- 建筑参数 -->
    <designationCategory>Production</designationCategory>
    <size>(2,2)</size>
    <cost>
      <Steel>100</Steel>
      <Wood>50</Wood>
    </cost>
    <researchPrerequisites>
      <li>MyMod_BasicTech</li>
    </researchPrerequisites>
  </ThingDef>

</Defs>
```

## 2. CompProperties 模板

**用途**：定义Comp的配置和参数
**保存位置**：在 ThingDef 的 `<comps>` 内

### 2.1 自定义 Comp 模板

```xml
<li Class="MyMod.CompProperties_MyFeature">
  <!-- 必需字段 -->
  <compClass>MyMod.Comp_MyFeature</compClass>

  <!-- 配置参数（按需添加） -->
  <enabledByDefault>true</enabledByDefault>
  <maxValue>100</maxValue>
  <defaultValue>50</defaultValue>
  <tickInterval>60</tickInterval>

  <!-- 文本内容 -->
  <label>功能标签</label>
  <description>功能描述</description>
</li>
```

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| compClass | string | ✓ | C#类的完整路径（命名空间.类名） |
| enabledByDefault | bool | ✗ | 是否默认启用（默认true） |
| maxValue | float | ✗ | 最大值（范围值时使用） |
| defaultValue | float | ✗ | 默认值 |
| tickInterval | int | ✗ | Tick间隔（0=每Tick, 60=每秒, 3600=每分钟） |

## 3. 常用字段定义参考表

### 3.1 基础数据类型

| 字段名 | 类型 | 范围 | 说明 |
|--------|------|------|------|
| label | string | - | 显示名称 |
| description | string | - | 详细描述 |
| defName | string | - | 唯一ID（必须英文） |
| enabled | bool | true/false | 是否启用 |
| count | int | 0-9999 | 数量 |
| value | float | 0.0-1.0 | 百分比或概率 |
| chance | float | 0.0-1.0 | 概率（0=0%, 1=100%） |
| offset | IntVec3 | (x,y,z) | 坐标偏移 |
| color | Color | (R,G,B,A) | 颜色（0-1范围） |

### 3.2 资源成本定义

```xml
<cost>
  <Steel>100</Steel>
  <Wood>50</Wood>
  <ComponentIndustrial>5</ComponentIndustrial>
</cost>
```

### 3.3 研究前置定义

```xml
<researchPrerequisites>
  <li>MyMod_BasicTech</li>
  <li>MyMod_AdvancedTech</li>
</researchPrerequisites>
```

## 4. 字段验证规则

| 字段 | 验证规则 | 示例 |
|------|---------|------|
| defName | 必须唯一，英文字母+下划线+数字 | MyMod_Building1 |
| label | 长度 1-100字符 | "我的建筑" |
| float 值 | 0.0 - 1.0 (概率) 或特定范围 | 0.5, 0.75 |
| int 值 | 必须是整数 | 100, 60 |
| Color | RGBA 各分量 0-1 | (1, 0.5, 0.2, 1) |
| 类路径 | 命名空间.类名 | MyMod.Comp_MyFeature |

## 6. 常见问题

**Q: CompProperties 和 Comp 是什么关系？**
A: CompProperties 是XML配置(数据)，Comp 是C#类(逻辑)。XML中定义的字段会被加载到Comp对象。

**Q: 如何添加新的配置字段？**
A: 在C#的Comp类中定义public字段，XML中用相同名称即可自动加载。

**Q: 字段顺序重要吗？**
A: XML中字段顺序不重要，但建议按逻辑分组以提高可读性。
