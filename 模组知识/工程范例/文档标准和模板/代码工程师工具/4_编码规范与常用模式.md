## 设计理由
- **为什么需要**：统一的编码规范和模式能减少代码审查时的讨论，提高编码效率

## 1. 命名约定

### 1.1 C# 标识符命名

| 对象类型 | 约定 | 示例 | 备注 |
|---------|------|------|------|
| 类 | PascalCase | `Comp_MyFeature`, `Utility_Helper` | 包含前缀（Comp/Job/Building等） |
| 接口 | PascalCase + I前缀 | `IMyInterface` | 始终以I开头 |
| 方法 | PascalCase | `GetCurrentValue()`, `ProcessData()` | 动词开头 |
| 公共属性 | PascalCase | `MyProperty { get; set; }` | 首字母大写 |
| 公共字段 | PascalCase | `public int MyField;` | 避免使用，优先用属性 |
| 私有字段 | camelCase + 下划线 | `private int _myValue;` | 前缀下划线 |
| 本地变量 | camelCase | `int myCounter = 0;` | 小写开头 |
| 常量 | UPPER_SNAKE_CASE | `const int MAX_TRIES = 10;` | 全大写 |
| XML元素 | PascalCase | `<defName>`, `<compClass>` | 保持RimWorld风格 |
| defName | snake_case 或 PascalCase_snake | `my_mod_building`, `MyMod_Building1` | 保持一致 |

### 1.2 命名前缀规范

| 类型 | 前缀 | 示例 |
|------|------|------|
| Thing的Comp | Comp_ | `Comp_Power`, `Comp_MyFeature` |
| Job | JobDriver_ | `JobDriver_Mine`, `JobDriver_Haul` |
| 工具/辅助类 | Utility_ 或 Utils_ | `Utility_Math`, `Utils_Validation` |
| 常数/配置类 | Const_ 或 Config_ | `Const_GameRules`, `Config_Difficulty` |
| 扩展方法 | Ext_ | `Ext_PawnExtensions` |

## 2. 代码结构规范

### 2.1 类的成员组织顺序

```csharp
public class Comp_MyFeature : ThingComp
{
    // 1. 字段（常量在上，然后私有字段，最后公开字段）
    public const int DEFAULT_VALUE = 100;
    private int _internalCounter;
    public int PublicValue;

    // 2. 属性
    public int CurrentLevel { get; private set; }

    // 3. 构造方法
    public Comp_MyFeature() { }

    // 4. 生命周期方法（PostSpawnSetup, PostDeSpawn, CompTick等）
    public override void PostSpawnSetup(bool respawningAfterLoad)
    {
        base.PostSpawnSetup(respawningAfterLoad);
    }

    // 5. 公开方法（公开API）
    public void DoSomething() { }

    // 6. 私有方法（内部实现）
    private void InternalMethod() { }

    // 7. 序列化方法（ExposeData）
    public override void ExposeData()
    {
        base.ExposeData();
        Scribe_Values.Look(ref _internalCounter, "internalCounter", 0);
    }
}
```

### 2.2 方法大小
- **目标**：方法不超过 50 行
- **超过**：考虑拆分为多个小方法

### 2.3 缩进和格式
```csharp
// 使用 4 空格缩进
if (condition)
{
    DoSomething();
}

// 单行 if 可简写
if (condition) return;

// Lambda 表达式
var result = list.Where(x => x.IsValid).ToList();
```

## 3. 常用编码模式

### 3.1 模式：Comp 的基础实现

```csharp
public class Comp_MyFeature : ThingComp
{
    // 配置字段（从XML加载）
    public int maxValue = 100;
    public int tickInterval = 60;

    // 内部状态
    private int _currentValue = 0;
    private int _tickCounter = 0;

    // Comp 特有方法
    public override void PostSpawnSetup(bool respawningAfterLoad)
    {
        base.PostSpawnSetup(respawningAfterLoad);
        // 初始化
        _currentValue = maxValue / 2;
    }

    public override void CompTick()
    {
        base.CompTick();

        _tickCounter++;
        if (_tickCounter >= tickInterval)
        {
            _tickCounter = 0;
            UpdateValue();
        }
    }

    private void UpdateValue()
    {
        _currentValue = Mathf.Max(0, _currentValue - 1);
    }

    public override void ExposeData()
    {
        base.ExposeData();
        Scribe_Values.Look(ref _currentValue, "currentValue", 0);
        Scribe_Values.Look(ref _tickCounter, "tickCounter", 0);
    }
}
```

### 3.2 模式：Gizmo 返回

```csharp
public override IEnumerable<Gizmo> CompGetGizmosExtra()
{
    foreach (var g in base.CompGetGizmosExtra())
        yield return g;

    // 单个Gizmo
    yield return new Command_Action()
    {
        defaultLabel = "我的功能",
        defaultDesc = "详细描述",
        icon = ContentFinder<Texture2D>.Get("UI/Commands/MyIcon"),
        action = () => ExecuteAction()
    };

    // 带快捷键
    yield return new Command_Toggle()
    {
        defaultLabel = "切换功能",
        icon = ContentFinder<Texture2D>.Get("UI/Commands/Toggle"),
        isActive = () => _isEnabled,
        toggleAction = () => _isEnabled = !_isEnabled
    };
}

private void ExecuteAction()
{
    Messages.Message("执行了某个操作", MessageTypeDefOf.PositiveEvent);
}
```

### 3.3 模式：安全的 Pawn 引用获取

```csharp
private Pawn GetPawnParent()
{
    if (this.parent is Pawn pawn)
        return pawn;
    return null;
}

public override void CompTick()
{
    base.CompTick();

    var pawn = GetPawnParent();
    if (pawn != null && pawn.Spawned)
    {
        // 安全地使用 pawn
        pawn.needs.food.CurLevel -= 0.1f;
    }
}
```

### 3.4 模式：集合遍历

```csharp
// ✓ 推荐：使用 foreach
foreach (var item in list)
{
    item.DoSomething();
}

// ✓ 推荐：使用 LINQ
var filtered = list.Where(x => x.IsValid).ToList();

// ✗ 避免：索引遍历（除非需要修改集合）
// for (int i = 0; i < list.Count; i++) { }
```

## 4. 反模式（避免）

| 反模式 | 问题 | 正确做法 |
|--------|------|---------|
| `this.parent as Pawn` 后直接使用 | NPE异常 | 先判空再使用 |
| 在 `Tick` 中做重操作 | 卡帧 | 使用 `tickInterval` |
| 忘记序列化关键数据 | 读档后数据丢失 | 实现 `ExposeData` |
| 硬编码数值 | 难以调整参数 | 使用XML配置或常量 |
| 过度耦合 | 代码难以扩展 | 依赖注入或接口 |

## 5. 性能建议

### 5.1 常见优化

| 场景 | 优化方法 |
|------|---------|
| 每帧重复计算 | 缓存结果，使用tickInterval延迟更新 |
| 频繁创建对象 | 使用对象池或静态缓存 |
| 遍历大集合 | 使用 LINQ 过滤或缓存 |
| 调用外部API | 只在需要时调用，缓存结果 |

### 5.2 RimWorld 特定优化

```csharp
// ✓ 推荐：Tick间隔更新
public override void CompTick()
{
    if (Find.TickManager.TicksGame % 60 == 0) // 每秒一次
    {
        ExpensiveOperation();
    }
}

// ✓ 推荐：条件检查
public override void CompTick()
{
    if (!this.parent.Spawned) return; // 快速返回

    // 其他逻辑
}
```

## 6. 注释规范

### 6.1 何时添加注释

| 情况 | 做法 |
|------|------|
| 复杂算法 | ✓ 添加注释 |
| 业务逻辑 | ✓ 简短注释说明为什么 |
| 显而易见的代码 | ✗ 不需要注释 |
| 公开API | ✓ XML 文档注释 |

### 6.2 注释示例

```csharp
/// <summary>
/// 计算当前的能量消耗
/// </summary>
/// <returns>能量消耗值（单位：Wd）</returns>
public float GetEnergyConsumption()
{
    // 基础消耗 + 负载消耗
    return BASE_CONSUMPTION + (_currentLoad * LOAD_FACTOR);
}
```
