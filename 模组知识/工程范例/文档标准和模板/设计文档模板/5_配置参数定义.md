## 参数验证规则

代码工程师应该在哪些地方验证参数？

### 加载时验证

```csharp
public override void PostExposeData()
{
    base.PostExposeData();

    // 验证参数范围
    param1_A = Mathf.Clamp(param1_A, 0, 100);
    param2_A = Mathf.Max(0, param2_A);

    // 验证参数一致性
    if (param3_A && param1_A < 50) {
        Log.Warning("param3_A启用但param1_A太低，可能导致预期外的行为");
    }
}
```

### 运行时验证

```csharp
public void SetParam1_A(int value)
{
    // 修改前验证
    if (value < 0 || value > 100) {
        Log.Error("param1_A值无效：" + value);
        return;
    }

    param1_A = value;

    // 修改后触发更新
    InvalidateCache();
}
```

## 与平衡的关系

参数应该让游戏平衡师能调整游戏难度和玩法。

### 平衡师需要调整的参数

```
困难度设置
  低：param1_A=75, param2_A=0.3
  中：param1_A=50, param2_A=0.5
  高：param1_A=25, param2_A=0.7
```

**代码工程师需要知道**：
- 参数应该对玩家体验有**显著的影响**
- 太小的参数变化（比如差异<5%）不值得添加配置
- 参数名应该易于理解（避免内部变量名）

## 常见的参数模式

### 模式1：百分比参数

```
用途：调整某个数值的倍数
范围：0.0 - 2.0（0% - 200%）
默认值：1.0（100%，即正常）

XML：<speedMultiplier>1.0</speedMultiplier>
应用：实际速度 = 基础速度 * speedMultiplier
```

### 模式2：等级参数

```
用途：离散的选择（不是连续的）
范围：1-5（5个等级）
默认值：3（中等）

XML：<difficultyLevel>3</difficultyLevel>
应用：switch (difficultyLevel) { case 1: ...; case 5: ...; }
```

### 模式3：阈值参数

```
用途：控制触发条件
范围：0-100（百分比）
默认值：50

XML：<triggerThreshold>50</triggerThreshold>
应用：if (currentValue > triggerThreshold) { Trigger(); }
```
