## 交互场景

针对不同的使用场景，描述各模块怎么协作。

### 场景1：（待填充，如"玩家启动功能"）

**触发条件**：（待填充）
```
玩家点击[启动功能]按钮
```

**参与模块**：UI → ModuleA → ModuleB

**详细流程**：

```
时间轴：

[T=0] 用户点击按钮
     ↓
[T=1] UI → ModuleA
     调用：ModuleA.StartFeature()
     参数：{ ... }
     ↓
[T=2] ModuleA计算
     读取：ModuleA.currentValue
     调用：ModuleB.GetOptimalStrategy()
     返回：strategy对象
     ↓
[T=3] ModuleB处理
     读取：ModuleB.configuration
     计算：生成方案
     返回：结果
     ↓
[T=4] ModuleA接收结果
     更新：ModuleA.strategy = 结果
     ↓
[T=5] 返回UI
     返回值：{ success: true, message: "..." }
     ↓
[T=6] UI更新
     显示：结果窗口
     ↓
完成
```

**数据流**：

| 步骤 | 源→目标 | 数据 | 类型 |
|------|---------|------|------|
| 1 | UI → ModuleA | 用户输入 | 参数对象 |
| 2 | ModuleA → ModuleB | 需要处理的数据 | 数据对象 |
| 3 | ModuleB → ModuleA | 处理结果 | 结果对象 |
| 4 | ModuleA → UI | 最终结果 | 响应对象 |

**异常处理**：

| 异常 | 处理方式 |
|------|---------|
| 参数无效 | ModuleA.StartFeature()返回error |
| ModuleB超时 | 使用默认策略 |
| UI未更新 | Log.Warning并继续 |

## 模块通信协议

### 模块间通信方式

| 通信方式 | 用途 | 例子 |
|---------|------|------|
| **直接调用** | 同步操作，需要立即返回 | ModuleA.GetValue() |
| **事件/回调** | 异步通知，模块A发生变化通知模块B | ModuleA发送事件 → ModuleB监听 |
| **查询接口** | 模块B提供查询接口，模块A按需查询 | ModuleA.Query(条件) |

### 模块A → 模块B 的调用接口

```csharp
// ModuleA需要调用ModuleB的方法
public interface IModuleB_Interface
{
    // 同步调用：计算
    StrategyResult GetOptimalStrategy(InputData input);

    // 查询：读取状态
    int GetCurrentLoad();

    // 命令：修改状态
    void SetConfiguration(ConfigData config);

    // 事件：异步通知（可选）
    event Action<ResultData> OnResultChanged;
}
```

**代码工程师需要**：
- 在ModuleB中实现这个接口
- 在ModuleA中调用这些方法，而不是直接访问ModuleB的字段

## 并发和竞态

多个地方同时修改同一个数据时怎么办？

### 并发场景

| 场景 | 参与方 | 风险 | 解决方案 |
|------|--------|------|---------|
| **两个Tick同时更新** | ModuleA, ModuleB | 数据不一致 | 使用互斥锁或队列化操作 |
| **UI修改 + 游戏逻辑修改** | UI, Tick | 冲突 | UI改变发送命令，Tick处理命令 |

### 推荐的并发模型

```
Tick线程（游戏主线程）
  ├─ 读取输入队列
  ├─ 更新游戏逻辑
  ├─ 修改数据（单线程，不竞争）
  └─ 准备下一帧的UI数据

UI线程（输入）
  ├─ 检测用户操作
  ├─ 发送命令到"命令队列"
  └─ 读取上一帧的数据（只读）
```

## 性能考虑

### 高频操作（每Tick调用）

| 操作 | 预期频率 | 优化方案 |
|------|----------|---------|
| **属性1更新** | 60次/秒 | 缓存结果，避免重复计算 |
| **查询ModuleB** | 60次/秒 | 批量查询或异步查询 |

### 低频操作（偶尔调用）

| 操作 | 预期频率 | 优化方案 |
|------|----------|---------|
| **计算全局方案** | 1次/10秒 | 无需优化 |

## 扩展点

如果要添加新功能，怎么与现有模块集成？

### 新模块集成流程

```
新模块ModuleC要添加
  ↓
1. 定义接口（IModuleC_Interface）
  ↓
2. 确定依赖（依赖ModuleA还是ModuleB？）
  ↓
3. 定义与现有模块的交互
  ├─ ModuleA → ModuleC：怎么调用？
  ├─ ModuleC → ModuleA：怎么回调？
  └─ 数据怎么共享？
  ↓
4. 更新这个文档（时序图、数据流、并发模型）
```

## 常见的交互模式

### 模式1：查询-应答

```csharp
// ModuleA查询ModuleB的状态
int value = moduleB.GetValue(key);
if (value > threshold) {
    DoSomething();
}
```

**使用场景**：模块A依赖ModuleB的状态决策

### 模式2：命令-应答

```csharp
// ModuleA命令ModuleB执行操作
bool success = moduleB.Execute(command);
if (success) {
    // 继续处理
}
```

**使用场景**：ModuleB是一个服务提供者

### 模式3：发布-订阅

```csharp
// ModuleB发生变化时，自动通知ModuleA
moduleB.OnStateChanged += ModuleA.HandleStateChange;
```

**使用场景**：ModuleA需要响应ModuleB的变化，但不想主动查询

## 调试和日志

模块之间通信时，怎么调试？

### 关键位置的日志

| 位置 | 日志内容 |
|------|---------|
| 模块间调用前 | "ModuleA调用ModuleB.GetValue()" |
| 调用返回 | "ModuleB返回结果：{result}" |
| 异常处理 | "ModuleB调用失败：{原因}" |

**代码示例**：
```csharp
Log.Message("ModuleA.StartFeature() 调用ModuleB");
var result = moduleB.GetOptimalStrategy(input);
if (result == null) {
    Log.Warning("ModuleB返回空，使用默认值");
} else {
    Log.Message("ModuleB返回结果：" + result);
}
```
