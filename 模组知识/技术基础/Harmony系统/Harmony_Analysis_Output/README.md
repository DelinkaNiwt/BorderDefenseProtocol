## 📌 核心概念速记

### Harmony的三层结构

```
应用层 (HarmonyMod)
  ↓ [初始化、设置、UI集成]
框架层 (HarmonyLib)
  ↓ [补丕处理、属性系统、反射工具]
底层实现 (MonoMod + Mono.Cecil + Iced + 平台库)
  ↓ [IL操作、指令编码、内存管理]
```

### 补丕执行流程

```
前缀补丕们 → 原方法 → 后缀补丕们 → 返回值
  ↓
前缀可以阻止原方法，直接返回自定义结果
```

### 补丕应用的四个阶段

```
信息收集 → 补丕汇总 → 方法重写 → 全局更新
```
