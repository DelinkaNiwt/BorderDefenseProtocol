# 项目文档

此目录存储项目全生命周期的工作流文档，遵循**9步工作流**框架。

## 文件结构

```
docs/
├── README.md                        # 此文件
├── 1_创意方案.md                     # 创意总监的创意方案
├── 2_需求和系统设计.md              # 需求分析和系统设计
├── 3_知识获取清单.md                # 项目所需的知识和参考资源
├── 4_设计文档_验证清单.md           # 最终设计文档和RiMCP验证清单
├── 5_原型代码.md                    # 原型开发记录和实验
├── 6_测试报告.md                    # 测试结果和完善过程
├── 7_正式版.md                      # 正式版本的代码和说明
├── 8_经验总结.md                    # 项目经验沉淀（供知识库提炼）
└── 草稿/                            # 临时草稿和讨论文件
```

## 9步工作流说明

### 阶段1：创意阶段（第1-2步）

**1_创意方案.md**
- 创意总监或设计者的初始想法
- 模组的核心概念和特色
- 目标和受众
- 粗略的实现方向

**2_需求和系统设计.md**
- 需求分析（功能、限制、依赖）
- 系统架构设计
- 数据结构和交互流程
- 技术可行性评估

### 阶段2：知识和设计阶段（第3-4步）

**3_知识获取清单.md**
- 项目所需的知识点
- 参考模组和资源
- 需要验证的API
- 学习资料和文档链接

**4_设计文档_验证清单.md**
- 完整的设计文档
- RiMCP验证清单（API验证）
- 设计决策说明
- 风险评估

### 阶段3：实现阶段（第5-7步）

**5_原型代码.md**
- 脏原型的实现记录
- 快速迭代过程
- 技术细节和关键决定
- 遇到的问题和解决方案

**6_测试报告.md**
- 测试用例和结果
- 发现的问题和Bug
- 完善过程记录
- 性能评测

**7_正式版.md**
- 正式版本的代码
- 完整的功能说明
- 使用手册
- 已知问题和限制

### 阶段4：总结阶段（第8步）

**8_经验总结.md**
- 项目总体回顾
- 技术心得和最佳实践
- 发现的陷阱和解决方案
- 如何改进（供下一个项目参考）
- **此文档用于知识库提炼**

## 使用指南

### 1. 按顺序填充文档

每个阶段完成后，填充对应的文档：
- 创意阶段完成 → 完成1_创意方案.md
- 设计完成 → 完成2_需求和系统设计.md
- 知识获取完成 → 完成3_知识获取清单.md
- 等等...

### 2. 保持文档同步

每个文档记录**当前阶段**的最新进展：
- 不是回顾过去，而是记录现在
- 下一阶段开始时，旧内容会自然被新阶段的文档覆盖

### 3. 临时内容使用草稿/

不确定的想法、讨论、实验放在 `草稿/` 中：
- 不会污染主文档
- 便于后续整理或删除
- 在项目完成后清理

### 4. 模板示例

每个文档可以包含以下部分（根据需要调整）：

```markdown
# [阶段名称]

## 摘要
（简短概述）

## 核心内容
（详细说明）

## 关键决定
（重要决策点）

## 遇到的问题
（问题和解决方案）

## 后续计划
（下一步行动）

## 参考资源
（相关文档和链接）

## 修改历史
- 时间：改动说明 | 修改者
```

## 元信息（每个文档都应包含）

```markdown
## 元信息

**摘要**：本文档的内容概括

**版本**：v1.0

**修改时间**：YYYY-MM-DD

**关键词**：逗号分隔的关键词

**标签**：[定稿]/[草稿]/[待审]
```

## 最佳实践

### 1. 保持简洁
- 关注关键信息
- 避免冗长的细节
- 使用清晰的结构

### 2. 定期更新
- 每个阶段完成后立即更新
- 不要等到项目结束才补充文档

### 3. 链接相关资源
- 在适当的地方链接规范文档
- 参考相关的知识库条目
- 引用参考模组

### 4. 为知识库提炼做准备
- 8_经验总结.md中的内容最有可能被提炼到知识库
- 标注可复用的模式和最佳实践
- 清晰说明应用场景和局限

## 示例：如何填充文档

### 示例：第一次创意方案

```markdown
# 1_创意方案

## 摘要
一个添加激光武器系统的模组。

## 创意概念

- 射击时的视觉效果
```

## 常见问题

**Q: 我应该在第5步（原型）还是第7步（正式版）添加代码？**

A: 两步都添加。第5步是脏原型，第7步是最终版本。

**Q: 如果项目被中止怎么办？**

A: 在8_经验总结.md中记录为什么中止、学到了什么，这些内容对后续项目仍有价值。

**Q: 文档可以跨越多个版本吗？**

A: 可以。在第7步后如果继续迭代，可以重新从第5步开始，或者在同一文档中添加版本控制。
