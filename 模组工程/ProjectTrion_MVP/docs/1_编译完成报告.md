## 解决的编译问题

### 问题 1: FloatMenu API 不兼容
**症状**: `FloatMenu.Add()` 不存在
**根因**: FloatMenu 不支持构造后添加选项
**解决方案**: 改为 `new FloatMenu(List<FloatMenuOption>)` 模式
```csharp
// 错误方式 ❌
var menu = new FloatMenu(new List<FloatMenuOption>());
menu.Add(new FloatMenuOption(...));

// 正确方式 ✅
var options = new List<FloatMenuOption>();
options.Add(new FloatMenuOption(...));
var menu = new FloatMenu(options);
```

### 问题 2: Pawn.modData 不存在
**症状**: `Pawn` 未包含 `modData` 的定义
**根因**: RimWorld 1.6 中无通用 modData 字典
**解决方案**: 改用 Hediff 内部存储 + ExposeData() 持久化
```csharp
// Hediff_TrionGland.cs 中实现：
private TalentGrade takenTalent = TalentGrade.C;
private bool talentInitialized = false;

public override void ExposeData() {
    base.ExposeData();
    Scribe_Values.Look(ref takenTalent, "takenTalent", TalentGrade.C);
    Scribe_Values.Look(ref talentInitialized, "talentInitialized", false);
}
```

### 问题 3: Pawn.components 不存在
**症状**: Pawn 未包含 `components` 的定义
**根因**: 应使用 `Pawn.AllComps` 属性
**解决方案**: 改为 AllComps
```csharp
// 正确方式 ✅
var allComps = pawn.AllComps;
if (!allComps.Contains(comp)) {
    allComps.Add(comp);
}
```

## 关键设计决策

### 1. 天赋初始化延迟化
- **MVP 阶段**: 无全局 Harmony 补丁
- **触发点**: Building_TrionDetector.ScanPawn()
- **存储方式**: Hediff_TrionGland 内部字段 + ExposeData()
- **持久化**: 自动通过存档系统保存/加载

### 2. 组件插入方式
- **时机**: Hediff_TrionGland.PostAdd() 中自动创建
- **实例化**: 手动 new CompTrion() + 设置属性
- **初始化**: 调用 PostSpawnSetup()
- **关键**: 通过 Pawn.AllComps 添加到组件列表

### 3. 浮窗菜单构造
- **模式**: 先构造 List，后通过构造函数传入
- **对应代码**: Building_TrionDetector.OpenScanMenu()
- **符合 RimWorld 1.6 API**
