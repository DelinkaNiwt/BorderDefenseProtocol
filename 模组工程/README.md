# 模组工程

本目录存储RimWorld模组项目的源代码和工作流文档。

## 目录结构

```
模组工程/
├── 活跃项目.yaml              # 当前工作的项目指针（主Agent维护）
├── _模组规范文档/             # 模组开发规范和文档模板
├── _工作流模板/               # 工作流模板（新项目从这里复制）
│   ├── README.md
│   ├── 项目上下文/
│   ├── 智能体工作/            # 7个Agent的工作目录模板
│   ├── 交付物/
│   └── .claude/事件/
├── 示例/                      # 项目结构示例（参考用）
├── RimTrion/                  # 现有项目示例
│   ├── [模组源代码]
│   └── 工作流/                # 项目工作流（从模板初始化）
│       ├── 项目上下文/
│       ├── 智能体工作/
│       ├── 交付物/
│       └── .claude/事件/
└── [其他项目]/               # 其他项目（结构同上）
```

## 工作流指导 (新系统)

### 1. 创建新项目

当用户想要启动新项目时：

```bash
# 主Agent会：
1. 读取 活跃项目.yaml
2. 检查当前没有激活的项目
3. 创建新的项目目录结构
4. 从 _工作流模板/ 复制工作流目录
5. 初始化项目元数据
6. 启动Agent1（头脑风暴官）
```

### 2. 项目工作流

```
创建项目
  ↓
初始化工作流（复制模板）
  ↓
Agent1: 头脑风暴官 (创意探索) → 概念框架
  ↓
Agent2: 需求分析官 (需求整理) → 需求清单
  ↓
Agent3: 架构设计官 (系统设计) → 架构文档
  ↓
Agent4: 原型验证官 (概念验证) → 原型代码
  ↓
Agent5: 开发实现官 (代码实现) → 完整代码
  ↓
Agent6: 调试修复官 (问题修复) → 解决方案
  ↓
Agent7: 更新维护官 (版本维护) → 发布版本
```

### 3. 项目结构

每个项目包含：

```
项目名/
├── [模组源代码和资源]
└── 工作流/                     # 项目工作流
    ├── 项目上下文/
    │   ├── 项目元数据.yaml
    │   ├── 工作流状态.md
    │   └── 版本日志.md
    ├── 智能体工作/
    │   ├── 01_头脑风暴Agent/
    │   ├── 02_需求分析Agent/
    │   ├── ... (03-07)
    │   └── 07_更新维护Agent/
    ├── 交付物/
    │   ├── 模组包/
    │   ├── 发布说明.md
    │   └── 使用指南.md
    └── .claude/事件/           # 项目事件记录
```

## 快速开始

### 启动新项目

```
告诉主Agent（项目主管）："我想开始一个新模组项目"
```

主Agent会：
1. 初始化新项目
2. 启动Agent1进行创意探索
3. 逐步推进7个Agent的协作开发

### 继续现有项目

```
告诉主Agent："继续RimTrion项目" 或 "下一步"
```

主Agent会：
1. 加载项目元数据
2. 读取当前工作流状态
3. 启动下一个Agent
