## ✨ 本版本亮点

### 1. 天赋初始化机制
框架现已支持自动天赋初始化，可根据单位的天赋等级自动计算Capacity。

**支持的天赋等级**：S, A, B, C, D, E (6个等级)

**工作流程**：
```
新建Pawn
  ↓
PostSpawnSetup(false)
  ↓
Strategy.GetInitialTalent() → 获取天赋
  ↓
RecalculateCapacity()     → 根据天赋计算Capacity
  ↓
初始化完成
```

### 2. 完善的新建/读档区分
框架现已正确区分新建和读档两个分支：
- **新建分支**：调用Strategy获取天赋，计算Capacity
- **读档分支**：直接恢复存档数据，无需重新计算

### 3. 灵活的初始化方式
应用层可选择两种天赋初始化方式：
- **方式A（推荐）**：通过TalentCapacityProvider委托提供查表
- **方式B**：在Strategy.GetInitialTalent()中直接设置Capacity

```csharp
    B = 4,
    C = 3,
    D = 2,
    E = 1   // 最低等级
}
```

### 改动2：ILifecycleStrategy 接口扩展
**文件**：`Core/ILifecycleStrategy.cs`

新增方法：
```csharp
TalentGrade? GetInitialTalent(CompTrion comp);
```

**返回值说明**：
- 返回有效的TalentGrade：框架会调用RecalculateCapacity()计算Capacity
- 返回null：框架跳过自动计算，由应用层自己管理Capacity

### 改动3：CompTrion 组件增强
**文件**：`Components/CompTrion.cs`

#### 新增静态委托
```csharp
public static Func<TalentGrade, float> TalentCapacityProvider;
```

#### 改造 PostSpawnSetup() 方法
- 区分新建（respawningAfterLoad=false）和读档（respawningAfterLoad=true）
- 新建时调用GetInitialTalent()获取天赋
- 根据天赋调用RecalculateCapacity()计算Capacity
- 读档时直接恢复数据，无需重算

#### 新增 RecalculateCapacity() 方法
```csharp
private void RecalculateCapacity(TalentGrade talent)
{
    if (TalentCapacityProvider != null)
    {
        _capacity = TalentCapacityProvider(talent);
    }
    else
    {
        _capacity = _props.capacity;  // 使用默认值
    }

    // 验证有效性
    if (_capacity <= 0)
    {
        _capacity = Mathf.Max(100f, _props.capacity);
    }

    ValidateDataConsistency();
}
```

### 步骤2：设置 TalentCapacityProvider 委托

在应用层mod的OnModLoad或其他初始化时机：

```csharp
// 方式A：使用委托（推荐）
CompTrion.TalentCapacityProvider = (talent) =>
{
    return talent switch
    {
        TalentGrade.S => 2000f,
        TalentGrade.A => 1500f,
        TalentGrade.B => 1000f,
        TalentGrade.C => 700f,
        TalentGrade.D => 500f,
        TalentGrade.E => 300f,
        _ => 1000f
    };
};
```

或者

```csharp
// 方式B：在Strategy中直接设置
public TalentGrade? GetInitialTalent(CompTrion comp)
{
    var talent = GenerateTalent();
    comp.Capacity = CalculateCapacityFromTalent(talent);
    return talent;
}
```

## 🤝 技术支持

### 常见问题

**Q: 如何选择两种初始化方式？**
A: 方式A（委托）更简洁，建议首选。方式B用于需要复杂计算的场景。

**Q: Strategy返回null会怎样？**
A: 框架会打印日志提示，由应用层自己设置Capacity。这支持特殊的初始化逻辑。

**Q: 旧mod能直接用v0.7吗？**
A: 可以。旧mod不需要修改即可继续工作。如果想使用新的天赋功能，才需要实现GetInitialTalent()。
