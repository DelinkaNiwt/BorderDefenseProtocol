### 改动4：修改CompTrion.PostSpawnSetup()方法

**文件**：`Components/CompTrion.cs`
**行数**：第144-216行

#### 新建分支流程：
1. 创建Strategy
2. 调用Strategy.GetInitialTalent(this)获取天赋
3. 根据天赋调用RecalculateCapacity()计算Capacity
4. 初始化能量四要素（_reserved=0, _consumed=0）
5. 初始化Mounts列表
6. 初始化快照

#### 读档分支流程：
1. 直接恢复Strategy（通过反射重新创建）
2. Capacity等数据从PostExposeData自动恢复
3. 无需重新计算

**关键代码片段**：
```csharp
// ===== 读档分支 =====
if (respawningAfterLoad)
{
    // 从存档恢复时，直接恢复数据，无需重新计算
    InitializeStrategy();
    if (_strategy == null)
    {
        Log.Error($"CompTrion: 读档时无法为 {parent?.Label} 提供Strategy");
        return;
    }
    return; // 读档完成，跳出
}

// ===== 新建分支 =====
InitializeStrategy();
if (_strategy == null)
{
    Log.Error($"CompTrion: 无法为 {parent?.Label} 创建Strategy");
    return;
}

// 调用Strategy获取初始天赋
TalentGrade? talent = _strategy.GetInitialTalent(this);

// 根据天赋计算Capacity
if (talent.HasValue)
{
    RecalculateCapacity(talent.Value);
}
else
{
    Log.Message($"CompTrion: {parent?.Label} 的Strategy返回null天赋，由应用层自己设置Capacity");
    _capacity = _props.capacity;
}
```

**验证**：✅ 流程正确，注释清晰，区分新建/读档两个分支

### 改动5：新增RecalculateCapacity()方法

**文件**：`Components/CompTrion.cs`
**行数**：第252-280行

```csharp
/// <summary>
/// 根据天赋等级重新计算并设置Capacity。
/// 此方法由PostSpawnSetup在新建时调用，通过Strategy.GetInitialTalent的返回值触发。
///
/// Recalculate Capacity based on talent grade.
/// Called by PostSpawnSetup after Strategy.GetInitialTalent() returns a valid talent.
/// </summary>
private void RecalculateCapacity(TalentGrade talent)
{
    // 使用TalentCapacityProvider委托查表
    if (TalentCapacityProvider != null)
    {
        _capacity = TalentCapacityProvider(talent);
    }
    else
    {
        Log.Error("CompTrion: TalentCapacityProvider未设置，无法计算Capacity");
        _capacity = _props.capacity;  // 使用默认配置值
    }

    // 验证Capacity有效性
    if (_capacity <= 0)
    {
        Log.Error($"CompTrion: 天赋{talent}计算出无效Capacity={_capacity}");
        _capacity = Mathf.Max(100f, _props.capacity);  // 使用最小值或配置值
    }

    ValidateDataConsistency();
}
```

**验证**：✅ 方法逻辑完整，包括：
- 调用委托查表
- 委托未设置时的错误处理
- Capacity有效性验证
- 数据一致性检查

#### 2. 设置TalentCapacityProvider委托
在应用层mod的OnModLoad时：
```csharp
CompTrion.TalentCapacityProvider = (talent) =>
{
    return talent switch
    {
        TalentGrade.S => 2000f,
        TalentGrade.A => 1500f,
        TalentGrade.B => 1000f,
        TalentGrade.C => 700f,
        TalentGrade.D => 500f,
        TalentGrade.E => 300f,
        _ => 1000f
    };
};
```

#### 3. 或使用替代方案：在Strategy中直接设置Capacity
```csharp
public TalentGrade? GetInitialTalent(CompTrion comp)
{
    var talent = GenerateTalent();

    // 应用层直接计算并设置
    comp.Capacity = CalculateCapacityFromTalent(talent);

    return talent;
}
```

## 🎯 架构设计原则

本改动遵循框架设计原则：

✅ **框架的责任**：
- 定义天赋等级枚举
- 提供GetInitialTalent()接口
- 提供RecalculateCapacity()执行能力

- 不生成天赋（应用层负责）
- 不硬编码天赋→容量映射表（应用层负责）
- 不实现Strategy（应用层负责）
