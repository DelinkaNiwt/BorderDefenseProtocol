## 🔧 Strategy注入机制（方案A）

### 实现步骤

```csharp
// 步骤1：在CompTrion初始化时从ThingDef读取
public override void PostSpawnSetup(bool respawningAfterLoad)
{
    base.PostSpawnSetup(respawningAfterLoad);

    // 从comp属性读取Strategy类名
    var props = this.props as CompProperties_Trion;
    if (props != null && !props.strategyClassName.NullOrEmpty())
    {
        var strategyType = GenTypes.GetTypeInAnyAssembly(props.strategyClassName);
        if (strategyType != null)
        {
            this.strategy = (ILifecycleStrategy)Activator.CreateInstance(strategyType, this.parent);
        }
    }
}

// 步骤2：CompProperties_Trion的Def配置
public class CompProperties_Trion : CompProperties
{
    public string strategyClassName;  // 如："ApplicationLayer.Strategy_HumanCombatBody"
    public float capacity = 1000f;
    public float recoveryRate = 2.0f;
    public float leakRate = 0.5f;
    // ...其他属性
}

// 步骤3：应用层在XML中配置
// <comp Class="ProjectTrion.CompTrion">
//   <strategyClassName>ApplicationLayer.Strategy_HumanCombatBody</strategyClassName>
//   <capacity>1000</capacity>
// </comp>
```

### 关键点

- ✅ 解耦最好：应用层只需修改XML，无需编译
- ✅ RimWorld惯例：与其他模组的Comp配置一致
- ✅ 存档兼容：Strategy类名存储在Scribe中，加载时自动反射
- ⚠️ 注意：Strategy类必须有 `public MyStrategy(Thing parent)` 构造函数

public class CompTrion : ThingComp

## 🎯 CompTick执行流程

### 每60Tick的核心流程

```csharp
public override void CompTick()
{
    if (!IsInCombat) return;

    // 步骤1：基础维持消耗
    float baseMaintenance = strategy.GetBaseMaintenance();

    // 步骤2：组件激活消耗
    float mountConsumption = 0f;
    foreach (var mount in Mounts.Where(m => m.IsActive))
    {
        mountConsumption += mount.GetConsumptionRate();
    }

    // 步骤3：泄漏消耗（根据伤口）
    float leak = CalculateLeak();

    // 步骤4：累加并消耗
    float totalConsumption = baseMaintenance + mountConsumption + leak;
    Consume(totalConsumption);

    // 步骤5：检查耗尽
    if (Available <= 0)
    {
        TriggerBailOut();
    }

    // 步骤6：委托给Strategy处理复杂逻辑
    strategy.OnTick(this);
}

private float CalculateLeak()
{
    // 遍历所有Hediff，累加泄漏速率
    float leak = 0f;
    foreach (var hediff in parent.health.hediffSet.hediffs)
    {
        if (hediff is Hediff_Injury injury)
        {
            leak += GetLeakRateForInjury(injury);
        }
    }
    return leak;
}
```

### 性能约束

- ✅ **不可每Tick计算持续消耗**，必须每60Tick批量
- ✅ **不在CompTick中做复杂逻辑**（AI决策等交给Strategy.OnTick）
- ✅ **性能目标**：< 1ms/CompTick（宽松），允许100个单位同时战斗

        }
    }

## 🔄 Bail Out触发机制（最终确认）

### 触发条件（Tier 1：两个条件）

```csharp
public void TriggerBailOut()
{
    // 条件1：Trion供给器官被摧毁
    // 条件2：Trion可用量 ≤ 0

    if (!strategy.CanBailOut(this))
    {
        // 无法Bail Out，直接破裂
        DestroyBody(DestroyReason.ForcedDeactivation);
        return;
    }

    // 获取传送目标
    var target = strategy.GetBailOutTarget(this);

    // 执行传送
    GenTeleport.TeleportTo(parent, target);

    // 摧毁战斗体（被动解除）
    DestroyBody(DestroyReason.PassiveDeactivation);
}
```

### 关键澄清

- ✅ **特殊槽位不会失效**：只有供给器官（心脏）被摧毁时才失效
- ✅ **Bail Out系统本身不会被破坏**：它是特殊槽位的组件，不涉及四肢部位
- ✅ **两个触发条件**：缺一不可（AND关系）

## ⚙️ 数据一致性检查

### 约束检查（4个关键点）

```csharp
private void ValidateDataConsistency()
{
    // 检查1：Capacity必须 > 0
    if (Capacity <= 0)
        Log.Error($"CompTrion: Capacity不能≤0，当前值{Capacity}");

    // 检查2：Reserved不能超过Capacity
    if (Reserved > Capacity)
        Log.Error($"CompTrion: Reserved({Reserved}) > Capacity({Capacity})");

    // 检查3：Consumed不能为负
    if (Consumed < 0)
        Log.Error($"CompTrion: Consumed不能为负，当前值{Consumed}");

    // 检查4：Reserved + Consumed不能超过Capacity
    if (Reserved + Consumed > Capacity)
        Log.Error($"CompTrion: Reserved({Reserved}) + Consumed({Consumed}) > Capacity({Capacity})");
}

// 在关键操作后调用
private void Consume(float amount)
{
    Consumed = Mathf.Clamp(Consumed + amount, 0, Capacity);
    ValidateDataConsistency();
}

private void SetReserved(float amount)
{
    Reserved = Mathf.Clamp(amount, 0, Capacity);
    ValidateDataConsistency();
}
```

## ⚠️ 关键注意事项

### 实现陷阱

1. **Reserved的生命周期**
   - ✅ 战斗体生成时设置：Reserved = 所有组件占用值之和
   - ✅ 组件激活/停用时动态更新：Reserved += 新激活组件占用值
   - ✅ 战斗体解除时清空：Reserved = 0
   - ❌ 错误：Reserved被应用层直接修改

2. **Available的派生性**
   - ✅ 必须是只读属性，不能存档
   - ✅ 每次读取时动态计算
   - ❌ 错误：将Available作为字段存储

3. **Consumed的不可逆性**
   - ✅ 只能增加，不能减少
   - ✅ 被动解除时Reserved转为Consumed
   - ❌ 错误：允许应用层减少Consumed

4. **快照的一致性**
   - ✅ 快照仅包含物理状态（健康、装备、物品）
   - ✅ 不包含心理状态（技能、心情、经验）
   - ✅ 回滚时仅恢复快照内容
   - ❌ 错误：快照包含心理状态或允许选择性回滚

5. **Strategy反射的性能**
   - ✅ 反射仅在PostSpawnSetup时执行一次
   - ✅ 缓存Strategy实例
   - ❌ 错误：每个CompTick都反射

### World Trigger设定特殊性

- **虚拟伤害系统**：物理伤害完全被拦截，不可绕过
- **Trion消耗的不可逆性**：消耗的Trion永远回不来（仅能恢复）
- **快照与回滚**：战斗体一旦摧毁，肉身立即回滚，无过渡
- **心理状态保留**：战斗中的压力/创伤/学习完全保留，这很关键！
