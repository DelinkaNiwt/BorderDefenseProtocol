### 查询ID: 20250123_003 - Hediff系统验证

**查询目标**: 验证TrionSnapshot的快照/恢复机制在RimWorld中的可行性
**RiMCP工具**:
- `get_item("Verse.Pawn_HealthTracker")`

**查询时间**: 2026-01-23 14:20

**关键发现**:

1. **Pawn_HealthTracker API**:
```csharp
public void RemoveAllHediffs() // 清空所有Hediff
public Hediff AddHediff(HediffDef def, BodyPartRecord part, ...)
public void RemoveHediff(Hediff hediff)
```

2. **HediffMaker API**:
```csharp
public static Hediff MakeHediff(HediffDef def, Pawn pawn, BodyPartRecord partRecord)
{
    Hediff obj = (Hediff)Activator.CreateInstance(def.hediffClass);
    obj.def = def;
    obj.pawn = pawn;
    obj.Part = partRecord;
    obj.PostMake(); // 重要: 调用PostMake()初始化
    return obj;
}
```

**可行性结论**: ⚠️ **需要验证**

框架API完整，但需要原型验证GC安全性。

**重要发现**:
- `HediffMaker.MakeHediff()`会自动调用`PostMake()`，这对复杂Hediff很关键
- 快照中只能存HediffDef+Severity数据，不能存Hediff引用（避免GC问题）

## 关键学习和经验

### 1. Gene_Resource的设计哲学

官方代码展示了清晰的责任分离：
- `Gene`类管理数据（cur, max）
- `GeneGizmo`类管理UI（进度条、颜色）
- `ExposeData`管理存档

**启示**: 继承官方基类可以免费获得这些功能。

### 2. Harmony Patch的时序

RimWorld的伤害处理流程：
```
PreApplyDamage → ApplyDamage → PostApplyDamage
```

**PreApplyDamage**: 可以拦截伤害（通过修改`absorbed`）
**PostApplyDamage**: 伤害已应用，hediffSet已更新

**启示**: 护盾格挡在Pre阶段，泄漏计算在Post阶段。

### 3. HediffMaker的PostMake重要性

官方代码明确调用`obj.PostMake()`，这对某些复杂Hediff至关重要。

**启示**: 快照恢复时必须使用`HediffMaker.MakeHediff()`，不能手工构造。

### 4. Component生命周期

MapComponent和GameComponent的生命周期完全由官方框架管理，无需手工注册。

**启示**: 只需继承基类并实现方法，框架会自动发现和调用。
