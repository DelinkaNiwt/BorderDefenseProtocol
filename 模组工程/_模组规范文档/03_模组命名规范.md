# RimWorld 模组命名规范

## 元信息

**摘要**：统一RimWorld模组开发中的命名规范，涵盖C#类名、XML Def名称、文件名和目录名，确保代码可读性和模组互操作性。

**版本**：v1.0
**修改时间**：2026-01-07
**关键词**：命名规范、代码风格、Def命名、文件命名、一致性
**标签**：[定稿]

---

## 概述

统一的命名规范能够：

1. **提升代码质量**：便于阅读、维护和代码审查
2. **避免命名冲突**：使用模组前缀隔离不同模组的定义
3. **提高可互操作性**：使第三方能更容易地集成和扩展
4. **降低学习成本**：新开发者易于理解命名逻辑

## 模组前缀

### 定义

每个模组必须有一个唯一的前缀，用于隔离该模组定义的命名空间。

### 格式

```
[简称/首字母缩写]_[模组名称简化]
```

### 示例

| 模组名称 | 前缀 | 说明 |
|---------|------|------|
| MyAwesomeMod | MAM / MyAM | 首字母缩写或简化 |
| 天工铸造3 | CTA / CeleTech | 英文简称或通用缩写 |
| 米莉拉 | Milira / MR | 英文名或缩写 |
| WeaponFitting | WF / WFitting | 缩写或简化 |

### 选择建议

1. **使用英文**：便于国际兼容
2. **简短**：3-5个字母最佳
3. **易记**：最好与模组名关联
4. **避免冲突**：查询Steam工坊确保唯一性

### 范例

| 使用场景 | 示例 |
|---------|------|
| C#命名空间 | `namespace MyAwesomeMod` |
| XML Def名称 | `<defName>MAM_SomeItem</defName>` |
| C#类名 | `public class MAM_ItemManager` |
| 文件名 | `MAM_Configuration.xml` |

## C# 类名规范

### 基本规则

遵循C# PascalCase命名规范：

```csharp
public class MyClassName { }      // ✓ 正确
public class myClassName { }      // ✗ 错误（应为PascalCase）
public class MY_CLASS_NAME { }    // ✗ 错误（应避免全大写和下划线）
```

### 带模组前缀的类名

**格式**：`[模组前缀]_[功能类型]_[具体类名]`

**示例**：

```csharp
// 组件类
public class MAM_Comp_AdvancedStorage { }
public class MAM_Comp_EnergyCore { }

// 工作类
public class MAM_Job_MineAdvanced { }
public class MAM_JobDriver_AssembleItem { }

// 补丁类
public class MAM_Patch_ThingTick { }

// 工具类
public class MAM_Utility_Calculator { }
public class MAM_Utils_MathHelper { }

// 界面类
public class MAM_Window_ConfigPanel { }
public class MAM_FloatMenu_CustomOptions { }

// 管理器类
public class MAM_Manager_ResourcePool { }

// Harmony补丁
[HarmonyPatch(typeof(Pawn), nameof(Pawn.GetPosture))]
public class MAM_Patch_Pawn_GetPosture { }
```

### 类型前缀约定

| 前缀 | 用途 | 示例 |
|------|------|------|
| `Comp` | Component（组件） | `MAM_Comp_Storage` |
| `Job` | Job定义 | `MAM_Job_Assemble` |
| `JobDriver` | Job驱动 | `MAM_JobDriver_Assemble` |
| `Patch` | Harmony补丁 | `MAM_Patch_Thing_Tick` |
| `Utility` / `Utils` | 工具函数 | `MAM_Utility_Math` |
| `Window` | UI窗口 | `MAM_Window_Config` |
| `Dialog` | 对话框 | `MAM_Dialog_Settings` |
| `FloatMenu` | 浮动菜单 | `MAM_FloatMenu_Options` |
| `Manager` | 管理器 | `MAM_Manager_Resource` |
| `Factory` | 工厂类 | `MAM_Factory_Weapon` |
| `Helper` | 辅助类 | `MAM_Helper_Xml` |

### 命名空间

**格式**：`[模组名].[功能模块]`

**示例**：

```csharp
namespace MyAwesomeMod
{
    namespace Components
    {
        public class MAM_Comp_Storage { }
    }

    namespace Jobs
    {
        public class MAM_JobDriver_Assemble { }
    }

    namespace Patches
    {
        public class MAM_Patch_Thing { }
    }

    namespace Utilities
    {
        public class MAM_Utility_Math { }
    }
}
```

## XML Def 名称规范

### 基本格式

```xml
<defName>[模组前缀]_[分类]_[具体名称]</defName>
```

### 详细说明

| 部分 | 说明 | 示例 |
|------|------|------|
| 模组前缀 | 3-5字母 | `MAM` |
| 分类 | 功能分类（可选） | `Weapon`, `Building`, `Research` |
| 具体名称 | 描述性的名称 | `LaserRifle`, `AdvancedForge` |

### 分类列表（常用）

| DefType | 常见分类 | 示例 |
|---------|----------|------|
| ThingDef | 类型 | `MAM_Weapon_LaserRifle` |
|  | | `MAM_Building_AdvancedForge` |
|  | | `MAM_Item_ComponentA` |
| RecipeDef | 工作场所 | `MAM_Recipe_AtAdvancedForge` |
| ResearchDef | 层级/阶段 | `MAM_Research_Tier1_Basics` |
| HediffDef | 类型 | `MAM_Hediff_Bionic_Arm` |
|  | | `MAM_Hediff_Disease_Virus` |
| ThoughtDef | 心情类型 | `MAM_Thought_Positive_Victory` |
|  | | `MAM_Thought_Negative_Defeat` |
| BuildingDef | 功能 | `MAM_Building_Power_Reactor` |
|  | | `MAM_Building_Storage_Vault` |
| JobDef | 动作 | `MAM_Job_Assemble` |
| WorkGiverDef | 工作类型 | `MAM_WorkGiver_Assembly` |

### 具体示例

#### ThingDef（物品）

```xml
<!-- 武器 -->
<ThingDef>
  <defName>MAM_Weapon_LaserRifle</defName>
  <label>laser rifle</label>
</ThingDef>

<!-- 建筑 -->
<ThingDef>
  <defName>MAM_Building_AdvancedForge</defName>
  <label>advanced forge</label>
</ThingDef>

<!-- 物品 -->
<ThingDef>
  <defName>MAM_Item_ComponentA</defName>
  <label>component A</label>
</ThingDef>

<!-- 弹药 -->
<ThingDef>
  <defName>MAM_Projectile_LaserBlast</defName>
  <label>laser blast</label>
</ThingDef>
```

#### RecipeDef（配方）

```xml
<!-- 按工作台分类 -->
<RecipeDef>
  <defName>MAM_Recipe_CraftAdvancedComponent</defName>
  <label>craft advanced component</label>
  <workSpotDef>CraftingSpot</workSpotDef>
  <workBenchRequirements>
    <li>MAM_Building_AdvancedForge</li>
  </workBenchRequirements>
</RecipeDef>

<!-- 手术配方 -->
<RecipeDef>
  <defName>MAM_Recipe_Surgery_InstallBionicArm</defName>
  <label>install bionic arm</label>
</RecipeDef>
```

#### ResearchDef（科技）

```xml
<!-- 按阶段分类 -->
<ResearchProjectDef>
  <defName>MAM_Research_Tier1_Basics</defName>
  <label>MAM basics</label>
  <baseCost>500</baseCost>
</ResearchProjectDef>

<ResearchProjectDef>
  <defName>MAM_Research_Tier2_Advanced</defName>
  <label>MAM advanced</label>
  <prerequisites>
    <li>MAM_Research_Tier1_Basics</li>
  </prerequisites>
</ResearchProjectDef>
```

#### HediffDef（特性/增强）

```xml
<!-- 仿生增强 -->
<HediffDef>
  <defName>MAM_Hediff_Bionic_Arm</defName>
  <label>bionic arm</label>
  <hediffClass>HediffWithComps</hediffClass>
</HediffDef>

<!-- 疾病 -->
<HediffDef>
  <defName>MAM_Hediff_Disease_AdvancedVirus</defName>
  <label>advanced virus</label>
  <hediffClass>Hediff_Disease</hediffClass>
</HediffDef>

<!-- 特质/职业 -->
<HediffDef>
  <defName>MAM_Hediff_Trait_Engineer</defName>
  <label>engineer</label>
</HediffDef>
```

#### BuildingDef（建筑）

```xml
<!-- 按功能分类 -->
<ThingDef>
  <defName>MAM_Building_Power_Reactor</defName>
  <label>advanced reactor</label>
  <thingClass>Building</thingClass>
</ThingDef>

<ThingDef>
  <defName>MAM_Building_Storage_Vault</defName>
  <label>advanced vault</label>
  <thingClass>Building_Storage</thingClass>
</ThingDef>
```

## 文件命名规范

### Def文件（XML）

**格式**：`[模组前缀]_[DefType][_分类].xml`

**示例**：

```
MAM_ThingDefs.xml                    # 基础ThingDef
MAM_ThingDefs_Weapons.xml            # 武器ThingDef
MAM_ThingDefs_Weapon_Melee.xml       # 近战武器
MAM_ThingDefs_Building_Power.xml     # 电力建筑
MAM_RecipeDefs.xml                   # 基础配方
MAM_RecipeDefs_Tier1.xml             # 第一阶段配方
MAM_RecipeDefs_Surgery.xml           # 手术配方
MAM_ResearchDefs_Tier1.xml           # 第一阶段研究
MAM_HediffDefs_Bionic.xml            # 仿生增强
MAM_BuildingDefs_Power.xml           # 电力建筑
```

**规则**：
1. 使用DefType的全名或简称
2. 用下划线分隔层次
3. 层次不超过2级（简化为"_分类"）
4. 避免使用数字作为唯一标识（如_1, _2）

### 代码文件（C#）

**格式**：`[模组前缀]_[功能].cs` 或 `[类名].cs`

**示例**：

```
MAM_Comp_Storage.cs
MAM_Job_Assemble.cs
MAM_JobDriver_Assemble.cs
MAM_Patch_Thing.cs
MAM_Utility_Math.cs
MAM_Window_Config.cs
```

**规则**：
1. 通常以类名命名
2. 一个文件一个类（大型系统可多类）
3. 使用PascalCase
4. 使用模组前缀避免冲突

### 其他文件

**配置文件**：
```
MAM_Settings.xml                     # 设置
MAM_Configuration.xml                # 配置
MAM_Defaults.xml                     # 默认值
```

**文档文件**：
```
MAM_API.md                           # API文档
MAM_Changelog.md                     # 更新日志
MAM_README.md                        # 说明
```

## 目录命名规范

### 游戏资源目录

遵循RimWorld标准目录名：

```
Defs/
├── ThingDefs/               # ✓ 标准
├── thing_defs/              # ✗ 错误（应为PascalCase）
├── RecipeDefs/              # ✓ 标准
├── Recipe_Defs/             # ✗ 不标准
└── BuildingDefs/            # ✓ 标准
```

**标准目录名**（不要改动）：
- Defs/
- Assemblies/
- Languages/
- Textures/
- Sounds/
- Patches/
- Source/
- Content/
- Mods/
- About/

### 自定义子目录

**格式**：PascalCase，与RimWorld风格一致

**示例**：

```
Defs/
├── ThingDefs/
│   ├── ThingDefs_Weapons/      # ✓ 正确
│   ├── Things_Weapons/         # ✗ 不一致
│   └── weapon_defs/            # ✗ 错误（应为PascalCase）
├── BuildingDefs/
│   ├── BuildingDefs_Power/     # ✓ 正确
│   └── building_power/         # ✗ 错误

Textures/
├── MAM/                        # ✓ 模组名
│   ├── UI/                     # ✓ 功能分类
│   ├── Things/                 # ✓ 功能分类
│   └── ui_icons/               # ✗ 应为UI/Icons
```

## Label（游戏显示文本）

### 格式

**原则**：label是游戏界面显示的文本，需要本地化

```xml
<defName>MAM_Weapon_LaserRifle</defName>
<label>laser rifle</label>              <!-- 英文label -->
<labelPlural>laser rifles</labelPlural>

<!-- 在Languages/ChineseSimplified/Keyed/MAM.xml中 -->
<MAM_Weapon_LaserRifle_label>激光步枪</MAM_Weapon_LaserRifle_label>
```

### Label命名规约

| 内容类型 | Label格式 | 示例 |
|---------|----------|------|
| 武器 | 小写，用空格分隔 | `laser rifle`, `advanced sword` |
| 建筑 | 小写 | `advanced forge`, `power reactor` |
| 物品 | 小写 | `component A`, `advanced chip` |
| 研究 | 小写或标题大小写 | `laser technology`, `Advanced Engineering` |
| 特性 | 小写 | `bionic arm`, `engineer` |

### 规则

1. **小写开头**：绝大多数label小写
2. **可读性优先**：用户理解最重要
3. **简洁**：避免过长
4. **一致性**：相似内容用相似表述

## 国际化（i18n）关键字

### 格式

使用`Keyed/`文件中的关键字映射

```xml
<!-- Defs/ThingDefs/MAM_ThingDefs_Weapons.xml -->
<defName>MAM_Weapon_LaserRifle</defName>
<label>MAM_Weapon_LaserRifle_label</label>

<!-- Languages/ChineseSimplified/Keyed/MAM.xml -->
<MAM_Weapon_LaserRifle_label>激光步枪</MAM_Weapon_LaserRifle_label>

<!-- Languages/English/Keyed/MAM.xml -->
<MAM_Weapon_LaserRifle_label>laser rifle</MAM_Weapon_LaserRifle_label>
```

### Keyed命名规范

**格式**：`[模组前缀]_[类型]_[具体名称]_[属性]`

**示例**：

```xml
<!-- Weapon -->
<MAM_Weapon_LaserRifle_label>激光步枪</MAM_Weapon_LaserRifle_label>
<MAM_Weapon_LaserRifle_description>一把高科技激光步枪</MAM_Weapon_LaserRifle_description>

<!-- Building -->
<MAM_Building_Forge_label>高级熔炉</MAM_Building_Forge_label>
<MAM_Building_Forge_description>用于制作高级物品</MAM_Building_Forge_description>

<!-- Research -->
<MAM_Research_LaserTech_label>激光技术</MAM_Research_LaserTech_label>
<MAM_Research_LaserTech_description>解锁激光武器制作</MAM_Research_LaserTech_description>

<!-- UI/Message -->
```

## 特殊情况处理

### 1. Parent继承

```xml
<!-- 定义基础模板（不实际存在） -->
<ThingDef Name="MAM_Weapon_Base" Abstract="True">
  <soundImpactDefault>BulletImpact_Ground</soundImpactDefault>
  <equippedAngleOffset>0</equippedAngleOffset>
</ThingDef>

<!-- 具体武器使用模板 -->
<ThingDef ParentName="MAM_Weapon_Base">
  <defName>MAM_Weapon_LaserRifle</defName>
  <label>MAM_Weapon_LaserRifle_label</label>
</ThingDef>
```

### 2. 组件（CompProperties）

```csharp
public class MAM_CompProperties_Storage : CompProperties
{
    public int storageCapacity = 100;

    public MAM_CompProperties_Storage()
    {
        this.compClass = typeof(MAM_Comp_Storage);
    }
}

public class MAM_Comp_Storage : ThingComp
{
    public MAM_CompProperties_Storage Props => (MAM_CompProperties_Storage)props;
}
```

### 3. Harmony补丁

```csharp
// 格式：MAM_Patch_[类名]_[方法名]
[HarmonyPatch(typeof(Pawn), nameof(Pawn.Tick))]
public class MAM_Patch_Pawn_Tick
{
    [HarmonyPrefix]
    public static void PreFix(Pawn __instance)
    {
        // ...
    }
}
```

## 命名风格一致性检查表

创建新内容时，检查以下项目：

- [ ] DefName遵循 `[前缀]_[分类]_[名称]` 格式
- [ ] 类名使用PascalCase，带模组前缀
- [ ] 文件名使用PascalCase（代码）或下划线分隔（Def）
- [ ] Label指向Keyed关键字（支持本地化）
- [ ] 没有使用中文字符在代码或defName中
- [ ] 组件、补丁、工具函数有明确的类型前缀
- [ ] 目录名遵循RimWorld约定（PascalCase）

## 常见错误及改正

| 错误 | 原因 | 改正 |
|------|------|------|
| `defName="weapon_laser_rifle"` | 使用下划线和小写 | `MAM_Weapon_LaserRifle` |
| `defName="WeaponLaserRifle"` | 缺少模组前缀 | `MAM_Weapon_LaserRifle` |
| `label="激光步枪"` | 直接使用中文 | `label="MAM_Weapon_LaserRifle_label"` |
| `class MyClass { }` | 缺少模组前缀 | `class MAM_SomeClass { }` |
| `ThingDefs_weapon_Melee.xml` | 不一致的大小写 | `ThingDefs_Weapon_Melee.xml` |
| `MAM_AWeapon.cs` | 避免首字母缩写 | `MAM_Weapon_Advanced.cs` |
