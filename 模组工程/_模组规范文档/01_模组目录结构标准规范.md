# RimWorld 模组目录结构标准规范

## 元信息

**摘要**：定义RimWorld模组的标准目录结构，提供三个规格模板（轻量/标准/完整），支持新模组快速启动和既有模组规范化。

**版本**：v1.0
**修改时间**：2026-01-07
**关键词**：模组结构、目录规范、版本管理、文件组织、LoadFolders
**标签**：[定稿]

---

## 概述

RimWorld模组必须遵循特定的目录结构才能被游戏正确加载。本规范基于对9个参考模组的分析，提供了三个规格的标准模板，支持不同规模和复杂度的模组项目。

### 核心原则

1. **必需基础**：About/、版本目录（1.6等）、Defs/（通常必需）
2. **版本灵活**：支持单版本（1.6）或多版本（1.4/1.5/1.6）
3. **共享机制**：使用Content/或Common/目录实现跨版本资源共享
4. **条件加载**：利用LoadFolders.xml的IfModActive实现DLC/模组感知
5. **模块独立**：各功能目录独立，便于维护和扩展

## 三个规格模板

### 规格1：轻量模板（小型工具/补丁）

**适用场景**：
- 功能简单的补丁或工具
- 仅XML定义，无C#代码
- 单个版本（推荐1.6）
- 目录数 < 50

**结构**：

```
MyMod_轻量/
├── About/                           # 必需：模组元数据
│   ├── About.xml                    # 必需：模组信息
│   ├── ModIcon.png                  # 推荐：64x64 或 256x256 像素
│   └── PublishedFileId.txt          # 可选：Steam工坊ID
│
├── 1.6/                             # 必需：版本目录
│   ├── Defs/                        # 游戏定义（按类型分类）
│   │   ├── ThingDefs/               # 物品定义
│   │   ├── RecipeDefs/              # 配方定义
│   │   └── ResearchDefs/            # 科技树定义
│   │
│   └── Languages/                   # 游戏内文本（可选）
│       └── ChineseSimplified/
│           └── Keyed/
│               └── MyMod.xml
│
├── LoadFolders.xml                  # 版本加载配置
├── LICENSE                          # 推荐：开源许可证
└── README.md                        # 推荐：模组说明
```

**文件数量**：10-20 个目录

**配置示例**：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<loadFolders>
  <v1.6>
    <li>1.6</li>
  </v1.6>
</loadFolders>
```

**适用场景**：
- 中等规模的内容模组
- 包含Defs、纹理、可能有简单代码
- 支持2-3个版本（1.5/1.6）
- 包含DLC/模组兼容性
- 目录数 50-150

**结构**：

│   ├── About.xml                    # 必需

│   └── PublishedFileId.txt

│   │   └── MyMod.dll
│   │
│   ├── Defs/                        # 定义文件（按类型分类）
│   │   ├── ThingDefs/
│   │   ├── RecipeDefs/
│   │   ├── ResearchDefs/
│   │   ├── HediffDefs/
│   │   └── Jobs/
│   │
│   └── Patches/                     # 补丁文件（可选）
│       └── VanillaExpanded_Patch.xml
│
├── 1.6/                             # 版本1.6（同1.5结构）
│   ├── Assemblies/
│   ├── Defs/
│   └── Patches/
│
├── Content/                         # 跨版本共享内容
│   ├── Languages/                   # 国际化（共享）
│   │   ├── ChineseSimplified/
│   │   │   └── Keyed/

│   │       └── Keyed/

├── LoadFolders.xml                  # 版本加载配置

**文件数量**：50-100 个目录

**配置示例**：

```xml

    <li>1.6</li>
    <li>Content</li>

    <li IfModActive="Ludeon.RimWorld.Biotech">1.6/Mods/Biotech</li>
  </v1.6>
</loadFolders>
```

## 核心目录说明

### About/（必需）

**用途**：模组的元数据和品牌资源

**必需文件**：
- `About.xml` - 模组信息（名称、作者、版本、依赖、兼容性等）

**推荐文件**：
- `ModIcon.png` - 模组图标（Steam显示）
- `Preview.png` - 预览图（更大、更详细）
- `ModIcon.dds` - DDS格式（游戏内备用）

**可选文件**：
- `PublishedFileId.txt` - Steam工坊ID（发布用）

### 版本目录（1.4/1.5/1.6 等）

**用途**：按RimWorld版本组织所有游戏资源

**子目录说明**：

| 目录 | 用途 | 必需 | 说明 |
|------|------|------|------|
| **Assemblies/** | 编译的C#程序集 | 否 | 仅在有代码时需要 |
| **Defs/** | XML游戏定义 | 通常是 | 大多数模组都有 |
| **Languages/** | 国际化文本 | 否 | 支持多语言时需要 |
| **Patches/** | XML补丁 | 否 | 修改其他内容时 |
| **Textures/** | 游戏纹理资源 | 否 | 有图形内容时 |
| **Sounds/** | 音频资源 | 否 | 有声音内容时 |
| **Mods/** | DLC特定内容 | 否 | 1.5+版本，支持DLC |

### Defs/ 目录分类

**标准的Def文件分类**（详见"RimWorld Defs 分类规范"）：

```
Defs/
├── Ability/                 # 特殊能力
├── Apparel/                 # 服装
├── BodyDefs/                # 身体定义
├── BodyPartDefs/            # 身体部位
├── BuildingDefs/            # 建筑（可细分）
│   ├── BuildingDefs_Power/
│   ├── BuildingDefs_Storage/
│   └── BuildingDefs_Security/
├── DamageDefs/              # 伤害类型
├── EffectDefs/              # 视觉效果
├── HairDefs/                # 发型
├── HediffDefs/              # 生化增强/特性
├── JobDefs/                 # 工作类型
├── RecipeDefs/              # 制作配方
├── ResearchDefs/            # 科技树
├── Stats/                   # 统计值
├── ThingDefs/               # 物品定义
├── ThoughtDefs/             # 思想
├── TraitDefs/               # 特质
├── WeaponDefs/              # 武器
└── WorkGiverDefs/           # 工作分配
```

### Content/ 目录（跨版本共享）

**用途**：存储在多个版本间完全兼容的资源，避免重复存储

**适合存储**：
- 语言文件（在版本间不变）
- 纹理资源（一般跨版本兼容）
- 完全兼容的定义

**不适合存储**：
- 版本特定的API调用
- 版本间有差异的定义

### Mods/ 目录（DLC特定内容）

**用途**：为特定DLC提供额外内容，通过LoadFolders.xml的`IfModActive`条件加载

**命名约定**：
```
Mods/
├── Royalty/        # 贵族DLC (Ludeon.RimWorld.Royalty)
├── Biotech/        # 生物技术DLC (Ludeon.RimWorld.Biotech)
├── Ideology/       # 意识形态DLC (Ludeon.RimWorld.Ideology)
└── Odyssey/        # 奥德赛DLC (Ludeon.RimWorld.Odyssey)
```

### Source/ 目录（C#源代码）

**用途**：存储完整的C#源代码，便于版本控制和开发

**结构建议**：

```
Source/
├── Properties/
│   └── AssemblyInfo.cs
├── 主命名空间/
│   ├── Components/      # 组件
│   ├── GameParts/       # 游戏系统
│   ├── HarmonyPatches/  # Harmony补丁
│   ├── Utilities/       # 工具函数
│   └── ... (其他模块)
├── *.csproj             # C#项目文件
└── *.sln                # 解决方案文件
```

## LoadFolders.xml 配置

### 作用

控制游戏在加载模组时选择哪些文件夹，实现：
- 版本特定内容加载
- DLC条件加载
- 跨版本共享资源

### 基本语法

```xml
<?xml version="1.0" encoding="UTF-8"?>
<loadFolders>
  <v1.6>              <!-- RimWorld版本号 -->
    <li>1.6</li>      <!-- 加载版本特定目录 -->
    <li>Content</li>  <!-- 加载共享目录 -->
    <!-- 条件加载：仅当DLC激活时 -->
    <li IfModActive="Ludeon.RimWorld.Royalty">1.6/Mods/Royalty</li>
  </v1.6>
</loadFolders>
```

### 常见DLC/模组的IfModActive值

**官方DLC**：
```
Ludeon.RimWorld.Royalty     # 贵族
Ludeon.RimWorld.Biotech     # 生物技术
Ludeon.RimWorld.Ideology    # 意识形态
Ludeon.RimWorld.Anomaly     # 异常
Ludeon.RimWorld.Odyssey     # 奥德赛
```

**常见第三方模组**：
```
Ancot.KiiroRace             # Kiiro种族
Ancot.MiliraRace            # Milira种族
VanillaExpanded.VWE         # 原版扩展框架
CETeam.CombatExtended       # 战斗扩展
MYDE.AerocraftFramework     # 飞行器框架
```

### 加载顺序规则

⚠️ **重要**：LoadFolders.xml中的加载顺序很重要

```xml
<!-- 正确：先加载基础内容，再加载版本特定，最后DLC覆盖 -->
<li>Content</li>           <!-- 基础 -->
<li>1.6</li>               <!-- 版本特定 -->
<li IfModActive="...">...  <!-- DLC覆盖 -->

<!-- 错误：顺序会导致DLC补丁被覆盖 -->
<li IfModActive="...">...
<li>1.6</li>
<li>Content</li>
```

## 版本管理策略

### 策略A：完整多版本镜像

**结构**：
```
MyMod/
├── 1.4/ (完整)
├── 1.5/ (完整)
├── 1.6/ (完整)
└── LoadFolders.xml
```

**优点**：
- 每个版本独立完整
- 易于维护版本差异

**缺点**：
- 文件重复多
- 存储空间大

**适用**：大型模组，版本间差异大

---

### 策略B：共享Content + 版本特定

**结构**：
```
MyMod/
├── 1.5/
├── 1.6/
├── Content/ (共享)
└── LoadFolders.xml
```

**优点**：
- 减少重复
- 便于维护共享内容

**缺点**：
- 需要仔细区分共享和版本特定

**适用**：中等模组，版本间兼容度高

**推荐使用此策略**

---

### 策略C：Common/跨版本共享（替代方案）

**结构**：
```
MyMod/
├── 1.5/
├── 1.6/
├── Common/ (共享)
└── LoadFolders.xml
```

**说明**：与策略B类似，仅名称不同

**适用**：小型模组，明确区分"通用"资源

---

### 策略D：单版本无LoadFolders

**结构**：
```
MyMod/
├── About/
├── 1.6/
├── Defs/
├── Textures/
└── (无LoadFolders.xml)
```

**说明**：仅支持1.6版本，根目录的Defs/Textures会被加载

**适用**：新模组、小型工具、仅支持最新版本

## 命名规范

详见"RimWorld模组命名规范"文档。

### 简要说明

- **类名**：遵循C#命名规范（PascalCase）
- **XML文件名**：使用_X_方式分类（如ThingDefs_Weapons.xml）
- **Def Name属性**：使用下划线分隔（MyMod_ThingName）
- **目录名**：英文，PascalCase（ThingDefs、RecipeDefs等）

## 常见错误及解决

| 错误 | 现象 | 解决方案 |
|------|------|---------|
| About.xml缺失 | Steam无法识别模组 | 创建About.xml，内容参考模板 |
| LoadFolders.xml错误 | 游戏加载错误的版本内容 | 检查IfModActive条件、加载顺序 |
| Defs/目录混乱 | 难以维护，易产生重复 | 按标准分类重新组织Defs/ |
| 多版本重复过多 | 存储空间浪费 | 迁移共用部分到Content/ |
| 没有版本目录 | 游戏找不到内容 | 创建1.6/等版本目录 |
