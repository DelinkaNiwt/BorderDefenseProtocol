## 方式1：使用轻量模板（5分钟）

**适用**：工具类、补丁类、小型Def添加

### 步骤1：复制模板

```bash
# 从模板目录复制轻量模板
cp -r "模组工程/模板/轻量模板_工具" "模组工程/我的新模组"
cd "模组工程/我的新模组"
```

### 步骤2：修改About.xml

```xml
<!-- 编辑 About/About.xml -->
<name>My Awesome Tool</name>
<author>Your Name</author>
<description>Your mod description</description>
<packageId>your.unique.modid</packageId>
```

### 步骤3：添加你的内容

```
1.6/Defs/ThingDefs/
├── ThingDefs.xml      (删除示例，添加你的定义)
```

### 步骤4：测试

```bash
# 在RimWorld中启用模组，进行测试
# 确保About.xml格式正确，Def名称唯一
```

### 完成！

现在你有了一个可以发布到Steam工坊的模组。

## 方式2：使用标准模板（10分钟）

**适用**：中型内容模组、支持多个版本

### 步骤1：复制模板

```bash
cp -r "模组工程/模板/标准模板_内容" "模组工程/我的新模组"
cd "模组工程/我的新模组"
```

### 步骤2：修改About.xml

```xml
<name>My Awesome Content</name>
<author>Your Name</author>
<supportedVersions>
  <li>1.5</li>
  <li>1.6</li>
</supportedVersions>
```

### 步骤3：组织Defs

根据《RimWorld Defs分类规范》创建所需的Def目录：

```
1.6/Defs/
├── ThingDefs/          # 物品
├── RecipeDefs/         # 配方
├── ResearchDefs/       # 研究
├── BuildingDefs/       # 建筑
└── HediffDefs/         # 特性
```

### 步骤4：创建1.5版本（可选）

```bash
# 如果需要1.5支持，复制1.6/Defs到1.5/Defs
cp -r 1.6/Defs 1.5/
# 如有版本差异，修改相应文件
```

### 步骤5：配置LoadFolders.xml

```xml
<!-- 如果使用Content/ 存储共享资源 -->
<v1.6>
  <li>Content</li>
  <li>1.6</li>
  <li IfModActive="Ludeon.RimWorld.Royalty">1.6/Mods/Royalty</li>
</v1.6>
```

### 完成！

现在你有了支持多个版本的模组。

## 方式3：使用完整模板（15分钟）

**适用**：大型系统、种族模组、框架模组

### 步骤1：复制模板

```bash
cp -r "模组工程/模板/完整模板_系统" "模组工程/我的新模组"
cd "模组工程/我的新模组"
```

### 步骤2：修改About.xml

```xml
<name>My Complete System</name>
<supportedVersions>
  <li>1.4</li>
  <li>1.5</li>
  <li>1.6</li>
</supportedVersions>
```

### 步骤3：组织多版本Defs

```
1.4/Defs/  (1.4特定内容)
1.5/Defs/  (1.5特定内容)
1.6/Defs/  (1.6特定内容)
Content/Defs/  (共享内容)
```

### 步骤4：处理版本差异

对于各版本都有的Def，如果内容完全相同：
- 放在 Content/ 中，在LoadFolders中加载

对于版本特定的Def：
- 放在 1.4/、1.5/、1.6/ 各自的目录中

### 步骤5：设置C#项目（如有代码）

```
Source/
├── Properties/AssemblyInfo.cs
├── MyMod.csproj
├── MyMod.sln
└── MyMod/
    ├── Components/
    ├── GameParts/
    └── Utilities/
```

### 步骤6：配置LoadFolders.xml

```xml
<v1.4><li>Content</li><li>1.4</li></v1.4>
<v1.5><li>Content</li><li>1.5</li><li IfModActive="...">1.5/Mods/</li></v1.5>
<v1.6><li>Content</li><li>1.6</li><li IfModActive="...">1.6/Mods/</li></v1.6>
```

### 完成！

现在你有了一个完整的、多版本支持的大型模组框架。

## 核心文件检查表

新建模组时，确保以下文件存在：

### 必需
- [ ] `About/About.xml` - 模组信息
- [ ] 版本目录（如 `1.6/`）
- [ ] `LoadFolders.xml` - 版本加载配置
- [ ] 至少一个Def文件（如 `1.6/Defs/ThingDefs/...`)

### 推荐
- [ ] `About/ModIcon.png` - 模组图标（64x64或256x256）
- [ ] `About/Preview.png` - 预览图
- [ ] `README.md` - 模组说明
- [ ] `LICENSE` - 开源许可证
- [ ] `.gitignore` - Git忽略配置

### 可选
- [ ] `Source/` - C#源代码（如有）
- [ ] `docs/` - 文档

## 命名约定（快速参考）

### 模组前缀

选择一个3-5字母的前缀作为模组ID：
- `MAM` - My Awesome Mod
- `WF` - WeaponFitting
- `TS` - ToolSet

### Def命名格式

```xml
<!-- 格式: [前缀]_[分类]_[名称] -->
<defName>MAM_Weapon_LaserRifle</defName>
<defName>MAM_Building_AdvancedForge</defName>
<defName>MAM_Recipe_CraftAdvanced</defName>
```

### 类名（如有代码）

```csharp
// 格式: MAM_[类型]_[功能]
public class MAM_Comp_Storage { }
public class MAM_Patch_Thing { }
public class MAM_Utility_Math { }
```

## 第一个Def示例

**场景**：添加一个简单的工具物品

```xml
<!-- 文件: 1.6/Defs/ThingDefs/ThingDefs_Tools.xml -->
<Defs>
  <ThingDef>
    <defName>MAM_Tool_BasicShovel</defName>
    <label>basic shovel</label>
    <description>A simple shovel for digging.</description>

    <thingClass>ThingWithComps</thingClass>
    <category>Item</category>
    <stackLimit>1</stackLimit>
    <mass>0.5</mass>

    <graphicData>
      <texPath>Things/Item/BasicShovel</texPath>
      <graphicClass>Graphic_Single</graphicClass>
    </graphicData>

    <statBases>
      <MarketValue>50</MarketValue>
    </statBases>
  </ThingDef>
</Defs>
```

### 对应的本地化

```xml
<!-- 文件: Content/Languages/ChineseSimplified/Keyed/MAM.xml -->
<MAM_Tool_BasicShovel_label>基础铲子</MAM_Tool_BasicShovel_label>
```

## 发布到Steam工坊

当你的模组完成后：

1. **验证About.xml**
   ```xml
   <packageId>your.unique.modid</packageId>
   ```

2. **创建模组封面**
   - `About/ModIcon.png` (必需)
   - `About/Preview.png` (推荐)

3. **在Steam工坊发布**
   - 打开RimWorld
   - 创建/编辑mod
   - 上传你的模组文件夹
   - Steam会生成PublishedFileId

4. **更新About.xml**
   ```xml
   <publishedFileID>生成的ID</publishedFileID>
   ```

## 常见问题

**Q: 我的模组加载时没有出现？**

A: 检查以下项目：
- [ ] About/About.xml 是否存在且格式正确
- [ ] LoadFolders.xml 是否存在
- [ ] Def文件是否在正确的版本目录中
- [ ] 是否有XML语法错误（使用XML验证工具检查）

**Q: 我想支持多个版本，应该怎么做？**

A:
1. 创建1.5/和1.6/目录，各含一份完整的Defs
2. 使用Content/存储两个版本都适用的资源
3. 配置LoadFolders.xml，按版本加载

**Q: 我的Def名称重复了，会怎样？**

A: 后加载的定义会覆盖先加载的。为避免冲突，始终使用模组前缀（如MAM_）

**Q: 如何添加C#代码？**

A:
1. 在Source/中创建.cs文件
2. 编译为.dll文件
3. 将.dll放在1.6/Assemblies/目录
4. 确保命名空间和类名遵循规范
