### HediffComp_CombatBodyActive 内层FSM

```
Channeling ──[计时完成]──→ Established ──[玩家Gizmo]──→ DismissChanneling
    │                          │                              │
    │[中断/伤害]              │[破裂条件,M3补全]             │[计时完成]
    ↓                          ↓                              ↓
  Gene.OnCombatBodyEnded    Gene.OnCombatBodyEnded         Gene.OnCombatBodyEnded
  (Passive, 跳过快照还原)   (Passive)                      (Voluntary)
```

**Channeling完成时执行的原子操作**（NR-004，11步）：
1. 拍摄快照（Hediff + 需求值 + 衣物 + 物品）
2. 移除所有现有Hediff → 添加战斗体Hediff（已存在，此处为逻辑确认）
3. 衣物转移至快照ThingOwner
4. 物品转移至快照ThingOwner
5. 禁用生理需求（由HediffDef XML驱动：`hungerRateFactor=0` + `restFallFactor=0` + `disablesNeeds: Comfort`）
6. 芯片注册（CompTrion.Allocate）
7. 冻结Trion恢复（CompTrion.SetFrozen(true)）
8. 注册维持消耗（CompTrion.RegisterDrain）
9. 强制征召（pawn.drafter.Drafted = true）
10. Spawn战斗体外观Apparel
11. 初始化影子HP（M2补全，M1中为空方法占位）

### CombatBodySnapshot（Memento容器）

```csharp
public class CombatBodySnapshot : IExposable
{
    // Hediff快照：字段值快照，非实例持有
    private List<HediffRecord> hediffSnapshots;

    // 需求值快照：Dictionary<NeedDef, float>
    private Dictionary<NeedDef, float> needValues;

    // 衣物容器：ThingOwner（物理转移）
    private ThingOwner<Apparel> apparelContainer;

    // 物品容器：ThingOwner（物理转移）
    private ThingOwner<Thing> inventoryContainer;

    public void TakeSnapshot(Pawn pawn) { ... }
    public void RestoreSnapshot(Pawn pawn) { ... }
    public void ExposeData() { ... }
}
```

**ExposeData说明**：
```csharp
public void ExposeData()
{
    // HediffRecord是自定义IExposable结构体，只含基础字段，无游离实例风险
    Scribe_Collections.Look(ref hediffSnapshots, "hediffSnapshots", LookMode.Deep);
    Scribe_Collections.Look(ref needValues, "needValues", LookMode.Def, LookMode.Value);
    Scribe_Deep.Look(ref apparelContainer, "apparelContainer", this);
    Scribe_Deep.Look(ref inventoryContainer, "inventoryContainer", this);
}
```

**快照回滚技术要点**（遵循架构文档v1.1修正）：
- Hediff回滚采用**健康系统API优先**：通过`Pawn_HealthTracker.AddHediff/RemoveHediff`操作，不直接改`hediffSet.hediffs`
- 对回滚时不希望触发的副作用采用**受控还原上下文（RestoreScope）** + 定向拦截
- 衣物恢复：先从快照ThingOwner转移出，再`pawn.apparel.Wear(dropReplacedApparel:false)`
- 物品恢复：使用`ThingOwner.TryTransferToContainer`恢复到`pawn.inventory.innerContainer`

**IThingHolder要求**：CombatBodySnapshot需实现IThingHolder接口，ThingOwner构造时传入this：
- GetChildHolders：AppendThingHoldersFromThings(outChildren, apparelContainer)
- GetDirectlyHeldThings：return inventoryContainer

**HediffRecord（自定义IExposable）**：
- defName: string
- severity: float
- bodyPartDefName: string（空字符串表示全身性hediff）
- bodyPartIndex: int（同名部位的索引，区分左右手等）
- isPermanent: bool

### 伤害管道流程

```
Patch_DamageInterception (Harmony Prefix on Pawn_HealthTracker.PreApplyDamage)
  → 检查Pawn是否有BDP_CombatBodyActive Hediff
  → 若是：absorbed = true（原版HP完全不变）
  → 调用 CombatBodyDamagePipeline.Process(dinfo, pawn, hediffComp)

CombatBodyDamagePipeline.Process:
    输入：DamageInfo dinfo, Pawn pawn, HediffComp_CombatBodyActive comp
    ├─ 获取受伤部位 P 和伤害量 x
    ├─ CompTrion.Consume(f(x))              直接消耗（f(x) > x，占位值 f(x)=x*2）
    ├─ comp.shadowHP[P] -= x                影子HP扣减
    └─ 分支：
        ├─ shadowHP[P] > 0（部位未被毁）
        │   ├─ P上已有BDP_Wound → 合并：severity增大，更新drain注册
        │   └─ P上无BDP_Wound → 新建BDP_Wound，注册Trion流失（占位速率）
        │
        └─ shadowHP[P] ≤ 0（部位被毁）
            ├─ 移除P的BDP_Wound（若有）+ 注销drain
            └─ 标记部位被毁（后果处理在M3实现，M2仅做标记）
```

### 影子HP

- 数据位置：HediffComp_CombatBodyActive内，`Dictionary<BodyPartRecord, float>`
- 初始化：激活时遍历`pawn.RaceProps.body.AllParts`，每部位 = `part.def.GetMaxHealth(pawn)`
- 植入物处理（NR-027）：快照中有植入物覆盖的缺失部位 → 以原生最大HP初始化（视为完整部位）
- 序列化：`CompExposeData()`中用`Scribe_Collections` + `Scribe_BodyParts.Look`保存

### 伤口Hediff设计

- 基类：`HediffWithComps`（不继承`Hediff_Injury`）
  - 天然屏蔽：出血、自然愈合、包扎、伤口老化、伤口合并、永久伤转化、伤口视觉覆盖层
- `tendable = false`（不可包扎）
- 同部位多次受伤 → 合并为一个（severity增大），不堆叠
- 每个部位维持唯一一条Trion持续流失注册（`CompTrion.RegisterDrain`，key含部位标识）

### 覆盖需求

| 需求 | 实现方式 |
|---|---|
| NR-001 完全伤害隔离 | Harmony absorbed=true，原版HP完全不变 |
| NR-017 耐久度统一 | 所有战斗体使用相同的影子HP初始化 + 相同f(x)，仅Trion池大小不同 |
| NR-020 无法自愈 | HediffWithComps天然不触发自愈（原版硬编码`is Hediff_Injury`） |
| NR-032 伤害机制映射 | 完整管道：原版HP无效→f(x)直接消耗→伤口→流失 |
| NR-041 影子HP系统 | Dictionary<BodyPartRecord,float> + 序列化 + 植入物处理 |
| NR-042 伤口Hediff规范 | HediffWithComps + 合并机制 + drain注册 |
| NR-004 补全 | 影子HP初始化（M1中占位，M2补全具体逻辑） |

### 验证场景

1. **基本伤害**：激活 → Dev Mode造成伤害 → 检查原版HP不变、影子HP减少、Trion消耗
2. **伤口创建**：受伤后检查Pawn健康面板出现BDP_Wound
3. **伤口合并**：同部位多次受伤 → BDP_Wound severity增大（不堆叠多个）
4. **Trion流失**：受伤后检查Gizmo_TrionBar显示drain速率增加
5. **影子HP序列化**：受伤后保存/读档 → 影子HP值一致、伤口Hediff一致
6. **植入物处理**：真身有义肢 → 激活战斗体 → 该部位影子HP为满HP → 可正常受伤

| Patch_DownedInterception | `Pawn_HealthTracker.MakeDowned` | 战斗体中拦截倒地→触发被动破裂 | M3 |

### 6个被动破裂条件

| 条件 | 检测方式 | 时机 |
|---|---|---|
| 1. Trion可用量耗尽 | HediffComp.CompTick轮询 `CompTrion.Available ≤ 0` | 运行时 |
| 2. 头部被毁 | 伤害管道中 shadowHP[Head] ≤ 0 → 即时 | 受伤时 |
| 3. 心脏被毁 | 伤害管道中 shadowHP[Heart] ≤ 0 → 即时 | 受伤时 |
| 4. 主动紧急脱离 | Gizmo按钮（M4实现，M3预留接口） | 玩家操作 |
| 5. Pawn离开地图 | M4实现 | 事件触发 |
| 6. Pawn倒地 | Patch_DownedInterception / Notify_Downed | 倒地时 |

### NR-035 非弱点器官被毁处理

| 部位 | 后果 |
|---|---|
| 头部 / 心脏 | 直接触发被动破裂（NR-008条件2/3） |
| 双腿同时被毁 | 导致倒地 → 倒地即破裂（NR-036） |
| 其他重要器官（肾、肝、胃等） | 阻止死亡 + 注册高额Trion流失（占位速率），间接导致Trion耗尽→破裂 |

### 两条解除路径对比

```
路径A：主动解除 (Voluntary)             路径B：被动破裂 (Passive)
─────────────────────────              ─────────────────────────
DismissChanneling引导                   无引导（立即执行）
│ 引导期间被动破裂优先
▼ 引导完成                             ▼ 条件触发
清理所有BDP_Wound + 注销全部drain       清理所有BDP_Wound + 注销全部drain
CompTrion.Release(allocated)           CompTrion.ForceDeplete()
CompTrion.SetFrozen(false)             CompTrion.SetFrozen(false)
注销维持消耗drain                      注销维持消耗drain
快照恢复（Hediff/Need/Apparel/Item）   紧急脱离判定（M4）
销毁外观Apparel                        快照恢复（Hediff/Need/Apparel/Item）
保持征召状态                           销毁外观Apparel
RemoveHediff(BDP_CombatBodyActive)     枯竭debuff（BDP_Exhaustion）
OuterState = Cooldown（短CD占位值）    保持征召状态
                                       RemoveHediff(BDP_CombatBodyActive)
                                       OuterState = Cooldown（长CD占位值）
```

**被动破裂优先级**：主动解除DismissChanneling期间，若满足被动破裂条件 → 中断解除引导，走被动破裂路径。

### 紧急脱离责任链

```
    ├─ Level 1: 查找最近的紧急脱离信标（自定义建筑）→ 建筑旁可站立格
```


### 跨层调用解耦

Gene_TrionGland（Core/ L0）被动破裂时需要调用EscapeRouter（Combat/ L2），但L0不能依赖L2。

解决方案：Gene定义静态事件`OnPassiveBreak`，Combat/模块在`StaticConstructorOnStartup`中注册处理逻辑（检查脱离芯片、执行传送）。Gene只广播事件，不知道谁在处理。与`QueryCanActivate`同样的零耦合模式。

### 离开地图安全网（NR-043）

在HediffComp中检测Pawn离开地图事件。方案选择留详设阶段确认（候选：override `Hediff.Notify_MapRemoved()` / Harmony监听`ExitMap`）。检测到则立即触发被动破裂。
