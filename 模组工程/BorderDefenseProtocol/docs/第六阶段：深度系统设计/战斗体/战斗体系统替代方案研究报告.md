# 战斗体系统替代方案研究报告

## 一、研究背景

### 1.1 当前方案问题

**现有设计**：伤害拦截 + 影子HP + 自定义Hediff

**遇到的问题**：
- 方案笨重，难以维护
- 实际测试遇到诸多bug
- 调试困难

### 1.2 核心需求回顾

1. 战斗体能受伤，但不使用原版伤口逻辑
2. 不流血、不疼痛、不感染、不影响心情
3. 有选择性地影响部位机能（行动/战斗相关）
4. 部位可被破坏，但除关键部位外不致死
5. 关键部位被毁或倒地时拦截，改为"战斗体破裂"
6. 支持自定义Hediff影响效果

## 二、原版API核心机制分析

### 2.1 伤害应用流程

```
Thing.TakeDamage(DamageInfo dinfo)
  ├─ PreApplyDamage(ref dinfo, out absorbed)  ← 拦截点1
  │   ├─ Pawn.PreApplyDamage (调用基类)
  │   │   ├─ genes.FactorForDamage()
  │   │   └─ health.PreApplyDamage()
  │   └─ ThingWithComps.PreApplyDamage (遍历Comps)
  │
  ├─ DamageWorker.Apply(dinfo, this)  ← 核心伤害计算
  │   └─ 各种DamageWorker子类实现
  │
  └─ PostApplyDamage(dinfo, totalDamageDealt)
```

**关键发现**：
- `PreApplyDamage`是**最早的拦截点**，可以完全阻止伤害进入原版HP系统
- `absorbed`参数设为true可以完全吸收伤害
- ThingComp可以在PreApplyDamage中拦截伤害

### 2.2 Hediff_Injury核心属性

```csharp
class Hediff_Injury : HediffWithComps
{
    float BleedRate { get; }           // 流血速率
    float PainOffset { get; }          // 疼痛值
    bool destroysBodyParts;            // 是否破坏部位

    void Heal(float amount);
    bool TryMergeWith(Hediff other);   // 伤口合并机制
}
```

**关键发现**：
- 原版伤口类`Hediff_Injury`自带流血、疼痛等副作用
- 如果不继承`Hediff_Injury`，可以天然避免这些副作用

### 2.3 死亡和倒地判定

```csharp
Pawn_HealthTracker
{
    bool ShouldBeDead();
    bool ShouldBeDowned();
    void CheckForStateChange(DamageInfo? dinfo, Hediff hediff);
    void MakeDowned(DamageInfo? dinfo, Hediff hediff);
}
```

**关键发现**：
- `CheckForStateChange`在每次Hediff变化后调用
- 可以通过Harmony patch拦截`MakeDowned`和死亡逻辑

## 三、社区模组实践分析

### 3.1 方案A：HediffComp护盾模式（VEF框架）

**实现者**：Vanilla Expanded Framework

**核心代码**：
```csharp
public class HediffComp_Shield : HediffComp
{
    public virtual void PreApplyDamage(ref DamageInfo dinfo, ref bool absorbed)
    {
        if (absorbed) return;
        if (ShieldActive && AbsorbDamage(ref dinfo))
        {
            absorbed = true;
            // 视觉效果
            // 能量消耗
        }
    }

    protected virtual bool AbsorbDamage(ref DamageInfo dinfo)
    {
        if (useEnergy)
        {
            var loss = dinfo.Amount * Props.energyLossPerDamage;
            if (loss < energy)
            {
                energy -= loss;
                dinfo.SetAmount(0f);  // 完全吸收
                return true;
            }
            Break();  // 护盾破裂
            dinfo.SetAmount(dinfo.Amount - energy / Props.energyLossPerDamage);
            return false;
        }
        dinfo.SetAmount(0f);
        return true;
    }
}
```

**配套Harmony Patch**：
```csharp
[HarmonyPatch(typeof(Pawn_HealthTracker), "PreApplyDamage")]
class Ancot_Pawn_HealthTracker_PreApplyDamage_Patch
{
    [HarmonyPrefix]
    static bool Prefix(Pawn_HealthTracker __instance, DamageInfo dinfo, ref bool absorbed)
    {
        // 遍历装备的所有Comp，调用自定义PreApplyDamage
        foreach (var comp in pawn.equipment.Primary.AllComps)
        {
            comp.PostPreApplyDamage(ref dinfo, out absorbed);
            if (absorbed) return false;  // 阻止原版处理
        }
        return true;
    }
}
```

**优点**：
- ✅ 利用原版Comp系统，架构清晰
- ✅ 完全拦截伤害，不进入原版HP系统
- ✅ 可以自定义能量/资源消耗逻辑
- ✅ 社区成熟方案，稳定性高

**缺点**：
- ❌ 需要Harmony patch拦截`Pawn_HealthTracker.PreApplyDamage`
- ❌ 护盾模式不记录"伤口"，无法实现部位受损状态

**适用性评估**：⭐⭐⭐
- 适合"完全吸收伤害"的护盾系统
- 不适合需要"记录伤口状态"的战斗体系统

### 3.2 方案B：自定义Hediff + 伤害转换模式

**核心思路**：
1. 使用Harmony patch拦截伤害
2. 不让伤害进入原版HP系统
3. 创建自定义Hediff记录"伤口状态"
4. 自定义Hediff不继承`Hediff_Injury`，天然无流血/疼痛

**伪代码实现**：
```csharp
[HarmonyPatch(typeof(Pawn_HealthTracker), "PreApplyDamage")]
class CombatBody_DamageIntercept_Patch
{
    [HarmonyPrefix]
    static bool Prefix(Pawn_HealthTracker __instance, DamageInfo dinfo, ref bool absorbed)
    {
        var pawn = GetPawn(__instance);
        var combatBodyComp = pawn.GetComp<CompCombatBody>();

        if (combatBodyComp == null || !combatBodyComp.IsActive)
            return true;  // 不在战斗体状态，走原版逻辑

        // 拦截伤害
        absorbed = true;

        // 自定义处理
        combatBodyComp.HandleDamage(dinfo);

        return false;  // 阻止原版处理
    }
}

class CompCombatBody : ThingComp
{
    private Dictionary<BodyPartRecord, float> shadowHP;  // 影子HP
    private List<Hediff_CombatBodyWound> wounds;         // 伤口记录

    public void HandleDamage(DamageInfo dinfo)
    {
        var part = dinfo.HitPart;

        // 1. 直接消耗Trion
        float trionLoss = CalculateTrionLoss(dinfo.Amount);
        TrionSystem.Consume(trionLoss);

        // 2. 更新影子HP
        shadowHP[part] -= dinfo.Amount;

        // 3. 处理伤口Hediff
        if (shadowHP[part] > 0)
        {
            // 部位未被毁，添加/更新伤口Hediff
            var wound = GetOrCreateWound(part);
            wound.severity += CalculateSeverity(dinfo.Amount);
            RegisterTrionLeak(wound);  // 注册持续流失
        }
        else
        {
            // 部位被毁
            HandlePartDestroyed(part);
        }
    }
}

// 自定义伤口Hediff（不继承Hediff_Injury）
class Hediff_CombatBodyWound : HediffWithComps
{
    // 不继承Hediff_Injury，天然无流血/疼痛

    public override void Tick()
    {
        // 自定义Trion流失逻辑
        float leakRate = CalculateLeakRate(severity);
        TrionSystem.Leak(leakRate);
    }
}
```

**优点**：
- ✅ 完全控制伤害处理流程
- ✅ 自定义Hediff不继承原版伤口类，天然无副作用
- ✅ 可以实现影子HP + 伤口记录的双轨制
- ✅ 灵活性极高，可以实现任意自定义逻辑

**缺点**：
- ❌ 需要Harmony patch，有一定侵入性
- ❌ 需要手动管理影子HP和伤口Hediff的同步
- ❌ 复杂度较高

**适用性评估**：⭐⭐⭐⭐⭐
- **最适合战斗体系统的需求**
- 可以完整实现所有43条需求

### 3.3 方案C：Hediff属性覆盖模式

**核心思路**：
1. 不拦截伤害，让伤害正常进入原版系统
2. 通过"战斗体Hediff"的属性覆盖原版伤口的副作用
3. 使用HediffDef的XML配置实现大部分特性

**HediffDef配置示例**：
```xml
<HediffDef>
  <defName>BDP_CombatBodyState</defName>
  <hediffClass>BDP.Hediff_CombatBody</hediffClass>

  <!-- 禁用需求 -->
  <disablesNeeds>
    <li>Food</li>
    <li>Rest</li>
  </disablesNeeds>

  <!-- 阻止精神崩溃 -->
  <preventsBreakdown>true</preventsBreakdown>

  <!-- 免疫环境效果 -->
  <stages>
    <li>
      <painFactor>0</painFactor>  <!-- 疼痛免疫 -->
      <totalBleedFactor>0</totalBleedFactor>  <!-- 流血免疫 -->
      <capMods>
        <li>
          <capacity>Consciousness</capacity>
          <offset>0</offset>  <!-- 不影响意识 -->
        </li>
      </capMods>
    </li>
  </stages>
</HediffDef>
```

**优点**：
- ✅ 利用原版Hediff系统，无需Harmony patch
- ✅ 配置简单，大部分特性通过XML实现
- ✅ 稳定性高，不侵入原版代码

**缺点**：
- ❌ **无法阻止伤害进入原版HP系统**
- ❌ 原版HP会真实减少，可能触发死亡
- ❌ 无法实现"影子HP"独立追踪
- ❌ 无法实现"部位被毁但不死亡"（除非拦截死亡）

**适用性评估**：⭐⭐
- 不适合战斗体系统
- 只能实现部分特性（疼痛免疫、需求禁用等）

### 3.4 方案D：Altered Carbon模式（躯体替换）

**核心思路**：
1. 将真身"冻结"在一个容器中
2. 生成一个新的临时Pawn作为战斗体
3. 战斗体死亡时，恢复真身

**实现概要**：
```csharp
class CombatBodySystem
{
    // 激活战斗体
    public Pawn ActivateCombatBody(Pawn originalPawn)
    {
        // 1. 保存真身状态
        var snapshot = SavePawnState(originalPawn);

        // 2. 生成战斗体Pawn
        var combatBodyPawn = GenerateCombatBodyPawn(originalPawn);

        // 3. 隐藏真身（Despawn或放入容器）
        HideOriginalPawn(originalPawn);

        // 4. Spawn战斗体
        GenSpawn.Spawn(combatBodyPawn, originalPawn.Position, originalPawn.Map);

        return combatBodyPawn;
    }

    // 解除战斗体
    public void DeactivateCombatBody(Pawn combatBodyPawn)
    {
        // 1. 获取真身
        var originalPawn = GetOriginalPawn(combatBodyPawn);

        // 2. 恢复真身状态
        RestorePawnState(originalPawn, snapshot);

        // 3. Spawn真身
        GenSpawn.Spawn(originalPawn, combatBodyPawn.Position, combatBodyPawn.Map);

        // 4. 销毁战斗体
        combatBodyPawn.Destroy();
    }
}
```

**优点**：
- ✅ 完全隔离真身和战斗体
- ✅ 战斗体是独立Pawn，可以自由定制
- ✅ 不需要复杂的伤害拦截逻辑

**缺点**：
- ❌ **极其复杂**，需要处理大量边缘情况
- ❌ Pawn替换会破坏很多游戏系统（社交、工作、队伍等）
- ❌ 存档兼容性问题严重
- ❌ 性能开销大（频繁Spawn/Despawn）

**适用性评估**：⭐
- **不推荐**，复杂度和风险极高
- Altered Carbon模组用于"永久换躯体"，不适合"临时战斗体"

### 4.2 最终推荐：方案B（自定义Hediff + 伤害拦截）

**推荐理由**：
1. ✅ 可以完整实现所有43条需求
2. ✅ 架构清晰，职责分离
3. ✅ 社区有成熟的Harmony patch实践
4. ✅ 性能开销低
5. ✅ 可维护性好

**核心改进点**（相比当前方案）：
1. **简化影子HP管理**：使用`Dictionary<BodyPartRecord, float>`直接存储
2. **简化伤口Hediff**：不继承`Hediff_Injury`，避免原版副作用
3. **统一拦截点**：只在`Pawn_HealthTracker.PreApplyDamage`一处拦截
4. **清晰的职责分离**：
   - `CompCombatBody`：状态管理、激活/解除逻辑
   - `Hediff_CombatBodyWound`：伤口记录、Trion流失
   - Harmony Patch：伤害拦截

## 五、方案B详细设计

### 5.1 核心类设计

```
CompCombatBody (ThingComp)
├─ 状态管理
│  ├─ bool IsActive
│  ├─ CombatBodySnapshot snapshot
│  └─ Dictionary<BodyPartRecord, float> shadowHP
├─ 激活/解除
│  ├─ Activate()
│  ├─ Deactivate()
│  └─ Break()
└─ 伤害处理
   ├─ HandleDamage(DamageInfo)
   ├─ HandlePartDestroyed(BodyPartRecord)
   └─ CheckBreakConditions()

Hediff_CombatBodyState (HediffWithComps)
├─ 战斗体主Hediff
├─ 提供疼痛免疫、需求禁用等特性
└─ 通过HediffDef配置大部分属性

Hediff_CombatBodyWound (HediffWithComps)
├─ 伤口记录Hediff
├─ 不继承Hediff_Injury
├─ Tick()中处理Trion流失
└─ 支持伤口合并

CombatBody_DamageIntercept_Patch (Harmony)
└─ 拦截Pawn_HealthTracker.PreApplyDamage
```

### 5.2 伤害处理流程

```
攻击命中
  ↓
Pawn_HealthTracker.PreApplyDamage
  ↓
[Harmony Patch拦截]
  ├─ 检查是否在战斗体状态
  ├─ 是 → absorbed = true，调用CompCombatBody.HandleDamage()
  └─ 否 → 走原版逻辑
  ↓
CompCombatBody.HandleDamage(DamageInfo)
  ├─ 1. 直接消耗Trion：f(x)
  ├─ 2. 更新影子HP：shadowHP[part] -= x
  ├─ 3. 判断部位状态
  │   ├─ shadowHP > 0 → 添加/更新伤口Hediff
  │   └─ shadowHP ≤ 0 → HandlePartDestroyed()
  └─ 4. 检查破裂条件
      ├─ 头/心脏被毁 → Break()
      ├─ Trion耗尽 → Break()
      └─ 倒地 → Break()
```

### 5.3 关键技术点

#### 5.3.1 影子HP序列化

```csharp
public class CompCombatBody : ThingComp
{
    private Dictionary<BodyPartRecord, float> shadowHP;

    public override void PostExposeData()
    {
        base.PostExposeData();

        // 序列化影子HP
        if (Scribe.mode == LoadSaveMode.Saving)
        {
            var keys = shadowHP.Keys.ToList();
            var values = shadowHP.Values.ToList();
            Scribe_Collections.Look(ref keys, "shadowHP_keys");
            Scribe_Collections.Look(ref values, "shadowHP_values");
        }
        else if (Scribe.mode == LoadSaveMode.LoadingVars)
        {
            List<BodyPartRecord> keys = null;
            List<float> values = null;
            Scribe_Collections.Look(ref keys, "shadowHP_keys");
            Scribe_Collections.Look(ref values, "shadowHP_values");

            shadowHP = new Dictionary<BodyPartRecord, float>();
            for (int i = 0; i < keys.Count; i++)
            {
                shadowHP[keys[i]] = values[i];
            }
        }
    }
}
```

#### 5.3.2 伤口Hediff合并

```csharp
public class Hediff_CombatBodyWound : HediffWithComps
{
    public override bool TryMergeWith(Hediff other)
    {
        if (other is Hediff_CombatBodyWound wound && wound.Part == this.Part)
        {
            // 合并severity
            this.Severity += wound.Severity;

            // 更新Trion流失注册
            UpdateTrionLeak();

            return true;
        }
        return false;
    }
}
```

#### 5.3.3 倒地拦截

```csharp
[HarmonyPatch(typeof(Pawn_HealthTracker), "MakeDowned")]
class CombatBody_DownedIntercept_Patch
{
    [HarmonyPrefix]
    static bool Prefix(Pawn_HealthTracker __instance)
    {
        var pawn = GetPawn(__instance);
        var comp = pawn.GetComp<CompCombatBody>();

        if (comp != null && comp.IsActive)
        {
            // 战斗体状态下倒地 → 触发破裂
            comp.Break();
            return false;  // 阻止原版倒地逻辑
        }

        return true;
    }
}
```
