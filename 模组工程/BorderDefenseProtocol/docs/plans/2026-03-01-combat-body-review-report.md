## 二、关键技术发现

### 2.1 HediffStage原生能力——"战斗体Hediff"的基础

通过源码分析，发现RimWorld的`HediffStage`提供了大量原生字段，可以直接用于实现战斗体特性，**极大减少所需的C#代码量**：

| HediffStage字段 | 用途 | 对应战斗体需求 |
|---|---|---|
| `painOffset = -999f` 或 `painFactor = 0f` | 免疫疼痛 | NR-019 疼痛免疫 |
| `totalBleedFactor = 0f` | 禁止出血 | 战斗体不出血 |
| `naturalHealingFactor = 0f` | 禁止自然愈合 | NR-020 无法自愈 |
| `hungerRateFactor = 0f` | 消除饥饿积累 | 生理冻结 |
| `restFallFactor = 0f` | 消除疲劳积累 | NR-018 无限体力 |
| `disablesNeeds = [Food, Rest, Comfort]` | 禁用指定Need | 冻结生理需求 |
| `blocksMentalBreaks = true` | 阻止精神崩溃 | 战斗体中不崩溃 |
| `makeImmuneTo = [...]` | 免疫指定Hediff | 免疫中毒、疾病 |
| `preventVacuumBurns = true` | 防止真空灼伤 | 战斗体环境抗性 |
| `capMods` | 能力修改器 | 耐久度统一、能力提升 |
| `statOffsets` / `statFactors` | 属性修改 | 各种数值调整 |

### 2.2 需求禁用的心情影响机制

所有需求相关的ThoughtWorker在需求被禁用（对象为null）时，均返回`ThoughtState.Inactive`：

```csharp
// 以饥饿为例，其他需求同理
if (p.needs.food == null) return ThoughtState.Inactive;
```

**结论**：通过`disablesNeeds`禁用生理需求后：
- 不会产生该需求的**正面**思想（如"吃了精美一餐+12"）
- 不会产生该需求的**负面**思想（如"饥饿-6"）
- 需求直接从Pawn的需求列表中移除
- 解除战斗体时需要通过`AddOrRemoveNeedsAsAppropriate()`重新添加并恢复快照值

### 2.3 食尸鬼转化机制参考（Pawn_MutantTracker）

RimWorld原版的食尸鬼转化通过`Pawn_MutantTracker.Turn()`实现"状态突变"，其模式可供参考：
- **Turn()**：添加Hediff、修改需求、重置基因、修改外观、处理装备
- **Revert()**：移除Hediff、恢复需求、恢复派系/意识形态、清理外观
- 保存原始状态：`originalFaction`、`originalIdeo`字段

## 三、已发现的遗漏（v1文档缺失的需求）

### 遗漏1：真身状态快照（最重要）

**问题**：v1文档NR-004"激活效果"未提及"真身状态快照"步骤，但这是伤害隔离和状态回滚的技术前提。

**已确认的设计决策**：
1. **快照时机**：激活完成的同时，与状态切换是一套原子操作
2. **激活被打断**：快照不生成
3. **引导期间受伤**：伤口和状态会被记录在快照中（因为快照在激活完成时才生成）
4. **快照归属**：属于Pawn

### 遗漏2：战斗体Hediff替换策略

**问题**：v1文档未明确如何在技术层面实现"伤害隔离"。

**已确认的设计决策**：
- 激活时：记录快照 → 移除所有现有Hediff → 添加"战斗体Hediff"
- 解除时：移除所有战斗体期间的Hediff（包括战斗体Hediff本身）→ 从快照恢复原有Hediff
- 两者无缝衔接，不会有中间状态

### 遗漏3：衣物/外观处理

**问题**：v1文档未提及战斗体状态下的衣物和外观表现。

**已确认的设计决策**：
- 衣物：隐藏（记录在快照中，解除时恢复）
- 外观：应有新的贴图替换覆盖，作为"战斗体"的身体表现
- 武器：伪命题，激活前提是手持触发体

### 遗漏4：携带物品（Inventory）处理

**状态**：⚠️ 用户未决定，需继续讨论

### 遗漏5：生理 vs 心理的明确划分

**问题**：v1文档NR-025"生理状态冻结"的范围不够明确。

**已确认的分类原则**：只冻结生理，不冻结心理。

## 四、已确认的设计决策汇总

### 4.1 生理（冻结）vs 心理（不冻结）分类

| 系统 | 分类 | 处理方式 | 说明 |
|---|---|---|---|
| **饥饿（Hunger/Food）** | 生理 | 冻结（disablesNeeds） | 战斗体无进食需求 |
| **休息（Rest）** | 生理 | 冻结（disablesNeeds + restFallFactor=0） | 战斗体无疲劳 |
| **舒适度（Comfort）** | 生理 | 冻结（disablesNeeds） | 战斗体无舒适度感受 |
| **体温** | 生理 | 免疫（makeImmuneTo或拦截） | 战斗体不受环境温度影响 |
| **疼痛（Pain）** | 生理 | 免疫（painFactor=0） | NR-019 |
| **出血（Bleeding）** | 生理 | 禁止（totalBleedFactor=0） | 不出血，改为Trion泄漏机制 |
| **中毒（Toxicity）** | 生理 | 免疫 | 战斗体免疫中毒 |
| **疾病进度** | 生理 | 冻结（快照恢复） | 解除后回滚到激活时的阶段和进度 |
| **自然愈合** | 生理 | 禁止（naturalHealingFactor=0） | NR-020 无法自愈 |
| **精神崩溃** | 心理 | 不冻结但阻止触发（blocksMentalBreaks=true） | 心情可以下降，但不触发崩溃 |
| **心情（Mood）** | 心理 | 不冻结 | 正常运行 |
| **思想/记忆（Thoughts）** | 心理 | 不冻结 | 战斗经历正常产生思想 |
| **技能经验（Skills）** | 心理 | 不冻结 | 战斗经验正常增长 |
| **人际关系（Relations）** | 心理 | 不冻结 | 战斗互动正常影响关系 |
| **娱乐（Recreation/Joy）** | 心理 | 不冻结 | 正常运行（⚠️ 待最终确认） |
| **美观（Beauty）** | 心理 | 不冻结 | 正常运行（⚠️ 待最终确认） |
| **户外（Outdoors）** | 心理 | 不冻结 | 正常运行（⚠️ 待最终确认） |

### 4.2 伤害机制映射

| 原作概念 | 游戏映射 | 说明 |
|---|---|---|
| 掉血 | 扣Trion可用值 | CompTrion.Consume() |

## 五、待继续讨论的开放问题

### 开放问题1：娱乐/美观/户外的最终归类

**现状**：建议归为心理层面不冻结，但未得到用户最终确认。

**影响**：如果不冻结，长时间战斗会导致娱乐需求下降、产生负面思想。这增加了战斗体的时间压力（精神层面），但需确认是否是期望的游戏体验。

### 开放问题2：携带物品（Inventory）处理

**选项**：
- A：保持可用，战斗体可使用
- B：冻结不可用，解除时恢复
- C：清空

### 开放问题3：精神崩溃的`blocksMentalBreaks`方案确认

**现状**：建议使用HediffStage原生的`blocksMentalBreaks = true`，但未得到用户最终确认。

### 开放问题4：疾病进度冻结的exploit风险

**问题**：战斗体可以"冻结"疾病进度，玩家可能利用此机制避免疾病恶化。

**选项**：
- A：接受，这是战斗体的附带好处
- B：通过Trion维持消耗自然限制战斗体持续时间
- C：解除时补偿经过的时间

### 开放问题5：快照的具体存储结构

**已确认**：快照属于Pawn。但具体的数据结构和存储方式未定。

**已确认方向**：利用HediffStage原生字段实现大部分特性。但具体的Hediff定义（各字段具体值、分几个Stage等）未定。

## 六、需求文档v1中发现的描述歧义

### 歧义1：NR-017"耐久度统一"

**原文**："所有战斗体的血量/护甲值相同，不受真身属性影响"

**歧义**："血量/护甲值"的具体含义不明确。是指：
- A：所有战斗体的HP相同？（通过capMods设置固定值）
- B：所有战斗体的护甲值相同？（通过statOffsets设置固定值）
- C：两者都是？

**建议**：在v2中明确具体数值和实现方式。

### 歧义2：NR-018"无限体力"

**原文**："战斗体状态下不消耗体力，或体力恢复速度极快"

**已有解法**：`restFallFactor = 0f`可直接实现"不消耗体力"，不需要"恢复速度极快"的备选方案。建议在v2中删除备选方案。

### 歧义3：NR-014"传送目标建筑"

**原文**："如果没有建造，紧急脱离芯片无法生效（或传送到地图边缘）"

**歧义**：两个备选行为应该只保留一个。建议在v2中明确。

### 歧义4：NR-025"生理状态冻结"中的"可以做这些活动"

**原文**："可以做这些活动，但没有结果意义"

**歧义**：如果需求被disablesNeeds移除，部分AI行为（如自动寻找食物）将不再触发。这里的"可以做"是指"如果玩家强制命令"还是"AI也会自发去做"？按照disablesNeeds的机制，应该是"AI不会自发去做，但玩家强制命令可以执行"。需要在v2中明确。
