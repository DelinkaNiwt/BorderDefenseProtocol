# 芯片类别系统使用指南

## 概述

BDP模组v3.1引入了**芯片类别标签系统**和**槽位限制枚举**，用于多维度分类芯片，支持筛选、分组、批量设置等功能。

## 1. 槽位限制（ChipSlotRestriction）

### 枚举定义

```csharp
public enum ChipSlotRestriction
{
    None = 0,        // 无限制，可插任何槽位
    SpecialOnly = 1, // 只能插特殊槽位
    HandsOnly = 2    // 只能插左右手槽位
}
```

### XML配置示例

```xml
<comps>
  <li Class="BDP.Trigger.CompProperties_TriggerChip">
    <!-- 无限制（默认） -->
    <slotRestriction>None</slotRestriction>

    <!-- 只能插特殊槽位 -->
    <slotRestriction>SpecialOnly</slotRestriction>

    <!-- 只能插左右手槽位 -->
    <slotRestriction>HandsOnly</slotRestriction>
  </li>
</comps>
```

## 2. 芯片类别标签（Categories）

### 标签维度体系

#### 功能维度
- `Weapon` - 武器类
- `Shield` - 护盾类
- `Ability` - 能力类
- `Utility` - 辅助类
- `System` - 系统类

#### 战斗方式维度
- `Melee` - 近战
- `Ranged` - 远程

#### 距离维度
- `Close` - 近距离（0-15格）
- `Medium` - 中距离（15-30格）
- `Long` - 远距离（30+格）

#### 伤害/能量类型维度
- `Physical` - 物理伤害
- `Energy` - 能量伤害
- `Explosive` - 爆炸伤害

#### 武器细分维度
- `Blade` - 刀刃类
- `Blunt` - 钝器类
- `Rifle` - 步枪类
- `Sniper` - 狙击类
- `Shotgun` - 霰弹类
- `Launcher` - 发射器类

#### 特殊机制维度
- `Tracking` - 追踪类
- `Guided` - 引导类
- `Volley` - 齐射类
- `Burst` - 连射类
- `Passthrough` - 穿透类

#### 消耗特性维度
- `HighCost` - 高消耗
- `Continuous` - 持续消耗
- `Passive` - 被动触发

### XML配置示例

```xml
<!-- 近战武器芯片 -->
<comps>
  <li Class="BDP.Trigger.CompProperties_TriggerChip">
    <chipEffectClass>BDP.Trigger.WeaponChipEffect</chipEffectClass>
    <slotRestriction>HandsOnly</slotRestriction>
    <categories>
      <li>Weapon</li>
      <li>Melee</li>
      <li>Close</li>
      <li>Physical</li>
      <li>Blade</li>
      <li>Burst</li>
    </categories>
  </li>
</comps>

<!-- 远程狙击芯片 -->
<comps>

    <chipEffectClass>BDP.Trigger.WeaponChipEffect</chipEffectClass>
    <slotRestriction>HandsOnly</slotRestriction>
    <categories>
      <li>Weapon</li>
      <li>Ranged</li>
      <li>Long</li>
      <li>Energy</li>
      <li>Sniper</li>
      <li>Guided</li>
      <li>HighCost</li>
    </categories>
  </li>
</comps>

<!-- 系统芯片 -->
<comps>
  <li Class="BDP.Trigger.CompProperties_TriggerChip">
    <chipEffectClass>BDP.Combat.EmergencyEscapeChipEffect</chipEffectClass>
    <slotRestriction>SpecialOnly</slotRestriction>
    <categories>
      <li>System</li>
      <li>Passive</li>
      <li>Utility</li>
      <li>HighCost</li>
    </categories>
  </li>
</comps>
```

## 3. 代码使用示例

### 查询芯片类别

```csharp
// 检查芯片是否包含指定类别
if (chipProps.HasCategory(ChipCategories.Weapon))
{
    // 这是武器类芯片
}

// 检查芯片是否包含任意一个类别
if (chipProps.HasAnyCategory(ChipCategories.Ranged, ChipCategories.Melee))
{
    // 这是远程或近战芯片
}

// 检查芯片是否包含所有指定类别
if (chipProps.HasAllCategories(ChipCategories.Weapon, ChipCategories.Energy))
{
    // 这是能量武器芯片
}

// 获取所有类别标签
foreach (var category in chipProps.GetCategories())
{
    Log.Message($"类别: {category}");
}
```

### 筛选芯片

```csharp
// 获取所有武器类芯片
var weaponChips = allChips.Where(c =>
{
    var props = c.TryGetComp<TriggerChipComp>()?.Props as CompProperties_TriggerChip;
    return props?.HasCategory(ChipCategories.Weapon) ?? false;
});

// 获取所有远程+能量类芯片
var energyRangedChips = allChips.Where(c =>
{
    var props = c.TryGetComp<TriggerChipComp>()?.Props as CompProperties_TriggerChip;
    return props?.HasAllCategories(ChipCategories.Ranged, ChipCategories.Energy) ?? false;
});

// 获取所有高消耗芯片
var highCostChips = allChips.Where(c =>
{
    var props = c.TryGetComp<TriggerChipComp>()?.Props as CompProperties_TriggerChip;
    return props?.HasCategory(ChipCategories.HighCost) ?? false;
});
```

### UI分组显示

```csharp
// 按功能维度分组
var grouped = chips.GroupBy(c =>
{
    var props = c.TryGetComp<TriggerChipComp>()?.Props as CompProperties_TriggerChip;
    if (props == null) return "Unknown";

    if (props.HasCategory(ChipCategories.Weapon)) return "武器";
    if (props.HasCategory(ChipCategories.Shield)) return "护盾";
    if (props.HasCategory(ChipCategories.Ability)) return "能力";
    if (props.HasCategory(ChipCategories.Utility)) return "辅助";
    if (props.HasCategory(ChipCategories.System)) return "系统";

    return "其他";
});

foreach (var group in grouped)
{
    Log.Message($"类别: {group.Key}");
    foreach (var chip in group)
    {
        Log.Message($"  - {chip.LabelCap}");
    }
}
```

## 4. 扩展自定义标签

除了使用`ChipCategories`常量类中的标签，你也可以定义自己的标签：

```xml
<categories>
  <li>Weapon</li>
  <li>Ranged</li>
  <li>CustomTag_MySpecialType</li>  <!-- 自定义标签 -->
  <li>Experimental</li>              <!-- 自定义标签 -->
</categories>
```

## 5. 最佳实践

1. **多维度标签**：一个芯片应该有多个维度的标签，便于从不同角度筛选
   - 功能维度：Weapon/Shield/Ability/Utility/System
   - 战斗方式：Melee/Ranged
   - 距离维度：Close/Medium/Long
   - 伤害类型：Physical/Energy/Explosive
   - 特殊机制：Tracking/Guided/Volley/Burst/Passthrough
   - 消耗特性：HighCost/Continuous/Passive

2. **标签一致性**：使用`ChipCategories`常量类中的标签，避免拼写错误

3. **槽位限制**：
   - 武器/护盾/能力芯片：使用`HandsOnly`
   - 系统芯片（如紧急脱离）：使用`SpecialOnly`
   - 通用芯片：使用`None`

4. **向后兼容**：`categories`字段为null时不影响现有功能
