# 工作流模板使用指南

## 概述

`_工作流模板` 是RimWorld模组开发工作流的标准模板目录。当创建新项目或初始化项目工作流时，可以从本模板复制完整的目录结构和文件。

## 模板结构

```
_工作流模板/
├── README.md                     # 本文件
├── 项目上下文/                   # 项目元数据和工作流状态
│   ├── 项目元数据.yaml          # 项目的基本信息
│   ├── 工作流状态.md            # 各阶段的进度状态
│   └── 版本日志.md              # 版本更新历史
├── 智能体工作/                   # 7个Agent的工作目录
│   ├── 01_头脑风暴Agent/
│   │   ├── 状态.yaml
│   │   ├── 输出.md
│   │   └── RimWorld查询/
│   ├── 02_需求分析Agent/
│   │   ├── 状态.yaml
│   │   ├── 输出.md
│   │   └── RimWorld查询/
│   ├── ... (03-07以此类推)
│   └── 07_更新维护Agent/
│       ├── 状态.yaml
│       ├── 输出.md/维护日志.md
│       └── RimWorld查询/
├── 交付物/                       # 最终交付内容
│   ├── 模组包/                  # 编译后的模组包
│   ├── 发布说明.md              # 用户可读的发布说明
│   └── 使用指南.md              # 用户使用指南
├── .claude/                      # Claude AI系统相关
│   └── 事件/                    # 项目事件和决策记录
│       └── README.md
```

## 使用方法

### 1. 新建项目时

当用户启动新项目时，主Agent应该：

```
1. 读取模板结构
2. 在 模组工程/[项目名]/ 下创建 工作流/ 目录
3. 从 _工作流模板/ 复制所有内容到新建的工作流目录
4. 更新 项目上下文/项目元数据.yaml 中的项目信息
5. 初始化 工作流状态.md
```

### 2. 复制命令示例

```bash
# 复制模板到新项目
cp -r 模组工程/_工作流模板 模组工程/[新项目名]/工作流

# 或在Windows下
xcopy 模组工程\_工作流模板 模组工程\[新项目名]\工作流 /E /I /Y
```

### 3. 初始化新项目

复制后，主Agent需要：

1. 填写 `项目上下文/项目元数据.yaml`：
   - 项目名称
   - 版本
   - RimWorld版本支持
   - 核心目标

2. 更新 `项目上下文/工作流状态.md`：
   - 项目名称
   - 当前阶段：01_头脑风暴Agent
   - 更新时间戳

3. 初始化 `.claude/事件/`：
   - 创建项目初始化事件记录

## 每个目录的说明

### 项目上下文 (项目上下文/)

**用途**：存放项目的元信息和工作流状态

**文件**：
- `项目元数据.yaml`：项目基本信息、目标、版本
- `工作流状态.md`：各阶段进度状态
- `版本日志.md`：版本更新历史

### 智能体工作 (智能体工作/)

**用途**：7个Agent的工作目录

**结构**：
```
01_头脑风暴Agent/       ← Agent编号_Agent名
├── 状态.yaml           ← Agent工作状态
├── 输出.md             ← Agent的主要输出（可能有多个，如架构验证报告.md等）
└── RimWorld查询/       ← RiMCP查询记录目录（可选）
    └── [查询记录].md   ← 具体的查询记录
```

**文件约定**：
- `状态.yaml`：必需，记录Agent的当前状态、验证状态等
- `输出.md`：必需，Agent的主要输出文档
- `RimWorld查询/`：可选，存放使用RiMCP的查询日志

### 交付物 (交付物/)

**用途**：项目最终交付物

**文件**：
- `模组包/`：编译后的模组文件目录
- `发布说明.md`：用户可读的版本发布说明
- `使用指南.md`：用户使用指南

### Claude事件 (.claude/事件/)

**用途**：记录Claude AI系统的重要事件和决策

**文件**：时间戳+事件描述的.md文件，记录各类关键事件

## 模板文件的内容

### 项目元数据.yaml

包含：
- 项目名称、版本
- RimWorld版本支持
- 核心目标、用户需求
- 项目类型、难度
- 当前阶段、进度
- 关键决策、约束

### 工作流状态.md

包含：
- 7个阶段的进度表
- 每个Agent的状态说明
- 最新事件列表
- 下一步行动

### 版本日志.md

包含：
- 版本历史记录
- 版本规划表
- 更新频率和兼容性说明

### Agent状态.yaml

包含：
- Agent名称、当前状态
- 启动和完成时间
- 用户验证状态
- 输出文件清单

### 事件README.md

说明如何记录项目事件

  ↓

## 维护建议

1. **模板更新**：当发现模板中有不合理的地方，更新 `_工作流模板/` 本身

2. **新项目参考**：新项目应该定期参考最新的模板结构

3. **模板版本**：考虑对模板进行版本管理，确保一致性

4. **工具支持**：可以编写脚本自动从模板初始化新项目

## 常见问题

**Q: 可以修改模板中的文件吗？**
A: 是的。新建项目时，复制后的文件完全属于该项目，可以自由修改。修改模板本身需要谨慎，因为会影响后续新项目。

**Q: 需要所有文件都填充内容吗？**
A: 不需要。Agent工作过程中会逐步填充相应的输出文件。模板只是框架。

**Q: RimWorld查询目录什么时候创建内容？**
A: 当Agent使用RiMCP进行查询时，应该记录查询内容和结果到该目录。

**Q: 如何处理不需要的文件？**
A: 可以删除不需要的模板文件。例如，如果某个Agent不需要特定的输出文件，可以删除。
