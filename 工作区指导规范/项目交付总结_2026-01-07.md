## 项目概述

**项目名称**：RimWorld 模组开发标准规范与模板体系构建

**完成日期**：2026-01-07

**项目类型**：知识提炼与标准化

## 🚀 使用指南

### 对于新建模组

**推荐流程**：

1. 📖 **阅读快速启动指南** `模组快速启动指南.md`
   - 选择合适的模板（轻量/标准/完整）
   - 了解目标模组规模

2. 📋 **复制对应模板**
   ```bash
   cp -r 模组工程/模板/标准模板_内容 模组工程/我的新模组
   ```

3. ⚙️ **参考规范文档**
   - 修改About.xml时参考《命名规范》
   - 创建Defs时参考《Defs分类规范》
   - 组织目录时参考《目录结构标准规范》

4. 📚 **查看示例**
   - 参考 `示例/` 中的完整示例
   - 参考 `示例/README.md` 了解说明方式

### 对于既有模组改进

**推荐流程**：

1. 🔍 **对标规范**
   - 用《目录结构标准规范》检查当前结构
   - 用《Defs分类规范》检查定义组织
   - 用《命名规范》检查命名一致性

2. ✏️ **逐步改进**
   - 补充缺失的About.xml
   - 添加LoadFolders.xml
   - 重新组织Def文件

3. 📖 **参考示例**
   - 对照示例模组的结构和注释方式
   - 学习如何编写清晰的文档

## 💡 最佳实践总结

### 新建模组"五分钟快速启动"检查表

- [ ] 复制合适的模板
- [ ] 编辑About.xml（模组名、作者、ID）
- [ ] 创建LoadFolders.xml（版本配置）
- [ ] 添加Def文件到对应目录
- [ ] 在RimWorld中测试

### Defs文件组织"黄金法则"

- 每个文件不超过200个定义
- 按功能而非数量分类
- 使用模组前缀避免冲突
- 在Keyed中集中管理本地化文本

### 命名一致性"三部曲"

- 选择3-5字母的模组前缀
- 所有defName格式：`[前缀]_[分类]_[名称]`
- 所有类名格式：`[前缀]_[类型]_[功能]`
