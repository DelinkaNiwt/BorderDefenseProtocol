## 前言

本文档是BDP模组战斗体系统的第3版需求设计文档。整合了v1（25条初始需求）、v2（查漏补缺+8条新增）、以及v2→v3讨论（解决6个开放问题+架构审查7条新增）的全部成果。

**设计原则**：
1. **有选择性的机制移植**：不是100%还原原作，而是为RimWorld游戏体验服务
2. **利用现有机制**：HediffStage原生字段覆盖大部分战斗体特性，XML优先于C#
3. **增加战术深度**：激活/解除引导时间、两条解除路径的Trion结算差异、冷却CD
4. **保持游戏平衡**：不实现非Trion免疫、不修复缺陷、维持消耗+冷却防止滥用
5. **生理/心理分离**：生理冻结回滚，心理正常运行保留
6. **最简原则**：保留完整人体BodyDef拦截致死逻辑，不做高风险的躯体结构替换

**关键技术发现**（v2）：RimWorld的`HediffStage`原生字段可覆盖大部分战斗体特性（painFactor、totalBleedFactor、naturalHealingFactor、disablesNeeds、blocksMentalBreaks、makeImmuneTo等），极大减少C#代码量。

**文档状态**：v3为需求终版，所有开放问题已解决，无遗留待确认项。

**变更标记说明**：
- 🆕 = v3新增的需求
- ✏️ = v3修改的需求（相比v2有实质变化）
- ✅ = v2已有，v3未变

### NR-001: 完全伤害隔离 ✅
- 战斗体受伤完全不影响真身
- 解除后真身回到激活时的状态
- 这是战斗体系统的根本价值，不可妥协

## 二、激活机制

### NR-002: 激活前置条件 ✅
- 装备触发体（触发器装备）
- 当前Trion ≥ 激活所需的占用量
- 不在战斗体状态下
- 不在冷却CD中（🆕 v3追加，详见NR-038）

### NR-003: 激活过程 ✅
- 引导时间：1-3秒
- 引导期间可被打断（受到伤害/被击倒/失去意识等）
- 引导被打断 → 激活失败，无任何效果发生
- 引导完成后立即进入战斗体状态

### NR-004: 激活效果 ✏️（v3补充物品冻结）
激活完成时以下操作作为**一套原子操作**同时执行：

1. **真身状态快照**：记录真身当前的完整生理状态（详见NR-026）
2. **Hediff替换**：移除所有现有Hediff，添加"战斗体Hediff"（详见NR-027）
3. **衣物隐藏**：隐藏当前穿戴的衣物，转移至快照容器（详见NR-028）
4. **物品冻结**（🆕 v3）：将背包物品从Inventory转移至快照容器（详见NR-028）
5. **需求冻结**：禁用生理类需求（详见NR-029）
6. 芯片注册：已装载的触发芯片向Trion资源池注册，锁定占用量
7. 状态切换：进入战斗体状态
8. Trion冻结：设置CompTrion.frozen = true，停止Trion自然恢复
9. **维持消耗注册**（🆕 v3）：注册战斗体基础维持消耗drain（详见NR-034）

**关键约束**：
- 激活被打断 → 上述操作均不执行（快照不生成）
- 引导过程中受伤 → 伤口和状态在快照时一起被记录（快照在激活完成瞬间生成）

```
激活流程：

  [前置检查] ─ 条件不满足 ─→ 失败（提示）
      │ 条件满足
      ▼
  [引导开始] ─ 被打断 ─→ 失败（无任何效果）
      │ 引导完成
      ▼
  [原子操作] ← 以下步骤不可分割、不可打断
      ├─ 快照（Hediff + 需求值 + 衣物 + 物品）
      ├─ 移除所有Hediff → 添加战斗体Hediff
      ├─ 衣物 + 物品 → 转移至快照容器
      ├─ 禁用生理需求
      ├─ 芯片注册（锁定占用量）
      ├─ frozen = true
      └─ RegisterDrain（维持消耗）
      │
      ▼
  [战斗体状态]
```

## 三、解除机制

### 3.1 主动解除（Trigger Off）

#### NR-005: 主动解除触发条件 ✅
- 玩家主动发出解除指令

#### NR-006: 主动解除过程 ✏️（v3新增引导时间+物品恢复）

1. **引导时间**（🆕 v3）：与激活类似，有短暂引导，引导期间可被打断
2. 引导被打断 → 仍在战斗体状态，解除失败，需重新尝试
3. 引导完成后执行以下步骤：
4. 战斗体状态结束
5. 芯片注销：调用CompTrion.Release(allocated)，占用的Trion返还
6. Trion解冻：设置CompTrion.frozen = false，恢复Trion自然恢复
7. 维持消耗注销：UnregisterDrain
8. **Hediff回滚**：移除所有战斗体期间的Hediff（包括战斗体Hediff），从快照恢复原有Hediff
9. **需求恢复**：重新添加被禁用的需求，恢复到快照记录的值
10. **衣物恢复**：从快照容器恢复穿戴衣物
11. **物品恢复**（🆕 v3）：从快照容器恢复背包物品
12. 进入冷却CD（🆕 v3，详见NR-038）

#### NR-007: 主动解除Trion结算 ✅
- 占用量返还到可用量
- 已消耗的Trion不返还
- 玩家可以较快重新激活（受CD限制）

#### NR-037: 解除引导时间 🆕
- 主动解除与激活类似，有短暂引导时间（具体时长在详细设计阶段确定）
- 引导期间可被打断（受到伤害/被击倒/失去意识等）
- 引导被打断 → 解除失败，仍在战斗体状态，需重新尝试
- 引导不消耗Trion，打断无任何副作用

### 3.2 被动破裂

#### NR-008: 被动破裂触发条件 ✏️（v3明确范围）
以下任一条件满足时触发：
1. Trion可用量耗尽（Available ≤ 0）
2. 头部被击毁（拦截原版死亡结果）
3. 心脏被击毁（拦截原版死亡结果）
4. 玩家主动触发紧急脱离（战术撤退）

#### NR-009: 被动破裂过程 ✏️（v3补充物品恢复+冷却）
1. 战斗体破裂（视觉效果：裂纹扩散→消散）
2. 芯片强制注销：调用CompTrion.ForceDeplete()，占用量随战斗体流失
3. 紧急脱离判定：
   - 如果装备了紧急脱离芯片 且 存在传送目标建筑：传送真身到建筑旁
   - 如果装备了紧急脱离芯片 但 无传送目标建筑：回退到基地附近安全位置（🆕 v3）
   - 如果没有装备紧急脱离芯片：真身在原地释放
4. Trion解冻：设置CompTrion.frozen = false
5. 维持消耗注销：UnregisterDrain + 清除所有战斗体期间注册的drain
6. **Hediff回滚**：同NR-006步骤8
7. **需求恢复**：同NR-006步骤9
8. **衣物恢复**：同NR-006步骤10
9. **物品恢复**（🆕 v3）：同NR-006步骤11
10. **添加枯竭debuff**（🆕 v3，详见NR-040）
11. 进入冷却CD（🆕 v3，详见NR-038）

#### NR-010: 被动破裂Trion结算 ✅
- 所有Trion清零（cur = 0, allocated = 0）
- 需要长时间恢复才能重新激活

```
解除流程（两条路径）：

  ┌───────────────────────────────────────────────────────┐
  │                   战斗体状态                           │
  └───────────┬───────────────────────────┬───────────────┘
              │                           │
     [玩家主动解除]              [触发条件满足]
              │                           │
      [引导开始]                   NR-008四个条件任一
       │      │                           │
    被打断  完成                     [被动破裂]
       │      │                           │
    仍在     [主动解除]              [紧急脱离判定]
    战斗体       │                     │        │        │
              │                  有建筑    无建筑    无芯片
              │                  传送到    基地附近    原地
              │                  建筑旁    安全位置    释放
              │                     │        │        │
              ▼                     ▼        ▼        ▼
  ┌──────────────────────────────────────────────────────┐
  │  [共同恢复流程]                                       │
  │   注销drain → Hediff回滚 → 需求恢复                  │
  │   → 衣物恢复 → 物品恢复 → 进入CD                     │
  └───────────┬───────────────────────────┬──────────────┘
              │                           │
       Trion: 占用量返还            Trion: 全部清零
       无枯竭debuff                 添加枯竭debuff
```

## 四、弱点系统与死亡拦截

### NR-011: 弱点定义 ✅
- **头部**：Trion中继中心（概念层面）
- **心脏**：Trion供给系统（概念层面）

以上两种情况直接触发被动破裂（NR-008）。

### NR-012: 致命伤拦截——弱点被毁 ✏️（v3明确范围）
- 当**头部或心脏**被毁时 → 拦截原版死亡结果 → 触发被动破裂流程（NR-009）
- 真身不会死亡，安全释放

### NR-035: 非弱点器官被毁处理 🆕
战斗体保留完整的人类BodyDef（不做躯体结构替换）。非弱点部位（双腿同时被毁、肾、肝、胃等重要器官）被毁时：
- **不触发原版死亡逻辑**：战斗体是Trion构成的躯体，不依赖这些器官维持运作
- **注册高额Trion流失**：该器官被毁后向CompTrion注册高drain，使Trion快速流失
- **间接导致破裂**：高流失会在短时间内使Trion可用量耗尽，触发NR-008条件1
- 技术实现：Harmony patch死亡判定，战斗体状态下非弱点器官被毁时阻止死亡

```
战斗体死亡拦截决策树：

  [战斗体受到伤害]
        │
  [部位被毁？] ── 否 ──→ [扣Trion + 正常Hediff]
        │ 是
        ▼
  [哪个部位？]
        │
  ┌─────┼──────┬─────────────────────┐
  │     │      │                     │
 头部  心脏  双腿同时          其他重要器官
  │     │      │              （肾、肝、胃等）
  └─────┘      │                     │
     │         └──────────┬──────────┘
     │                    │
  [直接触发          [阻止死亡]
   被动破裂]         [注册高额drain]
   (NR-008/009)      [间接→Trion快速耗尽→破裂]
```

## 五、紧急脱离系统

### NR-013: 紧急脱离芯片 ✅
- 类型：触发芯片（特殊类型）
- 槽位：只能插在触发器的"特殊槽位"，不能插在"左右槽位"
- 效果：战斗体破裂时自动传送真身到安全位置

### NR-014: 传送目标与回退规则 ✏️（v3解决歧义）
**主传送目标**：
- 新增自定义建筑（如"紧急脱离信标"或"战略传送点"）
- 玩家在基地建造此建筑作为传送坐标
- 有多个建筑时传送到最近的一个

**回退规则**（🆕 v3，建筑缺失时）：
- 采用与轨道贸易空投（`DropCellFinder.TradeDropSpot`）同类的多级回退逻辑：
  1. 殖民者建筑附近寻找安全位置
  2. 地图上随机可站立未遮蔽位置
- 确保真身不会释放在极端危险位置

### NR-015: 紧急脱离触发 ✅
- 触发条件：战斗体被动破裂时自动触发
- 传送效果：
  - 真身传送到传送目标（建筑旁或回退位置）
  - 传送过程中真身无敌（不可被攻击）
  - 传送完成后真身出现在目标位置

### NR-016: 紧急脱离Trion消耗 ✅
- 紧急脱离触发时Trion已经清零（被动破裂路径）
- 不需要额外消耗Trion

## 六、战斗体特性

### NR-017: 耐久度统一 ✏️（v3重新定义）
**含义**：所有战斗体在受伤层面完全相同。

- 相同攻击、相同条件下，对不同战斗体的判定结果完全一致
- HP、护甲值、伤口效果、Trion流失速率对所有战斗体一视同仁
- **差异仅体现在Trion池大小**：Trion多的人受同样的伤后还能维持更久，Trion少的人更快破裂
- 不通过Trion量缩放任何战斗体属性

```
耐久度统一示意：

  Trion 1000的战斗员          Trion 100的战斗员
  ┌──────────────┐            ┌──────────────┐
  │ 手臂受50伤   │            │ 手臂受50伤   │
  │ → 手臂被毁 ✓│            │ → 手臂被毁 ✓│ ← 完全相同
  │ → +2 drain/s │            │ → +2 drain/s │ ← 完全相同
  │              │            │              │
  │ 剩余可用:950 │            │ 剩余可用: 50 │ ← 仅此不同
  │ 还能战斗很久 │            │ 即将破裂     │
  └──────────────┘            └──────────────┘
```

### NR-018: 无限体力 ✅
- 战斗体状态下不消耗体力
- 技术实现：`restFallFactor = 0f`（HediffStage原生字段）

### NR-019: 疼痛免疫 ✅
- 战斗体状态下免疫疼痛效果
- 技术实现：`painFactor = 0f`（HediffStage原生字段）

### NR-020: 无法自愈 ✅
- 战斗体受伤后不会自然愈合
- 只能通过解除重建来"治疗"
- 技术实现：`naturalHealingFactor = 0f`（HediffStage原生字段）

### NR-021: 不实现非Trion免疫 ✅
- 战斗体对所有武器都是正常受伤的
- 出于游戏平衡考虑

### NR-022: 不实现缺陷修复 ✏️（v3明确镜像含义）
- 战斗体**镜像真身的身体缺陷**：真身缺一条腿，战斗体也缺腿
- 不是"战斗体全新完美躯体"——是"和真身一样的Trion复制体"
- 出于游戏平衡和实现简洁性考虑
- 简化了快照机制：不需要在激活时补全部位，也不需要在解除时重新移除

### NR-034: 战斗体维持消耗 🆕
- 战斗体存在即持续消耗Trion（"存在税"）
- 定位为**低背景噪声**——单独不构成紧迫压力
- 主要目的：防止玩家长期维持战斗体来规避真身的生理需求
- 战斗中的压力主要来自使用消耗和伤口产生的流失消耗，维持消耗只是底噪
- 技术实现：激活时CompTrion.RegisterDrain("combatBody", X)，解除时UnregisterDrain

## 七、战斗体状态下的限制与行为

### NR-023: Trion无法恢复 ✅
- 战斗体状态下CompTrion.frozen = true
- Trion只减不增，一切消耗不可逆

### NR-024: 无法使用原版治疗 ✅
- 战斗体状态下无法使用医疗床/药物治疗
- 因为伤害不影响真身，治疗无意义

### NR-025: 生理状态冻结 ✅
- **原则**：只冻结生理层面，不冻结心理层面
- 战斗体状态下没有进食/睡眠等**生理**需求
- 心理层面（心情、思想、技能、人际关系）正常运行
- 解除时**生理**状态回滚到激活时的快照
- 心理状态不回滚（记忆/经验保留）
- 详细分类见NR-029~NR-033

### NR-036: 击倒状态处理 🆕
战斗体状态下Pawn被击倒（Downed）时的行为：

| 击倒原因 | 处理方式 |
|---------|---------|
| **双腿同时被毁** | 注册高额Trion流失（同NR-035），很快因Trion耗尽而破裂 |
| **意识降低至倒地阈值** | 保持战斗体状态倒地，等待事件自然发展 |

以上两种情况Pawn均倒地无法行动。
- 战斗体**持续存在**，维持消耗继续
- Pawn无法行动，无法执行解除操作
- 最终Trion耗尽 → 被动破裂 → 真身释放（走紧急脱离或原地释放）
- 或等待友方救助

> 注：由于painFactor=0（NR-019），疼痛不会导致倒地。意识降低主要来自头部受损等极端情况。

### NR-039: 战斗体征召无限制 🆕
- 战斗体状态下**不强制征召**，玩家可自由控制
- 取消征召后Pawn执行正常殖民者AI行为（搬运、建造等）
- 生理需求已禁用，不会产生进食/睡觉的AI行为
- 战斗体维持消耗持续，Trion是天然的时间约束

## 八、真身状态快照系统

### NR-026: 快照内容 ✅
快照记录以下数据：

| 记录项 | 说明 | 回滚方式 |
|---|---|---|

### NR-027: Hediff替换策略 ✅
- **激活时**：记录快照 → 移除所有现有Hediff → 添加"战斗体Hediff"
- **解除时**：移除所有战斗体期间的Hediff（包括战斗体Hediff本身）→ 从快照恢复原有Hediff
- 两者无缝衔接
- "战斗体Hediff"是一个特殊Hediff，通过HediffStage原生字段实现大部分战斗体特性

### NR-028: 衣物、物品与外观 ✏️（v3解决物品处理）
**衣物**：
- 激活时：从Pawn身上移除，转移至快照容器
- 解除时：从快照容器恢复穿戴

**物品（Inventory）**（🆕 v3已确认）：
- 激活时：从Pawn_InventoryTracker.innerContainer转移至快照容器
- 解除时：从快照容器转移回innerContainer
- 物品物理转移后，游戏引擎天然不会让AI使用不存在的物品，无需额外拦截

**外观**：
- 战斗体应有独特视觉外观（MVP可先用简单指示器，后续版本实现贴图替换）

**武器不是问题**：激活战斗体的前提是手持触发体

## 九、生理/心理分类详细需求

### NR-029: 禁用的生理需求 ✅
以下需求在战斗体状态下通过战斗体Hediff的`disablesNeeds`禁用：
- 饥饿（Food）
- 休息（Rest）
- 舒适度（Comfort）

**效果**：需求从Pawn的需求列表中移除，相关ThoughtWorker返回Inactive（不产生正面或负面心情影响）

### NR-030: 不禁用的心理需求 ✏️（v3最终确认）
以下需求在战斗体状态下正常运行：
- 心情（Mood）：正常计算
- 思想/记忆（Thoughts）：战斗经历正常产生思想
- 技能经验（Skills）：战斗经验正常增长
- 人际关系（Relations）：战斗互动正常影响关系
- **娱乐（Joy）**：正常运行（✅ v3确认，心理层面）
- **美观（Beauty）**：正常运行（✅ v3确认，心理层面）
- **户外（Outdoors）**：正常运行（✅ v3确认，心理层面）

> 设计意图：Joy/Beauty/Outdoors归为心理层面不冻结，为战斗体增加隐性精神压力——长时间维持战斗体会导致心情下降。配合blocksMentalBreaks（NR-031），心情可以跌到谷底但不会崩溃，解除后才面临崩溃风险。

### NR-031: 精神崩溃阻止 ✅（v3确认）
- 战斗体状态下**不冻结心情下降**
- 但**阻止触发精神崩溃**
- 技术实现：HediffStage的`blocksMentalBreaks = true`（纯XML，零C#）

### NR-032: 伤害机制映射 ✅
战斗体的伤害不走原版的出血/疼痛逻辑，而是映射为Trion消耗：

| 原版概念 | 战斗体映射 | 技术实现 |
|---|---|---|
| 掉血（HP减少） | 扣Trion可用值 | CompTrion.Consume() |
| 流血（持续出血） | Trion泄漏（维持消耗速率增加） | CompTrion.RegisterDrain() |
| 部位被毁 | 扣Trion可用值 + 部位被毁 | Consume() + Hediff |
| 出血（视觉） | 禁止 | totalBleedFactor = 0 |
| 疼痛 | 禁止 | painFactor = 0 |

### NR-033: 环境免疫 ✏️（v3确认exploit接受度）
战斗体状态下免疫以下环境效果：
- 体温影响（不受极端温度伤害）
- 中毒（免疫毒素）
- 疾病（不会感染新疾病，已有疾病进度冻结于快照）

**关于疾病冻结的exploit**（✅ v3确认）：
- 接受疾病进度冻结作为战斗体的附带好处
- Trion维持消耗+战斗消耗+冷却CD是天然时间约束，exploit空间有限
- 世界观自洽——真身在快照中，疾病没理由继续恶化

## 十、冷却与惩罚机制

### NR-038: 激活/解除冷却CD 🆕
- 解除战斗体后进入冷却期，期间不可重新激活
- 主动解除和被动破裂均触发CD
- 主要目的：防止玩家频繁激活/解除来规避真身风险
- 冷却时长为设计变量，具体数值在详细设计阶段确定
- 被动破裂的CD应长于主动解除的CD（惩罚破裂）

### NR-040: 被动破裂枯竭debuff 🆕
- 被动破裂后真身获得"Trion枯竭"debuff（Hediff）
- 具体效果在详细设计阶段确定（可能包含虚弱、移动减速、工作效率降低等）
- 主动解除**不产生**枯竭debuff
- 进一步拉大两条解除路径的惩罚差异

### 设计原则总结

1. **有选择性的机制移植**：不100%还原原作，为游戏服务
2. **利用现有机制**：HediffStage原生字段、DropCellFinder回退逻辑
3. **增加战术深度**：引导时间、两条路径、冷却CD
4. **保持游戏平衡**：不实现非Trion免疫、不修复缺陷、维持消耗+CD防滥用
5. **生理/心理分离**：生理冻结回滚，心理正常运行保留
6. **XML优先**：能通过HediffStage配置的特性优先XML
7. **最简原则**：保留原版BodyDef拦截致死逻辑，不做高风险结构替换
