# 2.5 Trion战斗系统流程

## 前置说明

本文档整合以下核心概念，展示它们在战斗中的协作关系：
- **Trion资源池**：能量的存储、占用与消耗机制
- **触发器三层架构**：触发体、触发芯片、Trion表现
- **战斗体**：意识载体与伤害隔离机制

## 第一章：核心概念关系图

### 1.1 三角协作关系

```
┌─────────────────────────────────────────────────────────┐
│                    Trion（生物能量）                      │
│              具有能量-物质二象性，可被"编程"转化           │
└─────────────────────────────────────────────────────────┘
        ↑ 产生                    ↓ 通过媒介转化
┌───────────────┐         ┌─────────────────────────────┐
│  Trion腺体    │         │      触发器（转化媒介）        │
│  （生产者）   │         │  ┌─────────────────────────┐ │
│               │         │  │ 触发体（物理载体）       │ │
│               │         │  │ └→ 触发芯片（功能配置） │ │
│               │         │  │    └→ Trion表现（物化） │ │
│               │         │  └─────────────────────────┘ │
└───────────────┘         └─────────────────────────────┘
                                      ↓ 输出
                    ┌─────────────────────────────────────┐
                    │           战斗体（意识载体）          │
                    │  - 完全由Trion构成的替代躯体          │
                    │  - 真身收容在触发体内                 │
                    │  - 伤害不回传真身                     │
                    └─────────────────────────────────────┘
```

### 1.2 系统协作概览

| 系统 | 核心职责 | 与Trion的关系 |
|------|---------|--------------|
| **Trion资源池** | 能量存储与分配 | 是能量本身 |
| **触发器系统** | 能量转化与控制 | 消耗Trion实现功能 |
| **战斗体系统** | 意识载体与保护 | 由Trion构成并维持 |

## 第二章：Trion资源池机制

### 2.1 核心公式

```
Trion可用量 = Trion总量 - 占用量 - 累计消耗量
```

**关键理解**：Trion是一个资源池，可视为**血量+蓝量+战术分配**的综合体。

### 2.2 两种状态对比

| 状态 | Trion用途 | 恢复能力 | 类比 |
|------|----------|---------|------|
| **日常状态** | 全部可用于释放能力、转化燃料等 | 可自然恢复 | 纯"蓝量" |
| **战斗体状态** | 部分被芯片占用，剩余为战斗可用 | **无法恢复** | 血量+蓝量+战术分配 |

> **关键差异**：战斗体状态下，Trion器官无法恢复能量。一切消耗都从可用量中扣除，直到耗尽。

### 2.3 消耗类型分类

| 类型 | 触发时机 | 示例 | 特点 |
|------|---------|------|------|
| **占用消耗** | 生成战斗体时锁定 | 触发芯片注册 | 锁定不可用，解除后释放 |
| **维持消耗** | 持续消耗 | 战斗体维持、变色龙激活 | 时间累积 |
| **使用消耗** | 动作触发 | 射击、技能释放 | 单次扣除 |
| **流失消耗** | 受伤触发 | 伤口泄漏 | 约1分钟后自动停止 |

### 2.4 Trion总量影响因素

| 因素 | 影响程度 | 说明 |
|------|---------|------|
| 个体天生差异 | **最大** | 决定基础上限 |
| 黑触发器使用 | 临时扩增 | 仅战斗体状态下，优先被消耗 |
| 成年前锻炼 | 少量 | 可略微提升 |
| 植入体（触发角） | 中量 | 近界民常用 |

## 第三章：触发器三层架构在战斗中的角色

### 3.1 三层架构总览

| 层级 | 名称 | 战斗中的角色 | Trion关系 |
|------|------|-------------|----------|
| 物理载体层 | **触发体** | 承载芯片、收容真身、提供内置系统 | 不直接消耗 |
| 功能配置层 | **触发芯片** | 定义可用能力，注册时占用Trion | **占用**Trion |
| 能量表现层 | **Trion表现** | 实际战斗工具（武器、护盾等） | **消耗**Trion |

### 3.2 触发体（物理载体层）

**定义**：无刃柄状的物理硬件平台，是整个系统的物理基础。

**8槽系统**：

| 侧面 | 槽位数 | 用途 | 切换规则 |
|------|--------|------|----------|
| 主触发（利手侧） | 4槽 | 主要武器和攻击能力 | 同侧同时仅能激活1个 |
| 副触发（另一侧） | 4槽 | 副武器、防御、辅助 | 两侧可同时各激活1个 |

**同时激活上限**：主副各1个，共2个。

**内置标配（不占槽位）**：

| 功能 | 说明 |
|------|------|
| 紧急脱离 | 战斗体失效时自动传送真身到安全位置 |
| 战斗体收容 | 存储和释放Trion战斗体的空间 |
| 雷达 | 探测周边敌人和队友位置 |
| 通讯 | 与本部和队友的实时通信 |

### 3.3 触发芯片（功能配置层）

**定义**：小型芯片状的配置模块，决定Trion能量以何种形式表现。

**三大功能分类**：

| 类型 | 功能定位 | 代表芯片 |
|------|----------|----------|
| 攻击触发器 | 生成武器和直接攻击能力 | 弧月、蝎子、小行星、鹭 |
| 防御触发器 | 生成防护能力 | 护盾、护城盾 |
| 选项触发器 | 辅助和战术支援 | 蜘蛛、蓑衣虫、蚱蜢 |

**Trion占用机制**：
- 芯片在战斗体生成时注册，锁定部分Trion作为占用量
- 占用量在战斗中不可使用，但解除战斗体后释放

### 3.4 Trion表现（能量表现层）

**定义**：芯片被激活后，Trion能量转化而成的具体武器、能力或效果。

**表现形式**：
- **武器实体**：弧月剑、能量步枪、能量护盾等
- **子弹类型**：小行星（直线）、猎犬（追踪）、毒蛇（可控弹道）等
- **特殊效果**：能量丝线、光学迷彩、假人投影等

**本质理解**：
```
芯片 = 程序代码
Trion能量 = 运行环境
Trion表现 = 程序输出
```

## 第四章：战斗体生命周期

### 4.1 阶段1：激活（Trigger On）

**触发条件**：
- 持有触发体
- 足够Trion
- 明确激活意图（语音指令非必须）

**执行过程**：

```
1. 真身数据快照
   └→ 记录当前生理状态、装备物品

2. 真身收容
   └→ 移动到触发体内的近界空间
   └→ 进入完全停滞状态（不老化、不受伤、不消耗能量）

3. 触发芯片注册
   └→ 锁定占用Trion
   └→ 芯片配置在战斗中固定，无法更换

4. 战斗体生成
   └→ Trion能量物质化构建躯体
   └→ 意识从真身转移到战斗体
```

**结果状态**：进入战斗体状态，芯片配置锁定，Trion无法自然恢复。

### 4.2 阶段2：战斗过程

#### 芯片操作规则

| 操作 | 说明 | Trion影响 |
|------|------|----------|
| **激活** | 需要引导时间 | 消耗激活费用 |
| **切换** | 旧芯片关闭+旧表现消散 → 新芯片引导（空窗期）→ 新表现生成 | 切换本身无额外消耗 |
| **同侧互斥** | 同一侧同时只能激活1个 | - |

#### Trion消耗来源

| 来源 | 说明 | 可控性 |
|------|------|--------|
| 维持消耗 | 战斗体存在的基础消耗 | 不可避免 |
| 激活消耗 | 芯片每次激活的固定消耗 | 不可避免 |
| 使用消耗 | 武器/技能使用 | 可控制使用频率 |
| 流失消耗 | 受伤导致的额外流失 | 约1分钟后自动停止 |

#### 伤害机制

**伤害表现**：
- 伤害导致**Trion气态泄漏**
- 无血液、无肉体伤口
- 伤口呈现为Trion能量的流失点

**关键特性**：
- 战斗体对**非Trion攻击几乎免疫**（坦克炮弹、导弹等无效）
- 只有**Trion武器**才能造成实质伤害
- 战斗体**无法自愈**，只能解除重建

**双核心弱点**：

| 核心 | 位置 | 功能 | 被破坏后果 |
|------|------|------|-----------|
| Trion中继中心 | 脑部 | 传输控制信号 | 战斗体立即失效 |
| Trion供给系统 | 心脏处 | 调节Trion流动 | 战斗体立即失效 |

### 4.3 阶段3：解除

#### 解除类型对比

| 解除类型 | 触发条件 | 战斗体处理 | 真身状态 |
|---------|---------|-----------|---------|
| **主动解除** | 使用者意图 | 消散 | 安全释放 |
| **被动解除** | Trion耗尽/供给系统破坏/中继系统切断 | 破裂消散 | 原地释放（可能危险） |
| **紧急脱离** | 自动检测触发条件 | 破裂消散 | 传送到安全位置 |

#### 紧急脱离详解

**自动触发条件（4种）**：
1. 战斗体极限超过（Combat Body Limit Exceeded）
2. Trion极限超过（Trion Limit Exceeded）
3. 供给系统被破坏（Trion Supply System Destroyed）
4. 中继系统被切断（Relay System Severed）

**传送范围限制**：
- 三门市内：保证有效
- 远征任务：距离母舰**3km内**有效

**使用权限**：
- ✓ A/B级队员
- ✗ C级训练生（资源限制）
- ✗ S级队员（黑触发器不兼容）

### 4.4 阶段4：状态回滚

#### 真身回滚

```
真身状态 = 收容时的快照状态
         ↓
- 战斗体伤害完全不继承
- 生理状态回滚到收容时
- 心理状态不回滚（记忆保留）
```

> **核心战略价值**：这是战斗体系统的核心意义——实现"无伤战斗"。

#### Trion结算

| 解除方式 | Trion处理 | 后续状态 |
|---------|----------|---------|
| 主动解除 | 占用Trion释放，可用量恢复 | 可较快重新激活 |
| 被动解除 | 所有Trion消耗殆尽 | 进入枯竭状态，需长时间恢复 |

#### 重建时间

- 重建时间与Trion容量成正比
- 低Trion用户（如三云修）需要**1-2小时**
- 高Trion用户重建更快

## 第五章：通用战斗流程示例

### 5.1 完整流程阶段表

| 阶段 | 事件 | Trion变化 | 系统状态 |
|------|------|----------|---------|
| **准备** | 装备触发体 | 无变化 | 芯片预装载，未激活 |
| **激活** | Trigger On | 总量 → 占用 + 可用 | 真身收容，战斗体生成 |
| **战斗** | 激活芯片 | 可用 - 激活费 | 引导时间内无武器可用（空窗期） |
| **战斗** | 使用武器 | 可用 - 使用费 | 射击、挥砍等 |
| **战斗** | 受伤 | 可用 - 流失费/单位时间 | 约1分钟后自动停止 |
| **解除** | 主动/被动 | 占用 → 释放或消耗 | 真身回滚 |

### 5.2 Trion流转示意

```
战斗开始前：
┌─────────────────────────────────────┐
│  Trion总量 = 100%                    │
│  ████████████████████████████████   │
│  全部可用于日常活动                   │
└─────────────────────────────────────┘

Trigger On后：
┌─────────────────────────────────────┐
│  占用量(30%)  │  可用量(70%)         │
│  ██████████  │  ██████████████████  │
│  芯片注册锁定 │  战斗可用            │
└─────────────────────────────────────┘

战斗中（使用+受伤）：
┌─────────────────────────────────────┐
│  占用(30%)  │  已消耗(25%) │ 剩余(45%) │
│  ██████████ │  ░░░░░░░░░  │ ██████████│
│  锁定       │  不可恢复    │ 当前可用  │
└─────────────────────────────────────┘

主动解除后：
┌─────────────────────────────────────┐
│  已消耗(25%)  │  可恢复(75%)         │
│  ░░░░░░░░░░  │  ████████████████████│
│  需时间恢复   │  占用量已释放         │
└─────────────────────────────────────┘

被动解除（Trion可用值耗尽 / 弱点器官被毁 / 主动触发紧急脱离）后：
┌─────────────────────────────────────┐
│  已消耗(100%)                       │
│  ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░     │
│  需时间恢复     占用量流失消耗       │
└─────────────────────────────────────┘
```

### 5.3 关键决策点

| 决策点 | 考量因素 | 建议 |
|--------|---------|------|
| **何时激活** | 威胁等级、Trion储备 | 确认威胁后再激活 |
| **芯片配置** | 预期战斗类型、队伍配合 | 战前规划，无法中途更换 |
| **何时切换芯片** | 战术需要、引导时间 | 预判敌人动作 |
| **何时撤退** | 剩余Trion、伤害累积 | 低于30%考虑撤退 |
| **主动vs紧急脱离** | 安全环境、战斗体状态 | 安全时主动解除保留战斗体 |

### Trion消耗速查

| 消耗类型 | 触发时机 | 可控性 | 停止条件 |
|---------|---------|--------|---------|
| 占用 | 激活时 | 配置决定 | 解除时释放 |
| 维持 | 持续 | 不可避免 | 解除时停止 |
| 使用 | 动作触发 | 可控 | 停止动作 |
| 流失 | 受伤 | 被动 | 约1分钟自动停止 |

### 解除方式速查

| 方式 | 触发 | 战斗体 | 真身位置 | Trion |
|------|------|--------|---------|-------|
| 主动 | 意图 | 消散 | 原地 | 占用释放 |
| 被动 | 耗尽/破坏/主动触发紧急脱离 | 消散 | 原地（无配置紧急脱离）/ 传送（有配置紧急脱离） | 全部消耗 |
