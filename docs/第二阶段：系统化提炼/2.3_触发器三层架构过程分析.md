## 阶段1：配置——战前准备（触发体 ⊃ 触发芯片）

> 战斗尚未开始。队员在作战室中，手持触发体，进行芯片配置。这是三层架构中**第一层（触发体）与第二层（触发芯片）**建立关系的阶段。

### 1.1 触发体的物理状态

触发体（トリガーホルダー / Trigger Holder）是一个**无刃柄状的物理硬件平台**。此时它处于未激活状态：

| 属性 | 状态 |
|------|------|
| 外观 | 无刃柄状装置，可手持 |
| 芯片槽位 | 8个槽位可用（主4+副4） |
| 内置系统 | 紧急脱离/雷达/通讯/战斗体收容，均处于待机 |
| 战斗体存储空间 | 内部存储着预构建的Trion战斗体 |
| Trion消耗 | 无——未激活状态不消耗任何Trion |

> **原因说明**：触发体本身是纯物理硬件，不消耗Trion。只有激活后，内置系统和芯片才开始消耗能量。这类似于一台关机状态的电脑——硬件存在但不运行。

### 1.2 芯片装载过程

队员将触发芯片（小型芯片状配置模块）逐一装载到触发体的槽位中：

```
触发体 8槽配置系统：

┌─────────────────────────────────────────────────┐
│                  触发体（物理硬件）                │
│                                                   │
│  ┌─ 主触发（利手侧）──┐  ┌─ 副触发（另一侧）──┐ │
│  │ 槽1: [芯片A]       │  │ 槽5: [芯片E]       │ │
│  │ 槽2: [芯片B]       │  │ 槽6: [芯片F]       │ │
│  │ 槽3: [芯片C]       │  │ 槽7: [芯片G]       │ │
│  │ 槽4: [芯片D]       │  │ 槽8: [芯片H]       │ │
│  └────────────────────┘  └────────────────────┘ │
│                                                   │
│  ┌─ 内置系统（不占槽位）─────────────────────┐   │
│  │ · 紧急脱离  · 雷达  · 通讯  · 战斗体收容  │   │
│  └───────────────────────────────────────────┘   │
└─────────────────────────────────────────────────┘
```

**装载的关键约束**：
- **战前配置，战中不可更换**：芯片必须在战斗前装载完毕，一旦激活触发器进入战斗体状态，芯片配置即锁定
- **只能在已装载的芯片间切换**：战斗中可以切换激活哪个芯片，但无法装载新芯片或卸载已有芯片

> **原因说明**：这个约束的本质是——芯片在激活时会向Trion资源池"注册"并锁定占用量。战斗中更换芯片意味着重新分配Trion占用，这在战斗体状态下是不可能的（Trion器官无法恢复，资源池是固定的）。

### 1.3 配置策略——不同职位的典型思路

芯片配置直接决定了队员在战斗中的能力集合。不同战斗职位有不同的配置方向：

| 职位 | 主触发（利手侧）典型配置 | 副触发（另一侧）典型配置 | 配置思路 |
|------|------------------------|------------------------|---------|
| **攻击手** | 弧月+护盾 | 旋空+护盾+蚱蜢/蓑衣虫 | 弧月与旋空分两侧以便同时激活组合使用，两侧护盾保证攻防兼顾 |
| **枪手** | 突击步枪（小行星+猎犬）+护盾 | 手枪（流星）+护盾+蓑衣虫 | 枪型芯片自带弹种，一个芯片=形+弹一体，两侧护盾保证攻防兼顾 |
| **射手** | 子弹芯片A（小行星）+护盾 | 子弹芯片B（毒蛇）+护盾+蓑衣虫 | A和B分两侧以便同时激活创建合成弹，两侧护盾保证攻防兼顾 |
| **狙击手** | 鹭/闪电/朱鹮+护盾 | 蓑衣虫（必备）+护盾+蚱蜢 | 主侧狙击武器，副侧隐蔽+机动，两侧护盾保证攻防兼顾 |
| **全能手** | 近战武器+护盾 | 远程武器+护盾+辅助 | 两侧各有护盾，无论激活哪侧攻击，另一侧都能展开护盾防御 |

> **配置核心原则**：
> 1. **两侧护盾原则**：大多数队员会在**两侧都配置护盾芯片**——无论用哪侧武器攻击，另一侧都能激活护盾兼顾防御。只有刻意做双持攻击或组合技能时才舍弃某侧护盾。
> 2. **组合分侧原则**：有组合关系的芯片（如弧月+旋空、子弹A+子弹B用于合成弹）必须分配在**不同侧**，因为同侧同时只能激活1个芯片，放在同侧就无法同时使用。

> **原因说明**：配置策略的核心权衡是——8个槽位是有限的，每多装一个攻击芯片就少一个防御/辅助槽位。而且每个芯片都会在激活时占用Trion，装载过多高消耗芯片会压缩战斗可用Trion。

### 1.4 内置系统的存在

触发体除了8个可配置槽位外，还内置了4个标准系统，**不占用槽位**：

| 内置系统 | 功能 | 说明 |
|---------|------|------|
| **紧急脱离** | 战斗体失效时自动传送真身到安全位置 | 所有触发器中Trion消耗最大的系统 |
| **雷达** | 探测周边敌人和队友位置 | 可被蓑衣虫屏蔽 |
| **通讯** | 与本部和队友的实时通信 | 有距离限制 |
| **战斗体收容** | 存储和释放Trion战斗体的空间 | 激活时真身与战斗体互换 |

> **原因说明**：内置系统本质上也是触发芯片，但作为B级以上队员的默认标配，被固化在触发体中不占用可配置槽位。这保证了每个队员都有基础的生存保障（紧急脱离）和态势感知（雷达/通讯），不需要牺牲战斗配置来换取这些基础能力。

### 1.5 阶段1小结：三层架构的状态

```
阶段1结束时的三层状态：

第一层（触发体）：已装载芯片，待机状态
  └→ 8槽已配置完毕
  └→ 内置系统待机
  └→ 战斗体存储在内部空间

第二层（触发芯片）：已装载，未注册
  └→ 物理上已安装在触发体中
  └→ 尚未向Trion资源池注册
  └→ 不消耗任何Trion

第三层（Trion表现）：不存在
  └→ 芯片未激活，无任何Trion表现生成

Trion资源池：100%可用
  └→ 全部Trion储存在Trion腺体中
  └→ 无任何占用或消耗
```

## 阶段2：激活——从芯片到Trion占用（触发芯片 → Trion资源池）

> 队员发出激活意图（"Trigger On"），触发体启动。这是三层架构中**第二层（触发芯片）向Trion资源池注册**的阶段，也是第一层（触发体）执行真身收容和战斗体释放的阶段。

### 2.1 激活瞬间发生的事

激活是一个**瞬间且同时发生**的复合事件：

```
激活瞬间的事件序列（同时发生）：

1. 触发体执行真身收容
   └→ 真身移入触发体内的近界空间
   └→ 进入完全停滞状态

2. 触发体释放战斗体
   └→ Trion能量物质化，构建战斗体
   └→ 意识从真身转移到战斗体

3. 触发芯片向Trion资源池注册  ← 本阶段核心
   └→ 每个已装载的芯片锁定各自的Trion占用量
   └→ 占用量从可用Trion中扣除

4. 内置系统启动
   └→ 雷达、通讯开始工作
   └→ 紧急脱离进入监控状态
```

> **原因说明**：事件1和2属于"真身↔战斗体"的状态切换（详见"2.2_真身-战斗体-真身领域过程分析"），本文档聚焦事件3——芯片注册过程。

### 2.2 芯片注册的本质

芯片注册是触发芯片从"物理存在"变为"逻辑激活"的关键步骤：

| 注册前 | 注册后 |
|--------|--------|
| 芯片物理安装在触发体中 | 芯片在Trion资源池中占有"份额" |
| 不消耗Trion | 锁定一部分Trion作为占用量 |
| 无法被激活使用 | 可以在战斗中被激活生成Trion表现 |
| 可以被更换 | 配置锁定，不可更换 |

**类比理解**：
```
芯片注册 ≈ 程序加载到内存
  芯片 = 程序文件（存储在硬盘上）
  注册 = 加载到内存（占用内存空间）
  Trion占用量 = 程序占用的内存
  激活芯片 = 运行程序（消耗CPU资源）
```

### 2.3 占用量的计算逻辑

不同芯片占用不同量的Trion。占用量在注册时一次性锁定：

| 芯片类型 | 占用量特点 | 原因 |
|---------|-----------|------|
| 近战武器（弧月、蝎子等） | 较低 | 近战武器Trion消耗低是其核心优势 |
| 枪型触发器 | 中等 | 需要维持枪械实体 |
| 狙击武器（鹭、朱鹮等） | 较高 | 高威力武器需要更多Trion支撑 |
| 防御触发器（护盾等） | 中等 | 需要维持能量屏障 |
| 选项触发器 | 各异 | 蜘蛛低消耗，蓑衣虫持续消耗 |
| 内置系统（紧急脱离等） | 固定 | 作为标配，占用量固定 |

> **原因说明**：占用量的本质是——芯片注册后，一部分Trion**正在被芯片使用**，用于维持芯片的就绪状态。这就像程序加载到内存：每个程序（芯片）加载后会占用一部分内存（Trion），这些内存正在被程序使用以维持其运行就绪状态。占用量（程序占用的内存）在注册时锁定，芯片每次激活时的"激活费用"则是额外的运行开销——类似程序启动一个新任务时需要额外的CPU和内存资源，从可用量中扣除。

### 2.4 激活后的Trion资源池状态

```
激活前：
┌─────────────────────────────────────┐
│  Trion总量 = 100%                    │
│  ████████████████████████████████   │
│  全部可用                            │
└─────────────────────────────────────┘

激活后（芯片注册完成）：
┌─────────────────────────────────────┐
│  占用量(芯片注册)  │  可用量(战斗用)  │
│  ██████████████  │  ████████████████│
│  锁定不可使用     │  战斗中可消耗     │
└─────────────────────────────────────┘

占用量的构成：
┌─────────────────────────────────────┐
│ 芯片1 │ 芯片2 │ ... │ 芯片8 │ 内置 │
│ 占用  │ 占用  │     │ 占用  │ 系统 │
│       │       │     │       │ 占用 │
└─────────────────────────────────────┘
```

**关键约束**：
- 占用量在战斗中**不可使用**，但主动解除时会**释放返还**
- 被动破裂时占用量**随战斗体一同流失耗尽**
- Trion器官在战斗体状态下**无法恢复Trion**——一切消耗不可逆

### 2.5 阶段2小结：三层架构的状态

```
阶段2结束时的三层状态：

第一层（触发体）：已激活
  └→ 真身已收容
  └→ 战斗体已释放
  └→ 内置系统运行中

第二层（触发芯片）：已注册，待激活
  └→ 所有芯片已向Trion资源池注册
  └→ 各自锁定了占用量
  └→ 但尚未有芯片被"激活"生成Trion表现
  └→ 配置锁定，不可更换

第三层（Trion表现）：不存在
  └→ 虽然芯片已注册，但尚未激活任何芯片
  └→ 队员此时手中没有武器

Trion资源池：占用 + 可用
  └→ 占用量被芯片注册锁定
  └→ 可用量为战斗可消耗部分
  └→ 无法恢复
```

## 阶段3：战斗——芯片激活与Trion表现（触发芯片 → Trion表现）

> 战斗开始。队员需要武器和能力来战斗。这是三层架构中**第二层（触发芯片）激活生成第三层（Trion表现）**的阶段——三层架构的核心运作阶段。

### 3.1 芯片激活过程

队员选择一个已注册的芯片并激活它，芯片将Trion能量转化为具体的武器或效果：

```
芯片激活流程：

队员意图："激活弧月"
       ↓
┌─ 引导阶段 ─────────────────────────────┐
│  芯片开始引导（需要引导时间）            │
│  消耗：激活费用（从可用Trion中扣除）     │
│  引导期间队员无武器可用（空窗期）        │
└────────────────────────────────────────┘
       ↓ 引导完成
┌─ 生成阶段 ─────────────────────────────┐
│  Trion能量按芯片定义的"程序"物质化       │
│  生成对应的Trion表现（武器/护盾/效果）   │
│  队员可以开始使用                        │
└────────────────────────────────────────┘
```

**引导时间**：
- 每个芯片激活都需要引导时间
- 引导时间长短取决于芯片类型和使用者熟练度
- 引导期间队员处于**无武器的空窗期**，是最脆弱的时刻

> **原因说明**：引导时间造成的空窗期使得芯片激活和切换成为有风险的战术决策——选择在什么时机切换、如何利用队友掩护度过空窗期，是战斗中的关键判断。

### 3.2 Trion表现的三种形态

芯片激活后生成的Trion表现，根据芯片类型呈现为三种形态：

| 形态 | 说明 | 代表芯片 |
|------|------|---------|
| **武器实体** | 持续存在的物质化武器，可握持、挥砍、格挡 | 弧月（剑）、蝎子（变形刃）、光魂（重剑/盾） |
| **子弹/投射物** | 一次性消耗的能量投射物，发射后消散 | 小行星（直线弹）、猎犬（追踪弹）、流星（炸裂弹）、毒蛇（可控弹道） |
| **特殊效果** | 持续或一次性的非武器效果 | 护盾（能量屏障）、蜘蛛（金属线）、蓑衣虫（雷达隐身）、变色龙（光学隐身） |

**本质理解**：
```
芯片 = 程序代码（定义"做什么"）
Trion能量 = 运行环境（提供"用什么做"）
Trion表现 = 程序输出（实际"做出来的东西"）
```

### 3.3 芯片切换机制

战斗中，队员可以在已装载的芯片间切换。切换遵循以下规则：

**同侧互斥规则**：
- 同一侧（主触发或副触发）同时只能激活**1个**芯片
- 两侧可以同时各激活1个，共**最多2个**

**切换过程**：

```
切换流程（同侧）：

当前状态：主侧激活弧月（剑在手中）
队员意图："切换到蝎子"
       ↓
┌─ 关闭阶段 ─────────────────────────────┐
│  旧芯片（弧月）关闭                      │
│  旧Trion表现消散（弧月剑消失）           │
│  队员此时手中无武器                      │
└────────────────────────────────────────┘
       ↓
┌─ 引导阶段（空窗期）───────────────────┐
│  新芯片（蝎子）开始引导                  │
│  引导时间取决于熟练度                     │
│  队员处于无武器的脆弱状态                │
└────────────────────────────────────────┘
       ↓ 引导完成
┌─ 生成阶段 ─────────────────────────────┐
│  新Trion表现生成（蝎子刃出现）           │
│  队员恢复战斗能力                        │
└────────────────────────────────────────┘
```

> **原因说明**：切换过程中存在**空窗期**——旧武器消散后、新武器生成前，队员处于无武器状态。这使得切换时机的选择成为关键的战术判断：需要在安全的间隙切换，或依靠队友掩护度过空窗期。空窗期的长短取决于新芯片的引导时间和使用者的熟练度。

### 3.4 主副同时使用——双持战斗

两侧可以同时各激活一个芯片，实现双持战斗：

```
双持状态示例：

┌─ 主触发（右手）─────┐  ┌─ 副触发（左手）─────┐
│ 激活：弧月          │  │ 激活：护盾          │
│ 表现：右手持剑      │  │ 表现：左手展开护盾  │
│ 消耗：无(普通挥砍)  │  │ 消耗：使用费(抵挡时) │
└────────────────────┘  └────────────────────┘

双持组合示例：
· 弧月 + 护盾 = 攻防兼备（攻击手标配）
· 弧月 + 弧月 = 双刀流（纯攻击型）
· 蝎子 + 蝎子 = 双变形刃（奇袭型）
· 枪型 + 护盾 = 射击+防御（枪手标配）
· 鹭 + 鹭 = 双狙击（Twin Snipe，极少数人使用）
```

### 3.5 Trion消耗的四种来源

战斗中，Trion从可用量中持续消耗，来源有四种：

| 消耗类型 | 触发时机 | 可控性 | 停止条件 |
|---------|---------|--------|---------|
| **维持消耗** | 持续（主要来自战斗体本身+少数持续运行芯片） | 不可避免 | 解除战斗体时停止 |
| **激活消耗** | 每次激活芯片时 | 可控（减少切换次数） | 激活完成后停止 |
| **使用消耗** | 每次射击/技能释放/护盾抵挡攻击 | 可控（控制使用频率） | 停止动作后停止 |
| **流失消耗** | 受伤时 | 被动（不可控） | 约1分钟后自动停止 |

```
战斗中Trion资源池的变化：

战斗开始：
┌─────────────────────────────────────┐
│  占用(30%)  │  可用(70%)            │
│  ██████████ │  ██████████████████████│
└─────────────────────────────────────┘

激活芯片后：
┌─────────────────────────────────────┐
│  占用(30%)  │ 激活费(2%) │ 可用(68%) │
│  ██████████ │ ░░░░░░░░ │ ████████████│
└─────────────────────────────────────┘

战斗中（使用+受伤）：
┌─────────────────────────────────────┐
│  占用(30%)  │  已消耗(25%) │ 剩余(45%) │
│  ██████████ │  ░░░░░░░░░  │ ██████████│
│  锁定       │  不可恢复    │ 当前可用  │
└─────────────────────────────────────┘
```

> **原因说明**：四种消耗中，维持消耗是"存在税"（主要来自战斗体本身的存在成本，大多数芯片没有维持消耗），激活消耗是"启动税"（激活芯片的一次性开销），使用消耗是"行动税"（射击、技能释放、护盾抵挡攻击等具体行为的代价——注意近战武器普通挥砍不消耗Trion），流失消耗是"受伤税"（被打就要付出的代价）。队员需要在这四种消耗之间权衡，决定何时进攻、何时防御、何时撤退。

### 3.6 触发芯片的"形+弹"一体化设计

远程类触发芯片（枪手/射手/狙击手）都遵循**"形+弹"一体化**的设计——每个芯片同时定义了Trion表现的"形体"和"弹种"，是一个不可拆分的整体：

```
触发芯片的"形+弹"一体化：

枪手芯片：形（某种枪械）+ 弹（子弹类型）= 一个芯片
  例：突击步枪（小行星+猎犬）= 1个芯片，可切换2种弹药
  例：手枪（流星）= 1个芯片，1种弹药

射手芯片：形（Trion立方体）+ 弹（子弹类型）= 一个芯片
  例：小行星芯片 = 形:Trion立方体 + 弹:小行星
  例：毒蛇芯片 = 形:Trion立方体 + 弹:毒蛇

狙击手芯片：形（狙击枪）+ 弹（小行星，固定不可更改）= 一个芯片
  例：鹭 = 形:鹭狙击枪 + 弹:小行星（固定）
  例：朱鹮 = 形:朱鹮狙击枪 + 弹:小行星（固定）
```

#### 枪手芯片的特殊之处

枪手芯片可以在一个芯片中整合**最多2种子弹**，战斗中二选一切换使用：

| 特征 | 说明 |
|------|------|
| 弹种整合 | 一个枪型芯片最多整合2种子弹 |
| 切换方式 | 2种弹药之间二选一切换，不可同时使用 |
| 合成弹 | **不可用**——因为2种弹药在同一芯片内，无法同时激活来组合 |
| 命名方式 | 如"突击步枪（小行星+猎犬）"、"手枪（流星）" |

#### 射手芯片与枪手芯片的本质区别

射手芯片虽然也是"形+弹"一体化，但其"形"固定为Trion立方体，这带来了根本性的差异：

```
枪手的Trion表现生成：
  枪型芯片（形:枪械 + 弹:A/B）
       ↓ 激活
  Trion表现（枪械实体）
       ↓ 射击
  Trion表现（子弹A 或 子弹B，二选一）

射手的Trion表现生成：
  子弹芯片（形:Trion立方体 + 弹:A）
       ↓ 激活
  Trion表现（Trion立方体）
       ↓ 分割+发射
  Trion表现（子弹A，可调参数）
```

**射手的核心优势**——因为"形"是Trion立方体而非固定枪械，射手获得了以下能力：
1. 立方体可被**分割**为多发子弹（更多子弹 = 单发更弱）
2. 可实时调整子弹参数（威力/射程/速度）
3. 可散射、延迟或整体发射
4. 两侧各激活一个子弹芯片时，可**实时组合**创建合成弹

| 对比项 | 枪手 | 射手 |
|--------|------|------|
| 芯片结构 | 形:枪械 + 弹:A(+B) | 形:Trion立方体 + 弹:X |
| 操作方式 | 瞄准射击 | 操控立方体 |
| 射程 | +20%加成 | 基准 |
| 弹药灵活性 | 芯片内最多2种，二选一 | 实时调整参数 |
| 合成弹 | 不可用（弹药在同一芯片内） | 可实时组合（两侧各一个芯片） |
| 操作难度 | 简单 | 复杂 |

> **原因说明**：枪手和射手的区别本质上是"形"的不同——枪械是固定形态的发射器，Trion立方体是可自由操控的能量体。枪械简化了操作（瞄准射击即可），但也限制了灵活性；Trion立方体提供了极高的灵活性，但需要使用者同时控制创建、分割、参数调整和发射。

#### 狙击手芯片的本质

狙击手芯片与枪手芯片本质相同（形+弹一体化），但有两个关键区别：
- **弹种固定为小行星**，不可更改，也不可整合第二种弹药
- **以低连射速换取更远的射程和精度**

三种狙击枪在这个共同基础上，各自有不同的性能侧重：

| 狙击枪 | 性能侧重 | 与共同基础的区别 |
|--------|---------|----------------|
| 鹭（Egret） | 平衡型 | 射程最远，各项均衡 |
| 闪电（Lightning） | 速射型 | 弹速最快、射速最高，但威力最低 |
| 朱鹮（Ibis） | 威力型 | 威力最高，但射速最低、最重 |

### 3.7 合成弹系统——两种芯片组合产生新Trion表现

射手可以将两种基础子弹芯片组合，产生全新的Trion表现——**合成弹**：

```
合成弹生成流程：

  子弹芯片A（如小行星）  +  子弹芯片B（如毒蛇）
            ↓                        ↓
       Trion立方体A            Trion立方体B
            └──────── 组合 ────────┘
                      ↓
              合成弹（眼镜蛇）
              = 新的Trion表现
```

**已知合成弹**：

| 合成弹 | 组合 | 效果 |
|--------|------|------|
| 钻头弹（Gimlet） | 小行星+小行星 | 双倍规格，增强穿透力 |
| 眼镜蛇（Cobra） | 小行星+毒蛇 | 具体特性未明确 |
| 黄蜂弹（Hornet） | 猎犬+猎犬 | 更灵活精准的强力追踪 |
| 火蜥蜴（Salamander） | 猎犬+流星 | 带追踪能力的炸裂弹 |
| 战斧（Tomahawk） | 毒蛇+流星 | 可控弹道的炸裂弹 |

**枪手与射手的合成弹差异**：
- **枪手**：**无法使用合成弹**——枪手的2种弹药整合在同一个芯片内，无法同时激活来组合
- **射手**：可以在战斗中**实时组合**——两侧各激活一个子弹芯片，同时操控两个Trion立方体并合并

> **原因说明**：合成弹是三层架构中一个特殊现象——两个第二层（触发芯片）的输出在第三层（Trion表现）合并为一个新的表现。这要求两个子弹芯片**分别位于主副两侧并同时激活**，然后使用者同时操控两个Trion立方体并精确合并。枪手的弹药在同一芯片内无法满足这个条件，因此只有射手才能使用合成弹。

### 3.8 阶段3小结：三层架构的完整运作

```
阶段3中三层架构的完整运作：

第一层（触发体）：运行中
  └→ 提供芯片槽位平台
  └→ 内置系统持续运行（雷达/通讯/紧急脱离监控）
  └→ 真身安全收容在内部

第二层（触发芯片）：注册+激活
  └→ 所有芯片已注册（占用Trion）
  └→ 当前激活的芯片正在生成Trion表现
  └→ 可在已注册芯片间切换
  └→ 同侧互斥，最多同时激活2个

第三层（Trion表现）：活跃
  └→ 武器实体在手中（弧月剑/蝎子刃/枪械等）
  └→ 子弹在发射（小行星/猎犬/合成弹等）
  └→ 特殊效果在运行（护盾/蓑衣虫/蜘蛛线等）
  └→ 切换芯片时旧表现消散、新表现生成

Trion资源池：持续消耗中
  └→ 占用量锁定不变
  └→ 可用量因维持/激活/使用/流失持续减少
  └→ 不可恢复
```

│  │  已注册芯片池：[弧月][蝎子][护盾][蓑衣虫]...    │    │
│  │       ↓ 选择激活                                │    │

## 阶段4：解除——Trion表现消散与芯片关闭

> 战斗结束或战斗体失效。这是三层架构**逆向关闭**的阶段——第三层（Trion表现）先消散，第二层（触发芯片）注销，第一层（触发体）释放真身。

### 4.1 芯片主动关闭——Trion表现消散

队员可以在战斗中主动关闭某个芯片（不解除战斗体）：

```
芯片主动关闭：

当前状态：弧月激活中（剑在手中）
队员意图："关闭弧月"
       ↓
Trion表现消散（弧月剑消失）
       ↓
芯片回到"已注册但未激活"状态
       ↓
该芯片的占用量仍然锁定（不释放）
```

> **原因说明**：关闭芯片只是停止生成Trion表现，芯片本身仍然注册在Trion资源池中。占用量不会因为关闭芯片而释放——只有解除整个战斗体才能释放占用量。

### 4.2 芯片切换时的消散

当同侧切换芯片时，旧芯片的Trion表现会先消散，然后新芯片才开始引导：

```
切换消散过程：

弧月激活中（剑在手中）
       ↓ 队员切换到蝎子
旧芯片关闭：弧月剑消散
       ↓（空窗期开始）
新芯片引导中...（队员无武器）
       ↓ 引导完成（空窗期结束）
蝎子刃生成
       ↓
弧月回到"已注册未激活"
蝎子变为"已注册且激活"
```

### 4.3 战斗体解除时的全面消散

当整个战斗体解除时，**所有**Trion表现同时消散：

```
战斗体解除时的三层关闭顺序：

1. 第三层全面消散
   └→ 所有激活中的Trion表现同时消失
   └→ 武器、护盾、特殊效果全部消散

2. 第二层全面注销
   └→ 所有芯片从Trion资源池注销
   └→ 占用量的处理取决于解除方式（见4.4）

3. 第一层执行真身释放
   └→ 战斗体消散/破裂
   └→ 真身从触发体内释放
   └→ 意识转回真身
```

### 4.4 两条路径的Trion结算差异

战斗体解除有两条路径，对芯片占用Trion的处理截然不同：

| 对比项 | 路径A：主动解除 | 路径B：被动破裂 |
|--------|---------------|-----------------|
| **触发方式** | 使用者主动发出Trigger Off意图 | Trion耗尽/弱点被毁/主动触发紧急脱离 |
| **Trion表现** | 消散 | 消散 |
| **芯片注销** | 正常注销 | 强制注销 |
| **占用Trion** | **解锁返还**——占用量重新变为可用 | **随战斗体一同流失耗尽** |
| **Trion恢复起点** | 较高（保留了占用量+未消耗量） | 接近枯竭 |
| **重新激活等待** | 较短 | 较长 |

```
主动解除的Trion结算：
┌─────────────────────────────────────┐
│  已消耗(25%)  │  可恢复(75%)         │
│  ░░░░░░░░░░  │  ████████████████████│
│  需时间恢复   │  占用量已解锁返还     │
└─────────────────────────────────────┘

被动破裂的Trion结算：
┌─────────────────────────────────────┐
│  已消耗(100%)                        │
│  ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░    │
│  占用量也流失耗尽，需长时间恢复       │
└─────────────────────────────────────┘
```

> **原因说明**：这个差异的本质是——主动解除是"正常关闭程序"，系统有时间让芯片正常释放其占用的Trion（类似关闭程序释放内存）；被动破裂是"断电崩溃"，战斗体连同其中运行的所有芯片一起消散，芯片正在使用的Trion也随之丢失（类似断电崩溃，内存内容全部丢失）。这也是为什么经验丰富的队员会尽量选择主动解除——保留更多Trion意味着更快的恢复和更短的重新激活等待时间。

### 4.5 阶段4小结：三层架构的关闭状态

```
阶段4结束时的三层状态：

第一层（触发体）：回到待机
  └→ 真身已释放
  └→ 内置系统关闭
  └→ 芯片仍物理安装在槽位中（可重新配置）

第二层（触发芯片）：已注销
  └→ 从Trion资源池注销
  └→ 回到"已装载但未注册"状态
  └→ 可以更换芯片配置（为下次战斗准备）

第三层（Trion表现）：不存在
  └→ 所有Trion表现已消散
  └→ 武器、护盾、效果全部消失

Trion资源池：
  └→ 主动解除：占用量释放，开始自然恢复
  └→ 被动破裂：接近枯竭，需长时间恢复
```

## 对比分析：三种触发器体系的架构差异

以上四个阶段描述的是**BORDER触发器**的三层架构运作过程。但在《境界触发者》世界中，还存在另外两种触发器体系，它们的架构设计理念截然不同。

### 三种体系对比

| 对比项 | BORDER触发器 | 近界触发器 | 黑触发器 |
|--------|-------------|-----------|---------|
| **架构类型** | 模块化三层 | 一体化 | 生命意志 |
| **设计理念** | 标准化、可量产、灵活配置 | 个性化特制、发挥个体潜能 | 不可设计——由创造者生命力和意志自然形成 |
| **物理形态** | 无刃柄状握把 | 多样（手镯、融合器等） | 各异（与创造者相关） |
| **配置灵活性** | 高——8槽自由组合，战前可更换 | 无——能力固定不变 | 无——能力固定不变 |
| **性能上限** | 中等——受标准化设计限制 | 较高——个性化设计可突破标准 | 极高——远超普通触发器 |
| **获取方式** | 组织分配，可量产 | 各国技术制造 | 创造者倾注所有生命力创造，不可复制 |
| **Trion效率** | 标准——通过芯片注册分配 | 各异——取决于设计 | 极高——额外叠加创造者结晶化Trion |
| **使用者要求** | 训练即可 | 各异 | 需要与创造者人格兼容 |
| **可调控性** | 完全可控可调 | 不可调 | 不可调控 |

### 架构差异的本质

```
BORDER触发器（模块化三层）：
  触发体 → 触发芯片(可更换) → Trion表现(由芯片决定)
  特点：灵活但有上限

近界触发器（一体化）：
  触发器本体 = 能力(固定) → Trion表现(固定)
  特点：三层合一，无法拆分配置

黑触发器（生命意志）：
  创造者生命+意志 = 触发器 → 固定能力(极强)
  特点：超越技术框架，不遵循标准规则
```

> **原因说明**：BORDER的模块化设计是一种"组织化战略"——牺牲单体性能上限，换取可量产、可训练、可灵活配置的团队战斗力。近界触发器追求个体极致，黑触发器则完全超越了技术设计的范畴。三种体系各有优劣，没有绝对的高下之分。

## 总结：三层架构全过程一览

```
┌──────────────────────────────────────────────────────────────────┐
│                    触发器三层架构全过程                             │
├──────────────────────────────────────────────────────────────────┤
│                                                                    │
│  阶段1：配置（战前）                                               │
│  ┌─────────────────────────────────────────────────────────┐     │
│  │ 第一层（触发体）：待机状态，8槽可用                       │     │
│  │ 第二层（触发芯片）：物理装载到槽位，未注册                │     │
│  │ 第三层（Trion表现）：不存在                               │     │
│  │ Trion：100%可用                                          │     │
│  └─────────────────────────────────────────────────────────┘     │
│                          ↓ Trigger On                              │
│  阶段2：激活                                                       │
│  ┌─────────────────────────────────────────────────────────┐     │
│  │ 第一层（触发体）：真身收容，战斗体释放，内置系统启动      │     │
│  │ 第二层（触发芯片）：向Trion资源池注册，锁定占用量         │     │
│  │ 第三层（Trion表现）：尚未生成（等待芯片激活）             │     │
│  │ Trion：占用(锁定) + 可用(战斗用)                         │     │
│  └─────────────────────────────────────────────────────────┘     │
│                          ↓ 激活芯片                                │
│  阶段3：战斗                                                       │
│  ┌─────────────────────────────────────────────────────────┐     │
│  │ 第一层（触发体）：运行中，提供平台和内置系统              │     │
│  │ 第二层（触发芯片）：激活中，可切换（同侧互斥，最多2个）   │     │
│  │ 第三层（Trion表现）：活跃——武器/子弹/效果                │     │
│  │ Trion：占用(锁定) + 已消耗(不可逆) + 剩余可用            │     │
│  │                                                           │     │
│  │ 特殊机制：                                                │     │
│  │ · 射手：Trion立方体→分割→发射（绕过枪型中间层）          │     │
│  │ · 合成弹：两种子弹芯片组合→新Trion表现                   │     │
│  │ · 双持：主副各激活1个芯片，同时使用                       │     │
│  └─────────────────────────────────────────────────────────┘     │
│                          ↓ 解除                                    │
│  阶段4：解除                                                       │
│  ┌─────────────────────────────────────────────────────────┐     │
│  │ 第三层→消散：所有Trion表现消失                            │     │
│  │ 第二层→注销：芯片从Trion资源池注销                        │     │
│  │ 第一层→释放：真身从触发体释放                             │     │
│  │                                                           │     │
│  │ 路径A（主动解除）：占用Trion解锁返还 → 较快恢复          │     │
│  │ 路径B（被动破裂）：占用Trion流失耗尽 → 长时间恢复        │     │
│  └─────────────────────────────────────────────────────────┘     │
│                                                                    │
└──────────────────────────────────────────────────────────────────┘
```
