## 阶段0：Trion兵的本质——在追踪之前先认识它

> 在追踪一个Trion兵的生命历程之前，我们需要先理解它"是什么"，以及它与战斗体的根本区别。

### 0.1 一句话定义

**Trion兵是通过制造设施将Trion能量物质化而成的无意识人工战斗单位，本质是武器而非生命载体。**

这个定义包含三个关键要素：
- **制造设施为媒介**：与战斗体通过触发器生成不同，Trion兵通过专门的制造设施生产——这决定了它可以批量制造
- **无意识**：Trion兵没有自主意识，行为由预设程序或远程操控决定——这是它与战斗体的本质分界线
- **武器**：Trion兵的存在目的是执行任务（侦察/战斗/捕获），不是保护某个人的真身

> **与战斗体的根本区别**：战斗体是"人的延伸"——一个人通过触发器生成一个战斗体，意识转移其中，战斗体被摧毁人还在。Trion兵是"资源的消耗"——制造设施消耗Trion资源生产出一个无意识单位，被摧毁就是纯粹的资源损失。战斗体是一人一体、可恢复的；Trion兵是批量生产、不可回收的。

### 0.2 核心特性：制造设施作为媒介

Trion兵与战斗体共享同一个底层原理——**Trion需要媒介才能转化**（详见2.4阶段0）。但两者使用的媒介完全不同：

| 对比项 | 战斗体 | Trion兵 |
|--------|--------|---------|
| **媒介** | 触发器（个人装备） | 制造设施（工业设备） |
| **产出** | 一人一体 | 批量生产 |
| **意识** | 承载使用者意识 | 无意识 |
| **能量来源** | 使用者自身Trion | 制造时注入的Trion |
| **损失代价** | 无人员损失（Bail Out） | 纯资源损失 |
| **可恢复性** | 解除后可重新生成 | 被摧毁即永久消失 |

> **原因说明**：为什么近界民要发展Trion兵而不是只用战斗体？因为触发器使用者数量有限。Rhodokhroun甚至无法派遣人类战斗员，完全依赖Trion兵作战。Trion兵的战略价值在于：用工业产能弥补人力不足。

### 0.3 两个独立维度：功能复杂度 vs 部署规模

Trion兵的设计存在一个根本性的权衡——功能越复杂，制造成本越高，部署规模越小：

```
功能复杂度（单体性能）
  ↑
高│  ·Rabbit（高级捕获，变体定制）
  │     ·Idra（人形，手动操控）
  │        ·Ilgar（轰炸+自爆）
  │  ·Bander（捕获+炮击）
  │     ·Marmod（近战战斗）
  │        ·Dog三型号（侦察/支援/战斗）
低│           ·Rad（设置Gate）
  └──────────────────────────→ 部署规模
  少                          多（数千）

说明：
· 左上角 = 单体高性能路线（如Rabbit，少量精锐）
· 右下角 = 大规模部署路线（如Rad，数千单位）
· 两条路线不是非此即彼——同一次入侵中两者并用
```

### 0.4 阶段小结

> **快照**：Trion兵是通过制造设施批量生产的无意识武器，与战斗体共享"Trion物质化"的底层原理但走了完全不同的技术路线。它的设计在"单体性能"和"部署规模"之间权衡，两条路线在实战中互补使用。

## 阶段1：制造——Trion兵从何而来

> Trion兵不是天然存在的，它需要被制造出来。制造过程决定了Trion兵的类型、能力和存续时间。

### 1.1 制造原理：Trion物质化

Trion兵的制造本质是**Trion物质化**——通过制造设施（媒介）将Trion能量从能量态转化为固态物质形态。

这与战斗体的生成原理相同（都是Trion物质化），但过程完全不同：

| 对比项 | 战斗体构筑 | Trion兵制造 |
|--------|-----------|------------|
| **触发条件** | 使用者发出"Trigger On"指令 | 制造设施启动生产 |
| **能量来源** | 使用者自身Trion腺体 | 外部Trion能量注入 |
| **生成速度** | 瞬间（秒级） | 需要制造时间（批量生产） |
| **形态决定** | 触发器芯片配置 | 制造设施的设计模板 |
| **能量补充** | 战斗中持续消耗使用者Trion | 制造时一次性注入，**不可补充**（Rad除外） |

> **原因说明**：为什么Trion兵制造时一次性注入能量而非持续供给？因为Trion兵没有Trion腺体，无法自行产生能量。战斗体的使用者有Trion腺体持续供能（虽然战斗中不可恢复），但Trion兵连这个来源都没有。制造时注入多少，就只有多少。

### 1.2 两种制造策略

近界民在Trion兵制造上发展出两条截然不同的路线：

**策略一：单体高性能型**
- 设计理念：集中资源打造少量精锐单位
- 代表：Rabbit（高级捕获型）、Ilgar（轰炸型）
- 特点：功能复杂、制造成本高、单体战斗力强
- 适用场景：需要完成特定高难度任务（如捕获触发器使用者）

**策略二：团队战术型**
- 设计理念：4-5单位联动设计，功能互补
- 代表：Dog三型号（Takia侦察+Manverd支援+Batelier战斗）、Idra群体
- 特点：单体功能较少但注重团队配合，适合大规模部署
- 适用场景：需要覆盖大面积战场、执行多线任务

> **原因说明**：两种策略不是非此即彼的。Aftokrator大规模入侵时同时使用了大量Marmod（低成本战斗型）和少量Rabbit（高性能捕获型），前者消耗敌方资源，后者执行核心捕获任务。

### 1.3 改造与定制：Rabbit变体案例

Rabbit是Trion兵改造定制的典型案例，展示了如何在基础型号上叠加不同能力：

| 变体 | 叠加能力 | 能力来源 | 战术增强 |
|------|---------|---------|---------|
| **基础型（Plain）** | 背部尖刺电流 | 标准设计 | 物理力量最强 |
| **兰巴内因型** | 口中爆炸光束+喷气背包 | 兰巴内因的能力模板 | 远程攻击+机动性 |
| **艾内多拉型** | 手臂液化+地下刀刃 | 艾内多拉的能力模板 | 突袭+奇袭 |
| **修斯型** | 胸部磁性碎片 | 修斯的能力模板 | 控制+固定 |
| **雷普利卡型** | Boost/Shield/Bolt | 游真的黑触发器印记 | 跨技术体系整合 |

所有变体都保留了Rabbit的核心能力——**方块化捕获**（来自黑触发器Alektor）。变体是在核心能力之上叠加额外能力，而非替换。

> **原因说明**：为什么Aftokrator要制造这么多变体？因为不同变体针对不同类型的对手。面对远程型对手用兰巴内因型（有光束反击），面对群体用修斯型（磁性碎片控制），面对防御严密的目标用艾内多拉型（地下突袭）。这是"用工业定制对抗个体差异"的战略思路。

```
Trion兵制造能力层级：

┌─────────────────────────────────────────────┐

│  Rhodokhroun（专业化）                        │
│  · 专注Idra和Dog开发                          │
│  · 团队战术型设计（4-5单位联动）               │
│  · 技术输出能力（向Galopoula供应）             │
├─────────────────────────────────────────────┤

### 1.5 阶段小结

> **快照**：Trion兵通过制造设施将Trion能量物质化而成，制造时一次性注入能量（不可补充）。制造策略分为单体高性能和团队战术型两条路线。Aftokrator拥有最完整的制造体系，包括黑触发器整合和变体定制能力。此刻，Trion兵已被制造出来，等待被部署到战场。

## 阶段2：部署——Trion兵如何到达战场

> Trion兵制造完成后，需要通过门（Gate）从近界传送到目标世界。部署方式直接决定了入侵的节奏和战术效果。

### 2.1 部署前提：门（Gate）的开启

Trion兵无法自行跨越世界之间的距离，必须通过门（Gate）传送。

门的开启需要Trion能量。这就产生了一个"鸡生蛋"问题——要部署Trion兵需要门，但开门需要Trion。解决方案是**先派遣Rad**：

```
部署启动链：

近界民母星 ──→ 少量Rad先行潜入 ──→ Rad收集附近人类少量Trion
                                          │
                                          ↓
                                    Rad开启Gate
                                          │
                                          ↓
                              大量Trion兵通过Gate涌入
```

> **原因说明**：为什么Rad能收集人类的Trion？因为所有人类都拥有Trion腺体（只是大多数人的Trion量极低）。Rad通过从附近行人处收集少量Trion来积累开门所需的能量。这也是Rad通常隐藏在其他Trion兵（如Bamster）体内的原因——它需要被安全地运送到目标区域。

### 2.2 两种部署模式

两次入侵展示了截然不同的部署策略：

```
模式一：一次性涌入（Aftokrator大规模入侵）

时间 ─────────────────────────────────→
     │
     │  ████████████████  ← 超过100个Gate在1分钟内同时开启
     │  ████████████████  ← 大量Marmod+Bander+Rabbit同时涌入
     │  ████████████████  ← 多点开花，瞬间压垮防御
     │
特点：压倒性数量优势，不给防御方反应时间


模式二：分波次投放（Galopoula入侵）

时间 ─────────────────────────────────→
     │
     │  ██        ← 第一波：Dog侦察（约200只）
     │     ███    ← 第二波：Idra进攻（约95只）
     │        █   ← 第三波：战士渗透（6人）
     │
特点：层层递进，每波有明确战术目的
```

| 对比项 | 一次性涌入 | 分波次投放 |
|--------|-----------|-----------|
| **战术目的** | 压倒性打击+捕获 | 侦察+牵制+渗透 |
| **Gate数量** | 100+个同时开启 | 少量，分批开启 |
| **信息依赖** | 低（靠数量碾压） | 高（每波基于前波情报） |
| **资源效率** | 低（大量消耗） | 高（精确投放） |
| **适用方** | Aftokrator（资源充裕） | Galopoula（资源有限） |

### 2.3 嵌套战术：Trion兵藏Trion兵

Aftokrator发展出了一种精妙的嵌套部署战术——**在大型Trion兵体内隐藏小型Trion兵**：

```
嵌套结构：

┌─────────────────────────┐
│  Bamster（巨型捕获型）    │
│  ┌─────────────┐        │
│  │  Rabbit ×N  │        │  ← Bamster体内隐藏多个Rabbit
│  └─────────────┘        │
│  ┌─────────────┐        │
│  │  Rad ×N     │        │  ← Bamster体内也可隐藏Rad（用于开Gate）
│  └─────────────┘        │
└─────────────────────────┘

┌─────────────────────────┐
│  Bander（捕获型/炮击型）  │
│  ┌─────────────┐        │
│  │  Rabbit ×N  │        │  ← Bander体内同样可隐藏Rabbit
│  └─────────────┘        │
└─────────────────────────┘

战术效果：
· Bamster被击败 → Rabbit从中涌出（"特洛伊木马"）
· Bander被击败 → 同样可能释放Rabbit
· Rad隐藏在大型Trion兵体内，但其任务是潜伏开Gate，与Rabbit释放无关
· 防御方以为消灭了威胁，实际上更大的威胁刚刚释放
```

> **原因说明**：嵌套战术的核心价值是**欺骗**。防御方看到Bamster会按"大型低威胁单位"来应对，不会预料到内部藏着高性能Rabbit。这种战术利用了防御方对Trion兵类型的认知惯性。

### 2.4 阶段小结

> **快照**：Trion兵通过Rad开启的Gate从近界传送到目标世界。部署模式分为一次性涌入（压倒性打击）和分波次投放（精确渗透）两种。嵌套战术（大型Trion兵藏小型Trion兵）增加了部署的欺骗性。此刻，Trion兵已到达战场，进入战斗状态。

## 阶段3：战斗——Trion兵的核心运作

> 战斗是Trion兵存在的根本目的。本阶段是全文核心，分析Trion兵在战斗中的行为模式、操控机制、能量约束、群体协同和弱点。

### 3.1 自律型行为状态机

大多数Trion兵采用自律型（预设程序）运作。它们的行为可以用一个有限状态机来描述：

```
自律型Trion兵行为状态机：

                    ┌──────────┐
          部署完成  │          │  未发现目标
        ─────────→ │  巡逻中  │ ←─────────┐
                    │          │            │
                    └────┬─────┘            │
                         │                  │
                    发现目标                 │
                         │                  │
                         ↓                  │
                    ┌──────────┐            │
                    │          │  目标逃脱   │
                    │  攻击中  │ ───────────┘
                    │          │
                    └──┬───┬───┘
                       │   │
              目标失能  │   │  自身受重伤
                       │   │
                       ↓   ↓
                 ┌────────┐ ┌────────┐
                 │ 捕获中 │ │ 自爆中 │
                 │(部分型)│ │(Ilgar) │
                 └───┬────┘ └───┬────┘
                     │          │
                     ↓          ↓
                 ┌────────────────┐
                 │    任务完成     │
                 │  或 被摧毁     │
                 └────────────────┘

状态说明：
· 巡逻中：在指定区域内移动，搜索目标
· 攻击中：发现目标后自动攻击（近战/远程/轰炸）
· 捕获中：仅限捕获型（Bamster/Rabbit/Bander），将目标方块化或吞入
· 自爆中：仅限Ilgar，受重伤时装甲增强，冲向人群密集处
```

**各类型的状态差异**：
- **Marmod**：巡逻→攻击→被摧毁（最简单的状态机，无捕获/自爆）
- **Bamster/Rabbit**：巡逻→攻击→捕获→被摧毁（增加捕获状态）
- **Ilgar**：巡逻→轰炸→自爆→被摧毁（攻击分为轰炸和自爆两阶段）
- **Rad**：潜伏→收集Trion→开启Gate→被摧毁（完全不同的任务状态机）

> **原因说明**：为什么自律型的行为可预测？因为预设程序的状态转换条件是固定的——发现目标就攻击，受重伤就自爆。这种可预测性是自律型的固有缺陷，也是Border能够制定针对性战术的基础。

### 3.2 手动操控型模式切换：Idra的双模式

Idra是唯一明确支持手动操控模式（Manual Mode）的Trion兵，它拥有自律和手动两种运作模式：

```
Idra双模式状态机：

┌─────────────────────────────────────────┐
│              自律模式                     │
│  ┌────────┐    ┌────────┐    ┌────────┐ │
│  │ 巡逻中 │───→│ 攻击中 │───→│ 被摧毁 │ │
│  └────────┘    └────────┘    └────────┘ │
│       ↑              ↑                   │
│       │ 切换         │ 切换              │
│       ↓              ↓                   │
│  ┌────────┐    ┌────────┐               │
│  │ 待命中 │───→│ 精确   │               │
│  │        │    │ 操控中 │               │
│  └────────┘    └────────┘               │
│              手动模式                     │
│  （操控者远程控制，实时响应）              │
└─────────────────────────────────────────┘

关键差异：
· 自律模式：行为可预测，无需操控者，可大规模部署
· 手动模式：行为灵活，需要操控者注意力，可执行复杂战术
· 模式切换：操控者可随时在两种模式间切换
```

**手动模式的战术价值**：
- 可执行自律模式无法完成的复杂战术（如精确配合、战术撤退）
- Galopoula入侵时，特工伪装成Idra混入战场进行渗透
- 操控者被干扰时，Idra会退回自律模式——这是一个可被利用的弱点

### 3.3 战斗中的能量约束

Trion兵在战斗中面临一个根本性约束——**能量不可补充**（Rad除外）。

这与战斗体形成鲜明对比：

| 对比项 | 战斗体 | Trion兵 |
|--------|--------|---------|
| **能量来源** | 使用者Trion腺体持续供能 | 制造时一次性注入 |
| **战斗中补充** | 不可恢复（但有持续供给） | 完全不可补充 |
| **能量耗尽后果** | 战斗体破裂，Bail Out保护真身 | Trion兵直接消亡 |
| **战斗时长限制** | 受使用者Trion总量限制 | 受注入量限制，通常更短 |

> **原因说明**：能量不可补充这个约束深刻影响了Trion兵的战术定位。它们不是用来打持久战的——它们是"一次性消耗品"。这就是为什么近界民的入侵战术总是追求速战速决：Trion兵的能量在持续消耗，拖得越久，战斗力衰减越严重。

**Rad的特殊例外**：Rad可以通过收集附近人类的少量Trion来补充能量。这使得Rad可以长期潜伏（其他Trion兵做不到），但收集速度极慢，仅够维持基本存续和开启Gate。

### 3.4 群体协同战术

Trion兵的真正威力不在单体，而在群体协同。以下是三种典型的协同模式：

**模式一：Idra组合护盾**
```
单个Idra护盾：
  ┌───┐
  │ ▓ │  ← 单体护盾，耐久度有限
  └───┘

多个Idra组合护盾：
  ┌───────────────┐
  │ ▓▓▓▓▓▓▓▓▓▓▓▓ │  ← 多个Idra的护盾叠加
  │ ▓▓▓▓▓▓▓▓▓▓▓▓ │     耐久度成倍增加
  └───────────────┘

效果：N个Idra组合 → 护盾耐久度远超N倍（非线性增长）
```

**模式二：Dog三型号配合**
```
战术配合流程：

  Takia（侦察型）          Manverd（支援型）         Batelier（战斗型）
  ┌──────────┐            ┌──────────┐            ┌──────────┐
  │ 隐形侦察 │──情报──→  │ 护盾展开 │──掩护──→  │ 近战突击 │
  │ 发现目标 │            │ 光束支援 │            │ 头部刀刃 │
  └──────────┘            └──────────┘            └──────────┘

  侦察锁定目标 → 支援提供护盾和火力 → 战斗型发起近战
```

**模式三：嵌套释放战术（特洛伊木马）**
```
嵌套释放链：

  Bamster/Bander ──→ 防御方击败宿主 ──→ Rabbit从中涌出
  （大型低威胁）        （以为消灭威胁）     （高性能捕获型）
                                              │
                                              ↓
                                        防御方面临更大威胁
                                        需要重新分配精锐力量
```

> **原因说明**：嵌套释放战术的精妙之处在于——它利用了防御方对大型低威胁单位的认知惯性。Bamster/Bander体型巨大但战斗力不强，防御方会优先处理它们以清理战场。然而击败宿主的瞬间，预藏的高性能Rabbit涌出，防御方在消耗战力后立刻面对更强的对手。这是预先储备的延迟释放，不是无限增殖。

### 3.5 弱点与破坏方式

几乎所有Trion兵共享一个通用弱点——**位于口中的单眼**。

```
Trion兵弱点结构：

  ┌─────────────┐
  │  外部装甲    │  ← 坚韧皮肤/鳞片，常规攻击难以穿透
  │  ┌───────┐  │
  │  │ 口腔  │  │
  │  │ ┌───┐ │  │
  │  │ │ ◉ │ │  │  ← 单眼（核心弱点），足够伤害可摧毁整体
  │  │ └───┘ │  │
  │  └───────┘  │
  └─────────────┘
```

**弱点设计的因果逻辑**：
- **为什么弱点在口中？** 口腔是Trion兵少数没有装甲保护的部位（需要开口进行攻击/捕获）
- **为什么是单眼？** 单眼是Trion兵的感知核心，类似于"CPU"——摧毁它等于摧毁整个控制系统
- **为什么这个弱点被保留？** 因为完全封闭口腔会丧失攻击/捕获能力——这是功能性和防御性之间的必然权衡

**不同型号的破坏难度**：
- **低难度**：Rad（单体极弱，无需瞄准弱点）、Dog（群体较弱）
- **中难度**：Marmod（需要精确攻击眼部）、Bander（体型大但可被B级特工击败）
- **高难度**：Rabbit（需要精确瞄准+集中火力，单个A级特工无法击败）
- **特殊**：Ilgar（受重伤进入自爆模式，装甲增强，需要在自爆前拦截）

### 3.6 阶段小结

> **快照**：Trion兵在战斗中按自律型状态机运作（巡逻→攻击→捕获/自爆），Idra额外支持手动操控模式。能量不可补充是根本约束，决定了Trion兵是"一次性消耗品"。群体协同（组合护盾、三型号配合、嵌套释放战术）是真正的威力来源。口中单眼是通用弱点，但高级型号需要精确攻击+集中火力才能击败。

## 阶段4：消亡——Trion兵的终结

> 所有Trion兵最终都会消亡。与战斗体不同，Trion兵的消亡是不可逆的——没有Bail Out，没有重新生成，消亡就是永久消失。

### 4.1 三种消亡路径

```
Trion兵消亡路径：

                    ┌──────────┐
                    │ 战斗中的  │
                    │ Trion兵  │
                    └──┬─┬─┬───┘
                       │ │ │
          ┌────────────┘ │ └────────────┐
          │              │              │
          ↓              ↓              ↓
    ┌──────────┐  ┌──────────┐  ┌──────────┐
    │ 被击败   │  │ 能量耗尽 │  │ 自爆     │
    │          │  │          │  │          │
    │ 弱点被   │  │ 注入的   │  │ 主动引爆 │
    │ 精确攻击 │  │ Trion    │  │ 剩余能量 │
    │ 或遭受   │  │ 自然消耗 │  │ （Ilgar  │
    │ 压倒性   │  │ 殆尽     │  │  专属）  │
    │ 伤害     │  │          │  │          │
    └────┬─────┘  └────┬─────┘  └────┬─────┘
         │             │             │
         ↓             ↓             ↓
    ┌─────────────────────────────────────┐
    │         Trion物质消散               │
    │   （不可回收，纯资源损失）           │
    └─────────────────────────────────────┘
```

**路径一：被击败**——弱点（口中单眼）被精确攻击，或遭受超过结构承受极限的伤害。这是最常见的消亡方式。

**路径二：能量耗尽**——制造时注入的Trion自然消耗殆尽。由于Trion兵无法补充能量，长时间战斗后必然走向这个结局。这是"时间站在防御方一边"的根本原因。

**路径三：自爆**——Ilgar专属。受重伤时主动进入自爆模式，装甲增强，冲向人群密集处引爆剩余Trion能量。自爆是将"即将消亡"的资源转化为最后一次大范围攻击。

### 4.2 消亡后资源结算

Trion兵消亡后，构成其身体的Trion物质消散——**完全不可回收**。

这与战斗体形成又一个关键对比：

| 对比项 | 战斗体消亡 | Trion兵消亡 |
|--------|-----------|------------|
| **人员损失** | 无（Bail Out保护真身） | 不适用（无人员） |
| **Trion回收** | 部分回收（战斗体消散后Trion返回环境） | 完全不可回收 |
| **可重建性** | 可以（使用者重新Trigger On） | 不可以（需要重新制造） |
| **战略影响** | 暂时减员 | 永久消耗制造资源 |

> **原因说明**：Trion兵的不可回收性是其"消耗品"本质的直接体现。每消灭一个Trion兵，就是永久削减了入侵方的战斗资源。这就是为什么防御战的核心策略是"消耗战"——只要持续消灭Trion兵，入侵方的资源终将耗尽。

### 4.3 嵌套释放：宿主消亡触发Rabbit涌出

Bamster/Bander的嵌套部署打破了"消亡即威胁消除"的常规——击败大型宿主反而释放出预藏的Rabbit：

```
嵌套释放过程：

  ┌──────────────┐     击败      ┌──────────────┐
  │ Bamster/     │ ──────────→  │ 宿主消亡     │
  │ Bander       │              │ 结构崩解     │
  │ (内藏Rabbit) │              └────┬─────────┘
  └──────────────┘                   │
                                     ↓
                               ┌──────────┐
                               │ Rabbit   │
                               │ ×N 涌出  │
                               │ (预藏的) │
                               └────┬─────┘
                                    │
                                    ↓
                              ┌──────────┐
                              │ Rabbit   │
                              │ 战斗/    │
                              │ 捕获     │
                              └──────────┘

注意：这是单次释放，不是循环——Rabbit是预先储备在宿主体内的，
释放后不会再产生新的Rabbit。总资源量在制造阶段就已确定。
```

> **原因说明**：嵌套释放的本质是"预先储备的延迟释放"。Aftokrator在制造阶段就将Rabbit预置在Bamster/Bander体内，击败宿主只是触发了释放条件。这不是无限增殖，而是一次性的战术欺骗——让防御方在消耗战力击败大型单位后，立刻面对更强的Rabbit。

### 4.4 阶段小结

> **快照**：Trion兵有三种消亡路径——被击败、能量耗尽、自爆。消亡后Trion物质完全不可回收，是纯粹的资源损失。Bamster/Bander的嵌套释放是唯一的"消亡触发新威胁"例外，但本质是预储备的单次释放而非无限增殖。消亡的不可逆性决定了消耗战是防御方的核心策略。

## 阶段5：战役视角——Trion兵在完整入侵中的角色

> 前四个阶段分析了单个Trion兵的生命周期。本阶段拉高视角，分析Trion兵在完整战役中的战术运用模式。

### 5.1 Aftokrator入侵：消耗→升级→嵌套释放三阶段递进

Aftokrator大规模入侵（1月20日）中，Trion兵的投入呈现清晰的三阶段递进：

```
Aftokrator入侵Trion兵投入时序：

时间 ─────────────────────────────────────────→
     │
阶段1│  ████████████████████  ← 大量Marmod+Bander从多点涌入
消耗 │  标准Trion兵分散Border防御，消耗Border战力和Trion
     │
阶段2│         ▓▓▓▓▓▓▓▓▓▓  ← Rabbit引入战场
升级 │         单个A级特工无法击败，需要多个B级小队联合
     │         捕获型能力（方块化）直接威胁C级队员
     │
阶段3│              ░░░░░░░░░░░░░░  ← 嵌套释放启动
嵌套 │              击败Bamster/Bander → Rabbit从中涌出
释放 │              防御方以为清除威胁，实际释放更强对手
     │

威胁递进逻辑：
· 阶段1：数量压制（消耗资源）
· 阶段2：质量升级（引入精锐）
· 阶段3：嵌套释放（击败大型单位→Rabbit涌出，战术欺骗）
· 每个阶段都建立在前一阶段的基础上
```

**战术效果**：
- 阶段1消耗了Border的Trion和注意力，使其无法集中力量应对阶段2
- 阶段2的Rabbit迫使Border分散精锐力量，为阶段3的嵌套释放创造条件
- 阶段3利用了防御方对大型低威胁单位的认知惯性——击败Bamster/Bander后Rabbit涌出，防御方在消耗战力后立刻面对更强对手

**情报利用**：Aftokrator利用情报（C级队员无法Bail Out）针对性地用Rabbit捕获C级队员。最终32名C级成员被捕获——这是Trion兵（而非近界民战士）直接造成的战果。

### 5.2 Galopoula入侵：侦察→牵制→渗透掩护

Galopoula入侵（2月19日）中，Trion兵的角色完全不同——不是主攻力量，而是掩护工具：

```
Galopoula入侵Trion兵投入时序：

时间 ─────────────────────────────────────────→
     │
阶段1│  ··················  ← 约200只Dog先行侦察
侦察 │  收集Border防御布置信息
     │  评估防御力量和弱点
     │
阶段2│       ▓▓▓▓▓▓▓▓▓▓  ← 约95只Idra发起进攻
牵制 │       吸引Border注意力到外围
     │       消耗Border战力
     │
阶段3│            ████  ← 6名Galopoula战士趁机渗透
渗透 │            利用隧道触发器从地下接近远征船
掩护 │            Trion兵在外围持续牵制
     │

核心逻辑：
· Trion兵不是目的，是手段
· 真正的攻击力量是6名战士
· Trion兵的任务是"让Border看向错误的方向"
```

**与Aftokrator的根本差异**：
- Aftokrator：Trion兵是主攻力量（Rabbit直接执行捕获任务）
- Galopoula：Trion兵是掩护力量（真正的攻击由战士执行）

### 5.3 两次入侵对比分析

| 对比项 | Aftokrator入侵 | Galopoula入侵 |
|--------|----------------|---------------|
| **Trion兵角色** | 主攻力量 | 掩护工具 |
| **部署规模** | 数千单位 | 约295单位（200 Dog + 95 Idra） |
| **类型多样性** | 高（Marmod/Bander/Rabbit/Rad） | 低（Dog + Idra） |
| **Trion兵来源** | 自主制造 | Rhodokhroun提供 |
| **战术目标** | 捕获高Trion者 | 破坏远征船 |
| **递进模式** | 消耗→升级→嵌套释放 | 侦察→牵制→渗透掩护 |
| **Trion兵战果** | 直接捕获32名C级（核心战果） | 牵制成功但未造成决定性战果 |
| **最终结果** | 部分成功（捕获目标但损失Enedra） | 失败（远征船未被摧毁） |

> **原因说明**：两次入侵的差异根源在于资源差距。Aftokrator作为最强近界国，有能力制造大量高性能Trion兵作为主攻力量。Galopoula作为附属国，只能使用他国提供的Trion兵作为辅助，真正的攻击力量必须依赖有限的人类战士。

### 5.4 阶段小结

> **快照**：在战役层面，Trion兵的角色取决于入侵方的资源水平。资源充裕方（Aftokrator）将Trion兵作为主攻力量，通过消耗→升级→嵌套释放的三阶段递进实现战术目标。资源有限方（Galopoula）将Trion兵作为掩护工具，通过侦察→牵制→渗透掩护为人类战士创造机会。

```

### 与Border战斗体系统的根本差异

| 维度 | Border战斗体 | 近界民Trion兵 |
|------|-------------|--------------|
| **设计哲学** | 保护人、增强人 | 替代人、消耗资源 |
| **核心资源** | 人（触发器使用者） | 工业产能（制造设施） |
| **损失代价** | 低（Bail Out保护真身） | 高（不可回收） |
| **战术弹性** | 高（人的判断力） | 低（预设程序）或中（手动操控） |
| **规模上限** | 受限于人数 | 受限于制造资源 |
| **持久性** | 可恢复（解除后重新生成） | 不可恢复（消亡即永久消失） |
| **本质** | 人的延伸 | 资源的消耗 |

> 一句话总结：**Border用人驱动战斗体，近界民用资源驱动Trion兵。前者的瓶颈是人才，后者的瓶颈是产能。**
