## 阶段0：Trion的本质——在追踪之前先认识它

> 在追踪一份Trion的旅程之前，我们需要先理解它"是什么"。

### 0.1 一句话定义

**Trion是一种具有能量-物质二象性的生物能量，可以通过各种媒介（如触发器）被"编程"为各种形态和功能。**

这个定义包含三个不可拆分的要素：
- **生物能量**：由生物体内的Trion腺体产生，不是凭空存在的
- **能量-物质二象性**：可在能量态和物质态之间转换——这是Trion区别于一切"魔力值"的核心特性
- **可被编程**：Trion本身没有固定形态，它的表现完全取决于转化媒介的"指令"

### 0.2 核心特性：需要媒介才能转化

Trion最重要的特性是——**它自己什么都做不了**。

一份Trion储存在体内时，就是纯粹的能量。它不会自动变成剑、不会自动变成护盾、不会自动变成子弹。只有通过媒介（触发器、制造设施等），Trion才能被"编程"为具体的形态和功能。

> **原因说明**：这个特性是整个世界观的基石。正因为Trion需要媒介，才有了触发器技术、才有了BORDER的芯片系统、才有了近界各国不同的Trion技术路线。如果Trion能自行转化，触发器就没有存在的必要了。

### 0.3 量与质——两个独立维度

Trion存在两个独立的衡量维度：

```
Trion双维度模型：

输出功率（行为阈值）
  ↑
高│          ·Hyuse（触发角增强）
  │
  │     ·普通B级队员
  │
低│  ·三云修        ·雨取千佳
  └──────────────────────────→ 容量（总量）
  低                          高

说明：
· 容量 = 能量总量，决定"能做多少事"
· 输出功率 = 单位时间内能释放的Trion强度，决定"能做什么级别的事"
· 两者独立——高容量不等于高输出功率，反之亦然
· 千佳：容量极高(38)但输出功率未必最高
· 三云修：容量极低(2)但输出功率未知
```

> **原因说明**：为什么Trion不是简单的"魔力值"？因为传统RPG的魔力值只有一个维度（量），而Trion有两个独立维度。原作中的"质"在游戏化时被具象为**Trion输出功率**——它作为**行为阈值**发挥作用：某些高级能力需要输出功率达到一定水平才能施展。例如，一名队员可能有足够的Trion容量来激活旋空芯片，但如果输出功率不达标，就无法施展旋空能力。这为角色差异化提供了丰富的设计空间——不仅是"能做多久"的差异，还有"能做什么"的差异。

### 0.4 三个派生属性——Trion的个体化参数

Trion作为基础资源，在不同个体身上表现出三个派生属性，它们贯穿Trion生命周期的各个阶段：

| 派生属性 | 定义 | 性质 | 影响的生命周期阶段 |
|---------|------|------|------------------|
| **Trion天赋(1a)** | 天生个体差异的影响因子 | 影响因子 | 阶段1（产生）：决定Trion腺体的基础产量和容量上限 |
| **Trion输出功率(1b)** | 允许执行某些行为的阈值 | 阈值 | 阶段4（消耗）：决定能否施展特定能力 |
| **Trion恢复速率(1c)** | Trion自然恢复速度的影响因子 | 影响因子 | 阶段6（恢复）：决定从消耗中恢复的速度 |

> **原因说明**：这三个属性是Trion系统个体化的核心。天赋决定"起点有多高"，输出功率决定"能做什么级别的事"，恢复速率决定"多快能再次行动"。三者共同构成了角色在Trion维度上的完整画像。后续各阶段中会自然关联这些属性。

### 0.5 阶段0小结

```
阶段0：认识Trion

Trion的本质：
  · 生物能量，具有能量-物质二象性
  · 自身无法转化，需要媒介"编程"
  · 量（容量）和质（输出功率）是两个独立维度

三个派生属性：
  · Trion天赋——天生个体差异的影响因子
  · Trion输出功率——行为阈值，决定能否施展特定能力
  · Trion恢复速率——自然恢复速度的影响因子

此时Trion的状态：
  · 尚未产生——我们只是在认识它
  · 接下来进入阶段1：它从何而来
```

## 阶段1：产生——Trion从何而来

> 一份Trion的旅程从这里开始。它在人体内的Trion腺体中诞生。

### 1.1 Trion腺体——生产者

**Trion腺体**（トリオン器官 / Trion Gland）是产生和储存Trion能量的器官：

| 属性 | 说明 |
|------|------|
| **位置** | 心脏旁边 |
| **可见性** | 不可见的器官 |
| **普遍性** | 每个人都拥有，但在人类世界中不为人知 |
| **功能** | 产生Trion能量 + 储存Trion能量（双重功能） |

### 1.2 产生机制

Trion由Trion腺体**代谢转化产生**——不是凭空生成，而是通过生物代谢过程将某种原料转化为Trion能量。恢复方式是日常生活：进食、休息。

> **原因说明**：原作中具体的代谢机制不重要（也未详细说明），重要的是"代谢转化"这个本质——它意味着Trion的产生需要消耗生物资源，因此有上限、需要时间、可以被增强也可以衰退。

### 1.3 发育规律——20岁的分水岭

Trion腺体的发育有明确的时间窗口：

```
Trion腺体发育时间线：

出生                    ~20岁                    老年
  |========================|.........................|
  |    可训练增长期         |   停止发育期            |
  |                        |                        |
  | · 如同肌肉，定期使用    | · 约20岁后停止发育      |
  |   可增加产量            | · 不锻炼会开始衰退      |
  | · 良好营养和休息有益    | · 成年后训练效果大幅降低 |
  | · 最佳训练期            |                        |
```

> **原因说明**：为什么20岁后停止发育？这个设定直接解释了BORDER为什么招募年轻人——不是因为年轻人更勇敢或更听话，而是因为**只有年轻人的Trion腺体还在发育期，训练才有意义**。这也是Aftokrator购买高Trion儿童植入触发角的战略原因——在发育期植入，效果最大化。

### 1.4 个体差异——天生的不平等

每个人的Trion腺体功能不同，决定了Trion容量的天生差异。这种差异的根源就是**Trion天赋(1a)**——一个天生的个体差异影响因子，它决定了Trion腺体的基础产量和容量上限：

```
已知Trion值分布（BORDER参数评分制）：

  38 ████████████████████████████████████████ 雨取千佳（异常值）
  37 ███████████████████████████████████████  迅悠一（黑触发器加成）
   9 █████████  高级战斗员
   7 ███████    迅悠一（普通触发器）/ 普通战斗员
   4 ████       低级战斗员
   2 ██         三云修（BORDER战斗员最低）
  ~1 █          操作员（非战斗人员）
     └──────────────────────────────────────→
     低                                    高

说明：
· 参数评分制是相对评分，非绝对数值
· 典型BORDER战斗人员范围：4-9
· 千佳的38是极端异常值
· 迅的37包含黑触发器的额外Trion加成
```

### 1.5 阶段1小结

```
阶段1：Trion已产生

Trion的状态：
  · 由Trion腺体代谢转化产生
  · 储存在Trion腺体中
  · 全部处于"自由状态"——未被任何系统占用或消耗
  · 可通过日常生活（进食、休息）持续补充

关键特性：
  · 产量由腺体功能决定（天生差异+训练）
  · 20岁前可训练增长，20岁后停止发育
  · 个体差异巨大（参数2-38）
  · 接下来进入阶段2：储存与增强
```

## 阶段2：储存与增强——Trion的"水库"与"扩容"

> Trion产生后，储存在Trion腺体中。但这个"水库"的容量并非一成不变——有三种方式可以"扩容"。

### 2.1 Trion腺体作为储存容器

Trion腺体同时承担**生产者**和**储存容器**的双重角色。产生的Trion能量就地储存在腺体中，等待被使用。

BORDER使用参数评分制来量化个体的Trion容量：
- 使用参数评分制（1或更高）
- 这是相对评分，非绝对数值
- 典型BORDER战斗人员范围：4-9

### 2.2 三种增强路径

Trion容量可以通过三种路径增强，每种路径的机制和效果截然不同：

```
三种增强路径的"水库"类比：

路径A：训练（自然增强）
┌──────────────┐        ┌──────────────────┐
│  原始水库    │  训练   │  略微扩大的水库    │
│  ████████  │ ──→    │  ██████████████  │
│  容量:7     │        │  容量:8(+1)       │
└──────────────┘        └──────────────────┘
特点：缓慢、有限、20岁前有效

路径B：触发角（人工增强）
┌──────────────┐        ┌────────────────────────┐
│  原始水库    │  植入   │  大幅扩大的水库          │
│  ████████  │ ──→    │  ██████████████████████│
│  容量:7     │        │  容量:15(+8)             │
└──────────────┘        └────────────────────────┘
特点：大幅提升量和质、有风险（侵入大脑/影响人格/可能致死）

路径C：黑触发器（持有期间永久扩容）
┌──────────────┐        ┌──────────────┬──────────┐
│  原始水库    │  持有   │  原始水库    │ 扩容水库  │
│  ████████  │ ──→    │  ████████  │ ████████│
│  容量:7     │        │  容量:7     │ +30(扩容) │
└──────────────┘        └──────────────┴──────────┘
特点：持有期间永久扩容，消耗后可恢复填充，优先消耗扩容部分
```

**三种路径详细对比**：

| 对比项 | 训练 | 触发角（Trion接收器） | 黑触发器 |
|--------|------|---------------------|---------|
| **机制** | 锻炼Trion腺体增加产量 | 植入物大幅提升容量和质量 | 创造者结晶化Trion扩增使用者上限 |
| **效果** | 少量提升 | 大幅提升量和质 | 持有期间永久扩增Trion上限（可恢复填充） |
| **持续性** | 永久 | 永久（植入后） | 持有期间永久有效（消耗后可恢复） |
| **时间窗口** | 20岁前最佳 | 无明确限制 | 无限制 |
| **风险** | 无 | 侵入大脑、影响人格、可能致死 | 需要与创造者人格兼容 |
| **使用者** | 所有人 | 近界民（特别是Aftokrator） | 特定适格者 |

### 2.3 高Trion者vs低Trion者的战术差异

Trion容量的高低直接影响战术选择：

| 对比项 | 高Trion者（如千佳） | 低Trion者（如三云修） |
|--------|-------------------|---------------------|
| **战斗持续时间** | 长——可用Trion充裕 | 短——可用Trion有限 |
| **武器选择** | 可使用高消耗武器 | 需选择低消耗武器 |
| **受伤容忍度** | 高——泄漏对总量影响较小 | 低——泄漏快速耗尽储备 |
| **战斗体耐久度** | **与高Trion者相同** | **与高Trion者相同** |
| **恢复时间** | 较长（容量大，恢复慢） | 较短（容量小，恢复快） |
| **战术风格** | 可持续输出、消耗战 | 速战速决、精准打击 |

> **原因说明**：为什么"耐久度与Trion容量无关"是关键设计？因为这意味着低Trion者的战斗体**同样坚固**——三云修的战斗体和千佳的战斗体一样难以被击伤。差异不在于"能不能扛住一刀"，而在于"扛住之后还剩多少Trion"。这使得低Trion者不是"脆皮"，而是"续航短"——完全不同的战术定位。

### 2.4 阶段2小结

```
阶段2：Trion已储存，可能已增强

Trion的状态：
  · 储存在Trion腺体中（可能已通过训练/触发角增强容量）
  · 全部处于"自由状态"——未被任何系统占用或消耗
  · 量和质两个维度已确定

三种增强路径：
  · 训练：缓慢、有限、永久
  · 触发角：大幅、有风险、永久
  · 黑触发器：持有期间永久扩容、可恢复填充、需兼容性

关键洞察：
  · 耐久度与容量无关——低Trion者不是"脆皮"而是"续航短"
  · 接下来进入阶段3：分配（进入战斗体状态）
```

## 阶段3：分配——Trion的"占用与可用"（进入战斗体状态）

> 队员激活触发器，Trion从"全部自由"变为"被占用"。这是Trion生命周期中最关键的转折点——从此刻起，Trion不可恢复。

### 3.1 核心公式

```
Trion可用量 = Trion总量 - 占用量 - 累计消耗量
```

这个公式是理解Trion在战斗中流转的核心。激活触发器的瞬间，Trion从"纯蓝量"变为"血量+蓝量+战术分配"的综合体。

### 3.2 芯片注册的本质：从"自由能量"到"被锁定状态"

激活触发器时，已装载的触发芯片向Trion资源池"注册"，锁定各自的占用量：

```
激活前（日常状态）：
┌─────────────────────────────────────┐
│  Trion总量 = 100%                    │
│  ████████████████████████████████   │
│  全部自由——可用于任何用途             │
│  Trion器官可自然恢复                 │
└─────────────────────────────────────┘

激活后（战斗体状态）：
┌─────────────────────────────────────┐
│  占用量(~30%)   │  可用量(~70%)      │
│  ██████████████│  ██████████████████│
│  芯片注册锁定   │  战斗中可消耗       │
│  不可使用       │  不可恢复           │
└─────────────────────────────────────┘

占用量的构成：
┌─────────────────────────────────────┐
│ 芯片1 │ 芯片2 │ ... │ 芯片8 │ 内置 │
│ 占用  │ 占用  │     │ 占用  │ 系统 │
│       │       │     │       │ 占用 │
└─────────────────────────────────────┘
```

> **原因说明**：为什么需要"占用"机制？芯片注册后，一部分Trion**正在被芯片使用**——用于维持芯片的就绪状态，使其随时可以被激活生成Trion表现。这就像程序加载到内存：程序（芯片）加载后会占用一部分内存（Trion），这些内存正在被程序使用以维持其运行就绪状态，不是"存着以后花的"，而是"正在被占用的"。主动解除时芯片释放这些Trion（类似正常关闭程序释放内存），被动破裂时连同战斗体一起消散（类似断电崩溃，内存内容丢失）。详见"2.3_触发器三层架构过程分析"阶段2。

### 3.3 日常状态vs战斗体状态——Trion的两种存在模式

| 对比项 | 日常状态 | 战斗体状态 |
|--------|---------|-----------|
| **Trion用途** | 全部可用于释放能力、转化燃料等 | 部分被芯片占用，剩余为战斗可用 |
| **恢复能力** | 可自然恢复（进食、休息） | **无法恢复**——Trion器官停止工作 |
| **消耗方向** | 无持续消耗 | 维持消耗持续扣除 |
| **类比** | 纯"蓝量" | 血量+蓝量+战术分配 |

> **原因说明**：为什么战斗体状态下无法恢复Trion？这是战斗节奏的核心约束。如果战斗中能恢复Trion，战斗就会变成无限消耗战，失去紧迫感和战术深度。正因为不可恢复，每一次射击、每一次受伤都是不可逆的代价，迫使队员精打细算、权衡取舍。详见"2.2_真身-战斗体-真身领域过程分析"阶段1。

### 3.4 阶段3小结

```
阶段3：Trion已被分配

Trion的状态：
  · 一部分被芯片注册锁定（占用量）——不可使用但主动解除时可返还
  · 剩余部分为战斗可用量——可消耗但不可恢复
  · Trion器官在战斗体状态下停止工作

关键转折：
  · 从此刻起，Trion不可恢复——一切消耗都是不可逆的
  · 占用量的命运取决于解除方式（阶段5详述）
  · 接下来进入阶段4：消耗
```

## 阶段4：消耗——Trion的"燃烧"

> 战斗开始。Trion从可用量中持续流失，每一种消耗都是不可逆的。这是Trion生命周期中最"剧烈"的阶段。

### 4.1 四种消耗类型穷举

Trion在战斗中的消耗可以穷举为四种类型，每种都有独特的触发机制和特点：

| 消耗类型 | 类比 | 触发时机 | 可控性 | 停止条件 |
|---------|------|---------|--------|---------|
| **维持消耗** | "存在税" | 持续——战斗体存在就要付出 | 不可避免 | 解除战斗体时停止 |
| **激活消耗** | "启动税" | 每次激活芯片时 | 可控（减少切换次数） | 激活完成后停止 |
| **使用消耗** | "行动税" | 每次射击/技能释放/护盾抵挡攻击 | 可控（控制使用频率） | 停止动作后停止 |
| **流失消耗** | "受伤税" | 受伤时 | 被动（不可控） | 约1分钟后自动停止 |

**关于维持消耗的重要澄清**：

维持消耗主要来自**战斗体本身**的存在成本，以及少数需要持续运行的芯片（如蓑衣虫的雷达隐身、变色龙的光学隐身）。**大多数芯片没有维持消耗**——它们以"激活消耗（+使用消耗）"为主要消耗模式。

| 芯片类型 | 维持消耗 | 主要消耗模式 |
|---------|---------|------------|
| 近战武器（弧月、蝎子等） | 无 | 激活消耗（普通挥砍不消耗Trion） |
| 枪型触发器 | 无 | 激活消耗 + 使用消耗（射击） |
| 护盾 | 无（激活后默认0成本维持） | 使用消耗（成功抵挡攻击时消耗） |
| 蓑衣虫/变色龙 | **有**（持续运行） | 维持消耗（持续隐身效果） |
| 战斗体本身 | **有**（存在即消耗） | 维持消耗（底噪） |

> **原因说明**：近战武器的Trion效率极高——弧月芯片只有激活消耗，普通挥砍不消耗Trion。"旋空弧月"等组合技的消耗走的是旋空芯片的使用消耗逻辑，而非弧月本身。护盾的设计也很精妙——激活后默认0成本维持，只有成功抵挡攻击时才消耗Trion。这意味着护盾抵挡攻击只扣Trion不产生伤口，而战斗体直接受伤则产生伤口导致额外的流失消耗。此外，**Trion输出功率(1b)**在消耗阶段也发挥作用——某些高级能力（如旋空）需要输出功率达标才能施展，即使有足够的Trion容量来支付消耗。

### 4.3 不同职位的消耗模式差异

| 职位 | 主要消耗来源 | 消耗模式 | 续航特点 |
|------|------------|---------|---------|
| **攻击手** | 激活消耗（芯片切换） | 极低频——普通挥砍不消耗Trion，组合技走旋空等芯片的使用消耗 | 续航最长 |
| **枪手** | 使用消耗（射击） | 中频中耗——子弹发射数量决定 | 中等续航 |
| **射手** | 使用消耗（射击+操控） | 高频高耗——分割越多消耗越大 | 续航较短 |
| **狙击手** | 使用消耗（高威力射击） | 低频高耗——单发威力大但消耗高 | 取决于射击频率 |

### 4.4 紧急脱离——消耗最大的系统

紧急脱离（Bail Out）是所有触发器中**Trion消耗最高**的功能。它的高消耗是有意设计的——用巨大的Trion代价换取真身的绝对安全。

> **原因说明**：为什么紧急脱离是消耗最大的系统？因为它执行的是最复杂的操作——将真身从触发体内的近界空间传送到预设安全位置，传送过程中真身处于异空间免疫攻击。这种跨空间传送需要极大的能量支撑。高消耗也是一种平衡机制——如果紧急脱离消耗很低，队员就可以无顾忌地冒险，战斗的紧迫感会大幅降低。

### 4.5 泄漏的物理表现

受伤导致的Trion泄漏有明确的物理表现：

| 属性 | 说明 |
|------|------|
| **形态** | 气态 |
| **颜色** | 绿色→深灰色（中央血管处颜色更深） |
| **持续时间** | 约1分钟后自动停止 |
| **停止机制** | 战斗体的内置保护机制 |
| **已流失部分** | 无法恢复——已消散的Trion不会回来 |

详见"2.3_触发器三层架构过程分析"阶段3。

```
  · 可用量：因四种消耗持续减少
    - 维持消耗（存在税）：持续、不可避免，主要来自战斗体本身
    - 激活消耗（启动税）：每次激活芯片时

关键约束：
  · 泄漏表现为绿色→深灰色气体，约1分钟自动停止

  · 接下来进入阶段5：结算
```

## 阶段5：结算——Trion的"清算"（两条路径）

> 战斗结束或战斗体失效。Trion面临最终的"清算"——它的命运在这里分叉为两条截然不同的路径。

### 5.1 路径A：主动解除（"有序清算"）

使用者主动发出Trigger Off意图，战斗体有序消散：

```
路径A：主动解除的Trion流向

战斗中状态：
┌─────────────────────────────────────┐
│  占用(30%)  │  已消耗(25%) │ 剩余(45%) │
│  ██████████ │  ░░░░░░░░░  │ ██████████│
└─────────────────────────────────────┘
       ↓ 主动解除（Trigger Off）
       ↓ 芯片正常注销，占用量解锁返还
主动解除后：
┌─────────────────────────────────────┐
│  已消耗(25%)  │  可恢复(75%)         │
│  ░░░░░░░░░░  │  ████████████████████│
│  需时间恢复   │  占用量已解锁返还     │
└─────────────────────────────────────┘
```

**关键**：占用量**解锁返还**——之前被芯片锁定的Trion重新变为可用。

### 5.2 路径B：被动破裂（"破产清算"）

战斗体因各种原因被动失效，连同占用的Trion一起流失：

```
路径B：被动破裂的Trion流向

战斗中状态（Trion可用值即将耗尽）：
┌─────────────────────────────────────┐
│  占用(30%)  │  已消耗(65%) │ 剩余(5%)  │
│  ██████████ │  ░░░░░░░░░  │ ██        │
└─────────────────────────────────────┘
       ↓ 被动破裂（Trion耗尽/弱点被毁/主动触发紧急脱离）
       ↓ 战斗体破裂消散，占用量随之流失耗尽
被动破裂后：
┌─────────────────────────────────────┐
│  已消耗(100%)                        │
│  ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░    │
│  占用量也流失耗尽，需长时间恢复       │
└─────────────────────────────────────┘
```

**关键**：占用量**随战斗体一同流失耗尽**——没有机会执行释放流程。

### 5.3 两条路径的完整分叉图

```
两条结算路径的Trion流向分叉：

                    战斗体状态
                   ┌──────────┐
                   │ 占用+可用 │
                   └────┬─────┘
                        │
              ┌─────────┴─────────┐
              ↓                   ↓
     路径A：主动解除        路径B：被动破裂
     ┌──────────────┐     ┌──────────────┐
     │ "有序清算"    │     │ "破产清算"    │
     │              │     │              │
     │ 战斗体消散   │     │ 战斗体破裂   │
     │ 芯片正常注销 │     │ 芯片强制注销 │
     │ 占用量→返还  │     │ 占用量→流失  │
     │ 剩余量→保留  │     │ 全部→耗尽    │
     └──────┬───────┘     └──────┬───────┘
            ↓                     ↓
     ┌──────────────┐     ┌──────────────┐
     │ 恢复起点高   │     │ 恢复起点低   │
     │ 等待时间短   │     │ 等待时间长   │
     └──────────────┘     └──────────────┘
```

### 5.4 两条路径的完整对比

| 对比项 | 路径A：主动解除 | 路径B：被动破裂 |
|--------|---------------|-----------------|
| **触发方式** | 使用者主动发出Trigger Off意图 | Trion耗尽/弱点被毁/主动触发紧急脱离 |
| **战斗体去向** | 消散 | 破裂消散（裂纹扩散→爆炸成烟雾） |
| **芯片注销** | 正常注销 | 强制注销 |
| **占用Trion** | **解锁返还** | **随战斗体一同流失耗尽** |
| **Trion恢复起点** | 较高（占用量+剩余量） | 接近枯竭 |
| **重新激活等待** | 较短 | 较长（低Trion者如三云修需1-2小时） |
| **真身位置** | 原地释放 | 原地释放（无紧急脱离）/ 传送至安全位置（有紧急脱离） |

> **原因说明**：为什么主动解除能返还而被动破裂不能？本质区别是"正常关闭程序"vs"断电崩溃"。主动解除时，系统有时间让芯片正常释放其占用的Trion（类似关闭程序释放内存）；被动破裂时，战斗体连同其中运行的所有芯片一起消散，芯片正在使用的Trion也随之丢失（类似断电崩溃，内存内容全部丢失）。这也是为什么经验丰富的队员会尽量选择主动解除——保留更多Trion意味着更快的恢复和更短的重新激活等待时间。详见"2.2_真身-战斗体-真身领域过程分析"阶段3。

### 5.5 阶段5小结

```
阶段5：Trion已被结算

路径A（主动解除）后的Trion状态：
  · 占用量已解锁返还
  · 战斗消耗部分已消散（不可恢复）
  · 剩余Trion = 占用量 + 未消耗的可用量
  · 恢复起点较高

路径B（被动破裂）后的Trion状态：
  · 占用量随战斗体流失耗尽
  · 全部Trion接近枯竭
  · 恢复起点极低

共同点：
  · 战斗体已消散——所有Trion表现消失
  · 真身已释放——意识转回真身
  · 接下来进入阶段6：恢复
```

## 阶段6：恢复——Trion的"再生"

> 战斗结束，真身回归。Trion器官重新开始工作，通过日常生活缓慢恢复消耗的Trion。

### 6.1 恢复的前提条件

**必须解除战斗体回到真身**。这是恢复的绝对前提——Trion器官在战斗体状态下无法恢复Trion。

恢复方式：日常生活——进食、休息。没有"快速恢复药水"或"充能站"之类的捷径。

### 6.2 恢复时间的影响因素

恢复速度的核心参数是**Trion恢复速率(1c)**——一个与个体Trion天赋相关的影响因子，决定了Trion腺体的产出速率。

| 因素 | 影响方向 | 说明 |
|------|---------|------|
| **消耗量** | 消耗越多，恢复越久 | 与消耗的Trion量成正比 |
| **Trion恢复速率(1c)** | 恢复速率越高，恢复越快 | 个体天生差异，决定腺体产出速率 |
| **个体容量** | Trion容量越大，恢复越久 | 大水库填满需要更多时间 |
| **解除方式** | 被动破裂恢复更久 | 从枯竭状态恢复比从半满状态恢复慢得多 |

> **原因说明**：为什么Trion容量越大恢复时间越长？这是一个反直觉但合理的设定。Trion腺体的产出速率由**Trion恢复速率(1c)**决定，是相对固定的（受个体天赋影响），而恢复时间 = 需恢复量 / 产出速率。高容量者需要恢复的绝对量更大，因此时间更长。这创造了一个有趣的权衡：高Trion者战斗中更持久，但战斗后恢复更慢。

### 6.3 不同场景的恢复时间对比

| 场景 | 解除方式 | 消耗程度 | 恢复时间 | 代表案例 |
|------|---------|---------|---------|---------|
| 轻度战斗后主动解除 | 路径A | 低 | 较短 | 训练赛后 |
| 激烈战斗后主动解除 | 路径A | 中 | 中等 | Rank Wars后主动撤退 |
| 战斗体被摧毁 | 路径B | 高（枯竭） | 较长 | 被击败触发紧急脱离 |
| 低Trion者被摧毁 | 路径B | 高（枯竭） | 1-2小时 | 三云修战斗体被毁 |
| 高Trion者被摧毁 | 路径B | 高（枯竭） | 更长 | 高容量者从枯竭恢复 |

### 6.4 阶段6小结

```
阶段6：Trion正在恢复

Trion的状态：
  · Trion器官重新开始工作
  · 通过日常生活（进食、休息）缓慢恢复
  · 恢复速度取决于：Trion恢复速率(1c)、消耗量、个体容量、解除方式

关键约束：
  · 必须在真身状态下才能恢复
  · 没有快速恢复手段
  · 高容量者恢复时间更长（大水库填满需要更多时间）
  · 接下来进入阶段7：再循环与战斗外应用
```

## 阶段7：再循环与战斗外应用——Trion的"更广阔世界"

> Trion恢复完成后，可以再次激活触发器进入新一轮循环。但Trion的用途远不止战斗——它是整个近界文明的基石。

### 7.1 再循环——每次都是全新开始

恢复完成后，队员可以再次激活触发器。**每次激活都是全新构筑一具战斗体**——上一次战斗体的伤害、状态完全不存在，新的战斗体总是以完好状态开始。

这意味着Trion的生命周期是一个**循环**，而非线性过程：

```
产生 → 储存 → 分配 → 消耗 → 结算 → 恢复 → 再次分配 → ...
  ↑                                              │
  └──────────────── 循环 ─────────────────────────┘
```

### 7.2 Trion在战斗之外的六大应用领域

Trion不仅是战斗能量，更是近界文明的基础资源。以下穷举Trion在战斗之外的六大应用领域：

**1. 交通运输——远征船燃料**

| 属性 | 说明 |
|------|------|
| 用途 | 远征船（Away Ship）的燃料 |
| 消耗规律 | 船体大小与Trion消耗成正比 |
| 设计影响 | 为节省Trion，船舱设计紧凑 |
| 特殊案例 | 可利用高Trion者（如千佳）增加船体容量和燃料储备 |
| 补给方式 | 远距离航行时需在近界各国停靠补充Trion |

**2. 空间传送——门与紧急脱离**

| 类型 | 说明 |
|------|------|
| 门（Gate） | 连接地球与近界的次元通道，近界各国接近地球时可开启 |
| 紧急脱离 | 将真身传送到安全位置，是Trion消耗最高的触发器功能 |
| 身体存储 | 激活战斗体时真身存储到触发体内的近界空间 |

**3. 星球维持——母触发器/"神"系统**

| 属性 | 说明 |
|------|------|
| 核心设备 | 母触发器（Mother Trigger）/ 星触发器 |
| 功能 | 维持近界星球国家的存在 |
| 供能方式 | 必须由高Trion容量的个体（称为"神"）持续供能 |

## 特殊情况：打破常规的Trion流动

> 以上阶段0-7描述的是Trion的"标准生命周期"。但存在两个重要的例外，它们打破了某些常规规则。

### 游真的永久战斗体——可在战斗体状态下恢复Trion

空闲游真（Yūma Kuga）是**唯一可以在战斗体状态下恢复Trion**的已知案例。

| 属性 | 说明 |
|------|------|
| **背景** | 游真在战斗中身受致命伤，父亲用生命创造黑触发器封存其濒死真身 |
| **打破的规则** | "战斗体状态下Trion器官无法恢复"——游真可以 |
| **原因** | 其肉体处于濒死状态被黑触发器封存，永久战斗体成为其唯一存在形式 |
| **其他特殊性** | 自我修复能力（唯一）、耐久度低于标准、无需睡眠、身体停止生长 |

> **原因说明**：为什么这是"例外"而非"规则"？因为游真的情况需要极其特殊的条件——创造者（父亲）牺牲生命创造黑触发器，且使用者（游真）的真身处于濒死状态。这不是一种可复制的技术，而是一次不可重现的奇迹。它证明了Trion系统的规则并非绝对，但打破规则的代价极其高昂。

### 黑触发器——持有期间的永久上限扩容

黑触发器使用时，创造者的结晶化Trion会**永久扩增使用者的Trion上限**（持有期间有效）：

```
黑触发器的Trion扩容机制：

普通状态：
┌──────────────┐
│  自身Trion    │
│  ████████    │
│  容量:7       │
└──────────────┘

持有黑触发器后（上限永久扩容）：
┌──────────────┬──────────────────────────────┐
│  自身Trion    │  黑触发器扩容部分（可恢复填充）│
│  ████████    │  ██████████████████████████████│
│  容量:7       │  +30（创造者的结晶化Trion上限）│
└──────────────┴──────────────────────────────┘

消耗顺序：优先消耗黑触发器扩容部分的Trion
恢复机制：自身Trion恢复满后，可继续填充扩容部分
```

| 属性 | 说明 |
|------|------|
| **机制** | 创造者的Trion被"结晶化"保存在触发器中，作为上限扩容 |
| **扩容效果** | 持有期间永久扩增使用者的Trion上限 |
| **消耗优先级** | 优先消耗黑触发器扩容部分的Trion |
| **可恢复性** | **可恢复**——自身Trion恢复满后，可继续填充扩容部分 |
| **超越原主** | 输出可高于创造者原有水平 |
| **限制** | 需要与创造者人格兼容才能激活 |

> **原因说明**：为什么黑触发器的Trion"优先被消耗"？这是一个合理的设计——扩容部分的Trion是"外来能量"，系统优先使用外来能量，保护使用者自身的Trion储备。与之前版本描述不同的关键点是：黑触发器的扩容不是"一次性消耗品"，而是**持有期间的永久上限扩容**——消耗后可以通过正常恢复机制重新填充，就像扩容后的水库可以重新蓄水一样。

### 每个阶段的一句话总结

| 阶段 | 一句话总结 |
|------|-----------|
| **阶段0：本质** | Trion是具有能量-物质二象性的生物能量，需要媒介才能转化，量（容量）和质（输出功率）是两个独立维度，有三个派生属性（天赋、输出功率、恢复速率） |
| **阶段1：产生** | Trion由Trion腺体代谢转化产生，20岁前可训练增长，个体差异巨大（参数2-38） |
| **阶段2：储存与增强** | Trion储存在腺体中，可通过训练/触发角/黑触发器（持有期间永久扩容）三种路径增强容量 |
| **阶段3：分配** | 激活触发器后Trion被分为占用量和可用量，从此刻起不可恢复 |
| **阶段4：消耗** | 战斗中Trion因维持/激活/使用/流失四种消耗持续减少，全部不可逆 |
| **阶段5：结算** | 主动解除返还占用量（有序清算），被动破裂全部流失（破产清算） |
| **阶段6：恢复** | 回到真身后通过日常生活恢复，高容量者恢复时间更长 |
| **阶段7：再循环** | 恢复后可再次激活（全新构筑），Trion还有六大战斗外应用领域 |
| **特殊情况** | 游真可在战斗体状态下恢复（唯一例外），黑触发器持有期间永久扩容（可恢复填充，优先消耗） |
