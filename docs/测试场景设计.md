# BorderDefenseProtocol模组测试场景设计

## 测试环境准备

### 基础配置
- **游戏版本**: RimWorld 1.6.4633
- **难度设置**: 建议使用"冒险家"或"血与尘"难度
- **讲述者**: 建议使用Cassandra Classic（稳定袭击节奏）
- **开发者模式**: 必须开启
- **必备Mod**: 仅BDP模组（避免其他战斗Mod干扰）

### 测试用控制台命令
```
# 快速生成敌人
调试工具栏 -> Spawn -> Pawn -> 选择派系和种类

# 快速推进时间
调试工具栏 -> Time -> +1 day / +1 hour

# 查看Pawn状态
调试工具栏 -> View Settings -> Write pawn stats

# 治疗/恢复
调试工具栏 -> Health -> Heal all / Remove hediff
```

## 场景1：早期入门测试（游戏开局1-5天）

### 测试目标
- 验证战斗体激活流程的流畅性
- 测试基础战斗能力（小行星芯片）
- 评估早期Trion消耗的合理性
- 检验新手友好度

### 场景设置
**殖民者配置**:
- 1名殖民者，已安装Trion腺体基因
- 装备：BDP_TriggerBody_Test（触发体）
- 芯片：小行星芯片（左手槽位1）
- 其他装备：无护甲，普通衣物

**敌人配置**:
- 3名部落弓箭手
- 距离：20-25格
- 掩体：无

**地形**:
- 平原，无障碍物
- 测试纯粹的战斗能力

### 测试步骤
1. 激活战斗体（检查激活时间和动画）
2. 使用小行星芯片射击敌人
3. 记录击杀时间和弹药消耗
4. 观察Trion消耗速度
5. 战斗结束后观察Trion恢复速度

### 评估指标
| 指标 | 目标值 | 记录值 | 评价 |
|------|--------|--------|------|
| 激活战斗体耗时 | <3秒 | | |
| 3敌人击杀时间 | 15-30秒 | | |
| Trion消耗 | <100点 | | |
| 受伤情况 | 轻伤或无伤 | | |
| 操作流畅度 | 主观评分1-10 | | |

### 预期结果
- 战斗体激活流畅，无卡顿
- 能够轻松击败早期弱敌
- Trion消耗在可控范围内
- 新手能够理解基本操作

### 常见问题检查
- [ ] 激活战斗体后装备是否正确替换？
- [ ] 芯片是否能正常切换？
- [ ] Trion消耗提示是否清晰？
- [ ] 战斗体解除后装备是否恢复？

- 毒蛇芯片命中率更高（引导机制）
- 小行星芯片伤害更高（9 vs 5）

## 场景4：持续战斗Trion管理测试（游戏中期20-30天）

### 测试目标
- 测试长时间战斗的Trion消耗
- 验证Trion恢复机制
- 评估战斗体维持成本的合理性
- 检验Trion枯竭后的应对能力

### 场景设置
**殖民者配置**:
- 1名殖民者
- 装备：BDP_TriggerBody_Test
- 芯片：小行星芯片 + 蚱蜢芯片
- Trion状态：满Trion（700点）

**敌人配置**:
- **第一波**：5名海盗枪手（间隔30秒）
- **第二波**：5名海盗枪手（间隔30秒）
- **第三波**：5名海盗枪手
- 总计：15名敌人，持续战斗约5-10分钟

**地形**:
- 防御阵地，有掩体

### 测试步骤
1. 激活战斗体，Trion=700
2. 应对第一波敌人，记录Trion消耗
3. 短暂休整（30秒），观察Trion恢复
4. 应对第二波敌人，记录Trion消耗
5. 短暂休整（30秒），观察Trion恢复
6. 应对第三波敌人，记录Trion消耗
7. 观察是否出现Trion枯竭

### 评估指标
| 指标 | 第一波 | 第二波 | 第三波 | 总计 |
|------|--------|--------|--------|------|
| 战斗时间 | | | | |
| Trion消耗 | | | | |
| Trion剩余 | | | | |
| 是否枯竭 | | | | |

### 预期结果
- 单波战斗Trion消耗<200点
- 短暂休整能恢复部分Trion
- 三波战斗后Trion接近枯竭但不会完全耗尽
- Trion管理成为战术考量因素

### 常见问题检查
- [ ] Trion恢复速度是否合理？
- [ ] 战斗体维持成本是否过高？
- [ ] Trion枯竭警告是否及时？
- [ ] 枯竭后是否有应对手段？

## 场景5：破裂机制测试（游戏中期30-40天）

### 测试目标
- 验证战斗体破裂触发条件
- 测试破裂期间的无敌机制
- 评估破裂后的debuff影响
- 检验紧急脱离芯片的效果

### 场景设置
**殖民者配置**:
- 1名殖民者
- 装备：BDP_TriggerBody_Test
- 芯片：
  - 左手槽位1：小行星芯片
  - 特殊槽位1：紧急脱离芯片（可选）
- Trion状态：满Trion

**敌人配置**:
- 10名海盗枪手（高火力）
- 距离：15-20格
- 掩体：无（故意让殖民者暴露）

**地形**:
- 开阔地形
- 如果测试紧急脱离，需要放置紧急脱离信标

### 测试步骤
1. 激活战斗体
2. 故意暴露在敌人火力下
3. 等待关键部位被破坏（头部/躯干）
4. 观察破裂触发（90 ticks延时）
5. 记录破裂期间的状态
6. 观察破裂后的debuff效果
7. 如果装备紧急脱离芯片，验证传送效果

### 评估指标
| 指标 | 目标值 | 记录值 | 评价 |
|------|--------|--------|------|
| 破裂触发时间 | 90 ticks | | |
| 破裂期间是否无敌 | 是 | | |
| 破裂后移动速度 | 80% | | |
| 破裂后操作能力 | 80% | | |
| 紧急脱离是否成功 | 是 | | |

### 预期结果
- 关键部位破坏后90 ticks触发破裂
- 破裂期间完全无敌
- 破裂后有明显debuff但不至于无法行动
- 紧急脱离芯片能成功传送

### 常见问题检查
- [ ] 破裂触发是否稳定？
- [ ] 破裂期间是否真的无敌？
- [ ] 破裂后debuff持续时间是否合理？
- [ ] 紧急脱离芯片是否消耗Trion？

## 测试技巧和注意事项

### 1. 测试前准备
- **备份存档**：每次测试前备份存档，避免数据丢失
- **清理日志**：清空RimWorld日志文件，方便查看错误
- **关闭其他Mod**：避免其他Mod干扰测试结果
- **准备录屏工具**：记录关键战斗场景

### 2. 测试中注意
- **单变量原则**：每次只改一个数值，避免混淆
- **多次重复**：同一场景至少测试3次，确保结果稳定
- **记录细节**：记录所有异常现象，即使看起来不重要
- **保持客观**：避免主观偏见，用数据说话

### 3. 测试后分析
- **对比数据**：与目标值对比，找出偏差
- **分析原因**：思考为什么会出现这个结果
- **提出方案**：给出具体的调整建议
- **验证修复**：修改后重新测试，确认问题解决

### 4. 常用开发者工具

#### 调试工具栏快捷操作
```
Spawn（生成）:
- Spawn pawn：生成敌人/殖民者
- Spawn thing：生成物品/装备
- Spawn weapon：生成武器

Health（健康）:
- Heal all：治疗所有伤害
- Remove hediff：移除特定Hediff
- Add hediff：添加特定Hediff

Time（时间）:
- +1 hour：快进1小时
- +1 day：快进1天
- Speed 4：4倍速

View Settings（查看设置）:
- Write pawn stats：输出Pawn详细数据
```

- 使用成本：每次调整±2-5点
