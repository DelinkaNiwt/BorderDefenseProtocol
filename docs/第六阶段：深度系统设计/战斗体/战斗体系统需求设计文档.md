# 战斗体系统需求设计文档 v4

## 前言

本文档是BDP模组战斗体系统的第4版需求设计文档。

**战斗体是什么**：
战斗体（Combat Body） 是BDP模组中一个由 Trion能量 构成的临时替代躯体系统                         
用一句话描述：玩家消耗Trion能量，将自己的殖民者包裹在一副Trion构成的"战斗用替身躯体"内，用这个替身去承受所有伤害，而真身毫发无损地保存在"快照"里。


**设计原则**（同v3）：
1. **有选择性的机制移植**：不是100%还原原作，而是为RimWorld游戏体验服务
2. **利用现有机制**：RimWorld原生Hediff系统可覆盖大部分战斗体特性
3. **增加战术深度**：激活/解除引导时间、两条解除路径的Trion结算差异、冷却CD
4. **保持游戏平衡**：不实现非Trion免疫、不修复缺陷、维持消耗+冷却防止滥用
5. **生理/心理分离**：生理冻结回滚，心理正常运行保留
6. **最简原则**：保留完整人体结构拦截致死逻辑，不做高风险的躯体结构替换

**关键设计决策**（v4确认，v4.1调整）：
- 原版HP系统对战斗体**完全无意义**，所有伤害均被拦截，不进入原版HP体系
- 使用**影子HP系统**独立追踪部位受损状态（不依赖原版HP）
- **伤口Hediff**不继承原版伤口类，不触发出血、疼痛等原版伤口机制
- 战斗体**激活时强制征召**，但**解除时保持当时状态**（不强制改变）
- **倒地即破裂**（🆕 v4.1）：战斗体状态下任何倒地事件等同于被动破裂触发
- **伪造“身体替换”的实质，只做表象**，世界观设定上是用一副战斗体躯壳替换原本的肉身，但实现逻辑上决策为只实现更换战斗体后的所影响的表象，不实现“更换身体”这个行为实质

**变更标记说明**：
- 🆕 = 该版本新增的需求
- ✏️ = 该版本修改的需求（相比上版有实质变化）
- ✅ = 上版已有，该版未变
- （标注末尾括号为首次引入该需求的版本）

## 一、核心价值定位

### NR-001: 完全伤害隔离 ✅（v1）
- 战斗体受伤完全不影响真身
- 解除后真身回到激活时的状态
- 这是战斗体系统的根本价值，不可妥协

## 二、激活机制

### NR-002: 激活前置条件 ✅（v1）
- 装备触发体（触发器装备）
- 当前Trion ≥ 激活所需的占用量
- 不在战斗体状态下
- 不在冷却CD中（详见NR-038）

### NR-003: 激活过程 ✅（v1）
- 引导时间：1-3秒
- 引导期间可被打断（受到伤害/被击倒/失去意识等）
- 引导被打断 → 激活失败，无任何效果发生
- 引导完成后立即进入战斗体状态

### NR-004: 激活效果 ✏️（v4补充强制征召+外观Apparel；v4.1去除技术实现标注）
激活完成时以下操作作为**一套原子操作**同时执行：

1. **真身状态快照**：记录真身当前的完整生理状态（详见NR-026）
2. **Hediff替换**：移除所有现有Hediff，添加"战斗体Hediff"（详见NR-027）
3. **衣物隐藏**：隐藏当前穿戴的衣物，转移至快照容器（详见NR-028）
4. **物品冻结**：将背包物品转移至快照容器（详见NR-028）
5. **需求冻结**：禁用生理类需求（详见NR-029）
6. **芯片注册**：已装载的触发芯片向Trion资源池注册，锁定占用量
7. **状态切换**：进入战斗体状态
8. **冻结Trion恢复**：停止Trion自然恢复
9. **注册维持消耗**：注册战斗体基础维持消耗（详见NR-034）
10. **强制征召**（🆕 v4）：将Pawn置为征召状态
11. **生成战斗体外观**（🆕 v4）：Spawn战斗体专属Apparel并穿戴（详见NR-028）

**关键约束**：
- 激活被打断 → 上述操作均不执行（快照不生成）
- 引导过程中受伤 → 伤口和状态在快照时一起被记录（快照在激活完成瞬间生成）
- **影子HP初始化**（🆕 v4）：激活完成时同步初始化影子HP系统（详见NR-041）

```
激活流程：

  [前置检查] ─ 条件不满足 ─→ 失败（提示）
      │ 条件满足
      ▼
  [引导开始] ─ 被打断 ─→ 失败（无任何效果）
      │ 引导完成
      ▼
  [原子操作] ← 以下步骤不可分割、不可打断
      ├─ 快照（Hediff + 需求值 + 衣物 + 物品）
      ├─ 移除所有Hediff → 添加战斗体Hediff
      ├─ 衣物 + 物品 → 转移至快照容器
      ├─ 禁用生理需求
      ├─ 芯片注册（锁定占用量）
      ├─ 冻结Trion恢复
      ├─ 注册维持消耗
      ├─ 强制征召
      ├─ Spawn战斗体Apparel并穿戴
      └─ 初始化影子HP系统
      │
      ▼
  [战斗体状态]
```

## 三、解除机制

### 3.1 主动解除（Trigger Off）

#### NR-005: 主动解除触发条件 ✅（v1）
- 玩家主动发出解除指令

#### NR-006: 主动解除过程 ✏️（v4.1去除API调用标注）

1. **引导时间**：与激活类似，有短暂引导，引导期间可被打断
2. 引导被打断 → 仍在战斗体状态，解除失败，需重新尝试
3. 引导完成后执行以下步骤：
4. 战斗体状态结束
5. 芯片注销：占用的Trion释放，返还至可用量
6. 恢复Trion自然恢复
7. 注销所有持续消耗源（含伤口流失、维持消耗等）
8. **清理伤口Hediff**（🆕 v4）：移除所有战斗体伤口Hediff，注销对应流失（详见NR-042）
9. **Hediff回滚**：移除所有战斗体期间的Hediff（包括战斗体Hediff），从快照恢复原有Hediff
10. **需求恢复**：重新添加被禁用的需求，恢复到快照记录的值
11. **衣物恢复**：从快照容器恢复穿戴衣物
12. **物品恢复**：从快照容器恢复背包物品
13. **销毁战斗体外观Apparel**（🆕 v4）：移除并销毁激活时Spawn的外观Apparel
14. 进入冷却CD（详见NR-038）

#### NR-007: 主动解除Trion结算 ✅（v1）
- 占用量返还到可用量
- 已消耗的Trion不返还
- 玩家可以较快重新激活（受CD限制）

#### NR-037: 解除引导时间 ✅（v3）
- 主动解除与激活类似，有短暂引导时间（具体时长在详细设计阶段确定）
- 引导期间可被打断（受到伤害/被击倒/失去意识等）
- 引导被打断 → 解除失败，仍在战斗体状态，需重新尝试
- 引导不消耗Trion，打断无任何副作用

### 3.2 被动破裂

#### NR-008: 被动破裂触发条件 ✏️（v4新增离开地图；v4.1新增倒地）
以下任一条件满足时触发：
1. Trion可用量耗尽（Available ≤ 0）
2. 头部被击毁（拦截原版死亡结果）
3. 心脏被击毁（拦截原版死亡结果）
4. 玩家主动触发紧急脱离（战术撤退）
5. **Pawn离开当前地图**（🆕 v4，详见NR-043）
6. **Pawn倒地**（🆕 v4.1，详见NR-036）

**优先级规则**（🆕 v4）：
- 主动解除引导期间，若同时满足被动破裂条件（如Trion耗尽） → **被动破裂优先**，中断主动解除，立即走被动破裂路径

#### NR-009: 被动破裂过程 ✏️（v4补充清理伤口Hediff；v4.1去除API标注）
1. 战斗体破裂（视觉效果：裂纹扩散→消散）
2. Trion全部强制清零（含已占用量），不返还
3. **清理伤口Hediff**（🆕 v4）：移除所有战斗体伤口Hediff，注销对应流失
4. 紧急脱离判定：
   - 如果装备了紧急脱离芯片 且 存在传送目标建筑：传送真身到建筑旁
   - 如果装备了紧急脱离芯片 但 无传送目标建筑：回退到基地附近安全位置
   - 如果没有装备紧急脱离芯片：真身在原地释放
5. 恢复Trion自然恢复
6. 注销所有持续消耗源
7. **Hediff回滚**：移除所有战斗体期间的Hediff（含战斗体Hediff），从快照恢复原有Hediff
8. **需求恢复**：重新添加被禁用的需求，恢复到快照记录的值
9. **衣物恢复**：从快照容器恢复穿戴衣物
10. **物品恢复**：从快照容器恢复背包物品
11. **销毁战斗体外观Apparel**（🆕 v4）：移除并销毁激活时Spawn的外观Apparel
12. **添加枯竭debuff**（详见NR-040）
13. 进入冷却CD（详见NR-038）

#### NR-010: 被动破裂Trion结算 ✅（v1）
- 所有Trion清零（cur = 0, allocated = 0）
- 需要长时间恢复才能重新激活

### 3.3 特殊解除：离开地图（安全网）

#### NR-043: 离开地图安全处理 ✏️（v4新增；v4.1简化——倒地场景已由NR-036覆盖）
战斗体激活状态下Pawn离开当前地图时的安全网处理。

**触发时机**：离开地图事件，且战斗体处于激活状态。

**处理方式**：立即触发被动破裂（NR-009）。

**有紧急脱离时**：复用NR-014传送机制。

**无紧急脱离时**：真身随Pawn离开，走原生处理流程。

> 注：战斗体倒地即破裂规则（NR-036）已覆盖大多数被俘场景（倒地时战斗体已破裂，真身以普通状态被搬走）。NR-043作为安全网处理非倒地情况下的边缘案例。

## 四、弱点系统与死亡拦截

### NR-011: 弱点定义 ✅（v1）
- **头部**：Trion中继中心（概念层面）
- **心脏**：Trion供给系统（概念层面）

以上两种情况直接触发被动破裂（NR-008）。

### NR-012: 致命伤拦截——弱点被毁 ✅（v2）
- 当**头部或心脏**被毁时 → 拦截原版死亡结果 → 触发被动破裂流程（NR-009）
- 真身不会死亡，安全释放

### NR-035: 非弱点器官被毁处理 ✏️（v4.1：双腿被毁改为直接破裂；去除技术实现标注）
战斗体保留完整的人类身体结构（不做躯体结构替换）。各类部位被毁时的处理：

**双腿同时被毁**：
- 直接触发被动破裂（导致倒地，而倒地即破裂，详见NR-036）

**其他非弱点器官被毁**（肾、肝、胃等）：
- **不触发原版死亡逻辑**：战斗体是Trion构成的躯体，不依赖这些器官维持运作
- **注册高额Trion流失**：该器官被毁后向Trion系统注册高额持续流失
- **间接导致破裂**：高流失会在短时间内使Trion可用量耗尽，触发NR-008条件1

```
战斗体死亡拦截决策树：

  [战斗体受到伤害]（已被拦截，走NR-042管道）
        │
  [影子HP判断部位是否被毁] ── 未被毁 ──→ [f(x)+伤口Hediff，正常流程]
        │ 被毁
        ▼
  [哪个部位？]
        │
  ┌─────┼──────┬────────────────────────────────┐
  │     │      │                                │
 头部  心脏  双腿同时                      其他重要器官
  │     │      │                           （肾、肝、胃等）
  └─────┴──────┘                                │
         │                                      │
  [直接触发被动破裂]                       [阻止死亡]
   (NR-008/009)                            [注册高额流失]
                                           [间接→Trion耗尽→破裂]
```

## 五、紧急脱离系统

### NR-013: 紧急脱离芯片 ✅（v1）
- 类型：触发芯片（特殊类型）
- 槽位：只能插在触发器的"特殊槽位"，不能插在"左右槽位"
- 效果：战斗体破裂时自动传送真身到安全位置

### NR-014: 传送目标与回退规则 ✅（v3）
**主传送目标**：
- 新增自定义建筑（如"紧急脱离信标"或"战略传送点"）
- 玩家在基地建造此建筑作为传送坐标
- 有多个建筑时传送到最近的一个

**回退规则**（建筑缺失时）：
- 采用与轨道贸易空投同类的多级回退逻辑：
  1. 殖民者建筑附近寻找安全位置
  2. 地图上随机可站立未遮蔽位置
- 确保真身不会释放在极端危险位置

### NR-015: 紧急脱离触发 ✅（v1）
- 触发条件：战斗体被动破裂时自动触发  / 玩家通过按钮“主动触发”
- 传送效果：
  - 真身传送到传送目标（建筑旁或回退位置）
  - 传送过程中真身无敌（不可被攻击）
  - 传送完成后真身出现在目标位置

### NR-016: 紧急脱离Trion消耗 ✅（v1）
- 紧急脱离触发时Trion已经清零（被动破裂路径）
- 不需要额外消耗Trion

## 六、战斗体特性

### NR-017: 耐久度统一 ✅（v3）
**含义**：所有战斗体在受伤层面完全相同。

- 相同攻击、相同条件下，对不同战斗体的判定结果完全一致
- 伤口效果、Trion流失速率对所有战斗体一视同仁
- **差异仅体现在Trion池大小**：Trion多的人受同样的伤后还能维持更久，Trion少的人更快破裂
- 不通过Trion量缩放任何战斗体属性

```
耐久度统一示意：

  Trion 1000的战斗员          Trion 100的战斗员
  ┌──────────────┐            ┌──────────────┐
  │ 手臂受50伤   │            │ 手臂受50伤   │
  │ → 影子HP-50 ✓│            │ → 影子HP-50 ✓│ ← 完全相同
  │ → 伤口记录  │            │ → 伤口记录  │ ← 完全相同
  │ → 流失/天   │            │ → 流失/天   │ ← 完全相同
  │              │            │              │
  │ 剩余可用:950 │            │ 剩余可用: 50 │ ← 仅此不同
  │ 还能战斗很久 │            │ 即将破裂     │
  └──────────────┘            └──────────────┘
```

### NR-018: 无限体力 ✏️（v4.1去除技术实现标注）
- 战斗体状态下不消耗体力

### NR-019: 疼痛免疫 ✏️（v4.1去除技术实现标注）
- 战斗体状态下免疫疼痛效果

### NR-020: 无法自愈 ✏️（v4.1去除技术实现标注）
- 战斗体受伤后不会自然愈合
- 只能通过解除重建来"治疗"

### NR-021: 不实现非Trion免疫 ✅（v1）
- 战斗体对所有武器都是正常受伤的
- 出于游戏平衡考虑

### NR-022: 不实现缺陷修复 ✅（v3）
- 战斗体**镜像真身的身体缺陷**：真身缺一条腿，战斗体也缺腿
- 不是"战斗体全新完美躯体"——是"和真身一样的Trion复制体"
- 出于游戏平衡和实现简洁性考虑
- **植入物例外**（🆕 v4）：植入物覆盖的缺失部位视为完整（详见NR-027）

### NR-034: 战斗体维持消耗 ✏️（v4.1去除技术实现标注）
- 战斗体存在即持续消耗Trion（"存在税"）
- 定位为**低背景噪声**——单独不构成紧迫压力
- 主要目的：防止玩家长期维持战斗体来规避真身的生理需求
- 战斗中的压力主要来自使用消耗和伤口产生的流失消耗，维持消耗只是底噪
- 激活时向Trion系统注册该持续消耗，解除时注销

## 七、战斗体状态下的限制与行为

### NR-023: Trion无法恢复 ✏️（v4.1去除技术字段标注）
- 战斗体状态下Trion自然恢复被冻结
- Trion只减不增，一切消耗不可逆

### NR-024: 无法使用原版治疗 ✅（v1）
- 战斗体状态下无法使用医疗床/药物治疗
- 因为伤害不影响真身，治疗无意义

### NR-025: 生理状态冻结 ✅（v1）
- **原则**：只冻结生理层面，不冻结心理层面
- 战斗体状态下没有进食/睡眠等**生理**需求
- 心理层面（心情、思想、技能、人际关系）正常运行
- 解除时**生理**状态回滚到激活时的快照
- 心理状态不回滚（记忆/经验保留）
- 详细分类见NR-029~NR-033

### NR-036: 击倒状态处理 ✏️（v4.1完全重写：倒地即破裂）
战斗体状态下Pawn倒地（Downed）时的行为：

- **任何原因导致的倒地，立即触发被动破裂（NR-008条件6 → NR-009流程）**
- 不存在"倒地但战斗体仍维持"的中间状态

**设计意图**：
- 战斗体的概念是意识高度集中、全力运作的状态——倒地本身就代表控制崩溃
- 双腿被毁、意识降低至倒地阈值等情况均视为战斗体失控，统一走破裂路径
- 简化了被俘场景（倒地瞬间已破裂，真身以普通状态被搬走）

> 注：由于疼痛免疫（NR-019）、生理需求冻结（NR-029）、环境免疫（NR-033），战斗体状态下能导致倒地的原因已大幅减少。主要风险来自头部受损（已由NR-012覆盖）和双腿同时被毁（已由NR-035更新覆盖）。

### NR-039: 战斗体征召行为 ✏️（v4明确激活强制征召与解除保持；v4.1去除字段标注）
- **激活时**：强制征召，由NR-004步骤10执行
- **战斗体状态中**：玩家可自由切换征召/自由状态，无限制
- **解除时**：保持解除前的征召状态（不强制改变）
  - 解除前是征召 → 解除后仍是征召（保持征召，等待玩家命令）
  - 解除前是自由 → 解除后仍是自由（自动恢复工作队列，相当于"快速征召+取消"）
- 快照**不需要记录**征召状态（解除后保持当时状态，无需恢复）
- 生理需求已禁用，自由状态下不会产生进食/睡觉的AI行为
- Trion维持消耗是天然的时间约束

## 八、真身状态快照系统

### NR-026: 快照内容 ✅（v2）
快照记录以下数据：

| 记录项 | 说明 | 回滚方式 |
|---|---|---|
| **Hediff列表** | 所有伤口、疾病（含阶段和进度）、永久伤残、植入物等 | 解除时从快照恢复 |
| **需求值** | 被冻结的生理需求的当前数值（饥饿值、休息值、舒适度等） | 解除时恢复到快照值 |
| **衣物列表** | 当前穿戴的所有衣物及其状态 | 解除时恢复穿戴 |
| **物品列表** | 背包（Inventory）中的所有物品 | 解除时恢复到背包 |

**不记录的数据**（心理层面，正常运行不回滚）：
- 心情/思想/记忆
- 技能经验
- 人际关系
- 工作设置/时间表
- 征召状态（🆕 v4：解除后保持当时状态，无需快照）

**快照归属**：属于Pawn，需持久化存档

### NR-027: Hediff替换策略 ✏️（v4补充植入物处理规则）
- **激活时**：记录快照 → 移除所有现有Hediff → 添加"战斗体Hediff"
- **解除时**：移除所有战斗体期间的Hediff（含战斗体Hediff本身）→ 从快照恢复原有Hediff
- 两者无缝衔接
- "战斗体Hediff"是一个特殊Hediff，通过其属性实现大部分战斗体特性

**植入物处理规则**（🆕 v4）：
- 植入物Hediff（义肢、器官替换等）与其他Hediff一同存入快照
- **影子HP初始化时**：植入物覆盖的缺失部位，以该部位的**原生最大HP**初始化影子HP（视为完整部位）
- 战斗体生成后该部位不缺失，可正常受伤
- 解除时随快照恢复，植入物重新生效

```
植入物处理示意：

  真身（有钢腿义肢）             战斗体
  ┌─────────────────┐           ┌─────────────────┐
  │ 左腿：缺失(残肢) │  激活     │ 左腿：完整       │
  │ + 钢腿Hediff    │ ──────→  │ 影子HP = 最大HP  │
  │（义肢覆盖缺失）  │           │（义肢=完整部位） │
  └─────────────────┘           └─────────────────┘
  快照中保存：[缺失Hediff, 钢腿Hediff, ...]
```

### NR-028: 衣物、物品与外观 ✏️（v4具体化MVP外观；v4.1去除内部类名）
**衣物**：
- 激活时：从Pawn身上移除，转移至快照容器
- 解除时：从快照容器恢复穿戴

**物品（Inventory）**：
- 激活时：从背包（Inventory）转移至快照容器
- 解除时：从快照容器转移回背包
- 物品物理转移后，游戏引擎天然不会让AI使用不存在的物品，无需额外拦截

**战斗体外观**（🆕 v4具体化）：
- MVP实现：Spawn战斗体专属Apparel穿戴到Pawn上（改变视觉外观）
- 激活时Spawn，解除时销毁（不存入快照，不持久化）
- 后续版本：可通过专属建筑配置每个Pawn的战斗体外观

**武器不是问题**：激活战斗体的前提是手持触发体

### NR-031: 精神崩溃阻止 ✏️（v4.1去除技术实现标注）

| 原版概念 | 战斗体映射 | 相关需求 |
|---|---|---|
| 原版HP减少 | **无效**（伤害全部拦截，不进入原版HP体系） | NR-041 |

| 流血（持续） | 伤口持续造成Trion流失 | NR-042 |

| 出血（视觉） | 禁止 | — |
| 疼痛 | 禁止 | — |

> 完整伤害管道详见NR-041（影子HP系统）和NR-042（伤口Hediff规范）。

### NR-033: 环境免疫 ✅（v3）

- 中毒（免疫毒素）
- 疾病（不会感染新疾病，已有疾病进度冻结于快照）

**关于疾病冻结的exploit**（已确认接受）：

- 世界观自洽——真身在快照中，疾病没理由继续恶化

## 十、冷却与惩罚机制

### NR-038: 激活/解除冷却CD ✅（v3）

- 主动解除和被动破裂均触发CD

- 进一步拉大两条解除路径的惩罚差异

## 十一、Trion消耗模型总览

以下汇总战斗体状态下所有Trion消耗来源：

| 消耗类型 | 触发时机 | 可控性 | 停止条件 | 来源需求 |
|---------|---------|--------|---------|---------|
| **占用消耗** | 激活时芯片注册 | 配置决定 | 解除时释放（主动）或流失（被动） | NR-004 |
| **维持消耗** | 战斗体存在即消耗 | 不可避免 | 解除时停止 | NR-034 |
| **使用消耗** | 芯片使用（射击等） | 可控 | 停止使用 | 触发器模块 |
| **直接消耗** | 受伤时，每次拦截 f(x) | 被动 | 伤害停止 | NR-042 |
| **流失消耗** | 伤口的持续流失，y点/天 | 被动 | 解除战斗体时清理 | NR-042 |
| **器官被毁流失** | 非弱点器官被毁 | 被动 | 持续至破裂 | NR-035 |

> 注：v4将"流失消耗"拆分为"直接消耗f(x)（瞬时）"和"伤口流失y（持续）"两种，通过解除战斗体清理伤口记录来停止持续流失。

## 十四、MVP范围

6. **战斗体特性** (NR-017~022, NR-034)

   - 不修复缺陷（镜像真身），植入物视为完整部位

7. **伤害管道** (NR-041, NR-042)
   - 影子HP系统
   - 伤口记录（合并机制 + 流失注册）

8. **生理/心理分类** (NR-029~033)
   - 生理需求禁用
   - 心理需求正常（含Joy/Beauty/Outdoors）
   - 伤害机制映射
   - 环境免疫

9. **状态处理** (NR-036, NR-039, NR-040)
   - 倒地即破裂
   - 征召行为（激活强制/解除保持）
   - 枯竭debuff

10. **Trion消耗与恢复限制** (NR-023~025)

    - 无法使用原版治疗

## 十六、伤害管道详细机制

### NR-041: 影子HP系统 ✏️（v4新增；v4.1去除数据结构细节，增加序列化要求）

战斗体使用独立的影子HP系统追踪部位受损，**不依赖原版HP**。

**核心概念**：
- 为每个身体部位独立维护一个"当前耐久值"
- 不继承真身当前伤势：每次激活以各部位满状态出发
- **植入物处理**：植入物覆盖的缺失部位，以该部位原生最大HP初始化（视为完整部位，详见NR-027）

**初始化规则**：
- 每个部位从**该部位原生最大HP**开始
- 影子HP随受伤递减，不低于0

**两个用途**：

| 用途 | 判断条件 | 触发结果 |
|------|---------|---------|
| **部位被毁判断** | 影子HP ≤ 0 | 触发NR-042的"部位被毁"分支 |
| **伤口大小参照** | 最大HP - 当前影子HP | 作为伤口记录severity的参照 |

**存档要求**（🆕 v4.1 A变更）：
- 影子HP需要**序列化存档**
- 战斗体激活状态下保存游戏，读档后影子HP必须还原（否则读档相当于"治愈"所有战斗体伤势）

```
影子HP生命周期：

  激活
   │
   ▼
  初始化（每部位 = 最大HP）
   │
   ▼ ← 每次受伤循环
  受到x伤害
  影子HP[部位] -= x
   │
   ├─ 影子HP > 0 → 更新伤口记录大小（NR-042）
   │
   └─ 影子HP ≤ 0 → 部位被毁（NR-042被毁分支）
                     │
                     ├─ 头/心脏/双腿 → 被动破裂（NR-008）
                     └─ 其他 → 高额流失（NR-035）
   │
   ▼
  战斗体解除
  销毁影子HP记录
```

### NR-042: 伤口Hediff规范与完整伤害管道 ✏️（v4新增；v4.1去除实现细节，增加f(x)>x约束）

#### 42.1 完整伤害管道

```
受到攻击（伤害计算完成后，应用前拦截；原版HP不变）
    │
    ▼
  即将受到 x 点伤害（作用于部位 P）
    │
    ├─→ 直接消耗一定量Trion【瞬时，消耗量 = f(x)，f(x) > x（大数值体系）】
    │
    ├─→ 部位 P 的影子HP减少 x
    │
    └─→ 伤口记录处理（作用于部位 P）：
            │
            ├─ 影子HP > 0（部位未被毁）
            │       ├─ P 上已有伤口记录
            │       │     └→ 合并：severity增大，更新Trion持续流失注册
            │       │
            │       └─ P 上无伤口记录
            │             └→ 新建伤口记录，注册Trion持续流失速率 y
            │
            └─ 影子HP ≤ 0（部位被毁）
                    ├─ 移除 P 的伤口记录（若有）
                    ├─ 注销对应Trion持续流失
                    └─ 走 NR-035/NR-036 逻辑
```

**函数说明**（具体公式在详细设计阶段确定）：
| 函数 | 含义 | 约束 |
|------|------|------|
| `f(x)` | 伤害x → 直接Trion消耗量 | f(x) > x（Trion大数值体系，单位不同） |
| `g(x)` | 伤害x → 伤口severity大小 | 可能参照影子HP剩余比例 |
| `y` | severity → Trion流失速率（/天） | 具体上限待数值设计确定 |

#### 42.2 伤口Hediff规范

**设计原则**：BDP专用伤口Hediff，**不继承原版伤口类**，不触发出血、疼痛等原版伤口机制。

**原因**：原版伤口类有复杂的出血/疼痛计算逻辑。BDP伤口Hediff通过使用基础Hediff类型，天然屏蔽这些副作用，无需针对性拦截。

**合并机制**：
- 同一部位受多次击打 → Hediff**合并为一个**（severity增大），而非堆叠多个
- 合并时同步更新Trion流失注册的速率值

**生命周期**：
- 创建：受伤且部位未被毁时创建
- 更新：同部位再次受伤时合并
- 移除A：部位影子HP降至0（被毁），移除并走被毁逻辑
- 移除B：战斗体解除时（主动或被动），全部移除并注销流失

**与Trion系统的对接**：
- 每个部位维持唯一一条持续流失注册
- 受伤时注册（或更新），被毁/解除时注销

### 设计原则总结

1. **有选择性的机制移植**：不100%还原原作，为游戏服务
2. **利用现有机制**：Hediff系统原生属性、空投回退逻辑
3. **增加战术深度**：引导时间、两条路径、冷却CD
4. **保持游戏平衡**：不实现非Trion免疫、不修复缺陷、维持消耗+CD防滥用
5. **生理/心理分离**：生理冻结回滚，心理正常运行保留
6. **最简原则**：保留原版身体结构拦截致死逻辑，不做高风险结构替换
7. **影子HP解耦**（🆕 v4）：伤害判定完全独立于原版HP
