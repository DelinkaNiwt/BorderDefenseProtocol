# 战斗体伤害系统问题诊断与解决方案

## 一、问题诊断

### 1.1 核心问题：拦截点选择错误

**当前实现**：
```csharp
// Patch_DamageWorker_FinalizeAndAddInjury.cs
[HarmonyPatch(typeof(DamageWorker_AddInjury), "FinalizeAndAddInjury")]
public static bool Prefix(Pawn pawn, Hediff_Injury injury, DamageInfo dinfo)
{
    // 拦截原版已构造的 Hediff_Injury
    CombatBodyDamageHandler.HandleDamage(pawn, injury, dinfo);
    return false;  // 阻止原版 AddHediff
}
```

**问题所在**：

这个拦截点在原版伤害流程中的位置：
```
DamageWorker_AddInjury.Apply()
  └─ ApplyToPart()
      ├─ GetExactPartFromDamageInfo()  ← 选择命中部位
      ├─ ArmorUtility.GetPostArmorDamage()  ← 计算护甲减伤
      └─ FinalizeAndAddInjury()
          ├─ 构造 Hediff_Injury 对象  ← 包含流血、疼痛等逻辑
          ├─ 设置武器信息
          ├─ 设置 severity
          └─ pawn.health.AddHediff()  ← 你们在这里拦截
```

**你们拦截的时机太晚了**：
- ✅ 原版已经计算好护甲减伤
- ✅ 原版已经选择好命中部位
- ❌ 原版也构造好了 `Hediff_Injury` 对象
- ❌ `Hediff_Injury` 包含流血、疼痛、感染等你们不需要的逻辑

### 1.2 导致的问题

#### 问题A：无法复用原版Hediff逻辑

你们必须重新实现：
1. **武器信息记录**（WoundAdapter.cs:116-123）
2. **伤口标签和颜色**（Hediff_CombatWound.cs:26-119）
3. **部位缺失判定**（PartDestructionHandler.cs）
4. **部位左右区分**（PartDestructionHandler.cs:187-210，用字符串匹配）

但实现得不完整：
- ❌ 没有伤害扩散逻辑（`CheckDuplicateDamageToOuterParts`）
- ❌ 没有永久伤残逻辑（`HediffComp_GetsPermanent`）
- ❌ 没有即死保护逻辑（`allowInstantKillChance`）
- ❌ 部位左右区分不可靠（依赖LabelShort字符串）

#### 问题B：两套系统不兼容

**原版系统**：
- 使用 `Hediff_Injury` 记录伤口
- 使用 `Hediff_MissingBodyPart` 标记部位缺失
- 使用 `pawn.health.hediffSet.GetPartHealth()` 查询部位HP

**你们的系统**：
- 使用 `Hediff_CombatWound` 记录伤口（不继承 `Hediff_Injury`）
- 使用 `BDP_CombatBodyPartDestroyed` 标记部位缺失
- 使用 `gene.ShadowHP.GetHP()` 查询影子HP

两套系统并存，导致：
- 原版UI显示的是原版HP（但你们拦截了伤害，原版HP不变）
- 你们的影子HP和原版HP不同步
- 部位缺失判定混乱（原版用 `PartIsMissing`，你们用自定义Hediff）

### 1.3 根本原因

**你们想要的**：
- 使用原版的部位选择、护甲计算、伤害扩散、武器信息记录
- 但不要原版的流血、疼痛、感染、HP减少

**但你们选择的拦截点**：
- 在原版构造完 `Hediff_Injury` 之后拦截
- 导致必须重新实现一遍原版逻辑

## 二、解决方案：更早的拦截点

### 2.1 最佳拦截点

**在 `ApplyDamageToPart` 中，护甲计算之后、构造Hediff之前拦截**：

```csharp
// DamageWorker_AddInjury.ApplyDamageToPart() 的流程：
private void ApplyDamageToPart(DamageInfo dinfo, Pawn pawn, DamageResult result)
{
    // 1. 选择命中部位
    BodyPartRecord exactPartFromDamageInfo = GetExactPartFromDamageInfo(dinfo, pawn);
    dinfo.SetHitPart(exactPartFromDamageInfo);

    // 2. 计算护甲减伤
    float num = ArmorUtility.GetPostArmorDamage(pawn, dinfo.Amount, ...);

    // 3. 检查是否完全格挡
    if (num <= 0f) {
        result.deflected = true;
        return;
    }

    // 4. 调用 FinalizeAndAddInjury
    //    ← 在这里拦截！
    FinalizeAndAddInjury(pawn, num, dinfo, result);
}
```

**新的拦截策略**：


**改进2：复用原版Hediff_Injury的武器信息逻辑**

不要自己设置武器信息，而是**让原版构造Hediff_Injury，然后复制信息**：

```csharp
public static void HandleDamage(Pawn pawn, float totalDamage, DamageInfo dinfo, DamageWorker.DamageResult result)
{
    // 1. 让原版构造 Hediff_Injury（但不添加到Pawn）
    HediffDef hediffDefFromDamage = HealthUtility.GetHediffDefFromDamage(dinfo.Def, pawn, dinfo.HitPart);
    Hediff_Injury vanillaInjury = (Hediff_Injury)HediffMaker.MakeHediff(hediffDefFromDamage, pawn);
    vanillaInjury.Part = dinfo.HitPart;

    // 2. 让原版设置武器信息（复制原版逻辑）
    SetWeaponInfo(vanillaInjury, dinfo);

    // 3. 创建战斗体伤口，复制武器信息
    var combatWound = CreateCombatWound(pawn, dinfo.HitPart, dinfo.Def, totalDamage);
    CopyWeaponInfo(vanillaInjury, combatWound);

    // 4. 不添加 vanillaInjury 到Pawn（让它被GC回收）
}

private static void SetWeaponInfo(Hediff_Injury injury, DamageInfo dinfo)
{
    // 完全照搬原版逻辑（DamageWorker_AddInjury.FinalizeAndAddInjury）
    Pawn pawn2 = dinfo.Instigator as Pawn;
    injury.sourceDef = dinfo.Weapon;
    if (pawn2 != null && pawn2.IsMutant && dinfo.Weapon == ThingDefOf.Human)
        injury.sourceLabel = pawn2.mutant.Def.label;
    else
        injury.sourceLabel = dinfo.Weapon?.label ?? "";
    injury.sourceBodyPartGroup = dinfo.WeaponBodyPartGroup;
    injury.sourceHediffDef = dinfo.WeaponLinkedHediff;
    injury.sourceToolLabel = dinfo.Tool?.labelNoLocation ?? dinfo.Tool?.label;
}
```

### 2.3 完整的新架构

```
战斗体伤害处理流程（新）：

1. 原版计算护甲和部位
   ↓
2. Harmony拦截 FinalizeAndAddInjury（早期重载）
   ├─ 参数：totalDamage（护甲后）, dinfo.HitPart（命中部位）
   └─ 阻止原版构造 Hediff_Injury
   ↓
3. 战斗体处理
   ├─ 直接消耗Trion：f(totalDamage)
   ├─ 更新影子HP：shadowHP[part] -= totalDamage
   ├─ 判断部位状态
   │   ├─ 未被毁 → 创建战斗体伤口Hediff
   │   └─ 被毁 → 标记部位缺失（仅在影子HP中）
   └─ 检查破裂条件
   ↓
4. UI显示
   ├─ Harmony patch GetPartHealth() → 返回影子HP
   ├─ Harmony patch PartIsMissing() → 检查影子HP
   └─ 原版UI自动显示正确的部位状态
```

## 三、具体实现方案

### 3.1 新的Harmony Patch

```csharp
[HarmonyPatch(typeof(DamageWorker_AddInjury), "FinalizeAndAddInjury",
    new Type[] { typeof(Pawn), typeof(float), typeof(DamageInfo), typeof(DamageWorker.DamageResult) })]
public static class Patch_DamageWorker_FinalizeAndAddInjury_V2
{
    public static bool Prefix(
        Pawn pawn,
        float totalDamage,
        DamageInfo dinfo,
        DamageWorker.DamageResult result,
        ref float __result)
    {
        // 检查战斗体状态
        var gene = pawn?.genes?.GetFirstGeneOfType<Gene_TrionGland>();
        if (gene == null || !gene.IsCombatBodyActive)
            return true;  // 走原版逻辑

        // 战斗体伤害处理
        float damageDealt = CombatBodyDamageHandler.HandleDamage(
            pawn, totalDamage, dinfo, result);

        // 设置返回值（原版返回实际造成的伤害）
        __result = damageDealt;

        return false;  // 跳过原版
    }
}
```

### 3.2 简化的伤害处理器

```csharp
public static class CombatBodyDamageHandler
{
    public static float HandleDamage(
        Pawn pawn,
        float totalDamage,
        DamageInfo dinfo,
        DamageWorker.DamageResult result)
    {
        BodyPartRecord hitPart = dinfo.HitPart;
        var gene = pawn.genes.GetFirstGeneOfType<Gene_TrionGland>();

        // 1. Trion消耗
        float trionCost = CalculateTrionCost(totalDamage);
        gene.CompTrion.Consume(trionCost);

        // 2. 更新影子HP
        gene.ShadowHP.TakeDamage(hitPart, totalDamage);
        bool partDestroyed = gene.ShadowHP.IsDestroyed(hitPart);

        // 3. 处理伤口或部位破坏
        if (!partDestroyed)
        {
            // 添加战斗体伤口（复用原版武器信息）
            AddCombatWound(pawn, hitPart, dinfo, totalDamage);
        }
        else
        {
            // 部位被毁：只在影子HP中标记，不添加Hediff
            // UI通过patch GetPartHealth显示
            if (IsCriticalPart(hitPart))
            {
                // 关键部位破坏 → 触发破裂
                gene.TriggerCollapse("关键部位破坏");
            }
        }

        // 4. 更新result（让原版系统知道伤害已处理）
        result.totalDamageDealt += totalDamage;
        result.wounded = true;
        result.AddPart(pawn, hitPart);

        return totalDamage;
    }

    private static void AddCombatWound(
        Pawn pawn,
        BodyPartRecord part,
        DamageInfo dinfo,
        float damage)
    {
        // 获取战斗体伤口Def
        HediffDef combatWoundDef = GetCombatWoundDef(dinfo.Def);

        // 查找现有伤口
        var existing = pawn.health.hediffSet.hediffs
            .OfType<Hediff_CombatWound>()
            .FirstOrDefault(h => h.Part == part && h.def == combatWoundDef);

        if (existing != null)
        {
            // 合并
            existing.Severity += damage / 10f;
            existing.TryGetComp<HediffComp_CombatWound>()?.hitCount++;
        }
        else
        {
            // 新建
            var wound = (Hediff_CombatWound)HediffMaker.MakeHediff(combatWoundDef, pawn, part);
            wound.Severity = damage / 10f;

            // 复制武器信息（照搬原版逻辑）
            SetWeaponInfo(wound, dinfo);

            pawn.health.AddHediff(wound);
        }
    }

    private static void SetWeaponInfo(Hediff wound, DamageInfo dinfo)
    {
        // 完全照搬原版 DamageWorker_AddInjury.FinalizeAndAddInjury 的逻辑
        Pawn instigator = dinfo.Instigator as Pawn;
        wound.sourceDef = dinfo.Weapon;

        if (instigator != null && instigator.IsMutant && dinfo.Weapon == ThingDefOf.Human)
            wound.sourceLabel = instigator.mutant.Def.label;
        else
            wound.sourceLabel = dinfo.Weapon?.label ?? "";

        wound.sourceBodyPartGroup = dinfo.WeaponBodyPartGroup;
        wound.sourceHediffDef = dinfo.WeaponLinkedHediff;
        wound.sourceToolLabel = dinfo.Tool?.labelNoLocation ?? dinfo.Tool?.label;
    }
}
```

### 3.3 UI显示的Harmony Patch

```csharp
// Patch 1: 让原版UI显示影子HP
[HarmonyPatch(typeof(HediffSet), "GetPartHealth")]
public static class Patch_HediffSet_GetPartHealth
{
    public static void Postfix(
        HediffSet __instance,
        BodyPartRecord part,
        ref float __result)
    {
        var pawn = __instance.pawn;
        var gene = pawn?.genes?.GetFirstGeneOfType<Gene_TrionGland>();

        if (gene != null && gene.IsCombatBodyActive && part != null)
        {
            // 返回影子HP而不是原版HP
            __result = gene.ShadowHP.GetHP(part);
        }
    }
}

// Patch 2: 让原版UI正确判断部位缺失
[HarmonyPatch(typeof(HediffSet), "PartIsMissing")]
public static class Patch_HediffSet_PartIsMissing
{
    public static void Postfix(
        HediffSet __instance,
        BodyPartRecord part,
        ref bool __result)
    {
        var pawn = __instance.pawn;
        var gene = pawn?.genes?.GetFirstGeneOfType<Gene_TrionGland>();

        if (gene != null && gene.IsCombatBodyActive && part != null)
        {
            // 检查影子HP是否为0
            __result = gene.ShadowHP.IsDestroyed(part);
        }
    }
}

// Patch 3: 部位标签颜色（可选）
[HarmonyPatch(typeof(HealthUtility), "GetPartConditionLabel")]
public static class Patch_HealthUtility_GetPartConditionLabel
{
    public static void Postfix(
        Pawn pawn,
        BodyPartRecord part,
        ref string __result)
    {
        var gene = pawn?.genes?.GetFirstGeneOfType<Gene_TrionGland>();

        if (gene != null && gene.IsCombatBodyActive && part != null)
        {
            float hp = gene.ShadowHP.GetHP(part);
            float maxHP = part.def.hitPoints;

            if (hp <= 0f)
                __result = "已破坏";
            else if (hp < maxHP * 0.3f)
                __result = "严重受损";
            else if (hp < maxHP * 0.7f)
                __result = "受损";
            else
                __result = "良好";
        }
    }
}
```

## 七、总结

### 7.1 根本问题

你们的问题不是"方案笨重"，而是**拦截点选择错误**：
- 拦截太晚，原版已经构造了Hediff_Injury
- 导致必须重新实现一遍原版逻辑
- 但实现得不完整，产生各种bug

### 7.2 解决方案

**更早拦截 + 复用原版 + Patch UI查询**：
1. 在原版构造Hediff之前拦截（早期重载）
2. 直接使用原版计算结果（护甲、部位）
3. 照搬原版武器信息逻辑（不要自己实现）
4. 不添加部位缺失Hediff，只在影子HP标记
5. Patch原版UI查询方法，返回影子HP

### 7.3 核心原则

**"不要重新发明轮子，而是在轮子上加个外壳"**：
- 让原版做它擅长的事（部位选择、护甲、武器信息）
- 只在最后一步拦截，替换成你们的逻辑
- 通过Patch让原版UI显示你们的数据
