## 核心特性

### 1. 延时破裂机制
- 破裂延时：90 ticks（1.5秒）
- 触发条件：Trion耗尽或关键部位破坏
- 状态转换：Active → Collapsing → Cooldown

### 2. 延时期间特性
- ✅ 无敌：所有伤害被拦截
- ✅ 禁止攻击：Manipulation设为0
- ✅ 允许倒地：移除了倒地拦截补丁
- ✅ 武器不掉落：扩展了武器掉落拦截
- ✅ 移动速度降低：-50%移动速度

### 3. 视觉反馈
- Hediff显示："战斗体破裂中 (破裂倒计时: X ticks)"
- 关键部位破坏时显示："[部位名]结构失效"
- 详细的日志输出

## 架构设计

### 状态机扩展
```
Inactive → Active → Collapsing → Cooldown → Inactive
                ↑                    ↓
                └────────────────────┘
```

**新增状态**：`Collapsing`（延时破裂中）

**新增字段**：
- `collapseStartTick`：破裂开始时间
- `collapseReason`：破裂原因

**新增方法**：
- `TransitionToCollapsing(string reason)`
- `TransitionToCooldownFromCollapsing(int cooldownTicks)`
- `CanManualDeactivate()`
- `IsInvulnerable()`
- `GetCollapseRemaining()`

### 事件驱动倒计时
使用`Hediff_CombatBodyCollapsing`的`Tick()`方法管理倒计时，避免全局轮询。

**优势**：
- 无需全局轮询，只有Collapsing的Pawn才Tick
- Hediff自动跟随Pawn，无需手动管理生命周期
- 存档/读档自动序列化
- 性能优秀

### Pipeline原子性
在伤害处理Pipeline入口检查Collapsing状态，提前返回，不影响原版逻辑。

```csharp
if (ctx.Runtime != null && ctx.Runtime.State.IsCollapsing)
{
    Log.Message($"[BDP] 延时破裂阶段无敌，忽略伤害");
    return; // 提前返回，不执行Pipeline
}
```

## 关键代码片段

### 1. 状态转换（CombatBodyState.cs）
```csharp
public void TransitionToCollapsing(string reason)
{
    if (outerState != OuterState.Active)
        throw new System.InvalidOperationException($"Cannot collapse from {outerState}");

    outerState = OuterState.Collapsing;
    collapseStartTick = Find.TickManager.TicksGame;
    collapseReason = reason;
}
```

### 2. 无敌检查（CombatBodyDamageHandler.cs）
```csharp
if (ctx.Runtime != null && ctx.Runtime.State.IsCollapsing)
{
    Log.Message($"[BDP] 延时破裂阶段无敌，忽略伤害: {pawn.LabelShort}");
    return;
}
```

### 3. 破裂触发（CombatBodyDamageHandler.cs）
```csharp
// 转换到Collapsing状态
ctx.Runtime.State.TransitionToCollapsing(collapseReason);

// 添加延时破裂Hediff（管理倒计时）
pawn.health.AddHediff(Core.BDP_DefOf.BDP_CombatBodyCollapsing);

// 关键部位破坏时添加视觉Hediff
if (criticalPartDestroyed && hitPart != null)
{
    pawn.health.AddHediff(Core.BDP_DefOf.BDP_CombatBodyPartPending, hitPart);
}
```

### 4. 倒计时管理（Hediff_CombatBodyCollapsing.cs）
```csharp
public override void Tick()
{
    base.Tick();

    if (Find.TickManager.TicksGame >= collapseAtTick)
    {
        ExecuteCollapse();
    }
}
```

### 5. 破裂执行（Hediff_CombatBodyCollapsing.cs）
```csharp
private void ExecuteCollapse()
{
    // 移除临时Hediff
    pawn.health.RemoveHediff(this);

    // 移除部位待失效标记
    var pendingHediffs = pawn.health.hediffSet.hediffs
        .Where(h => h.def == BDP_DefOf.BDP_CombatBodyPartPending)
        .ToList();
    foreach (var hediff in pendingHediffs)
    {
        pawn.health.RemoveHediff(hediff);
    }

    // 执行解除流程
    var runtime = CombatBodyRuntime.Of(pawn);
    runtime?.Deactivate(isEmergency: true);
}
```
