## 设计检查清单

```
□ defName是否唯一且符合命名规范（BDP_Chip_XXX）
□ 芯片效果类型是否选择正确
□ 成本配置是否合理（激活成本、占用成本、持续消耗的平衡）
□ 槽位限制是否符合设计意图
□ 分类标签是否完整且准确
□ 是否有互斥冲突（exclusionTags）
□ 如果是武器类，Verb配置是否完整
□ 如果是Hediff类，HediffDef是否已创建
□ 如果是Ability类，AbilityDef是否已创建
□ 如果是被动类，外部触发逻辑是否已实现
□ 贴图资源是否准备好
□ 市场价值是否与功能匹配
□ 时间成本（前摇/后摇）是否符合游戏体验
```

## 附录：芯片系统架构说明

### 芯片效果类型详解

#### 1. VerbChipEffect - 武器类芯片
- **作用**: 向触发体注入Verb/Tool配置，实现近战和远程武器功能
- **适用场景**: 所有武器类芯片（近战、远程、狙击等）
- **机制**: 激活时调用SetSideVerbs按侧存储Verb数据，支持双武器系统

#### 2. HediffChipEffect - 状态效果芯片
- **作用**: 激活时添加Hediff，关闭时移除
- **适用场景**: 护盾、增益、减益等持续状态效果
- **机制**: 通过RimWorld的Hediff系统实现

#### 3. AbilityChipEffect - 能力类芯片
- **作用**: 激活时授予Ability，关闭时移除
- **适用场景**: "点击→选目标→执行"的一次性动作
- **机制**: 通过RimWorld的Ability系统实现

#### 4. PassiveChipEffect - 被动芯片
- **作用**: 空壳标记，激活时不执行任何操作
- **适用场景**: 紧急脱离、雷达隐身等被动触发的芯片
- **机制**: 实际逻辑由外部系统检测芯片是否激活并执行相应行为

### 成本系统说明

1. **激活成本（activationCost）**: 激活时一次性消耗的Trion
2. **占用成本（allocationCost）**: 战斗体激活时锁定的Trion量
3. **持续消耗（drainPerDay）**: 每天持续消耗的Trion（0=无持续消耗）
4. **每发消耗（trionCostPerShot）**: 每次射击时消耗的Trion（仅武器类）

### 槽位系统说明

- **LeftHand**: 左手槽位
- **RightHand**: 右手槽位
- **Special**: 特殊槽位（全部同时激活/关闭，不参与左右切换逻辑）

### 分类标签系统说明

芯片采用**多维度标签分类**，一个芯片可以有多个类别标签。标签用于：
- 筛选和匹配芯片
- 组合能力系统的匹配
- UI显示和分类
- 互斥检查
