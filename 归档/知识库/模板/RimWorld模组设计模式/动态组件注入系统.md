# 动态组件注入系统

## 摘要
动态组件注入系统是一种设计模式，允许通过多种入口点（Gene、Hediff、Building等）动态添加功能组件到Pawn或其他Thing上。该模式将核心逻辑与游戏系统集成点解耦，提高代码复用性和可维护性。

## 模式概述

### 适用场景

- 功能需要通过多种方式激活（基因、药物、装备、建筑等）
- 核心逻辑需要在多个游戏系统中共享
- 需要动态启用/禁用功能
- 希望避免代码重复

### 核心价值

1. **解耦性**: 核心逻辑独立于游戏系统集成点
2. **复用性**: 同一核心逻辑可通过多个入口点激活
3. **灵活性**: 支持运行时动态添加/移除组件
4. **可维护性**: 修改核心逻辑一处生效，无需修改多个入口点

### 优缺点

**优点**:
- 代码复用率高
- 易于添加新的激活方式
- 核心逻辑集中管理
- 支持动态启用/禁用

**缺点**:
- 初始实现复杂度较高
- 需要理解ThingComp机制
- 调试时需要追踪组件生命周期

### 核心类结构

#### 1. ThingComp（核心组件）

```csharp
[StaticConstructorOnStartup]
public class CompYourFeature : ThingComp, IExposable
{
    public bool enabled = true;
    
    public override void PostSpawnSetup(bool respawningAfterLoad)
    {
        base.PostSpawnSetup(respawningAfterLoad);
        // 初始化逻辑
    }
    
    public override void CompTick()
    {
        if (!enabled) return;
        
        // 核心功能逻辑
        // 例如：检测条件、执行动作、播放效果等
    }
    
    public override void PostExposeData()
    {
        base.PostExposeData();
        Scribe_Values.Look(ref enabled, "enabled");
    }
}
```

#### 2. Gene入口点

```csharp
public class Gene_YourFeature : Gene
{
    private CompYourFeature runtimeComp;
    private GeneExtension_YourFeature Ext => def?.GetModExtension<GeneExtension_YourFeature>();
    
    public override void PostAdd()
    {
        base.PostAdd();
        TryAttachComp();
    }
    
    public override void PostRemove()
    {
        base.PostRemove();
        DetachComp();
    }
    
    public override void Tick()
    {
        base.Tick();
        if (Active != previouslyActive)
        {
            if (Active) TryAttachComp();
            else DetachComp();
            previouslyActive = Active;
        }
    }
    
    private void TryAttachComp()
    {
        if (runtimeComp == null && pawn?.AllComps != null && Ext != null)
        {
            runtimeComp = new CompYourFeature
            {
                parent = pawn,
                props = Ext.ToThingCompProps()
            };
            pawn.AllComps.Add(runtimeComp);
            runtimeComp.PostSpawnSetup(respawningAfterLoad: false);
        }
    }
    
    private void DetachComp()
    {
        if (runtimeComp != null && pawn?.AllComps != null)
        {
            pawn.AllComps.Remove(runtimeComp);
            runtimeComp = null;
        }
    }
}
```

#### 3. Hediff入口点

```csharp
public class HediffComp_YourFeature : HediffComp
{
    private CompYourFeature runtimeComp;
    private HediffCompExtension_YourFeature Ext => Props?.GetModExtension<HediffCompExtension_YourFeature>();
    
    public override void CompPostMake()
    {
        base.CompPostMake();
        TryAttachComp();
    }
    
    public override void CompPostPostAdd(DamageInfo? dinfo)
    {
        base.CompCompPostAdd(dinfo);
        TryAttachComp();
    }
    
    public override void CompPostRemoved()
    {
        base.CompPostRemoved();
        DetachComp();
    }
    
    public override void CompPostTick(ref float severityAdjustment)
    {
        base.CompPostTick(ref severityAdjustment);
        if (runtimeComp != null && !runtimeComp.enabled)
        {
            runtimeComp.enabled = true;
        }
    }
    
    private void TryAttachComp()
    {
        if (runtimeComp == null && parent.pawn?.AllComps != null && Ext != null)
        {
            runtimeComp = new CompYourFeature
            {
                parent = parent.pawn,
                props = Ext.ToThingCompProps()
            };
            parent.pawn.AllComps.Add(runtimeComp);
            runtimeComp.PostSpawnSetup(respawningAfterLoad: false);
        }
    }
    
    private void DetachComp()
    {
        if (runtimeComp != null && parent.pawn?.AllComps != null)
        {
            parent.pawn.AllComps.Remove(runtimeComp);
            runtimeComp = null;
        }
    }
}
```

1. **ThingComp.AllComps**: Pawn的组件集合

   ```csharp
   pawn.AllComps.Add(runtimeComp);

   ```

2. **ThingComp.PostSpawnSetup()**: 组件初始化
   ```csharp
   runtimeComp.PostSpawnSetup(respawningAfterLoad: false);
   ```

3. **ModExtension**: 配置扩展
   ```csharp
   var ext = def.GetModExtension<YourExtension>();
   ```

## 完整示例

### 示例场景：自动治疗功能

假设我们要实现一个自动治疗功能，可以通过基因获得，也可以通过药物获得。

#### 核心组件

```csharp
[StaticConstructorOnStartup]
public class CompAutoHeal : ThingComp, IExposable
{
    public float healInterval = 60f;
    public float healAmount = 1f;
    public float lastHealTick = 0f;
    
    public override void PostSpawnSetup(bool respawningAfterLoad)
    {
        base.PostSpawnSetup(respawningAfterLoad);
        lastHealTick = Find.TickManager.TicksGame;
    }
    
    public override void CompTick()
    {
        if (!parent.IsHashIntervalTick(60)) return;
        
        float ticksSinceLastHeal = Find.TickManager.TicksGame - lastHealTick;
        if (ticksSinceLastHeal >= healInterval)
        {
            HealPawn();
            lastHealTick = Find.TickManager.TicksGame;
        }
    }
    
    private void HealPawn()
    {
        Pawn pawn = parent as Pawn;
        if (pawn != null && !pawn.Dead && !pawn.Downed)
        {
            HealthUtility.HealNonPermanentInjuries(pawn, healAmount);
        }
    }
    
    public override void PostExposeData()
    {
        base.PostExposeData();
        Scribe_Values.Look(ref healInterval, "healInterval");
        Scribe_Values.Look(ref healAmount, "healAmount");
        Scribe_Values.Look(ref lastHealTick, "lastHealTick");
    }
}
```

## 常见问题

### Q1: 组件注入后不生效？

**A**: 检查以下几点：
- 确认runtimeComp已正确添加到AllComps
- 确认已调用PostSpawnSetup
- 检查enabled状态
- 查看日志是否有错误

### Q2: 组件移除后仍然生效？

**A**: 确保正确调用DetachComp，并从AllComps中移除组件

### Q3: 如何处理多个入口点同时激活？

**A**: 在TryAttachComp中添加检查，避免重复注入
```csharp
if (runtimeComp != null) return;
```

### Q4: 如何保存组件状态？

**A**: 在ThingComp中实现IExposable接口，在PostExposeData中序列化必要字段
