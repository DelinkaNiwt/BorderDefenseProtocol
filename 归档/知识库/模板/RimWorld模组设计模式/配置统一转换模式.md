# 配置统一转换模式

## 摘要
配置统一转换模式是一种设计模式，用于统一管理不同游戏系统（Gene、Hediff、ThingComp等）的配置参数。该模式通过ModExtension和转换方法，将分散在多个Def中的配置集中管理，避免重复定义和不一致问题。

## 模式概述

### 适用场景

- 功能需要在多个系统中使用相同参数（Gene、Hediff、ThingComp等）
- 需要避免在多个Def中重复定义相同配置
- 希望统一管理功能参数，便于维护和更新
- 需要支持配置继承和覆盖

### 核心价值

1. **单一配置源**: 所有参数在一个地方定义，避免不一致
2. **减少重复**: 避免在多个Def中重复定义相同参数
3. **易于维护**: 修改配置只需更新一处
4. **灵活扩展**: 支持配置继承和自定义覆盖

### 优缺点

**优点**:
- 配置集中管理
- 减少代码重复
- 避免配置不一致
- 易于维护和更新

**缺点**:
- 需要额外的ModExtension类
- 转换逻辑增加复杂度
- 需要理解ModExtension机制

### 步骤1: 定义CompProperties

1. 创建CompProperties_YourFeature类
2. 继承CompProperties
3. 添加所有配置属性
4. 在构造函数中设置compClass

### 步骤2: 创建ModExtension类

1. 为每个系统创建对应的ModExtension类
2. 添加与CompProperties相同的配置属性
3. 实现ToThingCompProps()转换方法
4. 设置默认值

### 步骤3: 在入口点中使用转换

1. 在Gene、Hediff等入口点中获取ModExtension
2. 调用ToThingCompProps()转换
3. 将转换后的CompProperties传递给ThingComp

## 完整示例

### 示例场景：自动治疗功能配置

假设我们要为自动治疗功能创建统一配置，通过Gene和Hediff都可以激活。

#### CompProperties定义

```csharp

{
```

#### Gene中使用

```csharp
public class Gene_AutoHeal : Gene
{
    private CompAutoHeal runtimeComp;
    private GeneExtension_AutoHeal Ext => def?.GetModExtension<GeneExtension_AutoHeal>();
    
    public override void PostAdd()
    {
        base.PostAdd();
        TryAttachComp();
    }
    
    private void TryAttachComp()
    {
        if (runtimeComp == null && pawn?.AllComps != null && Ext != null)
        {
            runtimeComp = new CompAutoHeal
            {
                parent = pawn,
                props = Ext.ToThingCompProps()
            };
            pawn.AllComps.Add(runtimeComp);
            runtimeComp.PostSpawnSetup(respawningAfterLoad: false);
        }
    }
}
```

## 高级用法

### 配置继承

```csharp
public class GeneExtension_AutoHeal : GeneExtension_AutoHealBase
{
    public override CompProperties_AutoHeal ToThingCompProps()
    {
        var props = base.ToThingCompProps();
        // 可以添加特定于Gene的额外逻辑
        return props;
    }
}

public class GeneExtension_AutoHealBase : DefModExtension
{
    public float healInterval = 60f;
    public float healAmount = 1f;
    
    public virtual CompProperties_AutoHeal ToThingCompProps()
    {
        return new CompProperties_AutoHeal
        {
            healInterval = this.healInterval,
            healAmount = this.healAmount
        };
    }
}
```

### 配置验证

```csharp
public class GeneExtension_AutoHeal : DefModExtension
{
    public float healInterval = 60f;
    public float healAmount = 1f;
    
    public override void ResolveReferences(Def def)
    {
        base.ResolveReferences(def);
        if (healInterval <= 0)
        {
            Log.Error($"[YourMod] {def.defName} has invalid healInterval: {healInterval}");
            healInterval = 60f;
        }
        if (healAmount <= 0)
        {
            Log.Error($"[YourMod] {def.defName} has invalid healAmount: {healAmount}");
            healAmount = 1f;
        }
    }
}
```

### 配置覆盖

```csharp
public class CompProperties_AutoHeal : CompProperties
{
    public float healInterval = 60f;
    public float healAmount = 1f;
    
    public void OverrideWith(GeneExtension_AutoHeal ext)
    {
        if (ext.healInterval > 0) this.healInterval = ext.healInterval;
        if (ext.healAmount > 0) this.healAmount = ext.healAmount;
    }
}
```

## 常见问题

### Q1: 如何处理不同系统的特殊参数？

**A**: 在ModExtension中添加特定于该系统的额外参数，在转换方法中处理

### Q2: 如何确保配置一致性？

**A**: 使用相同的ModExtension类，或在不同的ModExtension类中使用相同的参数名和默认值

### Q3: 如何支持配置热更新？

**A**: 在CompProperties中添加Reload方法，在运行时重新加载配置

### Q4: 如何处理配置验证失败？

**A**: 在ResolveReferences中验证，失败时记录错误并使用默认值

## 最佳实践

### 命名规范

- ModExtension类名: `{System}Extension_{Feature}`
- CompProperties类名: `CompProperties_{Feature}`
- 转换方法名: `ToThingCompProps()`

### 默认值设置

- 为所有参数设置合理的默认值
- 默认值应该是最常用的配置
- 避免使用0或null作为默认值

### 配置验证

- 在ResolveReferences中验证配置
- 记录配置错误
- 使用安全的默认值

### 文档注释

- 为每个参数添加XML注释
- 说明参数的作用和取值范围
- 提供示例配置
