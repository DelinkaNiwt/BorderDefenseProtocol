# 多版本兼容模式

## 摘要
多版本兼容模式是一种设计模式，用于支持多个RimWorld版本的模组开发。该模式通过LoadFolders.xml、版本目录结构和API差异处理，实现代码复用最大化，减少维护成本，简化版本升级流程。

## 模式概述

### 适用场景

- 模组需要支持多个RimWorld版本（如1.5和1.6）
- 不同版本间存在API差异
- 希望减少代码重复和维护成本
- 需要简化版本升级流程

### 核心价值

1. **代码复用**: 共享代码放在Common目录，避免重复
2. **版本隔离**: 版本特定代码放在对应版本目录
3. **易于升级**: 添加新版本只需创建新目录
4. **维护简单**: 修改共享代码一处生效

### 优缺点

**优点**:
- 代码复用最大化
- 版本差异隔离清晰
- 易于添加新版本
- 维护成本低

**缺点**:
- 需要理解LoadFolders机制
- 目录结构稍复杂
- 需要处理API差异

## 实现细节

### 目录结构

```
模组根目录/
├── About/                          # 模组信息（所有版本共享）
│   ├── About.xml
│   ├── Preview.png
│   └── PublishedFileId.txt
│
├── LoadFolders.xml                 # 版本加载配置（必须）
│
├── 1.5/                           # 1.5版本专用内容
│   ├── Assemblies/                # 1.5专用DLL
│   ├── Defs/                      # 1.5专用Def
│   ├── Patches/                   # 1.5专用补丁
│   └── Source/                    # 1.5专用源代码
│
├── 1.6/                           # 1.6版本专用内容
│   ├── Assemblies/                # 1.6专用DLL
│   ├── Defs/                      # 1.6专用Def
│   ├── Patches/                   # 1.6专用补丁
│   └── Source/                    # 1.6专用源代码
│
└── Common/                        # 跨版本共享内容
    ├── Languages/                 # 本地化文件
    ├── Textures/                  # 纹理资源
    ├── Sounds/                    # 音频资源
    └── Source/                    # 共享源代码（可选）
```

### LoadFolders.xml配置

```xml
<?xml version="1.0" encoding="utf-8"?>
<loadFolders>
  <v1.5>
    <li>1.5</li>
    <li>Common</li>
  </v1.5>
  <v1.6>
    <li>1.6</li>
    <li>Common</li>
  </v1.6>
</loadFolders>
```

**说明**:
- `<v1.5>`: 定义1.5版本加载的目录
- `<li>1.5</li>`: 加载1.5目录
- `<li>Common</li>`: 加载Common目录
- 加载顺序：先加载版本目录，再加载Common目录

### API差异处理

#### 方法1: 条件编译

```csharp
#if RW1_6
    // 1.6版本特定代码
    var newAPI = someObject.NewMethod();
#else
    // 1.5版本特定代码
    var oldAPI = someObject.OldMethod();
#endif
```

#### 方法2: 运行时版本检测

```csharp
public static class VersionUtility
{
    public static bool IsVersion1_6OrLater
    {
        get
        {
            return VersionControl.CurrentVersion >= new Version(1, 6);
        }
    }
    
    public static void CallVersionSpecificMethod()
    {
        if (IsVersion1_6OrLater)
        {
            // 1.6版本方法
        }
        else
        {
            // 1.5版本方法
        }
    }
}
```

#### 方法3: 适配器模式

```csharp
public interface IVersionAdapter
{
    void DoSomething();
}

public class Version1_5Adapter : IVersionAdapter
{
    public void DoSomething()
    {
        // 1.5版本实现
    }
}

public class Version1_6Adapter : IVersionAdapter
{
    public void DoSomething()
    {
        // 1.6版本实现
    }
}

public static class AdapterFactory
{
    public static IVersionAdapter CreateAdapter()
    {
        if (VersionUtility.IsVersion1_6OrLater)
        {
            return new Version1_6Adapter();
        }
        return new Version1_5Adapter();
    }
}
```

### 代码组织策略

#### 策略1: 完全隔离

```
1.5/Source/
└── ModName/
    ├── Feature1.cs
    └── Feature2.cs

1.6/Source/
└── ModName/
    ├── Feature1.cs
    └── Feature2.cs

Common/Source/
└── ModName/
    └── SharedUtility.cs
```

**优点**: 版本差异完全隔离
**缺点**: 代码重复

#### 策略2: 共享核心

```
Common/Source/
└── ModName/
    ├── Core/
    │   ├── Feature1.cs
    │   └── Feature2.cs
    └── VersionSpecific/
        ├── Version1_5.cs
        └── Version1_6.cs

1.5/Source/
└── ModName/
    └── Version1_5.cs

1.6/Source/
└── ModName/
    └── Version1_6.cs
```

**优点**: 核心代码共享
**缺点**: 需要处理版本差异

#### 策略3: 部分共享（推荐）

```
Common/Source/
└── ModName/
    ├── Shared/
    │   ├── Feature1.cs
    │   └── Feature2.cs
    └── VersionSpecific/
        ├── IVersionAdapter.cs
        └── VersionUtility.cs

1.5/Source/
└── ModName/
    ├── Version1_5/
    │   ├── Feature1.cs
    │   └── Feature2.cs
    └── Adapters/
        └── Version1_5Adapter.cs

1.6/Source/
└── ModName/
    ├── Version1_6/
    │   ├── Feature1.cs
    │   └── Feature2.cs
    └── Adapters/
        └── Version1_6Adapter.cs
```

**优点**: 平衡共享和隔离
**缺点**: 结构稍复杂

### 示例场景：支持1.5和1.6版本

#### LoadFolders.xml

```xml

  </v1.5>

    <li>1.6</li>

  </v1.6>

```

#### 版本检测工具

```csharp
using Verse;

namespace YourMod
{
    public static class VersionUtility
    {
        public static bool IsVersion1_6OrLater
        {
            get
            {
                return VersionControl.CurrentVersion >= new Version(1, 6);
            }
        }
        
        public static bool IsVersion1_5
        {
            get
            {
                return VersionControl.CurrentVersion.Major == 1 && 
                       VersionControl.CurrentVersion.Minor == 5;
            }
        }
    }
}
```

#### 版本适配器

```csharp
namespace YourMod
{
    public interface IFeatureAdapter
    {
        void ExecuteFeature();
    }
    
    public class FeatureAdapter1_5 : IFeatureAdapter
    {
        public void ExecuteFeature()
        {
            // 1.5版本实现
            Log.Message("Executing feature in 1.5 mode");
        }
    }
    
    public class FeatureAdapter1_6 : IFeatureAdapter
    {
        public void ExecuteFeature()
        {
            // 1.6版本实现
            Log.Message("Executing feature in 1.6 mode");
        }
    }
    
    public static class AdapterFactory
    {
        public static IFeatureAdapter CreateFeatureAdapter()
        {
            if (VersionUtility.IsVersion1_6OrLater)
            {
                return new FeatureAdapter1_6();
            }
            return new FeatureAdapter1_5();
        }
    }
}
```

#### 共享功能

```csharp
using Verse;

namespace YourMod.Shared
{
    public class SharedFeature
    {
        private IFeatureAdapter adapter;
        
        public SharedFeature()
        {
            adapter = AdapterFactory.CreateFeatureAdapter();
        }
        
        public void Execute()
        {
            // 共享逻辑
            Log.Message("Executing shared feature");
            
            // 版本特定逻辑
            adapter.ExecuteFeature();
        }
    }
}
```

## 常见问题

### Q1: LoadFolders.xml不生效？

**A**: 检查以下几点：
- 确认文件在模组根目录
- 确认XML格式正确
- 确认版本号与游戏版本匹配
- 查看日志是否有错误

### Q2: 如何处理API变更？

**A**: 使用适配器模式或条件编译，隔离版本差异

### Q3: 共享代码如何访问版本特定代码？

**A**: 使用接口或抽象类，通过工厂模式创建版本特定实例

### Q4: 如何测试不同版本？

**A**: 使用多个RimWorld版本进行测试，或使用条件编译切换版本
