## 设计模式列表

### 1. 动态组件注入系统
**文档**: [动态组件注入系统.md](./动态组件注入系统.md)

**适用场景**: 需要通过多种入口点（Gene、Hediff、Building等）动态添加功能到Pawn或其他Thing的模组

**核心价值**:
- 解耦核心逻辑与游戏系统集成点
- 支持多路径激活同一功能
- 提高代码复用性和可维护性

**参考来源**: AutoBlink自动折跃模组

---

### 2. 配置统一转换模式
**适用场景**: 需要统一管理不同系统（Gene、Hediff、ThingComp）的配置参数，避免重复定义的模组

**核心价值**:
- 单一配置源，避免不一致
- 简化参数维护和更新
- 支持配置继承和覆盖

**参考来源**: AutoBlink自动折跃模组

---

### 3. 多版本兼容模式
**适用场景**: 需要支持多个RimWorld版本的模组，减少代码重复和维护成本

**核心价值**:
- 代码复用最大化
- 版本差异隔离
- 简化版本升级流程

**参考来源**: AutoBlink自动折跃模组

## 使用指南

### 如何选择合适的模式

1. **动态组件注入系统**: 当你的功能需要通过多种方式激活时使用
   - 例如：玩家可以通过基因获得能力，也可以通过药物获得，还可以通过装备获得

2. **配置统一转换模式**: 当你的功能需要在多个系统中使用相同参数时使用
   - 例如：冷却时间、效果强度等参数在Gene和Hediff中需要保持一致

3. **多版本兼容模式**: 当你的模组需要支持多个RimWorld版本时使用
   - 例如：同时支持1.5和1.6版本

### 组合使用建议

这些模式可以灵活组合：

- **动态组件注入 + 配置统一转换**: 最常见的组合，实现多入口点激活且配置统一
- **多版本兼容 + 其他模式**: 确保所有模式都支持多版本

### 模板使用流程

1. 阅读对应模式的完整文档
2. 根据需求调整模板代码
3. 遵循文档中的实施步骤
4. 参考检查清单验证实现

如果您想添加新的设计模式模板，请遵循以下步骤：
