## 第1章 Def数据定义系统 ★★★

> 所有游戏内容的数据基础，所有RimWT实体都需要定义Def

### 1.1 Def体系总览与XML继承

RimWorld的所有游戏内容都通过Def（Definition）定义。核心Def类型：

| Def类型 | 命名空间 | 定义内容 | 典型实例 |
|---------|---------|---------|---------|
| `ThingDef` | Verse | 物品/建筑/Pawn种族 | 钢铁、工作台、Human |
| `HediffDef` | Verse | 健康状况 | 流感、仿生眼、切割伤 |
| `GeneDef` | Verse | 基因 | 强壮、吸血、毛皮 |
| `AbilityDef` | Verse | 能力 | 吸血、传送、心灵冲击 |
| `TraitDef` | RimWorld | 特性 | 勤劳、嗜血、和平主义 |
| `FactionDef` | RimWorld | 派系 | 海盗、帝国、虫族 |
| `IncidentDef` | RimWorld | 事件 | 袭击、贸易、疾病 |
| `PawnKindDef` | Verse | Pawn变体 | 海盗步兵、帝国骑士 |
| `NeedDef` | RimWorld | 需求 | 饥饿、休息、心情 |
| `StatDef` | RimWorld | 属性 | 移动速度、射击精度 |
| `DamageDef` | Verse | 伤害类型 | 切割、钝击、火焰 |

**`ParentName`继承机制**：XML中通过`ParentName`实现模板继承，`Abstract="True"`标记抽象模板不生成实例。子Def继承父Def的所有字段，可覆盖任意字段。

```xml
<!-- 抽象父模板 -->
<HediffDef ParentName="DiseaseBase" Abstract="True">
  <hediffClass>HediffWithComps</hediffClass>
  <isBad>true</isBad>
  <tendable>true</tendable>
</HediffDef>

<!-- 具体子Def -->
<HediffDef ParentName="DiseaseBase">
  <defName>Flu</defName>
  <label>flu</label>
  <!-- 继承DiseaseBase的所有字段，只需定义差异 -->
</HediffDef>
```

📖 `Hediff分类与继承体系`、`基因系统结构与继承体系`、`伤害类型分类与DamageDef体系`

### 1.2 ThingDef与PawnKindDef双层架构

Pawn的定义采用双层架构：

| 层级 | Def类型 | 定义内容 | 粒度 |
|------|---------|---------|------|
| 种族层 | `ThingDef`（race配置） | 身体类型、基础属性、种族特征 | 种族共性 |
| 变体层 | `PawnKindDef` | 装备、技能、外观、战斗力 | 个体变体 |

一个`ThingDef`（如Human）可以对应多个`PawnKindDef`（如海盗步兵、帝国骑士、商人）。

**ThingDef关键字段体系**（装备/物品/建筑通用）：

| 字段组 | 关键字段 | 说明 |
|--------|---------|------|
| 基础 | `defName`, `label`, `description` | 标识和显示 |
| 分类 | `category`, `thingClass` | 物品类别和C#类 |
| 属性 | `statBases` | 基础属性值列表 |
| 组件 | `comps` | Comp组件列表 |
| 装备 | `apparel`, `equippedStatOffsets` | 服装/武器配置 |
| 生产 | `recipeMaker`, `costList` | 生产配方和材料 |

📖 `原版机械族单位总览`、`装备大类与穿戴规则`、`物品分类与堆叠系统`

### 1.3 HediffDef的三层效果配置

HediffDef通过三层机制配置效果：

```
HediffDef
├── 层1: Def字段（全局属性）
│   ├── initialSeverity, lethalSeverity, maxSeverity
│   ├── isBad, chronic, tendable
│   └── preventsDeath, countsAsAddedPartOrImplant
├── 层2: stages阶段列表（Severity驱动切换）
│   ├── HediffStage[0]: minSeverity=0
│   │   ├── statOffsets / statFactors
│   │   ├── capMods（能力修改）
│   │   ├── painOffset / painFactor
│   │   └── hediffGivers（阶段触发器）
│   └── HediffStage[1]: minSeverity=0.5
│       └── ...更严重的效果
└── 层3: comps组件列表（行为扩展）
    ├── HediffComp_SeverityPerDay（每日变化）
    ├── HediffComp_Immunizable（免疫竞赛）
    ├── HediffComp_TendDuration（治疗效果）
    └── ...其他组件
```

**Severity驱动阶段切换**：阶段按`minSeverity`升序排列，Severity越高进入越靠后的阶段。Severity≤0时自动移除，≥`lethalSeverity`时致死。

📖 `Hediff分类与继承体系`、`植入体效果机制`、`Severity严重程度机制`

### 1.4 GeneDef的字段体系

GeneDef拥有80+字段，覆盖16种效果类型：

| 效果类型 | 关键字段 | 说明 |
|---------|---------|------|
| 属性修改 | `statOffsets`, `statFactors`, `conditionalStatAffecters` | 加值/倍率/条件修改 |
| 能力赋予 | `abilities` | 赋予能力列表 |
| 能力修改 | `capMods` | 视力/操作等能力修改 |
| 特质控制 | `forcedTraits`, `suppressedTraits` | 强制/压制特质 |
| Need控制 | `enablesNeeds`, `disablesNeeds` | 启用/禁用需求 |
| 外观 | `renderNodeProperties`, `skinColor*`, `fur`, `bodyType` | 视觉效果 |
| 免疫 | `makeImmuneTo`, `hediffGiversCannotGive`, `damageFactors` | 免疫和抗性 |
| 三维约束 | `biostatCpx`, `biostatMet`, `biostatArc` | 复杂度/代谢/远古科技 |

**`statOffsets` vs `statFactors`**：Offset是加值（所有来源累加），Factor是倍率（所有来源相乘）。计算顺序：先累加Offset，再乘以Factor。

📖 `基因效果分类与实现机制`、`代谢效率与Biostat系统`

## 第2章 Comp组件模式 ★★★

> RimWorld最核心的扩展模式——通过挂载组件为实体添加行为，无需修改基类

### 2.1 ThingComp生命周期与挂载

ThingComp是Thing（物品/建筑/Pawn）的行为组件。XML中通过`<comps>`列表声明，运行时自动实例化。

**生命周期方法**：

| 方法 | 调用时机 | 典型用途 |
|------|---------|---------|
| `CompTick()` | 每tick（或按interval） | 状态更新（如CompShield能量恢复） |
| `PostSpawnSetup(respawningAfterLoad)` | Thing生成时 | 初始化组件状态 |
| `PostDeSpawn(map)` | Thing从地图移除时 | 清理资源 |
| `PostPreApplyDamage(ref DamageInfo, out bool absorbed)` | 伤害应用前 | 拦截伤害（如CompShield） |
| `PostPostApplyDamage(DamageInfo, float)` | 伤害应用后 | 受伤反应 |
| `CompGetGizmosExtra()` | UI刷新时 | 添加操作按钮 |
| `PostExposeData()` | 存档读写时 | 序列化组件数据 |
| `CompAllowVerbCast(Verb)` | 攻击前检查 | 限制攻击方式（如护盾禁远程） |

**挂载方式**：XML中`<comps>`列表 → C#中`CompProperties`子类 → 运行时`ThingComp`子类实例化。

```xml
<ThingDef>
  <comps>
    <li Class="CompProperties_Shield" />           <!-- 护盾组件 -->
    <li Class="CompProperties_Forbiddable" />      <!-- 可禁止组件 -->
    <li Class="CompProperties_ProjectileInterceptor">  <!-- 带参数的组件 -->
      <radius>3.9</radius>
      <hitPoints>50</hitPoints>
    </li>
  </comps>
</ThingDef>
```

**关键设计**：CompShield的案例证明——护盾腰带的`thingClass`就是普通`Apparel`，所有护盾行为完全由`CompShield`组件提供。这是Comp模式的核心价值：**行为与实体类型解耦**。

📖 `护盾机制与伤害吸收系统`

### 2.2 HediffComp生命周期与交互

HediffComp是Hediff（健康状况）的组件系统，与ThingComp平行但独立。

**生命周期方法**：

| 方法 | 调用时机 | 典型用途 |
|------|---------|---------|
| `CompPostTick(ref float severityAdjustment)` | 每tick | 实时效果（如范围光环） |
| `CompPostTickInterval()` | 每`checkInterval` ticks（默认200） | 定期检查（如严重度触发） |
| `CompPostPostAdd(DamageInfo?)` | Hediff添加后 | 初始化 |
| `CompPostPostRemoved()` | Hediff移除后 | 清理 |
| `Notify_PawnPostApplyDamage(DamageInfo, float)` | Pawn受伤后 | 受伤反应 |
| `Notify_PawnDied(DamageInfo?, Hediff)` | Pawn死亡时 | 死亡效果 |

**9个关键HediffComp按功能分类**：

| 类别 | Comp | 核心行为 |
|------|------|---------|
| 给予类 | `HediffComp_GiveHediff` | Severity≥阈值时给予新Hediff |
| 给予类 | `HediffComp_GiveHediffsInRange` | 范围光环，每5 ticks刷新 |
| 替换类 | `HediffComp_ReplaceHediff` | Severity≥阈值时替换为新Hediff |
| 移除类 | `HediffComp_RemoveIfOtherHediff` | 检测到互斥Hediff时移除自身 |
| 反应类 | `HediffComp_ReactOnDamage` | 受特定伤害时触发反应 |
| 摧毁类 | `HediffComp_DestroyOrgan` | Severity达上限时摧毁器官 |
| 免疫类 | `HediffComp_Immunizable` | 免疫竞赛机制 |
| 治疗类 | `HediffComp_TendDuration` | 治疗效果持续时间 |
| 消失类 | `HediffComp_Disappears` | 定时自动消失 |

**HediffGiver的三层配置**：HediffDef层（该Hediff固有）、Stage层（阶段性触发）、种族层（`HediffGiverSetDef`，种族固有如出血→失血）。

📖 `Hediff互相影响与触发机制`

### 2.3 GeneComp与AbilityComp

**Gene没有Comp系统**——Gene类直接通过重写生命周期方法扩展：

| Gene方法 | 调用时机 | 典型用途 |
|---------|---------|---------|
| `PostAdd()` | 基因添加时 | 注入Hediff、修改外观 |
| `PostRemove()` | 基因移除时 | 清理效果 |
| `Tick()` | 每tick | 运行时逻辑 |

社区扩展基因的4种模式：

| 模式 | 需要C# | 代表 | 适用场景 |
|------|--------|------|---------|
| 纯XML配置 | 否 | GeneDef 80+字段 | 属性/能力/外观等标准效果 |
| DefModExtension | 是 | VEF `GeneExtension` 50+字段 | 需要额外数据字段 |
| 自定义Gene子类 | 是 | `Gene_AutoBlink` | 需要运行时逻辑 |
| Harmony补丁 | 是 | HAR种族限制 | 修改原版行为 |

**AbilityComp**（`CompAbilityEffect`子类）：Ability的效果组件，30+种子类。通过XML组合多个CompAbilityEffect实现复合能力效果（详见1.5节）。

📖 `模组自定义基因与扩展模式`、`Ability系统基本结构`

## 第3章 Worker委托模式 ★★☆

> 将具体行为逻辑委托给可替换的Worker类，Def只持有数据和Worker类型引用

### 3.1 DamageWorker与伤害处理

DamageWorker是伤害处理的核心委托——DamageDef通过`workerClass`字段指定Worker子类，决定伤害如何分配到身体部位。

**继承树**（14个子类）：

```
DamageWorker (Verse, 基类)
├── DamageWorker_Stun          ← 不造成伤害，仅眩晕
├── DamageWorker_Extinguish    ← 灭火
└── DamageWorker_AddInjury     ← 默认：单部位伤害
    ├── DamageWorker_Cut       ← 劈砍溢出到相邻部位
    ├── DamageWorker_Blunt     ← 40%概率冲击内部 + 眩晕
    ├── DamageWorker_Stab      ← 穿刺传播（内40%/外75%）
    ├── DamageWorker_Scratch   ← 分裂到2个部位
    ├── DamageWorker_Bite      ← 同Cut行为
    ├── DamageWorker_Flame     ← 点燃目标 + 地面起火
    ├── DamageWorker_Frostbite ← 仅Outside+有冻伤脆弱性的部位
    ├── DamageWorker_Nerve     ← 造成伤害后追加眩晕
    ├── DamageWorker_Vaporize  ← 内圈蒸发摧毁部位
    └── DamageWorker_MiningBomb ← 高建筑伤害
```

**Worker的两个核心虚方法**：

| 方法 | 职责 | 子类差异点 |
|------|------|-----------|
| `ChooseHitPart(DamageInfo)` | 选择命中部位 | Cut/Scratch强制Outside，Stab概率穿透Inside |
| `ApplySpecialEffectsToPart(Pawn, float, DamageInfo, DamageResult)` | 应用特殊效果 | Cut溢出、Blunt内部冲击、Flame点燃 |

```xml
<!-- 自定义伤害类型示例 -->
<DamageDef>
  <defName>MyCustomDamage</defName>
  <workerClass>MyMod.DamageWorker_Custom</workerClass>
  <armorCategory>Sharp</armorCategory>
  <hediff>Cut</hediff>
  <isRanged>true</isRanged>
</DamageDef>
```

📖 `伤害类型分类与DamageDef体系`

### 3.2 IncidentWorker与事件执行

IncidentWorker是事件执行的委托——IncidentDef通过`incidentClass`字段指定Worker子类。

**核心方法**：

| 方法 | 职责 | 说明 |
|------|------|------|
| `CanFireNow(IncidentParms)` | 前置检查（15+项） | 地图/派系/冷却/人口等条件 |
| `TryExecute(IncidentParms)` | 执行事件逻辑 | 返回bool表示是否成功 |

**事件触发调用链**：

```
Storyteller.StorytellerTick()              ← 每1000 ticks
  └── MakeIncidentsForInterval()
      ├── StorytellerComp.MakeIntervalIncidents()  ← 21个子类各自生成
      └── TryFire(FiringIncident)
          ├── IncidentWorker.CanFireNow()
          └── IncidentWorker.TryExecute()
```

**两种调度模式**：
- `StorytellerComp_RandomMain`（Randy）：MTB随机，平均1.35天触发一次，类别加权随机
- `StorytellerComp_OnOffCycle`（Cassandra/Phoebe）：确定性开关周期，按天数/威胁点数计算事件数

📖 `事件触发机制与叙事者系统`

### 3.3 其他Worker模式

RimWorld中Worker委托模式广泛应用：

| Worker类型 | Def类型 | 职责 | 典型子类 |
|-----------|---------|------|---------|
| `DamageWorker` | DamageDef | 伤害分配到部位 | 14个子类 |
| `IncidentWorker` | IncidentDef | 事件执行逻辑 | `IncidentWorker_RaidEnemy`等 |
| `StatWorker` | StatDef | 属性计算逻辑 | `StatWorker_MeleeAverageDPS`等 |
| `RecipeWorker` | RecipeDef | 配方执行逻辑 | `Recipe_InstallImplant`等 |
| `PlaceWorker` | ThingDef | 建筑放置验证 | `PlaceWorker_NeedsFuelingPort`等 |
| `RaidStrategyWorker` | RaidStrategyDef | 袭击策略 | `RaidStrategyWorker_ImmediateAttack`等 |
| `PawnsArrivalModeWorker` | PawnsArrivalModeDef | 到达方式 | `PawnsArrivalModeWorker_EdgeWalkIn`等 |

**模式共性**：Def持有数据 + `workerClass`字段指定C#类 → 运行时实例化Worker → 调用虚方法执行逻辑。这使得同一Def类型可以有完全不同的行为。

📖 `袭击策略与到达模式`

## 第4章 Tracker状态管理 ★★★

> Pawn的各子系统通过独立Tracker管理，每个Tracker封装一个领域的完整状态和逻辑

### 4.1 Pawn_HealthTracker

`Pawn_HealthTracker`是最核心的Tracker——管理Hediff集合、伤害应用、死亡判定、状态变化。

**核心职责**：

| 方法 | 职责 | 调用时机 |
|------|------|---------|
| `AddHediff(Hediff, BodyPartRecord, DamageInfo?)` | 添加健康状况 | 受伤/患病/植入 |
| `RemoveHediff(Hediff)` | 移除健康状况 | 治愈/手术 |
| `PreApplyDamage(ref DamageInfo, out bool absorbed)` | 伤害预处理（拦截链） | 伤害管线阶段4 |
| `CheckForStateChange(DamageInfo?, Hediff)` | 状态变化检查 | 每次Hediff增减后 |
| `ShouldBeDead()` | 死亡判定（5个条件） | CheckForStateChange内部 |

**ShouldBeDead()的5个判定条件**（短路求值）：

| # | 条件 | 判定逻辑 |
|---|------|---------|

## 第5章 Stat属性计算系统 ★★★

> 所有数值属性通过StatDef定义、StatWorker计算、StatPart修饰的三层管线

### 5.1 StatDef与StatWorker

**StatDef**定义一个属性（如移动速度、射击精度、护甲值），**StatWorker**负责计算最终值。

**StatWorker计算管线**：

```
StatWorker.GetValueUnfinalized(StatRequest req)
  1. 基础值: StatDef.defaultBaseValue 或 Thing的statBases
  2. + 加法偏移（遍历所有StatPart.TransformValue）
     ├── StatPart_Age          ← 年龄修饰
     ├── StatPart_Health       ← 健康修饰
     ├── StatPart_GearStatOffset ← 装备Stat加法传递
     ├── StatPart_GearAndEquipmentStatOffset ← equippedStatOffsets
     └── ...其他StatPart
  3. × 乘法因子
     ├── StatPart_GearStatFactor ← 装备Stat乘法传递
     └── ...
  4. Clamp(minValue, maxValue)
```

**StatPart**是Stat计算的插件——每个StatDef可以挂载多个StatPart，在计算过程中依次修饰值。这是Stat系统的核心扩展点。

### 5.2 多来源属性叠加

装备通过三条路径影响Pawn属性：

| 路径 | 机制 | 叠加方式 | 典型应用 |
|------|------|---------|---------|
| `equippedStatOffsets` | ThingDef上声明的穿戴偏移 | 加法 | 武器射击精度加成 |
| `StatPart_GearStatOffset` | 装备自身Stat值传递给Pawn | 加法 | 护甲值（服装→Pawn） |
| `StatPart_GearStatFactor` | 装备自身Stat值作为乘数 | 乘法 | 射击精度因子、闪避因子 |

**Offset vs Factor计算顺序**：先累加所有Offset（加法），再连乘所有Factor（乘法）。

```xml
<!-- equippedStatOffsets示例 -->
<ThingDef>
  <defName>Gun_Revolver</defName>
  <equippedStatOffsets>
    <ShootingAccuracyPawn>0.5</ShootingAccuracyPawn>
    <MoveSpeed>-0.1</MoveSpeed>
  </equippedStatOffsets>
</ThingDef>
```

**品质影响**：`equippedStatOffsets`的值还会经过StatParts处理（如品质倍率），传奇品质武器的偏移效果更强。

**多来源叠加实例**（Pawn护甲计算）：

```
Pawn.ArmorRating_Sharp =
  基础值(0)
  + 服装1.GetStatValue(ArmorRating_Sharp)  ← StatPart_GearStatOffset
  + 服装2.GetStatValue(ArmorRating_Sharp)
  + 基因statOffsets                         ← GeneDef.statOffsets
  + Hediff.stages[].statOffsets             ← HediffStage.statOffsets
  × 基因statFactors                         ← GeneDef.statFactors
```

📖 `装备属性叠加计算`、`品质系统`

## 第6章 伤害管线与拦截架构 ★★★

> 从攻击发起到Hediff_Injury最终添加的7阶段流水线，提供10个拦截/修改点

### 6.1 7阶段伤害管线

```
阶段1  Verb.TryCastShot()
  ├── 近战: 命中判定 → 闪避判定 → DamageInfo
  └── 远程: 投射物生成 → 飞行 → 命中/掩体判定 → DamageInfo

阶段2  DamageInfo创建
  → new DamageInfo(Def, Amount, ArmorPen, Instigator, Weapon...)

阶段3  Thing.TakeDamage(dinfo)
  → ① ThingDef.damageMultipliers 倍率

阶段4  PreApplyDamage拦截链
  → ② Gene.damageFactors 基因倍率
  → ③ Hediff.damageFactors Hediff倍率
  → ④ Comp.PostPreApplyDamage (CompShield等，可absorbed)
  → ⑤ Apparel.CheckPreAbsorbDamage (护盾腰带，可absorbed)

阶段5  DamageWorker_AddInjury.ApplyDamageToPart()
  → ⑥ ArmorUtility.GetPostArmorDamage 逐层护甲判定
  → ⑦ IncomingDamageFactor Stat倍率

阶段6  FinalizeAndAddInjury()
  → ⑧ 即死保护（殖民者专属，最多7次降低Severity）
  → 创建Hediff_Injury + additionalHediffsThisPart

阶段7  PostApplyDamage()
  → ⑨ DamageDef.additionalHediffs 附加Hediff（毒素/感染）
  → ⑩ hediff.Notify_PawnPostApplyDamage 受伤后反应
  → ShouldBeDead() → Kill() 或 ForceDeathrestOrComa()
```

### 6.2 10个伤害拦截/修改点

| # | 位置 | 机制 | 效果 | 模组扩展方式 |
|---|------|------|------|-------------|
| ① | 阶段3 | `ThingDef.damageMultipliers` | 倍率修改伤害量 | XML配置 |
| ② | 阶段4 | `Gene.damageFactors` | 基因伤害倍率 | GeneDef XML |
| ③ | 阶段4 | `Hediff.damageFactors` | Hediff伤害倍率 | HediffDef XML |
| ④ | 阶段4 | `Comp.PostPreApplyDamage` | 完全吸收伤害 | 自定义ThingComp |
| ⑤ | 阶段4 | `Apparel.CheckPreAbsorbDamage` | 服装级吸收 | 自定义ThingComp |
| ⑥ | 阶段5 | `ArmorUtility.GetPostArmorDamage` | 偏转/半减/穿透 | Stat修改 |
| ⑦ | 阶段5 | `IncomingDamageFactor` | Stat倍率 | StatDef |
| ⑧ | 阶段6 | 即死保护 | 殖民者防一击致死 | 引擎内置 |
| ⑨ | 阶段7 | `additionalHediffs` | 附加毒素/感染 | DamageDef XML |
| ⑩ | 阶段7 | `Notify_PawnPostApplyDamage` | 受伤后反应 | HediffComp |

**护甲判定公式**（阶段5⑥）：

```
effectiveArmor = max(armorRating - armorPenetration, 0)
roll = Rand.Value
roll < eA×0.5 → 完全偏转(damAmount=0)
roll < eA     → 半减(damAmount/=2, Sharp→Blunt转换)
else          → 全额穿透
```

逐层判定：先遍历覆盖受击部位的每件服装，最后判定Pawn自身护甲。每层独立随机。

### 6.3 护盾的两种实现模式

| 维度 | CompShield | CompProjectileInterceptor |
|------|-----------|--------------------------|
| 拦截阶段 | 阶段4 PostPreApplyDamage | 投射物飞行阶段（阶段1-2之间） |
| 拦截范围 | 仅穿戴者/宿主 | radius范围内所有友方 |
| 拦截类型 | 远程+爆炸（不拦截近战） | 投射物（可配置地面/空中） |
| 能量机制 | 浮点energy，每tick恢复 | 整数hitPoints，周期充能 |
| EMP弱点 | 直接击破(energy=0) | disarmedByEmpForTicks |
| 典型实例 | 护盾腰带、机械体内置 | 建筑护盾、低护盾 |

**关键设计**：CompProjectileInterceptor在投射物飞行时拦截（最早），CompShield在伤害应用前拦截（阶段4）。如果投射物被前者拦截，根本不会到达后者。`DamageDef.ignoreShields`仅影响CompShield。

📖 `战斗伤害管线完整流程`、`伤害减免机制总览`、`护盾机制与伤害吸收系统`

## 第7章 Pawn形态切换技术架构 ★★★

> RimWorld有5种形态/状态切换先例，共同模式：状态标记 + 系统通知 + 副作用链

### 7.1 MutantDef与Pawn_MutantTracker

变异体转化（Anomaly DLC）是最全面的形态改变——行为/外观/需求/战斗/社交/装备/派系/基因全维度。

**核心架构**：`MutantDef`（100+字段定义变异形态）+ `Pawn_MutantTracker`（管理转化/恢复逻辑）。

**MutantDef字段按维度分类**：

| 维度 | 关键字段 | 说明 |
|------|---------|------|
| 行为 | `thinkTree`, `workDisables`, `canBeDrafted` | 替换AI行为树 |
| 外观 | `skinColorOverride`, `forcedHeadTypes`, `renderNodeProperties` | 皮肤/头型/动画 |
| 需求 | `disableNeeds`, `needWhitelist`, `foodType` | 禁用Need（可白名单） |
| 战斗 | `tools`, `verbs` | 自定义近战/远程攻击 |
| 装备 | `dropEquipment`, `destroyGearOnTurn` | 掉落/销毁装备 |

**Turn()转化流程**（20+副作用链）：

```
Pawn_MutantTracker.Turn(MutantDef)
  → 设置mutantDef标记
  → 应用外观变化（皮肤/头型/体型/动画）
  → 禁用Need（按白名单保留）
  → 掉落/销毁装备
  → 重置基因（可选）
  → 替换ThinkTree
  → 通知所有系统刷新
```

**Revert()恢复局限**：恢复不完全——外观可恢复，但已销毁的装备和已重置的基因无法恢复。

**3个MutantDef实例**：Shambler（行尸，不可征召/不可开门/爬行攻击）、Ghoul（食尸鬼，可征召/仅食肉）、Chimera（嵌合体，最复杂）。

## 附录B 关键XML字段速查表

按Def类型分组，列出模组开发最常用的XML字段。

### ThingDef（物品/建筑/Pawn种族）

| 字段 | 类型 | 说明 |
|------|------|------|
| `defName` | string | 唯一标识符 |
| `thingClass` | Type | C#类（默认Thing） |
| `category` | ThingCategory | Item/Building/Pawn等 |
| `statBases` | List | 基础属性值 |
| `comps` | List | Comp组件列表 |
| `equippedStatOffsets` | List | 穿戴属性偏移 |
| `damageMultipliers` | List | 伤害倍率 |
| `race` | RaceProperties | Pawn种族配置 |

### HediffDef

| 字段 | 类型 | 说明 |
|------|------|------|
| `hediffClass` | Type | C#类（默认Hediff） |
| `initialSeverity` / `lethalSeverity` / `maxSeverity` | float | 严重度控制 |
| `stages` | List | 阶段列表（Severity驱动切换） |
| `comps` | List | HediffComp组件列表 |
| `isBad` / `tendable` / `chronic` | bool | 基础属性 |
| `preventsDeath` | bool | 死亡豁免 |
| `hediffGivers` | List | Hediff给予器 |

### GeneDef

| 字段 | 类型 | 说明 |
|------|------|------|
| `geneClass` | Type | C#类（默认Gene） |
| `statOffsets` / `statFactors` | List | 属性加值/倍率 |
| `abilities` | List | 赋予能力 |
| `capMods` | List | 能力修改 |
| `forcedTraits` / `suppressedTraits` | List | 强制/压制特质 |
| `enablesNeeds` / `disablesNeeds` | List | 启用/禁用需求 |
| `biostatCpx` / `biostatMet` / `biostatArc` | int | 三维约束 |
| `damageFactors` | List | 伤害倍率 |

### DamageDef

| 字段 | 类型 | 说明 |
|------|------|------|
| `workerClass` | Type | DamageWorker子类 |
| `armorCategory` | DamageArmorCategoryDef | 护甲类别（Sharp/Blunt/Heat） |
| `hediff` | HediffDef | 默认创建的Hediff |
| `isRanged` / `isExplosive` | bool | 行为标志位 |
| `ignoreShields` | bool | 绕过护盾 |
| `additionalHediffs` | List | 附加Hediff（毒素/感染） |

### StatDef

| 字段 | 类型 | 说明 |
|------|------|------|
| `workerClass` | Type | StatWorker子类 |
| `defaultBaseValue` | float | 默认基础值 |
| `minValue` / `maxValue` | float | 值范围 |
| `parts` | List | StatPart插件列表 |
