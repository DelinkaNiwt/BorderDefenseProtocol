# 后续AI工作指引

## 一、项目目标

为RimWorld开发一个基于《境界触发者》(World Trigger)的模组，**不是还原原作世界观，而是借鉴原作设计以Trion为核心的新机制体系玩法**。

**核心定位**：
- 借鉴原作设计新玩法，而非复刻WT世界
- 以Trion机制体系为核心（能量→身体→装备→战斗）
- 双向成长："让自己和敌人都变强"
- 区别于现有RimWorld模组的新体验
- 时间线：RimWorld宇宙的平行设定

## 六、遵循原则

### 项目核心原则
1. **最简原则**：不做冗余设计，删除不必要的内容
2. **单一职责**：每个概念、机制只负责一个明确的功能
3. **以Trion为核心**：所有机制围绕Trion能量展开
4. **双向成长**：玩家和敌人都要有成长路径
5. **新玩法体验**：区别于现有RimWorld模组

### 文档编写原则
1. 遵循项目文档规范（元信息、标签、历史记录）
2. 简洁明确，避免冗余
3. 标注信息来源和价值
4. 中文编写，提供术语对照

### 概念设定原则
1. **现阶段不涉及技术实现**，仅明确概念和设定
2. 选择性借鉴原作，不是全盘复制
3. 以"对模组开发的价值"为导向
4. 明确标注哪些需要纳入模组、哪些仅作参考

## 七、关键问题提醒

在填充大纲时，需要明确以下问题：

1. **还原度取舍**：
   - 哪些设定必须还原？（如Trion的核心机制）
   - 哪些可以调整？（如数值平衡）
   - 哪些可以原创？（如新的Trigger类型）

2. **玩家身份定位**（目前未明确确定）：
   - 玩家是Border指挥官？
   - 玩家是偶然获得Trigger技术的殖民者？
   - 玩家是近界民在边缘世界建立据点？
   - *可在填充过程中与用户确认*

3. **模组范围界定**：
   - 明确哪些原作内容不纳入模组（如原作角色、剧情）
   - 明确MVP（最小可玩版本）的边界

2. 按章节逐一提取有价值信息

1. 基于大纲0的资料，设计核心机制概念
2. 每个机制明确：是什么、怎么运作、能设计出什么玩法
3. 以Trion为主线贯穿8章

### 阶段3：填充大纲B
1. 基于大纲A的机制，设计玩家体验
2. 每个模块描述：玩家会玩到什么、感受到什么
3. 确保体验完整且有吸引力

### 阶段4：需求分析与设计（更后期）
1. 基于大纲A和B，进行系统需求分析
2. 架构设计
3. 开发实现
