## 决策历史

### 20260127-001 | 项目基础结构设计

**决策时间：** 2026-01-27

**决策内容：**
采用模块化可扩展的知识库结构，而不是简单的文档堆砌。

**决策原因：**
1. 用户在整理术语词典时发现内容涉及Trion兵、副作用、BORDER体系等多个领域
2. 简单的单文件结构无法应对这些内容的复杂性
3. 用户强调"提到的不代表全部"，需要预留扩展空间
4. 分类有利于新AI快速定位和理解相关内容

**决策方案：**
- 知识库分为四个主要分类：核心机制、触发器图鉴、战斗体验档案、敌对势力
- 每个分类下有多个独立模块，每个模块是一个独立文档
- 建立各分类的索引文档，方便查阅和管理
- 预留"...可扩展"的位置，明确这不是最终完整列表

**影响范围：**
- 阶段1的所有知识库文档结构
- 后续各阶段的参考逻辑
- AI协作指南的描述

**签名：** Claude Code

### 20260127-002 | 提示词模板库

**决策时间：** 2026-01-27

**决策内容：**
为不同阶段创建专门的AI提示词模板，而不是让用户每次手动说明背景。

**决策原因：**
1. 用户指出"只是告诉我这么做不够，你还得帮我让AI知道该怎么做"
2. 新的对话窗口AI需要快速理解项目背景，文字指导容易遗漏
3. 完整的提示词模板能确保一致性，减少理解偏差
4. 降低用户的操作复杂度（只需复制粘贴）

**决策方案：**
- 创建 `提示词模板库` 目录
- 为阶段0、1、2各创建一个完整的提示词模板
- 每个模板包含：项目背景、当前阶段、工作方式、质量标准、关键位置
- 用户启动新对话时，复制对应阶段的提示词粘贴给AI

**影响范围：**
- `00_项目管理/提示词模板库/` 下的所有文件
- AI协作指南中的"工作流程"章节
- 用户的实际工作方式

**签名：** Claude Code

### 20260127-003 | 术语词典的处理方式

**决策时间：** 2026-01-27

**决策内容：**
原术语词典不删除，保存到 `_原始资料` 作为参考源，新文档在此基础上重新组织。

**决策原因：**
1. 用户已经投入时间整理术语词典（已有v1.7版本）
2. 词典中包含了原作动漫/漫画的大量信息，删除会造成浪费
3. 保留原件便于交叉验证和追溯信息来源
4. 新组织的文档可以有针对性地补充"体验维度"和"战术特点"

**决策方案：**
- 复制原术语词典到 `_原始资料\术语词典_境界触发者_v1.7.md`
- 根据规划文档中的"表格"重新拆分和整理内容
- 每个新文档保留对原术语词典的参考链接
- 发现新内容时，先在原词典中查找，没有才新增

**影响范围：**
- 所有知识库文档的内容来源
- 后续阶段的资料查阅方式
- 版本管理和更新记录

**签名：** Claude Code
