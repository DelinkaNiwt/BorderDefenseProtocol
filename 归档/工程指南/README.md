# 工程指南目录

## 📌 目录用途

本目录存放**系统级指导文件**——定义整个RimWorld模组工程体系的规范、框架和管理指南。

这些文件的特点是：
- ✅ **仅修改，不删除** — 这些文件定义了系统的基础规范
- ✅ **跨项目适用** — 所有项目和所有角色都要遵守这些规范
- ✅ **低变化率** — 这些文件的修改频率很低，通常是版本升级时才调整
- ✅ **优先级高** — 其他所有文档都应该引用这些文件，而不是复制内容

## ⚠️ 重要约束

1. **版本管理**：工程指南中的文件版本升级时，应该通知所有相关文档进行更新
2. **引用优先**：在角色设计或项目文档中，应该**引用**这些文件，而不是**复制**内容
3. **修改流程**：修改这些文件时应该通过：
   - 在版本历史中记录修改
   - 评估对其他文档的影响
   - 通知所有相关角色和项目
4. **回溯链接**：被修改的地方应该有明确的版本标注，便于追溯
