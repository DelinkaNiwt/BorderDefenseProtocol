# 日志管理

本目录包含RimworldModStudio项目的所有日志记录。

## 目录结构

```
日志/
├── README.md              # 本文件，日志管理说明
├── 开发日志/             # 开发过程中的日志、笔记、总结
│   ├── *.md              # 各类开发相关日志
│   └── *.png             # 开发过程中的截图、图表
└── Agent工作日志/        # 7个Agent的工作日志（使用时自动生成）
    ├── 01_头脑风暴Agent/
    ├── 02_需求分析Agent/
    ├── ...
    └── 07_更新维护Agent/
```

## 日志分类

### 开发日志 (开发日志/)

**用途**：记录开发过程中的各类信息

**包含**：
- 设计决策记录
- 技术探索笔记
- 问题排查总结
- 测试验证记录
- 性能分析报告
- 版本更新说明
- 项目进度记录

**命名规范**：
- 按日期或功能命名
- 格式：`YYYYMMDD_描述.md` 或 `功能名_描述.md`
- 示例：`20260123_项目初始化.md`、`性能优化_缓存策略.md`

### Agent工作日志 (Agent工作日志/)

**用途**：记录各Agent的工作进展（项目启动时自动创建）

**包含**：
- Agent的工作输出副本
- 每个Agent阶段的决策记录
- 质量检查结果
- 用户反馈

**目录结构**：
```
Agent工作日志/
├── [项目名]/          # 项目名目录
│   ├── 01_头脑风暴Agent/
│   │   ├── [时间戳]_思路整理.md
│   │   ├── [时间戳]_概念框架.md
│   │   └── ...
│   ├── 02_需求分析Agent/
│   ├── ...
│   └── 07_更新维护Agent/
```

## 日志维护指南

### 开发日志维护

1. **定期记录**：有重要进展时及时记录
2. **清晰描述**：标题清晰，内容完整
3. **及时更新**：重要文件更新时同步更新日志
4. **长期保留**：开发日志作为项目历史记录永久保留

### Agent工作日志维护

1. **项目启动时初始化**：由主Agent自动创建
2. **阶段完成时更新**：每个Agent完成工作后自动记录
3. **定期归档**：完成项目后归档到开发日志
4. **版本管理**：与模组版本关联

## 最佳实践

### 何时记录日志

✅ **应该记录的情况**：
- 重要的设计决策
- 遇到的问题和解决方案
- 性能优化或改进
- 版本更新和兼容性处理
- 重大项目里程碑
- 技术选型和权衡

❌ **无需记录的情况**：
- 日常代码编写
- 常见的编译报错（已在git中）
- 临时调试信息

### 记录内容规范

**好的日志**：
```markdown
# 性能优化：列表渲染缓存

## 问题
模组中大量的UI列表渲染导致帧率下降。

## 解决方案
实现了列表项的缓存机制...

## 效果
帧率从45提升到60...

## 相关RiMCP
- Verse.UI.Widgets: ListUI缓存机制
```

**不好的日志**：
```
修复bug
改进代码
```

### 文件命名

遵循以下规范：

- **日期戳格式**：YYYYMMDD (如 20260123)
- **时间戳格式**：YYYYMMDD_HHMM (如 20260123_1530)
- **功能名**：清晰简短，用下划线分隔单词
- **例子**：
  - `20260123_项目初始化.md`
  - `20260123_1530_RiMCP查证报告.md`
  - `RiMCP验证_ListUI缓存机制.md`

## 历史日志查询

要查找某个话题的日志：

```bash
# 查找包含"性能"的日志
grep -r "性能" 日志/

# 查找特定日期的日志
ls 日志/开发日志/20260123*

# 查找特定项目的Agent日志
ls 日志/Agent工作日志/RimTrion/
```

## 与git的关系

- 日志在版本控制中保留（不忽略）
- 重要的技术决策应该记录在日志中
- 日志是项目文档的重要补充
- 定期审查日志可以了解项目演进
