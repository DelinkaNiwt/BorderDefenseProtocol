## 背景

BDP 模组的双重攻击（`Verb_BDPDual`）在合成 `VerbProperties` 时，`soundCast` 固定取左手芯片的音效（`DualVerbCompositor.cs:285`）。用户希望左右手各自播放自己芯片配置的音效。

---

## 尝试方案及失败原因

### 方案一：在发射方法内临时替换 `verbProps.soundCast`

**思路**：在 `FireSideSimultaneous` / `FireSideSequential` / `FireBothSimultaneous` 发射前把 `verbProps.soundCast` 换成当前侧音效，发射后用 `finally` 还原。

**失败原因**：引擎播音效的时机在 `TryCastNextBurstShot` 里，`TryCastShot()` 返回**之后**才读取 `verbProps.soundCast` 播放。而我们的 `finally` 在 `TryCastShot` 内部就已执行，引擎读到的永远是还原后的旧值（左手音效）。

```
TryCastNextBurstShot()
  → TryCastShot()                    ← 我们在这里换音效、发射、finally还原
  → verbProps.soundCast.PlayOneShot() ← 引擎在这里播，此时已还原为旧值
```

### 方案二：在 `TryCastShotCore` 里手动播音效 + 置 null 阻止引擎重播

**思路**：在 `TryCastShotCore` 每次发射成功后，调用 `PlayShotSoundAndSuppress()`：手动播当前侧音效，然后 `verbProps.soundCast = null` 阻止引擎重复播。`Verb_BDPDual` 重写虚方法 `GetShotSound()` 按 `chipSide` 返回对应侧音效。

**失败原因**：`verbProps.soundCast = null` 修改的是 `VerbProperties` 对象上的字段。该对象在 `Verb_BDPDual` 的整个生命周期内共享，置 null 后**永久生效**，下一次射击时 `GetShotSound()` 读到 null，音效永远消失。

**表现**：第一声枪响后，单侧武器再也无法自己发出枪声，只有双重攻击时才有声音（因为双重攻击走 `TryCastShotCore` 路径，而单侧攻击的引擎路径读到了被置 null 的 `soundCast`）。

---

## 根本原因

引擎音效播放时机（`TryCastShot` 返回后）与代码可控时机（`TryCastShot` 内部）之间存在**结构性冲突**：

- 我们能控制的最晚时机：`TryCastShot` 返回前
- 引擎读取音效的时机：`TryCastShot` 返回后

任何在 `verbProps.soundCast` 上做临时替换的方案，都无法在引擎读取之后、下次发射之前完成还原，除非能 hook 到 `TryCastNextBurstShot` 的执行流程中。

---

## 当前状态

所有音效相关改动已回退，代码恢复干净状态。双重攻击仍使用左手芯片音效（原始行为）。

---

## 可行的后续方向

1. **Harmony Patch `TryCastNextBurstShot`**：在引擎播音效前（Prefix）把 `verbProps.soundCast` 换成当前侧音效，播完后（Postfix）还原。需要判断当前 Verb 是否为 `Verb_BDPDual`。

2. **Harmony Patch `SoundStarter.PlayOneShot`**：拦截音效播放，检查调用方是否为 BDP Verb，按需替换 SoundDef。侵入性较强。

3. **接受现状**：双重攻击统一播左手音效，不做区分。
