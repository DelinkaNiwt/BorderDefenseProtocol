## 🔍 第1章：Trion能量系统

### 1.1 四要素实现

**设定要求**：
```
Available = Capacity - Reserved - Consumed
关键规则：
- Capacity：个体固定属性，当前设计不可提升
- Reserved：战斗体生成时计算，主动解除时返还，被动解除时流失
- Consumed：不可逆，只能通过恢复补充
- Available：实时计算派生值，≤0触发Bail Out
```

**框架层实现**：✅ **完全符合**
- CompTrion.cs 正确实现四要素
- 关系式正确：`Available = Capacity - Reserved - Consumed`
- Reserved和Consumed的区分完整

**MVP层实现**：✅ **完全符合**
- DefaultTrionStrategy.GetBaseMaintenance() = 5.0f（维持消耗）
- Hediff_TrionGland正确存储天赋
- 天赋→容量映射正确

**结论**：✅ **无问题**

### 2.2 组件状态机

**设定要求**：
| 状态 | 条件 | 占用 | 提供功能 | 额外消耗 |
|------|------|------|---------|---------|
| 未连接 | 战斗体未生成/挂载点被毁 | 否 | 否 | 否 |
| 休眠 | 战斗体已生成，组件未激活 | 是 | 否 | 否 |
| 激活 | 战斗体已生成，组件已激活 | 是 | 是 | 部分 |

**状态转换规则**：
- 未连接→休眠：生成战斗体时自动
- 休眠→激活：支付激活费用+引导时间(1-5Tick)
- 激活→休眠：瞬间关闭，无延迟
- 休眠/激活→未连接：解除战斗体/挂载点毁

**框架层实现**：✅ **完全符合**
- 框架支持组件状态管理
- Reserved在休眠、激活时都锁定

**MVP层实现**：⚠️ **状态机不完整**
```
当前实现：
- IsActive: bool（仅二进制）
- 状态只有：Disconnected、Active

缺失状态转换：
❌ 无"Dormant"状态（休眠）
❌ 激活导引中间状态（Activating）
❌ 状态转换不包含引导阶段

问题表现：
- TriggerMount.Activate()直接激活，无引导
- 无法实现"激活→休眠→激活"的平衡设计
```

**修改方案**：
```csharp
public enum MountState
{
    Disconnected,   // 未连接
    Dormant,        // 休眠（锁定Reserved，无功能）
    Activating,     // 激活中（引导状态）
    Active          // 已激活（锁定Reserved，提供功能）
}

public class TriggerMount
{
    public MountState state = MountState.Disconnected;
    private int guidanceTicksLeft = 0;

    public void Tick()
    {
        if (state == MountState.Activating)
        {
            guidanceTicksLeft--;
            if (guidanceTicksLeft <= 0)
            {
                state = MountState.Active;
                // 通知UI激活完成
            }
        }
    }

    public bool IsReserving => state == MountState.Dormant ||
                               state == MountState.Activating ||
                               state == MountState.Active;
}
```

**缺失项**：🔴 **Dormant和Activating状态**

**严重性**：🔴 **高** - 影响组件生命周期完整性

**修改优先级**：⭐⭐⭐⭐⭐（第一阶段修复）

### 3.5 解除机制

#### 主动解除

**设定要求**：
```
触发方式：玩家手动操作
流程：
1. 战斗体消失
2. 组件注销（休眠/激活→未连接）
3. 占用量全部返还
4. 快照数据恢复到肉身
5. 生理活动恢复

惩罚：无
```

**框架层实现**：✅ **完全符合**
```csharp
public void DestroyCombatBody(DestroyReason reason)
{
    // 步骤1-2：战斗体消失+组件注销
    _isInCombat = false;
    foreach (var mount in _mounts)
    {
        mount.state = MountState.Disconnected;  // 未连接
    }

    // 步骤3：占用量返还
    if (reason == DestroyReason.Manual)
    {
        // 主动解除：Reserved直接返0
        _reserved = 0;  // 返还

    _snapshot.RestoreToPawn(pawn);
    // 生理恢复自动
}
```
