## 1. CompTrion 属性参数

### 1.1 Capacity（能量容量）

| 项 | 值 |
|-----|--------|
| **参数名** | Capacity |
| **含义** | Trion能量池的最大值 |
| **类型** | float |
| **有效范围** | > 0（必须为正数） |
| **默认值** | 应用层设置（框架不提供） |
| **影响范围** | 全局（决定战斗体耐久） |
| **变更影响** | 增加 → 更耐久；减少 → 更脆弱 |
| **存档兼容性** | 必须保存 |
| **可动态修改** | 可以（但影响是否应用由应用层决定） |

**约束条件**：
```
Capacity > 0
如果 Capacity <= 0: 框架不检查，应用层负责数据有效性
```

**参数验证**（应用层）：
```csharp
if (comp.Capacity <= 0)
{
    Log.Error("CompTrion: Capacity必须 > 0");
    comp.Capacity = 1000;  // 默认值
}
```

---

### 1.2 RecoveryRate（基础恢复速率）

| 项 | 值 |
|-----|--------|
| **参数名** | RecoveryRate |
| **含义** | 每60 Tick（1秒）的基础恢复量 |
| **类型** | float |
| **有效范围** | >= 0 |
| **默认值** | 应用层设置 |
| **影响范围** | CompTick恢复阶段 |
| **变更影响** | 增加 → 恢复快；减少 → 恢复慢 |
| **实际恢复量公式** | RecoveryRate * GetRecoveryModifier() |
| **存档兼容性** | 必须保存 |

**说明**：
- RecoveryRate是"基础"恢复速率
- 实际恢复量还受Strategy.GetRecoveryModifier()影响
- 修改系数可在不改变RecoveryRate的情况下调整恢复

---

### 1.3 LeakRate（泄漏速率）

| 项 | 值 |
|-----|--------|
| **参数名** | LeakRate |
| **含义** | 伤口导致的持续Trion泄漏速率 |
| **类型** | float |
| **有效范围** | >= 0 |
| **默认值** | 0（无泄漏）|
| **影响范围** | CompTick消耗计算 |
| **变更影响** | 增加 → 消耗快；减少 → 消耗慢 |
| **计算时机** | Strategy.OnDamageTaken()中更新 |
| **存档兼容性** | 必须保存 |

**应用层更新逻辑**：
```csharp
// 在Strategy中
public void OnDamageTaken(float damageAmount, BodyPartRecord hitPart)
{
    VirtualWound wound = new VirtualWound { part = hitPart, severity = damageAmount / 100f };
    wounds.Add(wound);

    // 重算泄漏速率
    float totalLeakage = 0;
    foreach (var w in wounds)
    {
        totalLeakage += w.severity * 2f;  // 配置：每单位伤口严重度泄漏2点/Tick
    }
    comp.LeakRate = totalLeakage;
}
```

## 2. TriggerComponentDef 参数

### 2.1 occupyingCapacity（占用容量）

| 项 | 值 |
|-----|--------|
| **参数名** | occupyingCapacity |
| **含义** | 装备此组件时占用的Trion容量 |
| **类型** | float |
| **有效范围** | >= 0 |
| **示例值** | 10, 20, 50（取决于组件复杂度） |
| **影响范围** | 战斗体初始化时 |
| **变更影响** | 增加 → 占用多；减少 → 占用少 |

**计算关系**：
```
装备此组件后：
Reserved += occupyingCapacity
Available = Capacity - Reserved - Consumed
```

**平衡建议**：
- 简单组件：10
- 中等组件：20-30
- 复杂组件：50+

---

### 2.2 activationCost（激活费用）

| 项 | 值 |
|-----|--------|
| **参数名** | activationCost |
| **含义** | 激活组件时一次性消耗的Trion |
| **类型** | float |
| **有效范围** | >= 0 |
| **示例值** | 5, 10, 50 |
| **影响范围** | TriggerMount.TryActivate()时 |
| **变更影响** | 增加 → 激活成本高；减少 → 激活容易 |

**触发时机**：
```
玩家激活组件时：
comp.Consume(componentDef.activationCost)
if (comp.Available < 0) 激活失败
```

**平衡建议**：
- 可以随意激活的组件：0
- 需要谨慎激活的组件：5-20
- 限制性很强的能力：50+

---

### 2.3 sustainCost（持续消耗）

| 项 | 值 |
|-----|--------|
| **参数名** | sustainCost |
| **含义** | 激活后每60 Tick的持续消耗 |
| **类型** | float |
| **有效范围** | >= 0 |
| **示例值** | 0, 1, 5, 10 |
| **影响范围** | CompTick每次执行 |
| **变更影响** | 增加 → 战斗体更快耗尽；减少 → 战斗体持久 |

**持续消耗公式**：
```
每Tick消耗：
totalConsumption = GetBaseMaintenance() + sum(mount.sustainCost) + LeakRate
Consumed += totalConsumption
```

**平衡建议**：
- 低消耗组件（常驻）：0-1
- 中等组件（可持续）：2-5
- 高消耗组件（短期使用）：10+

---

### 2.4 activationDelay（激活引导时间）

| 项 | 值 |
|-----|--------|
| **参数名** | activationDelay |
| **含义** | 激活需要的引导Tick数 |
| **类型** | int |
| **有效范围** | >= 1 |
| **示例值** | 1, 5, 60, 300 |
| **影响范围** | 组件从Activating → Active |
| **变更影响** | 增加 → 激活更慢；减少 → 激活更快 |

**时间转换**：
- 1 Tick = 约 16.7 ms
- 60 Tick = 1秒（游戏时间）
- 300 Tick = 5秒

**平衡建议**：
- 瞬间激活：1
- 短引导：5-10
- 中等引导：30-60
- 长引导（检查需求等）：300+

---

### 2.5 requiredOutputPower（输出功率需求）

| 项 | 值 |
|-----|--------|
| **参数名** | requiredOutputPower |
| **含义** | 激活此组件需要的最小OutputPower |
| **类型** | float |
| **有效范围** | >= 0 |
| **示例值** | 0, 30, 50, 100 |
| **影响范围** | TriggerMount.TryActivate()时检查 |
| **变更影响** | 增加 → 激活条件更严格；减少 → 激活容易 |

**检查逻辑**：
```csharp
if (componentDef.requiredOutputPower > comp.OutputPower)
    return false;  // 激活失败
```

**平衡建议**：
- 常规组件：0（无需求）
- 强力能力：30-50
- 终极能力：80-100

## 3. Strategy 参数（应用层定义）

### 3.1 GetBaseMaintenance() 返回值

| 项 | 值 |
|-----|--------|
| **含义** | 每60 Tick的基础消耗 |
| **范围** | >= 0 |
| **代表** | 战斗体维持所需消耗 |
| **示例值** | 1, 2, 5 |

**应用层实现**：
```csharp
public float GetBaseMaintenance()
{
    return 1f;  // 每秒消耗1点Trion维持战斗体
}
```

---

### 3.2 GetRecoveryModifier() 返回值

| 项 | 值 |
|-----|--------|
| **含义** | 恢复速率修正系数 |
| **范围** | >= 0（通常0-2）|
| **代表** | 应用RecoveryRate时的乘数 |
| **示例值** | 0.5, 1.0, 1.5, 2.0 |

**实际恢复量**：
```
实际恢复 = RecoveryRate * GetRecoveryModifier()
示例：RecoveryRate=10, GetRecoveryModifier()=0.5
实际恢复 = 10 * 0.5 = 5
```

---

### 3.3 GetLeakageRate() 返回值

| 项 | 值 |
|-----|--------|
| **含义** | 特定伤口的泄漏速率 |
| **范围** | >= 0 |
| **参数** | VirtualWound wound |
| **代表** | 应用层根据伤口计算泄漏 |

**应用层实现**：
```csharp
public float GetLeakageRate(VirtualWound wound)
{
    // 伤口越严重，泄漏越快
    return wound.severity * 2f;  // 配置：严重度 * 2
}
```

---

### 3.4 GetBaseOutputPower() 返回值

| 项 | 值 |
|-----|--------|
| **含义** | OutputPower的基础值 |
| **范围** | 0-100 |
| **代表** | 战斗体能输出的功率（逻辑值） |
| **应用** | UI显示、组件激活检查 |

**应用层实现**：
```csharp
public float GetBaseOutputPower()
{
    // 根据战斗体状态返回功率
    return 50f;  // 基础50点功率
}
```

## 5. 参数验证检查清单

代码工程师在加载参数时应检查：

```
【CompTrion参数检查】
- [ ] Capacity > 0
- [ ] RecoveryRate >= 0
- [ ] LeakRate >= 0
- [ ] OutputPower >= 0 && <= 100
- [ ] Reserved >= 0 && <= Capacity
- [ ] Consumed >= 0 && <= Capacity

【TriggerComponentDef参数检查】
- [ ] occupyingCapacity >= 0
- [ ] activationCost >= 0
- [ ] sustainCost >= 0
- [ ] activationDelay >= 1
- [ ] requiredOutputPower >= 0 && <= 100

【Strategy返回值检查】
- [ ] GetBaseMaintenance() >= 0
- [ ] GetRecoveryModifier() >= 0
- [ ] GetLeakageRate() >= 0
- [ ] GetBaseOutputPower() >= 0 && <= 100
```

## 6. 参数平衡示例

### 示例1：高耐久战斗体

```
Capacity: 2000
RecoveryRate: 20
GetBaseMaintenance(): 2
```

效果：容量大，恢复快，维持消耗中等

### 示例2：脆弱爆发型

```
Capacity: 500
RecoveryRate: 5
GetBaseMaintenance(): 0.5
```

效果：容量小，恢复慢，维持消耗低（适合短期高输出）

### 示例3：平衡型

```
Capacity: 1000
RecoveryRate: 10
GetBaseMaintenance(): 1
```

效果：中等容量，中等恢复，均衡消耗
