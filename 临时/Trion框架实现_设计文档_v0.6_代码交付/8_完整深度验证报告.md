**推荐实现方式**：在CompTrion中重写PostPreApplyDamage()

```csharp
public override void PostPreApplyDamage(ref DamageInfo dinfo, out bool absorbed)
{
    absorbed = false;  // 默认不拦截

    if (!parent.IsInCombat)
        return;  // 非战斗体，不干预

    // 检查虚拟伤害拦截条件
    float damageAmount = dinfo.Amount;
    if (compTrion.Available >= damageAmount)
    {
        // Trion足够，拦截伤害
        compTrion.Consume(damageAmount);
        absorbed = true;  // ✓ 设置为true
        return;
    }

    // Trion不足，允许伤害继续
    absorbed = false;
}
```

### 🔴 PreApplyDamage补丁的3个关键缺陷

**文件**：`Patch_Pawn_HealthTracker_PreApplyDamage.cs`（L21-50）

#### 缺陷1：无法访问absorbed参数

**当前签名**：
```csharp
public static void Prefix(Pawn ___pawn, ref DamageInfo dinfo)
{
    // 问题：完全无法访问absorbed参数！
}
```

**正确签名**：

```csharp
}
```

**影响**：补丁无法设置absorbed=true，完全无法使用标准的伤害拦截机制

---

#### 缺陷2：返回类型错误

**当前代码**：
```csharp
public static void Prefix(...)  // ← 返回void
{
    // ...
}
```

**问题**：
- Prefix应返回`bool`，表示是否继续执行原方法
- void返回意味着原方法**一定会执行**
- 无法控制虚拟伤害的拦截行为

**正确做法**：
```csharp
public static bool Prefix(...)  // ← 返回bool
{
    // ...
    if (shouldIntercept)
        return false;  // 跳过原方法
    return true;  // 继续执行原方法
}
```

**影响**：原方法（PreApplyDamage）一定会执行，即使补丁已处理了伤害

---

#### 缺陷3：仅修改Amount，未设置absorbed

**当前代码**：
```csharp
compTrion.Consume(damageAmount);
dinfo.SetAmount(0);  // ← 非标准做法
```

**问题分析**：

虽然Amount=0会阻止DamageWorker造成伤害，但：
1. **不遵循RimWorld标准**：应该设置`absorbed=true`
2. **语义模糊**：Amount=0可能被其他mod误解
3. **流程不完整**：absorbed仍为false，可能触发其他伤害逻辑

**RimWorld标准做法**：
```csharp
absorbed = true;  // ← 标准做法
return false;     // 跳过原方法
```

**影响**：虽然伤害被消耗，但伤害流程处理不符合RimWorld约定
