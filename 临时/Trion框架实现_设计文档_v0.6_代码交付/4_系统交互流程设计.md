## 1. 框架初始化流程

### 1.1 应用层启动

```
应用启动 (OnModLoad或ModInitializer)
  ↓
应用层代码
  ├─ 1. 创建/注册所有Strategy实现类
  │
  ├─ 2. 设置 CompTrion.StrategyProvider 委托
  │    CompTrion.StrategyProvider = (comp, thing) =>
  │    {
  │        if (...判断1...) return new Strategy_HumanCombatBody(comp);
  │        if (...判断2...) return new Strategy_TrionSoldier(comp);
  │        ...
  │        return null;
  │    };
  │
  └─ 3. 注册Harmony补丁
       [HarmonyPatch(Pawn_HealthTracker.PreApplyDamage)]
       ...

完成：框架已准备好接收战斗体
```

### 1.2 单位生成（PostSpawnSetup）

```
单位被生成 (Thing.PostSpawnSetup)
  ↓
框架 CompTrion.PostSpawnSetup(respawningAfterLoad)

  ├─【读档分支】
  │  if (respawningAfterLoad)
  │    ├─ 从存档恢复 Capacity、Reserved、Consumed
  │    ├─ 重新注入Strategy（通过StrategyProvider）
  │    └─ 跳到步骤6
  │
  └─【新建分支】
     if (!respawningAfterLoad)
       ├─ 1. 检查StrategyProvider是否已设置
       │    if (CompTrion.StrategyProvider == null) 抛异常
       │
       ├─ 2. 调用StrategyProvider，获取此单位的Strategy
       │    var strategy = CompTrion.StrategyProvider(this, parent);
       │    if (strategy == null) 抛异常("Strategy提供者返回null")
       │
       ├─ 3. 设置Strategy（保存引用）
       │    _strategy = strategy;
       │
       ├─ 4. 【天赋初始化】调用Strategy获取初始天赋
       │    var talent = _strategy.GetInitialTalent(this);
       │    ├─ 如果talent == null
       │    │  └─ 框架不设置Capacity，由应用层管理
       │    │
       │    └─ 如果talent != null
       │       ├─ 框架调用 RecalculateCapacity(talent)
       │       │  └─ 应用层提供的查表函数：TalentGrade → Capacity值
       │       └─ 设置 _capacity = 查表结果
       │
       ├─ 5. 初始化能量四要素
       │    _reserved = 0
       │    _consumed = 0
       │    重算 Available = Capacity - Reserved - Consumed
       │
       ├─ 6. 初始化Mounts（如有）
       │    InitializeMounts();
       │    └─ 根据应用层配置创建Mount对象
       │
       └─ 7. 完成：战斗体系统已就绪

可能结果：
  A. 成功 → 战斗体可用，Capacity已初始化
  B. 异常 → 框架报错，单位无Trion功能
```

## 2. CompTick 调度流程（核心心跳）

### 2.1 完整时序

```
游戏Tick
  ↓
if (TicksGame % 60 != 0) return;  // 仅每60 Tick执行一次
  ↓
CompTrion.CompTick()
  │
  ├─【第1步】累加消耗源
  │  totalConsumption = 0
  │  totalConsumption += strategy.GetBaseMaintenance()
  │  │  ├─ 应用层Strategy在此返回基础消耗值
  │  │  └─ 框架不计算，只使用返回值
  │  │
  │  ├─ for each mount in Mounts:
  │  │  totalConsumption += mount.TickAndGetConsumption()
  │  │  │  ├─ Mount检查是否有Active组件
  │  │  │  ├─ 如果有，返回该组件的sustainCost
  │  │  │  └─ 框架累加
  │  │  │
  │  │  └─ mount.TickActivation()
  │  │     └─ 推进激活计数器（如果有Activating组件）
  │  │
  │  └─ totalConsumption += LeakRate
  │     └─ LeakRate由应用层Strategy维护
  │
  ├─【第2步】执行消耗
  │  Consume(totalConsumption)
  │  ├─ Consumed += totalConsumption
  │  ├─ Clamp到[0, Capacity]
  │  │
  │  └─ 检查是否耗尽
  │     if (Available <= 0)
  │     {
  │         strategy.OnDepleted()
  │         │  └─ 应用层在此：快照恢复、解除战斗体等
  │         return;  // 本次Tick结束
  │     }
  │
  ├─【第3步】执行恢复
  │  if (strategy.ShouldRecover())  // 应用层决定是否恢复
  │  {
  │      float modifier = strategy.GetRecoveryModifier();
  │      Recover(RecoveryRate * modifier);
  │      └─ Consumed -= recoveryAmount
  │  }
  │
  ├─【第4步】策略Tick
  │  strategy.OnTick()
  │  └─ 应用层执行定期逻辑（如AI更新、状态检查）
  │
  └─【第5步】重算输出功率
     RecalculateOutputPower()
     └─ OutputPower = Clamp(strategy.ModifyOutputPower(...), 0, 100)

完成：此Tick调度结束
下一Tick：60 Tick后重复
```

### 2.2 流程图

```
CompTick START
  │
  ├─ Tick % 60 != 0?
  │  ├─ YES → 返回（不做任何事）
  │  └─ NO → 继续
  │
  ├─ GetBaseMaintenance() ──→ Strategy提供
  │  └─ 累加到 totalConsumption
  │
  ├─ for each mount:
  │  ├─ TickActivation()（推进激活计数）
  │  └─ TickAndGetConsumption() ──→ Mount计算
  │     └─ 累加到 totalConsumption
  │
  ├─ LeakRate ──→ Strategy维护
  │  └─ 累加到 totalConsumption
  │
  ├─ Consume(totalConsumption)
  │  └─ Available <= 0?
  │     ├─ YES → OnDepleted() ──→ Strategy处理 → 返回
  │     └─ NO → 继续
  │
  ├─ ShouldRecover()?
  │  ├─ YES → GetRecoveryModifier() ──→ Strategy提供
  │     └─ Recover()
  │  └─ NO → 跳过恢复
  │
  ├─ OnTick() ──→ Strategy执行
  │  └─ 应用层更新逻辑
  │
  └─ RecalculateOutputPower()
     └─ OutputPower更新 → UI显示
```

## 3. 伤害拦截流程

### 3.1 伤害发生时

```
游戏系统：Pawn受伤
  ↓
Pawn.TakeDamage(DamageInfo dinfo)
  ↓
Harmony补丁拦截 [前置补丁]
Patch_InterceptDamage.Prefix()
  ├─ 获取CompTrion
  │  comp = pawn.TryGetComp<CompTrion>()
  │  ├─ 无CompTrion? → 返回true（走原系统）
  │  └─ 有CompTrion? → 继续
  │
  ├─ 调用框架入口
  │  comp.PreApplyDamage(dinfo.Amount, dinfo.HitPart)
  │  │
  │  ├─【框架内】
  │  │  ├─ 询问Strategy
  │  │  │  shouldIntercept = strategy.ShouldInterceptDamage()
  │  │  │  ├─ true → 继续处理虚拟伤害
  │  │  │  └─ false → 跳过虚拟伤害逻辑
  │  │  │
  │  │  └─ 调用应用层
  │  │     strategy.OnDamageTaken(amount, hitPart)
  │  │     └─ 应用层完全控制伤害处理
  │  │        ├─ 可以消耗Trion
  │  │        ├─ 可以创建hediff
  │  │        ├─ 可以拦截原系统伤害
  │  │        └─ 完全自由
  │  │
  │  └─ 框架返回
  │
  └─ 补丁返回true或false
     ├─ true → 伤害继续走原系统（到肉身）
     └─ false → 原系统伤害被拦截（不到肉身）

可能结果：
  A. Strategy拦截 → 虚拟伤害，肉身无伤
  B. Strategy转化 → Trion消耗，原伤害走原系统
  C. Strategy忽略 → 走原系统（肉身受伤）
```

### 3.2 应用层Strategy中的伤害处理

```
Strategy.OnDamageTaken(damageAmount, hitPart)
  │
  ├─ 1. 识别伤害来源（可选）
  │    if (伤害来自特定子弹类型?)
  │    {
  │        // 特殊处理
  │    }
  │
  ├─ 2. 计算虚拟伤害转化
  │    float trionToConsume = damageAmount * conversionRate;
  │    comp.Consume(trionToConsume);
  │
  ├─ 3. 创建虚拟伤口
  │    VirtualWound wound = new VirtualWound()
  │    {
  │        part = hitPart,
  │        severity = damageAmount / 100f
  │    };
  │    wounds.Add(wound);
  │
  ├─ 4. 更新泄漏速率
  │    LeakRate = wounds.Sum(w => w.severity) * 2f;
  │
  ├─ 5. 创建hediff（如有伤势）
  │    var hediff = HediffMaker.MakeHediff(HediffDef_Wound, pawn);
  │    pawn.health.AddHediff(hediff);
  │
  └─ 完成：伤害已处理

框架此后不再干预，交由应用层Strategy维护
```

## 4. 部位丧失处理

### 4.1 流程

```
游戏系统：肢体断裂或截肢
  ↓
Pawn_HealthTracker.AddHediff(Hediff_MissingBodyPart)
  ↓
Harmony补丁检测
Patch_DetectAmputations.Postfix()
  ├─ 获取CompTrion
  │  comp = pawn.TryGetComp<CompTrion>()
  │  ├─ 无? → 返回
  │  └─ 有? → 继续
  │
  ├─ 识别丧失部位
  │  part = hediff.Part;
  │
  ├─ 调用框架入口
  │  comp.OnBodyPartLost(part);
  │  │
  │  ├─【框架内】
  │  │  ├─ 查找绑定此部位的Mount
  │  │  │  for each mount in Mounts:
  │  │  │  {
  │  │  │      if (mount.boundPart == part)
  │  │  │      {
  │  │  │          mount.OnPartDestroyed();
  │  │  │      }
  │  │  │  }
  │  │  │
  │  │  └─ Mount清理
  │  │     └─ 断开所有组件
  │  │        └─ 设置isFunctional = false
  │  │
  │  └─ 调用应用层
  │     strategy.OnBodyPartLost(part)
  │     └─ 应用层：
  │        ├─ 移除相关的VirtualWound
  │        ├─ 更新泄漏速率
  │        ├─ 应用惩罚效果
  │        └─ 可能触发战斗体破裂
  │
  └─ 框架返回

结果：
  A. 断肢挂载的组件失效
  B. 增加泄漏速率
  C. 可能导致战斗体破裂
```

## 5. 组件激活流程

### 5.1 玩家激活请求

```
玩家操作：点击按钮激活组件
  ↓
应用层UI
  ├─ 调用 mount.TryActivate(componentDef)
  │  │
  │  ├─【Mount内部检查】
  │  │  ├─ 1. 检查是否已在激活中
  │  │  │     if (activatingTrigger != null || activeTrigger != null)
  │  │  │     return false;
  │  │  │
  │  │  ├─ 2. 检查是否在装备列表中
  │  │  │     if (!equippedList.Contains(component))
  │  │  │     return false;
  │  │  │
  │  │  ├─ 3. 检查OutputPower是否满足
  │  │  │     if (componentDef.requiredOutputPower > comp.OutputPower)
  │  │  │     return false;
  │  │  │
  │  │  ├─ 4. 尝试支付激活费用
  │  │  │     comp.Consume(componentDef.activationCost);
  │  │  │     if (comp.Available < 0)
  │  │  │     {
  │  │  │         comp.Recover(activationCost);  // 恢复消耗
  │  │  │         return false;
  │  │  │     }
  │  │  │
  │  │  ├─ 5. 进入激活中状态
  │  │  │     activatingTrigger = component;
  │  │  │     activationTicksRemaining = componentDef.activationDelay;
  │  │  │
  │  │  └─ 6. 返回true
  │  │     return true;
  │  │
  │  └─ 应用层收到结果
  │     ├─ true → 显示"引导中..."
  │     └─ false → 显示失败原因
  │
  └─ 等待引导完成

【引导期间】
每Tick：mount.TickActivation()
  ├─ 递减 activationTicksRemaining
  ├─ 如未完成，继续等待
  └─ 完成时：
     ├─ 旧组件关闭（如有）
     │  activeTrigger.OnDeactivated();
     │  activeTrigger.state = Dormant;
     │
     ├─ 新组件激活
     │  activeTrigger = activatingTrigger;
     │  activeTrigger.OnActivated();
     │  activeTrigger.state = Active;
     │
     ├─ 激活标记清空
     │  activatingTrigger = null;
     │
     └─ 应用层UI更新
        显示"组件已激活"
```

## 6. 战斗体破裂流程

### 6.1 触发条件

```
【触发条件（满足其一）】
1. Available <= 0（Trion耗尽）
2. Trion器官被毁（应用层定义）
3. 玩家手动解除（如有Bail Out组件）
```

### 6.2 破裂流程

```
条件触发
  ↓
CompTick检测 Available <= 0
  ├─ 调用 strategy.OnDepleted()
  │  │
  │  ├─【应用层处理】
  │  │  ├─ 1. 取消激活的组件
  │  │  │     for each mount:
  │  │  │     {
  │  │  │         if (mount.activeTrigger != null)
  │  │  │         {
  │  │  │             mount.activeTrigger.OnDeactivated();
  │  │  │             mount.DisconnectAll();
  │  │  │         }
  │  │  │     }
  │  │  │
  │  │  ├─ 2. 应用快照恢复
  │  │  │     快照系统恢复肉身状态
  │  │  │     ├─ 恢复健康状态
  │  │  │     ├─ 恢复装备
  │  │  │     └─ 恢复肉身形态
  │  │  │
  │  │  ├─ 3. 应用生理状态恢复
  │  │  │     生理需求恢复计算
  │  │  │
  │  │  ├─ 4. 创建破裂效果
  │  │  │     播放动画、音效等
  │  │  │
  │  │  └─ 5. 可能创建debuff
  │  │     受伤、疲劳等负面影响
  │  │
  │  └─ 返回
  │
  └─ CompTick结束

结果：
  战斗体消失，肉身恢复
```

## 8. 通信协议（参数和返回值）

### 8.1 Strategy → 框架（数据提供）

| 调用 | 参数 | 返回值 | 含义 |
|------|------|--------|------|
| GetBaseMaintenance() | - | float | 本Tick基础消耗 |
| GetLeakageRate(wound) | VirtualWound | float | 伤口泄漏速率 |
| ShouldRecover() | - | bool | 是否允许恢复 |
| GetRecoveryModifier() | - | float | 恢复修正系数 |
| ShouldInterceptDamage() | - | bool | 是否拦截伤害 |

### 8.2 框架 → Strategy（事件回调）

| 调用 | 参数 | 含义 |
|------|------|------|
| OnInitialize() | - | 战斗体初始化 |
| OnTick() | - | 每Tick执行 |
| OnDamageTaken(amount, part) | 伤害量、部位 | 伤害处理 |
| OnBodyPartLost(part) | 丧失部位 | 部位丧失 |
| OnDepleted() | - | 能量耗尽 |

## 9. 错误处理流程

### 9.1 异常情况

| 异常 | 触发条件 | 框架处理 | 应用层处理 |
|------|---------|---------|-----------|
| StrategyProvider返回null | 无法判断单位类型 | 抛出异常 | 补充判断逻辑 |
| Capacity无效 | Capacity <= 0 | 不检查 | 应用层验证 |
| Strategy方法异常 | Strategy.OnTick()崩溃 | 不try-catch | 应用层处理 |
| Available计算溢出 | Reserved + Consumed > Capacity | Clamp到Capacity | 应用层维持不变量 |
