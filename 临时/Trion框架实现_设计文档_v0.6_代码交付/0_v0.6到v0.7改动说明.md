### 改动1：ILifecycleStrategy 接口新增方法

**文件**：`Core/ILifecycleStrategy.cs`

**改动内容**：在接口中添加新方法，用于初始化时获取天赋

```csharp
/// <summary>
/// 获取单位的初始Trion天赋。
/// 此方法在单位生成时调用（仅新建，读档时不调用）。
///
/// Called when unit is first generated to determine initial talent level.
/// (Not called during save/load)
/// </summary>
/// <param name="comp">CompTrion实例</param>
/// <returns>
/// TalentGrade: 框架会根据此天赋计算Capacity
/// null: 框架跳过容量计算，由应用层自己管理Capacity
```

**何时添加**：在`DestroyReason`枚举定义之前添加

**关键点**：
- 返回类型是 `TalentGrade?`（nullable，可以返回null）
- 此方法仅在 `CompTrion.PostSpawnSetup(!respawningAfterLoad)` 时调用
- 应用层负责天赋的生成逻辑（随机生成 or 从Pawn.modData读取）

### 改动3：修改 CompTrion.PostSpawnSetup() 方法

**文件**：`Components/CompTrion.cs`

**改动内容**：添加天赋初始化逻辑

```csharp
public override void PostSpawnSetup(bool respawningAfterLoad)
{
    // 确保Strategy提供者已设置
    if (StrategyProvider == null)
    {
        Log.Error("CompTrion: StrategyProvider未初始化，无法创建Strategy");
        return;
    }

    // ===== 读档分支 =====
    if (respawningAfterLoad)
    {
        // 从存档恢复时，直接恢复数据，无需重新计算
        // Capacity、Reserved、Consumed等将通过ExposeData自动恢复

        // 重新注入Strategy（Strategy对象无法序列化，需重新创建）
        _strategy = StrategyProvider(this, parent);
        if (_strategy == null)
        {
            Log.Error($"CompTrion: 读档时无法为 {parent.Label} 提供Strategy");
            return;
        }

        return;  // 读档完成，跳出
    }

    // ===== 新建分支 =====

    // 第1步：创建Strategy
    _strategy = StrategyProvider(this, parent);
    if (_strategy == null)
    {
        Log.Error($"CompTrion: StrategyProvider为 {parent.Label} 返回null");
        return;
    }

    // 第2步：【新增】调用Strategy获取初始天赋
    TalentGrade? talent = _strategy.GetInitialTalent(this);

    // 第3步：根据天赋计算Capacity
    if (talent.HasValue)
    {
        // 框架调用RecalculateCapacity来计算Capacity
        // RecalculateCapacity由应用层在Capacity属性Setter中实现
        // 或者框架可提供一个方法委托给应用层

        RecalculateCapacity(talent.Value);
    }
    else
    {
        // talent为null时，框架不干预
        // 应用层应该在其他地方设置Capacity
        Log.Message($"CompTrion: {parent.Label} 的Strategy返回null天赋，由应用层自己设置Capacity");
    }

    // 第4步：初始化能量四要素
    _reserved = 0f;
    _consumed = 0f;
    // Available由属性自动计算

    // 第5步：初始化Mounts（如有）
    InitializeMounts();
}
```

**关键要点**：
- 区分"新建"和"读档"两个分支
- 只在新建时调用 `GetInitialTalent()`
- 读档时Capacity已经存档恢复，不需要重新计算
- 必须重新注入Strategy（因为Strategy对象无法序列化）

### 改动4：新增 RecalculateCapacity() 方法

**文件**：`Components/CompTrion.cs`

**改动内容**：添加方法让框架能根据天赋计算Capacity

```csharp
/// <summary>
/// 根据天赋等级重新计算并设置Capacity。
/// 此方法由应用层的Strategy通过GetInitialTalent返回值触发。
///
/// Recalculate Capacity based on talent grade.
/// Called by PostSpawnSetup after Strategy.GetInitialTalent() returns a valid talent.
/// </summary>
private void RecalculateCapacity(TalentGrade talent)
{
    // 方式A：框架提供查表接口（推荐）
    // 应用层需要提供一个委托：TalentGrade → float
    if (TalentCapacityProvider != null)
    {
        _capacity = TalentCapacityProvider(talent);
    }
    else
    {
        Log.Error("CompTrion: TalentCapacityProvider未设置，无法计算Capacity");
        _capacity = 1000f;  // 默认值
    }

    // 验证Capacity有效性
    if (_capacity <= 0)
    {
        Log.Error($"CompTrion: 天赋{talent}计算出无效Capacity={_capacity}");
        _capacity = 100f;  // 最小值
    }

    ValidateDataConsistency();
}
```

**替代方案B**：如果应用层在Strategy中直接设置Capacity

```csharp
// 应用层Strategy.GetInitialTalent中
public TalentGrade? GetInitialTalent(CompTrion comp)
{
    var talent = GenerateTalent();

    // 应用层直接计算并设置
    comp.Capacity = CalculateCapacityFromTalent(talent);

    return talent;
}
```

**推荐**：使用方式A（委托方式），因为框架能控制计算逻辑，应用层只需提供查表。

## ⚠️ 常见陷阱

### 陷阱1：在读档时重新调用GetInitialTalent

**错误**：
```csharp
if (respawningAfterLoad)
{
    var talent = _strategy.GetInitialTalent(this);  // ❌ 读档时不应调用
    RecalculateCapacity(talent);
}
```

**正确**：读档时Capacity已从存档恢复，无需重新计算

### 陷阱2：忘记Strategy对象需要重新注入

**错误**：
```csharp
if (respawningAfterLoad)
{
    // ❌ Strategy对象读档时为null，需要重新创建
    _strategy.OnTick();  // 会空引用崩溃
}
```

**正确**：
```csharp
if (respawningAfterLoad)
{
    _strategy = StrategyProvider(this, parent);  // ✓ 重新创建
    _strategy.OnTick();  // 现在安全
}
```

### 陷阱3：Capacity设置为0或负数

**错误**：
```csharp
RecalculateCapacity(talent);
// 如果TalentCapacityProvider返回0或-1
// 会导致Available计算异常
```

**正确**：始终验证Capacity > 0

```csharp
if (_capacity <= 0)
{
    Log.Error("无效Capacity，设置默认值");
    _capacity = 1000f;
}
```
