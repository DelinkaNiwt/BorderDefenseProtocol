## 💡 重要概念：做什么 vs 怎么做

### 这套文档的职责（做什么）
```
✓ 功能是什么、有什么步骤
✓ 需要什么数据、什么关系
✓ 参数范围和含义
✓ 与其他模块的交互
✗ 不涉及具体API调用
✗ 不涉及代码组织方式
```

### 你的职责（怎么做）
```
✓ 选择什么RimWorld API实现功能
✓ 代码怎样组织结构
✓ 怎样避免性能问题
✓ 遵循什么编码规范
✗ 不重新定义功能本身
✗ 不改变设计决策
```

## 🔍 如何使用设计文档

### 场景1：实现某个功能时

```
问题："我不知道天赋初始化的流程"
  ↓
查看文档2（功能详细设计）→ 查找"天赋初始化"章节
  ↓
了解：首次扫描→生成天赋→存储→Capacity重算
  ↓
查看文档3（数据结构）→ 查找天赋的数据定义
  ↓
了解：存储位置是Pawn.modData，键名是"Trion_Talent"
  ↓
开始编码
```

### 场景2：不知道某个参数怎么处理

```
问题："Capacity初始值应该是多少？"
  ↓
查看文档5（配置参数定义）
  ↓
找到：初始值1400，天赋等级对应的查表值（S:2000, A:1800等）
  ↓
继续编码
```

### 场景3：调试模块间通信时

```
问题："战斗体激活时，Reserved怎么计算的？"
  ↓
查看文档4（系统交互流程）→ 查找生成战斗体的流程图
  ↓
了解：Reserved = Σ(所有装备组件的消耗值)
  ↓
调试数据流转
```

### 场景4：对设计决策有疑问

```
问题："为什么天赋要延迟到检测仪时才初始化？"
  ↓
查看文档6（技术方案选择）→ 查找"天赋初始化时机"决策
  ↓
了解：避免全局Harmony补丁，UI交互自然，符合世界观
  ↓
理解设计背景，心平气和地实现
```

## 🆘 遇到问题时

### 编译报错

```
问题：找不到ProjectTrion程序集
  ↓
检查项目文件：.csproj的<HintPath>是否指向正确的dll路径
  ↓
参考文档：无（这是IDE问题，检查项目配置）
```

### 功能实现卡住

```
问题：不知道GetInitialTalent()应该返回什么
  ↓
查看文档2："天赋初始化"功能说明
  ↓
查看文档3："天赋属性"数据定义
  ↓
理解：首次返回生成的TalentGrade，后续返回已有的
```

### 数据不一致

```
问题：Capacity值不对，Available计算错误
  ↓
查看文档3："能量四要素"的数据关系和计算公式
  ↓
检查你的代码是否遵循公式：Available = Capacity - Reserved - Consumed
```
