# Trion_MVP 配置参数定义 v0.3

## 框架层参数（ProjectTrion_Framework）

### CompProperties_Trion

| 参数 | 类型 | 默认值 | 范围 | 说明 | 影响 |
|------|------|--------|------|------|------|
| capacity | float | 1400 | >0 | Trion初始容量（会被天赋覆盖） | Capacity上限 |
| enableSnapshot | bool | true | true/false | 是否保存快照 | 战斗体摧毁时是否恢复 |
| enableBailOut | bool | true | true/false | 是否启用紧急脱离 | Available≤0时的行为 |
| strategyClassName | string | "" | 有效类名 | Strategy类的完整类名 | 生命周期回调 |

**验证规则**：
- capacity 必须 > 0（否则日志报错，使用最小值100）
- strategyClassName 必须存在（否则日志报错，CompTrion初始化失败）

## 应用层参数

### CompTrion.TalentCapacityProvider（委托）

**设置时机**：Mod启动时（在任何Pawn生成前）

**签名**：`Func<TalentGrade, float>`

**MVP实现**：
```csharp
CompTrion.TalentCapacityProvider = (talent) => talent switch
{
    TalentGrade.S => 2000f,
    TalentGrade.A => 1800f,
    TalentGrade.B => 1600f,
    TalentGrade.C => 1400f,
    TalentGrade.D => 1200f,
    TalentGrade.E => 1000f,
    _ => 1400f
};
```

**参数映射表**：

| 天赋 | Capacity值 | 相对优势 | 应用场景 |
|------|-----------|---------|---------|
| S | 2000 | +43% | 精英战士 |
| A | 1800 | +29% | 高级战士 |
| B | 1600 | +14% | 中高级战士 |
| C | 1400 | 0% | 平均水平 |
| D | 1200 | -14% | 普通战士 |
| E | 1000 | -29% | 新兵 |

**约束**：
- 每个天赋的Capacity值必须 > 0
- 建议按天赋等级递增（S > A > B > C > D > E）

## 消耗系统参数

### 基础维持消耗

| 参数 | 来源 | 默认值 | 范围 | 说明 |
|------|------|--------|------|------|
| baseMaintenance | Strategy.GetBaseMaintenance() | 5.0 | >0 | 每60Tick的基础维持消耗 |
| consumptionTickInterval | 框架固定 | 60 | 固定 | 消耗计算的Tick间隔 |

**计算**：
- 每60Tick消耗一次
- 每次消耗 = baseMaintenance + mountConsumption + leakRate

### 组件消耗参数

| 组件 | TrionCost | 激活消耗方式 | 备注 |
|------|-----------|-----------|------|
| 弧月 | 10 | 持续 | 激活后持续消耗 |
| 护盾 | 15 | 持续 | 激活后持续消耗 |
| 炸裂弹 | 10 | 持续 | 激活后持续消耗 |
| 变色龙 | 10 | 持续 | 激活后持续消耗 |
| 脱离 | 400 | 一次性 | 自动触发时消耗 |

**约束**：
- 消耗值 >= 1，建议 <= 500
- Reserved = Σ(所有装备组件的消耗值)

### 伤口泄漏参数

| 参数 | 值 | 范围 | 说明 | 约束 |
|------|-----|------|------|------|
| baseLeak | 0.5 | >=0 | 伤口基础泄漏 | - |
| severityLeakFactor | 0.1 | >=0 | 伤口严重程度系数 | - |
| vitalPartMultiplier | 2.0 | >1 | 关键部位加倍系数 | - |
| leakCacheDuration | 60 | >0 | 泄漏缓存有效期(Tick) | 固定60 |

**计算公式**：
```
leak = baseLeak + injury.Severity * severityLeakFactor
if (关键部位) leak *= vitalPartMultiplier
```

**示例**：
- 小伤口(severity=0.2)：leak = 0.5 + 0.2*0.1 = 0.52
- 重伤口(severity=0.8)：leak = 0.5 + 0.8*0.1 = 0.58
- 关键部位重伤：leak = 0.58 * 2 = 1.16

## 快照相关参数

| 参数 | 值 | 说明 |
|------|-----|------|
| enableSnapshot | true | 是否保存快照 |
| snapshotRestoreMode | "Full" | 恢复模式（"Full"=完全恢复） |
| inheritHediff | false | 战斗体伤口是否继承 |
| inheritPsychologic | true | 心理状态是否继承 |

**恢复规则**：
- 生命值：完全恢复
- 部位：完全恢复
- 装备：完全恢复
- 伤口：不继承新伤口（只保留战前旧伤口）
- 心理状态：继承（包括Debuff）

## Bail Out参数

| 参数 | 值 | 说明 | 约束 |
|------|-----|------|------|
| enableBailOut | true | 是否启用自动脱离 | 必须true |
| bailOutTriggerThreshold | 0 | Available≤此值时触发 | 固定0 |
| bailOutTargetType | "Position" | 脱离目标类型 | 仅支持Position |

**触发条件**（任一满足）：
- Available <= 0
- Trion供给器官被摧毁
- Strategy.CanBailOut() = false 时脱离失败

## 性能参数

| 参数 | 目标值 | 单位 | 说明 |
|------|--------|------|------|
| consumptionCalcTime | <5 | ms | 消耗计算耗时（目标） |
| leakCalcCacheDuration | 60 | Tick | 泄漏计算缓存周期 |
| snapshotRestoreTime | <10 | ms | 快照恢复耗时（目标） |

**优化策略**：
- 使用leakRate缓存，避免每Tick都计算
- Reserved使用加法，不遍历List
- Snapshot只在激活时捕获一次

## 数据验证范围

### Capacity
- 最小值：100
- 最大值：10000（建议）
- 非零检查：必须 > 0

### Consumed
- 最小值：0
- 最大值：Capacity（不能超过）
- 非负检查：必须 >= 0

### Reserved
- 最小值：0
- 最大值：Capacity（不能超过）
- 范围检查：0 <= Reserved <= Capacity

### Available
- 计算：Capacity - Reserved - Consumed
- 约束：自动钳制到 >= 0
- 触发条件：<= 0 时激活BailOut

## 参数修改影响分析

| 修改参数 | 影响范围 | 修改限制 | 恢复方式 |
|---------|---------|---------|---------|
| capacity初始值 | 新建Pawn | 低风险，修改即生效 | 重新生成Pawn |
| TalentCapacityProvider | 天赋等级计算 | 中风险，需读档后生效 | 新建Pawn或重读档 |
| baseMaintenance | 每帧消耗计算 | 低风险，实时生效 | 立即生效，持续影响 |
| 组件消耗值 | 组件被装备时 | 低风险，新装备生效 | 移除重新装备 |
| 泄漏参数 | 伤口泄漏计算 | 低风险，实时生效 | 清除缓存后生效 |
