# Trion_MVP 系统交互流程设计 v0.3

## 整体数据流

```
[玩家输入]
    ↓
[UI交互]
    ├─ 植入手术 → Hediff.PostAdd → CompTrion初始化
    ├─ 检测仪扫描 → ScanPawn → 天赋生成 → Capacity重算
    ├─ 装备组件 → TriggerMount.AddComponent → Reserved更新
    ├─ 激活战斗体 → GenerateCombatBody → Snapshot保存
    └─ 点击解除 → DestroyCombatBody → 快照恢复
    ↓
[Tick系统]
    ├─ 每Tick: CompTick() → 检查60Tick间隔
    ├─ 每60Tick: TickConsumption()
    │   ├─ baseMaintenance = Strategy.GetBaseMaintenance()
    │   ├─ mountConsumption = Σ激活组件
    │   ├─ leakRate = GetLeakRate()
    │   ├─ Consumed += total
    │   └─ if Available <= 0 → TriggerBailOut()
    └─ 伤口变化: InvalidateLeakCache()
    ↓
[数据持久化]
    ├─ 存档: PostExposeData() 保存状态
    └─ 读档: PostSpawnSetup(respawningAfterLoad) 恢复状态
```

## 植入→初始化 流程

```
玩家执行"植入Trion腺体"手术
        ↓
SurgeryDef 执行成功
        ↓
Hediff_TrionGland.PostAdd(DamageInfo?)
        ├─ 检查Pawn.GetComp<CompTrion>()
        ├─ 如无则创建CompProperties_Trion
        ├─ 创建CompTrion实例
        ├─ pawn.AllComps.Add(comp)
        ├─ comp.PostSpawnSetup(false) 【手动触发】
        │  ├─ 创建Strategy（反射）
        │  ├─ talent = Strategy.GetInitialTalent(this)
        │  ├─ if talent.HasValue
        │  │  └─ RecalculateCapacity(talent)
        │  └─ 初始化_reserved=0, _consumed=0, _mounts=[], _snapshot=null
        └─ Log: "植入成功"
        ↓
CompTrion已初始化，Capacity=1400，等待天赋检测
```

## 检测→天赋初始化 流程

```
玩家与检测仪交互
        ↓
Building_TrionDetector.ScanPawn(pawn)
        ├─ 检查pawn.GetComp<CompTrion>()
        ├─ 如无则提示"无腺体"返回
        ├─ 调用TrionTalentManager.GetTalent(pawn)
        │  ├─ 读Pawn.modData["Trion_Talent"]
        │  └─ 解析为TalentGrade? 返回
        │
        ├─ if talent == null
        │  ├─ talent = GenerateRandomTalent(pawn)
        │  │  └─ 从[S,A,B,C,D,E]中随机选一个
        │  ├─ TrionTalentManager.SetTalent(pawn, talent)
        │  │  └─ 写Pawn.modData["Trion_Talent"] = talent.ToString()
        │  ├─ comp.RecalculateCapacityFromTalent(talent)
        │  │  └─ _capacity = TalentCapacityProvider(talent)
        │  │  └─ 验证_capacity有效
        │  └─ Log: "首次扫描：天赋{talent}"
        │
        └─ ShowDetectionResult(pawn, talent)
           └─ UI显示：名字、天赋、新Capacity
           ↓
CompTrion.Capacity已重算，可进行战斗体激活
```

## 装备组件 流程

```
玩家选择组件和触发器
        ↓
Building_TriggerConfigBench.EquipComponent(comp, trigger)
        ├─ 检查trigger.GetComp<TriggerMount>()
        ├─ 调用mount.TryAddComponent(comp)
        │  ├─ 检查1：HasSlotSpace(comp.SlotSize)
        │  ├─ 检查2：CanCoexistWith(comp)
        │  ├─ 检查3：!compTrion.IsInCombat
        │  ├─ if 通过所有检查
        │  │  ├─ _equippedComponents.Add(comp)
        │  │  └─ 如战斗体已激活则更新Reserved
        │  └─ 返回 true/false
        │
        ├─ if 成功
        │  ├─ comp.Destroy(DestroyMode.Vanish) 移除物品
        │  ├─ 更新UI组件列表
        │  └─ Log: "装备{comp.ComponentName}"
        │
        └─ else
           └─ Log.Warning: "装备失败：{原因}"
           ↓
组件已添加到TriggerMount，Reserved准备好
```

## 生成战斗体 流程

```
玩家点击"激活战斗体"
        ↓
CompTrion.GenerateCombatBody()
        ├─ 检查_isInCombat(如已激活则警告返回)
        ├─ 创建Snapshot
        │  ├─ _snapshot = new CombatBodySnapshot()
        │  ├─ _snapshot.CaptureFromPawn(parent as Pawn)
        │  │  └─ 保存：生命值、部位、伤口、装备
        │  └─ 【此后Pawn受伤也不会改变快照】
        ├─ _isInCombat = true
        ├─ 【Reserved自动就位，因为组件已装备】
        ├─ Strategy.OnCombatBodyGenerated(this)
        └─ Log: "{Pawn}战斗体已生成，可用:{Available}"
           ↓
战斗体激活，进入60Tick消耗循环
```

## 消耗计算 循环（每60Tick）

```
CompTick() 被调用
        ↓
if !_isInCombat 返回
        ↓
if !IsHashIntervalTick(60) 返回
        ↓
TickConsumption()
        ├─ baseMaintenance = _strategy.GetBaseMaintenance()   [5f]
        ├─ mountConsumption = 0
        ├─ foreach mount in _mounts
        │  └─ foreach component in mount.EquippedComponents
        │     └─ if component.IsActive
        │        └─ mountConsumption += component.TrionCost
        │
        ├─ leakRate = GetLeakRate()  [有缓存]
        │  └─ if 缓存过期(60Tick)
        │     └─ leakRate = CalculateLeakRate()
        │        └─ foreach hediff in pawn.health.hediffs
        │           └─ leak += 0.5 + severity*0.1
        │           └─ if 关键部位 leak *= 2
        │
        ├─ totalConsumption = base + mount + leak
        ├─ Consume(totalConsumption)
        │  └─ _consumed += amount
        │
        ├─ if Available <= 0
        │  └─ TriggerBailOut()
        │
        └─ Strategy.OnTick(this)
           ↓
消耗循环完成，Available递减，等待下一个60Tick
```

## Bail Out 脱离 流程

```
Available <= 0 或 供给器官被摧毁
        ↓
TriggerBailOut()
        ├─ 检查_isInCombat
        ├─ 检查enableBailOut配置
        ├─ 调用Strategy.CanBailOut(this)
        │  └─ if false → BailOutFailed 路线
        │
        ├─ 调用Strategy.GetBailOutTarget(this)
        │  ├─ 返回 IntVec3 目标位置
        │  └─ if Invalid → BailOutFailed 路线
        │
        ├─ if BailOutFailed
        │  └─ DestroyCombatBody(BailOutFailed)
        │     └─ 战斗体直接破裂
        │
        └─ if 目标有效
           ├─ pawn.Position = target
           ├─ pawn.Notify_Teleported()
           ├─ DestroyCombatBody(BailOutSuccess)
           └─ Log: "{Pawn}Bail Out到{target}"
              ↓
脱离完成，快照恢复，战斗体摧毁
```

## 触发器装备流程

```
玩家在配置台选择触发器和组件
        ↓
Building_TriggerConfigBench.EquipComponent(trigger, component)
        ├─ 获取trigger的TriggerMount列表
        ├─ 遍历3个挂载点（左/右/特殊）
        ├─ 调用mount.TryAddComponent(component)
        │  ├─ 检查HasSlotSpace(component.SlotSize)
        │  ├─ 检查CanCoexistWith(component)
        │  └─ 检查compTrion.IsInCombat
        ├─ if 成功
        │  ├─ 移除ground上的component物品
        │  ├─ component.Destroy(DestroyMode.Vanish)
        │  ├─ 更新Reserved值
        │  └─ UI刷新Mounts列表显示
        └─ if 失败
           └─ Log.Warning & 提示玩家原因
           ↓
触发器已装备组件，Gizmo按钮可用
```

## 传送锚查询流程

```
TriggerComponent_BailOut 激活或 Available ≤ 0 时触发Bail Out
        ↓
CompTrion.TriggerBailOut()
        ├─ 检查enableBailOut配置
        ├─ 调用Strategy.CanBailOut(this)
        └─ if true 继续，else 破裂失败
           ↓
调用Strategy.GetBailOutTarget(this)
        ├─ 扫描当前地图所有Building_BailOutAnchor
        ├─ 过滤active=true且有效的锚点
        ├─ 计算每个锚点到pawn的距离
        ├─ 按距离排序，选择最近的
        ├─ if 找到有效锚点
        │  └─ 返回 anchor.Position
        └─ if 无有效锚点
           └─ 返回 IntVec3.Invalid
           ↓
if 目标位置有效
├─ pawn.Position = target
├─ pawn.Notify_Teleported()
└─ 继续Bail Out流程（见下）
```

## 战斗体破裂与Bail Out流程（新方案）

```
Available ≤ 0 或玩家手动Bail Out 或供给器官被摧毁
        ↓
TriggerBailOut()
        ├─ 检查_isInCombat
        ├─ 检查enableBailOut配置
        │
        ├─ 【传送目标查询】
        ├─ 优先级1：调用Strategy.GetBailOutTarget(this)
        │  └─ 扫描Building_BailOutAnchor列表
        │  └─ 返回最近有效锚点位置
        │
        ├─ 优先级2：如无有效锚点
        │  └─ 调用RCellFinder.RandomSpotJustOutsideColony(pawn)
        │  └─ 返回基地外围随机位置
        │
        ├─ 执行传送
        │  ├─ pawn.Position = target
        │  └─ pawn.Notify_Teleported()
        │
        ├─ 破裂处理：DestroyCombatBody(DestroyReason.Crashed)
        │  ├─ 快照恢复（如启用）
        │  ├─ Reserved → Consumed（能量消耗转化）
        │  ├─ 施加心理debuff
        │  └─ _isInCombat = false
        │
        └─ Log: "{Pawn}战斗体破裂，传送到{target}，Reserved→Consumed+debuff"
           ↓
破裂完成，Pawn传送到目标位置，战斗体摧毁，能量永久消耗
```

## 安全解除战斗体流程（新方案）

```
玩家点击"解除战斗体"按钮 + Available > 0
        ↓
DestroyCombatBody(DestroyReason.Manual)
        ├─ 检查Available > 0（如不符合则提示Bail Out）
        ├─ 快照恢复（如启用）
        │  └─ Pawn状态完全回滚
        ├─ Reserved 完全返还（不转化为Consumed）
        ├─ 无任何心理debuff施加
        ├─ 标记_isInCombat = false
        └─ Log: "{Pawn}安全解除战斗体，Reserved返还，无debuff"
           ↓
解除完成，Pawn状态恢复，能量返还，无任何代价
```

## 快照恢复 流程

```
DestroyCombatBody(reason: DestroyReason)
        ├─ 检查_isInCombat(如未激活则警告返回)
        │
        ├─ if enableSnapshot && _snapshot != null
        │  ├─ _snapshot.RestoreToPawn(parent as Pawn)
        │  │  ├─ 恢复生命值（总和+各部位）
        │  │  ├─ 恢复伤口列表
        │  │  ├─ 恢复装备列表
        │  │  └─ 【新产生的伤口不被恢复，直接丢弃】
        │  └─ Log: "{Pawn}状态已回滚"
        │
        ├─ 处理Reserved和Consumed
        │  ├─ if BailOutSuccess || TrionDepleted
        │  │  └─ Reserved → Consumed (被动解除)
        │  └─ else
        │     └─ Reserved 返还（主动解除）
        │
        ├─ _isInCombat = false
        ├─ _leakCacheTickExpire = 0  清除泄漏缓存
        ├─ Strategy.OnCombatBodyDestroyed(this, reason)
        └─ Log: "{Pawn}战斗体已摧毁，原因:{reason}"
           ↓
战斗体完全摧毁，Pawn可进行下一轮操作
```

## 存档/读档 流程

```
【存档】
玩家保存游戏
        ↓
CompTrion.PostExposeData()
        ├─ Scribe_Values.Look(ref _capacity, "capacity")
        ├─ Scribe_Values.Look(ref _reserved, "reserved")
        ├─ Scribe_Values.Look(ref _consumed, "consumed")
        ├─ Scribe_Values.Look(ref _isInCombat, "isInCombat")
        ├─ Scribe_Deep.Look(ref _snapshot, "snapshot")
        └─ Scribe_Collections.Look(ref _mounts, "mounts")
           ↓
所有数据序列化到存档文件

【读档】
玩家读取游戏
        ↓
PostExposeData() 反序列化
        ├─ _capacity, _reserved, _consumed 恢复
        ├─ _isInCombat 恢复
        ├─ _snapshot 恢复
        └─ _mounts 恢复
           ↓
PostSpawnSetup(respawningAfterLoad: true)
        ├─ 跳过新建分支（不重新生成天赋）
        ├─ 执行读档分支：InitializeStrategy()
        │  └─ 通过反射重新创建Strategy实例
        └─ 返回
           ↓
CompTrion完全恢复，可继续游戏
```

## 模块依赖关系

```
ProjectTrion_Framework
    ├─ CompTrion (框架核心)
    ├─ ILifecycleStrategy (策略接口)
    └─ TalentGrade (天赋枚举)

Trion_MVP
    ├─ DefaultTrionStrategy (ILifecycleStrategy实现)
    ├─ TrionTalentManager (天赋管理)
    ├─ Hediff_TrionGland (植入物)
    ├─ Building_TrionDetector (检测仪)
    ├─ TrionComponent (组件基类)
    ├─ TriggerMount (扩展)
    └─ UI组件 (可选)
```
