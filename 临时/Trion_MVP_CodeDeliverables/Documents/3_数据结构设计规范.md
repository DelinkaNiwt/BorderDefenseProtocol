# Trion_MVP 数据结构设计规范 v0.3

## CompTrion 组件数据

| 属性 | 类型 | 初始值 | 说明 | 可序列化 |
|------|------|--------|------|---------|
| _capacity | float | 1400 | Trion总容量 | ✓ |
| _reserved | float | 0 | 组件占用量 | ✓ |
| _consumed | float | 0 | 已消耗量 | ✓ |
| _isInCombat | bool | false | 战斗体激活状态 | ✓ |
| _snapshot | CombatBodySnapshot | null | 快照数据 | ✓ |
| _mounts | List<TriggerMount> | [] | 组件挂载点 | ✓ |
| _strategy | ILifecycleStrategy | null | 生命周期策略 | ✗ |
| _cachedLeakRate | float | 0 | 缓存的泄漏速率 | ✗ |
| _leakCacheTickExpire | int | 0 | 缓存过期时刻 | ✗ |

**公开属性**（只读）：
- `Available` → Capacity - Reserved - Consumed
- `IsInCombat` → _isInCombat
- `Snapshot` → _snapshot
- `Strategy` → _strategy
- `Mounts` → _mounts

## 能量四要素的数据关系

```
Capacity（容量）
    ├─ 决定因素：天赋等级
    ├─ 初始值：1400（临时，待天赋时重算）
    ├─ 计算：TalentCapacityProvider(talent) 委托返回
    └─ 不变点：战斗体激活后不再改变

Reserved（占用）
    ├─ 决定因素：装备的组件
    ├─ 计算：Σ(component.TrionCost)
    ├─ 更新时机：装备/移除组件时，或战斗体生成时
    └─ 约束：Reserved <= Capacity

Consumed（消耗）
    ├─ 决定因素：维持消耗、组件消耗、泄漏
    ├─ 更新频率：每60Tick累加一次
    ├─ 计算：+=  (baseMaintenance + mountConsumption + leakRate)
    └─ 不可逆：只能通过Recover()减少

Available（可用）
    ├─ 公式：Capacity - Reserved - Consumed
    ├─ 约束：Always >= 0（自动钳制）
    ├─ 触发条件：<= 0 时触发 BailOut
    └─ 只读：不能直接赋值
```

## 触发器物品数据

### TrionTrigger 物品属性（Apparel子类）

| 属性 | 类型 | 说明 | Apparel来源 |
|------|------|------|-----------|
| defName | string | TrionTrigger_Standard | 必须 |
| label | string | 标准触发器 | 必须 |
| bodyPartGroups | list | [Torso] | Apparel系统 |
| layers | list | [OnSkin] | Apparel系统 |
| dropsOnDeath | bool | true | Apparel系统 |
| mounts | TriggerMount[] | 组件挂载点列表（3个：左手、右手、特殊） | Trion自定义 |
| maxSlots | int | 9 | Trion自定义 |

**方法**：
- `Wear(pawn)` → void | 穿戴（继承自Apparel）
- `Unfold(pawn)` → void | 脱下（继承自Apparel）
- `GetOwnerCompTrion()` → CompTrion | 获取穿着者的CompTrion
- `GetMountCount()` → int | 获取当前挂载点数
- `CanEquipComponent(comp)` → bool | 检查是否可装备组件

**重要**：
- TrionTrigger 继承自 Apparel，NOT ThingWithComps
- 穿戴/脱下由RimWorld穿戴系统自动处理
- 存档/读档由穿戴系统自动处理（无需手动序列化mounts）

## 战斗体快照数据

### CombatBodySnapshot 结构

```csharp
public class CombatBodySnapshot
{
    // Pawn状态快照
    public float BodyHealthSum;        // 身体总生命值
    public Dictionary<BodyPartRecord, float> PartHealthDict;  // 各部位生命值

    // 伤口快照
    public List<Hediff_Injury> InjuryList;   // 伤口列表备份

    // 缺失部位快照
    public List<BodyPartRecord> MissingParts;  // 缺失部位列表

    // 装备快照
    public List<Apparel> ApparelList; // 穿着装备备份

    // 背包物品快照
    public List<ThingWithComps> InventoryList; // 背包物品备份

    // 捕获和恢复方法
    public void CaptureFromPawn(Pawn pawn) { /* ... */ }
    public void RestoreToPawn(Pawn pawn) { /* ... */ }
}
```

**捕获内容**：
- 身体总生命值和各部位生命值
- 所有伤口列表（Hediff_Injury）
- 缺失部位列表
- 穿着装备列表
- 背包物品列表

**恢复规则**：
- 生命值和部位完全还原
- 伤口和缺失部位完全还原
- 装备和背包完全还原
- **例外**：战斗体激活后产生的新伤口不被恢复

## 传送锚数据

### Building_BailOutAnchor 数据结构

| 属性 | 类型 | 说明 |
|------|------|------|
| defName | string | Building_BailOutAnchor |
| label | string | 传送锚 |
| Position | IntVec3 | 建筑位置 |

| anchorID | int | 唯一ID（用于保存/读档） |

**方法**：
- `IsValidTarget()` → bool | 检查是否是有效的传送目标

**查询规则**：
- 扫描整个地图的所有Building_BailOutAnchor
- 按距离排序，返回最近的有效锚点
- 如无有效锚点，BailOut失败

## 序列化规范

### PostExposeData() 保存内容

```csharp
// 能量数据
Scribe_Values.Look(ref _capacity, "capacity", 1400f);
Scribe_Values.Look(ref _reserved, "reserved", 0f);
Scribe_Values.Look(ref _consumed, "consumed", 0f);

// 状态数据
Scribe_Values.Look(ref _isInCombat, "isInCombat", false);

// 快照数据
Scribe_Deep.Look(ref _snapshot, "snapshot");

// 组件列表
Scribe_Collections.Look(ref _mounts, "mounts", LookMode.Deep);
```

### 不保存内容（读档时重建）
- `_strategy`：通过InitializeStrategy()重新创建
- `_cachedLeakRate`：重新计算
- `_leakCacheTickExpire`：重置为0

## 数据一致性约束

### 检查规则

```csharp
ValidateDataConsistency()
{
    // 规则1：Capacity > 0
    if (_capacity <= 0) 报错，设置为1

    // 规则2：Reserved <= Capacity
    if (_reserved > _capacity) 报错，钳制到_capacity

    // 规则3：Consumed >= 0
    if (_consumed < 0) 报错，设置为0

    // 规则4：Reserved + Consumed <= Capacity
    if (_reserved + _consumed > _capacity)
        报错，减少_consumed
}
```

### 何时触发检查
- 每次修改Capacity、Reserved、Consumed时
- 每次读档后
- 每次生成/摧毁战斗体时

## 组件槽位管理

### 槽位限制

```
触发器总槽位：9
├─ 左手：4个槽位
├─ 右手：4个槽位
└─ 特殊：1个槽位
```

### 装备规则

```
同时激活：最多2个
├─ 左手激活：1个
├─ 右手激活：1个
└─ 特殊自动：不占激活数

热切换：
├─ 战斗体未激活：可随意装备/移除
└─ 战斗体激活：组件锁定，只能激活/关闭
```
