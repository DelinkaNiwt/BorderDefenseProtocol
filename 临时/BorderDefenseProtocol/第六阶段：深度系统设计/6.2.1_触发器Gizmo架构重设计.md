# 6.2.1 触发器Gizmo架构重设计

## 前置文档

| 文档 | 关系 |
|------|------|
| 6.2 触发器模块详细设计 v4.2 | 父文档，本文档是其Gizmo架构的补充设计 |

## 1. 问题描述

### 1.1 根因

RimWorld引擎中，`VerbTracker.GetVerbsCommands()` 为每个 `hasStandardCommand=true` 的Verb生成独立的 `Command_VerbTarget`。但UI渲染阶段有Gizmo合并机制：

```csharp
// Command_VerbTarget.GroupsWith() — RimWorld引擎源码
public override bool GroupsWith(Gizmo other)
{
    if (!(other is Command_VerbTarget cmd)) return false;
    if (ownerThing != null && cmd.ownerThing != null)
        return ownerThing.def == cmd.ownerThing.def;  // 同一ThingDef就合并
    return base.GroupsWith(other);
}
```

所有从同一触发体生成的 `Command_VerbTarget`，其 `ownerThing.def` 完全相同，因此无论有多少个Verb，UI层全部合并成1个Gizmo。

### 1.2 影响

- 近战A+近战B：只显示1个Gizmo（应显示3个：左手/右手/双手）
- 远程A+远程B：只显示1个Gizmo（应显示3个）
- 近战+远程：只显示1个Gizmo（应显示2个），且行为矛盾（显示远程范围但执行近战）

## 2. 设计方案

### 2.1 核心思路

```
标准路径（VerbTracker → GetVerbsCommands）：只负责占位Tool的gizmo
自定义路径（CompGetEquippedGizmosExtra）：负责所有芯片攻击的gizmo
```

三个关键改动：
1. **IVerbOwner改造**：始终返回占位Verb + 芯片Verb，芯片Verb设 `hasStandardCommand=false`
2. **自定义Command类**：`Command_BDPChipAttack` 继承 `Command_VerbTarget`，重写 `GroupsWith()`
3. **Verb引用缓存**：`RebuildVerbs()` 后存储芯片Verb引用，`CompGetEquippedGizmosExtra()` 直接读取

### 2.2 IVerbOwner改造

**改造前**（互斥返回）：
```
有芯片Verb → 返回芯片Verb列表（不含占位）
无芯片Verb → 返回parent.def.Verbs（占位）
```

**改造后**（始终包含占位）：
```csharp
List<VerbProperties> IVerbOwner.VerbProperties
{
    var chipVerbs = DualVerbCompositor.ComposeVerbs(...);
    if (chipVerbs != null)
    {
        // 占位Verb（hasStandardCommand=true）+ 芯片Verb（hasStandardCommand=false）
        var result = new List<VerbProperties>(parent.def.Verbs);
        result.AddRange(chipVerbs);
        return result;
    }
    return parent.def.Verbs;  // 无芯片，只有占位
}

List<Tool> IVerbOwner.Tools
{
    // 始终返回占位Tools，移除v4.2的近战抑制逻辑
    return parent.def.tools;
}
```

**效果**：
- VerbTracker始终包含占位Verb + 芯片Verb
- 标准路径只为占位Verb生成gizmo（占位近战gizmo始终可见）
- AI的 `TryGetAttackVerb()` 仍能找到芯片Verb（`hasStandardCommand` 不影响AI选择）

### 2.3 DualVerbCompositor改动

`ComposeVerbs()` 返回的所有VerbProperties统一设置 `hasStandardCommand = false`。

其余组合逻辑（近战+近战、远程+远程、近战+远程、同芯片去重）不变。

### 2.4 Command_BDPChipAttack

```csharp
// 新增文件：Command_BDPChipAttack.cs
public class Command_BDPChipAttack : Command_VerbTarget
{
    // 攻击类别标识
    public string attackId;  // "left" / "right" / "dual" / "combo_{abilityDefName}"

    public override bool GroupsWith(Gizmo other)
    {
        if (other is Command_BDPChipAttack cmd)
        {
            // 同一触发体：相同attackId合并（同种芯片去重）
            if (verb?.EquipmentSource == cmd.verb?.EquipmentSource)
                return attackId == cmd.attackId;
            // 不同触发体（多选pawn）：暂时不合并
            // TODO: 未来实现统一"触发器攻击"合并
            return false;
        }
        // 不与标准Command_VerbTarget合并
        return false;
    }
}
```

**attackId四种类别**：

| attackId | 含义 | 来源 |
|----------|------|------|
| `"left"` | 左手触发 | 左手芯片独立攻击Verb |
| `"right"` | 右手触发 | 右手芯片独立攻击Verb |
| `"dual"` | 双手触发 | DualMelee/DualRanged合成Verb |
| `"combo_{name}"` | 组合技能 | ComboAbilityDef产生的Ability（预留） |

### 2.5 Verb引用缓存

CompTriggerBody新增字段：
```csharp
private Verb leftHandAttackVerb;    // 左手触发对应的Verb实例
private Verb rightHandAttackVerb;   // 右手触发对应的Verb实例
private Verb dualAttackVerb;        // 双手触发对应的Verb实例
// 不序列化——RebuildVerbs()后重建
```

`RebuildVerbs()` 执行 `InitVerbsFromZero()` 后，遍历 `AllVerbs`，按 verbClass 类型 + label侧别标签匹配：
```
Verb_BDPDualMelee + label="BDP_LeftHand"  → leftHandAttackVerb
Verb_BDPDualMelee + label="BDP_RightHand" → rightHandAttackVerb
Verb_BDPDualMelee + label无侧别标签       → dualAttackVerb
Verb_BDPShoot + label="BDP_LeftHand"      → leftHandAttackVerb
Verb_BDPShoot + label="BDP_RightHand"     → rightHandAttackVerb
Verb_BDPDualRanged                        → dualAttackVerb
其他标准Verb（占位Tool产生的）             → 忽略
```

### 2.6 CompGetEquippedGizmosExtra改造

```
CompGetEquippedGizmosExtra() 流程：

1. base.CompGetEquippedGizmosExtra()
   → 透传标准gizmo（占位Tool的攻击gizmo始终存在）

2. OwnerHasTrionGland() 检查

3. 芯片攻击Gizmo（新增）：
   if leftHandAttackVerb != null:
     yield return new Command_BDPChipAttack {
       verb = leftHandAttackVerb,
       attackId = "left",
       icon = 左手芯片uiIcon,
       defaultLabel = 左手芯片label
     }
   if rightHandAttackVerb != null:
     yield return new Command_BDPChipAttack {
       verb = rightHandAttackVerb,
       attackId = "right",
       icon = 右手芯片uiIcon,
       defaultLabel = 右手芯片label
     }
   if dualAttackVerb != null:
     yield return new Command_BDPChipAttack {
       verb = dualAttackVerb,
       attackId = "dual",
       icon = 触发体uiIcon（或自定义双武器图标）,
       defaultLabel = "双手触发"
     }

4. 状态Gizmo（Gizmo_TriggerBodyStatus，不变）
5. 拳头Gizmo（CreateFistGizmo，不变）
6. 调试Gizmo（godMode下，不变）
```

## 3. Gizmo预期效果

| 芯片组合 | 显示的Gizmo |
|----------|------------|
| 无芯片 | 占位近战（触发体柄） |
| 近战A(左) | 占位近战 + 左手触发(A) |
| 近战A(左) + 近战B(右) | 占位近战 + 左手触发(A) + 右手触发(B) + 双手触发(A+B) |
| 近战A(左) + 近战A(右) | 占位近战 + 左手触发(A)（同芯片去重） + 双手触发(A+A) |
| 远程A(左) + 远程B(右) | 占位近战 + 左手触发(A) + 右手触发(B) + 双手触发(A+B) |
| 近战A(左) + 远程B(右) | 占位近战 + 左手触发(A) + 右手触发(B) |
| 武器A(左) + 护盾B(右) | 占位近战 + 左手触发(A) |
| 武器A(左) + 辅助B(右) | 占位近战 + 左手触发(A) + B的Ability gizmo（由Ability系统生成） |

## 5. 设计决策

| # | 决策 | 选择 | 理由 | 否决方案 |
|---|------|------|------|---------|
| T44 | Gizmo生成路径 | 芯片Verb设hasStandardCommand=false，CompGetEquippedGizmosExtra自定义生成 | 完全控制Gizmo数量和行为，不受GroupsWith合并影响 | 保持标准路径+Harmony Patch GroupsWith（违反无Patch原则）；拦截替换base返回的gizmo（脆弱） |
| T45 | 占位gizmo处理 | IVerbOwner始终返回占位Verb+Tools，标准路径自然生成占位gizmo | 占位近战gizmo始终可见，无需额外代码 | 手动生成占位gizmo（多余） |
| T46 | 攻击类别分类 | 四种：left/right/dual/combo | 简洁清晰，不按芯片名区分，避免复杂性 | 按chipDefName+侧别（过于具体，容易混乱） |
| T47 | Verb引用管理 | RebuildVerbs后缓存到CompTriggerBody字段 | 识别逻辑集中一处，Gizmo生成只读取引用 | 每帧从VerbTracker反向查找（重复计算，脆弱） |
| T48 | 多选pawn合并 | 暂不合并，预留GroupsWith接口 | 当前优先级低，接口已预留 | 立即实现（YAGNI） |
