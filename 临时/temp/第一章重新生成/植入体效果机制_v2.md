# 植入体效果机制

**总览**：植入体是RimWorld中通过手术安装到Pawn身体部位的永久性Hediff，分为两大类：**Hediff_Implant**（纯植入体，如仿生眼、心灵连接器）和**Hediff_AddedPart**（替换部位，如仿生腿、义肢）。植入体通过多种路径实现效果：AddedBodyPartProps修改部位效率、HediffStage提供能力/属性修正、HediffComp实现复杂行为。所有植入体必须关联身体部位（Part），并通过ThingDef定义物品形态和HediffDef定义健康效果。

## 一、Hediff_Implant vs Hediff_AddedPart

### 1.1 类继承关系

```
HediffWithComps
└── Hediff_Implant（植入体基类）
    └── Hediff_AddedPart（添加部位子类）
```

### 1.2 核心区别

| 特性 | Hediff_Implant | Hediff_AddedPart |
|------|---------------|-----------------|
| **用途** | 纯植入体（不替换部位） | 替换缺失或损坏的部位 |
| **PostAdd行为** | 仅验证Part不为null | 调用RestorePart恢复部位功能 |
| **子部位处理** | 不处理 | 为所有子部位添加MissingPart |
| **典型实例** | 仿生眼、心灵连接器、仿生心脏 | 仿生腿、义肢、假肢 |
| **部位状态** | 部位保持原状态 | 部位从"缺失"变为"有效" |

### 1.3 Hediff_Implant源码

```csharp
// 文件：Hediff_Implant.cs
public class Hediff_Implant : HediffWithComps
{
    public override void PostAdd(DamageInfo? dinfo)
    {
        base.PostAdd(dinfo);

        // 唯一逻辑：验证Part必须存在
        if (base.Part == null)
        {
            Log.Error(def.defName + " has null Part. It should be set before PostAdd.");
        }
    }

    public override void ExposeData()
    {
        base.ExposeData();

        // 加载后验证：Part为null则移除此Hediff
        if (Scribe.mode == LoadSaveMode.PostLoadInit && base.Part == null)
        {
            Log.Error(GetType().Name + " has null part after loading.");
            pawn.health.hediffSet.hediffs.Remove(this);
        }
    }
}
```

**关键认知**：
- Hediff_Implant本身不实现任何效果逻辑
- 所有效果通过HediffDef配置（Stage、Comp、AddedBodyPartProps）实现
- Part字段是强制要求，缺失会导致错误

### 1.4 Hediff_AddedPart源码

```csharp
// 文件：Hediff_AddedPart.cs
public class Hediff_AddedPart : Hediff_Implant
{
    public override void PostAdd(DamageInfo? dinfo)
    {
        base.PostAdd(dinfo);

        // 1. 恢复部位功能（移除MissingPart Hediff）
        pawn.health.RestorePart(base.Part, this, checkStateChange: false);

        // 2. 为所有子部位添加MissingPart
        for (int i = 0; i < base.Part.parts.Count; i++)
        {
            Hediff_MissingPart hediff_MissingPart =
                (Hediff_MissingPart)HediffMaker.MakeHediff(HediffDefOf.MissingBodyPart, pawn);
            hediff_MissingPart.IsFresh = true;
            hediff_MissingPart.lastInjury = HediffDefOf.SurgicalCut;
            hediff_MissingPart.Part = base.Part.parts[i];
            pawn.health.hediffSet.AddDirect(hediff_MissingPart);
        }
    }
}
```

**关键认知**：
- **RestorePart**：移除目标部位的MissingPart Hediff，使部位重新生效
- **子部位MissingPart**：例如安装仿生腿后，脚趾会标记为缺失（因为仿生腿没有脚趾）
- **IsFresh = true**：标记为新鲜缺失，影响疼痛和治疗优先级

## 二、AddedBodyPartProps配置

### 2.1 字段定义

```csharp
// 文件：AddedBodyPartProps.cs
public class AddedBodyPartProps
{
    public float partEfficiency = 1f;      // 部位效率（1.0 = 100%）
    public bool solid = true;              // 是否为实体（影响某些伤害类型）
    public bool isGoodWeapon;              // 是否为良好武器（影响近战）
    public bool betterThanNatural;         // 是否优于天然（影响Pawn价值和某些事件）
}
```

### 2.2 字段详解

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| **partEfficiency** | float | 1.0 | 部位效率倍数。1.25表示125%效率，0.8表示80%效率。影响该部位对能力的贡献度 |
| **solid** | bool | true | 是否为实体部位。false表示空心（如假肢），影响某些伤害类型的计算 |
| **isGoodWeapon** | bool | false | 是否为良好武器。true时该部位在近战中有额外伤害加成 |
| **betterThanNatural** | bool | false | 是否优于天然部位。影响Pawn的市场价值、某些事件触发条件 |

### 2.3 partEfficiency的作用机制

**部位效率如何影响能力**：

1. **能力计算公式**：
   ```
   最终能力值 = Σ(部位效率 × 部位权重) × 其他修正器
   ```

2. **示例**（视力Sight能力）：
   - 两只眼睛，每只权重0.5
   - 左眼：天然（效率1.0）
   - 右眼：仿生眼（效率1.25）
   - 最终视力 = (1.0 × 0.5 + 1.25 × 0.5) = 1.125 = 112.5%

3. **超过100%的效果**：
   - 视力112.5%：射击精度提升、手术成功率提升
   - 操作能力超100%：工作速度加快
   - 移动能力超100%：移动速度加快

### 2.4 典型配置实例

**仿生眼（BionicEye）**：
```xml
<ThingDef ParentName="BodyPartBionicBase">

  <label>bionic eye</label>

  <addedPartProps>
```

```xml
  <addedPartProps>
```

## 三、植入体效果实现的四大路径

### 3.1 路径总览

| 路径 | 实现方式 | 适用场景 | 示例 |
|------|---------|---------|------|
| **1. AddedBodyPartProps** | partEfficiency修改部位效率 | 简单的能力增强/削弱 | 仿生眼125%视力 |
| **2. HediffStage** | statFactors/statOffsets修改属性 | 全局属性修正 | 心灵连接器增加心灵熵上限 |
| **3. HediffStage** | capMods修改能力 | 直接修改能力值 | 仿生心脏+15%血液泵送 |
| **4. HediffComp** | 组件实现复杂行为 | 特殊效果、交互逻辑 | 植入体发光、给予能力 |

### 3.2 路径1：AddedBodyPartProps（部位效率）

**机制**：
- 通过ThingDef的`<addedPartProps>`配置
- 修改部位对能力的贡献度
- 自动参与能力计算，无需额外代码

**优势**：
- 配置简单，只需一个数值
- 自动适配所有相关能力
- 性能开销最小

**局限**：
- 只能修改部位相关的能力
- 无法实现全局属性修正
- 无法实现特殊行为

**典型实例**：
```xml
<!-- 仿生腿：移动能力提升至125% -->
<addedPartProps>
  <partEfficiency>1.25</partEfficiency>
</addedPartProps>
```

### 3.3 路径2：HediffStage的statFactors/statOffsets

**机制**：
- 通过HediffDef的`<stages>`配置
- statFactors：乘法修正（如×1.2）
- statOffsets：加法修正（如+0.5）

**优势**：
- 可修改任意Stat（属性）
- 支持多阶段（根据Severity变化）
- 可配置大量属性

**局限**：
- 不直接修改能力（Capacity）
- 需要了解Stat系统

**典型实例**（心灵连接器）：
```xml
<HediffDef>
  <defName>PsychicAmplifier</defName>
  <hediffClass>Hediff_Psylink</hediffClass>
  <initialSeverity>1</initialSeverity>
  <maxSeverity>6</maxSeverity>

  <stages>
    <!-- 1级：基础效果 -->
    <li>
      <minSeverity>1</minSeverity>
      <statFactors>
        <PsychicEntropyMax>1.0</PsychicEntropyMax>
        <PsychicEntropyRecoveryRate>1.0</PsychicEntropyRecoveryRate>
      </statFactors>
    </li>

    <!-- 2级：提升33%熵上限 -->
    <li>
      <minSeverity>2</minSeverity>
      <statFactors>
        <PsychicEntropyMax>1.3334</PsychicEntropyMax>
        <PsychicEntropyRecoveryRate>1.125</PsychicEntropyRecoveryRate>
      </statFactors>
    </li>

    <!-- 3-6级：持续提升 -->
    <!-- ... -->
  </stages>
</HediffDef>
```

### 3.4 路径3：HediffStage的capMods

**机制**：
- 通过HediffDef的`<stages><capMods>`配置
- 直接修改Pawn的能力值（Capacity）
- 支持offset（加法）和setMax（设置上限）

**优势**：
- 直接修改能力，效果明确
- 可设置能力上限（如最大意识80%）
- 适合实现致残效果

**局限**：
- 只能修改预定义的能力
- 不如partEfficiency自然（不考虑部位权重）

**典型实例**（仿生心脏）：
```xml
<HediffDef ParentName="ImplantHediffBase">
  <defName>BionicHeart</defName>
  <label>bionic heart</label>

  <stages>
    <li>
      <capMods>
        <li>
          <capacity>BloodPumping</capacity>
          <offset>0.15</offset>  <!-- +15%血液泵送能力 -->
        </li>
      </capMods>
    </li>
  </stages>
</HediffDef>
```

### 3.5 路径4：HediffComp组件

**机制**：
- 通过HediffDef的`<comps>`配置
- 使用60+种内置HediffComp类型
- 可自定义Comp实现复杂逻辑

**优势**：
- 功能强大，可实现任意行为
- 模块化，易于组合
- 支持自定义扩展

**局限**：
- 配置复杂
- 可能需要编写C#代码

**常用Comp类型**：

| Comp类型 | 功能 | 典型用途 |
|---------|------|---------|
| HediffComp_VerbGiver | 给予动词（攻击方式） | 植入武器 |
| HediffComp_GiveAbility | 给予能力 | 心灵能力、特殊技能 |
| HediffComp_Effecter | 产生视觉效果 | 发光植入体 |
| HediffComp_DisappearsOnDeath | 死亡时消失 | 临时植入体 |

**典型实例**（发光植入体）：
```xml
<HediffDef ParentName="ImplantHediffBase">
  <defName>LuminousImplant</defName>
  <label>luminous implant</label>

  <comps>
    <li Class="HediffCompProperties_Effecter">
      <effecterDef>GlowEffect</effecterDef>
    </li>
  </comps>
</HediffDef>
```

## 四、植入体配置的完整流程

### 4.1 ThingDef配置（物品形态）

```xml
<ThingDef ParentName="BodyPartBionicBase">
  <defName>BionicEye</defName>
  <label>bionic eye</label>
  <description>...</description>

  <!-- 1. 基础属性 -->
  <statBases>
    <WorkToMake>26000</WorkToMake>
    <MarketValue>1200</MarketValue>
  </statBases>

  <!-- 2. 制作配方 -->
  <recipeMaker>
    <researchPrerequisite>Bionics</researchPrerequisite>
    <skillRequirements>
      <Crafting>8</Crafting>
    </skillRequirements>
  </recipeMaker>

  <!-- 3. 植入体属性 -->
  <addedPartProps>
    <solid>true</solid>
    <partEfficiency>1.25</partEfficiency>
    <betterThanNatural>true</betterThanNatural>
  </addedPartProps>

  <!-- 4. 移除时掉落 -->
  <spawnThingOnRemoved>BionicEye</spawnThingOnRemoved>
</ThingDef>
```

### 4.2 HediffDef配置（健康效果）

**方式1：使用通用模板**（推荐）

大多数植入体不需要单独的HediffDef，直接使用ThingDef配置即可。游戏会自动创建对应的Hediff。

**方式2：自定义HediffDef**（特殊效果）

```xml
<HediffDef ParentName="ImplantHediffBase">
  <defName>CustomImplant</defName>
  <hediffClass>Hediff_Implant</hediffClass>
  <label>custom implant</label>

  <!-- 阶段效果 -->
  <stages>
    <li>
      <statFactors>
        <WorkSpeedGlobal>1.2</WorkSpeedGlobal>
      </statFactors>
      <capMods>
        <li>
          <capacity>Consciousness</capacity>
          <offset>0.1</offset>
        </li>
      </capMods>
    </li>
  </stages>

  <!-- 组件效果 -->
  <comps>
    <li Class="HediffCompProperties_VerbGiver">
      <verbs>
        <li>
          <verbClass>Verb_MeleeAttack</verbClass>
          <meleeDamageDef>Blunt</meleeDamageDef>
          <meleeDamageBaseAmount>12</meleeDamageBaseAmount>
        </li>
      </verbs>
    </li>
  </comps>
</HediffDef>
```

### 4.3 RecipeDef配置（手术配方）

```xml
<RecipeDef ParentName="SurgeryInstallBodyPartArtificialBase">
  <defName>InstallBionicEye</defName>
  <label>install bionic eye</label>
  <description>Install a bionic eye.</description>
  <descriptionHyperlinks>
    <ThingDef>BionicEye</ThingDef>
  </descriptionHyperlinks>

  <!-- 手术目标部位 -->
  <appliedOnFixedBodyParts>
    <li>Eye</li>
  </appliedOnFixedBodyParts>

  <!-- 所需物品 -->
  <ingredients>
    <li>
      <filter>
        <thingDefs>
          <li>BionicEye</li>
        </thingDefs>
      </filter>
      <count>1</count>
    </li>
  </ingredients>

  <!-- 手术难度 -->
  <skillRequirements>
    <Medicine>5</Medicine>
  </skillRequirements>
</RecipeDef>
```

## 五、典型植入体实例分析

### 5.1 仿生眼（BionicEye）

**效果路径**：AddedBodyPartProps（partEfficiency）

**配置要点**：
- partEfficiency = 1.25（125%视力）
- betterThanNatural = true（优于天然）
- 安装到Eye部位

**效果**：
- 视力提升至125%
- 射击精度提升
- 手术成功率提升
- Pawn价值增加

### 5.2 心灵连接器（PsychicAmplifier / Psylink）

**效果路径**：HediffStage的statFactors + 自定义Hediff_Psylink类

**配置要点**：
- hediffClass = Hediff_Psylink（自定义类）
- Severity表示等级（1-6）
- 每个等级有不同的statFactors
- 安装到Brain部位

**效果**：
- 1级：基础心灵熵上限和恢复速率
- 2级：熵上限+33%，恢复速率+12.5%
- 6级：熵上限+200%，恢复速率+87.5%
- 解锁心灵能力（通过Ability系统）

### 5.3 仿生心脏（BionicHeart）

**效果路径**：HediffStage的capMods

**配置要点**：
- capMods修改BloodPumping能力
- offset = 0.15（+15%）

**效果**：
- 血液泵送能力提升至115%
- 提升整体健康状态
- 减少失血死亡风险

### 5.4 机械连接（Mechlink - Biotech DLC）

**效果路径**：自定义Hediff_Mechlink类 + HediffComp

**配置要点**：
- hediffClass = Hediff_Mechlink
- 复杂的C#逻辑（机械师能力）
- 使用多个HediffComp

**效果**：
- 解锁机械师能力
- 可控制机械体
- 影响社交互动

## 六、开发者要点

> **关键认知**：
>
> 1. **选择正确的基类**：
>    - 替换部位（如义肢）→ Hediff_AddedPart
>    - 纯植入体（如仿生眼）→ Hediff_Implant
>    - 特殊逻辑 → 自定义子类
>
> 2. **效果路径选择**：
>    - 简单能力修正 → AddedBodyPartProps.partEfficiency
>    - 全局属性修正 → HediffStage.statFactors
>    - 直接能力修正 → HediffStage.capMods
>    - 复杂行为 → HediffComp
>
> 3. **Part字段是强制的**：
>    - 所有植入体必须关联身体部位
>    - Part为null会导致错误和移除
>    - 使用RecipeDef的appliedOnFixedBodyParts指定部位
>
> 4. **AddedPart的子部位处理**：
>    - Hediff_AddedPart会为所有子部位添加MissingPart
>    - 例如：仿生腿安装后，脚趾会标记为缺失
>    - 这是设计行为，不是bug
>
> 5. **partEfficiency的自然性**：
>    - partEfficiency会自动参与能力计算
>    - 考虑部位权重（如双眼各占50%）
>    - 优于直接使用capMods（更符合游戏逻辑）
>
> 6. **spawnThingOnRemoved**：
>    - 移除植入体时掉落物品
>    - 通常设置为ThingDef本身
>    - 允许植入体回收利用
>
> 7. **betterThanNatural的影响**：
>    - 影响Pawn的市场价值
>    - 影响某些事件触发（如贵族要求）
>    - 影响AI的手术决策
>
> 8. **配置文件组织**：
>    - ThingDef：物品形态、制作、addedPartProps
>    - HediffDef：健康效果、Stage、Comp（可选）
>    - RecipeDef：手术配方、部位、技能要求
>    - 三者通过defName关联
