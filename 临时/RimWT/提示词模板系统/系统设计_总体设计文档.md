# RimWT提示词模板系统·总体设计文档

## 🎯 系统目标

当新对话窗口打开时，AI 能在 **30 秒内**：
1. 理解 RimWT 项目的完整背景
2. 定位当前所处的阶段和进度
3. 了解已完成的内容和决策约束
4. 清楚下一步的具体任务和成功标准
5. 知道需要查看哪些参考资料

---

## 📐 系统架构

### 三层设计（从上到下）

```
┌─────────────────────────────────────────────────────────┐
│  第0层：元结构（通用框架）                               │
│  定义：7大模块的通用结构                                  │
│  特点：项目无关、可适用于任何多阶段项目                  │
│  文件：00_元结构_通用提示词模板.md                       │
└─────────────────────────────────────────────────────────┘
              │
              ↓
┌─────────────────────────────────────────────────────────┐
│  第1层：场景模板（应用框架）                             │
│  定义：针对两种常见场景的具体模板                        │
│  场景1：阶段衔接（前阶段完成 → 后阶段开启）             │
│  场景2：阶段内继续（中断 → 恢复）                       │
│  文件：01_场景1_阶段衔接模板.md                          │
│       02_场景2_阶段内继续模板.md                        │
└─────────────────────────────────────────────────────────┘
              │
              ↓
┌─────────────────────────────────────────────────────────┐
│  第2层：具体实例（可直接使用）                          │
│  定义：针对具体阶段的完整提示词                          │
│  第1阶段实例：03_阶段1_启动提示词.md                    │
│  第2-14阶段：参照第1阶段实例创建                       │
└─────────────────────────────────────────────────────────┘
```

### 核心模块（7个）

```
【模块1】项目身份卡
├── 项目名称
├── 项目简述
├── 当前版本
└── 最后更新时间

【模块2】核心上下文
├── 项目本质（3-5句话）
├── 核心方法论（3-5条）
└── 技术栈信息

【模块3】进度定位
├── 当前所处阶段
├── 阶段目标和决策点
└── 项目总体进度

【模块4】已完成内容清单
├── 本阶段已完成文档
├── 已做出的关键决策
└── 前阶段的限制条件（场景1）

【模块5】当前任务状态
├── 进行中的文档（场景2）
├── 已完成部分（场景2）
├── 待完成部分（场景2）
└── 阻塞点说明（场景2）

【模块6】下一步行动
├── 立即执行任务
├── 任务的输入/输出/成功标准
└── 后续任务队列

【模块7】必读资料索引
├── 规范文档
├── 参考文档
└── 本阶段输入材料
```

## 🔄 使用场景矩阵

| 场景 | 触发条件 | 使用模板 | 关键操作 |
|------|---------|---------|---------|
| **新项目启动** | 项目刚开始，进入第1阶段 | 03（具体实例） | 复制提示词→新对话 |
| **阶段衔接** | 前阶段100%完成，准备进入后阶段 | 01（场景1） | 填充变量→保存→复制 |
| **阶段继续** | 本阶段进行中，对话中断，需恢复 | 02（场景2） | 统计进度→填充→复制 |
| **创建新阶段提示词** | 需要为阶段2-14创建启动提示词 | 03作为模板 | 复制03→改写为新阶段 |
| **优化系统本身** | 发现模板设计的问题 | 00（元结构） | 修改结构→更新场景模板 |

## 🛠️ 快速生成工作流

### 新阶段提示词生成（15分钟）

```
Step 1: 确定信息源
  从蓝图文档获取该阶段的信息
  ├─ 阶段名称和编号
  ├─ 文档清单
  ├─ 关键决策点
  └─ 前置约束

Step 2: 选择模板
  ├─ 如果是第1阶段 → 用03直接复制
  ├─ 如果是2-14阶段 → 用03作为模板
  └─ 如果不是新阶段开启 → 用场景2模板

Step 3: 填充变量
  按照"变量填充速查表"填充所有 <...> 部分
  ├─ 通用变量（5个）
  ├─ 阶段特定变量（3个）
  └─ 进度特定变量（4个）

Step 4: 保存文件
  命名规范：RimWT_阶段X_<场景>_<日期>.md
  文件位置：提示词模板系统文件夹

Step 5: 验证
  阅读一遍，确保：
  ├─ [ ] 所有 <...> 都已填充
  ├─ [ ] 文件路径正确
  ├─ [ ] 版本号和日期正确
  └─ [ ] 任务描述清晰
```

### 阶段继续提示词更新（20分钟）

```
Step 1: 收集本阶段进度
  从上次对话的输出统计：
  ├─ 已完成的文档列表
  ├─ 进行中的具体进度（完成了哪些子任务）
  ├─ 未开始的文档列表
  └─ 当前的阻塞点（如有）

Step 2: 复制场景2模板
  打开 02_场景2_阶段内继续模板.md

Step 3: 填充进度信息
  ├─ 项目基础信息（版本、日期等）
  ├─ 阶段整体信息（从蓝图复制）
  ├─ 进度快照（从对话输出复制）
  ├─ 进行中的工作（详细说明子任务）
  ├─ 未开始的工作（列表）
  └─ 下一步行动（明确立即执行的任务）

Step 4: 保存
  命名：RimWT_阶段X_继续_YYYYMMDD.md

Step 5: 验证
  [ ] 进度百分比计算正确
  [ ] 下一个立即任务清晰
  [ ] 成功标准可验证
```

## 🎓 系统使用的三个阶段

### 第一阶段：学习（首次使用）

**目标**：理解系统的设计逻辑和使用方法

**行动**：
1. 读元结构（00），理解7大模块
2. 看场景1和2的模板，了解应用方式
3. 查看阶段1实例（03），看实际效果
4. 参考使用指南，进行首次操作

**预计时间**：30-45分钟

### 第二阶段：应用（正常使用）

**目标**：熟练使用系统，自动化工作流

**行动**：
- 每次进入新阶段时，生成对应提示词
- 阶段中断时，更新进度快照
- 日常工作中积累使用经验

**预计时间**：每阶段 15-20 分钟

### 第三阶段：优化（持续改进）

**目标**：根据实践发现，优化系统本身

**行动**：
- 发现某些信息总是缺失？改进模块7
- 某个场景的操作困难？改进对应模板
- 发现新的使用场景？添加新模板

**预计时间**：根据发现的问题

## 🔧 扩展与定制

### 如何为新项目应用这套系统

**Step 1：创建项目版本的元结构**
```
复制 00_元结构
↓
调整项目特定的信息
  ├─ 项目背景信息
  ├─ 核心方法论
  └─ 技术栈
↓
保存为项目专用版本
```

**Step 2：创建项目版本的场景模板**
```
基于项目版本的元结构
↓
创建两个场景模板的变种
  ├─ 场景1：阶段衔接
  └─ 场景2：阶段内继续
↓
保存为项目专用版本
```

**Step 3：为每个阶段创建具体实例**
```
根据项目蓝图
↓
为每个阶段生成启动提示词
↓
保存到项目的提示词文件夹
```

### 什么时候需要定制

| 情况 | 定制方式 |
|------|---------|
| 项目只有5个阶段，不是14个 | 调整元结构的"项目总体进度"模块 |
| 需要新的场景模板 | 在场景1和2的基础上，创建第3个场景 |
| 某个模块的信息总是不足 | 在元结构中补充新的子模块 |
| 项目有特殊的审查流程 | 在模块7中添加"必读的审查文档" |

## 📈 系统效果指标

### 成功指标

| 指标 | 目标值 | 测量方法 |
|------|--------|---------|
| **AI上下文时间** | < 30秒 | 新对话中，AI多长时间内完全理解项目 |
| **重复工作率** | < 5% | AI是否重复做已完成的工作 |
| **任务清晰度** | 100% | AI是否清楚下一步具体要做什么 |
| **提示词准备时间** | < 20分钟 | 生成新阶段提示词需要多长时间 |
| **对话流畅性** | > 80% | 有多少比例的对话流畅进行，无需额外说明 |

### 评估方法

**第一次使用后**：
- 新AI是否在第一条消息中问项目背景？（应该不问）
- 新AI是否能准确说出当前任务？（应该能说）

**使用几个月后**：
- 有多少次对话因为"进度丢失"而重做工作？（应该接近0）
- 平均生成一份新提示词花费多久？（应该稳定在15-20分钟）

## 🔐 维护与管理

### 版本管理

**文件命名规范**：
```
RimWT_<内容>_<场景>_v<版本>.md
示例：
  RimWT_阶段1_启动提示词_v1.0.md
  RimWT_阶段2_阶段衔接_v1.0.md
  RimWT_阶段2_继续_20260127.md
```

**版本号规则**：
- 元结构和场景模板：语义化版本（v1.0, v1.1, v2.0）
- 具体实例：日期版本（无需版本号，用日期标记）

### 备份管理

**备份时机**：
- 元结构或场景模板更新时 → 备份旧版本
- 阶段完成，提示词最终版本 → 备份到归档文件夹

**备份位置**：
```
提示词模板系统/
├── 活跃文件（当前使用）
│   ├── 00_元结构...
│   ├── 01_场景1...
│   └── ...
└── 归档文件（历史版本）
    ├── v0.9版本
    └── ...
```

## 📞 故障排查

### 问题1：新AI仍然问项目背景

**原因**：提示词中缺少项目身份卡或核心上下文

**解决**：检查模块1和2是否完整

### 问题2：新AI做重复工作

**原因**：场景2的"已完成部分"填充不完整

**解决**：仔细统计上次对话的完成情况

### 问题3：提示词太长，AI似乎没注意

**原因**：超过最佳长度（2000-3000字）

**解决**：精简冗余信息，突出重点

### 问题4：不知道怎么填某个变量

**原因**：模板的说明不足

**解决**：参考"变量填充速查表"或查看阶段1实例
