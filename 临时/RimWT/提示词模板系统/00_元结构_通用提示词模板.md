## 📐 设计原则

1. **模块化**：每个信息块独立，便于修改
2. **可扩展**：新增阶段或场景只需复用结构
3. **精准定位**：新AI能在30秒内理解当前状态
4. **最小信息原则**：只包含必要信息，避免冗余

## 🔧 模板变量说明

### 必填变量
| 变量 | 说明 | 示例 |
|------|------|------|
| `<项目完整名称>` | 项目正式名称 | "RimWT - RimWorld境界触发者模组" |
| `<阶段名称>` | 当前阶段名 | "源材料理解" |
| `<进度>` | 完成数/总数 | "3/7" |
| `<路径>` | 文件绝对路径 | "C:\...\文档.md" |

### 可选变量
| 变量 | 说明 | 留空规则 |
|------|------|----------|
| `<阻塞点>` | 当前障碍 | 无阻塞时写"无" |
| `<关键决策>` | 重要决策记录 | 新阶段开始时可为空 |

## 📦 模板使用流程

```
1. 复制元结构 → 2. 填充阶段变量 → 3. 更新进度数据 → 4. 生成具体提示词
```

---

## 🎯 两大使用场景的差异点

### 场景1：阶段衔接（前阶段→后阶段）
**模块4重点**：前阶段的完整成果和关键决策
**模块5重点**：后阶段的起始任务
**模块6重点**：后阶段的第一个任务

### 场景2：阶段内继续（中断→恢复）
**模块4重点**：本阶段已完成的部分
**模块5重点**：未完成的具体子任务
**模块6重点**：从中断处继续的操作

## 🔄 扩展指南

### 新增阶段时
1. 复制本元结构
2. 填充该阶段的特定信息
3. 更新"项目总体进度"
4. 补充该阶段的必读资料

### 新增场景时
1. 基于元结构调整模块重点
2. 保持7大模块结构不变
3. 在场景差异点说明中添加新场景
