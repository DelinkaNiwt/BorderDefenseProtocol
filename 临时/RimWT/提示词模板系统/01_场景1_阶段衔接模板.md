# RimWT提示词模板·场景1·阶段衔接

**适用情形**：前阶段所有工作已完成，准备开启后阶段

**核心目标**：
1. 新AI快速了解前阶段的成果和决策
2. 明确后阶段的起始点和目标
3. 继承前阶段的决策约束

## 🎯 填写指南

### 字段说明
| 字段 | 来源 | 说明 |
|------|------|------|
| `<阶段号>` | 蓝图文档 | 当前阶段的序号 |
| `<阶段名>` | 蓝图文档 | 从RimWT流程文档的表格里复制 |
| `<完成度>` | 进度表 | 该阶段的完成数/总数 |
| `<决策内容>` | 前阶段的产出 | 从前阶段的设计文档里提取 |
| `<关键决策点>` | 蓝图文档表格最后一列 | 直接从蓝图复制 |

### 动态内容维护
- **前阶段成果总结**：每当前阶段完成时，从实际输出物中填充
- **关键决策**：从该阶段的正式设计文档中提取
- **限制条件**：从决策记录中总结出来

### 必读资料的优先级排序
1. 违反会被打回的规范文档（必须最先列出）
2. 涉及本阶段的技术/框架文档
3. 前阶段的输出物
4. 其他参考资料

## 🔄 使用流程

```
前阶段完成
  ↓
提取前阶段的关键输出 → 制作阶段衔接提示词
  ↓
用户确认无误 → 保存为"阶段X_启动提示词.md"
  ↓
新对话开启时 → 直接复制本提示词到对话框
  ↓
新AI快速上手 → 开始执行阶段内任务
```
