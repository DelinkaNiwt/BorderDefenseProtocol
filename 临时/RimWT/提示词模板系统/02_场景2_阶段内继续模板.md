# RimWT提示词模板·场景2·阶段内继续

**适用情形**：某阶段已开启但未完成，需要在新对话中继续工作

**核心目标**：
1. 新AI理解本阶段的整体目标和约束
2. 快速定位已完成的部分和未完成的部分
3. 从中断点立即恢复工作，无缝接棒

## 🎯 填写指南

### 场景2特有字段说明
| 字段 | 来源 | 说明 |
|------|------|------|
| `<上次中断时间>` | 历史记录 | 上次对话的结束日期 |
| `<已完成进度>` | 实时统计 | 该文档目前完成了百分之几 |
| `<已完成的子任务>` | 前一个对话的成果 | 列出所有已经做完的子任务 |
| `<当前阻塞点>` | 用户反馈 | 如果上个对话卡住了，说明是什么 |
| `<下一个立即任务>` | 逻辑推导 | 基于已完成部分，推导出下一步 |

### 关键的进度状态符号
| 符号 | 含义 | 什么时候用 |
|------|------|-----------|
| ☑ | 已完成 | 文档完全写好，已审核通过 |
| ◐ | 进行中 | 正在写，还有部分没完成 |
| ☐ | 未开始 | 完全没动过 |

### 三层信息优先级

**最高优先级**（一定要填）：
- 当前进行中的文档名
- 已完成的具体部分
- 下一个立即任务

**中等优先级**（尽量填）：
- 当前阻塞点
- 已做出的关键决策
- 后续任务队列

**参考优先级**（不填也行）：
- 性能对比数据
- 详细的技术背景

## 🔄 使用流程

```
对话1开启：制作场景1提示词 → 用户提供第一批输入
  ↓
对话1进行中：生成/完成部分成果 → 用户反馈调整
  ↓
对话1中断：总结当前进度，保存中断点
  ↓
对话2开启：制作场景2提示词（本模板） → 恢复工作状态
  ↓
对话2进行中：继续完成未完成的任务
```

## 与场景1的差异

| 对比维度 | 场景1（阶段衔接） | 场景2（阶段内继续） |
|---------|------------------|-------------------|
| **重点** | 前阶段的完整成果 | 本阶段已完成 + 未完成 |
| **模块4** | 前阶段完整清单 | 本阶段进度快照 |
| **模块5** | 后阶段的起始任务 | 进行中的任务 + 未开始的任务 |
| **模块6** | 后阶段的第一个任务 | 立即继续的任务 |
| **适用场景** | 上一阶段100%完成 | 当前阶段有工作进行中 |
