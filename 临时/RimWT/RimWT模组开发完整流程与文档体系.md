## 📝 进度标记说明

### 状态符号

| 符号 | 状态 | 说明 | 用途 |
|------|------|------|------|
| ☐ | **未开始** | 还没有着手这份文档 | 计划中的新文档 |
| ◐ | **进行中** | 正在编写、收集信息或审核 | 现在进行的工作 |
| ☑ | **已完成** | 完成初稿或最终版本 | 已产出可用的文档 |
| ⊘ | **暂停** | 因各种原因暂时搁置 | 有重要阻碍或依赖问题 |
| ✕ | **废弃** | 决定不做这个文档 | 需求变更或范围调整 |
| ● | **修订中** | 已完成但需要基于反馈修改 | 完成后发现需要改进 |

### 如何更新进度

**方法1：修改表格中的"状态"列**
```
| 1.1 | 文档名 | ... | ☐ 未开始 |  ← 修改这里
```

**方法2：修改对应阶段的「进度」标记**
```
### **第一阶段：源材料理解** [进度: 0/7]  ← 修改这里
```
例如完成3个时改为 `[进度: 3/7]`

**方法3：更新最上面的「总进度追踪」表**
```
| 第1-2阶段（源材料） | 2 | 0/13 |  ← 修改"完成"这一列
```

### 进度更新示例

- **文档开始编写时**：☐ 未开始 → ◐ 进行中
- **文档写好后**：◐ 进行中 → ☑ 已完成
- **发现需要修改**：☑ 已完成 → ● 修订中 → ☑ 已完成
- **决定不做了**：☐ 未开始 → ✕ 废弃（并注明原因）
- **卡住等待中**：◐ 进行中 → ⊘ 暂停（并注明原因）
