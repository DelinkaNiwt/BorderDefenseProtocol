## 五、常见问题

### Q1：框架v0.5还有问题吗？
**A**：没有。所有P0级致命错误和P1级高风险问题都已修复。框架代码可正常编译和运行。

### Q2：框架和应用层怎么分工？
**A**：
- **框架**：提供机制（虚拟伤害、快照回滚、状态转换、消耗计算等）
- **应用层**：提供数据（天赋、组件效果、建筑功能等）
- **接口**：Strategy通过方法向框架提供数据

### Q3：怎样添加新实体类型？
**A**：无需修改框架代码。只需：
1. 定义新ThingDef
2. 添加CompTrion
3. 在SelectStrategy中添加识别逻辑
4. 实现新的Strategy类

### Q4：可以不使用Harmony补丁吗？
**A**：不行。需要3个Harmony补丁：
1. 伤害拦截（PreApplyDamage）
2. 截肢检测（AddHediff）
3. 需求冻结（Need.NeedInterval）

### Q5：debuff效果能自定义吗？
**A**：可以。框架负责在什么时候施加debuff，具体效果由HediffDef的capMods定义，完全可自定义。
