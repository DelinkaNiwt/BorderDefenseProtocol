## 第一部分：Trion 能量的基本设定

### 1.1 Trion 是什么？

**我的理解：**
- Trion 是一种能量资源
- 存在于生物体内（或人造物内）
- 可用于生成战斗体、激活组件、使用能力  【正确但不完整，所有依赖于Trion系统的实体物品、行为交互、能力状态等都有可能需要使用Trion为资源，具体情况再分析】
- 会自然恢复（速度受多种因素影响） 【人体内的Trion只能自然恢复，恢复受饱食度、特性、休养舱影响，暂不考虑其他途径。未来计划有建筑能存储Trion，它只能通过某种仪式献祭（分解）物品来恢复，当前阶段不考虑实现】

**关键属性：**
- **总量（Total）**：Trion 的最大容量
- **占用（Reserved）**：装备组件时预留的容量
- **可用（Available）**：当前可实际使用的量 = 总量 - 占用 - 已消耗

**公式关系：**
```
总量 = 占用 + 可用 + 累计消耗
可用量 = 总量 - 占用 - 累计消耗
```

### 1.2 Trion 容量的决定因素

## 第二部分：Trion 的"占用"机制

### 2.1 什么是"占用"？

**我的理解：**
- 装备组件到触发器时，会**预留**一部分 Trion 容量  【片面。未生成战斗体时，组件处于未连接状态，不会占用容量。当处于战斗体状态下，组件处于休眠或激活状态时，才占用容量】
- 这部分 Trion 没有被消耗，只是被"锁定"
- 类似于"押金"或"保证金"

### 2.2 占用的生命周期

**装备组件时：**
```
触发器装备弧月（占用10）
→ 总量不变
→ 占用 +10
→ 可用 -10
【错误，占用0，可用=总量】
```

【**生成战斗体时：**
```
-> 所有组件注册（未连接 -> 休眠）
-> 占用 +460 （假设所有组件总需占用460）
-> 可用 -460
```
】

**主动解除战斗体时：**
```
主动解除
→ 所有组件卸载
→ 占用归零
→ 可用 +460（之前占用的全部返还）
```

**被动解除战斗体时（Trion耗尽/器官被毁）：**
```
战斗体破裂
→ 占用的 Trion 全部流失
→ 表现为一次性消耗
→ 通常导致 Trion可用值 归零
```

### 2.3 典型占用值（参考）

| 组件类型 | 占用值 | 示例 |
|---------|-------|------|
| 近战武器 | 10 | 弧月 |
| 远程武器 | 10 | 炸裂弹 |
| 辅助组件 | 10 | 护盾、变色龙 |
| 能力组件 | 10 | 弧月旋空 |
| 特殊系统 | 400 | 紧急脱离系统 |

**流程图示例总占用：**
```
左手：弧月(10) + 护盾(10) = 20
右手：旋空(10) + 炸裂弹(10) + 护盾(10) + 变色龙(10) = 40
特殊：紧急脱离(400) = 400
总占用 = 460
```

❓ **疑问：** 同一槽位装备多个组件，占用是累加还是只计最大值？
- 我理解是**累加**，因为流程图中左手2个组件、右手4个组件都分别列出了占用值  【正确】

## 第三部分：Trion 的消耗机制

### 3.1 消耗的两大类型

#### 类型 A：一次性消耗

**场景1：激活组件**
- 每次激活组件都要支付"激活费用"
- 流程图示例：
  - 激活变色龙：-5（一次性）
  - 激活炸裂弹：-5（一次性）
  - 激活护盾：-5（一次性）
【正确。提醒：激活费用并非统一，示例只为演示】

**场景2：射击**
- 远程武器组件射击需要弹药费用
- 流程图示例：射出2发弹药 → -2

**场景3：护盾抵挡**
- 护盾成功抵挡攻击时扣除费用
- 流程图示例：护盾抵挡 → -5

**场景4：能力释放**
- 触发器能力、组件能力、Trion能力
- 流程图示例：释放弧月旋空 → -50

**场景5：受伤**
- 战斗体受到未被护盾抵挡的攻击
- 流程图示例：
  - 左手受伤 → -10（一次性）
  - 右手臂被切除 → -50（一次性）

**场景6：战斗体破裂时的占用释放**
- 被动解除时，所有占用值被一次性消耗
- 流程图示例：破裂时占用的460全部流失

#### 类型 B：持续性消耗（每Tick） 【不可能每tick，性能负担太大。预计至少每60tick，应该就是CompTick()？】

**场景1：战斗体维持**
- 战斗体存在时的基础消耗
- 流程图示例：每单位时间 -1
- ❓ 疑问：这是固定值还是可变的？ 【基础的维持消耗暂定是固定值，没有可变的需求。可以保留可变接口】

**场景2：组件功效维持**
- 部分组件激活后需要持续消耗
- 流程图示例：
  - 变色龙激活后 +1/单位时间（额外消耗）
  - 移动时：基础-1 + 变色龙-1 = -2/单位时间
- ❌ 我之前错误理解：大多数组件都有持续消耗
- ✅ 正确理解：只有**少数**组件（如变色龙）有持续消耗

**场景3：伤口泄漏**
- 战斗体受伤后会持续泄漏 Trion
- 流程图示例：
  - 左手受伤：+2/单位时间（泄漏速率）
  - 右手切除：+5/单位时间（泄漏速率）
  - 总泄漏速率累加
- 表现为战斗体维持消耗增加

【以上类型B持续消耗的场景，全都是按相同的单位时间（每60tick）持续消耗，建议累加求和后统一计算逻辑】

### 3.2 不消耗 Trion 的行为

**我之前的错误理解 ❌：**
- 移动消耗 Trion
- 近战攻击消耗 Trion

**正确理解 ✅：**
- **移动不消耗** Trion
- **普通攻击（近战）不消耗** Trion（流程图：L→M 普通攻击标注"不消耗"）
【正确
- **远程攻击 消耗** Trion】

### 3.3 消耗计算示例（基于流程图）

**示例1：静止，无组件激活，无伤口**
```
每Tick消耗 = 战斗体维持(1) = 1 Trion/Tick
```

**示例2：移动，变色龙激活，无伤口**
```
每Tick消耗 = 战斗体维持(1) + 变色龙维持(1) = 2 Trion/Tick
```

**示例3：静止，护盾+炸裂弹激活，左手伤口**
```
每Tick消耗 = 战斗体维持(1) + 伤口泄漏(2) = 3 Trion/Tick
```

**示例4：移动，变色龙激活，左手伤口+右手切除**
```
每Tick消耗 = 战斗体维持(1) + 变色龙维持(1) + 左手泄漏(2) + 右手泄漏(5) = 9 Trion/Tick
```

【上述示例仅代表“单位时间持续消耗”，正确。但注意示例还隐含有一次性消耗未提及】

## 第四部分：战斗体系统

### 4.1 战斗体生成流程（人类专用）

**前置条件：**
- 殖民者已装备"触发器"（特殊装备）
- 触发器上已配置组件

**生成步骤：**
1. **快照肉身**：记录健康状态、装备、生理数据
2. **生理停滞**：肉身的生理活动暂停
3. **装备移除/禁用**：肉身装备被卸下或锁定
4. **生成战斗体**：显示战斗体形态
5. **注册组件**：触发器上的组件注册到系统，计算占用值
6. **锁定配置**：战斗体状态下，组件配置无法更改

**状态变化：**
```
生成前：
- 总量：1000
- 占用：0
- 可用：1000

生成后：
- 总量：1000
- 占用：460（根据装备的组件）
- 可用：540
```

### 4.2 战斗体解除流程

#### 主动解除

**触发方式：**
- 玩家手动解除

**流程：**
1. 解除战斗体
2. 组件注销
3. **占用值返还**（关键！）
4. 恢复肉身数据（快照）
5. 生理活动恢复
6. 装备可用

**状态变化：**
```
解除前：
- 总量：1000
- 占用：460
- 累计消耗：200
- 可用：340

解除后：
- 总量：1000
- 占用：0
- 累计消耗：200
- 可用：800（占用的460返还了）
```

#### 被动解除（战斗体破裂）

**触发条件（二选一）：**
1. Trion 可用值 ≤ 0
2. Trion 供给器官被击毁

**流程：**
1. 解除战斗体
2. 组件注销
3. **占用值全部流失**（关键！不返还）
4. 恢复肉身数据（快照）
5. 生理活动恢复
6. 装备可用
7. **施加 debuff：Trion 枯竭**（惩罚）

**状态变化：**
```
解除前：
- 总量：1000
- 占用：460
- 累计消耗：540
- 可用：0（触发条件）

解除后：
- 总量：1000
- 占用：0
- 累计消耗：1000（占用的460也被消耗了）
- 可用：0
- debuff：Trion枯竭
```

### 4.3 战斗体的关键特性

**虚拟健康：**
- 战斗体受伤不会扣肉身血量
- 伤害转化为 Trion 消耗
- 伤口/断肢是"虚拟"的

**回滚机制：**
- 解除战斗体后，肉身完全恢复到快照状态
- 战斗体受的伤不会继承到肉身
- ❓ 疑问：心理状态会回滚吗？流程图提示"心理状态不会回滚"

## 第五部分：组件系统

### 5.1 组件的三种状态

**状态1：无连接（Disconnected）**
- 触发条件：
  - 战斗体未生成
  - 或挂载点被毁（如左手被切除）
- 表现：组件无法使用

**状态2：休眠（Dormant）**
- 触发条件：
  - 战斗体已生成
  - 组件未激活
- 表现：
  - 已占用 Trion
  - 不消耗 Trion
  - 不提供功能

**状态3：激活（Active）**
- 触发条件：
  - 战斗体已生成
  - 组件已激活
- 表现：
  - 提供功能
  - 可能有持续消耗（少数）

### 5.2 组件激活规则

**单槽位限制：**
- 双手各自同时只能激活**1个**组件
- 合计最多**2个**激活组件

**激活引导：**
- 激活需要引导时间（1-5 Tick）
- 引导期间组件未生效
- 引导完成后才进入激活状态

**关闭规则：**
- 关闭立即生效，不需要时间
- 同槽位激活新组件时，旧组件瞬间关闭

**切换示例（流程图）：**
```
状态：变色龙激活中
操作：激活炸裂弹+护盾（引导2单位时间）
过程：
  Tick 1: 变色龙仍激活，继续消耗
  Tick 2: 变色龙仍激活，继续消耗
  Tick 3: 引导完成 → 变色龙瞬间关闭 → 炸裂弹+护盾激活
```

### 5.3 组件类型与功能

**武器组件：**
- 近战武器：如弧月
- 远程武器：如炸裂弹（需要弹药消耗）

**辅助组件：**
- 护盾：概率性抵挡攻击
- 变色龙：隐身（持续消耗型）

**能力组件：**
- 弧月旋空：大范围攻击能力
- 需要特定条件（如输出功率≥50）

**特殊组件：**
- 紧急脱离系统：Bail Out 功能

### 5.4 部位与挂载点的关系

**人类触发器挂载点：**
- 左手槽位：装备在左手
- 右手槽位：装备在右手
- 特殊槽位：不绑定部位

**部位被毁的影响：**
- 流程图示例：右手臂被切除 → 右手槽位所有组件断连
- 左手槽位不受影响
- 特殊槽位不受影响

## 第七部分：Bail Out（紧急脱离）系统

### 7.1 触发条件

**条件1：Trion 即将耗尽**
- 流程图示例：Trion 可用值接近预留量（如 < 400）
- 自动触发

**条件2：受到致命攻击**
- 流程图示例：Trion 供给器官被击毁
- 立即触发，优先于其他逻辑

### 7.2 触发效果

**必须装备"紧急脱离系统"组件（占用400）**

**效果：**
1. 折跃传送到"传送锚"建筑
2. 解除战斗体（被动解除）
3. 恢复肉身
4. 施加 debuff：Trion 枯竭

**如果没有装备紧急脱离系统：**
- 原地解除战斗体
- 其他流程相同

### 7.3 Bail Out 的代价

**Trion 消耗：**
- 紧急脱离系统的占用值（400）被消耗
- 其他组件占用值也被消耗
- 通常导致 Trion 归零

**debuff 惩罚：**
- Trion 枯竭状态
- ❓ 疑问：具体效果是什么？减速？虚弱？持续多久？

## 我的主要理解错误汇总（已纠正）

❌ **错误1：容量与体型成正比**
- 正确：人类由天赋决定，人造物由设计决定

❌ **错误2：移动消耗 Trion**
- 正确：移动不消耗

❌ **错误3：大多数组件有持续消耗**
- 正确：只有少数组件（如变色龙）有持续消耗

❌ **错误4：四维消耗模型**
- 正确：只有战斗体维持、组件维持、伤口泄漏三种持续消耗

❌ **错误5：占用即消耗**
- 正确：占用是预留，主动解除会返还

❌ **错误6：枯竭 debuff 由 Trion 耗尽触发**
- 正确：由战斗体被迫解除触发（无论是否 Bail Out）
