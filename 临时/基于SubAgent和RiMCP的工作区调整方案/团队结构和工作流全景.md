## **3️⃣ 主Agent的核心决策树**

```mermaid
graph TD
    Input["📥 接收用户指令"]
    
    Understand["🤔 理解意图<br/>这是什么类型的请求？"]
    
    Decision{"当前应该做什么？"}
    
    Path1["状态1: 启动新的阶段"]
    Path2["状态2: 继续当前阶段"]
    Path3["状态3: 返回修改"]
    Path4["状态4: 查询信息"]
    
    Plan["📋 制定工作计划<br/>确认用户是否同意"]
    
    Activate["⚡ 激活对应Agent<br/>提供系统提示词"]
    
    Execute["🔄 Agent执行工作<br/>调用RiMCP<br/>生成输出"]
    
    Collect["📦 收集输出<br/>状态.yaml<br/>输出.md<br/>RimWorld查询/"]
    
    QCheck["✅ 质量检查<br/>输出是否符合标准"]
    
    QPass{"质量是否<br/>达标？"}
    
    Present["🎯 呈现给用户<br/>总结输出<br/>指出关键决策点"]
    
    Wait["⏸️ 等待用户反馈"]
    
    Feedback{"用户反馈<br/>是什么？"}
    
    Continue["➡️ 继续<br/>启动下一个Agent"]
    Modify["🔄 修改<br/>反馈给Agent重做"]
    Revert["↩️ 返回<br/>回到某个阶段"]
    Query["❓ 查询<br/>提供详细信息"]
    
    Input --> Understand
    Understand --> Decision
    
    Decision -->|新项目/新阶段| Path1
    Decision -->|当前阶段继续| Path2
    Decision -->|问题需要修改| Path3
    Decision -->|需要了解细节| Path4
    
    Path1 --> Plan
    Path2 --> Plan
    Path3 --> Plan
    Path4 --> Plan
    
    Plan -->|用户同意| Activate
    Plan -->|用户不同意| Understand
    
    Activate --> Execute
    Execute --> Collect
    Collect --> QCheck
    
    QCheck --> QPass
    QPass -->|否| Modify
    QPass -->|是| Present
    
    Modify -->|返回Agent| Execute
    
    Present --> Wait
    Wait --> Feedback
    
    Feedback -->|继续| Continue
    Feedback -->|修改| Modify
    Feedback -->|返回| Revert
    Feedback -->|查询| Query
    
    Query --> Wait
    Continue --> Input
    Revert --> Input
```

# 📝 **文字总结：团队结构和工作流**

## **团队组成**

```text
1个主Agent（RimWorld项目主管）
  + 7个子Agent（各司其职）
  + RiMCP工具库（知识来源）
  + 文件系统（信息交接）
  + PromptX角色（主Agent增强）
  
= 完整的模组开发系统
```

## **工作流的3个关键特点**

### **特点1：串联而非并行**

```text
Agent1 → [用户审查] → Agent2 → [用户审查] → Agent3 → ...
        ↑ 可返回修改              ↑ 可返回修改
```

-   每个Agent完成后，用户审查
    
-   用户可以继续、修改或返回
    
-   不是并行，而是线性串联
    

### **特点2：主Agent是"指挥官"**

```text
用户指令 → 主Agent理解 → 主Agent规划 → 主Agent激活Agent
                              ↓
                        主Agent监督 → 主Agent呈现 → 用户反馈
```

-   所有的激活、协调、呈现都由主Agent负责
    
-   用户和Agent之间通过主Agent桥接
    
-   主Agent保持全局视图
    

### **特点3：RiMCP是"真理来源"**

```text
所有Agent的工作都基于RiMCP验证

Agent1（低使用）→ Agent2（低使用）→ Agent3（高使用 ⭐）
                                  ↓
                            基础已奠定

Agent3（高使用）→ Agent4（中使用）→ Agent5（高使用 ⭐）
                                  ↓
                            代码已验证

Agent5（高使用）→ Agent6（高使用 ⭐ 深度追踪）
```

## **交接的5个关键时刻**

### **交接点1：User → MainAgent**

```text
用户说："我想做一个贸易系统"
主Agent理解并规划
```

### **交接点2：MainAgent → Agent1**

```text
主Agent激活头脑风暴Agent
提供用户的需求
```

### **交接点3：Agent1 → MainAgent → User**

```text
Agent1完成输出
主Agent收集、整理
呈现给用户审查
```

### **交接点4：User批准 → Agent2**

```text
用户说"继续"
主Agent激活需求分析Agent
提供Agent1的输出作为输入
```

### **交接点5：Agent → Agent（通过文件系统）**

```text
Agent3输出架构设计
Agent4读取Agent3的输出
特别是RimWorld查询的结果
基于这些进行原型验证
```
