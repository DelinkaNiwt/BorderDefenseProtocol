# Trion组件系统详细设计 v0.1

> **说明**：本文档补充了MVP方案中对"组件物品系统"的详细设计。之前的方案中确实遗漏了组件在游戏中的物品实现细节。

---

## 核心问题

**问题**：框架中的TriggerMount管理"激活状态"和"消耗值"，但组件从哪里来？如何从物品库存装备到触发器？

**解决方案**：分为三个层次实现

### 1.2 TrionComponent 基础类设计

```csharp
// Core/TrionComponent.cs
public class TrionComponent : Thing
{
    /// <summary>
    /// 组件名称（UI显示用）
    /// </summary>
    public string ComponentName { get; set; }

    /// <summary>
    /// Trion消耗值（每激活周期）
    /// </summary>
    public int TrionCost { get; set; }

    /// <summary>
    /// 组件类型（用于分类和兼容性检查）
    /// </summary>
    public ComponentType ComponentType { get; set; }

    /// <summary>
    /// 效果描述
    /// </summary>
    public string ComponentEffect { get; set; }

    /// <summary>
    /// 稀有度（0-1，用于定价和掉落概率）
    /// </summary>
    public float Rarity { get; set; }

    /// <summary>
    /// 占用的槽位数
    /// </summary>
    public int SlotSize { get; set; } = 1;

    /// <summary>
    /// 是否可与其他同类型组件共存（如两个护盾）
    /// </summary>
    public bool CanCoexistWithSameType { get; set; } = false;

    /// <summary>
    /// 是否可同时激活（如双手各一个）
    /// </summary>
    public bool CanActivateConcurrently { get; set; } = true;

    public override string GetInspectString()
    {
        var sb = new System.Text.StringBuilder();
        sb.AppendLine(base.GetInspectString());
        sb.AppendLine($"类型: {ComponentType}");
        sb.AppendLine($"消耗: {TrionCost}/激活");
        sb.AppendLine($"效果: {ComponentEffect}");
        return sb.ToString();
    }
}

public enum ComponentType
{
    Melee,      // 近战武器
    Ranged,     // 远程武器
    Auxiliary,  // 辅助
    Special     // 特殊
}
```

## 第二层：触发器和组件装备流程

### 2.1 触发器（Trigger）扩展

触发器是装备品，需要存储已装备的组件列表：

```csharp
// 需要在 ProjectTrion_Framework/Components/TriggerMount.cs 中扩展
public class TriggerMount : ThingComp
{
    /// <summary>
    /// 装备在此触发器上的所有组件
    /// </summary>
    private List<TrionComponent> _equippedComponents = new List<TrionComponent>();

    public List<TrionComponent> EquippedComponents => _equippedComponents;

    /// <summary>
    /// 向触发器添加组件
    /// </summary>
    public bool TryAddComponent(TrionComponent comp)
    {
        // 检查1：槽位是否足够
        if (!HasSlotSpace(comp.SlotSize))
        {
            Log.Warning($"TriggerMount: 槽位不足，无法装备{comp.ComponentName}");
            return false;
        }

        // 检查2：是否与现有组件冲突
        if (!CanCoexistWith(comp))
        {
            Log.Warning($"TriggerMount: {comp.ComponentName}与已有组件冲突");
            return false;
        }

        // 检查3：如果在战斗体激活状态，不允许修改
        var compTrion = parent.GetComp<CompTrion>();
        if (compTrion?.IsInCombat ?? false)
        {
            Log.Warning($"TriggerMount: 战斗体激活中，无法修改组件");
            return false;
        }

        // 装备组件
        _equippedComponents.Add(comp);
        Log.Message($"TriggerMount: 装备组件{comp.ComponentName}");
        return true;
    }

    /// <summary>
    /// 从触发器移除组件
    /// </summary>
    public bool TryRemoveComponent(TrionComponent comp)
    {
        var compTrion = parent.GetComp<CompTrion>();
        if (compTrion?.IsInCombat ?? false)
        {
            return false;  // 战斗体激活中无法移除
        }

        return _equippedComponents.Remove(comp);
    }

    /// <summary>
    /// 计算所有已装备组件的总消耗值
    /// </summary>
    public int GetTotalComponentCost()
    {
        return _equippedComponents.Sum(c => c.TrionCost);
    }

    /// <summary>
    /// 获取总槽位使用数
    /// </summary>
    public int GetUsedSlots()
    {
        return _equippedComponents.Sum(c => c.SlotSize);
    }

    private bool HasSlotSpace(int slotsNeeded)
    {
        // 槽位限制：默认4个
        const int MaxSlots = 4;
        return GetUsedSlots() + slotsNeeded <= MaxSlots;
    }

    private bool CanCoexistWith(TrionComponent newComp)
    {
        foreach (var existing in _equippedComponents)
        {
            // 检查同类型冲突
            if (existing.ComponentType == newComp.ComponentType &&
                !newComp.CanCoexistWithSameType)
            {
                return false;
            }

            // 检查特殊冲突规则（可扩展）
            if (!newComp.CanActivateConcurrently && existing.IsActive)
            {
                return false;
            }
        }

        return true;
    }

    public override void ExposeData()
    {
        base.ExposeData();
        Scribe_Collections.Look(ref _equippedComponents, "equippedComponents", LookMode.Deep);
    }
}
```

### 2.2 装备流程

```
玩家操作
  ↓
Building_TriggerConfigBench.EquipComponent(comp, trigger)
  ├─ trigger.TryAddComponent(comp)
  │  ├─ 检查槽位 ✓
  │  ├─ 检查冲突 ✓
  │  ├─ 检查战斗体状态 ✓
  │  └─ 添加到equippedComponents列表
  ├─ 更新CompTrion.Reserved
  │  └─ Reserved += comp.TrionCost（如战斗体已激活）
  ├─ 日志记录
  └─ UI刷新
```

## 第三层：组件对能量系统的影响

### 3.1 Reserved（占用值）计算

当战斗体**未激活**时：
```
Reserved = 0（没有任何占用）
```

当战斗体**激活后**装备组件时：
```
Reserved = Σ(所有已装备组件的TrionCost)

示例：
  - 弧月10 + 护盾15 = Reserved 25
  - 可用 = Capacity - Reserved - Consumed
```

当战斗体**激活时**装备新组件：
```
新 Reserved = 旧 Reserved + 新组件.TrionCost
新 Available = Capacity - 新Reserved - Consumed
```

### 3.2 组件激活与消耗

**激活状态**（IsActive）：
- 某个组件当前处于"工作"状态
- 只能同时激活2个（左手1个+右手1个）
- 激活需要"引导时间"（可选优化）

**消耗计算**（每60Tick）：
```csharp
// CompTrion.TickConsumption()
float baseMaintenance = Strategy.GetBaseMaintenance();  // 5f
float componentConsumption = 0f;

foreach (var mount in _mounts)
{
    foreach (var component in mount.EquippedComponents)
    {
        if (component.IsActive)
        {
            // 只有激活的组件才消耗
            componentConsumption += component.TrionCost;
        }
    }
}

float leakRate = GetLeakRate();
float totalConsumption = baseMaintenance + componentConsumption + leakRate;
Consume(totalConsumption);
```

### 3.3 组件类型与消耗行为

| 类型 | 激活方式 | 消耗时机 | 特点 |
|------|---------|---------|------|
| **近战武器** | 手动按键 | 持续消耗 | 近距离伤害增强 |
| **远程武器** | 手动按键 | 持续消耗 | 远距离攻击 |
| **辅助** | 手动按键 | 持续消耗 | 被动保护 |
| **特殊** | 自动触发 | 一次性或持续 | 紧急脱离 |

## 第四层：UI交互

### 4.1 触发器配置台UI

```csharp
// Building_TriggerConfigBench.cs
public override IEnumerable<Gizmo> GetGizmos()
{
    yield return new Command_Action
    {
        defaultLabel = "选择触发器",
        defaultDesc = "选择要配置的触发器",
        icon = TexUI.RotLeftTex,  // 临时图标
        action = ShowTriggerSelection
    };

    if (selectedTrigger != null)
    {
        yield return new Command_Action
        {
            defaultLabel = "添加组件",
            defaultDesc = "从库存中选择组件装备",
            icon = TexUI.RotRightTex,
            action = ShowComponentSelection
        };

        // 显示已装备组件
        foreach (var comp in selectedTrigger.GetComp<TriggerMount>().EquippedComponents)
        {
            yield return new Command_Action
            {
                defaultLabel = $"移除 {comp.ComponentName}",
                defaultDesc = $"消耗: {comp.TrionCost}",
                action = () => RemoveComponent(comp)
            };
        }
    }
}

private void ShowTriggerSelection()
{
    var options = new List<FloatMenuOption>();

    // 扫描地图上所有触发器
    foreach (var thing in parent.Map.listerThings.ThingsOfDef(ThingDef.Named("Trigger")))
    {
        var trigger = thing;
        options.Add(new FloatMenuOption(
            trigger.Label,
            () => { selectedTrigger = trigger; }
        ));
    }

    Find.WindowStack.Add(new FloatMenu(options));
}

private void ShowComponentSelection()
{
    var options = new List<FloatMenuOption>();
    var mount = selectedTrigger.GetComp<TriggerMount>();

    // 扫描地图上所有TrionComponent物品
    foreach (var comp in parent.Map.listerThings.ThingsOfDef(typeof(TrionComponent)))
    {
        var component = comp as TrionComponent;
        string label = $"{component.ComponentName} (消耗:{component.TrionCost})";

        options.Add(new FloatMenuOption(
            label,
            () => EquipComponent(component, mount)
        ));
    }

    Find.WindowStack.Add(new FloatMenu(options));
}

private void EquipComponent(TrionComponent comp, TriggerMount mount)
{
    if (mount.TryAddComponent(comp))
    {
        // 如战斗体已激活，更新Reserved
        var compTrion = selectedTrigger.GetComp<CompTrion>();
        if (compTrion?.IsInCombat ?? false)
        {
            // Reserved自动更新（在GenerateCombatBody中计算）
        }

        // 从地图移除组件物品
        comp.Destroy(DestroyMode.Vanish);
    }
}
```

## 组件系统关键约束

### 约束1：槽位管理
```
双手组件槽位: 4 + 4 = 8个
特殊组件槽位: 1个
总计: 9个槽位
```

### 约束2：激活规则
```
同一时刻最多激活:
  - 左手1个
  - 右手1个
  - 合计2个

例外：特殊组件自动触发，不占激活数
```

### 约束3：热切换规则
```
战斗体未激活: 可随意装备/移除组件
战斗体激活:   组件锁定（不能修改，只能激活/关闭）
```

### 约束4：冲突规则
```
近战武器 <-> 近战武器: 不能共存（除非特殊设计）
护盾 <-> 护盾: 可共存
特殊 <-> 其他: 互斥（不能同时激活）
```
