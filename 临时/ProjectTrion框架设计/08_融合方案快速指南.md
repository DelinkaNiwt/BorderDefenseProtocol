## 三个核心融合点

### 1️⃣ 架构骨架：采用v3.0
```
采用：CompTrion (通用Comp) + ILifecycleStrategy (多个实现)
放弃：事件总线（太复杂，改用直接方法调用）
```

**好处：**
- ✅ 支持人、兵器、建筑等全实体类型
- ✅ 避免Comp膨胀
- ✅ 符合RimWorld设计哲学

---

### 2️⃣ 能量管理：采用v2.0
```
采用：四维消耗模型
- 维持消耗 (0.5/Tick)
- 基础动作 (1-5/Tick)
- 伤口泄漏 (伤口严重程度 × 0.5/Tick)
- 激活消耗 (一次性 5-50)
```

**好处：**
- ✅ 清晰的游戏感控制
- ✅ 易于参数调优
- ✅ 完整的能量资源博弈

---

### 3️⃣ 组件激活：采用v2.0的状态机
```
采用：激活状态机（TrionMountController中）
- Inactive
- Activating (1-5 Tick引导)
- Active
- Cooling
- Destroyed
```

**好处：**
- ✅ 清晰追踪组件状态
- ✅ 易于实现"激活引导"和"无法切换"的限制
- ✅ 支持冷却机制

## 融合方案的核心类

### CompTrion (通用入口)
```csharp
public class CompTrion : ThingComp
{
    // v3.0: 通用数据
    public float CurrentTrion;
    public float MaxCapacity;

    // v2.0: 能量细节
    public float MaintenanceCost = 0.5f;
    public float LeakRate;               // 伤口泄漏

    // v3.0: 策略引擎
    private ILifecycleStrategy _strategy;
    private List<TrionMountController> _mounts;

    // v2.0: 四维消耗计算
    float CalculateTotalDrain()
    {
        return 维持 + 动作 + 伤口 + 挂载;
    }
}
```

**职责：**
- 存储Trion数据
- 计算每Tick的总消耗
- 触发Strategy回调

---

### Strategy_HumanOverlay (人类逻辑)
```csharp
public class Strategy_HumanOverlay : ILifecycleStrategy
{
    // v2.0: 人类特有逻辑
    private PawnSnapshot _snapshot;      // 快照
    private List<WoundData> _wounds;     // 伤口追踪

    // v2.0流程
    OnInitialize() → 快照肉身、显示战斗体
    OnDamageTaken() → 注册伤口、增加泄漏
    OnDepleted() → 回滚肉身、判定Bail Out
}
```

**职责：**
- 快照与恢复
- 伤口管理
- Bail Out判定

### TrionMountController (挂载点)

### ❓ 问题3：如何处理与原版Pawn的冲突？
**答：**
```csharp
// 使用Harmony Patch拦截，而不是修改原始类
[HarmonyPatch(typeof(Pawn_HealthTracker), nameof(Pawn_HealthTracker.PreApplyDamage))]
public static bool Prefix(Pawn __instance, ref DamageInfo dinfo)
{
    var comp = __instance.GetComp<CompTrion>();
    if (comp == null) return true;  // 原版流程

    // 拦截伤害到Trion系统
    return comp.Strategy.ShouldInterceptDamage();
}
```
