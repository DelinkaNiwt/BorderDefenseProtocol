### 3. IsEssentialHediff的真实设计意图

**定义**：返回true表示"这个Hediff不是Snapshot恢复逻辑应该处理的对象"

**三类Hediff的区分**：

```
┌─ 快Snapshot中的Hediff
│  └─ IsEssentialHediff = false（需要处理：移除后重新添加）
│
├─ 战斗体植入的Hediff（如Trion腺体）
│  └─ IsEssentialHediff = true（不处理：保持原状）
│
└─ 战斗伤害Hediff
   └─ IsEssentialHediff = true（不处理：由其他逻辑清理）
```

**当前代码的错误**：
- 总是返回false
- 导致所有Hediff都被移除并重新添加
- 特别是Trion腺体会丢失

## 详细问题-修复映射表

### 问题1：Trion腺体丢失 ← 由IsEssentialHediff错误引发

```
问题：RestoreToPawn中，IsEssentialHediff总是false
    导致Trion腺体被当作普通Hediff处理
    被移除但不被重新添加 → 丢失

根本原因：
    IsEssentialHediff逻辑未实现
    判断标准不清（应该区分什么？）

修复方案（阶段1）：
    1. 在CompTrion中实现IsEssentialHediff
    2. 正确识别Trion腺体 (hediff.def == TrionDefs.HediffTrionGland)
    3. 正确识别战斗新增Hediff (_snapshot != null && !_snapshot.hediffs.Contains())
    4. 在GenerateCombatBody中建立回调关联

修复后：
    ✅ Trion腺体不会被移除
    ✅ 战斗新增Hediff不会被重新添加
    ✅ 快照Hediff会被正确恢复
```

### 问题2：Hediff对象身份冲突 ← Capture/Restore的对象引用问题

```
问题：保存Hediff对象引用 → 战斗中对象被修改 → Restore时使用修改后的对象
    例：旧伤口severity=2 → 战斗中改成severity=5 → Restore时恢复severity=5

根本原因：
    Capture保存对象引用（而不是值）
    对象生命周期与原Pawn绑定
    重新AddHediff可能违反RimWorld的约束

修复方案（阶段2）：
    1. 新增HediffSnapshot struct（保存def、severity、part等值）
    2. 修改CaptureFromPawn使用值快照
    3. 修改RestoreToPawn从值快照重建新对象
    4. 修改ExposeData序列化值快照

修复后：
    ✅ 恢复时使用快照时的值，不是修改后的值
    ✅ 避免对象引用冲突
    ✅ 支持RimWorld版本升级
```

### 问题3：快照恢复逻辑不清 ← 移除/添加顺序问题

```
问题：当前RestoreToPawn的两个循环逻辑不清
    - 哪些Hediff应该被移除？
    - 哪些Hediff应该被重新添加？
    - Trion腺体在哪个环节被处理？

根本原因：
    IsEssentialHediff设计意图不清
    判断标准不统一

修复方案（阶段1）：
    1. 明确IsEssentialHediff的含义
    2. 在移除循环中：跳过IsEssentialHediff==true的Hediff
    3. 在添加循环中：仅添加IsEssentialHediff==false的Hediff

修复后：
    ✅ Trion腺体不被移除，不被重新添加（保留原状）
    ✅ 快照Hediff被移除再重新添加（回滚）
    ✅ 战斗伤害被移除不被重新添加（清除）
```

### 风险1：IsEssentialHediff判断标准不唯一

**风险描述**：不同应用可能对"哪些Hediff应该保留"有不同理解

**缓解方案**：
- 提供清晰的文档说明IsEssentialHediff的含义
- 允许应用层通过回调自定义规则
- 提供默认实现和常见扩展示例

**风险等级**：🟢 低

### 风险2：旧存档兼容性问题

**风险描述**：修改Snapshot结构可能导致旧存档加载失败

**缓解方案**：
- 添加版本检查和迁移逻辑
- 保留向后兼容的ExposeData
- 提供测试用例验证兼容性

**风险等级**：🟡 中

### 风险3：Hediff的RimWorld生命周期约束未知

**风险描述**：RimWorld可能对Hediff对象的生命周期有特殊约束，导致重新AddHediff失败

**缓解方案**：
- 使用RiMCP验证HediffSet.AddHediff/RemoveHediff的API要求
- 在单元测试中覆盖各种Hediff类型
- 在游戏中进行长期测试（跨多个战斗）

**风险等级**：🔴 高
