## 1. Trion 能量系统

### 1.1 定义

**Trion**：一种特殊能量资源，储存于生物或人造物内部，可被消耗、可自然恢复。驱动战斗体、组件、能力等所有系统功能。

### 1.2 能量四要素

| 要素       | 符号       | 性质     | 定义                                   |
| ---------- | ---------- | -------- | -------------------------------------- |
| 总容量     | Capacity   | 固定     | Trion 能量池的最大值，个体差异         |
| 占用量     | Reserved   | 可逆     | 战斗体状态下组件锁定的容量             |
| 已消耗量   | Consumed   | 不可逆   | 累计消耗的 Trion，永久减少              |
| 可用量     | Available  | 派生     | 当前可实际使用的能量                   |

**数量关系**：
```
Available = Capacity - Reserved - Consumed
```

**关键规则**：
- 总容量：个体固定属性，当前设计不可提升
- 占用量：战斗体生成时计算，主动解除时返还，被动解除时流失（转为已消耗量）
- 已消耗量：所有消耗行为（一次性 + 持续性）的累计值，不可逆
- 可用量：实时计算的派生值，≤ 0 时触发战斗体破裂

### 1.3 容量决定机制

#### 人类殖民者

- **决定因素**：隐藏属性"Trion 天赋"
- **天赋等级**：S / A / B / C / D / E
- **个体表现**：固定数值
- **后天提升**：当前设计不可提升（保证稀缺性）
- **接口预留**：保留提升机制接口

#### 人造物（Trion 兵、建筑）

- **决定因素**：设计者指定
- **设计原则**：通常与体积、功能需求正相关，但无强制因果关系
- **多样性**：依需求分型号，无统一标准

## 2. 组件系统

### 2.1 组件定义

**组件（Component）**：可装备到触发器的功能模块。

### 2.2 组件属性

| 属性         | 说明                                     | 必须 |
| ------------ | ---------------------------------------- | ---- |
| 占用值       | 战斗体状态下锁定的 Trion 容量            | 是   |
| 激活费用     | 激活组件时的一次性 Trion 消耗            | 是   |
| 持续消耗     | 激活后每单位时间的额外消耗               | 否   |
| 使用费用     | 使用组件功能时的消耗（如射击、释放能力） | 否   |
| 功能         | 组件提供的具体效果                       | 是   |

**注意**：激活费用 ≠ 使用费用。例如能力型组件激活费用低，但释放能力时消耗高。

### 2.3 组件状态机

| 状态       | 英文         | 条件                           | 占用 Trion | 提供功能 | 额外消耗 |
| ---------- | ------------ | ------------------------------ | ---------- | -------- | -------- |
| 未连接     | Disconnected | 战斗体未生成 / 挂载点被毁      | 否         | 否       | 否       |
| 休眠       | Dormant      | 战斗体已生成，组件未激活       | 是         | 否       | 否       |
| 激活       | Active       | 战斗体已生成，组件已激活       | 是         | 是       | 部分组件 |

**状态转换规则**：
- 未连接 → 休眠：生成战斗体时自动转换
- 休眠 → 激活：支付激活费用 + 引导时间（1-5 Tick）
- 激活 → 休眠：瞬间关闭，无延迟
- 休眠/激活 → 未连接：解除战斗体 / 挂载点被毁

### 2.4 装备规则

#### 挂载点类型（人类触发器）

| 挂载点   | 装备数量 | 同时激活数量 | 绑定部位 |
| -------- | -------- | ------------ | -------- |
| 左手槽位 | 多个     | 1            | 左手     |
| 右手槽位 | 多个     | 1            | 右手     |
| 特殊槽位 | 多个     | 不限         | 无       |

#### 激活切换机制

- 同槽位内切换激活：新组件进入引导 → 引导完成后 → 旧组件瞬间关闭 → 新组件激活
- 引导期间旧组件仍产生消耗

### 2.5 组件示例（参考）

| 组件名       | 类型     | 占用值 | 激活费用 | 持续消耗   | 使用费用        | 功能                       |
| ------------ | -------- | ------ | -------- | ---------- | --------------- | -------------------------- |
| 弧月         | 近战武器 | 10     | 5        | 0          | 0               | 近战攻击                   |
| 炸裂弹       | 远程武器 | 10     | 5        | 0          | 1/发            | 射击                       |
| 护盾         | 辅助     | 10     | 5        | 0          | 5/次抵挡        | 概率抵挡攻击，按比例减伤   |
| 变色龙       | 辅助     | 10     | 5        | 1/单位时间 | 0               | 隐身                       |
| 弧月旋空     | 能力     | 10     | 5        | 0          | 50/次释放       | 大范围攻击，需输出功率≥50  |
| 紧急脱离系统 | 特殊     | 400    | 0        | 0          | 0               | 启用 Bail Out 功能         |

**说明**：所有数值仅供理解设定，非最终数值。

## 3. 战斗体系统

### 3.1 定义

**战斗体（Combat Body）**：由 Trion 构成的虚拟身体形态，完全覆盖并替代肉身。

**适用对象**：人类殖民者

### 3.2 生成机制

#### 前提条件

- 装备触发器
- 不要求配置组件（无组件也可生成）

#### 生成流程

| 步骤 | 操作                       | 说明                           |
| ---- | -------------------------- | ------------------------------ |
| 1    | 快照肉身状态               | 记录健康、装备、物品           |
| 2    | 冻结生理活动               | 生理需求停止变化               |
| 3    | 禁用肉身装备               | 表现形式待定                   |
| 4    | 生成战斗体形态             | 表现形式待定                   |
| 5    | 组件注册                   | 未连接 → 休眠                  |
| 6    | 计算占用量                 | 从可用 Trion 扣除              |
| 7    | 锁定配置                   | 战斗体状态下不可更改组件配置   |

#### 能量变化

```
生成前：Capacity 1000, Reserved 0, Available 1000
生成后：Capacity 1000, Reserved 460, Available 540
```

### 3.3 战斗体特性

#### 虚拟伤害系统

- 伤害转化为 Trion 消耗，不扣除肉身血量
- 战斗体伤势不继承到肉身
- 解除战斗体后肉身恢复到快照状态

#### 部位毁坏后果

| 部位类型 | 后果                           |
| -------- | ------------------------------ |
| 手臂     | 该手挂载点所有组件断连，增加泄漏速率 |
| 单腿     | 移速大幅下降，增加泄漏速率       |
| 双腿     | 失去移动能力，增加泄漏速率       |

#### 心理状态继承

- 战斗中产生的心情、压力、精神状态完整保留
- 战斗体解除不回滚心理状态

### 3.4 解除机制

#### 主动解除

**触发方式**：玩家手动操作

**流程**：
1. 战斗体消失
2. 组件注销（休眠/激活 → 未连接）
3. 占用量全部返还
4. 快照数据恢复到肉身
5. 生理活动恢复

**惩罚**：无

#### 被动解除（战斗体破裂）

**触发条件（满足其一）**：
- Trion 可用量 ≤ 0
- Trion 供给器官被摧毁
- 玩家主动触发紧急脱离系统（如有）

**流程**：
1. 战斗体瞬间崩溃
2. 组件注销
3. 占用量全部流失（永久消耗）
4. 快照数据恢复到肉身
5. 生理活动恢复
6. 施加 debuff："Trion 枯竭"

**代价**：占用量流失 + debuff

### 3.5 快照机制

#### 快照内容

**保存**：
- 健康数据（Hediff）：疾病、伤口、义肢、植入物
- 身体部位完整性
- 穿戴服装（Apparel）
- 持有武器（Equipment）
- 携带物品（Inventory）

**不保存**：
- 技能等级和经验值
- 心理状态和心情
- 社交关系

#### 回滚规则

- 仅恢复快照保存的内容
- 战斗中的心理压力完整保留

## 4. Trion 消耗规则

### 4.1 消耗类型分类

| 类型         | 频率               | 示例                           |
| ------------ | ------------------ | ------------------------------ |
| 一次性消耗   | 事件触发           | 激活组件、射击、受伤、释放能力 |
| 持续性消耗   | 每单位时间（60 Tick）| 战斗体维持、组件维持、伤口泄漏 |

### 4.2 一次性消耗场景

| 场景         | 消耗量                   | 说明                           |
| ------------ | ------------------------ | ------------------------------ |
| 激活组件     | 组件激活费用             | 每次激活时支付                 |
| 射击         | 组件弹药费用             | 按发计费                       |
| 护盾抵挡     | 护盾抵挡费用             | 成功抵挡时支付                 |
| 释放能力     | 能力使用费用             | 每次释放时支付                 |
| 受到伤害     | 伤害值（1:1转化）        | 同时注册伤口，增加泄漏速率     |
| 占用流失     | 全部占用量               | 仅在战斗体被动破裂时发生       |

#### 护盾机制详细

**判定流程**：
1. 受到攻击
2. 概率判定是否抵挡成功
3. 若成功：按比例减少伤害
4. 扣除护盾抵挡费用
5. 不注册伤口，不产生泄漏

**示例**：
- 原始伤害：10 Trion
- 护盾减伤比例：50%
- 实际消耗：5 Trion
- 伤口：无

### 4.3 持续性消耗场景

**性能约束**：不可每 Tick 计算，统一每 60 Tick 计算一次，所有持续消耗项累加求和。

| 场景             | 消耗量/单位时间 | 条件               |
| ---------------- | --------------- | ------------------ |
| 战斗体维持       | 固定值（如 1）  | 战斗体存在         |
| 组件功效维持     | 视组件而定      | 仅少数组件（如变色龙 +1） |
| 伤口泄漏         | 伤势严重度相关  | 每个伤口独立计算并累加   |

#### 泄漏速率示例（参考）

| 伤势类型 | 泄漏速率/单位时间 |
| -------- | ----------------- |
| 轻伤     | +2                |
| 断肢     | +5                |

**规则**：
- 伤口不自愈
- 泄漏速率不自然降低
- 仅通过解除战斗体清除伤口

#### 持续消耗计算示例

假设单位时间 = 60 Tick：

| 场景                                   | 计算式                | 总消耗/单位时间 |
| -------------------------------------- | --------------------- | --------------- |
| 静止，无组件激活，无伤口               | 1                     | 1               |
| 移动，变色龙激活，无伤口               | 1 + 1                 | 2               |
| 静止，护盾和炸裂弹激活，左手伤口       | 1 + 2                 | 3               |
| 移动，变色龙激活，左手伤口 + 右手断肢  | 1 + 1 + 2 + 5         | 9               |

### 4.4 不消耗 Trion 的行为

| 行为       | 说明                                 |
| ---------- | ------------------------------------ |
| 移动       | 不消耗（除非特殊组件产生持续消耗）   |
| 近战攻击   | 不消耗                               |

## 5. Bail Out 紧急脱离系统

### 5.1 定义

**Bail Out**：紧急脱离机制，允许濒死使用者瞬间传送到安全位置。

### 5.2 前提条件

- 必须装备"紧急脱离系统"组件（占用值示例：400）
- 未装备时无法触发 Bail Out，战斗体破裂时原地解除

### 5.3 触发方式

| 方式     | 类型 | 条件                   | 优先级 |
| -------- | ---- | ---------------------- | ------ |
| 方式一   | 自动 | Trion 可用量 ≤ 0       | 普通   |
| 方式二   | 自动 | Trion 供给器官被摧毁   | 最高   |
| 方式三   | 手动 | 玩家主动触发（UI 按钮）| 最高   |

### 5.4 效果

- 瞬间传送到"传送锚"建筑位置
- 战斗体立即解除（被动解除）
- 肉身恢复到快照状态
- 施加 debuff："Trion 枯竭"

### 5.5 代价

- 紧急脱离系统占用量（示例：400）永久流失
- 其他所有组件占用量永久流失
- 通常导致 Trion 完全归零

**本质**：用占用的 Trion 换取生命安全。

## 6. 输出功率系统

### 6.1 定义

**输出功率（Output Power）**：使用者操控 Trion 的质量指标，既是数值也可按区间划分等级。

**示例**：输出功率 30 → C 级，输出功率 50 → B 级

### 6.2 决定因素

| 因素         | 影响程度 | 说明                           |
| ------------ | -------- | ------------------------------ |
| Trion 天赋   | 主要     | 天赋越高，功率越高             |
| 小人特性     | 辅助     | 模组新增特性                   |
| 装备加成     | 临时     | 稀有装备临时提升（未来计划）   |

### 6.3 作用

#### 限制能力释放

- 高阶能力需要最低输出功率
- 不满足条件时能力按钮禁用

**示例**：弧月旋空需要输出功率 ≥ 50

#### 影响组件表现

| 影响对象             | 表现                   |
| -------------------- | ---------------------- |
| 武器伤害             | 伤害数值               |
| 远程武器射程         | 最大射程               |
| AOE 能力影响范围     | 作用半径               |
| 护盾减伤比例         | 减伤百分比             |
| 特殊效果持续时间     | 持续 Tick 数           |

## 7. Trion 恢复机制

### 7.1 恢复途径

#### 人类殖民者

- **当前设计**：仅自然恢复
- **限制原因**：保证 Trion 作为战略资源的稀缺性
- **接口预留**：物品/药物快速恢复机制

#### 人造物（建筑）

- **近期计划**：引入重要建筑，包含存储 Trion 功能
- **恢复方式**：献祭物品恢复 Trion
- **实施阶段**：当前阶段暂不实现

### 7.2 影响恢复速率的因素

| 因素           | 影响规则                                   | 接口预留 |
| -------------- | ------------------------------------------ | -------- |
| 饱食度         | 饥饿时不恢复（硬性规则），其余情况待定     | 是       |
| 小人特性       | 模组新增特性直接影响基础恢复速率数值       | 否       |
| 休养舱         | 加速"Trion 枯竭"debuff 消退，不提升恢复速率 | 否       |
| 恢复增幅建筑   | 在附近时提升恢复速率（待设计）             | 是       |

### 7.3 参数配置

**待定参数**：
- 基础恢复速率
- 各因素影响倍数
- 满恢复所需时间

**调整策略**：平衡性测试中调优

## 8. 实体类型差异

### 8.1 人类殖民者

**功能集**：
- 战斗体生成与解除：✓
- 快照与回滚机制：✓
- Bail Out 系统：✓
- 虚拟伤害系统：✓

**Trion 耗尽后果**：战斗体破裂 → 回滚肉身 → 施加 debuff → 不死亡

### 8.2 Trion 兵器

**功能集**：
- 战斗体生成与解除：N/A（无肉身概念）
- 快照与回滚机制：✗
- Bail Out 系统：✗
- 虚拟伤害系统：待定

**Trion 耗尽后果**：直接销毁（爆炸）

**设计状态**：方案未最终确定，需预留多种方案接口

### 8.3 Trion 建筑

**特点**：
- 有传统建材结构作为"肉身"
- Trion 使用体现在弹药类别和充能装填
- 受伤走游戏原版逻辑，不泄漏 Trion

**Trion 耗尽后果**：停机 → 不销毁 → 恢复后可重启

**重启方式**：待定（手动/自动）

**原则**：文档中所有数值参数仅供理解设定，非最终数值，需实际测试调优。

| 功能                       | 预留状态 | 优先级 |
