## 1️⃣ Harmony Patch 规范

### 1.1 Patch 文件组织

**文件命名规则**：
```
Patch_[TargetClass]_[TargetMethod].cs
```

示例：
- `Patch_VerbShoot_TryStartCastOn.cs`
- `Patch_Pawn_TicksPerMove.cs`
- `Patch_Building_Tick.cs`

**文件路径**：所有Patch放在 `Source/Aldon/Patches/` 目录

### 1.2 Patch 类结构模板

```csharp
using HarmonyLib;
using Verse;
using RimWorld;
using System.Collections.Generic;  // 如果需要

namespace Aldon.Energy.Patches
{
    /// <summary>
    /// [功能说明]
    /// 钩子点: [TargetClass].[TargetMethod]
    /// 目的: [为什么需要这个Patch]
    /// 风险: [可能的兼容性问题]
    /// </summary>
    [HarmonyPatch(typeof([TargetClass]), nameof([TargetClass].[TargetMethod]))]
    public class [PatchClassName]
    {
        // Prefix: 在原方法前执行，可以修改参数或阻止原方法执行
        static bool Prefix([MethodParameters])
        {
            // 实现逻辑
            return true;  // true = 继续执行原方法, false = 阻止执行
        }

        // Postfix: 在原方法后执行，可以修改返回值
        static void Postfix([MethodParameters])
        {
            // 实现逻辑
        }

        // Transpiler: 修改IL代码（高级，尽量避免）
        static IEnumerable<CodeInstruction> Transpiler(IEnumerable<CodeInstruction> instructions)
        {
            // IL修改代码
            yield return /* ... */;
        }
    }
}
```

### 1.3 Patch 最佳实践

**原则1：优先使用 Prefix，尽量避免 Transpiler**
```csharp
// ✅ 好的做法：Prefix
[HarmonyPatch(typeof(Verb_Shoot), nameof(Verb_Shoot.TryStartCastOn))]
static class Patch_VerbShoot_Energy
{
    static void Prefix(Verb_Shoot __instance)
    {
        var pawn = __instance.caster as Pawn;
        var energyComp = pawn?.GetComp<AldonEnergyComp>();
        energyComp?.TryConsume(10);
    }
}

// ❌ 避免：复杂的Transpiler
static IEnumerable<CodeInstruction> Transpiler(...)
{
    // IL修改难以维护
}
```

**原则2：防守式编程，检查所有可能的null**
```csharp
// ✅ 好的做法
static void Prefix(Verb_Shoot __instance)
{
    Pawn pawn = __instance?.caster as Pawn;
    if (pawn == null) return;  // 及时返回

    AldonEnergyComp comp = pawn.GetComp<AldonEnergyComp>();
    if (comp == null) return;  // 没有能量组件就返回

    comp.TryConsume(10);
}

// ❌ 避免：假设数据存在
static void Prefix(Verb_Shoot __instance)
{
    Pawn pawn = (Pawn)__instance.caster;  // 如果不是Pawn会崩溃
    pawn.GetComp<AldonEnergyComp>().TryConsume(10);  // 如果没有组件会崩溃
}
```

**原则3：在模组加载时初始化 Harmony**
```csharp
// AldonMod.cs（模组入口点）
public class AldonMod : Mod
{
    public AldonMod(ModContentPack content) : base(content)
    {
        // 初始化Harmony
        var harmony = new Harmony("aldon.energy.mvp");
        harmony.PatchAll();

        Log.Message("[Aldon] Harmony patches已加载");
    }
}
```

**原则4：标记高风险Patch**
```csharp
/// <summary>
/// ⚠️ 高风险Patch: 修改游戏的核心战斗系统
/// 兼容性问题: 可能与其他修改Verb_Shoot的模组冲突
/// 版本依赖: RimWorld 1.4+ (Verb系统改动)
/// </summary>
[HarmonyPatch(typeof(Verb_Shoot), nameof(Verb_Shoot.TryStartCastOn))]
public class Patch_VerbShoot_Energy { ... }
```

## 2️⃣ C# 命名规范

### 2.1 命名约定

| 类型 | 规则 | 示例 |
|------|------|------|
| **类 (class)** | PascalCase，前缀根据分类 | `AldonEnergyComp`, `Building_AldonDriver` |
| **接口 (interface)** | PascalCase，前缀 `I` | `IAldonSignal` |
| **方法 (method)** | PascalCase | `AddEnergy()`, `TryConsume()` |
| **属性 (property)** | PascalCase | `AvailableEnergy`, `IsActive` |
| **私有字段 (private field)** | camelCase，前缀 `_` | `_usedEnergy`, `_lockDict` |
| **常量 (const)** | UPPER_CASE | `CAPACITY`, `RECOVERY_RATE_PER_SECOND` |
| **枚举 (enum)** | PascalCase | `AldonPermissionLevel` |
| **枚举值 (enum value)** | PascalCase | `Authorized`, `Unauthorized` |

### 2.2 前缀使用

**模块前缀** (区分不同功能)

| 前缀 | 使用范围 | 示例 |
|------|---------|------|
| `Aldon*` | 所有关键类 | `AldonEnergyComp`, `AldonSignalManager` |
| `Building_*` | 建筑类 | `Building_AldonDriver` |
| `Patch_*` | Harmony Patch | `Patch_VerbShoot_Energy` |
| `*Comp` | ThingComp组件 | `AldonEnergyComp`, `AldonPermissionComp` |
| `*Manager` | 管理器类 | `AldonSignalManager` |

### 2.3 避免的命名

```csharp
// ❌ 避免：缩写和单字母
class EC { }  // 不清楚
void Calc() { }  // 是什么计算？
int x, y, z;  // 没有含义

// ✅ 使用清晰的全称
class AldonEnergyComp { }
void CalculateAvailableEnergy() { }
int usedEnergy, lockedEnergy, currentCapacity;
```

## 3️⃣ 代码结构规范

### 3.1 组件类的标准结构

```csharp
public class AldonEnergyComp : ThingComp
{
    // ===== 常量区域 =====
    // 所有 const 定义放在最前面
    public const int CAPACITY = 1000;
    public const int RECOVERY_RATE_PER_SECOND = 2;

    // ===== 状态数据区域 =====
    // 所有私有字段在常量之后
    private int usedEnergy = 0;
    private int lockedEnergy = 0;
    private Dictionary<string, int> locks = new Dictionary<string, int>();

    // ===== Tick驱动区域 =====
    private int tickCounter = 0;
    private const int TICKS_PER_RECOVERY = 30;

    // ===== 计算属性区域 =====
    // 只读属性放在数据之后，计算简单且高频访问
    public int Available => CAPACITY - lockedEnergy - usedEnergy;
    public float RecoveryPercent => (float)(CAPACITY - usedEnergy) / CAPACITY;

    // ===== 生命周期方法区域 =====
    // Verse/RimWorld框架回调放在这里
    public override void CompTick() { ... }
    public override void PostExposeData() { ... }

    // ===== 核心操作区域 =====
    // 主要业务逻辑方法
    public void AddEnergy(int amount) { ... }
    public bool TryConsume(int amount) { ... }
    public string RegisterLock(int amount, string source) { ... }
    public bool ReleaseLock(string lockID) { ... }

    // ===== 工具方法区域 =====
    private void SomeHelper() { ... }
}
```

### 3.2 文件大小限制

| 类型 | 最大代码行数 | 说明 |
|------|------------|------|
| **单个方法** | 50行 | 超过则考虑拆分 |
| **单个类** | 300行 | 超过则考虑拆分为多个类 |
| **单个文件** | 500行 | 文件太大不利于维护 |

如果一个类超过300行，应该：
1. 提取子类或分解职能
2. 或将逻辑分散到多个helper方法

## 4️⃣ 日志输出规范

### 4.1 日志级别使用

```csharp
// ⚠️ ERROR - 严重错误，影响功能
Log.Error("[Aldon] 严重错误: 无法初始化...");

// ⚠️ WARNING - 警告，可能有问题但不严重
Log.Warning("[Aldon] 警告: 组件未找到");

// ℹ️ MESSAGE - 正常流程信息
Log.Message("[Aldon] 能量恢复已启动");

// 🔍 DEBUG - 调试信息（仅在开发时）
if (Prefs.DevMode)
{
    Log.Message("[Aldon] DEBUG: Available=" + energyComp.Available);
}
```

### 4.2 日志格式规范

```csharp
// ✅ 标准格式: [模块名] 动作: 具体信息
Log.Message("[Aldon] 赋予权限: John 已获得 Lv1 权限");
Log.Message("[Aldon] 能量消耗: -10点, 剩余=" + comp.Available);
Log.Message("[Aldon] 驱动塔已激活 - 全球能量恢复开始");

// ❌ 避免过于冗长或含混
Log.Message("error");  // 太简短
Log.Message("Something happened to the energy system and it might be broken");  // 过于冗长
```

### 4.3 日志位置规范

| 位置 | 何时输出 | 示例 |
|------|---------|------|
| **初始化** | 模组加载、组件初始化 | `[Aldon] 能量系统已初始化` |
| **关键操作成功** | AddEnergy、TryConsume成功 | `[Aldon] 能量消耗: -10` |
| **关键操作失败** | TryConsume失败（权限/能量不足） | `Messages.Message("能量不足")` |
| **状态改变** | 信号激活/停用、权限赋予 | `[Aldon] 驱动塔已激活` |
| **异常情况** | null检查失败、非预期路径 | `Log.Warning("[Aldon] 异常: ...")` |

**重要**：不要在CompTick()内每帧输出日志，会刷屏！如需调试，用`if (Prefs.DevMode) { Log.Message(...); }`

## 5️⃣ 性能与内存考虑

### 5.1 Tick性能规范

**CompTick() 中的操作必须轻量化**：

```csharp
// ✅ 好的做法：轻量计算
public override void CompTick()
{
    if (usedEnergy <= 0) return;  // 快速检查，能量满时直接返回

    if (!AldonSignalManager.Instance.IsActive) return;  // 信号关闭时快速返回

    tickCounter++;
    if (tickCounter >= TICKS_PER_RECOVERY)
    {
        AddEnergy(1);  // 简单加法运算
        tickCounter = 0;
    }
}

// ❌ 避免：复杂计算或GC分配
public override void CompTick()
{
    // 每帧都创建新列表（GC压力）
    var allEnemies = Find.CurrentMap.mapPawns.AllPawnsSpawned.Where(p => p.HostileTo(parent)).ToList();

    // 每帧都进行复杂计算
    float complexValue = CalculateEnergyDecay();  // 复杂算法

    // 每帧都字符串拼接（GC分配）
    Log.Message("Current energy: " + usedEnergy);
}
```

### 5.2 内存分配规范

```csharp
// ✅ 好的做法：初始化一次，重用
private Dictionary<string, int> locks = new Dictionary<string, int>();  // 只初始化一次
private List<System.Action> callbacks = new List<System.Action>();      // 只初始化一次

// ❌ 避免：频繁分配
foreach (Pawn pawn in Find.CurrentMap.mapPawns.AllPawnsSpawned)
{
    var comp = pawn.GetComp<AldonEnergyComp>();
    var temp = new Dictionary<string, int>();  // 每次循环都分配！
}

// ❌ 避免：LINQ导致的GC
var activeComps = allPawns
    .Select(p => p.GetComp<AldonEnergyComp>())  // 创建��间数组
    .Where(c => c != null)                        // 创建中间数组
    .ToList();                                     // 创建最终列表（GC）

// ✅ 好的做法：手动循环，无GC
foreach (Pawn pawn in allPawns)
{
    var comp = pawn.GetComp<AldonEnergyComp>();
    if (comp != null)
    {
        // 处理
    }
}
```

### 5.3 避免的高GC操作

| 操作 | 为什么避免 | 替代方案 |
|------|-----------|---------|
| `string + string` | 每次都分配新字符串 | 用 `StringBuilder` 或 `$"..."` 格式化 |
| `.ToList()`, `.ToArray()` | 创建新集合 | 用 `foreach` 直接迭代 |
| 匿名委托捕获变量 | 创建临时对象 | 用具名方法或结构体 |
| `new` 大数组 | 内存峰值高 | 复用已有数组或使用pool |

## 6️⃣ 注释规范

### 6.1 XML文档注释（XML Doc Comments）

所有公开的类、方法、属性都应该有XML文档注释：

```csharp
/// <summary>
/// 尝试消耗能量。检查权限、信号、容量。
/// </summary>
/// <param name="amount">要消耗的能量值</param>
/// <returns>true = 消耗成功, false = 权限不足或能量不足</returns>
public bool TryConsume(int amount)
{
    // ...
}
```

### 6.2 内联注释规范

只在复杂逻辑处添加注释，不要过度注释：

```csharp
// ✅ 好的注释：解释"为什么"
// 检查信号而不是直接恢复，因为驱动塔可能被关闭
if (!AldonSignalManager.Instance.IsActive)
{
    return;
}

// ❌ 过度注释："为什么"显而易见
// 如果已消耗的能量> 0
if (usedEnergy > 0)
{
    // ...
}
```

### 6.3 风险标记

在可能出问题的地方添加特殊标记：

```csharp
// ⚠️ 注意: 这里假设parent一定是Pawn，如果不是会崩溃
Pawn pawn = parent as Pawn;
pawn.healthState = PawnHealthState.Healthy;  // 如果pawn为null会崩溃

// 🔴 已知问题: 当装备被直接删除时，锁定项不会释放
public string RegisterLock(int amount, string source) { ... }

// 🔷 需要改进: 目前没有持久化locks字典，存档读取后会丢失
private Dictionary<string, int> locks = new Dictionary<string, int>();
```
